# 4・1・6 機械システムの安全技術

> **章**: 第4章 現代の機械工学・機械技術と社会の関係
> **節**: 4・1 社会技術・安全技術

日本工業規格JIS Z 8115:2000「信頼性用語」によると、信頼性(reliability)とは、アイテムが与えられた条件のもとで、与えられた期間、要求機能を遂行できる能力と定義されている。このアイテム(対象)には人工物、コンピュータのソフトウェア、人間の行動までが含まれる。そして安全性(safety)とは、機能を果たせなくなった人工物に起因する危険から、人間の生命と財産を保護するコンセプトである。したがって、安全性保証の技術と管理の手法は、機械システムの機能を果たさせる信頼性保証の技術と管理の手法とほぼ共通している。そこで以下に、機械システムの安全性を、故障の発生を予防する信頼性技術の問題として論じることにする。信頼性の対象は、故障によって経済的損失が発生するおそれのある社会や事業者や個人が所有する工場、設備、構造物、車両、航空機、機器、家庭電器、コンピュータのハードウェアとソフトウェアなどの人工物である。そして最近の大規模機械システムでは、その構成要素になる材料、部品、メカニズム、ソフトウェア、人間の操作などのうち、どれか一つの要素の一部分に不具合が発生して故障すれば、システム全体の機能が停止する事故になり、さらに大事故から危険が発生することにもなる。またシステムの信頼性は、対象についての企画-設計-生産-試験-検査-稼働-保全-廃棄の全ライフサイクルにおける各プロセスでの検討項目である。実際の機械に信頼性が作り込まれるのは、おもに設計段階での構造と生産段階での品質である。信頼性設計の基本思想は、システムの異常現象を故障にまで発展させず、さらに故障から事故の発生を防止するメカニズムにすることである。そのためには、次のことが機械システムの設計に必要である。(1) 劣化しにくい材料の選定:稼働環境と使用条件において、機能劣化を起こさない材料を選定する。(2) 部品点数の最小化:機械全体の信頼度は各部品の信頼度の積になるので、機械の構成部品点数をできるだけ少なくする。(3) フールプルーフ設計:操作者に不適切な行為や過失があっても、故障の発生を不可能にするメカニズムにする。(4) 冗長設計:故障が発生しても、一つ以上の予備の手段をあらかじめ準備した多重系にし、機能をただちに回復させる。(5) フェイルセイフ設計:故障が発生しても、つねに安全側に機能するようなメカニズムにする。(6) 伝染防止設計:ある部分に故障が発生しても、その故障が他の部分へ伝染しないようなメカニズムにする。(7) 保全性設計:異常や故障をいち早く検出・診断し、機能を修復しやすい構造にする。(8) リサイクル設計:機械が寿命になり廃棄するとき、分解と材料の分別をしやすいメカニズムにする。(9) 環境試験:実働状態と同一の環境と条件で試作品および完成品の試験を行い、機能と耐久性の信頼度をあらかじめ確認する。(10) 設計審査制度:設計の各段階で、経験豊富な技術者や管理者が専門的立場から設計の可否を評価する制度と組織にする。(11) 故障の再発防止制度:類似のシステムについて、過去の故障と失敗の事例を収集し、原因を解析することで再発予防を学習し、類似事故の発生を防ぐ制度と組織にする。このように、信頼性設計の技術体系はほぼ確立されている。さらに、機械システムの稼働状態での異常を検出するモニタシステム、異常の原因を科学的に解明する故障物理学、またシステム操作者の認知心理学を採り入れ、人間が過ちを犯すことを前提にした人間信頼性工学の研究も進んでいる。こうして設計された機械システムの機能を保証する品質は、生産段階において機械の形として作り込まれる。その手法は品質管理と品質保証(quality assurance)の技術として体系化されている。その技術思想は、検査によって不良品を除去するのではなく、生産工程の中で初めから不良品を作らない管理と保証のための技術活動である。信頼性技術の実例として、例えば大規模交通システムである日本の新幹線がある。東海道新幹線は、1964年の営業運転開始から約40年後の今日まで、人身事故を1件も起こしていない。その原因は安全性を優先させた設計コンセプトにある。まず大量輸送・高速鉄道としてのシステム設計を行い、過去の事故統計に学んで踏切を全廃した。システム工学に基づいた列車集中制御装置(CTC)、フェイルセイフ設計の自動列車制御装置(ATC)、軌道環境の予防保全データを計測する軌道検測車の定期運行、夜間毎日の軌道環境の保全作業、車両の定期点検と修理の実施などもある。航空ジェットエンジン用転がり軸受だけに採用されていた長寿命で高信頼度の真空溶解鋼を、開業当時から車軸用転がり軸受の材料にまで使うなど、可能な限りの信頼性設計を徹底させてきた。また旅客用航空機の安全運行も同じで、徹底した信頼性設計と事故に学ぶ再発防止技術の制度化の結果、最近の事故率は1960年代の10分の1以下に低下している。一般に、人工物の商品が産業化され、社会に流通するまでのプロセスでは、有用性と信頼性の両面で、市場ユーザの厳しい選択と淘汰を受ける。そのため、人工物の構想・企画、設計、生産・施工、稼働・運用および保全・修理の各プロセスの現場では、徹底した信頼性の保証活動が地道に行われている。そこで、このような文明系環境において確立された信頼性に関する技術思想を、大量の人工物や人工生物が介在する系に変わった生物系および物質系環境にも適用することが望まれる。それは、うっかり行動や異常現象を過失や故障に発展させず、さらに事故にまで拡大させないという技術思想に基づく、生物系および物質系環境における信頼性技術の確立である。しかし実際には、人知を超えた思わぬ失敗から事故が発生するのがつねである。信頼性技術では、その失敗事例に学び、再発防止を地道に実行することが特に重要である。そのためには、技術者個人の倫理面から、さらに社会の制度や組織としても、システムの異常によって故障が起こり、事故に至ったプロセスについて完全な情報を開示し公開することが、絶対に必要な条件になるのである。
