# 1・1・2 高速化技術

> **章**: 第1章 情報システム
> **節**: 1・1 計算機ハードウェア

計算機の高速化技術としては，まず，メモリへのアクセスを速く
することが挙げられる．多くの計算機ではメインメモリ（main
memory）として，大容量化が容易なDRAM（dynamic random
access memory）が用いられている．DRAMは，電荷を蓄える回
路を記憶素子として集積したもので，単純で集積度を上げるのが容
易な反面，一定時間ごとに記憶保持のための信号を送る必要がある
など，メモリアクセス速度が比較的遅いという欠点がある．それに
対して，トランジスタを組合わせたフリップフロップ回路を記憶素
子として用いたSRAM（static random access memory）は，小
容量であるが高速なアクセスが可能である．そこで，DRAMのメ
インメモリとCPUの間にSRAMをおき，そこに頻繁にアクセス
するデータや命令を格納しておく仕組みがキャッシュメモリ
（cache memory）である．まず，CPUはアクセスの早いキャッシ
ュメモリを調べ，そこにデータがある場合はそのデータを用いる．
もしも，キャッシュメモリにない場合にはメインメモリにアクセス
し，データを取出すと同時にキャッシュメモリに格納しておく．そ
のデータを再度アクセスするときには，キャッシュメモリに対して
アクセスすることで高速にデータにアクセスすることができるよう
になる．プログラムは多くの場合，限られたデータにアクセスする
ことが多いので，プログラムの実行が高速化されることになる．同
じ動作はメモリ上の命令に対しても行われる．特に，命令を繰返す
ループの場合には同じ命令が用いられるためキャッシュ上にあるこ
とが多くなり，高速化される．
また，集積回路ではチップの外にアクセスするよりも，チップの
中で動作するほうが高速にアクセスできる．以前のCPUでは，集
積度に制限があったため，キャッシュメモリはチップの外に置かれ
ていたが，集積度が向上するにつれて，キャッシュメモリはチップ
の中に置かれるようになっている．さらに，一般的に，必要なデー
タを選ぶ回路はメモリが大きくなると複雑になるためにアクセスに
時間がかかるようになる．集積度が向上し，キャッシュメモリの容
量も大きくなるにつれ，それまでのキャッシュとCPUの間にさら
に少容量のキャッシュを入れて高速化するようになった．現在の多
くの計算機では，CPUの近いほうから，一次キャッシュ，二次キ
ャッシュというように2段のキャッシュメモリをもつようになって
いる．CPUは一次キャッシュにアクセスし，データがない場合に
は二次キャッシュ，さらにない場合にはメインメモリへアクセスす
る．アクセス頻度の高いデータほど，CPUに近いメモリにおかれ
ることになり，高速化されることになる．通常，図1･4に示すよう
に一次キャッシュは命令用とデータ用の別々の一次キャッシュをも
つ．
計算機を高速化するアーキテクチャ（architecture）の基本的な
仕組みは，多くの命令を並列に処理するということである．
パイプライン（pipeline）アーキテクチャはその典型的な方式の
一つである．CPUは命令を一つずつ順に実行するが，一つの命令
の実行手順を細かく見てみると，いくつかの手順に分かれる．例え
ば，典型的なCPUの処理として以下の大まかな手順が考えられ
る．
(１)実行するべき命令をメモリから読み出す（IF:instruction
fetch）．
(２)読み出した命令を解読して各部分に信号を送り出す（DE:
decode）．
(３)命令を実行する（execute）．オペランドの読出し（EX1）
と演算の実行（EX2 ）の二つの処理を考える．
(４)結果をレジスタやメモリに書き込む（WB:write back）．
これらの処理を行うそれぞれの回路はクロック（clock）で制御さ
れ，クロック信号が入力されるごとに順に処理が進められる．クロ
ックの周波数が高ければ高いほど高速に処理されることになるが，
クロックの間隔はそれぞれの処理の時間による．
この場合，5クロックで1命令実行されることになる．一つの命
令の実行が終わってから，次の命令の実行を開始するとすると，2
命令実行するのに10クロック， 命令実行するには5クロック
かかることになる．その様子を図1･5に示す．
これらの処理はCPUの各部分で行われているため，1命令ずつ

分は使われていないことになる．例えば，命令実行しているとき
