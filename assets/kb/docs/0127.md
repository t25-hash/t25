# 1・2・4 リアルタイムOS（real -time operating system）

> **章**: 第1章 情報システム
> **節**: 1・2 オペレーティングシステム

a.リアルタイム性とそのカテゴリ 計算機によってさまざま
な計算処理を行う場合，その計算処理時間に強い時間的な制約があ
る場合とない場合がある．例えば，集積されたデータに統計処理を
施したり，全地球的な大気循環のシミュレーションを行う，といっ
た計算処理は，その処理ができるだけ短時間で行われることが期待
されるにせよ，その処理結果が得られるべき時刻に強い時間的な制
約があるわけではない．その計算処理を行わせたユーザは，その計
算結果が出るまで単に待てばよい．一方，例えば機械システムのサ
ーボ系をソフトウェアで構成する場合には，その計算には強い時間
的な制約がある．ソフトウェアで実現するサーボ系では，数ミリ
秒，あるいはそれより短い時間の周期ごとに同様の計算処理を繰返
し行うソフトウェア構造を実装する．この場合，一回のサーボ周期
(４)現在実行中のタスクAの優先度より高い優先度をもつタ
るのである．
b.リアルタイムOSの必要条件 リアルタイムOSでは，そ
のOSの上での計算処理（タスク）が，ハードリアルタイムの意味
で実行されることを要求されたとき，そのタスクの実行がその意味
において保証されることが期待される．もちろんこのリアルタイム
性を保証しながらのタスク処理速度と同時実行可能なタスク数はそ
の計算機の性能に依存するので，限りない数のタスクを限りなく短
い時間で達成することは要求できない．したがって，(１)実行が要
求されたタスクがすべてハードリアルタイムの意味において実行可
能かどうかを判断し，かつ，(２)そのタスクに対するCPU割当て
スケジュールを計画し管理する，（リアルタイム）タスクスケジュ
ーリング（task scheduling）の理論が必要である．この理論に基
づくタスクスケジューリングアルゴリズムが実装されていること
が，リアルタイムOSであることの必要条件であるといってよいで
あろう．実際に，リアルタイム性を実現するタスクスケジューリン
グの理論やアルゴリズムは種々提案されており，文献(１)には，原
典の引用とともにそれらの詳細が解説されている．
一般にリアルタイムなタスクスケジューリングの理論は，
(１)そのタスクを要求するイベントが周期的なもののみか，散
発的（そのイベントが集中的に発生すると，しばらく発生がなく，
また集中的な発生することをいう．通信におけるパケットの到着と
情報の復元の場合などがこれに当たる）なものも含むのか否か
(２)そのタスクを行うCPUは一つか複数か
(３)そのタスクの優先度は固定（固定優先度）か，実行中に優
先度を変化させることがある（動的優先度）か
スクBの実行開始イベントが生起したとき，現在実行中のタスク
Aを中断してタスクBを実行開始（プリエンプト）できるか否か
(５) そのタスクが他のタスクとの間に相互依存関係（他のタス
クの計算結果がでるのをそのタスクが待つ，複数のタスクが共通の
ハードウェア資源を利用するために相互排除が必要，など）がある
か否か
などの前提を種々考慮しながら構成される．
代表的なスケジューリングアルゴリズムとしては，固定優先度，
単一プロセッサ，周期的タスク実行，その周期時間が計算終了時間
に等しい，相互に依存関係のないタスクのみなどの仮定のもとに設
計されたレートモノトニックスケジューリング，その周期時間が計
算終了時間に等しいとの仮定を外したデッドラインモノトニックス
ケジューリング，動的優先度スケジューリングの方法であるEDF
（earliest deadline fast ）スケジューリングやLL（least laxity）
スケジューリング，タスク間に依存関係がある場合の動的優先度ス
ケジューリングである優先度継承プロトコル，優先度シーリングプ
ロトコルなどがある．
タスク間に相互依存があるケースは，実用的なリアルタイムOS
では当然に仮定されるべきである．この場合，タスク間で優先度継
承が行えることは，リアルタイムOSでは必要な条件であるといっ
てよい．これは優先度の逆転と呼ばれる現象を防ぐために必要であ
る．優先度の逆転とは，(１)そのOSの上で実行されているタスク
A，B，Cがあり，(２)最低優先度タスクCの実行結果待ちで最高
優先度タスクAの処理を中断している状態で，(３)結果的に中優
先度タスクBのみが実行され，(４)Cも中断されたままで，結果
としてAにタスクの割当てがまわってこず，(５)最高優先度のA
の処理が進まずにそれより低い優先度のBの処理のほうが進む，
というような場合である．優先度継承は，この優先度の逆転を検出
し，自動的にCの優先度を上げてAの処理を結果的に速く進ませ
るようにすることである．このためにも，そのOSがプリエンプト
可能であるべきことは必要な条件となる．
さらにリアルタイムOSの設計と実装に当たり，そのOS自体に
CPU割当てがある期間においても，プリエンプト禁止や割込み禁
止で処理を行う時間を極力減らすこと，また，ハードウェア割込み
がかかってから，割込み処理が行われるまでの時間も極力短いこと
（割込み反応速度が速いこと）などの工夫も必要である．
