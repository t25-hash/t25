b.OSIにおける各層の働き 以下では，OSIの各層の機能に
ついて概説する．ここで，通信をする主体をノード（node）と呼
ぶことにする．
.物理層（physical layer ) 最下位層の物理層は，隣接す
るノード間でビット伝送を行うための物理的な伝送路を上位層に提
供する．このため，伝送媒体の種類や機械的・電気的整合条件を定
めるだけではなく，ケーブルのコネクタ形状やピンの配置などにつ
いても規定される．この層の代表的なものとして，EIA- 232E（旧
RS- 232C）や，V.35などがある．
.データリンク層（data link layer ) 第2層のデータリ
ンク層は，物理層を用いて，隣接するノード間で透過的かつ高信頼
的なビット伝送を提供する．そのため，物理層の機能に加え，デー
タのフレーム化，順序制御，誤り検出や回復，フロー制御などを行
う．この層の代表的なプロトコルとしてHDLC（high-level data
link control ）が挙げられる．また，このプロトコルを基にして，
パケット交換網ではLAPB（link access procedure- balanced）
が，ISDNではLAPD（link access procedure on the D channel ）
が規定されている．
.ネットワーク層（network layer ) 一般に，通信を行
うノードどうしが直接通信回線で接続されるとは限らない．このよ
うな場合，いくつかのノードに中継してもらうことで，end- to-

継ノードを介在して接続されるノード間に対して透過的かつ高信頼
的なビット伝送を提供する．
このような接続においては，どの中継ノードをどのように経由す
るかによって通信効率が変わってくる．この選択を経路選択または
ルーティング（routing）と呼び，ネットワーク層における重要な
役割の一つとなっている．また，中継ノードだけの働きが求められ
る場合には，図1･21に示すように，物理層からネットワーク層ま

と呼ばれ，ネットワークを相互接続する際の重要な役割を果たす．
ネットワーク層の代表的なプロトコルとして，パケット交換におけ

.セッション層（session layer ) セッション層では，さ
まざまな種類のアプリケーションが要求する通信手段を提供する．
その一つに同期点の設定がある．セッション層では，データの一連
の流れを会話（dialog），いくつかの会話による一つの仕事の単位
をアクティビティ（activity）と呼ぶ．例えば，複数のファイルを
まとめて転送する場合，一つのファイルを転送する作業が会話に相
当し，全ファイルを転送する作業がアクティビティに相当する．一
つのファイルを転送するたびに，通信当事者双方で同期を取る必要
があり，これを大同期点（major synchronization point ）と呼ぶ．
また，ファイルの転送途中で小同期点（minor synchronization
point ）と呼ばれるチェックポイントを設定することができる．こ
れらの同期点を設定することで，あるアクティビティを処理してい
る間に障害が生じた場合でも，アクティビティを始めからやり直す
のではなく，障害が生じる直前の同期点から作業を再開することが
できる．
さらに，セッション層ではデータ転送を半二重（half duplex）
または全二重（full duplex）で行うことを選択する機能が提供さ
れる．ここで半二重とは，トランシーバのように，双方向通信が可
能ではあるが，一度に伝送できる方向は一方に限られる通信形態を
指す．このとき，双方のノードから同時にデータを送信すると，両
者は衝突して失われるため，送信するノードを定めるための送信権
制御が行われる．全二重では，公衆電話回線のように，双方から同
時に伝送を行うことができるため，送信権制御は行われない．
.プレゼンテーション層（presentation layer ) プレゼン
テーション層は，アプリケーション層が扱うデータの形式を伝送可
能なデータ形式に変換することで，効率的に伝送を行う役割を果た
す．ここで，アプリケーション層が扱う構文（syntax）を抽象構
文（abstract syntax）と呼ぶ．抽象構文は送信するデータを特定
のプログラミング言語や計算機に依存しない形式で表したもので，
抽象構文を表現する言語としてASN.1（abstract syntax nota-
tion1）が規定されている．プレゼンテーション層が扱うデータ表
現形式を転送構文（transfer syntax）と呼ぶ．転送構文は抽象構
文で表現された形式をセッション層に渡されるビット列に変換する
ものであり，例えば，文字コードや整数のバイトオーダ，浮動小数
点数の形式などの変換が行われる．また，送信データの圧縮や，暗
号化を行う作業もプレゼンテーション層の役割である．
.アプリケーション層（application layer ) アプリケー
ション層は，ユーザに対してさまざまなアプリケーションサービス
を提供する．このサービスを提供するモジュールはASE（appli-
cation service element ）と呼ばれ，アプリケーション層は特定の
業務処理に対応したASEと，これらのASEに共通なASEの二つ
の副層から構成されている．
ここで，特定業務対応のASEとしては，電子メール（MHS:
message handling system）やファイル転送（FTAM:file trans-
fer,access and management ）などがある．一方，業務に依存しな
い共通ASEとしては，遠隔操作を行うROSE（remote opera-
tions service element ）や信頼性の高い転送を行うRTSE（reli-
able transfer service element ）などがある．
