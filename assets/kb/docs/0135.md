b.POP 次に，もう一つの代表的な電子メール用プロトコル
POP（post oﬃce protocol ）バージョン3について簡単に述べる．
詳細については，文献(13)(14)を参照されたい．
POPは，SMTPなどでメールサーバに到着した電子メールを読
み出すためのプロトコルである．このプロトコルは，ユーザの端末
が常時ネットワークに接続されていない場合や，SMTPサーバを
準備できない場合などに用いられる．POPは電子メールの転送と
サーバからの削除を主目的としており，より複雑な機能が必要な場
合はIMAP（internet message access protocol ) などを利用す
る．
POPの通信は，SMTPと同様に，クライアントからサーバへコ
マンドを送り，それに対してサーバがクライアントに対して応答を
返すことで行われる．クライアントから送られるコマンドは，コマ
ンドの種類と（必要ならば）引数を表す印字可能なASCII文字列
で構成される．これに対してサーバからの応答は，肯定応答ならば
「＋OK」で，否定応答ならば「－ERR」で始まる文字列で構成さ
れる．応答に続けてデータが送付される場合は，最後にピリオドの
みを含む行で終了する．ただし，ピリオド「. 」で始まる行をデー
タとして送付する場合は，その前にピリオドを一つ挿入して送る．
なお，コマンドと応答の行は，いずれも改行文字CR（ASCIIコー
ド13）とLF（ASCIIコード10）で終端される．

省略した場合は80番が仮定される． パス は，接続先ホスト内に

す．HTTPバージョン1.0では，サーバからの応答終了直後にコ

セージは次のような形式を取る．まず，最初の行に要求または応答
の種別を示す．必要であれば，その次の行からヘッダと呼ばれる付
加情報を続けることができる．そして，空行を挟み，メッセージ本
体を続ける．なお，メッセージ内の各行はいずれも，改行文字CR
（ASCIIコード13）とLF（ASCIIコード10）で終端される．
ここで，クライアントがサーバwww.gama.ac.jpに対し，ファ

からサーバへの要求を図1･31に，これに対するサーバからの応答

明する．
まず，図1･31の1行めにおいて，クライアントはGETメソッ
ド（method）を送っている．これは，その次に続くindex.htmlの
内容を送信するよう要求するものである．最後に，HTTPプロト
コルで用いられるバージョン番号が記される．GETメソッド以外
には，リソースの情報のみを通知させるHEADメソッドや，メッ
セージ本体にあるデータを指定したリソースに対する入力として送
るPOSTメソッド，そのリソースに対してどのような要求ができ
るかを問い合わせるOPTIONSなどがある．
GETメソッド以下の行は，すべてヘッダになる．ヘッダの最後
は空行で終端しており，メッセージ本体はない．
User -Agentはクライアントで使用しているプログラム名とその
バージョン番号が示される．Acceptはそのプログラムで受付け可
能なデータの種類を示す．図1･31ではテキスト（text ）と画像
（image）全般を受付け可能である．Accept - Encodingはレスポン
スで使用できる符号化方式を示す．この例では，gzipによる圧縮
を受付ける．Accept - Languageは，クライアントで利用できる言
語であり，この例では日本語（ja）と英語（en）を指定している．
ところで，英語に関してはさらに＝0.5が指定されているが，こ
の値はquality valueと呼ばれ，ユーザの好みを表している．最小
は0，最大は1であり，大きいほど好ましいことを意味する．ま
た，＝0のときは特に，その指定が受付けられないことを示す．
なお，この指定が省略されたときは＝1が仮定される．
最後のHostはクライアントがアクセスしたホスト名を示すもの
で，HTTPバージョン1.1では必須のヘッダとなっている．この
ため，一つのサーバで複数のサーバに見せかけることが容易に実現
できる．例えば，www.gama.ac.jpというホストにnews.gama.ac.
jpという別名を与えておき，クライアントから来たメッセージの
Hostの値がどちらの名前になっているかによって，サーバの挙動
を変えることができる．このようなサーバはmulti -homed server
と呼ばれる．

まず，最初の1行めにはサーバの応答の内容が示されており，サ
ーバのバージョン番号と状態コード，およびそれに対する人間向け

状態コードは，100番台はサーバからの情報通知（処理継続中），
200番台は要求に対する処理の成功，300番台は要求の実現にはさ

ー，そして500番台はサーバの処理エラーを意味する．
2行めから空行の直前（7行め）まではヘッダであり，Dateは処
理が行われた日時，Serverはサーバのプログラム名とそのバージ

また，Content -Lengthでメッセージ本体のサイズを表し，Con-

空行以降（9行め以下）はメッセージ本体であり，クライアント
