# 12・1・3 エキスパートシステムの構成と機能

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・1 エキスパートシステム

エキスパートシステムの構造的な特徴は，システムが知識ベース
と推論機構とに分離されていることにある．この構造が，専門家の
経験的知識を利用するというエキスパートシステムの機能の実現に
都合が良い．エキスパートシステムは図12･1に示すような六つの
要素から構成される．ただし，エキスパートシステムを実現するた
めには，必ずしも図12･1の構成に従う必要はない．従来型のソフ
トウェアとしても，エキスパートシステムの機能は実現可能であ
a.知識ベース エキスパートシステムで知識を適切な表現方
法に従って記述してコンピュータ上に格納し，必要に応じて引出せ
るようにしたものが知識ベースである．知識ベースの内容はシステ
ムが対象とする問題によって異なる．しかし，知識ベースの適当な
知識を書き換えるだけで，システムの機能拡張が可能であり，さら
には，知識全体を交換することで，全く別のシステムが構築できる
b.推論機構 利用者インタフェースやシステムインタフェー
スを通じて外部から入力される情報と，知識ベース内の知識を使っ
て問題解決の処理を行うのが推論機構である．ここで，推論とは探
索（searching）を記号的かつ形式的に行う方法全般を指す．推論
の過程では，推論機構はその局面にあった適切な知識を選定，適用
すると同時に，利用者からの各種の問い合わせに応答するための機
能を制御する．
