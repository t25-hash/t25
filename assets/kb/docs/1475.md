# 12・10・1 基本概念

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・10 オブジェクト指向

構造化プログラミングが，問題解決のためのアルゴリズムをモジ
ュール化して，ソフトウェアの再利用性を高めたのに対し，オブジ
ェクト指向では，オブジェクト（object ）というものを単位とし
て，再利用性を図っている．ソフトウェア設計の基本概念が異なる
ことが，オブジェクト指向の理解を難しくしている面がある．
a.オブジェクトとクラス 従来のアルゴリズム中心の設計法
では，問題解決の方法を詳細に検討し，タスクの分類と関連を完全
に行った後に実装（プログラミング）段階に移る必要があった．し
かし，人間の思考はもっと抽象的な概念を用いることによって，よ
り複雑なシステムを効率的に検討することが可能である．オブジェ
クト指向は，この自然な思考方法をソフトウェア開発に取り入れた
ものである．ここで，オブジェクトはものであって，人間の思考対
象の単位となっており，それ自身の性質・状態をもち，外界からの
刺激によって状態を変化させることができる．この性質・状態を属
性と呼び，他オブジェクトからの刺激をメッセージ，それに対する
ふるまいを定義したものを，操作あるいはメソッドと呼ぶ．オブジ
ェクト指向ソフトウェア開発では，システムに関連するオブジェク
トの発見と，オブジェクト間の関連を定義することが，最も重要に
なる．
クラス（class ）とは，オブジェクトを生成するための型となる
ものである．生成されるオブジェクトは，インスタンスとも呼ばれ
る．クラスには，オブジェクトの属性と操作がすべて定義されてい
る．プログラム開発はこのクラスを作成することにほかならない．
再利用を前提にしたクラス定義群をクラスライブラリと呼ぶ．
b.データ隠ぺい オブジェクト内のデータは，基本的には，
オブジェクトで定義されたメソッドからしかアクセスできない．こ
れをデータ隠ぺいと呼ぶ．この機構により，他のオブジェクトが不
当にデータを破壊することができなくなり，プログラムの堅牢性が
増す．さらに，オブジェクトの外からの見え方（インタフェース）
と実装を分離することができ，より効率的なデータ構造やアルゴリ
ズムを，他の部分を変更せずに採用することができる．このような
クラスを抽象データ型（abstract data type）と呼び，開発・テス
トがクラス単位で行えるため，開発コストの低減化が期待できる．
c .メッセージパッシング 上記のデータ隠ぺいの機構により，
オブジェクト内のデータは直接変更できない．そのオブジェクトに
何らかの機能を実行させたいときには，メッセージを送ることによ
って，機能の実行を依頼することができる．これをメッセージパッ

シングと呼ぶ．オブジェクトは他のオブジェクトにメッセージを送
ることにより，処理を依頼することができるのである．オブジェク
トは正しいメッセージにのみ対応して処理を行うので，不正なアク

処理が行われる．これを多態（ポリモフィズム，polymorphism）
と呼ぶ．多態は，複雑なシステムのインタフェースを単純化するの
に効果的である．
d.オブジェクトの関係 オブジェクト間の関係は，継承，依
存，関連の三つに分類される．
.継承（inheritance) 継承はis -a関係とも呼ばれ，二
つのオブジェクトの概念に包含関係が存在する．図12･38では，オ
ブジェクトAがオブジェクトBの概念を含んでいる．したがっ
て，BはAをより特化したものであるといえる．逆にBからAの
向きは汎化（generalization）と呼び，Aはより抽象度の高い概念
となっている．実装レベルでは，親となるオブジェクトの性質や動
作について，子に当たるオブジェクトではプログラミングを行う必
要がないため，新しく増えた部分のみ実装すればよい．これを差分
プログラミングと呼び，ソースプログラムの再利用による開発コス
トの低減と高信頼性を実現している．継承関係にあるオブジェクト
どうしには，同じ名前のメソッドが存在するが，異なる動作をして
もよい．これを継承による多態という．
.依存（dependency) あるオブジェクトが他のオブジェ
クトを使用している関係で，use- a関係とも呼ばれる．操作を行う
ときのパラメータとして他のオブジェクトを使う場合がそうで，パ
ラメータとして使われたオブジェクトに変更が加えられると，それ
を使用しているオブジェクトにも影響が及ぶ．
.関連（association) オブジェクト間に包含関係はな
く，何らかの関連を有するもの．関連の方向は一方向でも双方向で
もよく，それぞれのオブジェクトの役割（role ）を指定できる．特
に，オブジェクトの内部に他のオブジェクトを含み，全体と部品の
関係にあるものを集約（aggregation）といい，has -a関係とも呼
ばれる．さらに，部品に当たるものが一つのオブジェクトだけに属
するときは，複合（composition）関係と呼ぶ．
