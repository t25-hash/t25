# 12・10・2 ソフトウェア開発手順

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・10 オブジェクト指向

a.ラピッドプロトタイピング（rapid prototyping）とらせん型
開発 オブジェクト指向アプローチでは，オブジェクトという概
念を導入するだけでなく，開発手順も異なっている．従来の手法で
は，ソフトウェア開発サイクルの分析・設計・実装が，それぞれ各
段階で完成してから次の段階に進む方法がとられていた．途中での
システム設計変更が非常に難しいからである．オブジェクト指向を
用いれば，オブジェクト間の依存関係が少なくなるため，インタフ
ェースだけ最初に決定すれば，内部の詳細が未実装であっても，全
システムに組込むことが可能である．部分が未完成であるから，非
常に素早く全システムの概要を知り，問題点を把握することができ
る．この手法をラピッドプロトタイピングと呼ぶ．このシステムを
テストすることにより，最初の設計の問題点を洗い出し，システム
の再設計を行う．このように，開発において，同じ要素を何度もブ
ラッシュアップしながら完成に近づけていく方法を，らせん型開発
（spiral development ）という．

d.実装（implementation) 最後が実装である．このとき初
めてプログラミング言語を扱う．しかし，必ずしもオブジェクト指
向言語を使用する必要はない．オブジェクト指向のために新しく用
意された機能が必要なければ，従来の言語を利用してもよい．
