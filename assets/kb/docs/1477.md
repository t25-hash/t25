# 6・3・3 簡単なプログラム例

> **章**: 第6章 固体の離散系シミュレーション
> **節**: 6・3 メトロポリス法

メトロポリス法の計算プログラムの例(C言語)を図6・12に示した。この例では簡単のため、微視的状態は一次元変数としてあるが、実用的なプログラムでは原子の座標など多次元ベクトルである。関数energy( )内で、目的に応じて、に対応するエネルギーの値を計算する。実用的には全原子の座標などに依存したポテンシャルエネルギーであるが、この例では単純な二次関数とした。定数TEMPERATUREとして与えられている温度は、エネルギーや変数とともに適切な無次元化がされているものとする。図6・13はこの例題プログラムを用いて、平均エネルギーが計算ステップ数にどのように依存するかを見たものである。初期値の選び方にもよるが、この場合は10程度のステップ数で平衡に近づい図6・12メトロポリス法の計算プログラム例図6・14 平均エネルギーの収束状況の変位幅依存性図6・13 平均エネルギーの収束状況の初期値依存性ているものと判断できる。なお、TEMPERATURE=1.0の平衡状態では、平均エネルギーはexp-dexp-d=0.5である。また、図6・14は同じプログラムで、1ステップでの最大変位幅DX を変えた場合の収束状況である。これらの結果から、初期条件が平衡状態に近く、また1ステップで状態が大きく変わるほうが、計算の収束が早いことがわかる。これは、実用的なシミュレーションにおいても同様である。特に、各ステップでの状態変化の幅を適切に選ぶと、計算の収束はかなり加速される。次の状態の候補が採用される確率が0.2~0.5程度になるように、変化の幅を選ぶのがよいという目安もあるが、対象系によっても異なるので、大規模な系では、予備計算により適切な計算条件を選ぶのが望ましいだろう。特に、まれに起こる現象を対象とする場合(例えば、高い自由エネルギー障壁を越えて相転移が起こる場合)には、採用確率を1%以下にしてでも、大きな状態変化を位相空間中に生成するべきである。この場合には当然、十分な統計平均をとるためにステップ数も多くする必要がある。プログラム例中の関数drand( )は、0以上1以下の一様乱数(uniform random number)を返す倍精度実数型の関数で、C言語の標準ライブラリのrand( )を利用している。少し古いOS やライブラリを実装している計算機では、rand( )の性質が良くない(偏りがある、周期が短いなど)ことがあるので、ライブラリのドキュメントを調べるなどの用心が必要である。ライブラリ内で定義されている最大値RAND MAX が2=65536程度というひどい場合もあり、これだと数万ステップ後には同じ乱数が繰返し出現してしまう。実用的な計算では周期10程度の乱数が必要となることも多く、性質がある程度保証された乱数生成アルゴリズムを組込むのが望ましい。
