# 12・1・2 エキスパートシステムの歴史

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・1 エキスパートシステム

エキスパートシステムの最初の例は,1960年代半ばに開発が開始されたDENDRALである.これは,未知化合物の質量スペクトルデータと分子式から,その化合物の構造式を推定する解釈型システムであった.やはり1960年代に開発が始まった数式処理システムMACSYMA は,知識ベースシステムではないが第1の定義に従うエキスパートシステムであり,実用的なシステムとして有用であった.1970年代に入ると,医療診断システムとして,MYCIN,CASNET,Internist等が続けて発表された.そして,これらのシステムで共通に使われたプロダクションシステム(production sys-tem)の手法が,鉱物資源探査支援システムPROSPECTOR にも適用可能なことが明確になった時点で,エキスパートシステム技術を中心とする,AI(artificial intelligence,人工知能)ブームが訪れた.1970年代終わりから1980年代初めにかけて,知識システムの実現手段として,あるいは知識表現技術として成熟したのが,言語OPS ファミリーやCLIPS などに代表されるプロダクションシステム技術,フレーム(frame)やスキーマ(schema)に代表されるオブジェクト技術,そして,Prolog などに代表されるロジックプログラミング技術であった.1980年代後半には,これらの技術が成熟し,計算量の壁などの理論的な問題点が明確になった.この時期には産業界への応用システム例が数多く発表されるようになった.DEC社のコンピュータ機器構成の決定を行う,設計型システムとして有名なR 1/XCON が発表され,成功例としてもてはやされたのもこのころである.これらのシステムの成功は,システム構築を容易化するための,開発環境としての構築ツールの発展をうながした.また,同時にエキスパートシステム構築の問題点が指摘されるようになった.そのうち最も重要なものが知識獲得の問題である.1990年代以降,一時のAIブームは沈静化し,エキスパートシステム技術は,高度なコンピュータシステム開発の一手法として着実に普及した.21世紀に入ると,エキスパートシステムという用語そのものも使われなくなってきた.
