# 13・2・1 並列処理（parallel processing）の重要性

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

手続きには，順を追って逐次的に処理しなければならない部分と，13･19(ａ)に示すDOループは，配列Aに関して前のループの演算きず，逐次的に実行しなければならない．
これに対し，図13･明らかであり，並列処理が可能である．

これに対し，実際に場合，理論性能よりもかなり低い値となる．
これまで，計算機ハーチャにおける並列処理によって達成されてきた．
クロック高速化方，並列処理はベクトル計算，命令の並列実行，マルチプロセッサ化などで実現されている．
特に近年の計算機システムにおいては，処理が大きな寄与を果たしていることに注意しなければならない．
アダプティブメッシング（順応型要素再分割) 解析を並列に同時計算できる部分とに分けることができる．
例えば，図結果が必要なため，インデックスJについて並列処理することはで19(ｂ)の例では，インデックスJについて独立に計算できることは逐次処理を含めすべての処理は基本的に高速化される．
しかしながことを考えると，コンパイラや計算プログラムがハードウェアの有わかる．
すなわち，プログラムやデータを分析して，処理の逐次性が乖離しないような工夫が必要である．
算順序やデータ配置を並べ替えたり，並列化支援の通信ライブラリ正作業は，最適化あるいはチューニングと呼ばれる．
