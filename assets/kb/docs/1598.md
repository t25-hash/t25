# 12・1・3 エキスパートシステムの構成と機能

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・1 エキスパートシステム

エキスパートシステムの構造的な特徴は、システムが知識ベースと推論機構とに分離されていることにある。この構造が、専門家の経験的知識を利用するというエキスパートシステムの機能の実現に都合が良い。エキスパートシステムは図12・1に示すような六つの要素から構成される。ただし、エキスパートシステムを実現するためには、必ずしも図12・1の構成に従う必要はない。従来型のソフトウェアとしても、エキスパートシステムの機能は実現可能である。a. 知識ベースエキスパートシステムで知識を適切な表現方法に従って記述してコンピュータ上に格納し、必要に応じて引出せるようにしたものが知識ベースである。知識ベースの内容はシステムが対象とする問題によって異なる。しかし、知識ベースの適当な知識を書き換えるだけで、システムの機能拡張が可能であり、さらには、知識全体を交換することで、全く別のシステムが構築できるようになる。b. 推論機構利用者インタフェースやシステムインタフェースを通じて外部から入力される情報と、知識ベース内の知識を使って問題解決の処理を行うのが推論機構である。ここで、推論とは探索(searching)を記号的かつ形式的に行う方法全般を指す。推論の過程では、推論機構はその局面にあった適切な知識を選定、適用すると同時に、利用者からの各種の問い合わせに応答するための機能を制御する。c. 説明機能エキスパートシステムには、問題解決に利用した知識を提示することで、利用者にはわかりにくい専門知識の適用方法を説明するための機能を備えることができる。この説明機能には、なぜシステムが質問を出すのかを問うWhy、システムの推論経過を問うHow、条件を変化させたときのシステムの応答を調べるWhat-Ifなどの種類がある。d. 知識獲得インタフェース専門家から知識を獲得するためには、さまざまな方法が考案されている。その中でも特に重要なのが、大量の事実的知識またはデータからそれらに共通する問題解決知識を自動的に抽出する帰納的学習機能や、専門家の知識を対話的に抽出するインタビュー機能である。これらを総称して知識獲得インタフェースと呼ぶ。e. 利用者インタフェース利用者インタフェースは、人工知能技術の適用を容易にするための開発環境を備えた開発者インタフェースと、知識を提供する専門家向けに知識獲得機能を備えた専門家インタフェース、説明機能を含むシステムの利用を容易化するためのエンドユーザインタフェースに分けることができる。f. システムインタフェース他のプログラム言語や他のソフトウェアシステム、他の計算機との結合方式を定める。大規模なエキスパートシステムは既存プログラムとの結合・融合で成り立つので、複雑なシステムに人工知能技法を組込むためには、システムインタフェースの役割は重要である。図12・1 エキスパートシステムの構成
