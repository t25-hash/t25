# 12・1・4 エキスパートシステムの主要概念

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・1 エキスパートシステム

エキスパートシステムの開発に使われる諸概念は、基本的には人工知能のそれと等しい。すなわち、エキスパートシステムにおいては、知識を用いて対象となる問題空間を表現し、その空間を探索することで解を導く。通常、エキスパートシステムで使われる知識表現と推論方式は、ルール(rule)とフレームの概念に基づくものである。a. ルールを用いた知識表現と推論方法専門知識を記述する手段として、If-Then形式を用いるプロダクションシステムの手法は、広く利用されてきた。プロダクションシステムの優れたところは、ルールの記述性と理解性にある。これをうまく利用したのが帰納推論で得た分類知識をルール群、もしくはそれに相当する決定木で表現する手法であり、ファジィ変数をルール内に導入することで実現したファジィルールシステムである。その一方で、実行効率が悪いこと、実行制御が難しいことが欠点である。実行効率の改善については、ルールコンパイルの手法として、すでにさまざまな手法が提案されている。これにより、従来の手続き型プログラムに比較しても遜色のない程度まで効率の良いルールシステムが提供されている。ルールを使った推論方式の概要を図12・2に示す。これには、大きく分けて、前向き推論(forward reasoning)と後ろ向き推論(backward reasoning)がある。前向き推論あるいはデータ駆動型推論(data driven reasoning)と、それに関連する手法については、ルールコンパイル(rule compile)、またはルールベースプログラミング(rule base programming)手法として、標準的なスタイルが確立している。前向き推論は、一般に問題空間全域を広い範囲にわたって探索するのに適した推論方式である。後ろ向き推論あるいは目標駆動型推論(goal driven reason-ing)は、解候補の中から正解を選ぶという形で定式化される分類問題には、適切な方法である。b. フレームを用いた知識表現フレームは、図12・3に示す図12・2 ルールに基づく推論図12・3 フレームシステムの例6―167ように知識を分類階層化して整理し、操作するための表現形式であり、知識の単位としてのモジュール性が非常に高いという特徴をもつ。フレーム概念に対しては、スキーマ、ユニット(unit)などの用語を当てることも多い。また、フレームは、意味ネットワーク(semantic network)や、オブジェクト(object)の概念にも極めて近く、実現方式に大きな差異は存在しない。フレームは、対象システムの構造をサブシステムや部品などの概念を使って表現するには都合が良い。ただし、知識の階層のどの部分にどの概念を対応させるかを決定するのは、難しい問題である。特に、フレームのもつ概念継承機能(inheritance)については、論理的に微妙な問題が多く、非単調推論に直接関連する研究課題となっている。
