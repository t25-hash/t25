# 13・2・2 アーキテクチャにおける並列処理

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

a.ベクトル処理（vector processing) ベクトル処理はデー
タレベルでの並列化の一つである．ベクトルデータを同時に処理で
きるベクトルレジスタと，パイプライン（セグメント）化されたベ
クトル演算器との組合せによって，高速演算が実現される．図13･
20にベクトル処理とスカラ処理の概念図を示す．ベクトルデータ
をベルトコンベヤに乗って流れる多数のデータ列に例えると，ベル
トコンベヤに沿って並べて置かれた装置によって，例えば加算演算
の場合，メモリからの読み込み（R)，指数部比較（C)，指数部合
わせ（S)，加算（A)，正規化（N)，メモリへの書き込み（W）と
いった操作が，データに実行される．ベルトコンベヤがパイプライ
ンに相当する．これらの演算操作は，パイプライン上でオーバラッ
プしてベクトルデータに次々と実行される（パイプライン実行され
る）ため，並んだ装置の数だけ速度が向上することになる．こうし
たベクトル処理に対し，スカラ処理では，一つのデータがベルトコ
ンベヤを流れ終えてはじめて，次のデータをベルトコンベヤに投入
することができる．ベクトル型スーパーコンピュータでは，複数個
のベクトル演算機を装備することにより，さらに高速化が図られて
いる．
b.RISC RISC（reduced instruction set computer ，縮小命
令セット方式）と呼ばれている方式では，使用頻度が高く実行時間
の短い単純な命令セットのみを，メモリよりもアクセス速度の速い
キャッシュメモリとのアクセスにより，高速処理している．また，
命令実行がパイプライン化され，命令の見掛け上の1サイクル時間
が短縮化されている．スーパースカラと呼ばれる計算方式では，複
数の命令列のうち可能なものから並列実行される．この方式のプロ
セッサは，パーソナルコンピュータに広く用いられている．また，
並列計算機の要素プロセッサとしても用いられている．
c.マルチプロセッサ アーキテクチャ面での並列化のレベル
をもう一段階上げたものが，いわゆる並列計算機である．複数のプ
ロセッサにメモリをどう割当てるかによって，共有メモリ型，分散
メモリ型，共有・分散メモリ型の三つの方式がある（図13･21)．
共有メモリ型は，全プロセッサから等距離の一つの大容量メモリに
アクセスするタイプのもので，SMP（symmetrical multi proces-
sor ）とも呼ばれる．パーソナルコンピュータでもマルチプロセッ
サ搭載のSMPが登場している．分散メモリ型は，プロセッサとメ
モリの組を一つの単位（ノード）とし，それらをさまざまなトポロ
ジーを有するネットワークで接続したものである．共有・分散メモ
リ型は，共有メモリ型マルチプロセッサを1ノード（SMPノー
ド）とし，それらをネットワーク接続したアーキテクチャで，いわ
ば共有メモリ型と分散メモリ型を組合わせた方式である．多くのハ
イエンド並列計算機では，高速なネットワークを用いた共有・分散
メモリ型を採用している．
コモディティプロセッサを比較的安価で低速なLANでネットワ
ーク結合したものを，一般にクラスタ計算機あるいはPCクラスタ
と呼び，並列計算機として多く用いられている．世界のスーパーコ
ンピュータの性能一覧であるTOP500 にも，このタイプが多く
見られるようになっている．
d.グリッド（grid) グリッドあるいはグリッドコンピューテ
ィングとは，各地に散らばる計算機やデータベースなどのリソース
をインタネットや専用回線で結び，次世代の計算環境ひいては社会
基盤にしようというものである．その名称はパワーグリッド
（power grid，電力網）とのアナロジーから命名されている ．多
様なOS，計算機，ネットワーク，データベース等を統一的なツー
ルやインタフェースで取扱うために，さまざまなミドルウェアやプ
ロトコルが開発されている．有名なミドルウェアとしては，
Globus ，Ninf などがある．グリッドの利用形態としては，世
界中に分散しているスーパーコンピュータをネットワーク接続して
超大規模計算を目指すメタコンピューティング，大容量データの分
散配置や処理を行うデータインテンシブコンピューティング，遊休
PCなどの計算機資源を活用するボランティアコンピューティン
グ，地理的に分散した場所から研究者がバーチャルリアリティなど
のツールを援用して協同作業を行うテレイマージョン，ハイエンド
のスーパーコンピュータ等に実装されて動作が保証されている高性
能アプリケーションを遠隔地から利用するグリッドASPなどがあ
る．
