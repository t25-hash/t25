# 13・4・1 問題解決環境（PSE）とプログラム自動生成

> **章**: 第13章 シミュレーション支援法
> **節**: 13・4 問題解決環境

a.PSEの概説 PSE（problem solving environment ）は歴
史的に，PDE（partial diﬀerential equation，偏微分方程式）問題
の解決を支援するソフトウェアと位置付けられる．PDEは空間的
な広がりをもつ物理問題を扱うことから，計算時間の面でも記憶領
域の点でも挑戦的な問題である．したがって当初は，ソフトウェア
の生産性や再利用性よりも性能が重視された．またPDE問題は定
式化の自由度が大きく，また個々の問題に応じた個別の工夫も必要
とされることから，本質的に標準化が困難な面がある．しかし，こ
れらの性格が再利用への強い要求を生むこととなり，計算機性能の
向上とともに，1960年代に入ってから生産性と再利用性の向上に
向けた取組みがなされるようになった．
後にPSEと呼ばれるようになるPDE問題解決に向けた生産
性・再利用性向上への取組みは，利用者とのインタフェースの取り
方により次の3種類に分類される．
第1は，ある程度のはん用性と技法の一貫性を備えたアプリケー
ションソフトウェアの開発である．構造解析分野におけるNAS-
TRAN（1971）は，その典型と考えられる．
第2は共用性の高いソフトウェア部品を，ライブラリまたはモジ
ュールとして用意しておき，それを用いてアプリケーションを組立
てる方法である．ライブラリの機能としては，形状入力，メッシュ
分割，離散化，行列変形，線形計算，結果出力などが挙げられる．
利用者のインタフェースとして，特定の記述言語を用意しておいて
利用者に問題記述を行わせ，そこからライブラリ呼出しコード等を
自動生成するタイプのものと，利用者が既存の言語インタフェース
を介して，直接ライブラリコードを呼出すタイプのものがある．前
者の例としては，SALEM（1968），PDEL（1970），ELLPACK
（1977），DEQSOL（1985），NCAS（1993）などが挙げられる．こ
のうちDEQSOLとNCASについては，b項，c項にて説明する．
後者の例としては，双曲形方程式のためのDCG（1975），
HYPACK（1978），放物形方程式のためのPDEPACK（1975），
DSS/2（1976），MOL1D（1976），PDETWO（1976），POST
（1977），DISPL（1977），LSQPDE（1978），FORSIMVI （1978）
など，だ円形方程式のためのEIGEN（1974），ITPACK/
REGION（1977），EPDEI （1977），GENEPI （1977），FISH-
PACK（1978）など多数が挙げられる．
第3は，既存のプログラミング言語を拡張して，利用者に機能を
提供する方法である．PDELAN（1972）は，FORTRAN言語に
有限差分解法を記述するためのメッシュ宣言，分布量宣言，差分演
算定義の機能を追加している．PDELANによる記述は，プリプロ
セッサにより通常のFORTRANに展開される．
上記を含む1970年代のPSEについては，文献(78)に包括的なサ
ーベイがある．
b.DEQSOL DEQSOL（diﬀerential equation solver ）
は，おもにベクトル型スーパーコンピュータ用高性能コードの自動
生成を目的としたPSEである．利用者向けの高位記述に基づいて，
FORTANの計算コードを自動生成する方式をとっている．あらか
じめ用意されたライブラリは線形計算ソルバのみであり，この点で
はライブラリのドライバの性格が強いELLPACKとは異なる設計
となっている．これは，生成コードの自動ベクトル化の容易さをね
らったためである．DEQSOLによる問題解法記述は，離散化手法，
領域形状，メッシュ分割，変数/定数宣言，部分領域宣言，境界条
件，初期条件，解法手順などのセクションから成る．言語仕様上の
特徴としては，数理モデルと計算スキームの分離記述，簡潔な微分
記法，領域形状の言語定義などが挙げられる．機能面では，二次
元，三次元空間内の2階の微分を含む定常，非定常連立方程式系が
扱える．非線形問題は，線形化計算スキームを工夫することにより
対処する．離散化手法としては，有限差分法とガラーキン式有限要
素法の両者を提供しており，後者の場合要素の種類は，三角形，四
辺形，四面体，六面体を基本とする線形要素と二次要素である．境
界形状は，多角形や多面体に加え，スプライン曲線やスイープ曲面
を取扱うことができる．線形解法として，ガウス消去による直接解
法とICCG（incomplete Cholesky conjugate gradient )/ BCG（bi -
conjugate gradient ）系の反復解法を用意している．半導体の不純
物拡散シミュレーション，LSIチップの冷却シミュレーション，デ
ィスクヘッド周りの三次元磁場解析などの適用事例があり，対
FORTRANプログラム記述行数で10分の1以下，スーパーコン
ピュータ向けベクトル演算化率90%以上を達成している．
c.NCAS NCASでは数学モデルからC言語のシミュレ
ーションプログラムを生成支援する際，問題記述からプログラムの
設計・生成されたプログラム自身に至るまでの過程を可視化し，ユ
ーザによる変更・修正（ステアリング，steering）を可能にしてい
る ．この意味でNCASは，ホワイトボックス型PSE（white
box PSE）と呼ばれる．シミュレーション過程のすべてをユーザ
に可視化して開示し，修正変更をも可能にし，ユーザの意図を反映
したプログラムの生成支援を行う．多次元流体プログラムなど，
5000行程度のプログラムの生成が実現されている．P-NCASでは
並列プログラムの自動生成にも対応している．
NCASの設計思想の裏には，PDEで記述できる問題をコンピュ
ータでシミュレーションする場合，扱いたい問題に全面的に集中し
たいのに，なぜ人間が機械的な作業であるプログラミングを行わな
くてはならないのか，現在のコンピュータは本物のコンピュータに
進化しきれていないのではないか，という疑問があった．同時に，
差分法や有限要素法によってプログラムを作る際には共通点があ
り，PDE問題領域に限れば，プログラミングのある程度の機械化，
すなわち自動プログラミングが可能であることもわかった．こうい
った背景のもとNCASは，問題の本質の抽出などの人間が得意な
ところは人間が分担し，コンピュータが得意なところはコンピュー
タに任せることを設計思想としている．
P- NCASでは並列プログラムの設計過程の可視化とステアリン
グも実現した ．計算領域の領域分割により並列化をサポートし，
並列モデルにはSPMDモデルを採用した．メッセージパッシング
にはMPI関数を用いた．計算領域の分割は領域の可視化ウインド
ウにより，可視化された領域を見ながら行うことが可能であり，領
域分割方法・領域分割数を可視化ウインドウで確認しながら変更し
設定できる．領域分割に関する設定の後，NCAS内でプログラム
が設計され，設計されたプログラムはPAD（problem analysis
diagram）図により可視化される．PAD図の可視化ウインドウで
も，並列パラメータの変更・確認を行うことができる．プログラミ
ングや並列化過程の可視化とステアリングにより，プログラムの並
列化がより柔軟に，容易に行えるようになる．
記述から行えることを利用して，問題そのものからプログラムまで
の内容についてのドキュメントの自動生成も可能になる．プログラ
方向は大きく二つに分類することができる．単体のPSEシステム
をウェブなどを通して利用できるようにするネットワーク型
PSE ，もう一つはPSEシステムそのものがサブシステムモ
ジュールに分割され，ネットワーク上に分散している分散型問題解
決環境 である．両者は完全に独立なものではなく，両者がミッ
クスした分散・ネットワーク型PSEもある．
上した．一つは，PSEシステムの機能が高くなるに従い利用方法
が煩雑になり，ユーザのPSE利用方法の学習が難しくなってきた
こと，もう一つは，PSEシステムの開発とメンテナンスに多大な
時間と労力がかかるようになったことである．第一の問題を解決す
るために，PSEに使いやすいユーザインタフェースを付ける工夫
がなされ，さらにこのインタフェースをウェブから利用することが
考案された．PSILABのウェブ版であるPSIWEBや
FEELWEB ，ELLPACK ，CAPSE なども，ウェブから使
いやすいインタフェースを利用できる．もう一つの問題である
PSE開発にかかる多大な労力を軽減するため，PSEを機能ごとの
モジュールに分割する工夫が提案された．これによりモジュールご
との開発とメンテナンス，さらに過去のPSE資産を活用する道筋
また，プログラムの自動生成が基礎式や解法などの抽象的な問題
ミングおよびドキュメンテイションからの解放を目指している．
