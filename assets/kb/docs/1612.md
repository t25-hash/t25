# 12・4・3 設計解のコーディング

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・4 遺伝的アルゴリズム

実世界の生物では、個体の設計図は細胞の中にあるDNA(deoxyribonucleic acid、デオキシリボ核酸)に書き込まれてい6―173る。そこでは、形の異なる四つの分子、A(アデニン)、G(グアニン)、C(シトシン)、T(チミン)が二列に並んでおり、これらのATGCの塩基配列が、生物の形や機能を作るタンパク質の設計図となっている。ここで重要なことは、生物の形や機能という表現型の情報の基になっているものは、一つのタンパク質の設計図である塩基配列であり、これが遺伝子と呼ばれている。この概念を工学的に実現させたものがGA であり、基本的なGAでは連続変数はバイナリコーディング、すなわち2進数に変換される。この場合、通常のバイナリコード(binary code)を用いるか、あるいはグレイコード(gray code)を用いるかの選択がある。例えば、図12・16の親1がバイナリコーディングされた遺伝子情報だとすれば、その表現型は1263である。バイナリコードでは、例えば7=0111と8=1000のように、10進数で隣り合う数でもビット配列が大きく異なることがあり、これが突然変異や交差に悪影響を与えるとして、グレイコードを用いることも多い。グレイコードでは、10進数で隣り合う数のハミング距離(異なるビットの数)が1になるように設計されており、これによってバイナリコードで生じる問題のいくつかが解消できる。ただし、グレイコードでは、隣り合う数のハミング距離は1であるが、ハミング距離が1である二つの数がすべて隣り合うわけではなく、このため、突然変異で一つのビットが変化すると、バイナリコードと同様、表現型の数が大きく変化することがある。連続変数を2進符号化することは、いわば離散的表現に変更していることに相当する。このため、こうした考え方をとらずに、遺伝子型の情報においても連続値をもたせる方法もある。これが実数値GA(real-coded GA)である。連続変数をもつ連続最適化問題をGA で解く場合、バイナリコードを用いて離散化する必然性は特にない。強いていえば、生物の遺伝子情報が離散的であること、設計変数をできるだけ小さなブロックに分割することで、GA における積み木仮説が有効に働くという期待があるからである。積み木仮説(building-block hypothesis)とは、GA において最適解の染色体を多数の部分に分割したとき、各部分(これが積み木と呼ばれる)をもつ個体の平均適合度が増加し、したがって、適合度が高い個体は部分解を有しており、これが交差によって組合わされ、進化し、最適解が得られるという仮説である。このため、設計変数をできるだけ小さな部分で表現することにより、小さな部分解が見つかるということになる。例えば、自然数を2進数で表現し、もし1けためが0であるのが良好という傾向が得られれば、それは、解は偶数であるということにつながる。これは実数値GA では得られない結論である。
