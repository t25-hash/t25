性能化を図る方式である．複数の命令は同時にフェッチ，デコード
され，そのオペランドの依存関係がチェックされる．依存関係がな
いことが確認されれば，命令は複数の実行ユニット（データのロー
ドストアユニット，整数演算ユニット，浮動小数点ユニット，マル
チメディアユニット等）に送り込まれ並列に処理される．
この通常のスーパースカラプロセッサが処理できる1サイクル当
たりの命令数IPC（instruction per cycle）は4～6であるのに対し
て，アドバンストスーパースカラでは，
･多様な分岐予測
･大容量トレースキャッシュ
･大容量予約ステーション（約2000命令を一時保管）
･多数の最適化パイプラインユニット（24～48個）
･十分なオンチップデータキャッシュ

.トレースプロセッサ（trace processor ）方式 条件分
岐を含まない一連の基本ブロックのグループを，トレースと呼ぶ．
現在のプロセッサの実効単位は命令であるのに対し，トレースプロ
セッサはトレースを単位とする．トレースプロセッサは，そのトレ
ースレベルの並列性を見出し，その並列処理により高性能化を図る
方式である．命令の前処理段階でシーケンシャルコードからトレー
スが作り出され，トレースキャッシュに保管される．その後，トレ
ースをグローバルおよびローカルレジスタをもつ複数のPE
（processing element ，スーパースカラと同種）に送り出すことに
より，並列に処理を実行する．トレースレベルの並列処理に加え，
各PEはスーパースカラであるので，PE内でもILPの並列処理が
期待される．トレースプロセッサではトレースの予測が必要であ
り，そのため，他のプロセッサ同様非常に複雑な分岐予測機構を採
用する必要がある．ブロックダイアグラムの例を図13･45に示
