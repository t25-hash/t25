# 12・10・1 基本概念

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・10 オブジェクト指向

構造化プログラミングが、問題解決のためのアルゴリズムをモジュール化して、ソフトウェアの再利用性を高めたのに対し、オブジェクト指向では、オブジェクト(object)というものを単位として、再利用性を図っている。ソフトウェア設計の基本概念が異なることが、オブジェクト指向の理解を難しくしている面がある。a. オブジェクトとクラス従来のアルゴリズム中心の設計法では、問題解決の方法を詳細に検討し、タスクの分類と関連を完全に行った後に実装(プログラミング)段階に移る必要があった。しかし、人間の思考はもっと抽象的な概念を用いることによって、より複雑なシステムを効率的に検討することが可能である。オブジェクト指向は、この自然な思考方法をソフトウェア開発に取り入れたものである。ここで、オブジェクトはものであって、人間の思考対象の単位となっており、それ自身の性質・状態をもち、外界からの刺激によって状態を変化させることができる。この性質・状態を属性と呼び、他オブジェクトからの刺激をメッセージ、それに対するふるまいを定義したものを、操作あるいはメソッドと呼ぶ。オブジェクト指向ソフトウェア開発では、システムに関連するオブジェクトの発見と、オブジェクト間の関連を定義することが、最も重要になる。クラス(class)とは、オブジェクトを生成するための型となるものである。生成されるオブジェクトは、インスタンスとも呼ばれる。クラスには、オブジェクトの属性と操作がすべて定義されている。プログラム開発はこのクラスを作成することにほかならない。再利用を前提にしたクラス定義群をクラスライブラリと呼ぶ。b. データ隠ぺいオブジェクト内のデータは、基本的には、オブジェクトで定義されたメソッドからしかアクセスできない。これをデータ隠ぺいと呼ぶ。この機構により、他のオブジェクトが不当にデータを破壊することができなくなり、プログラムの堅牢性が増す。さらに、オブジェクトの外からの見え方(インタフェース)と実装を分離することができ、より効率的なデータ構造やアルゴリズムを、他の部分を変更せずに採用することができる。このようなクラスを抽象データ型(abstract data type)と呼び、開発・テストがクラス単位で行えるため、開発コストの低減化が期待できる。c. メッセージパッシング上記のデータ隠ぺいの機構により、オブジェクト内のデータは直接変更できない。そのオブジェクトに何らかの機能を実行させたいときには、メッセージを送ることによって、機能の実行を依頼することができる。これをメッセージパッ(c) 性質・動作の追加(a) 概念の包含関係(b) 継承関係シングと呼ぶ。オブジェクトは他のオブジェクトにメッセージを送ることにより、処理を依頼することができるのである。オブジェクトは正しいメッセージにのみ対応して処理を行うので、不正なアクセス、オブジェクトの想定外のふるまいを極力少なくすることができる。同じメッセージでも受取るオブジェクトが異なれば、異なる処理が行われる。これを多態(ポリモフィズム、polymorphism)と呼ぶ。多態は、複雑なシステムのインタフェースを単純化するのに効果的である。d. オブジェクトの関係オブジェクト間の関係は、継承、依存、関連の三つに分類される。. 継承(inheritance)継承はis-a関係とも呼ばれ、二つのオブジェクトの概念に包含関係が存在する。図12・38では、オブジェクトA がオブジェクトB の概念を含んでいる。したがって、B はA をより特化したものであるといえる。逆にB からA の向きは汎化(generalization)と呼び、A はより抽象度の高い概念となっている。実装レベルでは、親となるオブジェクトの性質や動作について、子に当たるオブジェクトではプログラミングを行う必要がないため、新しく増えた部分のみ実装すればよい。これを差分プログラミングと呼び、ソースプログラムの再利用による開発コストの低減と高信頼性を実現している。継承関係にあるオブジェクトどうしには、同じ名前のメソッドが存在するが、異なる動作をしてもよい。これを継承による多態という。. 依存(dependency)あるオブジェクトが他のオブジェクトを使用している関係で、use-a関係とも呼ばれる。操作を行うときのパラメータとして他のオブジェクトを使う場合がそうで、パラメータとして使われたオブジェクトに変更が加えられると、それを使用しているオブジェクトにも影響が及ぶ。. 関連(association)オブジェクト間に包含関係はなく、何らかの関連を有するもの。関連の方向は一方向でも双方向でもよく、それぞれのオブジェクトの役割(role)を指定できる。特に、オブジェクトの内部に他のオブジェクトを含み、全体と部品の関係にあるものを集約(aggregation)といい、has-a関係とも呼ばれる。さらに、部品に当たるものが一つのオブジェクトだけに属するときは、複合(composition)関係と呼ぶ。
