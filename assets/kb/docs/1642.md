# 12・10・2 ソフトウェア開発手順

> **章**: 第12章 インテリジェントシミュレーション
> **節**: 12・10 オブジェクト指向

a. ラピッドプロトタイピング(rapid prototyping)とらせん型開発オブジェクト指向アプローチでは,オブジェクトという概念を導入するだけでなく,開発手順も異なっている.従来の手法では,ソフトウェア開発サイクルの分析・設計・実装が,それぞれ各段階で完成してから次の段階に進む方法がとられていた.途中でのシステム設計変更が非常に難しいからである.オブジェクト指向を用いれば,オブジェクト間の依存関係が少なくなるため,インタフェースだけ最初に決定すれば,内部の詳細が未実装であっても,全システムに組込むことが可能である.部分が未完成であるから,非常に素早く全システムの概要を知り,問題点を把握することができる.この手法をラピッドプロトタイピングと呼ぶ.このシステムをテストすることにより,最初の設計の問題点を洗い出し,システムの再設計を行う.このように,開発において,同じ要素を何度もブラッシュアップしながら完成に近づけていく方法を,らせん型開発(spiral development)という.b. オブジェクト指向分析(OO analysis)上記の一段の開発(a) クラス定義(b) 継承(d) 集約(c) 依存図12・39クラス図の基本構成要素図12・40 ユースケース図の表記法過程の最初の部分が分析である.これは,問題の理解を目的とする.この過程で,問題にかかわるオブジェクトの洗い出しとその関係が,概念モデルとして構成される.c. オブジェクト指向設計(OO design)次に,クラスの設計に入る.ここで,属性・操作を定義し,オブジェクト間の関係を詳細なものにする.d. 実装(implementation)最後が実装である.このとき初めてプログラミング言語を扱う.しかし,必ずしもオブジェクト指向言語を使用する必要はない.オブジェクト指向のために新しく用意された機能が必要なければ,従来の言語を利用してもよい.
