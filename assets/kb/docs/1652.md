# 13・2・1 並列処理(parallel processing)の重要性

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

計算機ハードウェアの本来有する演算性能は、理論性能、カタログ性能、あるいはピーク性能などと呼ばれる。これに対し、実際に数値計算を行って測定される演算速度は実効性能と呼ばれ、多くの場合、理論性能よりもかなり低い値となる。これまで、計算機ハードウェアの性能向上は、プロセッサのクロック高速化とアーキテクチャにおける並列処理によって達成されてきた。クロック高速化は、半導体技術と高密度実装技術の高度化によるものであり、一方、並列処理はベクトル計算、命令の並列実行、マルチプロセッサ化などで実現されている。特に近年の計算機システムにおいては、そのトータルな性能の向上にはクロック高速化よりも、これら並列処理が大きな寄与を果たしていることに注意しなければならない。ソフトウェアの側から考えると、一般に計算プログラムに現れる手続きには、順を追って逐次的に処理しなければならない部分と、並列に同時計算できる部分とに分けることができる。例えば、図13・19(a)に示すDOループは、配列A に関して前のループの演算結果が必要なため、インデックスJ について並列処理することはできず、逐次的に実行しなければならない。これに対し、図13・19(b)の例では、インデックスJ について独立に計算できることは明らかであり、並列処理が可能である。クロックの高速化に伴い、逐次処理を含めすべての処理は基本的に高速化される。しかしながら、理論性能には並列処理がより大きな寄与を果たしているということを考えると、コンパイラや計算プログラムがハードウェアの有する並列性を、実際の計算でうまく引出せるということが重要だとわかる。すなわち、プログラムやデータを分析して、処理の逐次性や並列性を判断しつつプログラムを改良し、理論性能と実効性能とが乖離しないような工夫が必要である。並列実行が可能なように演DO J=1,NA(J+1)=A(J)+B(J)END DO(a) 逐次的に処理しなければならないDOループの例DO J=1,NA(J)=B(J)+C(J)END DO(b) 並列に処理できるDOループの例算順序やデータ配置を並べ替えたり、並列化支援の通信ライブラリや並列処理・ベクトル処理の指示文を挿入するなどのプログラム修正作業は、最適化あるいはチューニングと呼ばれる。
