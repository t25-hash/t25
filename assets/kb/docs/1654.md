# 13・2・3 アプリケーションレベルにおける並列処理

a. 並列化の対象プログラムを作成するときに,計算処理の流れをフローチャートで整理することが多いが,その標記法の中に並列処理を司る記号は見出せない.このことは,プログラムの基となる計算アルゴリズムは,通常,逐次処理のみから成ることを意味(a) 共有メモリ型(b) 分散メモリ型(c) 共有・分散メモリ型図13・21 並列計算機におけるネットワーク,メモリ,プロセッサの構成6―19 8計算力学している.したがって,開発されるアプリケーションプログラムは,最初は逐次処理のプログラムであるといえる.プログラムが実際に計算機上で稼働するには,コンパイラによって,それぞれのハードウェア環境に適応したロードモジュールに変換されなければならない.しかしながら,現在,並列処理に関していえば,コンパイラが並列ハードウェア環境の違いや並列処理をFortranやCなどの高級言語から完全に隠ぺいし,1プロセッサ上で開発された逐次処理プログラムから,並列計算機環境でも効率的に実行できるようなロードモジュールを生成するということはない.すなわち,アプリケーションプログラムのレベルにおいて,何らかの並列化を意識した手続きを加えなければならない.並列化を意識しなければならない対象には,ワーク(あるいはタスク)とデータがある.並列プログラミングとは,ワークやデータをどのように分散し,どのように同時実行するかをプログラムの中で指示することにほかならない.これらはそれぞれ,ワークシェアリング,データシェアリングとも呼ばれる.ワークを分散する形態には,複数のプロセッサでDOループを分担して処理する,複数のプロセッサで異なるプログラムを同時に実行する,などがある.また,分散配置されたノードのメモリ量に応じてデータを分散し,ノードごとに異なるデータに対して同じ演算を施す,などがある.実際にこうしたワークシェアリングやデータシェアリングをプログラムで指示するには,後述のように,メモリ分散方式の違いやプロセッサタイプの違いなどのハードウェア環境に応じて,いくつかの方式がある.例えば,並列化を支援する通信ライブラリ(MPIやPVMなどのメッセージパッシングライブラリ)を選択してそれらをアプリケーションプログラムとリンクさせる,アプリケーションプログラムの中に並列化の指示文(OpenMPなど)やベクトル化の指示文を挿入する,並列言語(HPFなど)を用いる,などである.b. 並列プログラムのタイプメモリ分散方式の違いやプロセッサタイプの違いなどのハードウェア環境に応じて,並列プログラムのタイプは以下の3種類に分類される.. データ並列(data parallel)DOループを分割して,配列要素に対する演算を複数のプロセッサでワークシェアリングする.データシェアリングを意識しなくてよい共有メモリ型並列計算機での並列処理スタイルを指すことが多い.共有メモリではプロセッサ間の通信は必要ないが,用いる並列化支援ライブラリによっては,プロセッサごとの局所変数とプロセッサ間で共通な大域変数とを使い分ける必要がある.リダクション演算(配列要素の積や内積演算など)においては,この点に注意する必要がある.. SPMD(single program multiple data)分散メモリ型並列計算機あるいは分散・共有メモリ型並列計算機では,大規模なデータについてはメモリの物理的な分散に応じて演算対象のデータをあらかじめ分散させ(データシェアリング),複数のプロセッサ(あるいはノード)でそれらを分担して計算実行する.どのプロセッサ(あるいはノード)でも同じプログラムが実行されるが,演算対象のデータはそれぞれに異なるというプログラミングスタイルである.データ並列とSPMD の二つの用語を同義に用いることも多い.. MPMD(multiple program multiple data)MPMDではプロセッサ(あるいはノード)ごとに異なるデータに対し,異なるプログラムが実行される.例えば,複数の物理事象の連成解析においては,境界条件を交換しながら,それぞれの個別の物理事象の解析が異なるプロセッサ(あるいはノード)上で実行される.c. 並列プログラミング方法b項において,並列プログラムはデータ並列,SPMD,MPMD のようなタイプに分類できると述べた.こうした並列プログラムは,以下のような方法で作成される.. メッセージパッシングライブラリ(message passinglibrary)メッセージパッシングライブラリは,分散されたメモリ間でネットワークを介してデータを送受信したり,プロセスの起動や同期などの制御を行う機能をサポートしたライブラリ群である.FortranやCなどのライブラリとして,逐次プログラムの中からコールすることで並列計算が可能となる.ほぼデファクトスタンダードとなっているメッセージパッシングライブラリに,MPI(message 並列化指示文用いる計算機システムのコンパイラが自動並列化の機能を有する場合には,それを利用することができる.しかしながら,すべての逐次プログラムが完全に自動的に並列化されることはなく,多くの場合,ワークシェアリングやデータシェアリングのための指示文(directive,ディレクティブ)をプログラムの中に挿入する.おもに,共有メモリの並列計算機システムにおいて,このような並列化が行われる.共有・分散メモリのノード内並列化も同様である.指示文はプログラムの中で見掛け上コメント行のように書かれるが,コンパイル時にオプションを指定することによって,その指示文が解釈されるようになる.システムによって指示文の形式が異なると,プログラムのポータビリティ(可搬性,移植性)が失われるため,これを統一化した規格がOpenMPである.図13・22にOpenMPの簡単な例を示す.なお,要素プロセッサがベクトルプロセッサの場合,ベクトルベクトル計算に関する指定も指示文の挿入によって行われる.. 計算機システムに対応した並列化戦略複数のプロセッサにメモリをどう割当てるかによって,並列計算機システムには共有メモリ型,分散メモリ型,共有・分散メモリ型の三つの方式がある.また,要素プロセッサのレベルでは,スカラプロセッサ,ベクトルプロセッサの違いがある.以下に,計算機環境に対応して,アプリケーションプログラムのレベルにおける並列化戦略をまとめる.なお,こうした並列処理やベクトル処理を効率的に行うには,あらかじめプログラムやデータを分析して処理の逐次性や並列性を判断し,必要に応じてデータを分割したり,演算順序を並べ替える必要があることに注意しなければならない.. 共有メモリ型個々の計算機システムのコンパイラがサポートする自動並列化のための指示文,もしくはOpenMPを逐次プログラムの中に挿入し,おもにDOループに対する並列化を行う.もしくは,MPIなどのメッセージパッシングライブラリを用いて並列化を行う.. 分散メモリ型MPIなどのメッセージパッシングライブラリを,逐次プログラムの中からコールすることで並列化を行う.もしくは,HPFを用いる.. 共有・分散メモリ型メモリが物理的に分散しているノード間の並列化についてはii項の分散メモリ型の戦略,ノード内の並列化についてはi項の共有メモリ型の戦略を組合わせる.もしくは,ノード間,ノード内の両方についてMPIなどのメッセージパッシングライブラリを用いる.. ベクトルプロセッサ要素プロセッサがベクトルプロセッサの場合,コンパイル時にベクトル化のオプションを指定する.ベクトル化は再内側のDOループに適用される.演算にループ依存性がある場合にはベクトル処理してしまうと誤った結果となるため,コンパイラはベクトル化しない.しかしながら,配列データの順序を並べ替えるなどの工夫によりループ依存性をなくし,ベクトル処理しても正しい結果が得られるようにできる場合がある.このような場合には,指示文を挿入して強制的にベクトル化する.なお,当然のことながら,指示文の仕様はコンパイラに依存する.
