# 13・2・4 領域分割(domain decomposition)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

有限要素法や差分法など、ある領域を対象として解析を行うような場合の並列解析では、領域を部分領域に分割してそれに対応したデータを分散メモリに割当て、各部分領域の計算を並列に実施する。ただし、一般に部分領域ごとの計算は完全には独立ではなく、領域全体の整合性をとるために部分領域間での通信が必要となる。したがって、領域分割に基づく並列処理が効率的に行われるためには、部分領域間で通信ができるだけ少なくて済むように、データが局所化されている必要がある。領域分割の考え方を実装するには、メッシュの段階で領域分割し(b) PE#0が保持する情報(隣接PEとのオーバラップあり)(a) 4領域への分割図13・23 有限要素法データの領域分割例6―19 9て、部分領域ごとの入力データを作成するのが自然である。並列プログラムの様式はSPMD となる。すなわち、各プロセッサに同じロードモジュールをロードし、それぞれの担当する部分領域についてのデータを入力して実行する。その場合、部分領域の入力データには、隣接する部分領域との間の節点や要素の接続情報を入れておくのがふつうである。この接続情報は通信テーブルとも呼ばれ、行列ベクトル積や内積計算において、領域全体の整合性をとるための通信を行う際に参照される。図13・23に有限要素法データでの領域分割様式の一例を示す。メッシュデータを領域分割するには、節点の座標値に基づいて分割する方法、節点の接続関係であるグラフ情報に基づいて分割する方法などがある。さらに、部分領域間で少しオーバラップさせる場合と、オーバラップさせない場合とがある。どの分割様式を選択するかは解析アルゴリズムに依存する。一般に、オーバラップ量が深いほど隣接領域の情報がより伝達されるために、領域全体で反復的なアルゴリズムを適用する場合にはその収束が速い。領域分割のツールとしては、METISが広く用いられている。
