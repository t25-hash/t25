# 13・2・5 オーダリング(ordering)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

ノード内並列やベクトル化は、DOループを分割して同時に計算を行うことで実現される。このため、もしDOループにループ依存性があり、番めの結果が+1番め以降の計算結果に影響を与えるような場合には、並列処理あるいはベクトル処理してしまうと誤った結果となってしまう。しかしながら、配列データの順序を並べ替えるなどの工夫によりループ依存性をなくすことができる場合には、コンパイラに強制的に並列処理やベクトル処理を指示することができる。このような並べ替えをオーダリングと総称する。オーダリングによって、繰返し計算の中で依存性のない演算部が選び出されグループ化される。依存性のない一連の演算部分に対しては、並列計算やベクトル計算が可能となる。DOループの演算が節点や要素についてのループである場合には、依存性の有無は節点(b) 行列のプロファイル(a) 節点番号図13・24 オーダリング前(a) 節点番号(b) 行列のプロファイル図13・25 オーダリング後(2色)6―200計算力学(a) ベクトル処理による時間短縮(a) 並列処理による時間短縮(b) γ=20の場合のスピードアップ図13・27 アムダールの法則(並列処理)図13・26 アムダールの法則(ベクトル処理)や要素の接続関係から判断することができる。例えば、次式のような演算を考える。욡욪욡욅욅(13・10)욡=욡-∑요ここで、添字は節点のインデックスを表す。このような演算は、連立一次方程式の解法など多くの行列演算に現れる。まず、図13・24(a)のような節点番号付けの場合、行列の非ゼロ成分のプロファイルは図13・24(b)のようになり、明らかに節点の演算の際に、それ以前に演算済みの情報が必要であることがわかる。これに対し、図13・25(a)に示すように、依存性のない節点を2色に色分け(黒と白)して番号を付け替えた場合、行列のプロファイルは図13・25(b)のようになる。図13・25(b)から、同一色に属する節点に関する演算は互いに依存性がないことがわかる。すなわち、図13・25(a)のような節点番号付けのもとでは、ノード内並列処理やベクトル処理が可能となる。依存性のない演算を2グループ(2色)に分けるような場合をレッドアンドブラック(red and black)法と呼ぶ。また、より多くの色に分ける場合にはマルチカラー法と呼ぶ。演算に依存性のない節点をハイパープレーンと呼ばれるグループに分類し、各ハイパープレーン上の節点についてノード内並列処理やベクトル処理を行うことも多い。ベクトル化のためのオーダリングについては、ベクトル化が最内側DOループのみについて適用されること、十分な長さのループ長(ベクトルレジスタのサイズよりも大きいこと)が必要であること、ベクトル長は色数やハイパープレーン数によっても変化することなどに注意しなければならない。
