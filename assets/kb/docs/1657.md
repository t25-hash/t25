# 13・2・6 アムダールの法則(Amdahl’s law)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・2 ベクトル・パラレル

プログラムのベクトル化率とベクトル化による実効性能向上の関係、あるいは、プログラムの並列化率と並列化による実効性能向上の関係は、ともにアムダールの法則と呼ばれる。まずベクトル処理の場合について説明する。図13・26(a)に示すように、プログラムをスカラプロセッサで実行したときの計算時間욦のうち、ベクトル処理可能な部分に要した時間の割合αをベクトル化率と呼ぶ。このプログラムをベクトルプロセッサで実行すると、ベクトル処理可能な部分の計算時間は加速率γで短縮されるため、全体の計算時間욋は次式のように表される。욋=1-α욦+α욦γ(13・11)(b) visualization、科学技術計算結果の可視化)と、インフォメーションビジュアリゼーション(information visualization、情報の視覚化)に大別される。エンジニアリングビジュアリゼーション、メディカルビジュアリゼーションのように目的や対象に応じて細分化されることもあるが、これらは、サイエンティフィックビジュアリゼーションとインフォメーションビジュアリゼーションを部分的に組合わせたものである。サイエンティフィックビジュアリゼーションは、技術計算や実験・観測の結果である時空間に及ぶ数値データに基づいて、着目する物体形状や物理量の分布をわかりやすく表現するものである。インフォメーションビジュアリゼーションは、数値データそのものよりも、データの情報性を出発点とする可視化である。両者の違いは、表示する際の座標軸に着目して考えるとわかりやすい。サイエンティフィックビジュアリゼーションでは、可視化対象となる数値データの多くが時空間の関数であることから、二次元、あるいは三次元の物理座標(計算空間や波数空間の場合もある)に対応した表示空間において表現される。一方、インフォメーションビジュアリゼーションでは、物理座標とは対応しないデータ、さらには四次元以上のデータを扱うことが多い。このため、データを選別・加工して、二次元あるいは三次元表示空間に対応付ける必要がある。例えば、身長、体重、胸囲、座高というデータを扱う場合、それぞれの要素を独立変数として扱い、四次元空間を形成する。次に表示空間を踏まえたデータの選別・加工を行う。四つのデータから身長と体重を選び、軸を身長、軸を体重とした座標軸の設定を行うといったことである。その後、散布図などを作成する。このようにインフォメーションビジュアリゼーションは、多次元量の表現手法として位置付けられることが多く、多次元量の視覚化と呼ばれる場合もある。また、情報間の関連性を示す樹木図やワークフロー図、さらには、グラフィックスユーザインタフェースのような表現手法も表示対象にしている。まとめると、データ構造に基づく表示データの選択(多次元量から何かしらの操作、あるいは写像によって表示量を選ぶ)、表示データの抽象化(三角形や四角形などの描画オブジェクトを情報と対応させる)、表示方法(ある領域に対する描画オブジェクトの最適配置、関連性の記述、強調表示)がインフォメーションビジュアリゼーションの構成である。視覚化技術の核は、配置図とグラフ表現である。ただし、サイエンティフィックビジュアリゼーションとインフォメーションビジュアリゼーションは、明確に分離できるものではなく、翼面上の圧力分布のグラフ表現など、両者の要素をもつ融合的な表現も多く存在する。ビジュアリゼーションを行ううえで考えなければならないのは、何のために、何を、どのように、どこで、見えるようにするか、何がわかるのか、ということである。この中で最も重要なことは、何のために、何を見るか、そして何がわかるのかという部分である。これらを支援する方法は提案されているが、解析の対象に強く依存し分野固有の知識が必要になるので、はん用的な支援策は見出されていない。一般には、どのように、どこで、見えるかという部分を熟知したうえで試行を繰返すことになる。そこで、本節では、どのように、どこで、を中心に、時空間の数値解の分布に関する可視化手法について、その概要を示すことにする。なお、インフォメーションビジュアリゼーションの詳細については、文献(56)等を参照してほしい。
