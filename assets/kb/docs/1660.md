# 13・3・2 データの加工と近傍場の決定(何を見るのか)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・3 ビジュアリゼーション

可視化作業を何度か繰返すと、その目的によって何をどのように見ればよいのかがある程度はわかるようになる。しかしながら、未知の対象を扱う場合は、計算データを単にデータとして扱うところから始めなければならない。数値シミュレーションによって直接的に得られるデータは、一次データ、あるいは生データ(raw data)と呼ばれる。生データは、図13・29のように散逸型(scattered data)、構造型(structureddata)、非構造型(unstructured data)の三つの型と、それらの複合型に分けることができる。また、物理座標や波数などの関数として表されるものと考える。はじめに可視化対象となる物理量、あるいは境界形状を決める。生データが可視化対象でない場合はデータを加工して、目的に合ったものを算出する必要がある。実験・観測との違いは、数値微分や図13・28 粒子追跡を中心とした可視化(a) 粒子法(散逸型) (b) 非構造型(c) 構造型図13・29データ構造6―202計算力学図13・30 補助格子数値積分の方法が計算の段階でわかっている点である。定義位置とデータ間の接続情報が付随しているので、ある位置の生データの近傍場が定義できる。例えば、速度場から渦度場のような微分による二次的データの算出は、機械的に行うことができる。ただし、可視化以前の局所的な補間をそのまま利用できる場合は、有限要素法やセミラグランジュ法による数値解の一部を除いて多くはない。データが採取された位置で、そのデータ自身がある分布の代表値を表している場合(散逸型)の多くは、重ね合わせによって近傍場を決定できる。例えば、粒子法によって得られた結果を可視化する場合、粒子の位置に対してその属性を表示する方法以外に、図13・30のような印で節点を示した補助格子を用いて可視化が行われる場合が多い。ここで、は粒子の総数である。補助格子上でデータを再構成できれば、その後の処理は非構造型や構造型と同じとなる。一方、光学系を模擬する場合、分布関数を直接利用する可視化もある。非構造型や構造型の場合は、接続情報によってまわりの点がわかるので、それらの点でのデータから局所補間が決定し、近傍場が求められる。この操作によって、離散的であるデータを局所補間により連続な空間として扱うことができるようになる。ただし、先述したように局所補間の性質が明らかなものは少なく、可視化する側で適当な補間関数を与えて処理を進めることが多い。よく用いられるものはラグランジュ補間である。
