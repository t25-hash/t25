# 13・3・3 可視化操作(どのように見るのか)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・3 ビジュアリゼーション

近傍場が求められ,計算領域全体が近似的に連続空間として扱えるようになると,その分布に基づいて可視化処理が進められる.どのように見るかを考えるうえで,基本的な可視化手法とその特徴を理解しておくことが重要である.数値解の存在する場所(物体表面など特定の部分)の表示を含めると,可視化処理は,・データの存在位置の表示・データの接続情報の表示・等値線(isoline)表示・等値領域表示・等値面(isosurface)表示・ベクトル表示・粒子追跡(particle tracking)表示・グラフ表示・画素ベース,イメージベースの表示法のように分類することができる.a. データの存在位置と接続情報の表示格子図やメッシュ図と呼ばれるものである.計算の前段階以外での単独表示は少なく,他の可視化法との組合せで用いられる.格子に起因する誤差の判定,着目領域の明示,あるいは物体形状そのものの表現として利用される.車の衝突解析など,対象となる物体をそのまま表示することで,必要な情報を得る場合もある.通常は,存在位置と接続情報は対で表示される.ただし,粒子法のようにデータの存在位置そのものが情報となる場合は,存在位置を点や多角形で示し,その大きさや色を用いて属性を示す場合もある.b. 等値線表示,等値領域表示,等値面表示着目するデータの値が等しい点群(等値線,等値面)や,ある範囲に存在する点群(等値領域)を表示することによって,場の様子を調べる方法である.値の範囲を操作することで,場の中の特徴的な領域を切り出すことが可能である.等値線,等値領域,等値面探索には,計算空間全体を対象とする探索アルゴリズムと計算要素単位のものがある.例えば,ある要素内で双一次補間によって表されたにおいて,=となる解を要素内で求め,解を表示するものが要素単位の等値線表示法である.補間関数が複雑になると計算量が大きくなるので,頂点の値から辺上に近似解が存在するか否かを判別し,パターンを調べることで等値線や等値面の探索を高速化する手法が提案されている.三次元では,マーチングキューブ法と呼ばれる六面体を対象としたものがよく用いられている.非構造格子の場合は,計算要素を三角すいに分割し,三角すい内で線形補間を行い,=を満たす平面を切り出す.三角すいの分割方法の任意性が問題となるが,高速であることから,さまざまな分野で利用されている.等値領域表示は,計算要素の頂点の値を色に対応させて要素を色付けする方法と,二つの等値線の間を指定した色で塗る方法の二つがある.後者のほうが定量的な表現となる.等値面表示は一つのレベルであっても,三次元空間における物理量の分布を知るうえで有効な方法であるが,半透明化した等値面を数枚重ねて表示することによって,空間の構造をより明確に示すという試みもなされている.この方法は,f項で後述する画素ベースのボリュームレンダリングと区別して,簡易型ボリュームレンダリングと呼ばれている.c. ベクトル表示速度や渦度などのベクトルは,データの存在位置を起点とする適当な長さの線や矢印によって表される.テンソル量に関しても,射影によってベクトルとして表すことが多い.可視化法としては単純なものである.しかしながら,三次元シミュレーションにおいては,線や矢印の見せ方が難しく,さまざまな表現法が検討されている.また,接続情報を与え,補間によって連続空間を再構成することにより,後述する粒子追跡による可視化を代用する場合もある.ベクトル表示に残された主たる課題は,データの存在位置が視覚に与える影響と重複表示の問題である.これらを解決するために,補助格子を用いた起点の均質化手法などが提案されている.d. 粒子追跡による表示ベクトル場をデータの存在位置を起点とする線分や矢印で表現する方法によって,場全体を示すことは可能である.しかしながら,局所情報の積み重ねとなるので,部分間の関係を明示することが問題となる.また,時間的に変化するベクトル場において,時空間の様子をとらえることが難しい.時空間に及ぶベクトル場全体を表現する方法として,流体解析であれば,流線,流跡線,流脈線表示が用いられている.これらは,あるベクトル場に従って移動する仮想粒子の軌跡の表示として扱うことができる.仮想粒子の位置をとすると,dd=となる.この式を解く場合,ベクトル場の構成方法が問題となる.シミュレーション結果を処理する場合,ベクトル場が計算領域全体で解析的に定義されることはまれであり,多くの場合,計算要素内での補間によって表される.このため,積分する場合に,(1) グラフ表示一般のグラフ表示以外に,翼形状を基準とした圧力係数の分布図や図13・31のような二次元円柱を過ぎる衝撃波のcarpet plot,あるいはfunctional plotと呼ばれる方法が用いられることがある.これは,データの存在位置(空間分布)と大きさを,グラフによって同時に表現するもので,現象の直感的,かつ定量的な理解にとって重要である.ただし,この方法では視点の違いによって解析者に異なる印象を与えるため,定量性を重視する場合は,右図のように等値線を重ねて表示する.f. 画素ベースの可視化法ここまでの可視化処理をまとめると,可視化は数値解の中から特定の部位を抽出し,幾何データに変換し,表示するものとみなせる.格子面上の等値領域表示を具体例として,少し詳しく説明する.シミュレーションで得られた結果は,格子の節点,または中心に存在している.これらのデータの近傍は,シミュレーション手法によって決まる補間関数,または近似6―203補間関数によって表される.次に,ある値の範囲にある幾何データが切り出され,切り出された幾何データは,三次元空間で回転,拡大などの視点変換が行われ,最終的には画素空間に写像され,表示装置に可視化結果が現れる.データに着目すると,a~e項までの可視化は,生データ(離散データ)→補間による連続データ→幾何データ→画素データ(ピクセル,ボクセル)で構成される.つまり,最終的には画素と呼ばれる点群で,可視化結果が表現されることに注目すれば,表示空間から見た可視化は,あるデータからある規則に従って,表示装置上のグループ化された点群を拾い出す作業と考えられる.一方,画素をベースにした可視化法は,直接的に画素を操作する可視化法である.属する方法として,ボリュームレンダリング(volume rendering)やLICがある.CG において提案された方法がほとんどであり,可視化対象となる計算データの規模が大きくなると,可視化処理に要する計算時間が膨大になる.また,定量性や可視化誤差の問題がある.このため,技術計算データの可視化に用いる場合は,プレゼンテーションなどの用途に限られていた.しかしながら,パーソナルコンピュータの高速化やアルゴリズムのハードウェア化によって,高速な可視化処理が可能となり,適用例は増えている.他の可視化手法と異なる効果を得るという点でも,いくつかの提案がなされている.例えば,粒子密度に対するボリュームレンダリングによる可視化によって,粒子が集中する部分が明確に識別できる.これを等値面によって可視化すると,等値レベルによっては面どうしが干渉し,空間の構造の把握が難しいことがわかっている.ただし,ボリュームレンダリングにはいくつもの可視化誤差が存在し,1枚の図から三次元構造を定量的に表し得ないことが知られている.意味のある結果を得るためには,他の可視化手法を併用することが望ましい.画素ベースの可視化の利用には,他の可視化手法で同等の結果を導けることが多いので,注意が必要であるが,表示装置の表示領域全体に対する情報を提示できるという性質は活用すべきである.例えば,雲は気流を表現するトレーサとしても用いられるが,雲のない領域での流れの状態を知ることは,流れ場を把握するうえで重要である.画素ベースの可視化は,表示領域においてある可視化法では可視とならない情報を,いかに提示するかという課題の一つの解答を与える.表示領域内であれば,情報をあますことなく提示できる可視化法を全体可視化と呼べば,ボリュームレンダリングは,スカラ場に対する全体可視化法と位置付けることができる.ベクトル場を,ベクトル表示や粒子追跡によって可視化すると,計算格子,粒子の発生点,重複表示に起因して,情報のあるところとないところができる.ベクトル場における画素ベースの可視化法であるLICを用いることで,全体可視化が可能になる.LICはコンピュータグラフィックスの分野で提案された方法であるが,原理が流れの可視化法と似通っていたために,すぐにシミュレーション結果の可視化に応用された.ただし,LICに属するほとんどのものが,速度場を画素空間に写像した後に処理を進めるために,可視化誤差が生じてしまう.また,他の可視化法との整合性の問題も生じる.これらの問題を解決するためには,つねに物理空間(計算空間)と画素空間の対応を考える必要がある.シミュレーションや他の可視化法との整合性を考え,可視化精度を高めた方法の一つがピクセル露光法(PEM : pixel exposure method)である.なお,処理をすべて画素空間で行うものは,イメージベースの可視化と呼ばれることが多い.画素ベースの可視化法のすべてがイメージベースの可視化ではなく,ピクセル露光法のように画素空間と計算空間の両者を使うものもある.
