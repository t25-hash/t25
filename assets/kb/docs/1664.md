# 13・3・5 可視化環境の多様化(どこで見るのか)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・3 ビジュアリゼーション

1990年代初期から,ビジュアリゼーションは画素ベースの可視化法の研究を除き,効率化と使いやすさを主眼として発展した.システムとして見た可視化は図13・33のようなものである.図中の可図13・32 限界流線表示に基づいた三次元流れ場の階層的可視化視化法の指定の段階では,可視化対象となるデータは計算データそのものである.対象領域の指定によってデータの選別が行われる.対象領域,処理法が決定すると,計算データは直交等間隔格子(場合によってはボクセル空間)上に写像されるか,対象を絞られた可視化のためのデータとなる.ボクセル空間上に可視化データが展開された場合は,直接的なボリュームレンダリングが可能となる.一方,多くの場合,可視化対象となるデータは,興味のある領域(ROI)の計算データそのものである.ここから,可視化手法に従って処理が進む.ある場合は表示座標系の情報を先読みして画素空間(pixel space,ビットマップ空間)に,別の場合は点,線などの幾何データとなる(幾何データの場合,この時点では表示座標系の影響を受けていないことに留意されたい).画素空間に写像された可視化データはそのまま表示されるか,表示座標系に従いテクスチャとして指定領域上に張られる.幾何データは表示座標系に従い,ラスタ変換の後にスクリーンに表示される.このラスタ変換はハード処理される場合も多い.計算環境が複雑になると,これらの処理をどこで行うと効率的なのかといった問題が生じる.例えば,シミュレーションの計算部に可視化処理を組込む場合にも,表示に際して,幾何データを取出すか,ラスタ変換後のデータを取出すかで,可視化システムの特色が生じる.近年では,ビジュアルコンピューティングや,リアルタイムビジュアリゼーションなどのシミュレーション計算と,可視化を同時に行う方法が定着しつつあるので,この選択が課題となっている.データが巨大化すると,一つの計算機上にデータをおくことができなくなるために,分散並列環境における可視化が必要になる.可視化の並列化において大きな問題は,陰線・陰面処理である.陰線・陰面処理の並列化は,分散するデータに対して等値面を表示する場合等に必要となる.通常は幾何データをある計算機に移行し処理されるか,計算機ごとにラスタ化され,その後,一つにまとめるという方法がとられる.ラスタデータは通信量が一定となることが有利ではあるが,各計算機のデータ自身が奥行方向にソートされていないと,正しく奥行情報を反映させることが難しい.可視化結果は,最終的にはモニタやプロジェクタなどによって表示される.現在の可視化システムの多くが,1980年代の表示装置に基づいて設計されており,近年のシミュレーションの高度化に十分に対応できているとはいえない.そこで,高臨場感のある表示や高解像度の可視化画像を表示するための装置や,表示方法が検討されている.
