# 13・4・1 問題解決環境(PSE)とプログラム自動生成

> **章**: 第13章 シミュレーション支援法
> **節**: 13・4 問題解決環境

a. PSE の概説PSE(problem solving environment)は歴史的に,PDE(partial differential equation,偏微分方程式)問題の解決を支援するソフトウェアと位置付けられる.PDEは空間的な広がりをもつ物理問題を扱うことから,計算時間の面でも記憶領域の点でも挑戦的な問題である.したがって当初は,ソフトウェアの生産性や再利用性よりも性能が重視された.またPDE問題は定図13・33 可視化処理の全体像式化の自由度が大きく,また個々の問題に応じた個別の工夫も必要とされることから,本質的に標準化が困難な面がある.しかし,これらの性格が再利用への強い要求を生むこととなり,計算機性能の向上とともに,1960年代に入ってから生産性と再利用性の向上に向けた取組みがなされるようになった.後にPSEと呼ばれるようになるPDE問題解決に向けた生産性・再利用性向上への取組みは,利用者とのインタフェースの取り方により次の3種類に分類される.第1は,ある程度のはん用性と技法の一貫性を備えたアプリケーションソフトウェアの開発である.構造解析分野におけるNAS-TRAN(1971)は,その典型と考えられる.第2は共用性の高いソフトウェア部品を,ライブラリまたはモジュールとして用意しておき,それを用いてアプリケーションを組立てる方法である.ライブラリの機能としては,形状入力,メッシュ分割,離散化,行列変形,線形計算,結果出力などが挙げられる.利用者のインタフェースとして,特定の記述言語を用意しておいて利用者に問題記述を行わせ,そこからライブラリ呼出しコード等を自動生成するタイプのものと,利用者が既存の言語インタフェースを介して,直接ライブラリコードを呼出すタイプのものがある.前者の例としては,SALEM(1968),PDEL(1970),ELLPACK(1977),DEQSOL(1985),NCAS(1993)などが挙げられる.このうちDEQSOLとNCAS については,b項,c項にて説明する.後者の例としては,双曲形方程式のためのDCG(1975),HYPACK(1978),放物形方程式のためのPDEPACK(1975),DSS/2(1976),MOL 1D(1976),PDETWO(1976),POST(1977),DISPL(1977),LSQPDE(1978),FORSIMVI(1978)など,だ円形方程式のためのEIGEN(1974),ITPACK/REGION(1977),EPDEI(1977),GENEPI(1977),FISH-PACK(1978)など多数が挙げられる.第3は,既存のプログラミング言語を拡張して,利用者に機能を提供する方法である.PDELAN(1972)は,FORTRAN 言語に有限差分解法を記述するためのメッシュ宣言,分布量宣言,差分演算定義の機能を追加している.PDELAN による記述は,プリプロセッサにより通常のFORTRAN に展開される.上記を含む1970年代のPSEについては,文献(78)に包括的なサーベイがある.b. DEQSOLDEQSOL(differential equation solver)は,おもにベクトル型スーパーコンピュータ用高性能コードの自動生成を目的としたPSEである.利用者向けの高位記述に基づいて,FORTAN の計算コードを自動生成する方式をとっている.あらかじめ用意されたライブラリは線形計算ソルバのみであり,この点ではライブラリのドライバの性格が強いELLPACK とは異なる設計となっている.これは,生成コードの自動ベクトル化の容易さをねらったためである.DEQSOLによる問題解法記述は,離散化手法,領域形状,メッシュ分割,変数/定数宣言,部分領域宣言,境界条件,初期条件,解法手順などのセクションから成る.言語仕様上の特徴としては,数理モデルと計算スキームの分離記述,簡潔な微分記法,領域形状の言語定義などが挙げられる.機能面では,二次元,三次元空間内の2階の微分を含む定常,非定常連立方程式系が扱える.非線形問題は,線形化計算スキームを工夫することにより対処する.離散化手法としては,有限差分法とガラーキン式有限要素法の両者を提供しており,後者の場合要素の種類は,三角形,四辺形,四面体,六面体を基本とする線形要素と二次要素である.境界形状は,多角形や多面体に加え,スプライン曲線やスイープ曲面を取扱うことができる.線形解法として,ガウス消去による直接解法とICCG(incomplete Cholesky conjugate gradient)/BCG(bi-conjugate gradient)系の反復解法を用意している.半導体の不純物拡散シミュレーション,LSIチップの冷却シミュレーション,ディスクヘッド周りの三次元磁場解析などの適用事例があり,対FORTRAN プログラム記述行数で10分の1以下,スーパーコン6―205ピュータ向けベクトル演算化率90%以上を達成している.c. NCASNCAS では数学モデルからC言語のシミュレーションプログラムを生成支援する際,問題記述からプログラムの設計・生成されたプログラム自身に至るまでの過程を可視化し,ユーザによる変更・修正(ステアリング,steering)を可能にしている.この意味でNCAS は,ホワイトボックス型PSE(whitebox PSE)と呼ばれる.シミュレーション過程のすべてをユーザに可視化して開示し,修正変更をも可能にし,ユーザの意図を反映したプログラムの生成支援を行う.多次元流体プログラムなど,5000行程度のプログラムの生成が実現されている.P-NCAS では並列プログラムの自動生成にも対応している.NCAS の設計思想の裏には,PDEで記述できる問題をコンピュータでシミュレーションする場合,扱いたい問題に全面的に集中したいのに,なぜ人間が機械的な作業であるプログラミングを行わなくてはならないのか,現在のコンピュータは本物のコンピュータに進化しきれていないのではないか,という疑問があった.同時に,差分法や有限要素法によってプログラムを作る際には共通点があり,PDE問題領域に限れば,プログラミングのある程度の機械化,すなわち自動プログラミングが可能であることもわかった.こういった背景のもとNCAS は,問題の本質の抽出などの人間が得意なところは人間が分担し,コンピュータが得意なところはコンピュータに任せることを設計思想としている.P-NCAS では並列プログラムの設計過程の可視化とステアリングも実現した.計算領域の領域分割により並列化をサポートし,並列モデルにはSPMD モデルを採用した.メッセージパッシングにはMPI関数を用いた.計算領域の分割は領域の可視化ウインドウにより,可視化された領域を見ながら行うことが可能であり,領域分割方法・領域分割数を可視化ウインドウで確認しながら変更し設定できる.領域分割に関する設定の後,NCAS 内でプログラムが設計され,設計されたプログラムはPAD(problem analysisdiagram)図により可視化される.PAD 図の可視化ウインドウでも,並列パラメータの変更・確認を行うことができる.プログラミングや並列化過程の可視化とステアリングにより,プログラムの並列化がより柔軟に,容易に行えるようになる.また,プログラムの自動生成が基礎式や解法などの抽象的な問題記述から行えることを利用して,問題そのものからプログラムまでの内容についてのドキュメントの自動生成も可能になる.プログラミングおよびドキュメンテイションからの解放を目指している.
