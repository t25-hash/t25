# 13・4・2 分散型問題解決環境(distributed PSE)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・4 問題解決環境

問題解決環境(PSE)システムの研究開発の流れは、ネットワークを介して利用するPSEの研究開発に向かっている。その方向は大きく二つに分類することができる。単体のPSEシステムをウェブなどを通して利用できるようにするネットワーク型PSE、もう一つはPSEシステムそのものがサブシステムモジュールに分割され、ネットワーク上に分散している分散型問題解決環境である。両者は完全に独立なものではなく、両者がミックスした分散・ネットワーク型PSEもある。現在までのPSE研究により、過去のPSEシステムの問題点が浮上した。一つは、PSEシステムの機能が高くなるに従い利用方法が煩雑になり、ユーザのPSE利用方法の学習が難しくなってきたこと、もう一つは、PSEシステムの開発とメンテナンスに多大な時間と労力がかかるようになったことである。第一の問題を解決するために、PSEに使いやすいユーザインタフェースを付ける工夫がなされ、さらにこのインタフェースをウェブから利用することが考案された。PSILAB のウェブ版であるPSIWEBやFEELWEB,ELLPACK,CAPSEなども、ウェブから使いやすいインタフェースを利用できる。もう一つの問題であるPSE開発にかかる多大な労力を軽減するため、PSEを機能ごとのモジュールに分割する工夫が提案された。これによりモジュールごとの開発とメンテナンス、さらに過去のPSE資産を活用する道筋6―206計算力学が開かれた。この方向で研究されているPSEには、分散型NCAS等がある。PSE研究は幅広い分野をカバーし、シミュレーション支援分野では、プログラムの自動生成を目指すPFE(programming freeenvironment)、熱流体や構造解析等によく使われるCAEパッケージや、専用計算プログラムの利用の支援を目指すPSE、可視化支援や式変形支援などのPSE機能の一部を実現するものなどがある。さらに、利用支援の中でもナビゲーションを行うPSE、専門的知識を知識ベースとして提供し支援を行うもの、利用マニュアルをデータベース化して利用支援を行うものなどがある。教育の分野でのPSEではe-Learning を支援することを目的に、e-Book 作成支援やe-Book 検索・履歴メモ支援なども行われている。以下では、PSE研究の中心を占めるシミュレーション分野での分散型問題解決環境について述べる。a. PSE モジュール構成シミュレーション支援PSEにおけるモジュールは、シミュレーションのプロセスに対応して、おもに以下の構成要素から成る。解決すべき工学問題から、本質となる物理モデルの生成支援を行う物理モデル化支援、問題記述支援、物理モデルから数学モデルを導出支援する数学モデル化支援、基礎式をコンピュータで解けるようにするための離散化支援(離散化支援は、数学モデルから離散式を導出する離散化支援と、空間の離散化支援とから成る)、解析プログラム設計支援、プログラム生成支援、シミュレーション実行支援、得られるデータの解析支援、可視化支援、解くべき問題と作成されるプログラムのドキュメンテーション支援が、中心となるモジュールである。PSEによりこれらすべてを備えた統合型PSEもあるが、計算空間を離散化するメッシュ生成モジュール、式の変形モジュールのみやマトリックス生成支援のみを行うもの、設計されたプログラムの流れをPAD などの図で可視化するもの、計算・解析データを可視化するものなど、さまざまなPSEが存在する。b. 分散型問題解決環境(distributed PSE)PSEの機能の一つあるいは複数のモジュールに限って開発を行うことにより、複数の開発者による開発が可能になり、メンテナンスの労力が軽減されると同時に、過去のPSE資産あるいは今後に開発されるモジュールを再利用しやすくなる。このことを目指して分散型のPSEが研究開発されている。以下にはその一例を述べる。上記a項の各モジュールを1台あるいはネットワークで結合したコンピュータ上におき、一つあるいは複数のPSEシステムを構成している。分散型問題解決環境の概要を図13・34に示す。分散型問題解決環境を利用する方法はいくつかあるが、各PSEモジュールの機能と存在位置をPSEサーバに事前に登録しておき、問題を抱えるユーザに問題を解決する手順をナビゲーションする方法が簡便である。モジュール間あるいはモジュールとPSEサーバとの通信には、XML(extensible markup language)などの公開規格を用いると、モジュール間のインタフェースを統一することができる。過去に開発されたモジュール等の場合では、独自のインタフェースをもっている場合がある。この場合には、そのモジュールの独自のインタフェースとXMLなどの統一された規格との間の、マッチングソフトウェアを用意する必要がある。問題記述モジュールに解くべき問題を記述し、あるいは支援モジュールにより解法を得て問題記述を完成させれば、その結果をXMLファイルとして保存する。そのXMLファイルを次の離散モジュールに渡し、結果をXMLファイルとして得る。式変形モジュールやプログラム設計モジュール、プログラム生成モジュール、ドキュメンテーション支援モジュール、実行支援モジュールなどと順に情報を伝え、ユーザは問題を解決することができる。各機能に対して複数のモジュールが存在する場合もある。ユーザが選択しやすくするため、各モジュールの特徴を記述しておく必要がある。分散型問題解決環境により開発とメンテナンスの負荷が軽減されること、過去の資産が有効活用できることから、PSE研究に新たなステップを与えるものとなった。c. ネットワーク型PSE(network linked PSE)上記のようにすでにいくつものPSEシステムが開発されてきており、それらを利用する側からの利用の容易さを求められている。要求に応える一つとして、ネットワーク型PSEの開発研究も盛んである。おおむねウェブからネットワークを通して各PSEを利用できるように設計開発している場合が多い。この研究の方向はおもに、複雑で大規模になってきているPSEシステムに対して、使いやすいインタフェースを開発し、利用を促進することをねらっている。企業側ではシミュレーションの身近なサービス化をねらって、携帯端末からでもPSEを利用できるシステムを開発している。PSEの機能を限定し、シンプルな利用ガイドを用意し、シミュレーションを一般に浸透させようとする試み、マニュアルや障害相談データベースを検索利用支援するPSE、科学技術分野では並列プログラム生成支援、並列支援、大規模シミュレーション実行支援など、多くの方向で発展している。
