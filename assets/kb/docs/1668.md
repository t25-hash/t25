# 13・4・3 アプリケーションベースの問題解決環境(PSE)

> **章**: 第13章 シミュレーション支援法
> **節**: 13・4 問題解決環境

アプリケーションベースのPSEは、科学・工学分野のシミュレーションプログラムをコンポーネントとして統合化するため、しばしば統合化環境と呼ばれる。統合化の単位は、あくまでも完結した実行形式プログラムである。コンポーネントには、ファイルを入力してファイルを出力する伝統的な処理形態のアプリケーションを統合化環境とのインタフェースを受持つコードで、ラッピングすることにより統合化するレガシーコンポーネント(図13・35)と、統合化環境が提供するAPIを直接使ってオブジェクトを直接入出力する、初めから統合化環境用に書かれたネイティブコンポーネント(図13・36)の2種類がある。どちらのタイプのコンポーネントも、一つないしは複数のオブジェクトと呼ばれる抽象化されたデータを入力して何らかの処理を実行し、オブジェクトを出力する。このようにすべてのアプリケーションプログラムおよびデータを、それぞれコンポーネントおよびオブジェクトとして抽象化して取扱うのが、アプリケーションベースのPSEの基本的な考え方である。したがって、ユーザは個々のコンポーネントの起動方法を覚える必要がなくなり、GUI(graphical user interface)上でコンポーネントのアイコンを選択して起動し、さらに、ダイアログに入力・出力オブジェクトを指定して必要な情報を入力するだけで、統合化環境が計算処理を実行してくれる。コマンドラインの書式や入出力データの種類といった本質的でない情報は統合化環境が記憶していて、ユーザの負担を軽減してくれる。大型計算機の時代とは異なり、現在のシミュレーション環境は、分散・ネットワーク環境である。OS、アーキテクチャの異なる計算機上に、さまざまなアプリケーションソフトウェアがインストー図13・34 分散型問題解決環境の概要図13・35 レガシーコンポーネント図13・36 ネイティブコンポーネント図13・37 統合化環境の動作環境ルされている。シミュレーションは複数のアプリケーションを組合わせて実施されるため、科学者・技術者は、コンピュータに関する専門知識をある程度もっていることが要求される。統合化環境はコンピュータや通信の専門家でない彼らの負担を軽減し、高度なPSE環境を構築できる手助けをすることを目的に設計されている。統合化環境が想定する分散型問題解決環境を図13・37に示す。ユーザはLAN 上またはインタネットを介して、統合化環境の統一されたGUIを用い、分散ネットワーク環境上に散在するアプリケーションを利用して問題解決に当たることができる。このアプリケーションベースのPSEの利点は、(1)ワープロやエクセルでのデータ処理から流体計算や分子計算等、大小さまざまな規模の処理を扱えること、また、(2)それらのプログラムを統合化して組合わせて利用できること、さらに、(3)過去のレガシーアプリケーションから最新の情報技術をベースに構築されたシステムをヘテロに利用可能なこと、(4)将来にわたって開発され続けるであろうシステムの取り込みも想定してアーキテクチャが設計されていることなどが挙げられ、この方向でのPSE研究が盛んに世界的に行われているのも事実である。以下に、国内外で開発されている、または開発されつつあるアプリケーションベースのPSE研究6―207表13・2 グリッドアーキテクチャレイヤ内容アプリケーション分散型超高速計算、協調計算、高スループット計算、データ集中型計算などプログラミングツール/PSE言語、コンパイラ、可視化、通信ライブラリ、PSE などコモンサービススケジューラ、セキュリティ、遠隔データへのアクセス、リソース管理、通信、二重化などインフラストラクチャコンピュータ、OS、大記憶装置、DB、実験機器、ネットワーク、プロトコルなどの事例を示す。・科学技術計算環境STA:日本原子力研究所・分散オブジェクトによる科学応用システムに関する研究:電力中央研究所・科学・工学のための問題解決環境CAPSEの構築:構造計画研究所、パデュー(Purdue)大学ほか・分散型科学技術計算のためのXML-RPCエージェント:フロリダ州立大学(XML-RPC Agents for Distributed ScientificComputing : Florida State University)・エージェントグリッド:エージェント方式によるリソース統合化型問題解決環境:オークリッジ国立研究所(The Agent Grid:Agent-Based Resource Integration in PSEs: Oak RidgeNational Laboratory)
