# 1・1・1 計算機アーキテクチャ

> **章**: 第1章 情報システム
> **節**: 1・1 計算機ハードウェア

計算機アーキテクチャ(architecture)とは,計算機システムをどのように構成するかなどの構成方式のことをいう.この中には,記憶装置や演算制御などのハードウェアと,コンピュータ言語やオペレーティングシステムなどのソフトウェアの双方が含まれる.アーキテクチャの内容で,そのコンピュータシステムの処理速度などの基本的な性質が決定される.現在の計算機は大まかにいって,CPU(central processingunit,中央演算処理装置)とプログラムやデータを格納するメモリ,外部とのデータのやり取りを行う入出力装置からなる.CPUとメモリ,入出力装置との間でデータのやり取りを行う信号線をバスという.CPU は,単にプロセッサと呼ばれることもある(図1・1).プログラムはCPU 内部での演算やメモリにあるデータの読み書きなどのCPU の動作を指令する機械語命令の列であり,CPU はメモリに格納されているプログラムの命令を順次読み出し,命令により指定された動作を行う.このように,メモリにプログラムを格納し,それを実行する方式は内蔵プログラム方式(stored pro-gram)あるいはノイマンアーキテクチャと呼ばれ,現在の計算機の最も基本的なアーキテクチャとなっている.計算機の歴史の初期には,紙カードや紙テープで外部からプログラムの命令を与えて実行していたが,メモリ上にプログラムをおくことにより,高速に実行できるだけでなく,外部装置にあるプログラムをメモリに読込んで実行するプログラムが可能になり,現在の計算機に不可欠な自律的に動作するためのオペレーティングシステムの基礎となった.メモリにはデータの書込みや書換えが可能なRAM(randomaccess memory)と読出し専用のROM(read only memory)の二つの種類がある.ROM は固定的なプログラムやデータの格納に用いられる.メモリには8ビット(bit),すなわち1バイト(byte)ごとに,番地(アドレス)が付けられており,CPU からはその番地を指定してメモリに格納されているデータにアクセスする.CPU の内部には,プログラムの実行を制御する実行制御部と,加減算,乗除算を行うALU(arithmetic logic unit,演算論理装置),演算するデータや結果を一時的に格納するための,少量のメモリであるレジスタ(アキュムレータと呼ばれることもある)がある.計算機の命令は通常,メモリのデータをレジスタに格納するロード命令,レジスタからメモリに格納するストア命令,演算を行うための演算命令,命令の実行順序を制御する分岐命令,外部装置とのデータのやり取りを行う入出力命令がある.各計算機ごとに命令が決められており,これを命令セット(instruction set)という.計算機アーキテクチャという用語は,IBM 社から1964年に発売されたS/360の開発において最初に用いられた.S/360以前の計算機では,計算機ごとに命令セットやデータ形式を用意したため,これらは個々の計算機に付属するものと考えられていたが,S/360の設計において,これらにハードウェアから独立した定義を与えることで,小型から大型までの幅広い機種で共通のソフトウェアが稼働し,また新たに開発する計算機へのソフトウェアの継承を容易に7―1した.したがって,狭い意味では計算機アーキテクチャとは計算機の基本的な構成とそれに対する命令セットのことをいう.計算機の命令は,通常,命令の動作を指定する命令コード部と動作する対象を指定するオペランド部からなる.例として,図1・2に示すような命令セットがある簡単な計算機を考えてみることにしよう.この計算機には,X とY という二つのレジスタがあり,このレジスタにデータを読込み,演算を行うことができる.1000番地(16進数で3E8)と1004番地(3EC)に格納されているデータを加算し,2000番地(7D0)に格納するプログラムは,バイトごとに並べると以下のようになる(番地は2バイトの16進数で表現し,下位のバイトが最初にくるものとする).30E8 03 31EC 03 21 10 D0 07...このプログラムはメモリ上に格納され,計算機は命令を順に読み出し,命令コードを解読し,指定された操作を行う.実行する命令が格納されている番地を記憶しているレジスタを特に,プログラムカウンタと呼ぶ.上で示したとおり,計算機により解釈実行されるプログラムはメモリ上の2進数の数値として表現されている.これを機械語(machine language)という.命令コードに人間にわかりやすい名前(ニーモニックと呼ぶ)を付けて記述したものが,アセンブリ言語(assembly language)である.例えば,上のプログラムは以下のようになる.LOAD X,1000LOAD Y,1004ADD X,YSTORE X,2000このアセンブリ言語を機械語に変換するプログラムがアセンブラ(assembler)である.当初,プログラミングにはアセンブリ言語が用いられていたが,プログラムが複雑になるにつれ,より人間に使いやすいプログラミング言語が考えられた.プログラミング言語によって書かれたプログラムをアセンブリ言語,さらに機械語に翻訳するプログラムをコンパイラと呼ぶ.計算機内で符号付き整数の演算を行う場合,負の数は2の補数表示(2’s complement notation)で表現される.これは,数をビットで表現する場合,2욗-を用いるもので,これにより演算回路が簡単になる,負の数と正の数の演算を同じ回路で行うことができるなどの利点がある.2の補数とは,1の補数すなわち,ビットを反転した数に1を加えたものである.1バイトすなわち8ビットでは,-128~127までの範囲の数を表現できる.16ビットあるいは32ビットの整数はそれぞれ2バイト,4バイトとしてメモリに格納される.この場合,上位のビット(けた)から,アドレスの小さい番地から大きい番地のバイトに格納する方式をbig en-dian,逆に下位のビット(けた)から入れていく方式をlittle en-dianという.例えば,16進表示で,16ビットの数13A2の場合,big endianでは1バイトめに13,2バイトめにA2が格納されることになる.プロセッサにより,どちらかの方式がとられている.小数をもつ実数を表現する場合には,浮動小数点(floating図1・1 計算機の構成図1・2 命令セットの例7―2point)形式が用いられる.これは,数を,正負を表す1ビットの符合部と,仮数部と指数部に分けて表現する方式で,×2욓の数を表現する.浮動小数点形式のデータを用いた計算を浮動小数点演算と呼び,表現できる数の範囲が広いため,科学技術計算に利用される.浮動小数点演算を行う専用の演算装置をFPU(floatingpoint unit,浮動小数点数演算装置)と呼ぶ.現在計算機で多く用いられている形式はIEEE形式と呼ばれるもので,32ビットの単精度と64ビットの倍精度がある.単精度では,1ビットの符号部,7ビットの指数部,24ビットの仮数部の計32ビットで表現し,表現できる値の範囲は-3.40282×10~3.40282×10で,精度は6けたである.倍精度では,1ビットの符号部,11ビットの指数部,52ビットの仮数部の計64ビットで表現し,値の範囲は-1.79769×10~1.79769×10で,精度は15けたである.このほかに数値を表現する形式としては,10進法の各けたを4ビットで表現したBCD(binary coded decimal)形式がある.計算機で文字情報を扱う場合には,文字はそれに対応する数として表現される.この文字と数の対応を文字コードと呼ぶ.英数字の文字コードとして,ASCII(American Standard Code for Infor-mation Interchange)コードが一般的に使われている.これは英数字を7ビットで表現するもので1バイトに格納される.日本語の場合には,2バイトで表現され,その文字コードとしては,JIS コード,シフトJIS コード,EUCコードなどのコード体系が使われている.計算機アーキテクチャを命令セットで分類する場合,CISCアーキテクチャとRISCアーキテクチャの二つに分けられる.CISC(complex instruction set computer)とは,CPU に対する命令の種類を多くし,高機能化することにより処理能力を向上させようというアーキテクチャで,それに対しRISC(reduced instructionset computer)は,命令の数を少なく抑えることにより内部回路をシンプルにして,命令の実行を高速化することにより性能を向上させようというものである.例えば,CISCでは一つの命令でメモリからデータを読み,レジスタとの加算を行うための命令があるが,RISCではデータからレジスタに読込む命令とレジスタ間での加算命令を使い,この二つの命令を組合わせて行う.CISCではすべての演算命令にオペランドがメモリの場合とレジスタの場合に命令をもっているが,RISCではメモリアクセスはメモリからデータを読込むロード命令,書き込むストア命令で行い,演算はレジスタ間だけでやるようにして単純化している.そのほか,命令を一定の長さの形式に固定したり,レジスタを多くするなどの工夫がされている.しかし,現在ではRISCとCISCの差は意味がなくなりつつある.例えば,現在PC(パーソナルコンピュータ)などに多く使われているインテル社の,いわゆるIA32アーキテクチャは代表的なCISCプロセッサであるが,このプロセッサではCISCの命令セットを実行するときに,命令の解読時に,ハードウェアにより内部的なRISC命令(マイクロオペレーションと呼ばれている)に変換し実行することにより高速化が図られている.また,命令セット上はごく少ないレジスタしかないが,内部的には多数のレジスタを使うことにより効率的に実行できるようになっている.一方,RISCアーキテクチャをもつプロセッサでも,プログラムを効率的にするために初期のRISCアーキテクチャに比べて複雑な命令が導入される場合が多くなっている.以上が計算機の基本的なアーキテクチャであるが,実際にはさまざまな機能が装備されている.その中でも重要な仕組みが仮想記憶(virtual memory)である.プログラムやデータはメモリに格納されて実行されるが,メモリに格納しきれない場合には実行できなくなってしまう.そこで,プログラムやデータの一部のみを実際のメモリにおいて実行し,残りをディスクなどの二次記憶装置においておき,必要に応じてメモリ上にロードして実行することが考えられる.仮想記憶とは,仮想的なメモリを考え,そのアドレス(論理ア図1・3 仮想記憶の仕組みドレスという)を実際にあるメモリ上のアドレス(物理メモリアドレスという)に変換する機構である.論理アドレスに対応する実メモリがある場合には,そのデータにアクセスするが,もしも対応するメモリがない場合には実行を中断し,オペレーティングシステムが二次記憶装置上にあるデータを実メモリ上のデータと入れ替え,それに対して論理アドレスを対応させ実行を再開させる.論理アドレスと物理アドレスの変換を行うハードウェアをメモリ管理ユニット(MMU : memory management unit)と呼ぶ(図1・3).メモリと二次記憶装置上のデータの入れ替えは数キロバイト単位で行われ,これをページと呼ぶ.二次記憶上のプログラム全体を格納しておく領域のことをスワップ領域といい,入れ替える動作をスワップ(swap)という.スワップはオペレーティングシステムによって行われるため,アプリケーションプログラムは実メモリがあるかどうかを気にせずに実行できることになる.ただし,組込み用途などのプロセッサでは,プログラムは通常ROM などのメモリに収まる小さなプログラムであることが多いため,仮想記憶がない場合も多い.入出力装置を扱うために重要な機能としては,割込み(inter-rupt)機能がある.これは外部装置から信号が入力された場合に実行中のプログラムの実行を中断し,その信号を処理するためのプログラムを動作させる機能である.この機能により,キーボード入力を検知してデータを読込んだり,ディスクなどの外部装置の制御を効率的に行うことができる.また,多くの計算機では時計機能(タイマ)を内蔵しており,一定の間隔で信号を発生させることにより割込み処理を行う.これにより,一定の間隔で機器を監視したり,ある時刻にいろいろな処理を行う場合に便利な機能である.この機能を用いることにより,一定の時間でプログラムを切換え,複数のプログラムを時分割して同時に実行することができる.これをタイムシェアリングシステム(TSS : time sharing system)という.
