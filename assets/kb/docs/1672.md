# 1・1・1 計算機アーキテクチャ

> **章**: 第1章 情報システム
> **節**: 1・1 計算機ハードウェア

計算機アーキテクチャ（architecture）とは，計算機システムを
どのように構成するかなどの構成方式のことをいう．この中には，
記憶装置や演算制御などのハードウェアと，コンピュータ言語やオ
ペレーティングシステムなどのソフトウェアの双方が含まれる．ア
ーキテクチャの内容で，そのコンピュータシステムの処理速度など
の基本的な性質が決定される．
現在の計算機は大まかにいって，CPU（central processing
unit，中央演算処理装置）とプログラムやデータを格納するメモ
リ，外部とのデータのやり取りを行う入出力装置からなる．CPU
とメモリ，入出力装置との間でデータのやり取りを行う信号線をバ
スという．CPUは，単にプロセッサと呼ばれることもある（図1･
1)．
プログラムはCPU内部での演算やメモリにあるデータの読み書
きなどのCPUの動作を指令する機械語命令の列であり，CPUは
メモリに格納されているプログラムの命令を順次読み出し，命令に
より指定された動作を行う．このように，メモリにプログラムを格
納し，それを実行する方式は内蔵プログラム方式（stored pro-
gram）あるいはノイマンアーキテクチャと呼ばれ，現在の計算機
の最も基本的なアーキテクチャとなっている．計算機の歴史の初期
には，紙カードや紙テープで外部からプログラムの命令を与えて実
行していたが，メモリ上にプログラムをおくことにより，高速に実
行できるだけでなく，外部装置にあるプログラムをメモリに読込ん
で実行するプログラムが可能になり，現在の計算機に不可欠な自律
的に動作するためのオペレーティングシステムの基礎となった．
メモリにはデータの書込みや書換えが可能なRAM（random
access memory）と読出し専用のROM（read only memory）の
二つの種類がある．ROMは固定的なプログラムやデータの格納に
用いられる．メモリには8ビット（bit），すなわち1バイト
（byte）ごとに，番地（アドレス）が付けられており，CPUからは
その番地を指定してメモリに格納されているデータにアクセスす
る．
CPUの内部には，プログラムの実行を制御する実行制御部と，
加減算，乗除算を行うALU（arithmetic logic unit，演算論理装
置），演算するデータや結果を一時的に格納するための，少量のメ
モリであるレジスタ（アキュムレータと呼ばれることもある）があ
る．計算機の命令は通常，メモリのデータをレジスタに格納するロ
ード命令，レジスタからメモリに格納するストア命令，演算を行う
ための演算命令，命令の実行順序を制御する分岐命令，外部装置と
のデータのやり取りを行う入出力命令がある．各計算機ごとに命令
が決められており，これを命令セット（instruction set）という．
計算機アーキテクチャという用語は，IBM社から1964年に発売
されたS/360の開発において最初に用いられた．S/360以前の計
算機では，計算機ごとに命令セットやデータ形式を用意したため，
これらは個々の計算機に付属するものと考えられていたが，S/360
の設計において，これらにハードウェアから独立した定義を与える
ことで，小型から大型までの幅広い機種で共通のソフトウェアが稼
働し，また新たに開発する計算機へのソフトウェアの継承を容易に

計算機アーキテクチャを命令セットで分類する場合，CISCアー
キテクチャとRISCアーキテクチャの二つに分けられる．CISC
（complex instruction set computer）とは，CPUに対する命令の
種類を多くし，高機能化することにより処理能力を向上させようと
いうアーキテクチャで，それに対しRISC（reduced instruction
set computer）は，命令の数を少なく抑えることにより内部回路を
シンプルにして，命令の実行を高速化することにより性能を向上さ
せようというものである．例えば，CISCでは一つの命令でメモリ
からデータを読み，レジスタとの加算を行うための命令があるが，
RISCではデータからレジスタに読込む命令とレジスタ間での加算
命令を使い，この二つの命令を組合わせて行う．CISCではすべて
の演算命令にオペランドがメモリの場合とレジスタの場合に命令を
もっているが，RISCではメモリアクセスはメモリからデータを読
込むロード命令，書き込むストア命令で行い，演算はレジスタ間だ
けでやるようにして単純化している．そのほか，命令を一定の長さ
の形式に固定したり，レジスタを多くするなどの工夫がされてい
る．
しかし，現在ではRISCとCISCの差は意味がなくなりつつあ
る．例えば，現在PC（パーソナルコンピュータ）などに多く使わ
れているインテル社の，いわゆるIA32アーキテクチャは代表的な
CISCプロセッサであるが，このプロセッサではCISCの命令セッ
トを実行するときに，命令の解読時に，ハードウェアにより内部的
なRISC命令（マイクロオペレーションと呼ばれている）に変換し
実行することにより高速化が図られている．また，命令セット上は
ごく少ないレジスタしかないが，内部的には多数のレジスタを使う
ことにより効率的に実行できるようになっている．一方，RISCア
ーキテクチャをもつプロセッサでも，プログラムを効率的にするた
めに初期のRISCアーキテクチャに比べて複雑な命令が導入される
場合が多くなっている．
以上が計算機の基本的なアーキテクチャであるが，実際にはさま
ざまな機能が装備されている．その中でも重要な仕組みが仮想記憶
（virtual memory）である．プログラムやデータはメモリに格納さ
れて実行されるが，メモリに格納しきれない場合には実行できなく
なってしまう．そこで，プログラムやデータの一部のみを実際のメ
モリにおいて実行し，残りをディスクなどの二次記憶装置において
おき，必要に応じてメモリ上にロードして実行することが考えられ
る．仮想記憶とは，仮想的なメモリを考え，そのアドレス（論理ア
ドレスという）を実際にあるメモリ上のアドレス（物理メモリアド
レスという）に変換する機構である．論理アドレスに対応する実メ
モリがある場合には，そのデータにアクセスするが，もしも対応す
るメモリがない場合には実行を中断し，オペレーティングシステム
が二次記憶装置上にあるデータを実メモリ上のデータと入れ替え，
それに対して論理アドレスを対応させ実行を再開させる．論理アド
レスと物理アドレスの変換を行うハードウェアをメモリ管理ユニッ
ト（MMU:memory management unit）と呼ぶ（図1･3)．メモ
リと二次記憶装置上のデータの入れ替えは数キロバイト単位で行わ
れ，これをページと呼ぶ．二次記憶上のプログラム全体を格納して
おく領域のことをスワップ領域といい，入れ替える動作をスワップ
（swap）という．スワップはオペレーティングシステムによって行わ
れるため，アプリケーションプログラムは実メモリがあるかどうか
を気にせずに実行できることになる．ただし，組込み用途などのプ
ロセッサでは，プログラムは通常ROMなどのメモリに収まる小さ
なプログラムであることが多いため，仮想記憶がない場合も多い．
入出力装置を扱うために重要な機能としては，割込み（inter-
rupt）機能がある．これは外部装置から信号が入力された場合に実
行中のプログラムの実行を中断し，その信号を処理するためのプロ
グラムを動作させる機能である．この機能により，キーボード入力
を検知してデータを読込んだり，ディスクなどの外部装置の制御を
効率的に行うことができる．また，多くの計算機では時計機能（タ
イマ）を内蔵しており，一定の間隔で信号を発生させることにより
割込み処理を行う．これにより，一定の間隔で機器を監視したり，
ある時刻にいろいろな処理を行う場合に便利な機能である．この機
能を用いることにより，一定の時間でプログラムを切換え，複数の
プログラムを時分割して同時に実行することができる．これをタイ
ムシェアリングシステム（TSS:time sharing system）という．
1・1・2高速化技術
計算機の高速化技術としては，まず，メモリへのアクセスを速く
することが挙げられる．多くの計算機ではメインメモリ（main
memory）として，大容量化が容易なDRAM（dynamic random
access memory）が用いられている．DRAMは，電荷を蓄える回
路を記憶素子として集積したもので，単純で集積度を上げるのが容
易な反面，一定時間ごとに記憶保持のための信号を送る必要がある
など，メモリアクセス速度が比較的遅いという欠点がある．それに
対して，トランジスタを組合わせたフリップフロップ回路を記憶素
子として用いたSRAM（static random access memory）は，小
容量であるが高速なアクセスが可能である．そこで，DRAMのメ
インメモリとCPUの間にSRAMをおき，そこに頻繁にアクセス
するデータや命令を格納しておく仕組みがキャッシュメモリ
（cache memory）である．まず，CPUはアクセスの早いキャッシ
ュメモリを調べ，そこにデータがある場合はそのデータを用いる．
もしも，キャッシュメモリにない場合にはメインメモリにアクセス
し，データを取出すと同時にキャッシュメモリに格納しておく．そ
のデータを再度アクセスするときには，キャッシュメモリに対して
アクセスすることで高速にデータにアクセスすることができるよう
になる．プログラムは多くの場合，限られたデータにアクセスする
ことが多いので，プログラムの実行が高速化されることになる．同
