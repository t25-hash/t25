# 1・1・2 高速化技術

> **章**: 第1章 情報システム
> **節**: 1・1 計算機ハードウェア

計算機の高速化技術としては、まず、メモリへのアクセスを速くすることが挙げられる。多くの計算機ではメインメモリ(mainmemory)として、大容量化が容易なDRAM(dynamic randomaccess memory)が用いられている。DRAM は、電荷を蓄える回路を記憶素子として集積したもので、単純で集積度を上げるのが容易な反面、一定時間ごとに記憶保持のための信号を送る必要があるなど、メモリアクセス速度が比較的遅いという欠点がある。それに対して、トランジスタを組合わせたフリップフロップ回路を記憶素子として用いたSRAM(static random access memory)は、小容量であるが高速なアクセスが可能である。そこで、DRAM のメインメモリとCPU の間にSRAM をおき、そこに頻繁にアクセスするデータや命令を格納しておく仕組みがキャッシュメモリ(cache memory)である。まず、CPU はアクセスの早いキャッシュメモリを調べ、そこにデータがある場合はそのデータを用いる。もしも、キャッシュメモリにない場合にはメインメモリにアクセスし、データを取出すと同時にキャッシュメモリに格納しておく。そのデータを再度アクセスするときには、キャッシュメモリに対してアクセスすることで高速にデータにアクセスすることができるようになる。プログラムは多くの場合、限られたデータにアクセスすることが多いので、プログラムの実行が高速化されることになる。同じ動作はメモリ上の命令に対しても行われる。特に、命令を繰返すループの場合には同じ命令が用いられるためキャッシュ上にあることが多くなり、高速化される。また、集積回路ではチップの外にアクセスするよりも、チップの中で動作するほうが高速にアクセスできる。以前のCPU では、集積度に制限があったため、キャッシュメモリはチップの外に置かれていたが、集積度が向上するにつれて、キャッシュメモリはチップの中に置かれるようになっている。さらに、一般的に、必要なデータを選ぶ回路はメモリが大きくなると複雑になるためにアクセスに時間がかかるようになる。集積度が向上し、キャッシュメモリの容量も大きくなるにつれ、それまでのキャッシュとCPU の間にさらに少容量のキャッシュを入れて高速化するようになった。現在の多くの計算機では、CPU の近いほうから、一次キャッシュ、二次キャッシュというように2段のキャッシュメモリをもつようになっている。CPU は一次キャッシュにアクセスし、データがない場合には二次キャッシュ、さらにない場合にはメインメモリへアクセスする。アクセス頻度の高いデータほど、CPU に近いメモリにおかれることになり、高速化されることになる。通常、図1・4に示すように一次キャッシュは命令用とデータ用の別々の一次キャッシュをもつ。計算機を高速化するアーキテクチャ(architecture)の基本的な仕組みは、多くの命令を並列に処理するということである。パイプライン(pipeline)アーキテクチャはその典型的な方式の一つである。CPU は命令を一つずつ順に実行するが、一つの命令の実行手順を細かく見てみると、いくつかの手順に分かれる。例えば、典型的なCPU の処理として以下の大まかな手順が考えられる。(1) 実行するべき命令をメモリから読み出す(IF:instructionfetch)。(2) 読み出した命令を解読して各部分に信号を送り出す(DE:decode)。(3) back)。これらの処理を行うそれぞれの回路はクロック(clock)で制御され、クロック信号が入力されるごとに順に処理が進められる。クロックの周波数が高ければ高いほど高速に処理されることになるが、クロックの間隔はそれぞれの処理の時間による。この場合、5クロックで1命令実行されることになる。一つの命令の実行が終わってから、次の命令の実行を開始するとすると、2命令実行するのに10クロック、命令実行するには5クロックかかることになる。その様子を図1・5に示す。これらの処理はCPU
