# 1・2・1 計算資源管理

> **章**: 第1章 情報システム
> **節**: 1・2 オペレーティングシステム

a. プロセス管理(CPU 時間管理)計算機内部での処理の流れはCPU 時間の経過とともに進む.どの処理にいつどれだけのCPU 時間を消費するかを管理するには,処理の流れを抽象化して扱いやすくする必要があるため,プログラム実行制御の流れをプロセス(またはスレッド)としてとらえる.実在するプロセッサ数がプロセス(process)の数より少なくても,各プロセスは,そのプログラムが,あたかもそれぞれに与えられたプロセッサ(擬似プロセッサと呼ばれる)により処理されるようにみなされる.具体的には,プロセスの状態は,それを処理するプロセッサの状態,プロセスのプログラムが占有している記憶領域や入出力の状態を併せたものと見ることができる.プロセスは通常計算機システム内に複数個存在し,実プロセッサの割当てを受けたプロセスが実行状態となって処理が進められる.実行中のプロセスが,入出力待ちなどで処理の続行ができなくなったときは,停止状態となる.プロセッサは,他の実行可能状態のプロセスに割当てられ,そのプロセスの処理が行われる.停止状態のプロセスは,実行可能状態になるとプロセッサの割当てを待つ.すなわちプロセスは,実行状態,停止状態,実行可能状態の三つの状態をこの順に遷移する.停止状態になったプロセスの代わりに別のプロセスを実行させるためには,プロセッサのレジスタ値などの状態や記憶空間の状態を切換えなければならない.このことをコンテクスト切換え(con-text switching)という.コンテクスト切換えには少し時間がかかるため,頻繁にプログラムの流れを切換える必要がある場合には,切換えに要する時間が多くなってしまい肝心の処理が遅くなる.そのため,処理の状態全体ではなくその一部だけを切換える軽量プロセス(light weight process)が考えられた.そのひとつにプロセッサの状態だけを切換え,記憶空間は切換えずに共有するスレッド(thread)がある.スレッドは最初Mach OS で実装されたが,Windows Win32 APIにも含まれ,UNIX 系のOS には,Posix 準拠のP-Threadなどが実装されている.またJava仮想マシンではユーザレベルでスレッドが使用できる.以下,スレッドも含めてプロセスと表記することにする.割込み処理,カーネルプロセス(OS 自身が必要とする仕事,プロセス)以外のプロセスは,実行可能状態となったものの中から,次にプロセッサに割当てられるものが選択され,プロセッサを割当てられ実行される.この選択をプロセッサスケジューリングという.スケジューリングの目的には以下のようなものがある.(1) CPU 稼働率を高くする.(2) スループット(throughput)を大きくする.(3) ターンアラウンドタイム(turnaround time)を短くする.(4) 待ち時間(waiting time)を短くする.(5) 応答時間(response time)を短くする.(6) 消費エネルギーを少なくする.(7) アプリケーションから要求されたQoS レベルに基づいて,CPU,メモリ,I/O,ネットワーク,エネルギーの管理を行う.スケジューリングの目的に基づいて実際にスケジューリングを行うため,スケジューリングアルゴリズムが考えられている.スケジューリングは,まず,プリエンプティブ(preemptive)とノンプリエンプティブ(non-preemptive)に分けられる.プリエンプティブスケジュールでは,プロセスは,実行が中断されることがある[実行を中断されたプロセスが後に続行されるものを,特にプリエンプティブレジューム(preemptive resume)であるという].中断することがないスケジュールは,ノンプリエンプティブである.さらに,基本的な例として以下のようなものがある.(1) first-come first-served:早く来たものから順に処理をする.(2) shortest-job-first:処理時間の短いものから処理する.(3) priority scheduling:優先順位の高いものから処理をする.(4) round-robin:一定の時間ずつ処理をし,時間内に処理できないものは,待ち行列の最後尾につける.これらは,例えばround-robinの待ち行列を複数用意し,それらの間に優先順位を付けるというように,組合わせて用いられることもある.b. 記憶管理記憶装置は,高速であるが高価なデバイスから,低速であるが安価なものまで幅があるため,図1・10のように階層的に組合わせて用いられる.高速な順に大まかに,キャッシュ記憶,主記憶,補助(二次)記憶がある.OS では主記憶と補助記憶の管理を行う.. 主記憶管理プロセスを実行するためには,その実行に必要な主記憶領域を割当てなくてはならない.この割当て方は利用可能な主記憶管理のためのハードウェアやプロセス管理の方式に合っていることが求められる.唯一のプロセスに主記憶を割当てる単一連続割当て方式は,主記憶管理のためのハードウェアを要しない.多重プログラミングのために主記憶に複数個のプロセスを割当てる場合,単一連続割当て方式では,多数のプロセスによる主記憶の再利用を経ると細切れの未使用領域(fragmentation)が多数生じるむだが起こる.これは細切領域を一つにまとめる操作(compaction)や,停止状態にあるプロセスを主記憶から二次記憶に退避(swap out)し,代わりに近い将来にスケジューリングされるプロセスを主記憶に再配置(swap in)するスワッピング(swapping)などで改善できる.この場合,ベースレジスタを設けるなど,プロセスが再配置可能(relocatable)であるようにしておく必要がある.これらの問題は,主記憶を固定長のページ(page)に分割する方式で解決されるが,プロセスのプログラムの大きさと区画の大きさがぴったり合わないことによる若干のむだが生じうる.仮想記憶(virtual memory)方式は,動的アドレス変換機構(DAT)を用いて,通常は主記憶の容量よりも大きい仮想アドレス空間[virtual address space,論理アドレス空間(logical address図1・10 記憶階層space)ともいう]の仮想アドレスを実行時に主記憶上の物理アドレスに変換する方式である.ページ方式(paging)による仮想記憶では,ページごとに仮想アドレスと主記憶上のアドレスの対応が記憶されており,主記憶上に存在しないアドレスへのアクセスが生じたときは,OS によりそのアドレスを含むページを二次記憶から主記憶に転送する.主記憶上に新たにページをおく空きがないときには,先に主記憶上のページの適当なものを二次記憶上に転送してページの置換え(pagereplacement)を行う.ページ置換えの際には,時間的に最も長い間参照されていないページを二次記憶に転送するLRU(leastrecently used)アルゴリズムがよく用いられる.ページングによりもたらされる仮想記憶空間を多重化するのにセグメント方式(segmentation)が用いられる.論理的に密接につながったプログラムやデータを納めたページの集まりをセグメントとし,各プロセスの仮想記憶空間をセグメントの集まりとして構成する.セグメントが論理的にまとまった情報であるため,プロセス間での情報の共有や,プロセスの実行中にプログラムやデータのリンクを行うダイナミックリンキング(dynamic linking)の機能,セグメント単位での保護機能などが実現しやすい.セグメントは可変長であるため仮想記憶空間の実際の大きさに柔軟性がもたらされる.. 補助記憶管理補助記憶は,次のような目的のために使われる.(1) 主記憶におけない多量の情報を一時的に格納する(2) 大量の情報を常時参照できるよう格納する(3) データ交換や常時使用しない情報を保存する(1)の目的のためには大型の計算機では高速の半導体ディスクなども用いられるが,(1)(2)の目的のためには現在は磁気ディスクが多く用いられている.(3)の目的のためには磁気テープ,光ディスクなどが用いられるが,情報の保存やバックアップのためには地理的,電気的に分散していることが望ましいので,データセンタやネットワークディスクが用いられることもある.また着脱可能なメモリも用いられる.補助記憶中の情報はファイル単位で管理される.ファイル内のデータの論理的な構造を扱うOS もあるが,ファイル内のデータ構造の扱いはそのデータを使用するアプリケーションに任せるOS もある.前者ではファイル内の構造に応じたアクセス方法がOS に用意される.ファイル管理システムには,必要な容量のファイルを高速に検索し,読み書きを行い,補助記憶装置に効率よく格納し,維持管理することが求められる.そのためファイルには識別子として名前が付けられる.また大量のファイルを管理するため,ファイル名の管理簿としてディレクトリを設ける.多くのファイルは分類して管理するほうが簡単に管理できる.分類のための構造に木構造を採用したUNIX のファイルシステムの例を図1・11に示す.木の根に当たるところをルート(root)ディレクトリと呼ぶ.図1・11中の枝分かれ部分がディレクトリで,先端の葉に当たるところにファイルがある.ディレクトリもファイルの管理情報を保持するための一種のファイルである.7―7OS ではファイルの名前と物理的な保存場所やサイズだけでなく,ファイルの所有者,作成やアクセスした日付と時刻,共有情報などの属性情報をディレクトリやファイル自体の一部に記録しておき,その情報を利用して管理する.一般にプロセスがアクセスするオブジェクトの管理リストをケーパビリティリストと呼び,ファイルもオブジェクトの一種としてアクセスの管理をすることもある.ファイルを実際に磁気ディスクなどに格納するには,磁気ディスク上の領域を割当てることが必要である.主記憶管理におけるページングと同様,固定長のブロック単位で割付けを行うことが多い.ファイルの記録媒体にはさまざまなものがあり,記録方法も多様であるが,取扱いのはん用性,継続性の点からは,アクセスする方法が論理的に同じであることが望ましい.その一方で,ファイルアクセスを高速に行うためには,ソフトウェアの移行性を犠牲にして,ブロックのサイズや物理的な配置をユーザが管理して入出力を行う機能も必要である.UNIX ではraw I/Oと呼ばれる機能がこれに当たる.アクセスの高速化には,順次ファイルを磁気ディスクの連続領域に割当てるというような領域割当てによる工夫や,主記憶上のシステムバッファにファイルのブロックをキャッシュしておくというような方法もある.読込みの際にデータがキャッシュしてあれば磁気ディスクからの読込みが省略できるだけでなく,書込みの際にもユーザバッファの内容をシステムバッファにコピーした時点でシステムコールから復帰させ,実際の磁気ディスクへの書込みは,ファイルのクローズや同期(sync)の指示があったときに行うことで見掛け上のアクセスを高速化できる.c. ネットワーク管理ネットワークもまた重要な計算資源である.複数のプロセスがネットワークを使用する場合には,プロセッサの割当てと同様にスケジューリングを行う.プロセスにプライオリティがある場合,簡単にはプライオリティに応じてネットワークの帯域幅(band width)を割当てるなどの方策がある.d. エネルギー管理近年バッテリをエネルギー源として働く情報機器が増えるに従い,エネルギーも計算資源という考え方が出てきている.CPU も単に休止状態にして電力消費を低減するだけではなく,クロック周波数を低下させて電源電圧を下げたり,部分的に稼働させたりすることにより,きめ細かくエネルギー消費を減らすことができるようになってきている.このような機能を使ってパーソナルコンピュータの省電力制御を行うためのインタフェース規格にACPI(advanced configurationand power interface)がある.ACPIによって,デバイスごとに複数の電力レベルを設定することや,復帰時間を定めることなどもできる.また,ACPIでは,USB(universal serial bus)やIEEE1394に接続されたデバイスなど,周辺機器の電源管理も可能であるため,エネルギー管理の範囲を広げることができる.このような仕組みにより,OS が電力制御の管理を行い,必要なデバイスにのみ電力を供給できるようになっている.OS による電力管理をOSPM(operating system directed power manage-ment)と呼ぶ.
