# 1・3・1 アルゴリズムとデータ構造

> **章**: 第1章 情報システム
> **節**: 1・3 計算機ソフトウェア

a. 基本データ型とそのデータ表現。 固定小数点表現と浮動小数点表現現在利用されている計算機の大多数はディジタル型計算機である。このディジタル型という言葉は日本語では計数型と訳されるように、すべての数値は有限個の離散的な値で表現される。このため、実数のようなちゅう密な量を表現することは不可能であり、その中からいくつかの値を選び出し、それらによってその付近の値を代表させることが必須となる。ディジタル型電子計算機(以下では単に計算機と呼ぶ)では、基本的な数値の表現方式として固定小数点(fixed point)表現と浮動小数点(floating point)表現と呼ばれる二つの表現手段を併用するものが多い。固定小数点表現の多くの場合においては、最も右のけたのすぐ右に小数点が存在することを仮定しているため、これを特に整数表現と称する。しかし、グラフィック演算や機器組込み用などの特殊な用途では、小数点の位置を最も左のけたの左隣やその他さまざまな位置に仮定するため注意が必要である。一方の浮動小数点表現では、データ表現の中に数値そのものに関する情報に加えて、小数点の位置に関する情報をも併せ持っている。この数値そのものに関する情報の部分を仮数部、小数点の位置に関する部分を指数部と呼ぶ。仮数部はその最も左のけたの左に仮7―11の小数点をおき、指数部で仮の位置から実際の小数点の位置までのけた数を表示する。仮数部はその最上位けたが零以外の値になるように指数部の値を増減しながら左右にシフトされるが、この操作を正規化と呼ぶ。また、仮数部の基数が2の場合は正規化された仮数部の最上位けたは必ず1になるため、このけたを省略することも多く行われる。指数部の基数が2よりも大きい場合は少ない仮数部のけた数で広い範囲の数値を表現することが可能となるが、表現しようとする値によっては仮数部内の数値の有効けた数が減ってしまう場合がある。また、数値を正規化すると指数部の数値が表現可能な範囲を逸脱してしまう場合があり、特にそれが零に近いときには指数部に最小の値を与えて値を正規化することをあきらめることがある。このような数値表現を非正規化数と呼ぶ。. 符号の表現数値を表現する場合に正負の符号をいかに表現するかも問題となる。用途から加減算を容易にすることが重要となる場合には、2の補数表示(2’s complement notation)もしくは数値に定数を加えた値で表示する下駄履き表示を行う。これら二つの表示方法においてはその値の増加方向が符号なし二進数の値の増加方向とつねに同じになるため、加減算を行うハードウェアを簡略化することができる利点がある。また、加減算より乗除算のほうが多く必要とされることが予想される場合には、乗除算の演算を容易にするために数値の絶対値と符号とを組合わせた表現を行う。一般に、整数型の表現では1を加えたり引いたりといった加減算の必要性が大きく、おもに2の補数表示が用いられる。また、浮動小数点表現では乗除算の高速化が必須となるため絶対値表示が用いられるが、この場合でも指数部については加減算しか必要とならないためおもに下駄履き表示が用いられる。. 数値の精度(precision)と表現範囲数値をより精度良く、より広範囲にわたって表現するためには、数値を表現するのに用いるメモリ領域をより多く割当てる必要がある。計算機における情報量の単位は二進法の一けたを示すビット(bit)であるが、10ビットで十進法のおよそ3けたに相当する。例えば、32ビットの符号付き整数を表現するには、1ビットの符号ビットと31ビットの数値ビットを用い、-1×2(=-2147483648)から2-1(=2147483647)までの範囲の整数を、およそ9.3けたの精度で表現することができる。これが32ビットの符号なし整数であれば0から2-1(=4294967295)までの範囲の整数を、また16ビットの符号付き整数であれば-1×2(=-32768)から2-1(=32767)までの範囲の整数をおよそ4.5けたの精度で表現することが可能となる。近年のプロセッサではハードウェアで64ビットの符号付き整数を扱えるものが現われた。これを用いれば十進法でおよそ19けたの整数を表現でき、例えば、国家予算を整数として表現できるようになった。浮動小数点型の表現では、一つの数値に割当てられたビット数をどのように仮数部と指数部とに割り振るかが問題となる。IEEE標準に規定される浮動小数点形式では仮数部の基数に2を用いるが、数値の符号に1ビットを割当てた後、32ビット浮動小数点形式では仮数部と指数部にそれぞれ23ビットと8ビットを、64ビット浮動小数点形式ではそれぞれ52ビットと11ビットを割当てる。このため、浮動小数点表現の数値の精度は32ビットの場合はおよそ7けた、64ビットの場合ではおよそ16けた程度となる。また、正規化数で表現可能な範囲は32ビットの場合は10の-38乗から38乗まで、64ビットの場合は10の-308乗から308乗までとなる。また128ビット浮動小数点形式の実装の一つでは、仮数部と指数部にそれぞれ112ビットと15ビットを割当て、正規化数で表現可能な範囲を10の-4932乗から4932乗までとしている。また、仮数部の基数に2以外を採用した例としてはIBM 社の360アーキテクチャ以降で用いられている16を基数とした表現形式がある。この形式では指数部のビット数を7ビットとし、数値全体に割当てられたビット数によらず一定としている。この場合には表現可能な範囲は10の-79乗から75乗までとなる。7―12情報・ソフトウェアもしも表現したい数値の有効けた数が、用いようとする数値表現形式で表現できる精度を上回っている場合には、何らかの丸めを行って近似値として表現するほかない。あるいは、表現したい数値が正規化数で表現できる範囲から逸脱している場合には、非正規化数を用いたり、零や正負の無限大で代表することになる。なお、浮動小数点数の表現体系の中に、無限大や演算結果が定義できないときに代用される値など、正確には数値でない値を表現するためのビットパターンなどを定義することが行われている。b. 数値演算の誤差(error)浮動小数点型で表現された数値は加減算を行う場合に誤差を発生する危険性がある。例えば精度が52ビットの表現をとっている場合に、指数部の値の差が52を超える二数を加減算すると、絶対値の小さいほうの数値が絶対値の大きいほうの数値に影響を及ぼすことなく失われてしまう。一方、ほとんど同じ絶対値をもつ二数を加減算すると、それらの符号によっては非常に零に近い結果を生じることがある。このとき、元の二数が十分な精度をもった値であっても、この演算によって一気に有効けたを失ってしまって、結果として得られる数字のうち意味のあるけた数がほんの少しになってしまう場合があり、これを情報落ちと呼ぶ。これら2種類の誤差は、いずれも計算機内で保持される数値の有効けた数が有限であることに起因するものであり、いずれも加減算によって発生するものである。これを避けるためには、演算を行う前に計算式を変形するなどしてけた落ちや情報落ちを発生しにくい演算順序に変形しておく必要がある。c. アルゴリズム(algorithm)と計算量(complexity)アルゴリズムとは、問題の解答を求めるために必要な計算の手順のことである。アルゴリズムにはすべての解答を求めた後に(ほかに解答がないことを示すために)停止することが必要であり、またつねに同じ計算経過をたどって同じ解答を出さなければならない。この意味では、オペレータが途中で演算を停止させることを前提としてエンドレスループにしたり、乱数を用いて解答を求めたりしたプログラムは、正確にはアルゴリズムとは呼べないことになる。アルゴリズムにはそれに対応した計算量の概念が存在する。入力データの分量を示す代表的な値(例えばデータ点数など)をとしたとき、例えば計算に必要な時間をの関数として表現することができる。この関数の=∞の極限における最高次の項から係数を除いたものを時間計算量(time complexity)のオーダと呼び、同様に計算に必要な記憶領域に関するものを領域計算量(space complexity)のオーダと呼ぶ。アルゴリズムのオーダはそのアルゴリズムを利用する限りどうしても必要になる時間や記憶容量を評価する手掛かりとなり、計算機の速度や記憶容量の向上が目覚しい近年においても、どの程度の規模の問題であれば現実に解答を得られるかという点で重要な概念である。d. データ型(data type)と基本演算データ型はデータが属する集合を示すものであり、記憶装置中では全く同じビットパターンで表現されるデータも異なった型に属するなら異なった解釈を与えられる。さらに、コンパイラなどでは変数に対してもそれが属するデータ型を定義し、実行時のデータの解釈の高速化を図っている。同じ型に属するデータに対しては同じ演算操作が適用可能であり、あるデータに適用可能な演算操作の集合やその演算結果のデータ型はそのデータの属する型によってのみ決定される。すなわち、データに適用可能な演算操作集合はそのデータの属する型の属性と考えられる。. 整数型正と負の比較的絶対値の小さな整数を表現するために用いられる型で、代入・等値性判定・大小比較のほかに、加減乗除算などの算術演算や、次の値や直前の値を求める演算が定義される。. 符号なし整数型零以上の比較的絶対値の小さな整数を表現するために用いられる型で、代入・等値性判定・大小比較のほかに、加減乗除算などの算術演算や、次の値や直前の値を求める演算が定義される。整数型とは表現可能な範囲が異なる。. 実数型比較的絶対値の小さな実数を表現するために用いられる型で、代入・等値性判定・大小比較のほかに、加減乗除算などの算術演算が定義される。また、ちゅう密な実数を離散的な数値で代表させるため、数値の表現にはある程度の誤差を伴う。. 列挙型その型に属するデータがとりうる値をすべて列挙したデータ型で、値そのものは数値ではなく定数識別子であることが多い。代入・等値性判定・大小比較のほかに、次の値や直前の値を求める演算が定義される。また、先頭の(最小の)要素から数えた順番を示す数値との相互変換も重要である。この型は順序尺度を実現するもので、算術演算は定義されない。. 文字型列挙型の特殊なケースで文字セットのすべての要素を列挙した列挙型と考えられる。. 論理型列挙型の特殊なケースで真と偽とを要素とする列挙型である。代入・等値性判定・大小比較のほかに、論理演算が定義される。. 部分範囲型整数型・符号なし整数型・列挙型の一部の範囲の値をとりうるデータ型で、上限と下限を指定して定義する。. 集合型列挙型もしくは整数型・符号なし整数型・列挙型の部分範囲型がとりうる値の集合のべき集合(powerset)をとりうる値とするデータ型である。集合の積・和・差演算や特定の要素を含むか否かといった演算が定義される。. 配列型任意の一つの型に属する複数個のデータをひとまとめにしたデータ構造である。要素データの型がすべて同じなので、それらに対して演算処理を施す場合には同質の処理を繰返し行うことが多く予想されるため、要素データの指定を添字と呼ばれる順序を示す数値を用いて行う。また、要素データの数が有限かつ既知であることを前提としているため、配列をメモリ内に割付けてその任意の要素のメモリ内での位置をあらかじめ決定することができる。配列の要素データに対しては、任意の要素に対して任意の順番で参照と代入ができる。また、配列型データ全体に適用可能な演算操作には代入や等値性判定がある。. 構造型複数個の任意の型に属するデータをひとまとめにしたデータ構造である。構造型のデータのとりうる値の集合は、各要素のとりうる値の集合の直積となる。各要素の型が互いに異なっていることから、個々の要素にははっきりとした機能の差があることが期待されるため、要素データの指定には識別子を用いて行う。要素データに対しては任意の要素に対して任意の順番で参照と代入ができる。また、構造型データ全体に適用可能な演算操作には代入や等値性判定がある。. 線形リスト(linear list)任意の一つの型に属する複数個のデータをひとまとめにしたデータ構造である点は配列型と同様であるが、要素データの数が定められていない点が異なる。このため、あらかじめメモリ中に記憶領域を固定することは不可能である。また、アクセスできる要素はリスト中の一つのみに限られ、他の要素をアクセスしたいときはアクセス可能な要素から前後に手繰っていかなければならない。. キュー(queue)とスタック(stack)線形リストでアクセス可能な要素を両端に限定することによって、データ構造にアクセスするための計算量をリストの長さと独立にしたものである。線形リストに挿入する端と反対の端から取出すことによって、先入れ先出しにしたものをキュー、挿入する端と同じほうから取出して、先入れ後出しにしたものをスタックと呼ぶ。適用可能な操作としては、要素の挿入・取出し・取出し端の要素の参照のみ・リストの長さの参照などが定義される。xiii. 二分木(binary tree)線形リストの要素が隣接する要素を二つもって線形に並んでいるのに対し、隣接要素を親一つと子二つの合計三つもって木構造をもつようにしたものが二分木である。一般に分木を想定することができるが、二分木はそれらの中で最も単純であり、基本となる構造である。要素を指定するためにはキーを与え、これによって既存の要素を見つけたり、しかるべき場所に要素を追加したりすることができる。二分木に対する演算としては、指定したキーをもつデータの挿入・参照・削除・存在するか否かの確認などが定義される。xiv. ポインタ任意のデータ構造を組合わせて利用できるように、あるデータ構造を特定できるようにした識別子である。多くの場合はデータ構造が存在するメモリ内の位置をもって識別子とするが、データ構造そのものと1対1対応するビット列であればそのパターンは任意のものであって構わない。ポインタに対する演算としては、代入や等値性判定に加え、データ構造を識別するポインタ型の値を求める演算と、ポインタ型の値からそのポインタが示しているデータ構造を取出す演算が定義できる。この最後の演算を参照はがしと呼ぶ。xv. 辞書構造可変長の、もしくは比較的長い固定長のデータをキーとして、それに対応するデータ構造を維持管理するためのデータ構造である。辞書などに必要となるデータ構造であるためこの名前があるが、この構造を実現するときによく使われるアルゴリズムの名前から、ハッシュと呼ばれることも多い。キーが固定長で充分短いものであれば、キーを添字とする配列で実現できるが、自然言語の辞書などではキーは可変長であり、またキーの間隔が広く空くことも多い。このため、キーを短い固定長に、もしくはポインタに変換するために、ハッシュや二分木探索アルゴリズムを併用する。基本演算としては、指定したキーをもつデータの挿入・参照・削除・存在するか否かの確認などが定義される。e. サーチアルゴリズム(search algorithm)さまざまな値を要素にもつ配列の中から、指定した値をもつ要素を捜し出すためのアルゴリズムである。しかし、キーが辞書のように昇順もしくは降順に並んでいることが期待できる場合には、log の二分探索を用いることができる。毛色の異なるサーチとして、文字列中から特定の文字列を見つけ出す問題がある。ファイルの中から特定の語を見つけ出すような問題に用いるものであるが、見つけようとする文字列ごとに一定の下準備をすることでとなるボイヤ・ムーア(Boyer-Moore)のアルゴリズムなどが知られている。f. 木探索アルゴリズム(tree searching algorithm)木構造の内部を操作するためのアルゴリズムである。二分木の場合、二つの子以下も木(部分木)になっているため、すべての要素節を巡回する場合には、二つの部分木の巡回前に節そのものの処理をする場合、二つの部分木の巡回の間にその節の処理をする場合、二つの部分木の巡回後に節の処理をする場合に分けられる。この条件を維持したまま任意のキーをもつ要素節を挿入・削除するアルゴリズムが存在し、要素節の参照が挿入よりもずっと頻繁に行われる場合に用いられる。g. ソートアルゴリズム(sort algorithm)さまざまな値を要素にもつ配列を、その要素の昇順もしくは降順に並べ替えるためのアルゴリズムである。要素の値として同じ値がいくつ重複して現れても構わないため、同じ値をもつもののグループに分類するアルゴリズムともいえる。アルゴリズムは前提として、ソートされていない値が配列内に詰め込まれた状態で開始され、二つの値を比較してそれらを交換するかそのままにするかの操作を繰返す。h. グラフアルゴリズム頂点と辺からなる有向もしくは無向グラフ上での各種の問題を解くためのアルゴリズムである。グラフ内の頂点を巡回して何らかの処理をするアルゴリズムとしては、深さ優先探索と幅優先探索がよく知られる。深さ優先探索は巡回開始点から開始し、一つの頂点を訪れて処理を行ってはそこから辺でつながっている頂点をすべていったんスタックに入れ、次にはスタックから取出した頂点を訪れる、という動作をする。この動作はとにかく一つの道を突き進み、突き当たったら一つ引き返して別の辺をたどる動作になる。幅優先探索はある頂点からつながっている頂点をキューに入れるところが異なり、探索開始点から辺をたどる回数の少ない順に巡回することとなる。グラフ特有の問題として、グラフの連結性・二重連結性・強連結性を確認するアルゴリズムなどが知られている。また、辺をたどるコストも与えられたときに、最短経路やパイプラインとして見た場合の最大流量、あるいは最小木を求めるアルゴリズムなども有用である。
