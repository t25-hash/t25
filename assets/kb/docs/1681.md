# 1・3・2 プログラミング言語

> **章**: 第1章 情報システム
> **節**: 1・3 計算機ソフトウェア

プログラミング言語とは、問題を解いて解答を得るまでの手順を書き下したものであるプログラムを記述するための、アルゴリズム記述用の言語体系である。計算機システムのハードウェアに1対1に対応した機械語もプログラミング言語の一種であるが、ここでは一般に高級言語と呼ばれる、ハードウェアに対する依存性が少なくアルゴリズムの記述に的を絞った言語群を紹介する。a. コンパイラ(compiler)とインタプリタ(interpreter)プログラミング言語の実行形態による分類であり、実行前に可能な限りの下準備をして効率的な機械語コードを用意して実行速度を最適化するのがコンパイラであり、一方あまり下準備を行わずに実行時にしか得られない情報をも生かして柔軟な処理を目指すのがインタプリタである。インタプリタでは、あらかじめの下準備などができないため実行効率は上がらないものの、プログラムの断片をデータの形で保持して編集しながら実行するなど、コンパイラには困難な処理も実現可能である。なお、コンパイラとインタプリタを総称して言語処理系(lan-guage processor)、もしくはランゲージプロセッサと呼ぶ。b. 変数のスコープと可視性プログラミング言語の多くは情報を記録する目的で変数を用いる。変数にはライフサイクルがあり、プログラムの実行に従って新しく生成されたり破壊されたりするが、この生成から破壊までをその変数のスコープ(scope)と呼ぶ。また、変数は名前によって識別されるが、実行中のいろいろな場面によって同じ名前が指し示す変数が異なる場合がある。存在している変数にその名前で参照可能であることを、その変数が可視であるという。複数の変数が同じ名前で参照される場合には、可視である変数はつねにそれらのうちの一つに限られる。変数のスコープの範囲内であっても可視でなければその変数を参照することができないが、その後可視になればまた参照することが可能になる。7―14情報・ソフトウェア変数のスコープは、プログラムの構造から決定できる場合と、実行時に初めて決定できる場合とがあり、前者を静的なスコープ、後者を動的なスコープと呼ぶ。CやPASCALなど手続き型の言語の多くは静的なスコープをもち、一方LISPの方言のうちの多くは動的なスコープをもつ。c. 関数引数とFUNARG 問題関数を引数として関数に渡すことのできるプログラミング言語においては、引数として渡す関数の内部の変数の参照をいつの時点の束縛に基づいて行うかが問題となる場合がある。すなわち、引数とされた関数の内部にある変数の値は、その関数が引数として引き渡された時点の値とするか、その引数が引き渡された先の関数の中で利用された時点の値とするかの選択が可能である。この二つの選択肢はある束縛の異なった時点での値のこともあり得るし、あるいは全く異なった変数がたまたま同じ名前で参照されているにすぎない場合もある。この問題を解決するために、関数引数を渡す際にはその関数を評価する環境を関数と組にして渡すことが行われる。これをクロージャと呼び、この中に関数そのものと関数の中から参照される変数の値を決定するための束縛に関する情報とをセットにして埋め込む。d. 手続型言語問題を解く過程をステップを追って記述する型のプログラミング言語であり、よく知られるプログラミング言語の多くがこの型に属する。ここでいう手続き(procedure)とは、解答を得るための手順を順を追って書き下していくことを意味し、原則として書き下された手順を上から順に実行していくが、制御構造と呼ばれる問題を解くための操作そのものではなく、問題を解く手順を制御する構文によって、書き下された手順の中の任意の場所から下へ順に操作するように指示したり、ある条件が成り立ったときだけ行う操作や、何回も繰返して行う操作などを指定できる。また、同じような手順が毎回少しだけ異なるデータに対して何度も繰返して行われるときには、これを別のプログラムの単位として独立させて扱う。この別単位のことを手続きもしくはサブルーチンと呼び、何らかの値を返すときには関数とも呼ぶ。. FORTRAN数値計算をおもな目的として設計されたプログラミング言語であり、その歴史は1956年にさかのぼる。その後も絶えず拡張が行われ、ISOで国際規格となり、また日本では1967年にJIS として規格化された後にもFORTRAN77やFORTRAN90などが発表され、後に開発された他の言語に備えられた種々の機能を取込んだ。ここでは現時点で最も広範に用いられていると思われるFORTRAN77を中心に述べる。プログラムの全体構造はMAIN プログラムと関数やサブルーチンなどから構成されており、大局変数は専用の宣言文によって宣言される。数値計算を目的としているため、いろいろな精度の数値データ型や、それらに関する数値演算関数などが豊富に含まれている。しかし、あまり非数値演算向きではなく、データ型の定義メカニズムやポインタ型などは実装されていない。制御構造は任意の点に制御を移すGO TO文や、条件的な実行を行うIF文、繰返しを行うDO文などが主であり、最近のコンパイラになって他の言語からの影響を受けて構造をもつ文などが書けるようになった。非常に長い歴史と伝統をもち、現在でも巨大な数値計算ライブラリはFORTRAN 向けになっているものが多い。特に並列計算機向きのコンパイラにはFORTRAN がよく使われている。. ALGOLアルゴリズムの記述を目的として設計された、FORTRAN と同様に非常に古いルーツをもつプログラミング言語で、ALGOL60やALGOL68などが著名である。アルゴリズムの教科書などで、特定のプログラミング言語にとらわれずにアルゴリズムの記述を行いたい場合には、ALGOL風の記述をするのが通例となっている。すなわち、制御構造を表現するために内部に他の文を抱え込むような構造をもった文や、beginとendで複数の文をくくって文法上では一つの文として扱えるようにする複合文など、そして現在はポピュラーになったポインタ型などはALGOLの流れを汲むものである。特にALGOL68では自動的な参照はがしの機能があり、代入文の左右などでデータの型がポインタ型になっているときには、代入がうまく行われるようにポインタの指しているものの内容が自動的に選択される。. PASCALプログラミング教育を目的に設計されたプログラミング言語であり、現在でも多くのプログラミングやアルゴリズムの教科書がPASCALで記述されている。またPASCALのコンパイラをPASCALで書いたものが、初めはPASCALの移植性を高めることを目的として作成されたようであるが、コンパイラの初学者の間でよく参照される。プログラムの全体構成は、関数などの定義が何重にも入れ子構造になったもので、定義される関数や変数の名前の可視性はこの関数などの入れ子構造によって静的に決定される。また単純だが強力なデータ型の定義機構をもち、実行時のチェックの厳しさもあって、プログラミングの演習程度の問題であればコンパイルエラーが取れればまず正しい答えが得られるともいわれた。最初は教育支援に徹して分割コンパイルなどは言語仕様に含まれていなかったが、次第に広く使われるようになっていろいろな拡張を含む方言ができ、最終的にはOS の記述に使われた例もある。. CUNIX オペレーティングシステムの記述用言語として開発されたプログラミング言語である。どちらかというと専門家に近い人達が今まで機械語で書いていたようなプログラムを、移植性良く記述できるようにするのを目的として設計されているようで、ビット操作などのハードウェアを幾分か意識した演算子をもっている。. ADAアメリカ国防総省が軍事用ソフトウェアの記述用に開発したプログラミング言語である。大筋ではALGOLやPASCALと同じ流れに属する言語であるが、いろいろなソフトウェアパッケージを作るためのモジュール機構やリアルタイムアプリケーションなどのための機能が強化されている。また、ADA コンパイラを名乗るためにはアメリカ国防総省の仕様適合性試験を受ける必要がある。e. 関数型言語プログラムを入力データから出力データを生成する関数ととらえるもので、原理的には変数という概念をもたない。また、関数の定義を他の関数の呼出しで行うため、プログラムの制御の流れという概念もなく、条件分岐や繰返しといった制御構造もない。. LISP二分木構造を基本とするリストデータ構造を扱うことを目的としたプログラミング言語である。その最大の特色は、プログラム自身や変数の束縛に関する情報などもデータと全く同じリスト構造で記述され、自身を自己修飾するようなプログラムを記述することができることである。このため、演算式を処理の対象とする数式処理や、ある条件が成り立ったときにとるべきふるまいなどを記述しなければならない人工知能の分野などで古くから利用されており、研究の内容や進展に応じてさまざまな必要性から思い思いに拡張が施され、非常に多くの方言が存在するに至っている。f. 論理型言語述語論理(predicate logic)に基づく処理を行うために設計されたプログラミング言語で、オブジェクトどうしの間に存在する関連を記述する表現である述語を中心として記述される。. PROLOG1階述語論理に基づく推論を行うことを目的として設計されたプログラミング言語である。PROLOG による推論に特徴的なものとして、ユニファイと呼ばれる操作がある。知識を組合わせて新しい知識を作り上げるルールには、当然変数を用いることになるが、推論の進行に応じて変数に値を割当てる操作が必要となる。この、述語論理での値の割当てに相当する作業をユニフィケーションと呼ぶ。ユニフィケーションでは変数に定数を割当てるだけでなく、変数に他の変数を割当てるような操作も行う。また、推論を行ううえでは、数多くの述語表現を組合わせて利用し、それらの引数のユニフィケーションを行う必要があるが、つねに的確な述語を選択できるとは限らず試行錯誤的な動作をする必要がある。このために必要な動作はバックトラック(back track)と呼ばれ、処理の過程でその少し前の時点にまで後戻りし、その間に行ったユニフィケーションなどもすべて取り消して、新しい選択を開始できる状態にする必要がある。PROLOG ではバックトラックで必要となる情報の回復はすべて処理系が行い、プログラム中にはバックトラックにかかわる記述をする必要がない。g. オブジェクト指向型言語オブジェクト(object)とは情報世界に存在するあらゆる事柄を表現するモデルであり、内部のデータ構造と自分自身に対して適用可能な演算操作の集合とから構成される概念である。オブジェクト自身は他のオブジェクトの内部データを見ることはできず、必ず相手のオブジェクトとメッセージをやり取りすることによって必要な情報を入手しなければならない。オブジェクト指向型言語はこのような計算モデルに忠実に作られた言語である。一般にオブジェクトの内部データ構造と適用可能な演算操作集合とを定義したオブジェクトをクラスと呼び、データ構造内部にいろいろな値をもったオブジェクトをインスタンスと呼ぶ。. Smalltalk-80オブジェクト指向型言語として初めて世に出たプログラミング言語である。すべての演算操作をオブジェクトどうしのメッセージのやり取りでモデル化したのみならず、すべての制御構造をもメッセージのやり取りで定義している。このために、Smalltalk-80ではブロックと呼ばれるプログラムの断片をオブジェクトとしたものが使われ、繰返しはブロックに対してその実行を要求するメッセージを繰返し送信することで、条件実行はある値に応じて複数のブロックのうちの一つだけに実行要求メッセージを送ることで実現する。
