# 1・4・1 ネットワークアーキテクチャ

> **章**: 第1章 情報システム
> **節**: 1・4 ネットワーク

複数の計算機を通信媒体で接続することで、互いにデータのやり取りをしたり、それらに接続された高価な周辺機器を共用することができる。このような通信媒体で接続された通信網をネットワーク(network)と呼ぶ。また、ネットワークを介して通信を行う際に定められる通信手順(通信規約)をプロトコル(protocol)と呼び、プロトコルを含めたネットワーク全体の体系化された構成をネットワークアーキテクチャ(network architecture)と呼ぶ。初期のネットワークは、主要な大型計算機メーカが自社製品を相互接続する目的で独自に作成したため、他社製品との相互接続は考慮されていなかった。しかし、ネットワークの普及とともに、全世界的な規模でネットワークを接続することが要請されることとなり、異なった製品間での接続が可能なネットワークアーキテクチャが不可欠となった。そこで、国際標準化機構ISOにより、OSI(open systems interconnection、開放型システム相互接続)基本参照モデルが制定された。a. OSI 基本参照モデルOSIでは、図1・18に示すように、ネットワークの構成を七つの層(layer)に分割(階層化)し、それぞれの層の機能や役割、そして、各層間でのプロトコルを規定している。OSIにおける各層を一般化すると図1・19のようになる。第層(=1~7)の処理を行う実体はエンティティ(entity)と呼ばれる。これを提供するため、エンティティは通信相手の第層にあるエンティティと情報のやり取りを行う必要があるが、これを行うための論理的な通信接続をコネクション(connection)と呼ぶ。そして、第層間どうしで行う通信手順をプロトコルと呼ぶ。プロトコルで用いられる通信データの単位は、-PDU(pro-tocol data unit)と呼ばれる。この-PDU は、図1・20に示すように、上位層から送られてきたデータである-SDU(servicedata unit)に、第層の制御用に用いるデータである-PCI(protocol control identifier)を付加することで構成される。この逆に、下位層から通信データを受取ると、-SDU と-PCIに分アプリケーション層プレゼンテーション層セッション層トランスポート層ネットワーク層データリンク層物理層図1・18 OSI基本参照モデル図1・19各層の基本機能7―18情報・ソフトウェア図1・20 データユニット割され、前者が上位層に渡される。ここで、第層では-SDUを透過的に、すなわち、第層で-SDU を参照したり、変更することなく、そのままの形で上位層に受渡すことが求められる。これにより、各層が独立となり、ある層を変更した場合でも、他の層にその影響が及ぶことを避けることができる。ところで、-PCIは上位層が送るデータに加えられる冗長なデータであり、OSIでは、実際の伝送媒体にデータが送られるまでの間に合計7層分のPCIがユーザのデータに付加されることになる。PCIのサイズが大きければ大きいほど、ユーザが利用できる伝送容量が相対的に低下することとなる。したがって、PCIは必要最小限の大きさにすることが望まれる。b. OSI における各層の働き以下では、OSIの各層の機能について概説する。ここで、通信をする主体をノード(node)と呼ぶことにする。. 物理層(physical layer)最下位層の物理層は、隣接するノード間でビット伝送を行うための物理的な伝送路を上位層に提供する。このため、伝送媒体の種類や機械的・電気的整合条件を定めるだけではなく、ケーブルのコネクタ形状やピンの配置などについても規定される。この層の代表的なものとして、EIA-232E(旧RS-232C)や、V.35などがある。. データリンク層(data link layer)第2層のデータリンク層は、物理層を用いて、隣接するノード間で透過的かつ高信頼的なビット伝送を提供する。そのため、物理層の機能に加え、データのフレーム化、順序制御、誤り検出や回復、フロー制御などを行う。この層の代表的なプロトコルとしてHDLC(high-level datalink control)が挙げられる。また、このプロトコルを基にして、パケット交換網ではLAPB(link access procedure-balanced)が、ISDN ではLAPD(link access procedure on the D channel)が規定されている。. ネットワーク層(network layer)一般に、通信を行うノードどうしが直接通信回線で接続されるとは限らない。このような場合、いくつかのノードに中継してもらうことで、end-to-endの接続を実現する。ネットワーク層では、このような複数の中継ノードを介在して接続されるノード間に対して透過的かつ高信頼的なビット伝送を提供する。このような接続においては、どの中継ノードをどのように経由するかによって通信効率が変わってくる。この選択を経路選択またはルーティング(routing)と呼び、ネットワーク層における重要な役割の一つとなっている。また、中継ノードだけの働きが求められる場合には、図1・21に示すように、物理層からネットワーク層までの機能があれば十分である。このような装置はルータ(router)と呼ばれ、ネットワークを相互接続する際の重要な役割を果たす。ネットワーク層の代表的なプロトコルとして、パケット交換におけるネットワーク・ユーザインタフェースX.25やネットワーク・ネットワークインタフェースX.75などが挙げられる。. トランスポート層(transport layer)ネットワーク層図1・21 中継ノードを介したネットワーク接続(a)(b)図1・22 多重化と分流では、それが利用している通信回線の種類により通信の品質や信頼性、料金などが異なっている。トランスポート層では、これらの違いを吸収するため、クラス0~クラス4までのプロトコルクラスを設定し、通信回線による違いを意識させずに、高品質で効率の良い通信機能を上位層に提供する。上記を実現するために、トランスポート層に用意されているおもな機能として多重化(multiplexing)と分流(parallelization)が挙げられる。図1・22(a)に示すように、多重化はネットワーク層の一つのコネクションをトランスポート層の複数のコネクションで共用する機能であり、高速なコネクションを複数のユーザで共有する場合などに利用される。分流は、図1・22(b)に示すように、一つのトランスポート層コネクションがネットワーク層の複数のコネクションを同時に使用する機能であり、これにより、一つのコネクションでは実現できない高速なコネクションを提供することができる。また、一つのコネクションが障害にあった場合でもコネクションを維持できるため、障害に強いコネクションが提供される。以上のように、物理層からトランスポート層までは、おもにネットワーク寄りのサービスを提供していた。これに対し、以下で述べる3層はアプリケーション向けのサービスを提供する。. セッション層(session layer)セッション層では、さまざまな種類のアプリケーションが要求する通信手段を提供する。その一つに同期点の設定がある。セッション層では、データの一連の流れを会話(dialog)、いくつかの会話による一つの仕事の単位をアクティビティ(activity)と呼ぶ。例えば、複数のファイルをまとめて転送する場合、一つのファイルを転送する作業が会話に相当し、全ファイルを転送する作業がアクティビティに相当する。一つのファイルを転送するたびに、通信当事者双方で同期を取る必要があり、これを大同期点(major synchronization point)と呼ぶ。また、ファイルの転送途中で小同期点(minor synchronizationpoint)と呼ばれるチェックポイントを設定することができる。これらの同期点を設定することで、あるアクティビティを処理している間に障害が生じた場合でも、アクティビティを始めからやり直すのではなく、障害が生じる直前の同期点から作業を再開することができる。さらに、セッション層ではデータ転送を半二重(half duplex)または全二重(full duplex)で行うことを選択する機能が提供される。ここで半二重とは、トランシーバのように、双方向通信が可能ではあるが、一度に伝送できる方向は一方に限られる通信形態を指す。このとき、双方のノードから同時にデータを送信すると、両者は衝突して失われるため、送信するノードを定めるための送信権制御が行われる。全二重では、公衆電話回線のように、双方から同時に伝送を行うことができるため、送信権制御は行われない。. プレゼンテーション層(presentation layer)プレゼンテーション層は、アプリケーション層が扱うデータの形式を伝送可能なデータ形式に変換することで、効率的に伝送を行う役割を果たす。ここで、アプリケーション層が扱う構文(syntax)を抽象構文(abstract syntax)と呼ぶ。抽象構文は送信するデータを特定のプログラミング言語や計算機に依存しない形式で表したもので、抽象構文を表現する言語としてASN.1(abstract syntax nota-tion 1)が規定されている。プレゼンテーション層が扱うデータ表現形式を転送構文(transfer syntax)と呼ぶ。転送構文は抽象構文で表現された形式をセッション層に渡されるビット列に変換するものであり、例えば、文字コードや整数のバイトオーダ、浮動小数点数の形式などの変換が行われる。また、送信データの圧縮や、暗号化を行う作業もプレゼンテーション層の役割である。. アプリケーション層(application layer)アプリケーション層は、ユーザに対してさまざまなアプリケーションサービスを提供する。このサービスを提供するモジュールはASE(appli-cation service element)と呼ばれ、アプリケーション層は特定の業務処理に対応したASEと、これらのASEに共通なASEの二つの副層から構成されている。ここで、特定業務対応のASEとしては、電子メール(MHS :message handling system)やファイル転送(FTAM : file trans-fer,access and management)などがある。一方、業務に依存しない共通ASEとしては、遠隔操作を行うROSE(remote opera-tions service element)や信頼性の高い転送を行うRTSE(reli-able transfer service element)などがある。c. OSI とインタネットOSI基本参照モデルとは独立に、アメリカ国防総省においてARPA(Advanced Research ProjectsAgency)ネットが開発され、これが今日のインタネット(inter-net)に発展した。OSIが国際標準であるのに対し、インタネットで用いられているTCP/IP(transmission control protocol/inter-net protocol)プロトコルは国際標準ではないが、世界的に広く用いられている事実上の標準(de facto standard)となっている。インタネットで用いられる典型的なネットワークアーキテクチャ図1・23 OSIとインタネット7―19を図1・23に示す。この図に示すように、インタネットでは5層の構成になっており、OSIでのアプリケーション層からセッション層までの役割を、すべてアプリケーション層で行っている。アプリケーション層としては、電子メールを送受信するSMTP(simplemail transfer protocol)や、WWW で用いられるHTTP、ファイル転送を行うFTP(file transfer protocol)などがある。トランスポート層は、コネクション指向型であり、伝送データの内容と順序関係を保証するTCP(transmission control proto-col)と、コネクションレス型であり、TCPのような確実性よりも速度を重視するUDP(user datagram protocol)から構成されている。ネットワーク層は、コネクションレス型で、伝送データの内容や到達確認を保証しないIP(internet protocol)と、その補助プロトコルから成っている。すなわち、ICMP(internet control mes-sage protocol)は、各種制御情報を交換および転送するために用いられる。ARP(address resolution protocol)とRARP(reverse address resolution protocol)は、IPアドレスとデータリンク層で用いられる物理アドレスとの変換を行う。図1・23のデータリンク層は、IEEEによる802標準によるLAN(local area network)を用いた場合を示している。図1・23に示したように、802標準では、データリンク層は接続する媒体に依存しないLLC(logical link control)層と媒体に依存した働きを行うMAC(media access control)層の二つの副層に分かれている。
