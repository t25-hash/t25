# 1・4・2 ネットワークトポロジーと伝送媒体

> **章**: 第1章 情報システム
> **節**: 1・4 ネットワーク

ネットワークにおいては、伝送媒体をどのような構成形態で接続するかにより、通信方式や伝送効率が変わり、また、選択できる伝送媒体が決定される。ネットワークの接続形態はネットワークトポロジー(network topology)と呼ばれ、利用する環境により、適切なネットワークトポロジーを選択する必要がある。また、伝送媒体としては、ツイストペア線、同軸ケーブル、光ファイバなどの有線のほか、無線も用いられる。無線による通信については1・4・3項で触れることとし、本項では有線によるネットワークトポロジーについて述べる。a. ネットワークトポロジー代表的なネットワークトポロジーとして、図1・24に示すスター型、バス型、リング型、そして、メッシュ型がある。なお、図において、○はノードを、□は通信制御装置を表す。. リング型リング型ネットワークでは、図1・24(b)に示すように、各ノードは環状の伝送路(リング)に接続される。あるノードから送られたデータは、リングに沿ってあらかじめ決められた方向に送信される。このデータは必ず隣接されたノードに受取られ、信号レベルを増幅して次の隣接ノードへ送られる。このため、伝送路全体の距離を容易に長くすることができる。また、リング型ではバス型のようにデータが衝突して失われることがないという特徴がある。リング型でおもに用いられる伝送媒体として、ツイストペア線と光ファイバがある。また、リング型ネットワークの代表的なものに、トークンリングやFDDI(fiber distributed data interface)がある。前者についてはb項に示す。後者は光ファイバによる二重リングで構成された、高速で信頼性の高いネットワークである。. バス型バス型ネットワークでは、図1・24(c)に示すように、1本の伝送路にすべてのノードが接続される。あるノードから送られたデータは、この伝送路全体に伝送され、すべてのノードがこれを受取ることができる。あるノードがデータを伝送している間に他のノードがデータを伝送すると、これらのデータは衝突し、両者のデータは破壊される。したがって、バス型では接続されるノード数が増加し、同時にデータを送信するノード数が増えるに従い、伝送効率は急激に低下する。しかし、リング型と異なり、伝送路があいていればデータをただちに送ることができるため、ノード数が十分少なければ、高い伝送効率が得られる。バス型ネットワークの代表的な方式として、トークンバスとCSMA/CD(b,c項参照)がある。バス型で用いられる伝送媒体として、従来はおもに同軸ケーブルが用いられた。これに対してCSMA/CD 方式では、全ノードに共通の伝送路をハブ(hub)と呼ばれる装置に収容し、これと各ノードの接続をツイストペア線または光ファイバで接続する場合が多い。. メッシュ型図1・24(d)に示すように、メッシュ型ネットワークでは通信制御装置どうしを網状に接続した形態を取る。ノード間で通信を行う際、どの通信制御装置を通過するかを決定する、すなわち、データを流す経路を選択する必要があるため、通信の制御が複雑になる。この反面、経路を冗長に用意することが可能であり、通信制御装置や通信媒体の故障に耐性をもつ。メッシュ型は、おもにMAN(metropolitan area network)やWAN(wide area network)といった広範囲にわたるネットワークで用いられる。b. トークンリング以下では、LAN の代表的な方式の一つであるトークンリング(token ring)について述べる。この方式では、リング型ネットワークを対象としているが、バス型でも同様な方式が実現可能である。このバス型による方式は、トークンバス(token bus)と呼ばれる。トークンリングでは、トークン(token)と呼ばれる制御用データがリング上をつねに巡回している。トークンには、ビジートークン(busy token)とフリートークン(free token)の2種類がある。前者は、他のノードがデータを伝送中であることを意味し、後者は他のノードが伝送路を使っていないことを意味する。そして、フリートークンを取得したノードは、データの送信権を獲得する。例えば、図1・25において、ノードaがノードcにデータを送信しようとしているとする。そして、図1・25(a)のように、ノードdからノードaにフリートークンが送られたとする。これにより、ノードaは送信権を得たので、図1・25(b)のようにフリートークンをビジートークンに変えて、これを隣接するノードbに送信するとともに、これに続けて伝送単位であるフレーム(frame)にデータを格納して、これを送信する。ノードbはビジートークンを受取ると、これをそのまま隣接するノードcに送る。そして、引き続き届いたデータフレームを受取り、これが自分宛のフレームであるかどうかを確認する。この例では、ノードb宛ではないので、このフレームもそのままノードcに送付する。一方、フレームの宛先であるノードcがこれを受取ると、フレーム内にあるデータを取得するとともに、同フレーム内の受信フラグを立て、これを隣接するノードdに送る。このようにして、最終的にビジートークンとデータフレームは、フレームを送信したノードaに戻ることになる。このとき、ノードaはビジートークンをフリートークンに変える。そして、それに続くデータフレームを取得し、これが受信相手のノードcに正しく受取られたかどうか、また、フレームが誤りなく戻ってきたかどうかを確認するとともに、このフレームを廃棄する。トークンリングでは、データどうしが衝突して失われることがないため、接続するノード数が多い場合でも伝送効率が低下しづらいという利点がある。また、一つのノードが伝送路を独占しないように、送信できるフレームの最大長が決まっている。したがって、リングの長さと接続ノード数から、次にフレームが送出可能になるまでの最大待ち時間が保証できることが大きな特徴である。しかし、トークンとフレームが1周するまでの間、他のフレームを送ることができない。このため、リング長が長ければ長くなるほど、CSMA/CD 方式と比べ、伝送媒体を十分効率的に利用できないという問題点がある。これを解決するため、ビジートークンが複数存在することを認めるマルチトークン(multi-token)方式も考えられるが、トークンの管理が非常に複雑になる。c. sence)。もし、信号が流されている場合は、これが終了するのを待つ。終了後、さらに9.6ビット分のデータを伝送できる時間(伝送速度が10Mbpsの場合は9.6 s)待っても新たなデータ信号が送られてこなければ、自分のフレームを送ることができる。この時間間隔は、IPG(inter-packet gap)と呼ばれる。ここで、ノードaから送られたフレームは、この伝送路全体に伝搬され、すべてのノードに受取られる。各ノードは、これが自分宛のフレームであればこれを取得し、そうでなければこれを廃棄す(a)(b)図1・25 トークンリング(a)(b)図1・26 CSMA/CDる。また、この性質を利用し、すべてのノードに対してフレームを送信(broadcast)することも可能である。ところで、フレームが伝送路全体に伝搬するまでには時間がかかるため、図1・26(b)のように、ノードaからフレームが送信されているのにもかかわらず、他のノード(図ではノードd)からフレームが送信されてしまう可能性がある。この場合、両方のデータは衝突(collision)し、互いのデータは破壊される。その後、衝突が発生したことを知らせる信号が伝送路全体に送信され、フレームを伝送しているノードはただちに伝送を中止する。衝突発生後、各ノードは以下で定めるランダム時間待ってからフレームを再送する。すなわち、そのフレームの連続再送回数が回めだとすると、一様なランダム値を以下の範囲で選択する。なお、再送は15回めまで認められており、それでも送信に失敗するとそのフレームは廃棄される。また、が指数関数的に増加することから、このアルゴリズムはTBEB(truncated binary exponentialbackoff)アルゴリズムと呼ばれる。ところで、図1・26(b)に示すように、データ送信を開始してから衝突が検出されるまでに、最大で信号が伝送路を往復する時間(往復伝搬遅延時間)がかかる。また、CSMA/CD ではフレームを送り切るとこのフレームの送信に成功したとみなされる。したがって、衝突検出信号は最小フレームを送信し終わるまでに戻ってこなくてはならない。このため、CSMA/CD では伝送路全体の長さの上限が制約される。例えば、伝送速度が10Mbpsであり、同軸ケーブルを使った10Base-5では、500mまでのケーブルをリピータ(repeater)と呼ばれる増幅器で5本まで直列に接続可能である。また、同じ伝送速度で10Base-5よりもやや細い同軸ケーブルを使う10Base-2では、185mまでのケーブルを同様に5本接続できる。一方、伝送路をハブと呼ばれる装置に収容し、ツイストペア線で接続する10Base-T では、100mまでのケーブルで4台までのハブをカスケード(cascade)接続できる。10Base-T は同軸ケーブルよりも軽量で接続が容易なツイストペア線を用いたため、ネットワークの構築が容易になった。また、より高速な伝送速度での接続が求められるようになり、伝送速度が100Mbpsまで拡張された100Base-TX や、ツイストペア線の代わりにマルチモード光ファイバで接続する100Base-FX などが開発された。しかし、伝送速度の高速化により、接続可能なケーブル長がさらに短くなり、例えば100Base-TX では、ハブ-ノード間のケーブルは100mまで、ハブ-ハブ間のケーブル長は5mまでで、ハブのカスケード接続は2台までとなった。さらに、CSMA/CD 方式のハブに代わり、スター型のネットワークを用いたスイッチングハブ(switching hub)が開発された。スイッチングハブにより、CSMA/CD でのフレームの衝突はなくなった。ただし、同一ノードへ同時にアクセスした場合は衝突とみなされる。また、CSMA/CD ではその通信方式から半二重通信しかできなかったが、スイッチングハブによって全二重通信が可能となった。その後、伝送速度1Gbpsを実現した1000Base-SX(マルチモード光ファイバ)および1000Base-LX(シングルおよびマルチモード光ファイバ)や、1000Base-T(ツイストペア線)が提供された。また、CSMA/CD 方式による伝送を完全に廃止し、10Gbpsの伝送速度を実現した10G Base-SR(マルチモード光ファイバ)、10G Base-LR/ER(シングルモード光ファイバ)なども提供されている。
