# 2・2・2 制御アルゴリズム

> **章**: 第2章 メカトロニクス
> **節**: 2・2 ソフトウェア

a. 制御ソフトウェアの構成メカトロニクスシステムの制御ソフトウェアの構成を,図2・3に示す.メカトロニクスシステムの制御ソフトウェアに要求される働きは,当該システムにおいて,その時々の状況に応じ,所望の動作を実現させることである.このためには,システム本体に取付けられたセンサの情報から現在のシステムの状況を認識・把握し,それに基づいて,システムのアクチュエータを適切に駆動させる必要があり,主として全体管理部のもとに,センサ情報入力モジュール,アクチュエータ駆動指令出力モジュール,および,センサ入力情報とアクチュエータの駆動パターンを関係付ける制御演算モジュールで構成された動作制御部と,オペレータとのインタフェース用HMI(human-machine interface)部が用意されている.動作制御部の制御演算モジュール内には,対象とするメカトロニクスシステムに所望の動作を行わせるための各種制御アルゴリズムが組込まれており,この制御アルゴリズムに従って,センサ入力情報からアクチュエータの駆動パターンを生成する演算処理が実行される.一般に,メカトロニクスシステムの制御ソフトウェアでは,上記動作制御部を構成する三つのモジュールより成る制御ループの繰返し計算が高速に実行されている.近年の計算機性能の飛躍的な向上で制御周期の短縮化が可能となり,位置制御ループが数msから数百s程度,モータの電流制御ループは数十sから数s程度で実現されるなど,従来ディジタル処理が困難であった部分についてもディジタル化が進んでいる.. センサ情報入力モジュールセンサ情報入力モジュールは,各種センサで得られた情報を読込む部分であり,アナログ信号やディジタル信号,アップダウンパルス信号等を取込む各種インタフェースのハードウェア動作用ドライバソフトウェアと,取込んだセンサ情報を処理して計算機内のメモリ等へ格納するソフトウェアで構成される.センサ情報の入力処理は,それぞれの入力インタフェースで用意された固有の制御コマンドを制御計算機から順次発行することで,各種センサ情報が制御計算機内に取込まれ,その後,単位換算等が行われる.. アクチュエータ駆動指令出力モジュールアクチュエータ駆動指令出力モジュールは,各種アクチュエータを駆動するための動作指令出力部である.各アクチュエータには通常それらの駆動回路が構成されており,駆動指令の種類に応じて,アナログ信号やディジタル信号の出力インタフェースが用いられる.したがって,アクチュエータの駆動指令出力モジュールは,出力インタフェースのハードウェア動作用ドライバソフトウェアと,出力インタフェースに合わせてアクチュエータ駆動指令値の換算処理を行うソフトウェアで構成される.例えば,電圧指令型の駆動回路をもつDCサーボモータの場合は,駆動回路への入力電圧に比例したトルクや速度が得られるように構成されており,必要な出力を得るためのアクチュエータ駆動指令値が制御計算機で計算され,D-A 変換インタフェースを通してアナログ電圧値に変換された後に,アクチュエータ駆動回路に入力される.. 制御演算モジュールセンサ情報入力モジュールで取込まれた各種センサ情報を利用して,対象とするメカトロニクスシス図2・3メカトロニクスシステムの制御ソフトウェア構成図2・5 モータの位置制御系7―31テムの現在の状況を認識・把握し,所望の動作を行わせるアクチュエータ駆動指令値の生成モジュールである.一般に,メカトロニクスシステムに要求される所望の動作は,アクチュエータを適切に駆動し,システムの駆動部における位置や速度,そこで発生する力やトルクを制御することで得られ,その結果として各種作業が実現される.そのためのセンサ情報(入力)とアクチュエータ駆動パターン(出力)の関係付けが制御アルゴリズムであり,所望の動作実現のための各種制御アルゴリズムが開発されている.制御演算モジュールでは,そこに組込まれた制御アルゴリズムに従って,センサ入力情報からアクチュエータの駆動パターンを生成する演算処理が行われる.b. 制御アルゴリズムメカトロニクスシステムの制御アルゴリズムは,対象とするシステムで実現する所望動作の内容により,駆動部の位置や速度,駆動部で発生する力やトルクを制御する.ここでは,まず最初に基本となる制御アルゴリズムとして,モータの速度制御(velocity control),位置制御(position control)のアルゴリズム,次いで代表的なメカトロニクス機器であるロボットアームを対象とした,力制御,コンプライアンス制御のアルゴリズムについて説明する.また,学習機能により状況に応じた制御アルゴリズムを獲得する手法についても紹介する.. 速度・位置制御のアルゴリズムメカトロニクスシステムの多くは,本体内部のモータを駆動することで,所望の動作を実現している.DCサーボモータは,入力電流値に正確に比例した出力トルクが得られるモータであり,多くの場合,ハードウェアで構成されたモータ出力トルクの制御駆動回路を内蔵している.この回路は,入力信号がトルク指令電圧であることから,モータの回転速度を制御するには,図2・4に示すように,速度指令値と実際の速度検出値との差を求め,それよりモータのトルク指令値を生成すればよい.通常,速度制御には式(2・1)に示すPI制御(PI control)がよく用いられる.τ욃=욋ω욃-ω+욡ω욃-ωd(2・1)ここで,τ욃:トルク指令値,ω욃:速度指令値,ω:速度検出値,욋,욡:ゲインであり,욋,욡の値を適切に設定することで,所望の応答を得ることができる.また,モータの回転角度(位置)を制御するためには,位置指令値と実際の回転位置検出値との差を求め,それよりモータの速度指令値を生成すればよい.速度指令を制御(control)で構成すると,ω욃=욣θ욃-θ(2・2)ただし,θ욃:位置指令値,θ:位置検出値,욣:ゲインとなり,この場合のモータトルク指令値は,式(2・2)を式(2・1)に代入することで,τ욃=욋욣θ욃-θ-ω+욡욣θ욃-θ-ωd(2・3)と求まる.これを図示したのが図2・5である.この制御系は,位置図2・4 system)と呼ぶ.メカトロニクスシステムの速度・位置制御のアルゴリズムは,上記の速度・位置制御系を基本として構成されるが,近年の高性能な情報機器等では,高速・高精度化,小型化を達成するため,振動抑制や外乱補償,ロバスト性向上に対応した,より高度な制御アルゴリズムが適用されている.例えば,磁気ディスク装置や光ディスク装置では,最適レギュレータやカルマンフィルタを用いた制振位置決めや制御によるロバスト性向上などにより,高速・高密度化が実現されている.. 力・コンプライアンス制御のアルゴリズム人間の作業の代替を目的とするロボットアームでは,手先の速度・位置を制御するだけでなく,手先の発生力を制御することで,作業対象に倣った動作やはめあい作業などが可能となり,作業性能が格段に向上する.ロボットアーム手先の力を制御するためには,手先の発生力が所望の値となるように,各関節に配置されたアクチュエータの発生トルクを制御する直接的な方法と,結果として手先の発生力が所望の値となるように,各関節アクチュエータの位置や速度を制御する間接的な方法がある.前者は高い応答性が期待できる反面,安定な制御系を構成するには高速な制御演算処理が要求されるなど,構築が容易ではない.ここでは後者の適用により安定なグラインダ作業を実現したロボット(図2・6)を例に,間接的な力制御(forcecontrol)のアルゴリズムについて説明する.間接的な力制御のアルゴリズムを適用したグラインダ作業ロボット(grinding robot)では,時刻におけるアーム手先の押付け方向目標位置욃욨を次式で与える.욃욨=욞욃-+욃욨-(2・4)ここで,욃욨-:一制御周期前の手先目標位置,욃:力目標値,:力検出値,욞:ゲインであり,押付け方向を正とする.式(2・4)により,押付け力が目標値욃より小さい場合はグラインダを把持した手先をワークに近づける方向に,逆に大きい場合はワークから遠ざける方向に目標位置욃욨が修正されるため,욃욨を実現する各関節アクチュエータ位置θ욃を求め各軸の位置制御目標値として与えれば,結果としては욃に一致し,グラインダを指定した力でワークに押付けることができる.本手法は,速度・位置制御系の外側に力のフィードバックループが設けられた構成となっているため,位置制御型のロボットアームにも適用できる,比較的構築が容易で実用的な手法である.ただし,ここで実現される押付け力の変化量を,手先の押付け方向移動量を욨,ロボットアームのばね定数を욥욌とすると,=욥욌욨(2・5)が成立することから,욥욌が大きくなるとロボットのわずかな動きで押付け力が大きく変化し,制御が難しくなる.この場合には,例えばロボットアームとグラインダの間に硬質ゴムダンパを挿入するなどにより,욥욌を適切な値に調整する必要がある.なお,押付け力の計測は,ロボット手首部に装着した6軸力覚センサの計測値等を用いる.力制御のアルゴリズムでは,作業の内容に応じて式(2・4)中の押付け力目標値욃を適切な値に設定する.グラインダ作業ロボットの場合,押付け方向は욃一定で押付け動作を行い,その垂直方向についてはグラインダの位置を制御する.このように,力制御と位置制御の座標系でおのおの独立のサーボ系を構成する方法を,位置と力のハイブリッド制御法(hybrid 制御アルゴリズムの獲得これまで紹介した制御アルゴリズムでは,メカトロニクスシステムへの適用の際に,対象システムの特性解析や所望の動作を実現するための動作目標値の明確化が必要である.しかしながら,必ずしも対象システムの特性が明確になるとは限らず,熟練作業者の技能の代替を目的とするシステムなどでは,動作目標値の明確な設定も容易ではない.このような場合にも適用可能な制御アルゴリズムとして,近年,学習機能を備えた制御アルゴリズムの研究が進められており,ニューラルネットワーク(neural network)の学習機能を利用したものや,遺伝的アルゴリズム(GA : genetic algorithm)を利用した最適解探索アルゴリズム等が提案されている.これらは,システム自身に状況に応じた動作生成能力の付加を目指したものであり,これらの研究の発展による,状況の変化に適応した動作が可能なメカトロニクスシステムの実現が期待される.メカトロニクスの分野においては,多くのアクチュエータやコントローラが積極的に利用されるようになってきている.2・3節では多数のアクチュエータがある分散構造を形成している場合や,自律性をもった複数のシステムが分散的に協調動作を行う場合を取上げる.分散化によって期待されるのは次のような特徴である.(1) 高信頼性,耐故障性(fault tolerance),自己修復(システムの冗長性に基づく)(2) 自己組織化(self-organization),自己組立(構成要素の自律性に基づく)(3) システムの拡張・縮小性(モジュール構造などに基づく)(4) 高度な適応性(5) 高速動作,即応性
