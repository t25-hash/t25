# 2・3・2 協調方式

> **章**: 第2章 メカトロニクス
> **節**: 2・3 分散化と協調

分散型のハードウェアが機能するためには、全体の統合が実現できるように協調の仕組みを埋め込む必要がある。現実にはハードウェアが分散的な形態をとっていても実質的に集中型の制御部が全体を統括する場合もあるし、上位の制御部が目標を生成し、分散配置された下位の制御部がこの目標に従って並列的に制御を実行していく階層的な場合もある。以下では、主として分散構造の各部が対等な位置付けをもっている場合にどのような手法が提案されているかについて述べる。a. 形の生成多数の要素が自律的にお互いの結合を組換えていくことにより、設計者が意図する形状を実現する手法が提案されている。無機的な例を求めれば、一つの規範は結晶の成長に見られる。結晶においては分子のレベルで相互に結合する腕が存在するが、分散システムを構成するそれぞれのユニットがこれに相当する結合部をもち、ある規則に従ってこの腕どうしが結合するようなシステムが考えられる。細川は均質なユニットに磁石を埋め込み、結合の選択性を実現したうえでユニット群に揺らぎを与えると種々の形状が創成されることを、二次元的なモデルで示した。ホワイトサイズらは多面体の面に簡単な電子回路を作成し、一部の面にはんだを付着させて多数個を高温の液体中に浮遊させ、はんだの凝集力により面どうしが結合して三次元的な構造と電子回路が同時に得られることを実験的に示した。結晶的手法は規則正しい形状を創成するが、人工物の場合はより複雑な形状の創成を要求される場合もある。村田らのフラクタムでは中心モジュールの周りに一層ずつモジュールが結合して形状を創成していく手法を提案している。層ごとに結合部のパラメータを記述し、これを各モジュールに設計図として埋め込んでおくことで多様な形状が生成可能なことを示した。また、さらにモジュールに故障が生じたときの自己修復(self-repair)の方法についても実現の方策を与えている。生物の骨などの機能的な形状の生成に範を求めた例もある。伊能ははりの初期形状を簡単な直方体などで与え、この各部に生じる応力を有限要素法で計算して応力の少ない部分は材料を徐々に省き、応力の大きい部分は材料を追加して補強していく、という方法により、機能的な形状を創成できることを示した。このような適応形状の生成は、梅谷らも成長変形法として提案しており、さまざまな研究が行われている。この方式は計算機内での成長を前提にしたものがほとんどである。実モジュールを用いればセンサによって応力を直接検出できるので、変形可能なモジュールとセンサの組合せを利用することも考えられている。自己組織型のモジュールをもつシステムにおいて、ある初期形状と目標形状を与えられて効率的にモジュールの移動を計画する問題に対しては、決定的な手法は確立されていない。吉田らは三次元問題で大局的なレベルから局所に向けて、順次階層的に形状を目標に収束させる方法を提案した。また、チリクジャンらはシミュレーテッドアニーリングにより二次元問題の準最適解を得る手法を提案している。b. 運動の生成生物の移動様式を作り出して人工物の移動に応用するものとして、ひも状生物の運動を取上げた研究がある。モジュールを関節で直列に結合して一次元の構造を作り、関節角を適当な方法によって制御して頭部から尾部に向かう進行波を作り出すと頭部方向に前進する力を得ることができる。広瀬はほふく生物(蛇など)に想を得て、モジュールを非駆動車輪で支え、体の長手(進行)方向には低摩擦で、これと直角の方向には高い摩擦力が存在するような支持をしたうえで、多数個を関節で接続した。関節角を制御して進行波を発生させると、全体が頭部方向に前進することを実験的に示した。この機構では頭部に正弦信号の発振器を置き、一定の遅れのもとに後部モジュールにこの信号を順に送り、関節角をこの信号に比例させて駆動することで進行波を作り出している。湯浅らは分散した発振器間の位相差をあるパターンに収束させる仕組みについてポテンシャル関数に基づく方法を構成した。モジュール間の角度の制御方法として、ある周波数の発振器をそれぞれの関節に設けたうえで、これらの発振器間の位相差をあるポテンシャル関数の値を減少する方向に制御し、一定の位相差に収束させる方法を示した。関節角がこの発振器の信号に比例して駆動されれば全体として進行波が発生するように設計できる。この方式では隣り合った発振器間の相互作用に基づいて分散的に位相差が定まっていくので、耐故障的な性能も期待できる。この方法は四足動物の歩容(ゲートパターン)の発生にも応用が示されている。飯島・嘉数らは魚に似た一次元の関節で結合されたモジュール群を水上に浮遊させ、学習的な手段で与えた目標点へ移動する運動を獲得させている。c. 判断・分担の協調高度な自律性をもった複数のロボットが作業を行うときには、まず全体が現在の状況を判断して共通の認識をもち、次に相互に情報を交換して必要な協力・分担の実現を目指すという手順が求められる。このような協調については次のような基本的な手法がある。. 黒板モデル(blackboard model)多数の自律型の移動体(エージェント)が相互に通信を行って状況を判断する手法に黒板モデルがある。黒板はエージェントがそれぞれの立場から構成した仮説が格納される共有の記憶場所で、全員がこの内容にアクセスできる。構成員は自分のおかれた状況から得られる観測情報や黒板から得られる仮説を基に、内部にもっている知識(例えばプロダクションルールのような形で与えられる)に基づいて新しい仮説を生成したり変更を加えたりでき、これを再び黒板におくことができる。このように順次分散的に仮説の処理を進め、上位の仮説に収束させて全体の状況の判断を行う。全体の仮説が矛盾をもたないように調停を実行するためにスケジューラが置かれている。. 契約ネット(contract net)エージェントが状況に応じてマネージャ(提案側)と契約者(請負側)に分かれ、マネージャは目的とするタスクを実現するために、これこれの条件で仕事が必要とされているという提案内容をエージェント集団に伝達する。受け取ったエージェントは自分が条件に適しているかを判断し、複数の提案の中から有望な仕事を選択して入札メッセージを送る。マネージャは受け取った入札メッセージの中で全体局面に最も適合するエージェントを選び、落札を通知する。これにより作業の割当てが実現できる。d. 制御方式分散システムを構成するうえで、各部分システムが自律性をもち、かつ全体として合目的的なシステムを構成する要件として、森らは自律分散概念を提案し、この概念に基づいて鉄道システムなどに関連した実システムを構成している。分散型構成をとる制御システムにおいて、そのサブシステムが動作を停止したときに残りのシステムが制御系としての機能を維持できるとき、自律可制御性(autonomous controllability)をもつとし、このための必要十分条件を与えている。このような性質はシステムの破局を避け、オンライン性を維持したまま拡張・保守などを行ううえで重要な性質である。
