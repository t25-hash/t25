# 2・4・3 応用事例

> **章**: 第2章 メカトロニクス
> **節**: 2・4 組込みシステム

a. 自動車組込みシステムの代表例として、自動車の電子制御システムを挙げることができる。電子制御システムは、近年の自動車において極めて重要なコンポーネントとなっており、車両によっては価格の25%を占めるともいわれている。自動車における電子制御の要求は、安全装備、環境対策、情報支援などの面で日増しに高まっており、ABS(anti lock brake system)、エアバッグ、エンジン制御、排ガス制御、トラクション制御、電子制御サスペンション、NAVI,ITS など多岐にわたる処理を行わねばならない。これらの中では、エンジン制御がリアルタイム性において難しい対象である。トヨタ自動車の例を挙げれば、従来は独自の経験的手法で、OSを用いずに割込みとフラグを組合わせることで制御を行っていたが、1999年ごろからリアルタイムOS が導入されている。これは、従来のソフトウェアの再利用が難しく、設計、品質管理、信頼性検証のコストが大きくなったためである。リアルタイムOS の導入に当たっては、既存のITRON 仕様を検討した結果、エンジン制御の面から不足な点があったため、トロン協会に提案、仕様の策定が行われ、自動車制御用プロファイルが標準化された。エンジン制御における問題点にはOS のオーバヘッドがあり、これは1%以下が必要であること、またカーネルのメモリ使用量の制限は32ビットCPU でROM が数kB,RAM が数百バイト以下というところである。また、応答時間も非常に厳しく、数sから数十sの精度が要求される(30°間隔のクランク角割込み信号は、エンジン回転数8000回転のとき、周期625 sであり、割込み処理のOS オーバヘッドは6.25 s以下)。これらの要求を満たすために、カーネルの仕様はITRON 4.0のスタンダードプロファイルより、かなり小さく設定されている。なお、自動車の組込みシステムを対象とする標準仕様として、1995年にドイツとフランスの自動車産業界から提案されたOSEK/VDX がある。OSEK はドイツ語でopen systems and the corre-sponding interfaces for automotive electronicsの頭字語であり、VDX はvehicle distributed executiveの略である。OSEK/VDXは、自動車に搭載されたECU(electronic control unit、電子制御ユニット)をネットワーク接続する際のプログラムインタフェースの標準形式を定めており、次の三つの仕様からなる。(1)OSEKOS は、リアルタイムOS の仕様であり、(2)OSEK COM(com-munication)は、ECU 内/ECU 間の通信仕様、(3)OSEK NM(network management)はネットワーク管理の仕様である。ECU のソフトウェアの高度化に対応するため、異なるメーカによるユニットの相互接続化やソフトウェアの部品化を行うことによって、ソフトウェアの再利用性、移植性を向上し、開発期間とコストを低減することを目標としている。自動車の組込みシステムで用いられる車載ネットワークの標準仕様としては、CAN(controller area network)がある。これは、ドイツのボッシュ社(Robert Bosch GmbH)が提唱している車載用のネットワーク仕様であり、ISO規格(ISO 11898:1993)として事実上の業界標準となりつつある。CAN は最大1Mbpsで動作する2線式のシリアルバスであり、車載用の特徴として、温度・湿度の許容範囲が広く、耐ノイズ性、電磁放射特性、耐振動性などに優れるため、自動車以外の環境条件が厳しい領域でも利用されている。7―38情報・ソフトウェアb. FA、ロボットなどFA 機器においては、多様な生産機械(加工、組立、搬送など)のそれぞれの機能を制御する部品として、PLC、サーボアンプ、インバータ、主軸アンプ、NC装置などが用いられている。これらに組込まれるシステムは、専用ハードウェアや1チップマイコンからDSP、はん用CPU に至るまで極めて多様な形態をとる。いずれの場合も、ハードウェアに特化された構成、リアルタイム性、コスト制約などの中で設計されるが、OS を用いずに組込みソフトウェアが搭載されることも多い。しかし、制御ソフトウェアの機能の高度化や複雑化、高度なマンマシンインタフェースの搭載、他の制御機器とデータ通信を前提としたCIM(com-puter integrated manufacturing)の構成での利用が要求されるに従って、リアルタイムOS を搭載する必要は高まる。近年のNC装置や産業用ロボットなどにおいては、リアルタイムOS を用いずに高機能を実現することは困難である。近年開発された二足歩行ロボットである本田技研工業のASIMOでは、リアルタイムOS のVxWorksが利用され、10以上のタスクの並行処理が利用されている。またソニーの二足歩行ロボットSDR シリーズでは、リアルタイムOS のAperios(ソニー製)が用いられ、SDR-4X では38箇所の関節の制御と画像・音声認識制御が2台の64ビットCPU で実現されている。1997年にNASAJPLが火星に着陸させた自動探査車ソジャーナ(Sojourner)では、その組込みCPU として、放射線耐性が強化され、かつ消費電力も小さい80C85が搭載され、リアルタイムOS としてはVxWorksが利用された。また、国産の人工衛星では、データ管理用(無線通信を含む)と姿勢軌道制御用のCPU が使用されることが多いが、ITRON を搭載した100MIPS クラスの高性能CPU により、統合制御システムを構築することも可能となってきている。c. 個人用情報通信機器個人用情報端末PDA(personal dig-ital assistance)の組込みOS では、以上の例と比べるとリアルタイム性の程度は少し緩和される。通信処理やマルチメディア処理などの優先度の高いタスクに起因して、GUIなどで処理遅延が発生しても致命的な問題とはならないからである。PDA のOS としては、Palm OS が著名である。Palm OS は、容量が小さく、高速に動作すること、かつ低消費電力で長時間使用できることを重視したOS であり、ユーザインタフェースに特殊な書き順の省略アルファベット(Graffiti)文字入力を採用しているなどの特徴がある。Palm OS 5.0では、ARM プロセッサ対応の32ビットOS となって速度が向上した結果、動画などのマルチメディア機能も使用可能となっている。PDA では、このほかにWindows CE.NET ベースの標準プロファイルであるPocket PC2002がPalm OS に次いで多く利用されている。一方、国内の携帯電話では、NTT ドコモのPDC機にITRONが採用されているが、今後の計画として、Linux の搭載を発表するメーカもあり、アプリケーション開発の効率などと関係してOSの選択も重要となってきている。ヨーロッパやアジアを中心に広く普及(全世界の7割以上)しているGSM(global system formobile communications)方式を進める多くのメーカは、web参照などの通信機能を有するスマートフォン(2.5世代や第3世代の端末)では、Symbian OS を採用する可能性が出てきている。上記のPalm OS、およびWindows CEベースのSmartphone 2002も、PDA 機能を内蔵する高機能スマートフォンのOS としての利点を主張している。これらのはん用的特性を有するOS の採用が予想されているのは、通信処理を行うベースバンドプロセッサに加えて、今後、アプリケーション専用のプロセッサも搭載されるのが通常となり、アプリケーション開発の負担が大きくなるためである。2002年時点で、携帯電話のソフトウェアサイズは最大16MB 程度(3G
