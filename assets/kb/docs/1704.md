# 3・2・2 パターンマッチング

> **章**: 第3章 ロボティクス/知能システム
> **節**: 3・2 パターン処理

パターン処理問題では、標本データがあることが前提となる場合が多い。この標本データを基に、いかにクラス分けするかという指標を決める作業が必須である。この指標はモデル、テンプレート、7―50情報・ソフトウェアあるいはプロトタイプなどと呼ばれる。いったんこの指標が決まれば、あとは観測データをその指標に照合(マッチング)することにより処理が進められる。さまざまなバリエーションの処理方法があり、ここに挙げる手法のほかにも有効な手段として、パーセプトロン(perceptron)や人工ニューラルネットワーク(artificial neu-ral network)、SVM(support vector machine)等が挙げられる。a. 最近傍則、最近傍則最近傍則(nearest neighbor rule)や最近傍則(-nearest neighbor rule)は、母集団の分布を推定しない非パラメトリックな手法で、決定境界を直接求める手法である。このとき、を識別するのに、′のラベルを用いるというのが最近傍則である。また、最近傍則ではの最近傍から個の′のラベルを調べて、その中でも数の多いラベルをその識別結果とする。b. テンプレートマッチングテンプレートマッチングは、最も簡単なパターン識別方法の一つである。あらかじめ、テンプレートとなる二次元部分画像をいくつか用意しておき、識別対象となる画像を得た後に、テンプレート画像がその画像中に含まれるかどうかを調べる方法である。計算量は多いが、演算をハードウェアで実装するのが比較的容易である。また近年、いわゆるマルチメディア命令セットをもつCPU では、この種の計算を得意としているので、テンプレートマッチングは、ますます適用が容易になってきた。なお、単純なテンプレートマッチングは、回転や拡大などゆがんだ画像の画像認識等に弱いので、あらかじめ想定された変形を施したテンプレートを用意しておくか、他の尺度の照合手法を用いなければならない。c. 記号的パターンマッチング手法記号的パターン処理の中でも最も基礎的なものとして、記号の羅列(文字列)の中から候補文字列を探し出す問題が挙げられる。この文字列発見問題は単純ではあるものの応用範囲は広い。ボイヤ・ムーア(Boyer-Moore)アルゴリズムは、非常に高速な照合アルゴリズムであり、単純な文字列照合アルゴリズムに比べて文字列比較操作を大幅に削減することに成功している。また、記号パターンのモデルとして文法規則が記述できるならば、オートマトンを作成することにより、効率の良い照合器が作成できる。正則表現(regular expression)は決定性有限オートマトン(deterministic finite automaton)に、文脈自由文法(contextfree grammar)はプッシュダウンオートマトン(push-downautomaton)によって、それぞれの規則を受理するオートマトンが作成可能である。正則表現や文脈自由文法を扱うソフトウェアライブラリは広く普及しており、それらを利用することにより、容易に開発することができる。その他、一般的な記号的パターン処理を行う際には、プログラミング言語自身にあらかじめ単一化(unification)やバックトラック機能が備わっているような言語(例えば論理型言語Prolog)を利用することにより、より早く開発できる場合もある。d. の学習には、バウム・ウェルチ(Baum-Welch)アルゴリズムあるいは前向き・後ろ向きアルゴリズムと呼ばれる方法がよく使われる。学習の手法についての詳しい説明は文献[例えば文献(16)の3.10など]に譲る。e. 決定木いくつか質問を用意しておけば、それらの答えによって、パターンを識別することができる。質問に対して“yes/no”,“真/偽”,“値∈集合”のスタイルで答えることができるので、非計量的データでも有用である。このような一連の質問は決定木(decision tree)として表示される。決定木が得られていれば、観測データに質問を適用しながら木をたどり、識別することは容易である。訓練データを使って決定木を作る方法として、CART(classification and regression tree、識別・回帰木)やID 3,C4.5などが提案されている。
