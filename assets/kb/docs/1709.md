# 3・3・1 項ではいかにして知識を表現するかとい

> **章**: 第3章 ロボティクス/知能システム
> **節**: 3・3 知能処理(人工知能)

う問題について述べたが、3・3・2項ではそのようにして計算機上に表現された知識をいかにして利用するかという問題、すなわち推論(reasoning,inference)について述べる。推論という行為を極めて一般的に定義すれば、既存の知識(事実)から新しい知識(結論)を導き出す過程ということになるが、実際の推論法は利用する知識表現法に強く依存し、完全に分離して議論することはできない。また、推論は探索(search)や3・3・5項に後述する学習(learning)などの事項とも関連が深い。以下では、これらの関連事項にも触れつつ、代表的な推論方法のいくつかを取上げ説明する。b. プロダクションシステムにおける推論プロダクションシステム(production system)は、おもにIF-THEN ルール形式で記述されたルールベース(rule base)、推論の状態を保持するワーキングメモリ(working memory)、およびインタプリタ(inter-preter)と呼ばれる推論機構の三つの要素から構成され、ワーキングメモリ上でルールを適用しながら初期状態からゴール状態までの経路(ルールの適用系列)を探索することによって推論を行う。より具体的には、この探索に基づく推論は、(1) ワーキングメモリ上の現状態と、ルールベースに含まれる各ルールの前提部(IFの部分)とを照合する(認識フェーズ)。(2) 照合のとれた(すなわち、適用可能な)すべてのルールの中から一つを選択する(競合解消フェーズ)。(3) 選択されたルールをワーキングメモリに適用する(動作フェーズ)。の三つを繰返すことによって実現される。プロダクションシステムは3・3・3項のエキスパートシステムの基礎となっている。c. 論理による推論
