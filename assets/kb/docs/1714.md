# 3・3・5 機械学習・データマイニング

> **章**: 第3章 ロボティクス/知能システム
> **節**: 3・24 ).

a. 概要機械学習(machine learning)は3・3・1項および3・3・2項で述べた知識表現や推論などとともに,人工知能において最も活発に研究されてきた分野の一つである.機械学習は文字どおり機械(計算機)による学習の実現を目的とする研究であるが,より一般的には経験によって自動的に性能を向上させる計算機プログラムを構築する方法に関する研究として定義することができる.一方,データマイニング(data mining)は電子的に蓄積されたデータの中から有益なパターンあるいは知識を計算機によって自動的もしくは半自動的に発見する過程として定義される.近年,計算機や大容量ストレージデバイスなどのハードウェアが安価になるとともに,インタネットに代表される巨大な情報源が出現するようになった.その結果,われわれは膨大なデータや情報を容易に入手することが可能になったが,そのデータの量は爆発的な勢いで増加し続けており,もはや人間が直接見て理解できる範囲をはるかに超えている.データマイニング研究が近年非常に大きな注目を集め,急速に発展しつつあるおもな背景には,そのような状況の解決策としての高い期待があると考えられる.したがって,データマイニング研究は従来の機械学習と比べてより応用指向が強く,また,単に経験によって性能が向上するだけでなく,学習結果が人間によって理解可能な明示的構造知識として獲得されることに重きがおかれる.しかし,実際にデータマイニングで用いられる技術の多くは機械学習研究で発達してきたものであり,両分野で用いられる手法という観点から区別することは必ずしも容易ではない.以下,本項では現在の機械学習およびデータマイニングの研究分野において主要と思われる問題・手法のうち,(1)分類学習,(2)クラスタリング,(3)相関ルール学習,(4)強化学習を取上げる.なお,本項での説明は割愛するが,前述したニューラルネットワーク(3・3・4項)や信念ネットワーク(3・3・2・e項)も機械学習・データマイニングの主要なアプローチである.b. 分類学習分類学習(classification learning)とは,事前にいくつかのクラスに分類された訓練データが与えられたときに,各クラスの一般化された記述を獲得し,未分類のテストデータを適切に分類する規則を学ぶ問題であり,教師あり学習(supervised learning),概念学習(concept learning),帰納学習(inductive learning)などとも呼ばれる.分類学習における各データは,インスタンスすなわち事例の集合7―57によって表され,各インスタンスは属性(attribute)とその値(value)の組の集合によって表現される.また,各インスタンスがどのクラスに属するかを表す変数をクラス属性(class attrib-ute)またはラベル(label)と呼ぶ.訓練データ(training data)中の各インスタンスではクラス属性の値すなわちクラス値(classvalue)が事前に与えられるのに対して,テストデータ(testdata)のインスタンスではこれが未知ということになる.例として,表3・3はある日の天気,気温,湿気,風の強さという四つの属性の値が与えられたときに,その日に洗濯をしたかしなかったかというクラス属性値を予測するという分類学習問題の訓練データ例を示したものである.分類学習は機械学習・データマイニングにおいて最も基本的かつ重要な問題であり,非常に多くの応用が考えられる.例えば,患者のさまざまな症状から病気名を診断する問題や,クレジットカードの使用状況から不正使用を検知する問題などがこれに含まれる.分類学習の手法としてはこれまでにさまざまなものが開発されているが,以下では代表的なものとして決定木学習,事例ベース学習,サポートベクトルマシン,帰納論理プログラミングの四つを紹介する.. 決定木学習決定木(decision tree)とは,図3・27のような木構造によって任意のインスタンスがどのクラスに分類されるかを表現する方法である.決定木の各ノードはそれぞれ特定の属性に関するテストを表しており,入力されたインスタンスをその属性値に従って下層のノードに割り振る.決定木に入力されたインスタンスは根ノード(rootnode)からツリーを順にたどっていき,最下層の葉ノード(leafnode)に到達した時点でその葉ノードに割当てられたクラスに分類される.決定木学習はこのような決定木構造を訓練データから構築し,未分類のテストデータ中の各インスタンスを適切に分類することを目的とする.決定木学習の代表例であるID 3やC4.5では,どの属性によって訓練データを分割したときに最も多くの情報量が得られるかという基準に基づいて,貪欲的かつ繰返し的にノードを展開していくことによって決定木を構築する.決定木学習アルゴリズムは今日までにさまざまな拡張が提案されており,その中には連続値属性や欠損属性値の扱いを可能にするものも含まれる.また,分類学習問題表3・3 事例ベース学習事例ベース学習(instance-basedlearning)は,訓練データを単純に記憶しておき,未分類の新しいインスタンスが提示されたときに類似のインスタンスを探し出して,そのクラス値に基づいて分類を行う学習法である.具体的に,事例ベース学習の一手法である最近傍分類法(near-est neighbor method)では,未知のインスタンスが入力されるとユークリッド距離などの距離尺度に基づいて,既知のインスタンス群の中で最も近いものが選び出され,そのインスタンスのクラス値が未知インスタンスにも割当てられる.また,同様に最近傍分類法(-nearest neighbor method)では,未知インスタンスの近傍の複数個のインスタンスの多数決によって分類が行われる.事例ベース学習は,学習結果をインスタンスそのものによって表現し,ルールやツリーなどの明示的な知識構造を作らないという点で他の分類学習法とは大きく異なっている.事例ベース学習におけるインスタンス間距離の計算には,もし属性がすべて数値属性で標準化されており,かつ全属性が同じ重要性をもつ場合は単純にユークリッド距離を用いることができる.しかし,属性が名目属性(nominal attribute)であったり,属性ごとの重要性が異なる場合には,他の距離尺度を用いたり,属性に対する何らかの重み付けを行う必要が生じる.また,訓練データの量が膨大なとき,すべてのインスタンスを記憶しておくことは計算時間や記憶容量の観点で現実的でない.そのため訓練インスタンスの適切な取捨選択法がもう一つの重要課題となっている.. サポートベクトルマシンサポートベクトルマシン(support vector machine)は,異なるクラスに属する訓練インスタンス集合を最も良く判別するような特徴量(属性)空間内の非線形境界面を求めることによって分類学習を行う手法である.より具体的には,元の特徴量空間を何らかの非線形変換によって別の空間に変換し,その空間における最大マージン超平面(maximummargin hyperplane)を求めることによって行われる.この非線形変換は,カーネル関数(kernel function)と呼ばれる変換先の空間におけるインスタンスどうしの内積値を定義する関数によって特徴付けられる.カーネル関数には,ガウス関数など,一定の条件を満たす関数が用いられるが,どのようなカーネル関数が適切であるかは個々の問題に依存する.サポートベクトルマシンは近年,テキスト分類問題やバイオインフォマティクスなどの分野で盛んに利用され,実績を上げている.. 帰納論理プログラミング帰納論理プログラミング(inductive logic programming)は,1階述語論理による知識表現に基づき,学習対象となる概念(クラス)の事例および背景知識(background knowledge)を入力として帰納推論を行い,対象概念を説明する規則(rule)集合を獲得する概念学習法である.帰納論理プログラミングでは,述語論理を知識表現言語として用いることにより,属性と属性値による知識表現を用いた,前述の他の分類学習手法では表現が困難な再帰などを含む複雑な規則を獲得することができる.また,目標概念の仮説空間を効率的に探索するために背景知識を利用する点も大きな特徴である.帰納論理プログラミングの実装システムの代表例としては,ProgolやFOILなどが挙げられ,電子テキストの分類やタンパク質の構造予測などさまざまな実世界問題に適用されている.実用的観点から見た場合の帰納論理プログラミングの問題点としては,実際の分類学習問題をこの手法で扱える形式に変換するために多くの知識と労力を要する点,および,大規模な問題を扱うためには根本的な処理効率の改善が必要とされる点などが挙げられる.c. クラスタリングクラスタリング(clustering)とは,あらかじめ明示的なクラスが与えられていないときに,インスタンス集合を何らかの類似度基準に基づいて自然なグループ,すなわちクラスタの集合に分類することであり,教師なし概念学習(unsupervised concept learning)などとも呼ばれる.クラスタリングが3・3・5・b項の分類学習と根本的に異なる点は,訓練インスタンスがあらかじめクラスに分類されていないだけでなく,そもそもクラス(クラスタ)の集合自体をもデータから獲得しなければならない点である.クラスタリングもまた分類学習と同様に,具体的な手法についてはクラスタの表現の仕方,評価関数およびアルゴリズムの違いなどによって非常に多様性に富んでいる.代表的なものとしては,まず,平均法(-means method)が挙げられる.平均法は,ユークリッド距離などの距離尺度に基づいて,(1)各インスタンスの最近傍クラスタへの帰属,(2)各クラスタの中心点の更新,という二つの手順を反復的に行うことによってインスタンス集合を排他的なクラスタ集合に分割する.次に,COBWEBに代表される漸次的階層型クラスタリングが挙げられる.COBWEB では,インスタンスが一つ入力されるごとに,カテゴリー有用度(category utility)と呼ばれる評価基準を局所最適化するように階層的なクラスタ構造を更新していくことによって,クラスタリングを行う.また,その他の代表的アプローチとしては,EM アルゴリズム(EM algorithm)やベイジアンクラスタリング(Bayesian cluster-ing)などの統計的(確率的)クラスタリング(statistical cluster-ing,probability-based clustering)がある.統計的クラスタリングでは,各インスタンスは複数のクラスタに確率的に属すると考え,各クラスタに対応する確率分布の混合分布パラメータをインスタンス集合から統計的に推定することによってクラスタリングを行う.d. 相関ルールマイニング相関ルールマイニング(association rule mining)は,与えられたデータベース(インスタンス集合)上において属性間の重要な相互関係パターンを発見することを目的としたデータマイニング手法である.この手法では,属性と値の組(例えば,Temperature=)をアイテム(item)と呼び,二つのアイテム集合間のIF-THEN ルールを相関ルール(association rule)と呼ぶ.例えば,,,を属性,,,をそれらの属性がとりうる値の一つとすると,IF=AND =THEN =は三つのアイテムによって表現される相関ルールである.一般にこのような相関ルールの候補は非常に多数存在するが,相関ルールマイニングでは,各ルールの重要性を図る指標として,(1) 支持度(support)ルールの条件部と結論部がともに成立するインスタンスの全データに占める割合(2) 確信度(confidence)ルールの条件部が成立するインスタンスの中で結論部も同時に成立するものの割合の二つを考慮し,それぞれが事前に指定した値よりも大きいようなルールだけが効率的に抽出される.相関ルールマイニングを用いた大規模データベースからの知識発見は,現在までに,マーケティングや医療など実世界のさまざまな問題に応用されている.e. 強化学習強化学習(reinforcement learning)とは,自律システム(エージェント)が事前知識の乏しい環境下においてそのゴールを達成するための最適な行動決定戦略を獲得することを目的とした学習である.より具体的には,エージェントは各時点ごとに状態(state)を観測し,行為(action)を実行することによって状態を遷移させるとともに,その行動の善し悪しを評価する報酬(reward)を外部から得る.このとき強化学習の目的は,将来にわたって得られる報酬の総和を最大化するような政策(policy),すなわちどの状態でどの行為を選択すべきかの意思決定を獲得することである.強化学習問題を特徴付けるものの一つが遅れ報酬(delayedreward)である.すなわち,エージェントのある時点における行動選択の評価は一般にその時点において獲得された報酬だけでなく,数ステップ後に得られた報酬に対するその行動の寄与も考慮に入れて行われる必要がある.また,学習データを得るための探索(exploration)と学習結果の利用(exploitation)とが同時に進められることも強化学習の重要な性質の一つである.つまり,3・3・5・b項で述べた分類学習問題のように事前にまとまった訓練データが与えられるのではなく,各時点で選択・実行された行動によって次時点で得られるデータが決定付けられる.現在までに研究されている強化学習手法の多くは,エージェントの次時点での状態は現時点での状態と行為にのみ依存するというマルコフ決定過程(Markov decision process)を仮定した環境モデルに基づいたものであるが,その代表的手法としてQ学習(Qlearning)が挙げられる.Q学習では,ある状態においてある行為を実行した場合の割引き報酬(discounted reward)の総和の最大期待値として定義される値(value),)が漸次的に学習される.Q学習以外の強化学習手法としては,Q学習の一般型である時間的差分法(temporal difference method)や,報酬が与えられた際にそれを適用されたルール(状態と行為の組)の系列に分配する利益共益法(profit sharing method)などが挙げられる.
