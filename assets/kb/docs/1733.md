# 4・4・4 VR 空間シミュレーション

> **章**: 第4章 ヒューマンインタフェース
> **節**: 4・14 のように直交した三つのコイルからなる発信機と受信機の組

VR システムで再現する空間のデータ(街並みや物体の形状データなど)は前もってシミュレーションサブシステム中に収められており,これと入力サブシステムで得られた体験者の行動情報を基に計算された出力が出力サブシステムから体験者に提示される(図は大量のデータが必要になり,体験者の行動に対する出力の計算処理にも時間がかかる.しかし,VR 技術はリアルタイム性を前提としているため,時間がかからない簡単な処理を行うと,今度は出力装置で提示されるVR 空間は品質の低いものとなってしまう.そのため,データの作成段階で体験者の行動を予測してデータ量を少なくする方法が取られるが,行動の制約が多いと映画のように決まったストーリーしか体験できなくなってしまう.このように,行動の自由度,リアルタイム性,出力の品質は互いにトレードオフの関係にあり,これらのバランスを考慮してアプリケーションを設計することが重要である.a. 画像の生成手法. レンダリング三次元の空間データから視点位置に応じた平面画像を生成することをレンダリング(rendering)と呼び,VR における画像生成の基本である.VR の三次元データはシーングラフ(scene graph)と呼ばれる木構造の階層をもってオブジェ図4・15 視体積とクリッピングクトごとに管理される.オブジェクトは三角パッチと呼ばれる三角形の集まりで表現され,三次元データには三角パッチの頂点の位置座標と色情報および頂点間の接続関係が記述される.レンダリングにおいては,三次元データが精密であればあるほど生成される画像の現実感は高まるが,それだけ処理時間も要するため,三角パッチの上に画像を張り付けるテクスチャマッピング手法を用いて,簡単な形状で現実感のある画像を生成する.. クリッピング三次元データから平面画像を描画する際には,VR 空間中に視点の位置・方向・視野角(FOV : field ofview)および近点と遠点を決め,図4・15のような視点を頂点として視野角の広がりをもつ四角錐台を設定する.この四角すい台を視体積といい,この内側にあるオブジェクトがレンダリング画像となって表示される.レンダリング計算のコストを減らすため,レンダリング時に視体積の外にあるオブジェクトを排除する手法をクリッピングという.一般にバウンディングボックスと呼ばれるオブジェクトに外接する直方体と視体積の交差関係を判定することでクリッピングを行う.図4・15の例では,視体積内にある立方体とバウンディングボックスが視体積と交差している円すいの二つのオブジェクトが描画対象となる.. レベルオブディテールVR の画像生成は人間にそれらしく見せることを目的としているため,近くにある物体は詳細な三次元形状データを用いて描画し,遠くにあるものは簡易な三次元形状で描画すればよい.このように,視点からの距離に応じてデータを切換えることをLOD(level of detail)といい,前もって用意された複数の詳細度の三次元データを視点から物体までの距離に応じて適当に切換えることで,描画の負荷を低減させる技術である.また,テクスチャ画像のLOD はミップマップ(Mip Map)と呼ばれ,距離に応じて画像の解像度を切換えてテクスチャマッピングを行う.. イメージベースドレンダリング先に解説したレンダリング手法は三次元の形状データが主であり,それを補う形で画像データを用いていたのに対して,画像データを主としてVR 画像を作成する手法がIBR(image based rendering)である.多数の実空画像を積極的に用いることで,三次元モデルでは表現できない質感の高い画像を生成することを目的としている.b. 触力覚の生成手法. 接触判定体験者に触力覚を体験させるには,体験者の指先または感覚提示の作用点を示すポインタとVR 空間中のオブジェクトの接触を判定する必要があり,一般にオブジェクトの形状とポインタの移動ベクトルの交差によって接触を判定する.基本的にすべての三角パッチについて接触判定の計算が必要となるが,判定の高速化のため,画像描画と同様にオブジェクトごとにバウンディングボックスを設定し,バウンディングボックスで粗い判定を行った後に,対象となるオブジェクトで三角パッチごとの接触判定を行う.. 反力計算オブジェクトの変形や表面の摩擦を考慮しない場合,提示する反力は接触面の法線方向の力となり,接触面からオブジェクトの内側に向かうにつれて反力が大きくなるように制御する.この領域が小さいと硬い感覚が得られ,大きいと軟らかく感じる.硬くて滑らかな物体の表面をなぞる感覚を提示するには,ポインタがオブジェクトの内側へ入り込まないように,接触面近くで大きな反力を提示すればよい.しかし,硬い物体の表現では制御周波数が十分速くないとポインタの制御が不安定になるため,安定した力覚提示には1kHz の制御周波数が必要であるといわれている.また,オブジェクトの運動や表面の摩擦など,力学的条件を考慮して作用点に提示する反力を計算すると,オブジェクトの重さや運動するオブジェクトとの接触による衝撃力などの表現も可能である.. 変形計算ii項で示した軟らかさの表現は,押しボタンスイッチを押込むような,接触面の形状が変化せずかつ移動方向が1軸であるものに限られる.より正確に軟らかさを表現するには,体験者の操作によるオブジェクトの変形をシミュレートする必要がある.変形のシミュレーション手法にはばねモデルや有限要素法(FEM)があるが,力覚提示では高速な計算が可能であるばねモデルが用いられることが多い.しかし,FEM のほうが高精度であるため,接触点近傍を細かいパッチで表現し離れた部分は粗いパッチで計算することでシミュレーションの高速化を図る手法もある.c. リアルタイムシミュレーション体験者の動作に応じてVR 空間のデータを動的に変化させるには,物理法則のシミュレーションが必要である.最も基本的なシミュレーションは重力や慣性などの運動を表現することである.これらのシミュレーションはオブジェクト単位で行えばよく,計算負荷も低いため,シーングラフのノードに運動を記述する項目を追加し,描画サイクルごとにオブジェクトの位置や姿勢を変化させることで実現する.一方,流体解析等の一般的な数値解析をVR 空間中でインタラクティブに行うには,リアルタイム性の点で障壁が大きい.一般的なシミュレーション手法は正確な値を得ることを目的としており,リアルタイム性は考慮されていない.現時点ではリアルタイムシミュレーション(real-time simulation)の一般的手法は確立されていないが,いくつかの提案はなされている.b・iii項で示した変形計算も一種のリアルタイムシミュレーション手法である.また,精密な計算結果の間を簡易な補間によってインタラクティブシミュレーションを行う手法も提案されている.d. VR データ作成装置. 広画角カメラ実映像はCG で作られる画像よりもはるかに高い臨場感を与えることができるため,VR 空間の作成に多く用いられる.三次元モデルに張り付けるテクスチャ画像として用いられることが多い実映像であるが,広画角の映像を用いることで実映像のみに基づくVR 空間の構築も可能となる.広画角カメラには,魚眼レンズを用いるもの,円すいミラーを用いるもの,多眼カメラを用いるものがあるが,魚眼レンズや円すいミラーを用いる手法は1台のカメラで広画角画像の撮影を行うため,画像の解像度は低い.多眼カメラを用いることでカメラの台数を増やして解像度を上げることが可能となり,撮影後に複数の画像を連続的に接続することで,広画角で高解像度の画像が得られる.また,多眼カメラには水平方向のみでなく上下方向の撮影も可能なものや,ステレオカメラを多眼に配置したものなどもある.. 広域映像実写画像を基にした広域なVR 空間を作ることを目的として,車両に複数のビデオカメラとGPS,方位センサを搭載したシステムが開発されている.撮影しながら走り回ることで,水平方向360°の画像が,いつ,どこで,どの向きで撮影されたのかという情報とともに記録される.これらの情報を基に,地図情報とリンクした画像データベースを作成しておき,再生時には体験者の位置に応じた映像を引出すことで自由に動き回ることのできる実映像VR 空間が構築できる.7―83. マルチカメラスタジオ広画角画像を撮影するための多眼カメラや広域画像撮影車両もマルチカメラシステムの一つであるが,これらは周囲の映像を撮影することを目的としており,カメラは外向きに配置される.これに対して,マルチカメラスタジオは人物や物体の実写映像を多方向から撮影することを目的としており,カメラは撮影対象を取り囲むように配置される.複数のカメラで撮影されたビデオ映像から,画像間の補間処理によって任意の視点のビデオ映像を作り出すことが可能であり,実映像ベースのVR 空間が構築できる.また,対象とするオブジェクトを背景と分離して抽出した複数の画像から,視体積交差法と呼ばれる方法を用いて,オブジェクトの三次元形状を再現することも可能である.. レンジファインダレンジファインダ(range finder)は物体の形状測定に用いられ,一般にレーザ光を2軸で走査して,レーザ光の反射から対象とする面までの距離を計測することで物体の表面形状を計測する.そのため,光を反射しないものまたは完全な鏡面反射をするものは計測できない.計測距離は数mのものから100m程度のものまであり,誤差は計測距離の1万分の1以下と高精度である.その反面,レーザの走査に時間を要するため,動きのある物体の計測には不向きである.e. ネットワーク複数のVR システムをネットワークで接続すると,複数の体験者があたかも同一の空間にいるかのような体験をすることが可能となる.街並みのモデルの中で立ち話をしたり,機械モデルの組立てを共同で行うなど,VR 装置を介して人と人とのコミュニケーションが可能となる.自然なコミュニケーションを行うには,ネットワークの実時間性が重要な要素となる.音声の会話では,データ転送に必要とされる帯域幅は無圧縮でもたかだか64kbps程度であるが,1秒以上の時間遅れがあるとぎこちない会話になってしまう.また,触力覚を用いたVR 空間共有を行う場合は,0.1秒の時間遅れでも制御が不安定になる.また,ネットワークに接続された先を実空間として,あたかもその場にいるかのような体験をすることをテレイグジスタンスという.例えば,VR システムとロボットをネットワークでつないで,VR システムの体験者があたかもロボットがいる場で作業しているかのような体験をすることである.コンピュータはますます安価になり,小型化している.初期の大型コンピュータが専門家の特殊な装置であったのに対して,現在では一般的な事務機器としてのコンピュータの数が圧倒的に増加して,コンピュータ応用の主流になっている.近未来のコンピュータは,生活のあらゆる場面で使われる日用品として使われると予想されている.初期の大型コンピュータのユーザインタフェースは,コンピュータの専門家が操作することを前提に作られていた.一方,現在のPC(パーソナルコンピュータ)は,一般的なオフィスワーカーが道具として使うことを可能にするインタフェースを採用している.これが,画面の中に表示されたウインドウ,アイコン,メニューを,マウスなどの指示装置を使って直接操作するグラフィカルユーザインタフェース(GUI: graphical user interface)である.その一方で,コンピュータの小型化と低価格化はさらに進み,安価で小型の組込みコンピュータ,携帯・ウェアラブルコンピュータが急速に普及している.その結果,コンピュータ応用分野は,デスクワークからそれ以外の実生活全般まで拡大している.デスクワークをサポートするために設計され,デスクトップメタファ(机上を模したインタフェース)を基調にしたGUIが,実世界の人々の活動のすべての場面に適しているわけではない.ウインドウ,アイコン,メニュー,ポインティングデバイス(WIMP)に頼らないポストGUI(post-GUI)の研究が盛んに行われている.ポストGUIのアプローチは多岐にわたるが,その一つがここで紹介する実世界指向インタフェースである.実世界指向インタフェ7―84情報・ソフトウェアースは,実世界の事物や状況を利用してコンピュータとインタラクションする手法である.実世界の物,人,場所,出来事などにコンピュータ情報を結びつけて提示したり,実世界の物の操作,人の移動などの行動により,コンピュータ情報を操作する.従来のGUIでは,情報の閲覧・操作の場はコンピュータ画面の中にあった.実世界指向インタフェースは,これをコンピュータ画面の外側,人が生活を営む実世界に展開するアプローチである.実世界指向インタフェースは,切り口の違いにより,コンピュータ強化環境,拡張現実(AR : augmented reality),タンジブルインタフェース,ユビキタスコンピューティングなどと呼ばれることもある.本節では,実世界指向インタフェースに関係する研究テーマを,コンピュータ強化環境,タンジブルインタフェース,ユビキタス/ウェアラブルコンピューティングの観点から解説する.ただし,これらは実世界でのインタラクションという大きなテーマを論じる切り口であり,明確な境界線のある分類ではない.
