# 4・6・3 テキスト処理

> **章**: 第4章 ヒューマンインタフェース
> **節**: 4・3 を見るとわかるように、自然言語の文法記述においては、基本

WWW(world wide web)と電子メールを起爆剤とした1990年代後半からの計算機とインタネット環境の普及によって、膨大な量の情報が電子テキストとしてはんらんしている。計算機によるこれらの電子テキストの効率的な処理(またはその支援処理)のことをテキスト処理という。テキスト処理(text processing)は、大量のテキスト情報からユーザが望む情報を探し出す情報検索と、ユーザがそれらの情報の内容を効率良く把握できるように情報を整理する情報の組織化に大別できる。なお、テキスト処理は実用性(実時間性、はん用性)が強く求められるため、前項までに述べた自然言語処理の諸手法をそのまま適用するのは現実的ではない。例えばテキストの検索を考えた場合、ユーザが自然言語文を用いて表現した検索要求の意味や意図を理解表4・4 談話構造の例して、それに適合した内容をもつテキストを提示するシステムができれば理想的であるが、現在の構文・意味解析のレベルでこのようなシステムを作るのは不可能である。よってテキスト処理では、以下に示すように、言語表現の意味や内容を扱う質的なアプローチではなく、言語表現の表層(文字列)から容易に得られる情報を用いた量的アプローチによるさまざまな手法が開発・利用されている。a. 情報検索情報検索(information retrieval)とは、ユーザの検索要求に適合するテキスト(文書)をテキスト集合から見つけ出すことである。そのためには、ユーザの検索要求と検索対象であるテキストを同じ表現形式に変換してから、両者の照合処理を行うことになる。検索システムでよく用いられる表現形式としては、語集合(単語や複合語の集合)や語ベクトル(各語を一つの次元として、その語とテキストの関連度をその次元の値とする)がある。ユーザの検索要求を検索システムで処理できる形式で表現したものが検索質問であり、これは通常ユーザがシステムに入力する。例えば、新宿にある中華料理のレストランに関する情報を検索する場合、語集合による検索質問は「新宿、中華料理、レストラン」となる。また検索性能を上げるために、ユーザの与えた検索質問に含まれる語の類義語・関連語(上記の例だと「歌舞伎町」や「飲茶」)をシソーラスから取出して検察質問に付加するといった、検索質問の自動拡張手法も提案されている。一方、検索対象のテキストも、あらかじめ検索質問と同じ表現形式(語集合や語ベクトル)に変換しておく必要がある。そのためには、索引語というテキストの内容を反映する語を決定する処理(索引付け)が必要であり、この処理は通常検索システムが行う。索引付けには、テキスト中に出現するほぼすべての語句を索引語として用いる方法や、あらかじめ定められた索引語集合から選択する方法などがある。特に最近のテキスト検索やWWW 検索では前者の手法が主流であり、形態素解析を用いてテキストを単語列に分割し、それらの生起位置を転置ファイル(テキストと索引語の対応表)として保存している。WWW の検索(検索エンジン)では、検索対象のHTMLテキスト集合自体が頻繁に変化するため、HTMLテキストの収集も検索システムの重要な処理となる。多くの検索エンジンでは、Webロボットと呼ばれるプログラムによる自動収集を行っている。検索質問と索引付けされた検索テキストとの照合処理に関しては、さまざまな方法(これを検索モデルという)が提案されている。最も単純な方法は、検索質問を構成する語を含むテキストを検索結果とする方法(ブーリアンモデル)である。しかし、この方法では複数の適合テキストが出力されるだけであり、適合の度合いによる順位付けは行えない。順位付けが可能な検索方法もいくつか開発されているが、代表的な方法がベクトル空間法(vector spacemodel)である。ベクトル空間法では、テキストと検索質問の両方を同じ多次元空間における語ベクトルとして表現し、それらの類似度をテキストと検索質問の適合度とする。b. 情報の組織化情報の組織化・整理は、元の情報を加工しない処理(情報のフィルタリングや分類)と、元の情報を加工する処理(情報の抽出や要約)に大別できる。. フィルタリング(filtering)情報検索がすでに蓄積されている情報(テキスト集合)から情報要求が生じた時点で適合する情報を選別するのに対して、すでに与えられている情報要求に適合する情報(テキスト)をそれが得られた時点で選別する処理がフィルタリングである。例えば、毎日送られてくる電子メールやニュース記事から、ある特定の話題やジャンルの情報のみをユーザに提示する処理が相当する。実際のフィルタリング処理では、上述した情報検索の諸手法を援用する場合がほとんどである。7―9 1. 分類ある基準に従ってテキスト集合を二つ以上のグループに分ける処理であり、あらかじめ与えられた分類体系にテキストを振り分けるカテゴリー付与(categorization)と、似ているテキストどうしをグループ化していくクラスタリング(clustering)に大別できる。カテゴリー付与はカテゴリーに関する情報を検索質問とする検索とみなすことができるので、情報検索の諸手法が用いられている。クラスタリングについては、ベクトル空間法の類似度などを用いて、各種のクラスタリングアルゴリズムが適用されている。. 抽出(extraction)あらかじめ決められた情報内容をテキスト集合から抜き出してくる処理である。例えば、新聞記事データベースからある特定の事件の内容を抽出したり、電子メールからスケジュール表やToDoリストを作成するといった処理である。抽出すべき情報内容をテンプレート(フレーム)の形式で表現し、それらがテキストのどのような箇所に現れるかをパターン化して、パターンマッチングによって情報抽出を行うのが一般的である。. 要約(summarization)あるまとまりのあるテキストをその内容を保持したまま短いテキストで簡潔に表現する処理である。人間の行う要約では意味内容が同じになるように短い文章を再構成するが、現在の技術レベルではそのような要約は不可能である。よって計算機による自動要約では、テキストの重要箇所(段落、文、節)を抽出し、それらを連結することによって要約テキストを生成する方法が基本となる(このような手法で生成されたテキストであることを明確にするために、要約ではなくて、抄録と呼ぶ場合もある)。重要箇所の同定では、テキスト中の単語の出現頻度(頻度の高い語を含む文は重要)、テキスト中の位置(新聞記事などではテキスト最初の数文が重要)、パラテキスト(タイトルや見出しに含まれる語は重要)、語彙的連鎖(関連する語句の文間でのつながり)、テキスト構造などの情報を総合して得点付けを行い、あるしきい値以上の箇所が重要箇所と判断される。文献(1)
