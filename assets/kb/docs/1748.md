# 5・3・1 全体の体系

> **章**: 第5章 技術情報システム
> **節**: 5・3 管理システム・企業情報システム

生産システムは、その時代の背景や商品に対する要請によって、図5・27に示すように発展してきている。すなわち、大量生産から多品種少量生産へ、そして最近では、変種変量の生産に移ってきている。それに伴い、生産を管理するためのシステムも発展してい~1950図5・27 生産情報システムの変遷る。それを図5・27に併せて示す。最近の管理・企業情報システムの多くはパッケージ化され、それらを組合わせて使用しているケースが多い。パッケージ化されたサブシステムの構成を図5・28に示す。この構成は、製造業の企業の階層モデルに従って理解するとわかりやすい。製造業の階層モデルとしてはCIMOSA(CIM opensystems architecture)やISOの6階層エンタプライズモデルが有名である。図5・29に6階層エンタプライズモデル(ISO TR10314:1989)を示す。この階層はある企業内における意思決定の階層と見なすこともできる。最近は、工場をもたない製造業(ファブレスファクトリー)や、お互いの得意分野を提供し合って顧客受注から設計、製造、搬送までを分担するような仮想企業(バリューチェーン)概念も出現してきている。しかし、意思決定というプロセスの流れを見れば、どこかでこれらの意思決定が行われており、製造業における意思決定階層としては有効と考える。製造レベルにおける意思決定処理の詳細について見ると、図5・30のように表すことができる。図5・28の各サブシステム(ERP: enter-prise resource planning,SCM : supply chain management,MES : manufacturing execution system)は、技術の萌芽期ではその目的に絞った比較的狭い範囲のシステムであるが、開発が進むにつれ、それらは上位、下位に機能を拡張していき、おのおのが互いに機能を包含し合うようになる。したがって、サブシステム間で機能の重複があり、その切り分けは単純にはできなくなる。表5・1に主目的とする機能の分担を示す。製造業における経営を自動車の運転になぞらえて大ざっぱにまとめれば、ERPとは、メータ等の7―109表示機器であり、トランザクションをリアルタイムで処理した製造、販売に関するデータを管理してそれを可視化する統制機構である。SCM とは、表示機器や進行方向の情報から短・中期的に方向を決めたり、アクセル・ブレーキや燃料補給を行う運転操作であり、運転者が計器の示す需要に合わせて生産量を計画的に制御し、作業指示する機構である。MES とは、実際の路面や方向、速度に合わせたハンドル操作やアクセル・ブレーキの踏込み量の調整であり、これは作業指示の作業内容に応じて実際の設備機器の操作量を統制することである。そして機器制御系は、それらの操作量に応じて車を実際にある方向に向けたり、短期間、安定的に速度を維持する機構であるといえる。トランザクション処理はリアルタイムが基本であり、制御もそれに応じてリアルタイムで行うのが基本である。しかし、企業情報システムでは、システムの反応速度は遅い(時定数が大きい)。すなわち、操作に応じた結果を出すには物理的製造時間や搬送時間は不可欠であり、また停滞や受発注の処理、計画系の処理のようなオーバヘッド時間もかかり、それなりのタイムラグが発生する。企業情報システムの課題とは、製造に必要な物理的時間以外の時間をいかに短くするかであるといえる。
