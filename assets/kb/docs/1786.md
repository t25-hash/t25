# 2・4・2 SoC概要

> **章**: 第2章 電子デバイスと電子回路
> **節**: 2・4 マイクロプロセッサと応用

SoCの概念は、図2・59に示したように、Verilog やVHDLといったハードウェア記述言語で記述したマイクロプロセッサのコアをはじめとした各種の構成要素(IP: intellectual propertyと呼ぶ)をアプリケーションの必要に応じて組合わせ、1チップに集積するもので、次のような手順で設計される。a. 設計手順。 IP の選択Verilog などで記述されたRTL(registertransfer level)の論理ファイルから、所望のLSIを構築するために必要な要素として、マイクロプロセッサのコアや各種IPを選択する。. レイアウト設計上記IPを物理的に構成するセルと呼ぶ要素を集めたセルライブラリ、これらのセルの信号遅延時間を与えるディレイ情報などを使って所望の性能、チップサイズを満たすようにレイアウト設計を行う。ここで重要なことは、必要な要素を集めて並べればよいというわけではなく、RTLの論理ファイルからゲートレベルへ変換する論理合成、さらにそれらを物理的に配置するレイアウト設計へと設計作業が必要で、高性能なSoCであればあるほど設計は難しくなる。そのため、SoCの設計環境、設計技術、設計ツールの位置付けは大変重要となる。SoCにおいては構成要素であるIPがRTLレベルの論理ファイル(ソフトコアと呼ぶ)の場合、これらをゲートレベルに展開する過程で設計力に差が出てくる。効率の良い設計環境と、熟練した設計技術が必要である。しかしながら、マイクロプロセッサコアのように複雑かつタイミング設計の難しいIPでは、レイアウトまで完了したもの(ハードコアと呼ぶ)を用いる場合が多い。ただし、高速動作を要求しないアプリケーションの場合、他の一般のIPと同様にソフトコアとして扱う要求があるため、複雑なマイクロプロセッサといえどもIP化は不可欠のことである。このことは、使用する半導体技術が異なっても、何も変更せずに同一の論理ファイルか8―32電気・電子の基礎、化学の基礎ら同じような設計を行える。b. SoCに使用するマイクロプロセッサ、メモリSoCの中心に位置付けられるのはやはりマイクロプロセッサである場合が多い。マイクロプロセッサをSoCのコアとする場合、ソフトウェア開発のことを考えると、世界に流通したマイクロプロセッサが都合がよい。ソフトウェア開発ツールなどが完備したものでなければならないからである。最近のマイクロプロセッサは、SoCのコアとして活用することを前提として作られたものが多く、ワークステーションなどの高性能マイクロプロセッサとは異なり、低消費電力かつコアサイズを小さくしているものが見受けられる。マイクロプロセッサとともに、SoCではメモリが重要な位置を占める。メモリとしては、SRAM,DRAM、その他の不揮発性メモリがある。c. IP の動向SoC構築の核となるマイクロプロセッサ、メモリなどの最大規模のIPに加え、各種標準インタフェースに対応したコントローラなどのIPが重要な位置を占める。これらのIPにはいろいろの可能性があるが、現状では、標準インタフェースであるPCI,Ethernet,USB などのコントローラをはじめ、マルチメディア分野における画像処理機能として、静止画像JPEG、動画像MPEG 1またはMPEG 2のエンコーダ/デコーダをはじめとして、表2・4に示したように、従来よりマイクロプロセッサの周辺機能として、割込み、DMA、各種メモリのコントローラなどが提供されており、マイクロプロセッサやメモリもIPに位置付けられる。今後、いろいろな形でのIPが開発されることになるが、SoCのさらなる発展にとって、ユニークなIPの開発が非常に大きな役割を果たすことになる。さらに、標準化も進展しており、VSIアライアンスと呼ぶ標準化活動が不特定多数のIPベンダの開発したIP製品(論理ファイル)の流通を加速しつつある。これを活用すれば、過去の設計財産であるIPを再利用することで設計工数を低減し、短い設計期間でのLSI完成が可能となる。また、これらのハードウェアIPおよびマイクロプロセッサに対して、これらを効率良く活用するミドルウェアと呼ぶソフトウェアについても同様に、ソフトウェアIPとして提供されることがSoCの発展に期待されている。SoCの現状および概要について述べてきたが、いわゆるカスタムLSIが短期間で、低価格で提供できるようになれば、半導体技術を十二分に発揮したユニークなシステム製品が生まれ、パーソナルコンピュータ、ディジタルカメラなどのディジタル機器、携帯情報機器などのエレクトロニクス分野だけでなく、これまでコスト高ゆえに、LSIの独自開発に消極的であったメカトロニクスなどの産業分野に対しても活性化を促すことになる。そのために、それぞれの分野で要求される性能、機能、信頼性などを分析し、新たなデバイス、IP、ソフトウェアの開発を行っていくことが必要不可欠なことである。
