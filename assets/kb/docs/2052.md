# 1・2・5 工程設計用データベース

> **章**: 第1章 加工学一般
> **節**: 1・2 加工工程の計画法

加工工程の設計を体系的に行うには，各種の必要な技術情報を標
準化し，データベース（data base）に格納しておくことが重要に
なる．特に，三次元CAD/CAE/CAMに代表される設計および解
位（加工特徴）を整理，分類したカタログ情報である．すなわち，
過去の生産対象製品の加工特徴，今後生産すると考えることができ
る加工特徴の形状，寸法，公差，表面性状，材質などを整理，標準
化したものである．これらの情報は，工程設計で有用であるだけで
なく，製品設計における参照情報として利用することで，加工工程
を考慮した製品設計を可能にすることができる．
c .加工技術情報 生産資源と加工特徴情報を結びつけるため
の技術情報を与える．すなわち，加工特徴情報に対して，その加工
を実行することができる生産資源を対応付けるとともに，その生産
資源で加工を行う際の作業指示書，あるいはNCデータなどを作
成する際に必要な技術データを与えるものである．
以上のデータベースを用いる場合に，製品設計データから加工工
程データに変換するプロセスにおけるデータベースの役割を図1･6
の下部に示す．すなわち，加工特徴のカタログは，CADで設計さ
れた製品データから加工特徴を抽出し，工程設計用の製品モデルを
作成するために用いることができる．さらに，加工特徴のデータか
ら，加工技術データベースおよび生産資源データベースを参照する
ことで，加工工程データ，加工指示書およびNCデータを作成す
ることが可能になる．このようなデータの変換プロセスおよびデー
タベースの整備は，工程設計を体系的に行ううえで極めて重要であ
る．
