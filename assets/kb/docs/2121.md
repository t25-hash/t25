# 13・2・2 誤差の原因と種類

> **章**: 第13章 数値解析
> **節**: 13・2 誤差

一般に数値解は、問題を数式化するときの誤差(固有誤差)と、これを数値的に解くときに発生する誤差(計算誤差)を含んでいる。固有誤差は、計算に用いる支配方程式がそもそも物理現象の近似であることに起因する。さらに、支配方程式中に用いられる定数や計算の入力値が不確かさを伴う場合には、データ誤差も固有誤差の原因となる。測定データを数値計算に用いる場合、偏り誤差(biaserror)と偶然誤差(precision error)が含まれることを認識しておく必要がある。方程式やデータに無視できない誤差があれば、いかに数値計算が正確であっても結果には大きな誤差が存在しうる。数値解析の関心は計算誤差にある。これには打切り誤差(trun-cation error)と丸め誤差(round-offerror)がある。打切り誤差は、数式化されたとおり正確に解かないで、ある種の近似をすることによるものである。具体的には、(a)三角関数を級数展開の有限項で打切る、(b)積分を台形則で近似する、(c)微分を有限個の数値の差分で置換える、(d)無限反復で収束する方法を有限回数で止める、などがある。丸め誤差は、四則演算を有限のけた数で行うことによる計算機内部の誤差である。現在広く用いられているのは、IEEE形式と呼ばれる2を基底とする浮動小数点表現である。32ビット1語では、単精度実数の内部表現は24ビット[けた数はlog 2=約7]と電卓並みである。倍精度実数なら53ビット(約16けた)となる。このように情報が消失することをけた落ち(cancellation)という。この種の情報欠落は計算の過程で警告されないし、演算を繰返すたびに累積していく。また、丸め誤差は確率的な性質をもつため解析が困難である。したがって、反復計算や行列計算では、例えば計算結果の上9 ―77位3けたしか必要でないとしても倍精度以上で行うことが推奨される。さらに、変数を変換したり演算順序を工夫したりして丸め誤差の発生やその累積を防ぐ対策も重要である。
