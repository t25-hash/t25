# 1・2・2 設計の評価

> **章**: 第1章 総論
> **節**: 1・2 設計の構成

1･2･1項に示した設計の検証は，モデルが設計スタート時に想定
した性質を備えているかどうかを数値シミュレーションまたは実験
によって確かめるものであった．しかしその検証は多くの場合一つ
の性質ごとに行われる．例えば強度のシミュレーション，熱的なシ
ミュレーション，振動・騒音のシミュレーション，コストシミュレ
ーションなどはそれぞれ単独に行われる場合が多い．
設計の評価という場合には，この個々のシミュレーション結果で
はなく，設計全体を評価・検討しなくてはならない．
個々の性質に着目したシミュレーションの結果は，設計全体の性
質を表す要素ベクトルと考えることができる．そしてその要素は設
計のパラメータを変更することによって変化する．
このように考えると，設計の検証というものは，設計パラメータ
ベクトルに設計モデルというマトリックスを乗じて，測定しうる性
質ベクトルを求めていることに相当する．したがって，設計全体を
評価するには，1･2･1項のさまざまなシミュレーション結果を要素
とするベクトルが設計に要求されている性質を十分に網羅している
かどうかを検証する必要がある．
シミュレーション結果が要求されている性質を網羅しているかど
うかの検証は非常に大切であるが，思いつくすべての性質について
シミュレーションを行うことは効率的ではない．シミュレーション
を行うまでもなく要求を満足していると考えられる性質について
は，設計者の判断でシミュレーションを省くことができる．しかし
省略するにはそれなりの注意が必要である．
設計パラメータを当初のものから変えていく過程で，問題になら
なかったはずの性質が許容限界を超えてしまう場合がありうる．性
能的にも，また製品としての価値も大いに高めたと思われた機器の
設計が熱的に破綻するということは十分ありうることである．
このようなシミュレーション上の注意とシミュレーション結果の
総合的判断のための手法は3章で述べられている．特に品質機能展
開（QFD:quality function deployment ）はシミュレーション結
果を製品としての価値との関係から総合的に判断するための有力な
武器になる．
同様に，必ずしも正解のない中で評価をする際に，いくつかの代
替案の中からよりよいものを選択するには，その製品が目指してい
るものを的確に表現した価値関数が定義できれば効果的である．ひ
とたび価値関数が定義されれば設計パラメータを変更して最適化す
る手法を効果的に利用することができる．これも3章で述べられて
いる．
