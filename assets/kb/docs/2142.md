# 1・1・1 項では、設計の本質は性質からモデルまたはシステムへの

> **章**: 第1章 総論
> **節**: 1・2 中左端のシステムから当面対象としない部分を除いてモデルを

飛躍であるが、その後に続く設計の検証過程や、最初の飛躍によって生まれたモデルを修正しながら望みの性質へ導く過程も設計にとって非常に重要であると書いた。この部分も含めて設計を定義すると、「作り始める前に考えること」となる。最初の飛躍が設計の本質であり、これを誤ると近傍の最適解に至っても他の解に負けてしまうことがあるのと同様に、技術においては設計の検証を含む設計の段階が製品の競争力、製品のもつ性格を決めてしまうところがある。技術は設計だけではなく、調達、製造、流通、保守、回収、リサイクルなど製品のライフサイクル全体を指すが、その中で技術の出発点である設計のもつ意味は非常に重要なのである。この設計の早い段階で製品のライフサイクル全体を考えなくてはならないというのも、設計に対する新しい考え方であり、『機械工学便覧』でもそのような視点に立って設計を考えている。設計が技術活動の最上流であることから、当然、担当する技術者には技術者倫理(engineering ethics)が要求される。設計者は自分が設計したもの(それはつねにいままで世の中になかったものである)が、社会に出ていったときの影響を予測しなくてはならない。社会に対する影響のうち、負の側面をも目を背けずに予測することが大切である。環境に与える影響については3・6節で扱う。しかし社会に対する影響はそれだけではない。むしろその製品、技術の本来もっている機能による影響が大きな場合もある。身近なところで携帯電話やパーソナルコンピュータおよびソフトウェアは、そこに新しい機能を追加するたびに社会を変化させる原動力になっている。その意味では、地味ではあるが工作機械などの機械類も同様である。いままであった限界を超えて加工が可能になることは、新しい世界を築くことである。このような、いままでなかったものを世の中に出すということが社会に与える変化に対して、技術者は責任をもたなくてはならない。これは、製品データでうそをついてはならないということや、機能を誇大に伝えてはならないというようなことよりも、いっそう技術の本質に近いところでの技術者の責任である。このようにいうと、いままで世の中になかったものを社会に出すといえるほどの設計はしていないと考える人もいるだろうが、従来のものからのささやかな変更であっても、そこで安易な材料を用いることで顧客の健康被害をもたらす可能性もあるし、不用意な設計のために顧客がけがをするという危険性も発生する。通常はカバーに覆われているという理由でバリを取らなかっただけで、そのような危険性が生じることもありうる。すべての設計は、いままで世の中になかったものを世の中に出すという活動なのである。それだけに設計者には細心の注意が要求される。当然払うべき注意を怠ることは技術者倫理の問題である。
