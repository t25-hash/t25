# 1・2・1 項に示した設計の検証は、モデルが設計スタート時に想定

> **章**: 第1章 総論
> **節**: 1・2 設計の構成

した性質を備えているかどうかを数値シミュレーションまたは実験によって確かめるものであった。しかしその検証は多くの場合一つの性質ごとに行われる。例えば強度のシミュレーション、熱的なシミュレーション、振動・騒音のシミュレーション、コストシミュレーションなどはそれぞれ単独に行われる場合が多い。設計の評価という場合には、この個々のシミュレーション結果ではなく、設計全体を評価・検討しなくてはならない。個々の性質に着目したシミュレーションの結果は、設計全体の性質を表す要素ベクトルと考えることができる。そしてその要素は設計のパラメータを変更することによって変化する。このように考えると、設計の検証というものは、設計パラメータベクトルに設計モデルというマトリックスを乗じて、測定しうる性質ベクトルを求めていることに相当する。したがって、設計全体を評価するには、1・2・1項のさまざまなシミュレーション結果を要素とするベクトルが設計に要求されている性質を十分に網羅しているかどうかを検証する必要がある。シミュレーション結果が要求されている性質を網羅しているかどうかの検証は非常に大切であるが、思いつくすべての性質についてシミュレーションを行うことは効率的ではない。シミュレーションを行うまでもなく要求を満足していると考えられる性質については、設計者の判断でシミュレーションを省くことができる。しかし省略するにはそれなりの注意が必要である。設計パラメータを当初のものから変えていく過程で、問題にならなかったはずの性質が許容限界を超えてしまう場合がありうる。性能的にも、また製品としての価値も大いに高めたと思われた機器の設計が熱的に破綻するということは十分ありうることである。このようなシミュレーション上の注意とシミュレーション結果の総合的判断のための手法は3章で述べられている。特に品質機能展開(QFD : quality function deployment)はシミュレーション結1―5果を製品としての価値との関係から総合的に判断するための有力な武器になる。同様に、必ずしも正解のない中で評価をする際に、いくつかの代替案の中からよりよいものを選択するには、その製品が目指しているものを的確に表現した価値関数が定義できれば効果的である。ひとたび価値関数が定義されれば設計パラメータを変更して最適化する手法を効果的に利用することができる。これも3章で述べられている。
