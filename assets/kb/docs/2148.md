# 1・2・4 設計に関わる業務

> **章**: 第1章 総論
> **節**: 1・2 設計の構成

従来のように紙の上で設計を行っていた時代には、設計を示す紙を中心に設計業務が進んでいたが、最近は設計に関わるすべての業務が計算機の中で行われることが多くなってきた。そのような状況で、設計情報として扱われるものが単なる形状情報にとどまらず、企画書、シミュレーション結果、さまざまな設計手法による検討結果、またはこのような形のあるものに変換できていないノウハウのようなものまで非常に多彩になってきている。このような情報の管理と再利用が設計を能率よく進めるためには非常に重要である。この部分については2・5節に設計情報と関連して取り上げている。また情報管理だけでなく設計業務全体の管理という形で4・2節に独立して取り上げた。最近の製品は小さなものでもマイクロプロセッサを積んで、ソフトウェアも搭載して動いていることが多く、一人の技術者が設計を始めから終わりまで担当することは少なくなっている。このことはともすると全体のわかる技術者がいなくなるという現象を生み、大きな問題である。この問題を極力防ぐためにも、全体像が見渡せる、情報やプロジェクトの管理が大切なのである。
