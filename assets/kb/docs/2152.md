# 1・3・3 設計工学の展開

> **章**: 第1章 総論
> **節**: 1・9 にも示すような、概念設計(conceptual design)・実体設計

a. 設計への情報技術の展開図1・8のもとで設計方法というものを念頭に置けば、設計工学がその発端からはん用的な方法の確立を目指してきたことを期待したいが、歴史の主流は必ずしもそうはなっていない。むしろ、設計が典型的な情報処理過程であることを受けて、1960年代以降、設計におけるさまざまな情報を効率的に操作するための技術がコンピュータ技術の発展に呼応しながら生み出され、それらが設計工学と称される初期の技術分野を形成した。‘CAD’(computer aided design、コンピュータ援用設計)の基盤技術である形状処理(geometric processing)、数理計画法(mathematical programming)を基盤とした最適設計(optimaldesign)などの技術はその典型である。学術面においても、1990年ごろまではその方面での研究が主流を占めており、それらの流れをくむデジタルエンジニアリング(digital engineering)は今日の設計や開発において不可欠なツールとなっている。一連のコンピュータ利用技術は、今後も設計工学における重要な領域であり続けるはずである。その背後にあって、図1・9の意味するところを改めて考えてみれば、一連の内容は設計対象の内容が比較的明らかになった後段の過程では有効であるものの、要求仕様(requiredspecifications)や設計概念(design concepts)を操作する前段での内容を処理するためのものではなく、それらだけでは設計が完結しないことが明らかである。図1・8 設計方法の座標軸図1・9設計のプロセスb. 設計研究の起こり上述のような歴史的展開の背後にあっても、設計という行為をはん用的に考えようとする試みは以前より一部で行われてきた。例えばサイモン(Simon)による設計についての試みは先駆的であり、さまざまな影響をのちに及ぼしている。また、いわゆるドイツ流の体系的方法論は、ヨーロッパでは一つの流れを古くから形成していた。一方、わが国においては、前出の一般設計学(general design theory)のほか、実践的な方面において品質機能展開(quality function deployment)やタグチメソッド(Taguchi method)などの方法論が早い時期に考案されている。しかしながら、各方法論の意義が広く認識されるのは後になってからのことであり、当時においてははん用的な意味での設計方法に関する工学の必要性があまり顕在化していなかったものと思われる。c. 知識に基づいた設計以上のような状況にあった設計方法を、はん用的に考えることが一つの潮流となるには、何らかの誘引が必要である。学術研究面での要因としては、1980年代の人工知能(AI:artificial intelligence)ブームの影響が一つのきっかけになっている。これは、設計における従来からのコンピュータ利用技術が比較的下流についてのものに限られていたことに対して、人工知能に基盤をおく知識処理(knowledge processing)技術やエキスパートシステム(expert system)技法が、設計の上流における知識をモデル化したり処理したりするための有効な手段になるという期待によるものであった。ブームを受けて、インテリジェントCAD(intelligent CAD)や設計エキスパートシステム(design expertsystem)についてのさまざまな試みが進められた。こうした期待には過度のものも含まれてはいたが、人工知能分野での技術を設計に移転するだけでは不足する部分も大きく、そのことが設計そのものについてのより本質的な理解に向けた動機づけになるなどして、結果的には設計研究の裾野を拡大して今日に至っている。さらに、一連の動きに呼応しながら、知識(knowledge)というものを明示的に認識して問題解決を考えていくというある種のパラダイムが設計関連分野に根づいたことは、暗黙的であるにせよ、設計工学の考え方にさまざまな影響を及ぼしている。d. コンカレントエンジニアリングより本質的に設計工学の姿に変ぼうをもたらしたものは、1980年代の国際的な製造業の状況を受けた1990年ごろ以降からの欧米におけるコンカレントエンジニアリング(concurrent engineering)に関連した動向であ1―7る。具体的には、製造業における日本脅威論のもと、製品競争力の質的な向上に向けて、機能や品質、コストなどの統合性を改善しようとして、例えば、設計と生産との融合を図ったり、開発や設計におけるチーム内での連係を効率的で合理的なものにするためのしくみや情報システムを導入したりすることが展開された。関連して、それらを支えるための技術基盤の確立を促進するべく、設計研究が政策的にも推進された。一連の動きに共通する着眼点は、図1・9に示した設計プロセスにおける上流の重要性であり、さまざまな要因についての多角的な検討を、たとえ内容があいまいであっても早い段階で行うことによって総合的に優れた設計を実施しようとしたことにある。そのような総合性に向けた設計方法の一分野がいわゆるライフサイクルエンジニアリング(life cycle engineer-ing)であり、その具体的な方法論がDfX(Design for X)ということになっている。なお、一連の内容については、日本国内においてもそれらの必要性や意義が徐々に認識されるようになりつつあることを踏まえて、本書でも多くの記述が行われている。
