# 2・2・1 二次元情報と三次元情報

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・2 三次元形状処理

の手段であり、すべての作業の基準であった。しかし、設計、生産作業に計算機を利用したシステムが導入されるにつれて、図面基準から数値基準へと作業が移行しており、情報も二次元から三次元へと変化してきている。三次元の形状情報が計算機で扱えることにより、単に三次元的な立体の絵(陰影図)が出力できるだけでなく、製品の機能や特性の解析、試作品の製作、量産化のための工具や金型の加工、製品の組立などにおいて、コンピュータシステムにより援助を行うための入力データとして必須のものとなっている。現在、設計から生産、さらには保守、回収と、製品のライフサイクルを通して、統合したシステム化が進められている。以上のような、計算機支援システムを実用化するうえで、その核となるデータが三次元形状であり、また、これを実際の製品と同じように計算機の中で扱うためには、単に数値的な三次元データがあることでは不十分である。立体としての各種演算や、他システムの要請に応じて必要なデータを出力するには、これらを構造化したモデルとして表現されていなければならない。このようなモデルを形状モデルと呼び、1960年代初めのコンピュータグラフィックスの研究と並行して開発が進められてきた。当初は、図面に書かれた稜線を三次元的に表現するワイヤフレームモデルが中心であり、その後、金型製作のための曲面モデルを経て、現在は三次元立体を表すソリッドモデルが実用化されている。さらに最近は、図面に書かれていたすべての情報を含んで表現する製品モデルへと発展している。
