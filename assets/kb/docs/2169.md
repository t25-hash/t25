# 2・5・2 ノウハウ(know-how)の蓄積と伝達の方法

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・5 設計における情報管理

野中らによれば、ノウハウを以下のように定義している。知識を大きく形式知と暗黙知の2種類に分類し、形式知(explicitknowledge)は言葉や数字で表現できるもので、厳密なデータや科学方程式、明示化された手続き、普遍的原則などの形でたやすく伝達・共有することができる特徴がある。一方、暗黙知(tacitknowledge)とは、基本的には目に見えにくく表現しがたい、暗黙的なものであり、主観に基づく洞察、直感、勘がこの暗黙知に属する。また、他人に伝達し共有することが難しい知識とし、さらに暗黙知には、個人の行動、経験、理想、価値観、情念も含まれると1―59定義している。ノウハウとはこの暗黙知の一側面であり、はっきりと示すことが難しい技能や技工が含まれると定義されている。また暗黙知の認知的側面として、メンタルモデル、思い、知覚が属し、われわれが世界をどのように感知するかに、大きな影響を与えているという。その意味でノウハウを体系的・論理的に処理することは難しい。この暗黙知たるノウハウの蓄積と伝達は、組織活動において問題となる。すなわちノウハウの例を個人レベルで考えるならば、自転車の乗り方、ピアノの弾き方など、経験的学習による手続き的知識またはノウハウとして考えることができ、そのノウハウを蓄積・伝達することについては、個人内で完結しているために問題とはならない。一方組織活動においては、生産活動における技能の継承問題に見られるように、ノウハウの蓄積と継承が企業活動における大きな課題となる。ノウハウの蓄積と伝達に関する考察は、野中によれば、組織的知識創造活動における、暗黙知の組織内部での伝達・共有の問題としてとらえることができる。暗黙知を組織内部で伝達・共有するには、誰にでもわかるように、言葉やイラストや数字に変換しなければならない。この暗黙知から形式知、形式知から暗黙知への変換を通して組織の知が形成され、ノウハウは組織内で蓄積・伝達されると主張している。言葉や数学で表現できる知識は、知識全体の氷山の一角にすぎない。また表現することは暗黙知を形式知へ変換する行為であり、暗黙知がアナロジー、コンセプト、仮説、モデルなどの形をとりながら次第に形式知として明示化することは、知識創造プロセスと理解できる。組織における知識の共有化は、暗黙知の共有を目指し、最初に暗黙知の形式知への変換(表出化)、形式知から形式知への連結化、形式知から暗黙知への変換による内面化(行動による学習)、暗黙知から暗黙知への共同化を通して、組織内への知識創造(個人知から組織知)がなされると主張している。このように組織的知識創造(すなわちノウハウの蓄積と伝達)とは、暗黙知と形式知が四つの知識変換のモードを通して、絶え間なくダイナミックに相互循環するプロセスと定義されている。いずれにしてもノウハウをすべて形式知に変換することはできず、その形式知から新たなノウハウが生じることから、人間の介在を無視したノウハウの蓄積と伝達は困難なように見える。
