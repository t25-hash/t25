# 2・5・3 PDM による設計情報の統合化

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・5 設計における情報管理

製品データ管理(PDM : product data management)は、製品データを管理するためのプロセスおよび、そのためのコンピュータシステムを含む概念である。PDM はアプリケーションソフトウェア市場で高い成長を遂げている分野の一つで、製品データ管理における諸問題を解決するツールと期待されている。例えば、CAD システムは製品の形状とその属性を定義し、保管する機能が重視されるが、材質との関係、要求変更に伴う設計変更、バージョン管理など、その製品に関する正しい情報の管理が必要とされ、その管理のためにPDM システムが必須のものとなっている。PDM の概念の以前には、EDM(engineering data management)の概念があった。このEDM は以下の項目を含むとされている。(1) エンジニアリング文書管理:紙、マイクロフィルム、ラスタイメージを含む、あらゆる形態の文書を管理するためのデータベースの利用形態(2) エンジニアリング図面管理:エンジニアリング文書管理を図面管理に特化させ、図面作成手順と実行に利用する(3) エンジニアリングデータ管理:異なる設計およびエンジニアリングを横断するデータを管理するために、独立なデータベースシステムを利用するこれらの項目は、以下の五つの機能に分解できる。(1) アクセス管理:許可されたユーザがデータにアクセスできるように管理する機能(2) 製品構造定義(product structure definition):構成する部品とその組立により、製品の構造を記述する機能(3) 構成管理(configuration management):製品とデータのバ1―60設計工学ージョンを正しい組合せと識別する機能(4) 設計レビューと承認:製品と製品データに対するレビューと承認を発行する記録の管理機能(5) アクション管理(action management):有効性(effec-tivity)とワークフローの管理PDM はこれらのすべての機能または一部を包含している。またPDM のカバーすべき領域と機能から、以下のように分けることができる。(1) CAD システムに対するCAD データの管理機能としてのPDM(2) 企業活動における成果物管理機能としてのPDM(3) 企業間での協調作業を支援するプロジェクト管理機能を包含するPDMこれらのPDM 分類は利用形態としての区分と見ることができ、図2・148に示すように現状のPDM は、商品の迅速な開発体制を実現するために、開発・設計・生産を通した製品情報・知識の共有と活用を意図した図面情報管理(drawing data management)、三次元製品データ管理(3D product data management)などの成果物管理に主眼をおいている。今後、協調設計管理、プロジェクト管理、プロセス管理などへの適用範囲の拡大が意図されている。すなわち、製品データをコアに企業活動における顧客をも含めた相手先との情報共有を確実にするため、製品データを安全に確実に流通させ、再利用できるPDM 機能が必要とされている。・PDM による設計情報の統合化の背景設計情報を単にディジタル化しただけでは、再利用ができる、修正が容易である程度の利点しか生じない。製品の企画・設計・生産の各部門においては、同じ製品の異なる観点での製品データを対象とするため、各部門を横断した設計情報のディジタル化による情報共有は、製品情報の利用目的に合った観点での製品データを、必要なときに提示できる機能をもつべきである。この機能を提供するものとしてPDM システムが実現した。このPDM システムによる効果は、企業における各部門に分散していた製品データを一元的に管理することにより、製品情報を利用目的(設計、生産、検査等)に合った観点で提示し、必要なときに、必要な情報を相手先(企画、設計、製造、生産準備、調達、サプライヤ等)に正確に伝達することで作業の後戻りを削減し、コストの削減、リードタイムの短縮、製品開発における生産性の向上、顧客ニーズにこたえる迅速な製品開発が可能となる。
