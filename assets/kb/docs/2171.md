# 2・5・4 PDM による設計データの一元管理

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・5 設計における情報管理

製品データの表現と交換を体系的に整理した規格（ISO10303
なされた．STEPはPDMにおける中立的な製品データ参照モデル
（product data reference model）を規定しているので，その概要
について述べる．
a.STEP PDMスキーマ STEPは，複数の分野別規格
（AP:application protocol）をもち，代表的な分野別規格として
AP203（三次元製品モデル交換規格），AP214（自動車工業会用規
格）等がある．これらのAPにおけるモデル管理データを整理統合
してSTEP PDMスキーマが提案されている．以下にその機能の
概要について述べる．
b.PDMスキーマの基本機能 PDMスキーマは基本的に以
下の機能をもっている．
(１)製品・部品に対する，識別と分類，特性，構成と関連
(２)ドキュメントに対する，識別と分類，特性，構成と関連
(３)製品とドキュメントとの関連付け，識別子の名称変更，承
認，有効情報，作業管理
(４)PDMスキーマの機能単位（unit of functionality）

(１)STEP手法を用いてデータを表現している．強固で国際的

(２)国際規格（ISO10303「STEP」）をベースにしているため，

設計データの一元管理とは，多くは企業内で設計開発に関わる設
計者が，いつでもどこでも製品と製品データの正しい組合せを検
索・利用・保管できる管理方法であって，基本的には設計データの
構成管理技術に依存するといえる．
2・5・5設計データ互換性の確保
設計データを，ここでは製品モデル，設計情報，設計ノウハウ，
製品構成管理データの総称と理解する．具体的には，製品モデル
（設計結果としての製品を表現したデータ），設計情報（例えば設計
仕様，標準部品など，製品設計に必要な情報），設計ノウハウ（設
計における暗黙知），失敗事例，成功事例等，製品構成管理情報
（BOM：bill of material）など，製品と設計データの正しい組合せ
情報とする．また設計データの互換性，相互運用性を以下のように
定義する．
(１)製品データの互換性（product data compatiblity)：ある
CADシステムを用いて生成されたCADデータ（設計データ）を，
他の異なるCADシステムで利用できること．すなわち，交換した
データが元のデータと同一に表示されていることが最低限要求され
る．
(２)製品データの相互運用性（product data interoper-
ability)：あるCADシステムを用いて生成されたCADデータ
（設計データ）を，他の異なるCADシステムで読み込み，等価な
