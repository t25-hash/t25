# 2・5・4 PDM による設計データの一元管理

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・5 設計における情報管理

製品データの表現と交換を体系的に整理した規格(ISO 10303「STEP: Standard for the Exchange of Product model data」)は、CAD データ交換のための国際標準化の観点から活動が開始され、図面データに限定されることなく、製品モデルを対象とし、製品構成などの管理データ(PDM データ)をも包含して規格開発がなされた。STEPはPDM における中立的な製品データ参照モデル(product data reference model)を規定しているので、その概要について述べる。a. STEP PDM スキーマSTEPは、複数の分野別規格(AP: application protocol)をもち、代表的な分野別規格としてAP203(三次元製品モデル交換規格)、AP214(自動車工業会用規格)等がある。これらのAPにおけるモデル管理データを整理統合してSTEP PDM スキーマが提案されている。以下にその機能の概要について述べる。b. PDM スキーマの基本機能PDM スキーマは基本的に以下の機能をもっている。(1) 製品・部品に対する、識別と分類、特性、構成と関連(2) ドキュメントに対する、識別と分類、特性、構成と関連(3) 製品とドキュメントとの関連付け、識別子の名称変更、承認、有効情報、作業管理(4) PDM スキーマの機能単位(unit of functionality)STEP PDM スキーマの具体的な23の機能モジュールを、表2・16に示す。STEP PDM スキーマの特徴を以下に示す。(1) STEP手法を用いてデータを表現している。強固で国際的なデータ表現である。(2) 国際規格(ISO 10303「STEP」)をベースにしているため、各種APとの親和性が高い。(3) 製品構成データの中立的表現であるため、PDM 間のデータ交換に利用できる。設計データの一元管理とは、多くは企業内で設計開発に関わる設計者が、いつでもどこでも製品と製品データの正しい組合せを検索・利用・保管できる管理方法であって、基本的には設計データの構成管理技術に依存するといえる。
