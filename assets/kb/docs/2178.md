# 3・1・5 製品企画の具体的方法

> **章**: 第3章 設計のための個別方法論
> **節**: 3・1 企画のための方法

a. 製品企画のプロセス製品企画のプロセスには、大別して論理積上げ型プロセスと仮説思考(hypothesis oriented think-ing)型プロセスとがある(図3・17)。. 論理積上げ型プロセス論理積上げ型プロセスは、製品企画における最適解を追求するのに適している。それでは、最適解を追求しうることが合理的であるのはどういう場合かというと、経営戦略の3Cにおける顧客(customer)と競合(competitor)という社外要因についての不確実性が小さい場合である。例えば、前述した業界協調型や製品ライフサイクルの成長期における製品企画は、この論理積上げ型が適している。このプロセスでは、まず、製品企画の最終結論に至る前に、そのために必要な顧客、競合、自社についての情報収集・分析が徹底的に行われる。そして、可能な選択肢をすべて洗い出した後に、最適解を選定する。また、顧客(customer)や競合(competitor)に対する不確実性が低いのであるから、自社(company)がとるべき製品企画についても、できるだけ確実性の高い方向を選択するのが合理的といえる。すなわち、3W 2H の体系でいえば、実現性(how)に最大の重点をおき、ついで収益性(how much)についても精緻な検討を重んじるという、いわば極めて正当なプロセスである。しかしながら、これまでの高度成長期に有効であったこのプロセスも、すでに成熟期を迎え、協調から競争へと移行しつつある日本図3・16 製品ライフサイクルにおけるイノベーション図3・17 製品企画における二つのプロセス[文献(20)に加筆]企業にとっては、必ずしも有効とはいえなくなった。すなわち、顧客(customer)や競合(competitor)についての不確実性が小さい場合は、いまではむしろ例外となりつつある。. 仮説思考型プロセス顧客(customer)や競合(com-petitor)についての不確実性が大きい場合は、自社(company)にとっての最適解を求めるといったことは論理的に不可能となる。顧客や競合がどう動くのかは、正確には予測できないからである。例えば、企画競争型や成熟期においてイノベーションを実現しようという場合である。このような場合には、最適解を求めることは断念して、妥当解を追求するほうが現実的な考え方となり、これに適したのが仮説思考型プロセスである。仮説思考型プロセスでは、すでにある情報だけを前提として、最初に仮説を作り上げてしまう。その次に、その仮説が妥当であるか否かを判断するのに必要な情報に限定して、情報を収集・分析する。この段階で、最初の仮説に自信をもてない場合は、改たに加わった情報を含めて、さらに仮説を再構築する。そして、この再構築された仮説をもとに、必要な情報の収集・分析を行う。あとは、このプロセスを自社(company)が自信のもてるまで繰返していく。ただし、顧客(customer)や競合(competitor)の動きから決定される許された時間の範囲でという制約がつく。b. 製品企画の発想法。 経営戦略からの発想法革新的な製品はつねによいものとは限らない。企業経営の視点からは、改良型の製品のほうが積極的に望ましい場合もあれば、真に革新型の製品が求められる場合もある。大別して、このどちらを目指して製品を企画するのか、この重要な点について示唆を与えるのが、経営戦略からの発想法である。具体的には、前述した経営戦略の3C、ポートフォリオ分析、ポーターの競争戦略論、業界別の競争類型、製品ライフサイクルなどの視点から、多面的に事業環境と自社の位置付けを分析することによって、革新を目指すのか、それとも改良を目指すのかの意思決定をすることが重要である(図3・18)。. 製品企画の体系(3 W 2 H)からの発想法(1) 顧客からの発想法顧客が何を求めているのか、そのニーズから発想しようとするアプローチである。これは、本節の冒頭で述べたニーズ指向と対応し、また3W 2H の体系では、市場性(why・その1)を重視した発想と位置付けられる(図3・19)。なお、同じニーズ指向といっても改良型の製品を目指す場合は、すでに顕在化しているニーズをより重視し、一方、革新的な製品を目指す場合は、潜在的なニーズ図3・18 経営戦略と製品企画図3・19製品企画の体系(3W 2H)と製品企画1―73(ウォンツとも呼ばれる)をより重視するという違いがある。(2) 独自技術からの発想法シーズ指向には、自社の独自技術から発想するものと、自社の得意な技術から発想するものがあり、これらは前述したように区別して考えなくてはならない。前者では、製品企画の競争優位性(why・その2)を重視している(図3・19)。この発想法が使われる典型例は、イノベーションをねらう場合である。3・1・4・c・ii項で前述したように、イノベーションとは、あくまでも市場の視点が重要なのであって、技術革新とは無関係である。しかし、苦労の末にイノベーションを成功させても、競争優位性が弱いと、競合他社にやすやすとまねをされてしまう。したがって、この追随をできるだけ防ぐために、独自技術で障壁を作ってしまうという考え方である。また、一般には競争の激しい業界で、この独自技術からの発想がしばしば有効である。(3) 得意技術からの発想法得意技術からの発想とは、製品企画の実現性をより重視(図3・19)した考え方であり、早期かつ確実な実用化を重視している。すでに述べたように、この自社の得意な技術と自社の独自技術を混同しないように注意する必要がある。. その他の代表的な発想法以上で説明した以外にも、さまざまな発想法が考えられている。ここでは、それらのうち代表的で、よく使われている、ブレインストーミング、KJ法、TRIZ、ポジショニング法、関連樹木法の五つの発想法について説明する。(1) ブレインストーミング(brain storming)簡単にできるため、最も広く使われている発想法である。数人が集まって、テーマに関する考えを発言し合い、別の人と触れ合うことで新しいものを生み出そうというものである。この方法で守るべきルールとしては、1他人の考えを批判しない、2自己規制しないで自由奔放に発言する、3できるだけ多量のアイデアを出す、4そして組合せ・結合の視点をもつ、などがある。また、企業活動の視点から、有効な成果を得るために配慮すべきこととして、参加者の質、および事前の準備がある。参加者については、そのテーマに関してできるだけ有能な人を集めることである。加えて、厳格な上下関係のない人を集めることも大切である。発言にいろいろな遠慮・配慮をしなくてはならないようでは、発想に集中できないし、無意識のうちに自社特有のバイアスがかかってしまう危険がある。事前の準備については、目標とする成果によってケースバイケースであるが、前提となる明らかな制約条件を整理しておくことは、有用なことが多い。さらに、単なる雑談になるのを防ぐためには、テーマを明確にして、事前に参加者全員に伝えておくのはもちろんだが、場のリード役が必要である。また、出し合ったアイデアを、メンバ全員につねに見えるようにしておく白板(黒板)は必需品である。そのつど、アイデアを書き加えたり、整理したり、関連付けたりと、発想を促すのに役立つ。(2) KJ 法(KJ method)ブレインストーミングと並んで、広く使われている発想法である。アイデアを発想するだけでなく、雑多なデータをもとに仮説を構築したり、全体像を組立てたりするのに向いている。例えば、市場調査のデータをもとに、消費者像を描き出したり、企画の全体構想を考えるときに有用である。具体的な方法は、1まず素材となるデータやアイデアを1件1葉でカードに書き出す。2次にできたカードを並べ替える。3続いて、いくつかのグループができたら、小グループの内容を新たなカードに書き出す。4さらに小グループをまとめて中グループを作り、それらから大グループを作っていく。5最後までグループ編成が終わったら、これを素材として図解あるいは文章化していく。これらのステップは、個々人が頭の中で実践してきたことを、頭の外に出して、目に見える形に表現したととらえることができる。一人でももちろんできるが、思考過程が明示化できるので、数名で1―74設計工学検討するのにも向いている。ただし、あまりにも初期の段階でKJ 法を行うと、本来ならば無意識に行っている思考過程を明示的に行ってしまうため、効率が悪くなることに注意が必要である。(3) TRIZTRIZ はロシア語の略称であり、英語ではTIPS :theory ofinventive problem solving ということになる。技術開発や研究を、科学的・システマチックに行うための発想法である。TRIZ は、過去の膨大な特許を分析し、統計的に一定の頻度で使用される解決法などを抽出することで生まれた。TRIZ を用いると、短期間での効率的な技術開発が可能ということで、技術者や研究者を中心に関心を集めている。TRIZ が他の方法と異なる利点は、問題定義と問題解決の二つの部分を明確にもっている点である。一般的に、何が問題かを分析するツールは多いが、どのように解決したらよいかを支援する方法は少ない。TRIZ の構成は、1問題定義の技法であるARIZ(英語ではAIPS)、2矛盾問題を解決する発明原理であるプリンシプル、3科学効果を有効利用するための事典であるエフェクト、4標準的な発明手段であるプレディクション、5技術の将来を予測する技術の進化トレンドの五つからなる(図3・20)。これらの構成をもとに、TRIZ では、まずARIZ で問題定義を行い、次にプリンシプル、エフェクト、プレディクョンに進んで問題解決を行い、同時に技術の進化トレンドを参照して、問題の解決のレベルアップを図っていく。(4) ポジショニング法(positioning method)二つの評価軸を組合わせて四つの象限を構成し、そのチャートの中にアイデアを位置付ける(すなわち、ポジショニングする)方法である。評価軸のとり方やメモリのどこに注目するかで、いままでになかった新しいアイデアを発見するのを促すことができる。ソフトドリンクを例にとると、どういう場面で飲むかと何をアピールするかの二つの評価軸によって整理し、そこから、美容ドリン図3・20 TRIZ の構成図3・21 ソフトドリンクのポジショニングクや、スポーツドリンク、ダイエットドリンク、クエンチングドリンクなどの商品カテゴリを整理・発想していくものである(図3・21)。(5) 関連樹木法(related tree method)検討する領域の枠組を、大枝、中枝、小枝、葉末といった具合に、樹木のように関連付けていく手法。樹木全体の形と葉の繁り具合から、途中の枝ぶりを想定してみるなどの発想が可能になる。あるいは、枝を伸ばしてみる、異質なものを途中から接ぎ木してみるという利用の仕方も可能で、新商品開発や商品ラインナップの拡大などに使える。また、この手法は製品企画以外のさまざまな分野にも応用されている。よく知られているのが、意思決定用の決定木である。テーマをレベルに沿って細分化していき、それぞれの重要度を相対評価していくというものである。その際、全体の構造を理解できるという点でも、関連樹木法は優れている。
