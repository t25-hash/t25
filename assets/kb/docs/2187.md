# 3・4・1 信頼性の設計の考え方

> **章**: 第3章 設計のための個別方法論
> **節**: 3・4 信頼性の設計

信頼性は製品が満足すべき最も重要な特性の一つである.製品のライフサイクルを通じて,ユーザの要求する機能を満足できるようにするためには,種々の方策が必要となるが,それらは以下の3種に大別することができる.(1) 故障が発生しないようにする.(2) 故障が発生しても,なるべく機能を保てるようにする.(3) 故障が発生しても,ただちに機能回復のための修復ができるようにする.また,これらの方策を講じる時期としては,設計・製作段階と,運転・保全段階がある.前者では信頼性を作り込む活動が行われ,後者では信頼性を発現させるための活動が行われる.ここでは,前者の信頼性の作り込みについて考える.この観点から,上記三つのカテゴリに該当する方策例を示したのが表3・12である.以下では,おもに(1)と(2)のカテゴリの方策に関しての基礎を与える信頼性工学について概説する.3・4・2項においては,まず故障をとらえる視点を整理し,3・4・3項で説明する確率論に立脚した信頼性理論の意味を明確にする.また,3・4・4項では,信頼性データに基づく信頼性特性値の推定について,3・4・5項では,特に上記(2)に関連する,要素とシステムの信頼度の関係および冗長化について述べる.さらに,3・4・6項では,必ずしも確率論にとらわれない広義の信頼性工学手法として,特に設計段階で対象システムの弱点を抽出するのに活用されているFMEA(failure mode andeffect analysis)とFTA(fault tree analysis)について述べる.信頼性工学の範囲は非常に広いが,ここでは,製品の信頼性を考えるうえで必要な基本的視点と,信頼性に関する手法の中でも特に1―9 3表3・12 信頼性設計のための方策カテゴリ設計段階での方策(1) 故障が発生しないようにする.・要素の信頼性向上・要素数の低減・構造の簡素化・余裕度の向上・操作性の向上による誤操作の防止(2) 故障が発生しても,なるべく機能を保てるようにする.・構成の冗長化並列化予備機の設置・機能の冗長化機能代替機能再構成(3) 故障が発生しても,ただちに機能回復のための修復ができるようにする.・故障検知,診断能力の向上・修復性の向上・分解・組立性の向上・アクセス性の向上図3・60 機械の階層と損傷・故障の因果関係広く利用されている,累積ハザード法,FMEA,FTA の三つの手法の理解を主眼に解説する.このため,故障物理,信頼性試験などの信頼性工学を構成する重要な分野をカバーしていない.これらについては多くの優れた教科書が出ているので,それらを参照されたい.
