# 3・4・1 信頼性の設計の考え方

> **章**: 第3章 設計のための個別方法論
> **節**: 3・4 信頼性の設計

信頼性は製品が満足すべき最も重要な特性の一つである．製品の
ライフサイクルを通じて，ユーザの要求する機能を満足できるよう
にするためには，種々の方策が必要となるが，それらは以下の3種
に大別することができる．
(１)故障が発生しないようにする．
(２)故障が発生しても，なるべく機能を保てるようにする．
(３)故障が発生しても，ただちに機能回復のための修復ができる
ようにする．
また，これらの方策を講じる時期としては，設計・製作段階と，
運転・保全段階がある．前者では信頼性を作り込む活動が行われ，
後者では信頼性を発現させるための活動が行われる．ここでは，前
者の信頼性の作り込みについて考える．この観点から，上記三つの
カテゴリに該当する方策例を示したのが表3･12である．
以下では，おもに(１)と(２)のカテゴリの方策に関しての基礎を

3・4・2故障現象の理解
故障を理解する場合，一般的には，その因果関係からとらえるこ
とが多い．例えば，自動車が動かないという故障に対して，その原
因がセルモータが回らないことであったとする．これは，システム
の故障の原因を構成要素の故障に求めるもので，全体部分の因果関
係になる．また，セルモータが回らない原因が，バッテリが上がっ
ているであれば，それは要素間の機能依存関係に基づく因果関係を
示すものである．さらに，バッテリが上がった原因が，極板の損耗
であるというのは，バッテリの故障をその構成要素の損傷に帰着さ
せるものである．このように故障の因果関係は一組の原因と結果で
表せるものではなく，因果の連鎖になっている．また，それらの因
果関係の種類も多様である．
ここで故障（failure）とは，アイテム（信頼性の対象となる部
品，ユニット，システムなどの総称，またはそのいずれか）が要求
機能の達成能力を失うことであり，機能的視点からのとらえ方であ
る．これに対して損傷（damage）は，アイテムの物理・化学的属
性の変化である．損傷はアイテムが受けるストレスによって生じ
る．損傷が生じる物理・化学的な機構を，損傷メカニズムと呼ぶ
（JIS Z8115：2000「ディペンダビリティ（信頼性）用語」では，
損傷を故障と区別せずに，故障メカニズムと呼んでいる）．
一般に機械の構造は階層的にとらえられるので，この階層に沿っ
て故障の因果関係を整理することができる．例えば，図3･60は工
作機械の加工精度不良という故障の因果関係を，階層構造に沿って
たどったものである．このように，一つの故障でも，着目するアイ
テムによって，故障現象はさまざまに表現される．ただし，最終的
には，その原因は特定のアイテムの損傷に帰着する．図3･60の場
合は，損傷は軸受転動面のはく離で，それを引き起こしたストレス
