# 3・4・2 故障現象の理解

> **章**: 第3章 設計のための個別方法論
> **節**: 3・4 信頼性の設計

故障を理解する場合、一般的には、その因果関係からとらえることが多い。例えば、自動車が動かないという故障に対して、その原因がセルモータが回らないことであったとする。これは、システムの故障の原因を構成要素の故障に求めるもので、全体部分の因果関係になる。また、セルモータが回らない原因が、バッテリが上がっているであれば、それは要素間の機能依存関係に基づく因果関係を示すものである。さらに、バッテリが上がった原因が、極板の損耗であるというのは、バッテリの故障をその構成要素の損傷に帰着させるものである。このように故障の因果関係は一組の原因と結果で表せるものではなく、因果の連鎖になっている。また、それらの因果関係の種類も多様である。ここで故障(failure)とは、アイテム(信頼性の対象となる部品、ユニット、システムなどの総称、またはそのいずれか)が要求機能の達成能力を失うことであり、機能的視点からのとらえ方である。これに対して損傷(damage)は、アイテムの物理・化学的属性の変化である。損傷はアイテムが受けるストレスによって生じる。損傷が生じる物理・化学的な機構を、損傷メカニズムと呼ぶ(JIS Z 8115:2000「ディペンダビリティ(信頼性)用語」では、損傷を故障と区別せずに、故障メカニズムと呼んでいる)。一般に機械の構造は階層的にとらえられるので、この階層に沿って故障の因果関係を整理することができる。例えば、図3・60は工作機械の加工精度不良という故障の因果関係を、階層構造に沿ってたどったものである。このように、一つの故障でも、着目するアイテムによって、故障現象はさまざまに表現される。ただし、最終的には、その原因は特定のアイテムの損傷に帰着する。図3・60の場合は、損傷は軸受転動面のはく離で、それを引き起こしたストレスは、転動体の通過による繰返し応力であり、損傷メカニズムは疲労1―9 故障の進展パターン図3・62時間軸上での故障発生のタイミング摩耗ということになる。図3・60は、故障を機械の階層構造に対応させてとらえたものであったが、その中の一つの階層で観測される現象を時間軸上でとらえると図3・61のように表すことができる。縦軸は、その階層における機能状態を示すパラメータである。時間の経過とともに、パラメータ値は種々の要因により変動するが、それが検出限界を超えた時点で劣化が認められたと判断できるものとする。さらに、劣化が進行し、要求機能から定まる機能限界を超えた時点で故障となる。までの期間が正常期、からまでの期間が兆候期、以降が故障である。このように、故障をその因果関係から理解しようという立場に対して、図3・62に示すように時間軸上での故障の発生のタイミングを確率事象としてとらえて、その特性を理解するという立場がある。故障の因果関係という点では、前者はホワイトボックス化のアプローチであり、後者はブラックボックス化のアプローチである。歴史的にいえば、複雑な因果関係をブラックボックス化し、発生のタイミングのみに着目することで、見事に故障現象を体系的にとらえたのが信頼性工学であった。しかし、信頼性工学の適用には後述する故障率等の統計データが必要となるが、信頼性が向上し故障数が減少するほど、このようなデータを収集することが困難になる。いわゆる数と時間の壁と呼ばれる問題である。この問題を解決するために、信頼性工学の分野においてもブラックボックスを開けて、故障のメカニズムを物理・化学的に追求することの重要性が認識され、故障物理と呼ばれる分野が発達した。一方、実務的には、製品開発段階で、重大な結果を引き起こす可能性のある故障をたとえ定性的にでも解析しておくことが重要であり、機械構造や要素特性に基づいて対象機械の解析を行う信頼性解析手法が発達した。以上のことから、信頼性の手法は以下の3種に大別される。(1) 機械構造、経験知識をもとにした解析製品開発の初期段階では、利用可能なデータが少ない。そこで、主として製品の構造や専門家の経験知識をもとに、設計上の問題点や製造上の注意点を解析することが求められる。このための手法としては、FMEA,FTA、デザインレビュー(設計審査)などがある。(2) 信頼性試験や故障物理をもとにした解析試作評価段階では、問題になりそうな故障について信頼性試験を行い、耐久性の確認をしたり、問題が発生した箇所については、故障物理に基づき原因となる損傷メカニズムを解析し対策を検討する。(3) 統計的なデータをもとにした解析製品が市場に出て、データが集まってきていたり、過去に蓄積したデータが存在するなど、ある程度まとまった数のデータが利用可能なときには、確率統計的解析により、故障の発生を予測したり、発生している故障の特性を分析することができる。
