# 3・4・5 システムの信頼性

> **章**: 第3章 設計のための個別方法論
> **節**: 3・68 に示すワイブル型累積ハザード紙である。

これまでは、一つのアイテムの故障について考えてきた。しかし、一般に機械は複数の要素からなる。そこで、これらの要素の信頼性と、それが組合わさったシステムの信頼性との関係を考える必要がある。a. 直列システム(series system)直列システムは、システムを構成しているどの要素が故障してもシステムの故障となるものであり、図3・69のように表される。信頼性工学の分野では、このような要素の機能的な関係を示す線図のことを信頼性ブロック図(reliability block diagram)と呼んでいる。すなわち、直列システムの故障率は、要素の故障率の和で与えられる。要素の故障が指数分布に従う偶発型のときは、故障率は時間によらず一定となり、また、式(3・11)よりMTTFが故障率の逆数となるので、システムと要素のMTTFの間には以下の関係が成り立つ。b. 冗長システム(redundant system)直列システムの場合は、要素数が増えれば増えるほど、システムの信頼度は低下する。これに対処する方法としては、個々の要素の信頼度を上げることが考えられるが、他の方法として、一機能に対して複数の要素を併用し、その一つが故障しても全体としての機能を維持できるようにすることがある。このような考え方のシステムを冗長システムと呼ぶ。その構成方法には、並列システム、冗長システム、待機冗長システムなどがある。並列システムは、信頼性ブロック図では図3・70のように表される。この場合、すべての要素が故障したときにシステムが故障となるので、各要素の不信頼度を(=1,...,)とすると、システムの信頼度は以下の式で表される。待機中の予備の状態によって、冷予備、温予備、熱予備に分類される。冷予備は、待機中の要素が作動状態にない場合で、熱予備は、待機要素が作動状態にあるが、機能的に接続されていない場合である。また、温予備は、待機中に動作に必要な一部のエネルギーの供給を受けている場合である。システムに冗長性をもたせる目的は、要素に故障が発生してもシステムとしての機能を保持できるようにすることであるが、そのようなシステムの属性を一般にフォールトトレランス(fault toler-ance)と呼ぶ。システムをフォールトトレランスにするには、ハードウェアを冗長化するだけでなく、故障発生時に要素の機能関係を再構成して要求機能を維持するという方法もある。このような方法を機能冗長と呼ぶ場合がある。また、故障時に機能の一部でも維持し動作を続行しようとすることをフェイルソフトという。なお、これに類似した用語にフェイルセイフがあるが、これは故障発生時に停止するなどしてたとえ信頼性を下げてでも安全性を確保することである。
