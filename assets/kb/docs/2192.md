# 3・4・6 信頼性の解析

> **章**: 第3章 設計のための個別方法論
> **節**: 3・68 に示すワイブル型累積ハザード紙である。

a. 信頼性解析手法製品の信頼性を高めるためには、設計段階において、製品に生じる可能性のある損傷や故障を予測し、問題となる箇所について適切な対策を講じていく必要がある。このためには、信頼性データよりMTTFや故障率といった確率統計的な信頼性指標を求めるだけでなく、製品構造に基づき、故障の因果関係やその発生のメカニズムに関する解析を行うことが必要である。このような目的をもった解析手法として、最も広く用いられているのがFMEA とFTA である。FMEA,FTA は、アメリカの軍事、宇宙開発に関連して1950年代から1960年代にかけて生まれた手法である。ともに開発・設計段階で、製品の信頼性に関する潜在的問題を摘出し、対策を講じるための手法として開発された。FMEA は、システムを構成する要素に生じる可能性のある故障を予測するとともに、その故障によってシステムがどのような影響を受けるかを、表を使って解析を進める手法である。これにより、重大なシステム故障につながる要因となる要素の故障モードを網羅的に摘出することができる。また、それぞれの要因の致命度を評価することで、対策の優先度付けを行うことができる。FMEA は要因側から結果側に向かって影響関係を分析するボトムアップ的手法であるが、このような考え方は応用範囲が広く、現在では、製品設計段階だけでなく、製造工程での品質改善、運用段階での保全計画など、製品ライフサイクル全般にわたって活用されている。例えば、製造工程を構成する4M(設備、材料、人、方法)の不具合が製品の品質にどのように影響するかを解析するために用いる場合は、工程FMEA と呼ばれる。一方、FTA は、システムにおいて発生する可能性のある望ましくない事象を頂上事象として設定し、その発生経路を原因側に向かってたどっていくトップダウン的手法である。FMEA がハードウェアの単一故障の影響を解析するのに適しているのに対して、FTA ではソフトウェアの欠陥や、人的ミスなどの要因を含めた多重故障の解析が可能である。このために、安全性に関わる解析にも広く活用されている。また、FTA では、事象間の因果関係をAND やOR といった論理記号を用いて展開していくためブール代数に基づく解析が可能である。さらに、最下位の基本事象の発生確率が与えられれば、トップ事象の発生確率を定量的に求めることができる。FMEA,FTA は信頼性解析手法として最も有力な手法といえるが、そのほかにも種々の解析手法が提案されており、FMEA,FTA の不足を補うために活用を図っていくことが望ましい。例えば、FMEA,FTA では、事象間の時間的順序関係を陽には表現できない。これに対しては、原因側から結果側に向かって時間軸上でどのように事象が波及し望ましくない結果に至るかを解析する手法としてはETA(event tree analysis)が知られている。b. 対象システムの理解と分解レベルの決定FMEA の実施のためには、対象システムの構造、機能、使用条件、環境条件などを理解しておく必要がある。これには、設計仕様書等の資料だけでなく、類似システムなども参照する。また、対象システムをどこまで分解して解析するかを決める必要がある。あまり細かく分解しても解析しきれないし、粗すぎると要因が的確に特定できず、対策につながらない。一般的には、2,3階層下まで分解するのが普通である。.
