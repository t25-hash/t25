# 3・6・2 項で述べたメンテナンス、リマニュファクチャリング、リユ

> **章**: 第3章 設計のための個別方法論
> **節**: 3・6 節では、この流れに沿って、環境調和性という面から、ライ

ース、マテリアルリサイクル、サーマルリサイクル、さらには、アップグレードや適正廃棄など、製品の循環経路に関する選択肢のことである。欧米では、end-of-life optionといわれることもあるが、メンテナンスやアップグレードなど狭義のend-of-lifeではないオプションも対象にする場合、ライフサイクルオプションと呼ばれる。図3・94では、おもな製品ライフサイクル特性として8項目挙げた。この中で製品の寿命[lifetime、物理寿命(physical life-time)、価値寿命(value lifetime)]が鍵となるのであるが、それ以外にも製品の販売期間、製品カテゴリの寿命(8ミリビデオなど製品カテゴリがどのくらいの期間、市場に存在するか)、使用量・廃棄量(単品生産物と大量生産するものではおのずからその処理方法が異なる)、使用方法・ユーザ、製品の構造、回収システムの有無などを十分に把握する必要がある。製品の寿命は、以下の2種類に分類して考えられる。すなわち、従来の意味で製品が故障する、劣化するといった物理寿命と、パーソナルコンピュータや携帯電話に見られるように、製品自体は正常に動作するが新製品に比べて機能が大幅に劣った、デザインが古くなったために捨てられてしまうという意味での価値寿命であり、実は現代の工業製品では価値寿命が支配的な製品が多い。製品の価値寿命を維持、向上させながら、物理寿命まで製品や部品を使い尽くすことがライフサイクル設計の一つの大きな課題である。先に述べた設計の視野の拡大にも関連するが、ライフサイクル設計の対象は単一の製品とは限らない。むしろ限定しないほうがよい。例えば、3・6・1・b項で述べたインバースマニュファクチャリングを前提とすれば、提供すべきものはユーザが要求するサービスであるので、製造業においても従来のようにハードウェアを販売するという形態ではなく、部分的にソフトウェア化しオンラインで販売したり、製品自体はレンタルするといった多様な機能実現の方法を考えるべきである。また、機械自体を対象とする場合にも、単一の製品のみで考えるのではなく、製品系列(シリーズ)や世代を視野に入れ(図3・95)、シリーズ間や世代間での設計情報や部品の共通化を検討することが有効である。
