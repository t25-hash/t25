構造解析ソフトによる結果の相違・成形シミュレーション種々の現象を工学的に数学モデルに置換えて大規模に計算機上で計算すること、と定義することもできる。コンピュータシミュレーション(computer simulation)の利点は、設計者が細かいことを知らなくてもある程度の答えを出してくれることにある。一方で欠点としては、ブラックボックス的になり、間違った答えが出ても設計者が評価できないケースが出てくることである。コンピュータシミュレーションは一種の数値実験であるが、実験の場合は計測という行為が入るために、ある程度結果を予測する必要があり、この行為が結果として現象の理解につながる。一方、コンピュータシミュレーションはこのような手順を踏まないでも使えるため、便利な反面、危険でもある。また、コンピュータシミュレーションは多くの場合、数値的に解いているため、解法の違いによって答えも違ってくる。図3・104に3種類の市販の構造解析ソフトで同じ問題を解いた結果を示す。非常に簡単な例題であるにもかかわらず、答えに差のあることがわかる。詳細に検討すればこれらの違いの原因・理由はわかるが、一般に設計者がそこまで理解することは困難である。実験結果と同様に、コンピュータシミュレーション結果にも誤差・精度という概念を持ち込む必要がある。次に、コンピュータシミュレーション手法について述べる。物理現象の多くは偏微分方程式で表現できる。これを計算機で数値的に解くためには、方程式を離散化する必要がある。離散化の方法としては差分法(finite difference method)、有限要素法(finiteelement method)、境界要素法(boundary element method)があり、離散化された方程式は多元の連立方程式となり、最終的にこれを解く。差分法は図3・105(a)に示すように、座標軸の方向に計算領域を格子分割し、各格子点における値を用いて方程式の微分項を近似的に表現する。格子線によって囲まれる各体積要素について、物理量の保存則をもとに定式化を行う有限体積法(finite volumemethod)も用いられる。有限要素法は図3・105(b)に示すように、計算領域を任意の三角形あるいは四角形要素に分割し、離散化方程式を得る。非構造格子を用いる解法ははん用性に優れているが、格子点の配置が不規則であるため、計算処理上は構造格子に比べて劣る。一方、熱伝導問題を表現するラプラス方程式のように、微分方程式の基本解が解析的に得られる場合は境界値に応じて基本解を重ね合わせることができる。この方法を境界要素法といい、図3・105(c)に示すように境界上のみを要素分割すればよいので、問題の次元を一次元落とすことができ、効率的な計算が可能となる。運動学、剛体力学、マルチボデーダイナミクスの場合には、上記の格子分割と異なり、等価な多質点に置換えて解析する。最終的に離散化された方程式は多元の連立方程式となる点は同じである。実際のコンピュータシミュレーションでは上記の計算領域の格子分割、多質点分割を含め、計算条件の設定、境界条件の設定、材料定数の設定等を行うプリプロセッサと、計算した結果(変位、応力、速度、圧力等)を視覚的に提示するポストプロセッサが必要となる。b. 設計シミュレーション:FOAコンピュータシミュレーションが現象を直接的に表現し、大規模に計算機で解を求めるのに対し、現象を目的に応じて、より単純化し、数式レベルで表現して解を求めるのが設計シミュレーション(simulation for design)である。モデルの単純化には設計者の経験・ノウハウ、実験結果等が用いられる。便覧等にある設計式も設計シミュレーションの一手法と考えられる。菊池が提案しているFOA(first order analysis)も究極的には設計シミュレーションを目指している。菊池によるとFOA とは「あたりをつける計算(シミュレーション)」であり、設計がまさにあたりをつける作業の連続であることから、この考えは非常に有益である。したがって、ここでは設計シミュレーションはFOA
