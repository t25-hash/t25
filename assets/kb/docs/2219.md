# 3・8・4 積層造形のデータ処理

> **章**: 第3章 設計のための個別方法論
> **節**: 3・8 ラピッドプロトタイピング

3D- CADの形状データに基づいて積層造形を行うためのデータ
処理の流れを概説する．
まず，3D- CADの立体形状データをSTL形式（stereolithogra-
phy）のデータに変換する．STL形式データとは，ラピッドプロ
トタイピングという分野の初期に商用化された光造形システムで使
用されたデータ形式であり，その歴史的経緯により現在では事実上
の標準（de facto standard）として用いられている．STL形式デ
ータは，図3･141(ａ)に示すように，曲面を含めて立体形状の表面
を三角形の平面の集まりによって近似表現したものである．この三
角形の平面のことをファセット（facet）と呼ぶ．STL形式データ
では，図3･141(ｂ)に示すように，立体形状全体をsolidとend-
solidというラベルでくくって定義する．その中には階層的に，
facetとendfacetというラベルでくくって三角形ファセットが定
義され，そのファセットの外向きの法線（normal）ベクトルの，， 成分，三角形の三つの頂点（vertex）それぞれの，， 座
標値が記述されている．このようなファセットの定義が，立体を構
成するすべてのファセットについて列挙されている．STL形式デ
ータは構文が簡単であり，積層の断面形状が平面と平面の交差に基
づき計算できる利点があるが，頂点，稜線，面の接続関係や数の関
係など位相情報を有しないため，計算精度などの問題によりファセ
ット間にすきまが発生したりファセットどうしが交差するなど，矛
盾を有するデータが生成される可能性がある．そのような場合に
は，ラピッドプロトタイピング装置の造形ソフトウェアなどで矛盾
を検出し，データを補正する必要がある．
一般的には3D-CADシステムから目的形状のSTL形式のデー
タファイルが出力され，ラピッドプロトタイピング装置の造形ソフ
トウェアがそれを読み込み，その後の処理を行う．例えば図3･133
の例では，3D-CADの内部でモデリングされた図3･133(ａ)の形
状データは，ファセットによる近似モデル（図3･142）に変換さ
れ，STL形式のファイルとして出力される．ラピッドプロトタイ
solid
ピング装置の造形ソフトウェアはSTLファイルを読み込み，その
立体形状を一定間隔の水平な平面で切断して，その断面の輪郭デー
タの集まりを作成する［図3･133(ｂ)］．このときに上下の断面形
状を比較し，必要な箇所にサポートの断面形状を挿入する．この断
面の輪郭は立体の表面のみで中身がないので，中身が詰まった実体
を造形するために，輪郭内部を塗りつぶす走査データを作成する．
その塗りつぶしデータに基づいて，レーザや樹脂ノズルなど，各造
形方式の制御データを作成し，それを造形機に送ることによって，
積層造形が自動的に行われる．
