# 3・8・4 積層造形のデータ処理

> **章**: 第3章 設計のための個別方法論
> **節**: 3・8 ラピッドプロトタイピング

3D-CAD の形状データに基づいて積層造形を行うためのデータ処理の流れを概説する。まず、3D-CAD の立体形状データをSTL形式(stereolithogra-phy)のデータに変換する。STL形式データとは、ラピッドプロトタイピングという分野の初期に商用化された光造形システムで使用されたデータ形式であり、その歴史的経緯により現在では事実上の標準(de facto standard)として用いられている。STL形式データは、図3・141(a)に示すように、曲面を含めて立体形状の表面を三角形の平面の集まりによって近似表現したものである。この三角形の平面のことをファセット(facet)と呼ぶ。STL形式データでは、図3・141(b)に示すように、立体形状全体をsolidとend-solidというラベルでくくって定義する。このようなファセットの定義が、立体を構成するすべてのファセットについて列挙されている。STL形式データは構文が簡単であり、積層の断面形状が平面と平面の交差に基づき計算できる利点があるが、頂点、稜線、面の接続関係や数の関係など位相情報を有しないため、計算精度などの問題によりファセット間にすきまが発生したりファセットどうしが交差するなど、矛盾を有するデータが生成される可能性がある。そのような場合には、ラピッドプロトタイピング装置の造形ソフトウェアなどで矛盾を検出し、データを補正する必要がある。一般的には3D-CAD システムから目的形状のSTL形式のデータファイルが出力され、ラピッドプロトタイピング装置の造形ソフトウェアがそれを読み込み、その後の処理を行う。例えば図3・133の例では、3D-CAD の内部でモデリングされた図3・133(a)の形状データは、ファセットによる近似モデル(図3・142)に変換され、STL形式のファイルとして出力される。ラピッドプロトタイsolid facet normal0.0 0.0 1.0outer loopvertex3.0 0.0 1.0vertex3.0 2.0 1.0vertex0.0 2.0 1.0endloopendfacet...(省略)。..facet normal1.0 0.0 0.0outer loopvertex3.0 2.0 0.0vertex3.0 2.0 1.0vertex3.0 0.0 0.0endloopendfacetendsolid(a) ファセット表現(b) STLデータ形式図3・141 ファセットによる立体表現とSTLデータ形式ピング装置の造形ソフトウェアはSTLファイルを読み込み、その立体形状を一定間隔の水平な平面で切断して、その断面の輪郭データの集まりを作成する[図3・133(b)].このときに上下の断面形状を比較し、必要な箇所にサポートの断面形状を挿入する。この断面の輪郭は立体の表面のみで中身がないので、中身が詰まった実体を造形するために、輪郭内部を塗りつぶす走査データを作成する。その塗りつぶしデータに基づいて、レーザや樹脂ノズルなど、各造形方式の制御データを作成し、それを造形機に送ることによって、積層造形が自動的に行われる。
