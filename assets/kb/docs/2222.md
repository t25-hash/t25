# 3・9・1 従来の工業デザイン方法

> **章**: 第3章 設計のための個別方法論
> **節**: 3・9 工業デザインのための方法

まず,工業デザイン(industrial design)の定義をしておこう.いろいろな考え方があるが,ここではユーザの要求事項を体系的に抽出して把握し,システムとしての提案を行い,それを可視化することをデザインと定義する.工業という制約があるため,工芸や特定の人向けの製品などはその対象には含めない.工業デザインという業務は1950年代に電機や自動車のメーカなどにおいて開始された.今日では,デザイナーの可視化手段がマーカやパステルから計算機に替わったが,その基本的なデザイン手法は当時からほとんど変わっていない.その基本手法は,まず最初に,ある程度固まった製品コンセプトに基づいて,各種の生活シーンを想定しつつアイデアスケッチを描く.次に,拡散させたアイデアスケッチをラフスケッチとして収れんさせる.この段階で製造やマーケティングの面から,関係者との打合せを行い,方針を固めていく.ここで関係者から出た要望を加味して,さらに何点かのレンダリング(完成予想図)にまとめ,会議を開催して最終デザイン案を決定するのが一般的であろう.なお,以上に述べたプロセスは,順調に進んだ場合でも,通常,ラフスケッチの作業レベルで数回に及ぶ打合せを行わざるを得ない場合や検討漏れが起こることも多い.従来のデザイン手法の作業手順は,上記のように,関係者の意見を取り入れつつコンセプトを具現化していくという点で自然なものである.そしてこの方法は,ここ半世紀,連綿と引き継がれてきた.しかし,この従来の方法は,製品の機能が単純で,開発のスピードを要求されず,機能面での優位性で製品が売れていた時代ではなんら問題はなかった.ところが,1985年以降,計算機化によって製品のブラックボックス化が進み,商品としての機能の差がなくなった結果,ユーザニーズに基づいた製品企画が最重要テーマになっている.このように激変したものづくりの環境へは従来の方法では対応が難しい.特に,1990年以降,製造物責任(PL:product lia-bility)法,顧客満足度(CS : customer satisfaction),人間中心設計プロセス(ISO13407:1999,Human centred design processfor interactive systems)などへの対応により,デザイン時に検討しなくてはならない事項は増加している.一連の変化の背景においてはものづくりのベクトルがハード面での要求事項の重視からユーザの立場からの要求事項の重視へとシフトしてきていることが大きく,新しいデザイン方法の展開が求められている.
