# 3・9・1 従来の工業デザイン方法

> **章**: 第3章 設計のための個別方法論
> **節**: 3・9 工業デザインのための方法

まず，工業デザイン（industrial design）の定義をしておこう．
いろいろな考え方があるが，ここではユーザの要求事項を体系的に
抽出して把握し，システムとしての提案を行い，それを可視化する
ことをデザインと定義する．工業という制約があるため，工芸や特
定の人向けの製品などはその対象には含めない．
工業デザインという業務は1950年代に電機や自動車のメーカな
どにおいて開始された．今日では，デザイナーの可視化手段がマー
カやパステルから計算機に替わったが，その基本的なデザイン手法
は当時からほとんど変わっていない．
その基本手法は，まず最初に，ある程度固まった製品コンセプト
に基づいて，各種の生活シーンを想定しつつアイデアスケッチを描
く．次に，拡散させたアイデアスケッチをラフスケッチとして収れ
んさせる．この段階で製造やマーケティングの面から，関係者との
打合せを行い，方針を固めていく．ここで関係者から出た要望を加
味して，さらに何点かのレンダリング（完成予想図）にまとめ，会
議を開催して最終デザイン案を決定するのが一般的であろう．な
お，以上に述べたプロセスは，順調に進んだ場合でも，通常，ラフ
スケッチの作業レベルで数回に及ぶ打合せを行わざるを得ない場合
や検討漏れが起こることも多い．
従来のデザイン手法の作業手順は，上記のように，関係者の意見
を取り入れつつコンセプトを具現化していくという点で自然なもの
である．そしてこの方法は，ここ半世紀，連綿と引き継がれてき
た．
しかし，この従来の方法は，製品の機能が単純で，開発のスピー
ドを要求されず，機能面での優位性で製品が売れていた時代ではな
んら問題はなかった．ところが，1985年以降，計算機化によって
製品のブラックボックス化が進み，商品としての機能の差がなくな
った結果，ユーザニーズに基づいた製品企画が最重要テーマになっ
ている．このように激変したものづくりの環境へは従来の方法では
対応が難しい．特に，1990年以降，製造物責任（PL:product lia-
bility）法，顧客満足度（CS:customer satisfaction），人間中心
設計プロセス（ISO13407：1999，Human centred design process
for interactive systems）などへの対応により，デザイン時に検討
しなくてはならない事項は増加している．一連の変化の背景におい
てはものづくりのベクトルがハード面での要求事項の重視からユー
ザの立場からの要求事項の重視へとシフトしてきていることが大き
く，新しいデザイン方法の展開が求められている．
3・9・2新しい工業デザインの方法
a.従来の方法との比較 1985年以前では，あいまいな企画や
スペックのまま，デザインを行うスタイルが多かったようである．
このような状況では，徐々にコンセプトを収れんさせる従来の方法
は，なんら問題はなかった．しかし，可視化の前に検討しなくては
ならない条件が多数ある状況では，このような方法では効率が極め
て悪い．それを克服する新しいデザインの方法では，システム設計
の考え方を取り入れつつ，コンセプトを徹底的に検討し構築した
後，可視化を行う必要がある．その理由は可視化の前に，頑強性の
高いコンセプトを構築することにより，試行錯誤の時間をなくし，
効率的なプロセスにすることが可能であるためである．この考え方
に基づいて考案された工業デザインの方法論の一つとしてヒューマ
ンデザインテクノロジー（HDT:human design technol-
ogy) の考え方を工業デザインに適用した方法を以下に紹介
する．

る．
(３)製品コンセプト構築ステップ：ユーザ要求事項に基づいて構
造化された製品のコンセプトを構築する．コンセプト項目は
ウエイト付けされる．
(４)デザイン（総合化）ステップ：構造化コンセプトに基づい
て，デザイン案を可視化する．
(５)デザイン評価ステップ：可視化されたデザイン案に対して，

(６)ユーザ使用実態調査：商品を購入し使用したユーザに対し行

(１) ヒューマンマシンインタフェース（HMI）の5側面から

用的側面からインタフェース上の問題点を抽出する．身体的側面

システムとユーザとの不都合は痕跡となって現れるので，痕跡を
見つけることにより問題点を抽出することができる．

情報は識別することにより特定できる．この点から問題点を探

.3 Pタスク分析（3P task analysis）の方法 タスク
分析とは機器を使う際のシーンごとのタスクに関して，問題点を抽
出する方法である．大まかにいうと，タスクが連続して切り替わっ
ていくシーケンシャル型と，タスクが階層構造になっている階層型
の2種類がある．この場合，ユーザを使って問題点を抽出すること
が理想的であるが，ユーザを使わず実験者がユーザになったつもり
で問題点を抽出してもよい．
従来のタスク分析の大部分はおもに機器の入出力部分の問題点を
抽出する構造になっていて，ユーザの情報処理レベルでの視点が弱
い．このような従来のタスク分析に対し，評価検討抜けがないこと
とユーザの視点で評価できるようにするため，ユーザの情報処理レ
ベルでの問題点抽出をできるようにしたものが3Pタスク分析（図
3･144）である．この方法では，情報処理レベルにおいて，情報入
手→理解・判断→操作の3ステップから問題点を抽出する．従来の
方法では問題点の抽出段階で終わっているが，さらに製品コンセプ
ト構築につなげるため，それぞれのタスクにおいて抽出された問題
点を要求事項に変換する欄を右端に設けている．変換は7項目の手
がかりを参考にして行う．この欄は現実的に解決できる項目を書く
欄と，現実にはコスト，技術的に無理ではあるが，近い将来実現が
見込まれる項目を書く欄の二つに分かれている．
三つのステップのうち，情報入手ステップは，ユーザが情報を入
手する段階で，情報をいかに容易に手に入れるかがポイントであ
る．人間の情報処理の感覚，知覚レベルに該当する．
理解・判断ステップは，知覚情報を理解して判断を下す段階であ
る．わかりやすさの構築がポイントである．
操作ステップは，判断された情報を人間の効果器（手や足など）
を使って機械に与える段階である．身体面での快適，効率性がポイ
ントである．
なお，以上の各ステップにより30機種の電化製品のタスク分析
を行ったときの問題発生頻度の高い評価項目は，以下のようなもの

b.状況把握の方法 企画する製品のイメージを構築するため
には，現在市場に出ている商品がユーザにどのように知覚されてい
るかを確認する必要がある．
これについては，コレスポンデンス分析（correspondence anal-
ysis method) を活用することができる．
調査したい商品群に対して，評価項目（高級感がある，モダン
な，など）がどの程度該当するのか，ターゲットユーザに対するア
ンケートにより各項目の頻度を求める．アンケート結果は，コレス
ポンデンス分析により二次元座標上に，商品と評価項目が同一面上
に布置され，それらの関係が深い場合，近くに布置される．見方は
原点からの方向と原点からの長さから検討する．それらの商品が置
かれていない領域をねらい競争を避けるか，あえて同じ土俵上で勝
負するのかを決める．
