# 3・9・3 デザイン開発プロセスの各ステップの方法

> **章**: 第3章 設計のための個別方法論
> **節**: 3・9 工業デザインのための方法

a. ユーザニーズ収集の方法ユーザのニーズは、3P(ポイント)タスク分析、直接観察やグループインタビュー、アクティブリスニング法、評価グリッド法などの方法を用いて収集することができる。HDT では、それらの中でも、ユーザの無意識の行動を分析することにより潜在化したユーザニーズを得ることができる3Pタスク分析や直接観察を重要視している。それらの方法は、ユーザを用いないことも可能であり、コストの面からも魅力的な手法である。図3・143 HDT のプロセス。 直接観察の方法(direct observation method)直接観察による具体的な方法を下記に示す。(1) ヒューマンマシンインタフェース(HMI)の5側面から問題点を抽出するHMIの身体的側面、頭脳的側面、時間的側面、環境的側面、運用的側面からインタフェース上の問題点を抽出する。身体的側面は、1トルク、2位置、3フィット性の3項目から身体面での問題点を抽出する。頭脳的側面では、情報のインタラクションでの問題点、時間的側面では作業時間に関する問題点、環境的側面は照明、空気調和など環境に関する問題点、そして運用的側面はHMIを効率よく運営していくうえでの問題点を抽出する。(2) 痕跡から問題点を探すシステムとユーザとの不都合は痕跡となって現れるので、痕跡を見つけることにより問題点を抽出することができる。(3) 操作、行動の手がかり(cue)から問題点を探すわれわれが操作したり行動したりする場合、手がかりが必要である。この手がかりが何か、あるいはそれが効果的なのかを検討する。(4) 識別性から問題点を探す情報は識別することにより特定できる。この点から問題点を探す。(5) システム側によるユーザに対する制約条件から問題点を探す券売機のコイン投入口での制約された操作など、ユーザの操作、行動に対する制約条件を調べることにより、問題点を抽出できる。以上(1)~(5)の観察により抽出された問題点は、グループごとにまとめて要求事項化される。. 3 P タスク分析(3P task analysis)の方法タスク分析とは機器を使う際のシーンごとのタスクに関して、問題点を抽出する方法である。大まかにいうと、タスクが連続して切り替わっていくシーケンシャル型と、タスクが階層構造になっている階層型の2種類がある。この場合、ユーザを使って問題点を抽出することが理想的であるが、ユーザを使わず実験者がユーザになったつもりで問題点を抽出してもよい。従来のタスク分析の大部分はおもに機器の入出力部分の問題点を抽出する構造になっていて、ユーザの情報処理レベルでの視点が弱い。このような従来のタスク分析に対し、評価検討抜けがないこととユーザの視点で評価できるようにするため、ユーザの情報処理レベルでの問題点抽出をできるようにしたものが3Pタスク分析(図手→理解・判断→操作の3ステップから問題点を抽出する。従来の方法では問題点の抽出段階で終わっているが、さらに製品コンセプト構築につなげるため、それぞれのタスクにおいて抽出された問題点を要求事項に変換する欄を右端に設けている。変換は7項目の手がかりを参考にして行う。この欄は現実的に解決できる項目を書く欄と、現実にはコスト、技術的に無理ではあるが、近い将来実現が見込まれる項目を書く欄の二つに分かれている。三つのステップのうち、情報入手ステップは、ユーザが情報を入手する段階で、情報をいかに容易に手に入れるかがポイントである。人間の情報処理の感覚、知覚レベルに該当する。理解・判断ステップは、知覚情報を理解して判断を下す段階である。わかりやすさの構築がポイントである。操作ステップは、判断された情報を人間の効果器(手や足など)を使って機械に与える段階である。身体面での快適、効率性がポイントである。なお、以上の各ステップにより30機種の電化製品のタスク分析を行ったときの問題発生頻度の高い評価項目は、以下のようなものである。これらの項目を手がかりにすると、予備知識がなくとも問題点の検出は容易になる。(1) 情報入手:1レイアウト(位置)が悪い、2見にくい、3強調されていない、4情報がない、5マッピングがされていない(2) 理解・判断:1意味不明、2アフォーダンスがない、3紛らわしい、間違える、4フィードバックがない、5手順が問題、6一貫性がない、7メンタルモデルが問題(3) 操作:1身体的特性と不一致[特に、姿勢、フィット性およびトルク(操作に必要な力)の3項目が重要],2面倒であるb. 状況把握の方法企画する製品のイメージを構築するためには、現在市場に出ている商品がユーザにどのように知覚されているかを確認する必要がある。これについては、コレスポンデンス分析(correspondence anal-ysis method)を活用することができる。調査したい商品群に対して、評価項目(高級感がある、モダンな、など)がどの程度該当するのか、ターゲットユーザに対するアンケートにより各項目の頻度を求める。アンケート結果は、コレスポンデンス分析により二次元座標上に、商品と評価項目が同一面上に布置され、それらの関係が深い場合、近くに布置される。見方は原点からの方向と原点からの長さから検討する。それらの商品が置かれていない領域をねらい競争を避けるか、あえて同じ土俵上で勝負するのかを決める。c. 製品コンセプト(product concept)構築の方法。 コンセプト項目の体系化ユーザニーズ収集により得られたユーザの要求事項は、同じ機能どうしでまとめて階層化を行うことによりコンセプト項目として体系化し、製品コンセプトを構築する。まず、上位の項目と最上位項目の構造の文脈から、新しい上位の項目があれば追加する。また、この要求事項はもともと問題点図3・144 3Pタスク分析から作られた項目であるため、その製品の必要十分条件の項目がない場合や企画者として追加したい項目がある場合、必要に応じてこれらも追加する。さらに上位項目、下位項目のウエイト付けを行う。このウエイト付けは非常に重要な作業であり、これにより製品コンセプトの骨格が決まる。ウエイト付けは、ユーザの意向を反映させるため、アンケート調査やAHP(analytic hierarchy proc-ess)等により行う。これらのウエイト値は製造のコスト比にもなるため、コストが合わない場合、ウエイト値が低い項目を排除することも検討できる(図3・144)。最後に、体系図の最下位のコンセプト項目に対して抽象化された意味(設計項目)を与えることにより製品コンセプトの体系図を完成させる。図3・145の上半分はその一例である。. 設計項目への対応付けとそれらの種別製品コンセプトを明確に整理するうえで設計項目の種類や分類を知っておければ、デザイン経験にかかわらず要求事項の具現化が可能となる。HDTでは、設計項目を八つの大設計項目とその下位の70の小設計項目に分類して体系化している。一連の項目のもと、製品コンセプトの最下層のコンセプト項目に関係する設計項目を選択すれば、これらの項目から具体的なアイデアを創出することができるようになる。例えば、壁掛け時計のデザインで、文字の見やすさの獲得が最下層のコンセプト項目であるならば、これに関係するユニバーサルデザイン設計項目の情報入手を参照して文字のサイズとか形状のアイデアを固める。この種の設計項目はすべての製品に対応できるわけではないが、基本的で重要な項目である。下記に、製品に必要な検討要素に対応した八つの設計項目を示す。また、この8項目には、合わせて70の下位の設計項目があるので、( デザインを総合化する方法デザインの具体案は、製品コンセプトの最下層のコンセプト項目とこれに関係する設計項目から具体的な絵か文字でアイデアを表現することを介して生成することができる。この作業は企業内のイントラネットを活用して、所定の時間に関係者にネットワークの端末に集まってもらい、関係部門の担当者がそれぞれの立場からアイデアを書き込んでいくことも可能である。この方法を活用すると、関係者が会議室に集合しなくとも意思決定が可能である。しかも何回も会議をしなくても済むので、従来の方法と比べて非常に効率的である。製品コンセプトとその各項目のウエイト値を参照しつつ、各部位分のアイデアを統合すれば最終デザイン案を構築することができる。図3・145の下半分はその具体例である。この段階では製品コンセプトとその項目のウエイト付けが終わっているが、コンジョイント分析(conjoint analysis)によりスペックの特定とウエイト付けを行うこともできる。まず、未定の製品属性に関する水準を組合わせたデザイン案を作成し、ターゲットユーザにそれらの案に順位をつけてもらう。次に、コンジョイント分析により属性(attribute)の寄与率(rate of contribution)と水準(level)の効用値(utility value)が計算されるので、高得点の項目を組合わせ、スペックを決めることができる。e. デザイン評価の方法前ステップで作成したデザイン案と市場に出回っている商品との評価をAHPを用いて行う。この評価によりある程度の評価を得ることができれば商品化へのめどがつくことになる。この場合、製品コンセプト項目を基準にして評価を行うが、このことは案の妥当性(verification、仕様の確認という意味)を検討する意味もある。さらに製品のモックアップ(製品を模した模型)や3Pタスク分析などで有効性の確認(validation)を行う。あるいは、製品は有用性(useful)、利便性(usable)と魅力的(desirable)の3属性から構成されていると考えて、この3項目からデザイン案の評価を行うこともできる。また、製品の特性に対応して3項目のサブ項目で評価を行ってもよい。
