# 3・10・1 システムの内容

> **章**: 第3章 設計のための個別方法論
> **節**: 3・10 システマティックデザイン

システム(system)という言葉は今日では日常語としても広範に用いられている。元来は、複雑なものの全体、相互に連結された部分からなる全体などを意味して、およそ、あらゆるものを指すが、人工物の複雑化を受けて、そのような性質を伴うハードウェアやソフトウェアがその狭義の意味となって、現在に至っている。設計工学への要請の一つは複雑化した人工物の設計において個別の技術領域における知識を統合化しようとすることにある。このことから、設計に関わる対象物などをシステムとして見ることは、設計工学における根幹をなす考え方の一つとなっている。なお、システムの厳密な意味は、JIS によれば、例えば、1967年制定のZ8121「オペレーションズリサーチ用語」には「体系(system)」が「多種の構成要素が有機的な秩序を保ち、同一目的に向って行動するもの」として、1994年制定のZ 8116「自動制御用語一般」には「系、システム(system)」が「所定の目的を達成するために要素又は系を結合した全体」として示されている。以上のようなシステムの内容は、以下に示す四つの条件によって具体的に定義される。(1) 構成要素(components)から成り立っている。(2) 構成要素どうしが連結されている。その際の連結の仕方を構造(structure)という。(3) 固有の使用目的(system purpose,objective)をもつ。(4) 外部からコントロールが可能である。システムについての議論を行うためには、その対象範囲を明らかにしておくことが必要である。その際、対象外とする範囲は環境(environment)、あるいは外部システム(external system)、対象とする範囲は内部システム(internal system)という。図3・147は以上のことを模式的にまとめたものである。また、システムが大規模になって全体をいくつかの部分に分ける場合、分割された部分をサブシステム(subsystem)という。システムを機能的な側面からとらえて、図3・148の上部に示すように、システムへの入出力がエネルギー・物質・情報という三つの形態をとるとした場合、図3・148中部のように、全体はいくつかのサブシステムに分割することができ、全体のシステムに対する入出力も対応してサブシステム間の入出力へと展開することができ、さらに図ムへと再帰的に分割できるということになる。このことは設計においても重要であり、全体システムについての要求が明確になってサブシステムへの展開が定義できれば、設計処理をサブシステムごとに分割することができたり、また既存のシステムに対して、あるサブシステムを代替することによって、新たな設計解を導き出したりすることができる可能性を暗に示している。
