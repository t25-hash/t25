# 3・10・2 システムの設計

> **章**: 第3章 設計のための個別方法論
> **節**: 3・148 下部のように,個別のサブシステムはより細かなサブシステ

上記のことを抽象的にとらえて設計を行うということを考えれば,図3・149のことがいえる.すなわち,設計を何らかの意味図3・147 システムの内容をもつ挙動を有する人工物の内容を定める問題としたとき,それがシステムとしての性質を伴っていることから,まず,想起された設計問題は目的とする挙動の内容に従って部分問題に分割することができ,そのもとで,各部分問題において該当する部分挙動を実現しうる部分設計解を得ることができれば,得られた一連の部分設計解を統合することによって全体としての設計解を導出することができるということになる.このような視点はおおよそ自明のものではあるが,それを基盤とすれば設計に関する有益な知見の意味も合理的に説明することができる.例えば,上述した設計の効率性は,図3・149のような状況のもとで以下のように説明することができる.すなわち,部分問題への分解が適切であれば,部分設計解の解決が容易になることが期待できる.その一方で,最終的には全体システムとしての整合性が求められることから,部分問題の解決で設計が完了するわけではなく,全体問題の解決と部分問題の解決とを繰返して行うことが避けられず,それに起因して,最終的に得られる設計解は絶対的な意味での最適解となることはなく,何らかの満足解とならざるを得ないということにもなっている.一方,設計プロセスを企画・概念設計・実体設計・詳細設計など段階的なものとして認識した場合に,コンカレントエンジニアリングなどにおいては企画や概念設計の重要性が指摘されている.このことは図3・149における階層性が再帰的に連鎖していると考えることにより理解が容易になる.すなわち,設計問題を部分問題へと展開する際により上位の設計問題を適切に解決しておく,つまり,部分問題への分割を適切に行って早期にその課題を明確にしておくことができれば,下位の設計問題の解決が容易なものとなり,ひいては最終的な設計解に至るまで繰返しの必要性を低減することができ,全体として設計に必要となる期間を短縮できることになる.
