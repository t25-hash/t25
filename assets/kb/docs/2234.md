# 3・10・5 機能の複雑化のもとでの設計の分化と共有

> **章**: 第3章 設計のための個別方法論
> **節**: 3・148 下部のように,個別のサブシステムはより細かなサブシステ

上記の内容は,大局的に見ればどちらかといえば明確な単一の目的をもつ機械についてのものであったが,機械のもつ機能そのものが複雑化した場合には,いうなれば機械から製品へと展開した場合には,3・10・4項での様相はそれほど単純ではなくなる.これは,設計対象が多機能化し,内容が複雑化した場合には,それを評価する際の社会法則も複雑化して,旧来のものとは異なったものとなることによる.一例として,耐久生産財と耐久消費財とを比べた場合,前者においては,性能面での明確な目的が存在している場合が多く,その性能をいかにして効率的に実現できるかが技術の目的となる.それに対して,後者においては,主要な性能のレベルやそれに対するコストのみならず,使い勝手や意匠などの付加価値も重要な指標になってきて,一連の項目間のバランスが優れた製品であるための必須の条件になってくる.そうなった場合には,図3・149に示したように,全体の問題を部分問題に分割して解いていくというよりも,部分問題そのものが重要な設計問題であって,相互の統合性(まとまりのよさ,integrity)が重要になってくる.その種の設計における要は,アーキテクチャ(architecture)という考え方によって整理することができる.アーキテクチャとは機能を実体に展開する際の部分機能と部分実体との対応関係を指し,まず,設計の目的を図3・149のように単一のものとして見るのではなく,複数の部分機能の実現とそれらのバランスの達成としてとらえる.そのもとで,図3・153にも示すように,部分機能と部分実体との対応関係が混然としている場合をインテグラルアーキテクチャ(integral architecture)と呼び,1対1対応の関係が明示的な場合をモジュラアーキテクチャ(modular architecture)と呼ぶ.ある優れた製品を設計するためには,インテグラルアーキテクチャのもとで各部分間の関係を細かくすり合わせていくことが有効であるが,例えばこのすり合せを行うには設計チーム内での多くの相互作用を必要とするため,設計に要する期間は長くなってしまうな(c) 貫流ボイラ(b) 水管ボイラ(a) 丸ボイラ(b) モジュラアーキテクチャ(a) インテグラルアーキテクチャ図3・153 システムのアーキテクチャ図3・154 システムにおける多面性と品質機能展開どの弊害も生じる.短期間で設計を行うには,むしろモジュラアーキテクチャのもと,事前に明確な仕分けを行って設計を進めたほうが好ましい場合もある.極端な場合,設計対象の内容が膨大である場合には,絶対的に設計に要する時間が膨大なものとなるために,設計そのものを実現するためには何らかのモジュラアーキテクチャが必須条件となることも考えられる.上記は時間のもつ意味から見たものであるが,顧客ニーズの多様化によるマスカスタマイゼーションの動向もアーキテクチャという考え方から整理することができる.図3・153におけるインテグラルアーキテクチャは特定の機能要求に特化して部分実体相互のすり合せを行っているため,それ以外の機能要求に対しては適合できない.これに対して,モジュラアーキテクチャのもとでは,各部分機能に対応する部分実体が比較的独立したモジュールとして他の部分からは分離されていることから,異なる顧客ニーズに対しては全体を入れ換えるのではなく特定のモジュラを入れ換えることによって対応できる.つまり,モジュラアーキテクチャは有効な選択肢となる.以上のようなモジュラアーキテクチャの有効性はある種の社会法則によるものであることから,その背後では自然法則による制約も同時に作用する.つまり,設計は両条件の内容に従ってモジュラアーキテクチャとインテグラルアーキテクチャの間で定まることになる.社会法則の影響に関しては,メンテナンスやサービスを向上させるために使用時や再使用時にシステムの内容を組換えることはアーキテクチャの成せる技であると見ることができる.また,ライフサイクル設計における成功例にも同様に理解できるものが存在している.
