# 4・1・4 VE(value engineering)

> **章**: 第4章 設計の管理
> **節**: 4・6 に示すように,設計,作り方,買い方の三つのカテゴリーに分

VEとは,最低のライフサイクルコストで,必要な機能を確実に達成するため,製品・サービスの機能分析に注ぐ組織的努力,と定義される.その源泉は,ゼネラルエレクトリック社の購買担当者であったマイルズによる製品や購入品の価値を(機能)/(コスト)と考え,機能は変えずよりコストの安い代替品を探すアプローチであるVA にあり,これを製品開発・設計段階でも適用するために発展・体系化したものである.現状の製品や部品の原価低減・改善活動から始まったVEも,その適用範囲を製品ライフサイクルの源流に拡大した.その意味で,現状製品・部品の原価低減を図る2nd-Look VEから,製品開発段階のVEとして1st-Look VE,さらに製品企画段階のVEとして図4・6 原価企画の三つの要素1―154設計工学0-Look VEという呼び方もある.ジョブプランと呼ばれるVEの主要手順は,マイルズによる,(1)機能定義(それは何か,その働きは何か),(2)機能評価(そのコストはいくらか,その価値はどのくらいか),(3)代替案の作成(ほかに同じ働きをするものはないか,そのコストはいくらか,それは必要な機能を確実に果たすか),という三つのステップから構成される.a. 機能定義機能定義では,製品の構成要素を部位に分解し,その部位ごとにその機能を,可測的名詞と広義の動詞を用い,簡潔で明確に表現する,表現を抽象化する,否定文の機能定義は用いないというルールのもとで定義する.可測的名詞とは,荷重を支えるといった場合の荷重に相当し,どのくらいの荷重を支えるかによって,この機能を達成するための手段に大きな違いを生じるからである.また抽象化は,アイデアを出しやすくするための発想法に共通した制約である.そして図4・7に示すように,その部位の基本的な機能から,その目的を達成するための手段,さらにそれを目的として達成手段を列挙するというように,系統的に機能を定義・整理する.このような整理法および図は機能系統図と呼ばれる.そして,その定義された機能の働きの程度(達成度,仕様,性能等)と,制約条件を付記する.これによって同一機能でより安いコストが達成できるアイデアを出す基準が明確となる.b. コスト係数(図4・8右上):次に機能を達成するためのコストを算定する必要がある.このコストを算定するのに単位となる部品や作業(構成単位と呼ぶ)と機能は必ずしも1対1には対応しない.したがって構成要素別にコストを集計し,これを機能に配分するという方法がとられる.図4・8の例はA,B,C三つの構成要素について,材料費,製造費等から見積もられたコストが,四つの機能に配分されている.これから機能ごとに集計されたコストが求められ,A,B,Cの合計である総コストで割ることによって各機能のコスト係数が求められる.(3) 価値係数(図4・8下側):各機能の機能係数,コスト係数から,機能の価値は,価値係数=機能係数/コスト係数によって求められる.この価値係数が低いものがコストダウンのターゲットとなる.図4・8の例ではF1やF3である.特にF1は,現状のコストの絶対額も大きく,これを改善することによってコストダウンの大きな効果が見込まれる.c. 代替の作成機能に対するコストダウンのターゲットが決まることによって,次はそのためのアイデアを発想し,その具体的代替案を作成,そしてその評価を通してVE提案がなされる.アイデアの発想では,ブレーンストーミングや製品企画と同様な方法論が用いられる.さらに具体的代替案の評価では,コスト見積りに加えてDR 的な視点でマイナスの品質を徹底的にチェックしておくことも忘れてはならない.原価企画でVEを有効活用するためには,アイデア創出や着眼点を支援するチェックリストをあらかじめ用意しておくことも必要である.これは過去のVE実績に基づき,有益な情報をまとめて提供することである.VRP(4・1・5項参照)と呼ばれる部品等の共用化,標準化の着眼点も,部品コストだけでなくその手配やハンドリングのための管理・間接コストを削減するためにも有効な概念である.加えて,設計図面の諸元や特徴から,ある程度の正確さで原価をすぐに見積もれるようなコストテーブルを作成しておくことも,原価企画活動を迅速に行うために有効である.表4・2は,VE手法を体系化したアメリカ国防総省(DOD
