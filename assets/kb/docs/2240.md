# 4・1・5 VRP(variety reduction program)

顧客や得意先の要求により、そのままでは製品やそれを構成する部品の数は増加の一途をたどる。そうすると製品原価の中で、特に取扱う部品や材料の点数に伴って発生する間接業務(設計、発注や品質管理)や多工程化に関するコストが飛躍的に増加する。このような多品種・多部品・多工程に着眼したコスト削減技法としてVRPがある。VRPは、表4・3に示すような視点に基づき、顧客の要求を満たしながら部品数や製品・設備(工程)数を削減し、それに関わる製品コストを削減するものである。また流用設計を促進することにもなり、設計の効率化や開発リードタイム短縮にもつながる。現状の多品種・多部品・多工程の問題を解決するとともに、製品開発時における原価を作り込むチェックリストとして必須事項であるといっても過言ではない。五つの視点のうち固定-変動、組合せ、系列化は、どちらかといえば製品アーキテクチャ(product バラエティリダクションの視点固定-変動製品群のベースとなる固定部分と市場ニーズに対応した変動部分の切分けの明確化。組合せ部品やユニットを簡素化し互換性をもたせることで、製品の多様化にユニットの組合せで対応。モジュール設計はその例。多機能・集約化部品ごとの機能の集約化を図ることによって、最少の部品で必要な多機能を満たす。レンジ化一つの部品がカバーする使用範囲を最大化し、部品の種類や型・治工具が増えないようにする。系列化部品や設備に要求されている性能や仕様値を一定の法則で配列し直すことによって、余分な部品・製品を排除、管理コスト等を削減する。の多様化にも変動部分だけの対応で済ませられる。それにより固定部分については徹底した生産合理化が図れる。組合せは、部品やユニットを簡素化し互換性をもたせることで、製品の多様化にユニットの組合せで対応するものであり、モジュール設計(modulardesign)はその典型例である。また系列化は、部品や設備に要求されている性能や仕様値を一定の法則で配列し直し体系化することによって、余分な製品や部品を削減でき、管理コストを含めて大幅なコスト削減が期待できる。一方、残り二つは部品レベルでの部品数削減の視点である。多機能・集約化は、部品ごとの機能の集約化を図ることによって、最少の部品で必要な多機能を満たすことをねらったものであり、レンジ化は、一つの部品がカバーする使用範囲を最大化し、部品の種類や1―155表4・4
