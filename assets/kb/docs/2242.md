# 4・2・1 開発プロジェクトのタイプ

> **章**: 第4章 設計の管理
> **節**: 4・2 設計開発プロジェクト計画と管理

製品開発や設計のプロジェクトは,おもに創造的作業から構成されるコンセプト作りから製品計画まで(製品企画)と,製品設計を生産へと橋渡しする製品化開発の二つの本質的に性質の異なるプロセスから成り立っている.この製品企画と製品化開発では,図4・9に示すようにプロジェクトの進め方やそのための有効な方法論が根本的に異なる.このことを言い表すために,ここでは前者にコンティンジェンシーエンジニアリング(contingency engineering),後者にコンカレントエンジニアリング(concurrent engineering)という名前を付している.よりリスクの高い創造的作業からなる製品企画までの効率化を扱うコンセプトが,コンティンジェンシーエンジニアリングである.コンティンジェンシーエンジニアリングの対象としては,アイデアの創造・評価・選定,製品コンセプト創造と製品仕様決定,マルチ開発プロジェクト戦略,市場・技術・競合他社のベンチマーキングがあり,開発プロジェクトの進行に付随するリスクと機会というコンティンジェンシー要因(不確実要因)のマネジメントが機軸になる.ここではコア設計を流用化して開発する製品シリーズ全体のマネジメントが重要なポイントとなり,必然的に複数の開発プロジェクトの効率化が対象になる.すなわち,後述するように,製品開発の進捗に沿った開発作業の特性変化に伴って,ダイナミックに組織構造を変化させる部門横断チームの垂直統合を実現するというような取組みが必要とされる.一方,技術仕様が固まりそれを製品化するプロセスでは,コンカレントエンジニアリングが有効である.コンカレントエンジニアリングは,作業間・工程間の同時並行化により,上流工程と下流工程とが情報交流を密接にしながら協調的に作業を進める開発方式であり,特定製品の個別開発プロジェクトでの製品仕様決定以降の製品化開発において,効率化を実現するものである.製品化開発では,工程間の相互交流と調整をとりながら開発作業を同時並行的に進めるやり方,すなわち同時並行アプローチにより,特にリードタイム短縮に有効である.このようにコンカレントエンジニアリングは,複数の開発工程間,例えば設計と生産技術といった部門横断的な水平統合を確立するコンセプトといえる.一般に,製品開発プロセス全体の効率化の概念,特に製品開発のステップをオーバラップし,同時進行的に遂行することによってリードタイム短縮を図るものとしてコンカレントエンジニアリングが扱われている.しかしながらその中味をよく吟味すると,それが本当に有効なのは開発の後半部分,製品化開発の部分であるということである.以上をまとめると,開発プロジェクトの進め方として,基本構想や仕様が固まる前,あるいは開発がオーソライズされる前までは,コンティンジェントな状況でのインフォーマル情報を含めた上下左右の情報の共有化をベースとした仕事の仕方が求められる.一方,仕様が決まり市場に投入されるタイミングが決定された後は,基盤要素技術の先行開発の最新成果を取込み,フォーマル情報を主体とした水平的な情報共有化に基づき,まさにコンカレント的にオーバラップさせた仕事の仕方が可能な部分である.CAD,CAEやそのデータ交換等のIT(information technology,情報技術)環境も,この後半部分で特に威力を発揮する.図4・10は,以上を踏まえた製品開発プロジェクトのあり方のイメージ図である.市場の状況に俊敏に反応してコンティンジェンシ図4・9コンティンジェンシーエンジニアリングとコンカレントエンジニアリングの概念と区分図4・10 開発プロジェクトの構造とあり方ー,コンカレントに短期間に製品化を進める狭義の製品開発,それとは切り離したより長期の戦略に基づく基礎技術,基盤要素技術を先行的に走らせる先行開発の組合せからなっている.コンカレントエンジニアリングは,製品開発の部分が対象となり,それも後半のステップがコンカレント性をフルに期待すべきところであるということである.それによって外部条件によって決まる市場投入の時期に仕様決定を引きつけることが可能となり,最新の技術成果を新製品に盛り込むことができる.
