# 4・2・2 製品開発組織のタイプと効率

> **章**: 第4章 設計の管理
> **節**: 4・2 設計開発プロジェクト計画と管理

a.部門横断チーム（cross functional team）とそのタイプ
開発プロジェクトにおいてリードタイムの長期化や設計完成度低下
の最大原因に，縦割り機能的組織による情報伝達の遅延がある．こ
の欠点を補完するために，多様な部門のメンバから構成される部門
横断チームが有効である．部門横断チームは，チームリーダのもと
にマーケティング，製品開発，生産，購買，品質管理などの関係者
を一つのチームに集め，これらのメンバの非公式な情報交流を促進
する環境を実現し，開発初期段階で製品ライフサイクル全体を考慮
するのに有効な組織形態である．例えば，製品に関する情報を全員
で共有化し，開発初期段階でチームメンバ間の異なるニーズをトレ
ードオフできるため，全体的な視点から最適解を見つけ出すことが
可能となる．また，チームに製品開発の責任をもたせることによ
り，意識高揚や一体感の醸成が図られるとともに，情報交流が加速
化され，製品開発と生産との対立解消にも通じる．
製品開発のような非定型的な作業では，多大な工数と期間が，技
術・専門知識やノウハウの情報収集に費やされている．すなわち，
開発作業そのものより開発環境の整備や事前準備に多くの時間が割
かれている．これは，情報の共有化が効率的にできていない環境に
あることによる．情報共有化の最善の方法は，組織的・物理的な近
接配置環境に関係者全員を集めることであり，相互に目で見る管理
ができる環境を作ることである．近接配置環境を創造する手段とし
て最も有効なのが，部門横断チームの採用である．部門横断チーム
を運用することによって，開発者個人の能力，考え方，ノウハウと
いった属人的属性や，開発作業の進捗状況や必要な技術レベルとい
った作業的属性を全員が明確に把握できるようになる．したがっ
て，必要とする情報の収集効率が飛躍的に向上することから，事前
準備作業の必要性が少なくなり，付加価値作業に専念できることに
なる．すなわち，部門横断チームは縦割り機能的組織の壁を打破す
る組織的・物理的近接によって，開発目標の共有化，情報伝達の迅
速化と開発作業の効率化を実現する組織形態といえる．しかしなが
ら，部門横断チームを有効活用できるか否かは，チームメンバ間で
情報が円滑に伝達される仕組みができているか否かにかかってお
り，チーム組織化の方法によってその成否に大きな差が出る．
製品開発プロジェクトの組織形態に関しては，大きく分けて二つ
の形態があるといわれている．一つは，専門知識や技術のインプッ
ト側に対応した組織形態で，機能的組織形態と呼ばれるものであ
る．一般に基礎研究分野の開発組織では，同じ研究分野の開発者を
一つの組織下に配属することによって，専門知識の伝達や共有を容
易にしている．これは，専門知識の生成と伝達が，一般に専門分野
別に行われていることからも明らかである．機能的組織形態では，
