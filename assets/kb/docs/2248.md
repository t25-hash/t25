# 4・2・6 不確実性を前提とした手法

> **章**: 第4章 設計の管理
> **節**: 4・6 の問題に対する山積みの様子は、図4・16の下に示すような山

多くの場合、開発プロジェクトを構成する各アクティビティの遂行には不確実性やリスクを伴う。場合によっては目的としたアクティビティを完了できずに、開発プロジェクトそのものが途中で中止になることもある。以下に、所要時間に不確実性を持ち込んだPERT 手法としての3点見積り法、そしてアクティビティそのものにリスクを伴うプロジェクト計画法としてPDPCについて紹介する。a. 3点見積り法(three points estimation method)PERTにおける3点見積りとは、プロジェクトを構成する個々のアクティビティの所要期間について、最初から不確実性を盛り込む方法である。ここでは、各アクティビティの所要時間の楽観値(下限値)、最も実現できそうな最頻値、そして悲観値(上限値)の3点があらかじめ見積もられる。そして、この3点から各アクティビティの所要期間の平均と分散が、以下の式に基づいて計算される。平均値=(楽観値+4×最頻値+悲観値)/6分散=(悲観値-楽観値)/6これは所要時間の分布がβ分布に従うことを仮定したものである。そして各アクティビティの所要時間の平均値を用いて、4・2・4項で示したPERT 計算等が行われる。そのとき、クリティカルパスの完了期間がプロジェクトの平均完了時刻となる。そしてその完了時刻の分散は、クリティカルパス上のアクティビティの分散の総計として計算される。この分散を用いてある指定された日程内にプロジェクトが完了できる確率や、逆に完了できないリスクの計算が可能となる。b. PDPC(process decision program chart)PDPCとは、開発プロジェクトのように不確実性やリスクを前提としたプロジェクトの計画立案に対して、それらをあらかじめ見込み、あるいは読みきってプロジェクトの成功に導くための過程を示す計画チャートである。新QC七つ道具(N 7)の一つとしても知られる手法である。実際の活用事例では表記の仕方、また成功確率を高めるための計画の立案だけでなく、事前のボトルネック技術の予測・発見、進捗過程でそのつど実績や新たな情報を取り込むやり方や、4・進行過程をチャート化して残すことにより、次のプロジェクトのための問題点を摘出する用法もある。そのために表記の仕方等多くのバリエーションが存在する。図の目的が最上段に指定された記号のもとで示され、最下段にゴールが示されている。そして図右上には成功と判断するための前提条件が書かれている。そして、この目的とゴールを結ぶまでの想定される実施事項、予測される結果、それに基づく判断(デシジョン)等が列挙される。図4・21の場合には、当初三つの代替案とその優先順位が示され、それぞれの結果と判断によって次に打つべき手、すなわち実施事項が結ばれている。新商品開発や設備開発では、プロセスに必要になるフローがある程度想定できるものについては、多くの場合、このような図の横(例えば左側)に工程フローを明示し、それとの関係でPDPCのチャートを描く。想定されるリスクやボトルネックを事前に関係者が描いて、それを特定、共有することによって、プロジェクトの短縮・効率化や、何より途中で中止せざるを得ないというような状況を最小限にしようというものである。なお、TOCの中の思考プロセス(thinking process)と呼ばれる手法の中にも、PDPCと類似した手法がある。前提条件ツリー(PT : prerequisite tree)と呼ばれているものである。これはまず目標を掲げ、それを達成するための前提条件がその下に列挙される。次に、その前提条件を達成するために中間目標が順序関係を含めてその下に列挙され、同時に中間目標から前提条件に到達するための障害が同時に書かれる。さらに、中間目標を達成するための手段と想定される障害がその下に展開される。このように目標に到達するための目標のブレークダウンとその達成に伴う障害やリスクを明確にし、事前にそれに対して手を打つことによって確実な成功に結びつけようというものである。図4・21 PDPC の例
