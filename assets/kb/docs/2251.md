# 4・3・1 ・a項で述べたように、コンカレントエンジニアリングは、

> **章**: 第4章 設計の管理
> **節**: 4・2 節で述べたように、コンカレントエンジニアリングの原型

設計、製造プロセスの分化や、細分化で先行したアメリカにおいて1980年代まで主流であった、逐次処理、大量生産型のいわゆるアメリカ方式が競争力を失ったことに対する反省に立ち、リーンでかつ並列処理を可能とする方式として定式化されたものである。図ーとして提案された代表例である。図4・26は理想的なコンカレントな製品設計と製造システムのプロセスを表している。製品の販売価格、製品設計、システムデザインは、同時に考えて決められなければならない。いったんこれらの条件が満たされたなら、詳細設計、製造、アセンブリーの工程へと進められる。現在ではコンカレントな設計はチームアプローチで行われるのが一般的である。いかなる有能なエンジニアでも、このよ1―169うな内容豊富なアクティビティを一人で実行するほどの知識を持ち合わせることは不可能である。したがって通常は異なるチームどうしでそれぞれの検討結果を反映しながら設計内容を変え、よりよい結論を導いていくことでコンカレント設計を行っている(図4・27)。このような考えをさらに進めたものとして、バーチャルエンジニアのコンセプトが提案されている。現在のコンカレントエンジニアリングにおいて中核となる技術(道具)は、三次元CAD であるが、さらに進めて、CAE(解析シミュレーション)に重心を移した概念である。これは、ソリッドモデル作成と同時にロバスト性の高い自動メッシュ作成を行うことで解析シミュレーションを駆使した、予測エンジニアリングを可能にしようとするものである。バーチャルエンジニアリングの目標は、設計プロセスにおいて、応力、強度、剛性、騒音、振動、熱、流れなどについて設計目標値や顧客要望を満たすために、早くて正確なメッシュを作り解析を行い評価することにある。これが実現すれば、図4・28に示すようなCAD を中心としたアプローチに比べ、未来の設計プロセスでは図21世紀に入り、CAD,CAEツールとともに、大規模並列処理システムが各ワークステーションに安価に搭載されるようになり、さらにラピッドプロトタイピングもデスクトップ上で簡単に使えることになろう。そうすればCAD を中核としたチーム中心の現在のコンカレントエンジニアリングより、さらに短時間に少人数で複雑な設計をやり遂げることが可能になるであろう。
