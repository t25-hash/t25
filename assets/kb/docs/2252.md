# 4・3・2 設計作業のコンカレント化の方法とその効果

> **章**: 第4章 設計の管理
> **節**: 4・29 のようにさらなる期間短縮が可能になる.

いわゆる日曜大工をする者は,企画,構想から設計,製作そして利用に至るまでを一人で行っている.一人で行うため作業工程を見るとシーケンシャルであるが,優れた技術をもった日曜大工なら図4・27 設計,生産システムデザインの戦略的方法図4・26統合化された製品設計,製造システムデザイナー,エンジニア,解析者がデザインを行う.自動メッシュ,自動予知エンジニアリング機能なし.CAD がプロセスを主導.図4・28 CAD を中核とした設計プロセス1―170設計工学CAE が自動メッシュと自動予知エンジニアリング機能によって概念レベルで解析を実施.予知エンジニアリングがプロセスを主導.ば,頭の中では同時に設計者,製作者,利用者などさまざまな立場で考えているであろう.このように一人の人格なら簡単に実現できるであろうことを大規模な業務の場合でも可能なように,組織,道具,技術,風土などを整備し,大人数であるために生じるコミュニケーションロスなどの不具合を抑え,複数の人や組織で対応できるがゆえに可能になる同時並行作業を最大限に取り入れ,開発を効率的に行うことが目的となる.a. 方法設計プロセスにはさまざまな言い方があるが,一般的には企画,構想,基本設計,詳細設計,試作,実験といったプロセスからなっている.これらのプロセスにおいて,コンカレントエンジニアリングでは単に設計者のみで検討するのではなく,生産準備技術者,購買担当,サプライヤ,営業部員,リサイクル担当技術者などが参画し,それぞれの立場から最適な設計ができるよう要求したり,あるいは他の立場からの異なる要求との間に矛盾が発生した場合も,その場で調整を行い意志決定につなげようとするものである.しかし一方で,コンカレントエンジニアリングはいつも万能というわけではなく,正しい仕組みや考えに基づかない場合にはかえって混乱や非効率を招くことになる.例えば,商品力より生産性を重視しすぎて売れないものを作ってしまっては本末転倒である.またいくら並行作業を行っても,設計変更を多発するようではかえってむだを増幅することになる.その意味でコンカレントエンジニアリングを真に効果あるものとするには,チームワークやメンバ間および部門間の相互理解に優れており,さらにトレードオフ問題を正しく判断する技術力,リーダシップがあることが極めて重要になる.コンカレントエンジニアリングを支える道具としてのキーテクノロジーは三次元CAD をベースとした設計,生産準備,生産システムである.以下にコンカレントエンジニアリングを支える技術といった視点で,CAD/CAM/CAE/PDM についてその役割を簡単に述べる.. コンカレントエンジニアリングを支援する技術(道具)(1) CAD三次元CAD(three dimensional CAD)の歴史は古く,40年近い歴史がある.最初はいわゆるワイヤフレームといわれる三次元の線画からスタートした.その後サーフェスモデルを経て,1990年代からはソリッドモデルが本格的に利用されるようになった.ソリッドモデルはすでに1973年に二つの基本技術が発表されており,実用化に十数年の歳月がかかったのはコンピュータリソースとして高速大容量のデータ処理が可能でなければならなかったことが最大の要因であった.さらにモデリングに多大な工数を要することも普及を困難にしていたが,PTC社のPro-Engineerというソフトでフィーチャ,パラメトリック処理,履歴管理,二次元と三次元のアソシエイティビティの確保など使い勝手上で大きなブレークスルーがあり,本格的な活用の時代に入ることとなった.従来は設計者の道具にとどまっていたCAD が,サーフェスモデル(surfacemodel)やソリッドモデル(solid model)の普及により,あたか図4・2921世紀の設計プロセスモデルも実物のように画面上で見ることができるようになったため,二次元図や,三次元のワイヤフレーム図ではほとんど理解が不可能だった複雑な形状や機能を,実物を試作することなく生産,購買,営業など,設計以外の部門の人たちも容易に理解することができるようになった.この三次元CAD の実用化により,コンカレントエンジニアリングを,単に人間系に頼るだけではなく,有効な道具を伴った形で実現することが可能となった.(2) CAE三次元CAD の活用は設計以外のチーム・部門に大きな効果をもたらすが,CAEは設計本来の仕事を直接的に支援する道具といえる.CAEは例えば車の衝突実験のように,実際に試作し実験すると大きなコストがかかったりする場合や,エンジンの燃焼室内の流れ解析など目に見えないものを可視化したりする場合に特に有効になる.より高い精度のシミュレーションを可能にするためには,正確な解析モデルの作成と実際の現象との間の相関が高いレベルでとれた数値解析ツール(ソルバ),およびそれらの道具を使いこなすノウハウが必要となる.CAD データはプリプロセッサを介して解析用モデルに変換され,解析シュミレーションが実行される.シミュレーションの結果はポストプロセッサにより視覚化される.これらの一連の作業には高度な専門性が要求されたため,これまでは専門家が担当することが多かったが,コンカレント設計の道具としては,できるだけ製品設計者が直接扱えるのが望ましい.最近は自動メッシュ分割や,使いやすいユーザインタフェースをもったCAEソフトが普及しつつある.(3) PDM(product data management)CAD 化が進むとその活用のため,大量のデータの保存,検索,流通が必要になるが,通常のファイルシステムでは限界があるため,PDM の利用が広まりつつある.PDM は大変広い概念であるが通常,(a)技術情報の管理,(b)部品構成情報の管理,(c)ワークフローの管理の三つの機能からなる.コンカレント設計では情報の共有が大変重要であるが,PDM はその中核となるデータベース管理の仕組みである.通常はん用のパッケージを用いることになるが,多くが欧米製であるため,日本の仕事にマッチした形にカスタマイズする必要がある.しかし日本のような企業ごとに業務の仕組みが異なる場合,標準ソフトを使うメリットを出しにくく,今後の普及に当たっては大きな課題である.(4) ビジュアライザ(visualizer)三次元CAD は使いこなしが容易でなく,設計者以外が用いることはかなり難しいといえる.そこでコンカレントエンジニアリングを効果的に実現するための道具として,三次元ビジュアライザが有効な道具と位置付けられるようになった.しかし,従来のように一元化プロセスにより作られたデータはCAM で利用するため,微細な面やフィレットのデータも正確に作られ,人間が扱うには重すぎて実用性を損なうようになった.そこで,今後は図4・30に示すような,精密なデータの流れと人間が扱うに適した軽いデータの流れ図4・30ディジタルツインプロセスに分け,その間の整合をできるだけ自動的にとれるようなデータ交換機能をセットにした2階建てのプロセスである,ディジタルツインプロセスの概念の導入が必要と考えられる.(5) ラピッドプロトタイプ(rapid prototyping)CAD/CAM やビジュアライゼーションといったデータだけでコンカレントな設計ができるのが理想であろうが,形状が複雑などの理由で,実物がないと検討が困難な場合もある.しかし実物を試作することになれば,時間やコストがかかり,コンカレントな設計のメリットがなくなってしまう.1980年代後半から市場に登場したラピッドプロトタイプシステムはCAD データさえあれば簡単に実物と同じ形状が作成できるため,コンカレント設計を支援する道具として大変有効である.通常紫外線硬化樹脂を用い,短時間に実物モデルを作成できることから,光造形システムとも呼ばれる.当初は高価なうえに精度が不十分であることに課題があったが,最近では価格が下がりつつあり,精度もかなり向上してきているので,コンカレント設計の効果を高めるためのさらなる活用が期待される.また加工が不可能な中空形状や入り組んだ形状もCAD データさえあればほぼ自動的に作れることや,樹脂だけでなく紙や粉末金属などを材料に用いて造形できるものも出てきており,単に形状確認にとどまらず,空気や水の流れなど,簡単な実験も可能である.. 組織・体制チーム設計が一般的である.定常的な組織を設置する場合と,プロジェクトごとにチームを組む場合があるが,後者の形をとることが多い.チームのコミュニケーションが極めて大切になるため,環境面,人間関係面の両面においてさまざまな方法でチームとしての一体感を醸成する工夫が必要となる.チームによる検討の効率を上げるため特別の部屋を設けることが多く,そこにはCAD 用ワークステーションや,たくさんの人が一緒に検討しやすいように大型のプロジェクタ,ラウンドテーブルなどが設置される.これらの部屋をDMDR(digital mock up designreview,ディジタルモックアップデザインレビュー)室やゼロルーム(設計変更零化を目指す意),あるいはプラニングセンターなどといった名称で呼ぶ場合がある.一方の人間関係の面では,協調しコミュニケーションをとることが大切であることから,特にチームリーダの役割が重要になる.自動車産業のような裾野が広い業種の場合,コンカレント設計をしようとすると,そのメンバが部品メーカなど他社のエンジニアになる場合も多い.そのような場合はしばしば部品メーカからチームのメンバとして参加してもらう必要が出てくる.そのような方法をデザインインといい,参加メンバをゲストエンジニア,レジデントエンジニアなどと呼ぶ場合がある.. 意識と文化の変革三次元CAD は従来の二次元設計とは相当異なっており,二次元の文化で三次元設計に取り組むと必ず失敗するといっても過言ではない.二次元の図面なら簡単に描けるが,三次元では工数がかかりすぎるといった不満はしばしば聞かれ1―171るものである.そのため,三次元設計を効果あるものにするためには,単にオペレーションに習熟するだけではなく,パラメトリック設計,フィーチャ設計,形状創成時の思考プロセスなど,コンカレントなデザインを行ううえでどうするのが最適な方法であるのか,十分に考えて使いこなすことが必要になる.また異なる部門の立場から要求を言い合うだけでは決して合意や最適化はできない.協調的に作業を進めるためには,部門最適の考えを排除し,他部門や部品メーカの立場を理解し合う新しい文化を作り出すことが重要である.このことについては4・3・5項のボーイング777の開発におけるコンカレント設計の例で詳述する.b. 効果一般的に情報化投資は,(1)省力・自動化投資,(2)プロセスイノベーション,(3)競争優位の獲得(軍備拡張競争と同質),(4)戦略的投資(新しいビジネスモデルの創出),(5)インフラストラクチャの構築などの目的で行われる.コンカレント設計は(2)に該当する投資といえる.プロセスを革新することは,ねらいが正しくかつシナリオどおりに変革が実現できれば,企業内,企業間の業務プロセスのあらゆる部分でそのパフォーマンスを上げることが可能である.製品開発期間の短縮,設計変更の削減,試作,実験費,型改修費などの開発総コストの削減など,品質,コスト,期間(日程)短縮のすべてにわたって大きな効果をもたらす.またプロセスのイノベーションという性格から期待される効果は,単なる改善的な取組みに比べるとかなり高いレベルを要求される.例えば開発期間を半減する,設計変更を5分の1に削減する,干渉問題に起因する設計変更をすべてなくす,試作を廃止する,などの思い切った目標となる.したがってコンカレントエンジニアリングはねらいどおりに実現されたときの効果が極めて大きい.
