# 4・3・1 項で述べたように、設計対象が大規

> **章**: 第4章 設計の管理
> **節**: 4・29 のようにさらなる期間短縮が可能になる。

模・複雑化すれば業務の細分化や組織の分化が必要になる。分化には、設計過程(時間軸)による分化と、設計対象の機能別の分化、作業内容による分化などがある。設計過程による分化としては、企画、基本設計、詳細設計、作図などのプロセスに分けるのが一般的である。また設計対象の機能別分化は、例えば自動車の場合、意匠設計、車体設計、シャシ設計、内装設計、エンジン設計、駆動設計、電子設計などに分ける。作業内容による職種で見れば、エンジニア、デザイナー、図面トレーサ、CAD オペレータなどに分かれるであろう。これらが企業、組織、職能などを形作り、それぞれが協調しながら機能分担することにより、複雑大規模な設計作業を効率的かつ間違いなく行えるようにする。設計の複雑高度化に対応するためには、設計作業の細分化は必然といってもよい。このように分化が進むと、分化した機能間の整合をとるためにオーバヘッドなどの新たな問題が発生する場合もあるが、あらかじめ連携をよくすることで逆に分化された機能を同時並行的に実行することができ、コンカレントエンジニアリングの視点からはより効果的な設計プロセスの構築が可能になる。設計作業の分化が進むと、例えば作図のプロセスなど、比較的定型的な作業を分社化したり、アウトソースまたは外注化する方向に進むことになる。b. マニュアル化設計作業のマニュアル化は新しいテーマではなく、古くから一般的に行われてきた。開発期間の短縮や作業の効率化、保障された設計品質の確保、ノウハウの伝承、設計手法の標準化、ベテランでなくてもある品質の設計が可能になる、などマニュアル化にはさまざまな目的や効果がある。しかしながら近年はCAD 化が進み、かつての紙の上のマニュアル化(文書化)だけではあまり意味がなくなってしまった。1980年代以降、いわゆるワイヤフレームやサーフェスをベースとした三次元CAD が進展したが、高価であるため、CAD を単なる作図の道具として使うだけでは十分に使いこなしているとはいいがたい。モデル作成の工数に見合った付加価値を得るため、特定の部位、部品、システムなどのまとまり単位をアプリケーションプログラムの形でCAD システムの中に作り込む、いわゆる自動設計の方向に向かっていった。特に1―172設計工学CAD 化が進んでいた造船、航空機、自動車産業などでは各社ごとに内製でCAD を開発していたため、各社独自の設計手法をプログラム化しCAD の機能として強化していった。これらの開発は、それまでにマニュアルとして文書化されていた設計手順を、オンライン対話型のCAD アプリケーションに改良、変換することで行ったものである。このようにCAD 化時代になってマニュアルは単に紙の形に作るだけではなく、CAD 化するために必要な前段階の文書化という意味をももつようになった。1990年代後半になって実用化域に入ったソリッドモデルをベースとした三次元CAD は、システムの基本機能としてフィーチャ、パラメトリック、操作履歴、アソシエイティビティ(二次元データと三次元データ間の関係)など、三次元形状創成にとどまらない、さまざまな設計そのものの概念や機能を含んでいる。この段階になると従来のような設計手順ではなく、新しい三次元CAD に適した設計手順を作り出すことで設計業務をさらに効率化できる可能性も生まれてきた。しかし一方で従来の設計者の思考プロセスを変える必要性があり、ベテラン設計者や過去の設計法に習熟した者にとってはかえってなじみにくい面もある。最近では新しいソリッドCAD を有効に活用する目的で、従来のマニュアルに該当するものをHow To DesignやKnowHow CAD といった形で作り直し、CAD のアプリケーションとして組込む動きが出つつある。図4・31,4・32はパラメトリック、キネマティクス、ソリッドモデル、操作履歴などの機能を利用したアプリケーションの例である。図4・31ではトランクリッドの開閉機構の設計時、4リンクヒンジを用いた場合のトランクリッドも、開度とリンクの動的軌跡を求め収納スペースの最適化を求めている。図4・32は、サスペンションで保持されたホイールとホイールオープニングが、旋回したとき、乗員数が違うとき、タイヤチェーンを装着したときなどに空間的にさまざまな相関関係を作り出すが、それらの動きに対し干渉の有無や最適クリアランスを自動的に求めるものである。近年の商品開発のスピード競争が開発期間や設計期間をさらに短縮することを求めており、限られた資源を有効に活用するため、定図4・31トランクリッドの設計型的な設計作業や部品設計、機能設計、システム設計などの対象をできるだけ自動化するために、今後CAD 化を前提にしたマニュアル化とその三次元CAD への組込みの重要性が増すと思われる。一方でいったんシステムに組込むと、次のシステムに移行する場合に移植や作り直しが必要になるなど、従来はなかった新たな問題も内包されることにもなる。今後多くのマニュアルがこのようなシステムに組込まれる形で作り込まれることになった場合、これらのノウハウをどのように移植・伝承していくか、試行錯誤を続けながらシステムと業務の両面から改善を進めていくことが重要である。
