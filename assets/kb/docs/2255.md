# 4・3・4 日産自動車における展開例

> **章**: 第4章 設計の管理
> **節**: 4・29 のようにさらなる期間短縮が可能になる。

日産自動車では早くから三次元CAD の導入による車両の開発期間の短縮に取り組んできた。1990年代の後半からは次世代型のソリッドモデリングCAD を採用し、プロセスのディジタル化による仮想開発を手がけてきた。三次元CAD 化をコンカレントエンジニアリングの観点から見たとき特に重要な意味をもった概念や活動について、自動車業界の取組みの例として日産自動車の場合について以下に説明する。a. データ衝活動日産自動車ではデータ衝という言葉を使っているが、同義として他社ではデータ基準やデータ正といった言葉(漢字)が使われている。ここで用いている衝という漢字の意味として、(1)通路、(2)つきあたり、(3)かなめ・要所・要衝などがある。つまり四方八方から集まるところ、転じて要所、かなめ、とある。要は扇子のかなめのことで、これがはずれると扇子の形をなさなくなる。扇子の1本1本の骨が各部門を意味すると考えれば、各部門をつなぎコンカレントな仕事を成り立たせるかなめはデータである、という意味から用いられているものである。このように、かなめとしてのデータは大変重要であり、その役割を十分果たした場合、極めて大きな効果を発揮することになる。一方その位置付けにあるデータが本来の役割を果たさない場合、たちまちにして各部門(扇子の骨)がばらばらになってしまう危険性をも意味している。その意味で、データ衝活動はコンカレントエンジニアリングのメリットを享受し、さらに発展させるために大変重要な活動である。物を基準にした開発プロセスと、データ衝による開発プロセスを、図4・33,4・34に示す。図4・33と図4・34を比較してわかるように、データ衝のプロセスにするということは、物ができてから行っていたさまざまな検討を、図面作成(部品検討)前にいかにやり遂げられる仕組みを構築するかにあるといっても過言でない。日産自動車におけるデータ衝の活動は、大きくは開発プロセスの革新と情報ディジタル革新の二つからなっている。開発プロセス革新活動のねらいは、(1) 設計初期品質の向上→サイマル活動の強化(2) 生産準備期間の短縮→試作型廃止(3) フィードバックサイクルの削減と短縮→開発試作と同時に製作着手(4) 上屋/プラットホーム分離開発などである。一方の情報ディジタル革新は、(1) 三次元データを衝とした活動→ディジタルモックアップデザインの推進(2) 業務管理/運用情報のディジタル流通(3) 品質、コスト、日程などの開発支援情報のディジタル化などである。長年にわたるデータ衝活動により設計開発期間は図最近ではデータ衝活動をさらに一歩進め、図面レス活動を、部品メーカを含めて展開している。すべての情報が電子情報に変われば、コンカレントエンジニアリング活動は企業をまたがった、あるいは海外開発拠点も含めた場合でも有効性を増し、最近のグローバル化や工場の海外移転、世界分業といった変化にも柔軟に対応できるようになる。b. 構造原図方式この方式は日産自動車が独自の方式とし図4・32ホイールオープニングの設計図4・33物を基準にした開発プロセス図4・34データ衝による開発プロセス図4・35開発期間短縮の歴史て、三次元データを有効に活用できる仕事の形態として考案し改良を重ねてきたデータ作成の手法である。自動車の開発において、CAD を導入する以前は、もととなる新車のデザインがクレイモデルで承認されたあと、いわゆる造形線図と、樹脂で作られたマスタモデルを基準にして設計作業以降が展開されていた。CAD 化の進展により、クレイモデルと二次元の造形線図が、意匠デザインCAD(スタイロ、stylo)の三次元の精密かつ直接金型切削に使え1―173図4・36構造原図作成プロセス図4・37構造原図の例る高精度でなめらかな面データに置換えられた。一方のマスタモデルはすべての外形、内装形状の物理的なマスタと位置付けられ、いわば新車のメートル原器のような役割を果たしていたが、1980年代の前半にマスタモデルが廃止され、図4・36にあるような、それに代わる基準として三次元データで構成された構造原図を用いる方式に変わっていった。構造原図は図4・37のようなデータで構成され、意匠デザインCAD で作られた意匠面データを基準として外板データ、車体構造部品データ、内板データを、あたかも実際の車を作るように組み上げていくものである。この作業は設計エンジニアと、三次元構造原図データを作成する専門ショップがコミュニケーションをとりながら作り上げられる。その間、構造検討、レイアウト検討、強度・剛性、衝突性能などさまざまな設計検討がコンピュータ内のデータを衝にして進められる。この作業が終了すれば、文字どおり車両のマスタモデルがコンピュータ内に作り上げられたことになる。さらにデータ作成に当たっては一元化、成長するデータの考えを遵守し、中間生成物としてのデータも使用目的に応じてむだなく使えるようにプロセスとデータベースが構築されている。c. CAD 100(CAD 100%)活動CAD 100活動は1990年代の前半に行われた活動で、図4・38に示すようにデータ衝コンセプトを実現するため開発の初めから生産まで、三次元データを100%活用する業務プロセスの実現をねらいとしたものである。全社的な組織であるCAD/CAM 委員会を設置し、その傘下の中核となる活動として推進された。これは、(1)車両レイアウト計画を、100%CAD データを用いて行えるようにする(レイアウト計画CAD)、(2)車両メーカと部品メーカとの間でレイアウトが成立する最適な部品の詳細設計を、CAD データを用いて相互に連携しながら進めるための連携活動(CAD-CAD 連携)、(3)意匠デザイン段階で作られる三次元CAD データからスタートし、レイアウト計画、部品設計を経た後、実際のものづくりの段階、すなわち金型を加工できる精密な部品データ、金型加工用NCデータ、治工具データの作1―174設計工学開発から生産までCAD データを基準とした業務プロセスを実現することにより、トータルな効率化を図る。図4・38 CAD 100活動の共通コンセプト成までを一貫して同一データあるいは成長するデータとして活用できる業務の仕組み作りとシステムの構築、の3点を目指した活動(CAD/CAM 一元化活動)である。この活動は、毎年設計変更や不具合件数についての低減目標や、対象部品データの拡大、データ化のレベル向上の目標値などを設定し数年にわたって続けられ、開発期間の大幅な短縮、品質の向上、設計変更の低減を実現し、データを衝とする開発プロセス構築に大きく貢献した。d. DMDR(digital mock up design review)DMDR は試作型を作らず直接量産型を作れるような開発プロセスを実現するためのキーとなる手法である。従来の開発プロセスは図4・34のように、図面を発行してから実際に物を試作し、実験する作業が中心となっていた。このため設計変更が多く、そのたびに実物を作り、実験を行うために期間、コスト、工数ともむだが多くならざるを得なかった。新しいプロセスは、可能な限り出図前に必要な検討を済ませることで設計変更や実物の試作および実験を削減することがねらいである。DMDR は図4・35にある開発プロセスを実現するための中核となる方策で、ディジタルデータでデザインレビューが行えるようCAD 端末や大型のプロジェクタを設置し、設計、生産準備、購買、販売、部品メーカなどから関係者が集まり、一室で議論できるようになっている。そこで大型画面を見ながらディジタルな世界で可能な限り検討を済ませることで、設計変更の格段の低減をねらったものである。この活動で多くの効果が出るようになり、三次元化の有効性が認識されるようになった。e. 図面レス活動ソリッドモデルを用いた三次元CAD の普及、設計者以外の人でも利用しやすいビジュアライゼーションソフトの開発、導入や設計手配など管理業務のディジタル化が進み、データ衝活動のねらいの最も高い目標といえる紙図面をなくす、図面レス活動が進められた。当初、社内で生産部門との連携に同一の三次元CAD を利用しようとしたが、機能が多すぎかえって使いづらい、高価なコンピュータ資源が必要、図面のほうが便利といったさまざまな理由でなかなか普及しなかった。この悩みを解決するため、軽くて設備・工程設計や検査などの設計以外の業務に必要な機能を備えた三次元のビジュアライゼーションソフトを開発することで図面レス化に取り組んだ。しかし自動車は70%が購入部品といわれるように、車両メーカ内のみの図面レス活動では効果が限定される。そのため関連メーカを含めた広範な活動に拡大する必要があった。現在は図面レスを実現するために必要なソフトの機能の充実と併せ、紙の図面がなくても仕事ができる仕組み作りの両面から取組みが行われている。しかし、部品メーカとの間でCAD-CAD 連携活動や図面レス活動を推進する中で、異なるシステム間のデータ交換が完全には行えないという問題が大きくなってきた。このままでは、データを基準とする仕事がねらいどおりの効果を発揮できなくなる可能性もある。今後はデータ交換を容易に実現する相互運用性の確保がますます重要になってくる。f. 活動の成果以上のような活動を通し、三次元CAD を活用したコンカレントエンジニアリングの取組みが推進された。従来は設計から図面が手配された後、生産準備作業が行われたが、三次元CAD データの早出しと生産準備上の要件をあらかじめその三次元データに反映することで、手戻りや後工程の業務を大幅に効率化することが可能となった。その効果はまず開発期間短縮と工数削減の面で大きな形となって現れ、さらにその結果としてコストの削減や品質の向上に大きく寄与することになった。
