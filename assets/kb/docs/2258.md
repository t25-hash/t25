# 4・4・1 製品系列の統合化における背景

> **章**: 第4章 設計の管理
> **節**: 4・4 製品系列の統合化と設計

a. 共通化・共有化・モジュール化設計がより優れた機械や製品の実現を目指していることには疑う余地はない.それがさまざまな制約のもとにあることも自明であるが,それにはさまざまな基盤設備や知識生産なども含まれていることまでには考えは及びにくい.しかしながら,設計の内容がさまざまな意味で高度化してくれば,その種の制約を考えておくことが優れた製品を提供していくうえで重要な視点ともなる.異なる製品の間で一定の部分を共通化(commonalization)したり,異なる企業間でそのような部分を共有化(sharing)したり,あるいは,製品の一定の部分を独立させてモジュール化(modularization)したりする動向は,上記の視点に関わるものである.それらの潜在的で本質的な意義は,複数の製品すなわち製品系列(product family)を統合的に考えることによって旧来からの意味を超えた優れた設計を実現しようとする点にある.本節では,共通化・共有化・モジュール化として象徴される方面における設計に関して,典型的な実践例,概念的な意味,アーキテクチャという視点から展開される理論,統合化(integration)における最適性やそれに向けての数理的な設計の方法論などについて示す.b. 製品系列の統合化についての実践例さまざまな機械の間でそれらの構成要素を共通化しようとする考え方は,なんら特殊なものではなく,各種ねじの標準化などにも見られる.標準化のねらいはおもに設計の効率化とコストの低減にあり,それらは製品系列の統合化がねらいとするところでもある.統合化が標準化を超えて改めて重要である理由は,それが機械や製品の性質そのものを左右する主要な部分を対象としている点にある.なお,本節の以下の部分では,機械の物理的な機能だけではなく価値やコストなどについての内容をも考える必要があることから,設計の対象物を製品と呼ぶことにする.以下の各製品は統合化が効果的に行われている実践例である.腕時計メーカのスウォッチ(Swatch)社は,独特のデザインをもつ極めて多くの種類の製品を次々と妥当な価格で提供している.その背後には,高性能な時計の心臓部が極めて安価に手に入るようになって以後,時計が正確な時間を刻むことは当たり前の機能になって,意匠が製品を差別化するようになったことが大きく影響している.つまり,文字盤や針,腕輪などの形状を統一的なものとすることによって,それらのもとで多様な意匠を組合せ的に展開することが可能になっている.民間輸送機は製造コストもさることながら開発コストが大きく,新規の開発は容易ではない.さりとて,航空需要は年ごとに変化して,新しい機体に対する需要は生まれてくる.これに対応する一つの方策は,既存の設計を流用しつつ新しい設計を生み出すことである.例えば,DC-9は胴体の伸張(ストレッチ)などを何回かにわたって行いMD-80に至っている.また,ボーイング777においては,1990年の開発開始の当初から,初期型に続いて,将来,胴体のストレッチによる座席数の多いタイプや,やがて導入が見込まれる高性能エンジンを積んだタイプなどを展開することが計画されていた.自動車の開発方法においても2000年前後から同様の傾向が見られるようになってきている.それについてはさまざまな要因が交錯しているが,環境問題への対応や衝突安全への対応は開発コストの増大に直結するものである.また,社会の成熟に伴って消費者ニーズは多様化するとともに付加価値に焦点が移ってきていて,開発期間の短縮や迅速な多品種展開を要請している.少なくとも従来の価格を維持しながら前者に対応するには,より大量の台数を生産して1―178設計工学車1台当たりのコストを圧縮すればよいが,それでは後者に対応することはできない.両者への対応を同時に実現する方法が,車の全体ではなく部分を共通化することによって,部分としては超大量生産を実現しつつ,車自体としては多品種展開を行うというものである.具体的には,企業内や企業グループ内でプラットホーム(platform,車台)を共通化し,そのもとでさまざまな車種を順次展開していくことが行われたり,ドアやコンソールなどについては,サプライヤが主体となり開発と生産を行うことによって,個別のモジュールについての開発に関する知識を集約的に投入したりすることが行われつつある.開発コストが膨大となる製品の典型はコンピュータ産業のそれであろうが,その実態は上記の2例とは趣が異なる.その形成期は別として,目下のところ,コンピュータの全体を単独で製造することができる企業は成立し得ない.標準化や共通化を超えて,開発に先だってある種の規格が策定され,そのもとでさまざまな企業が各部分についての開発と製造を行った後に,最終的には組立を担う別の企業が最終製品としての取りまとめを行うようになっている.それに至る理由は製品開発に関わるコストと関与する知識が膨大であることにあり,分業を通じてそれらを分担し合うことを前提として産業が成り立っている.c. 統合化の及ぼす副次的効果b項の内容は製品系列の統合化が著しい効果をもたらしている事例ではある.例えば自動車において,さまざまなオプションが用意されていたり,特定車種に異なる排気量のエンジンを搭載するモデルがあったり,右ハンドルの車と左ハンドルの車が並立していたりすることは,同様のことを小規模に行っている事例とみなすこともできる.つまり,製品系列を統合的に考えることは,製品を全体と部分の関係のもとで考える,すなわち,製品を適切な部分に分けたうえで,それぞれの部分についての設計と相互の関係についての設計をまとめあげることによって全体の設計を成立させるということになっている.b項のような統合化とは離れても,部分の設計を考えることには以下のことも付随していて,それによって製品の柔軟性を増大させることができる.(1) オプション追加:製品を多様化するために,基本的な製品に付加的な機能を任意に追加する.(2) アップグレード・順応:システム全体の性能を向上させたり,使用環境の変化に適合させるために,製品のある部分を入れ換える.(3) サービス性(修理・補充):製品の寿命を確保するために,摩耗するなどした部分を入れ換えたり,消耗したものを充てんしたりする.(4) 使用における柔軟性:異なる条件下で使用するために,一時的に製品のある部分を入れ換える.(5) 再利用:いっそうの経済性を確保するために,寿命を終えた製品のある部分を別の新しい製品に用いる.これらについての設計はそれぞれに重要な事項ではあるが,以下ではb項で示したような規模での統合化を主題とする.
