# 4・4・3 アーキテクチャから展開される理論

> **章**: 第4章 設計の管理
> **節**: 4・4 製品系列の統合化と設計

a. 製品内容の階層性と複数視点間の関係設計対象である製品をいくつかの部分に分解してとらえる考え方は各種の設計方法論の基盤となっている.品質機能展開(QFD)では,製品をとらえる視点をさまざまに想定して製品の構造を各視点のもとで階層的に分解したうえで,各視点間での価値配分の整合性についての分析が行われる.その背後にある関係性は,各種の視点として,例えば,設計の起点となる顧客ニーズについてのもの,それを実現する物理的な原理に関わる機能構造についてのもの,具体的な製品に作り上げるための製造における構造についてのものの三つを考えたうえで特定の製品に着目すれば,図4・44(a)のように模式化することができる.それに対して,製品系列を統合化する場合においては図4・44(b)のようになる.まず,複数の製品を同時に考える場合においては,起点となる顧客ニーズが複数存在することになる.それに伴って機能構造についても,何らかの基盤となるものが想定できるものの,いくつかの派生構造をも想定する必要がある.それを受けた製造における構造では,背後にある個別部分を何らかの意味で派生構造に関連させながら共通的に用いることによって,多様な顧客ニーズに対応する各種の製品を効果的に提供できる可能性があるということになっている.(a) アーキテクチャとその類別図4・44のような関係性において,なかでも重要であるのは,機能における構造と製造における構造との相互関係である.機能は対象を構成する相互の関係によって発現するものであることから,機能における部分は完全に分離できるものではない.しかしながら,顧客ニーズとの関連を踏まえれば,解釈としての部分を他から分離して想定することができる.一方,製造においては,一般に多種多様な部品を作りアセンブリーにまとめたうえでそれらを最終的な製品(実体)に組み上げることから,明確な構造を定義することができる.これら両構造の間の関係はアーキテクチャ(architecture)と呼ばれ,大きく以下の二つに分類することができる.(1) インテグラルアーキテクチャ(integral architecture):両者の間の関係が渾然一体となっていて,ある部分機能を担うある部分実体を特定することができない.(2) モジュラアーキテクチャ(modular architecture):ある部分機能を担うある部分実体が明確になっていて,両者の間の関係がおおよそ1対1になっている.なお,後者におけるモジュール(module)とは,1対1対応のもとでの部分機能を担う明確な部分実体を意味する.また,インタフェース(interface)とは,そのようなモジュールが相互に作用する場合における相互接続のための仕組みを意味する.両者の違いは図4・45のように模式化できる.モジュール化(modularization)とは後者のモジュラアーキテクチャを用いることであり,共通化や共有化とはそのもとである部分実体を置換えることによって異なる顧客ニーズに対応する製品を提供することであるといえる.製品系列の統合化に常の理があるならば,前者のインテグラルアーキテクチャは存在し得ないことになる.しかし,例えば,単一の製品のみを対象として優れた製品を目指せば,あらゆる部分を要求機能に向けてすり合わせていくことが求められる.すな1―179図4・46 クローズドアーキテクチャ(closed architecture):アーキテクチャが特定の企業内で完結している.(2) オープンアーキテクチャ(open architecture):アーキテクチャが特定の企業を超えて業種内などで標準化されている.なお,前者の場合,インテグラルアーキテクチャとするかモジュラアーキテクチャとするかは対象とする製品の性質によるが,後者の場合には,製品の各部分が企業間で行き交った後に最終的な製品となることから,モジュラアーキテクチャとなることが必然となる.c. スロット型(slot-modular):ある範ちゅうのモジュールを据え付ける場所が母体となるモジュール上で限定されており,それぞれにそれぞれの部分機能を担う部分実体が導入されることにより全体が機能する.パーソナルコンピュータにおけるCPU やメモリのマザーボードへの接続方法はこの典型である.(2) バス型(bus-modular):母体となるモジュール上に他のモジュールを据え付ける共通のバスが用意されており,その上にさまざまなモジュールを比較的任意に組合わせて導入することができる.パーソナルコンピュータにおける周辺機器の接続方法はこの典型である.(3) 断面型(sectional-modular):特定の母体となるモジュールは存在せず,モジュール群を特定のインタフェースで相互に連結することによって全体が機能する.配管系はその典型であり,オフィス家具においても用いられる場合が多い.なお,これらの中でスロット型が最も一般的であるが,モジュラアーキテクチャによる柔軟性を最大限に引き出す場合は,事前の調整が大規模なものになるものの,バス型や断面型のほうが有利となる.d. 実践例に見るアーキテクチャの意義以上のアーキテクチャやモジュールの種類はある種の規範である.そのため,個別の製品に着目した場合にいずれのものに類別できるかについてはいささかの解釈も必要となる.例えば,4・4・1・b項のスウォッチの場合,クローズドなモジュラアーキテクチャを導入したことが特徴的であり,それぞれは部品交換型・スロット型となっているといえる.その背後において,時計としての心臓部ははん用的なものであるはずであり,より広い腕時計という範ちゅうから見れば,部品共有型であり,ある種のオープンアーキテクチャとなっていると見ることもできる.自動車の場合は一般にはクローズドなインテグラルアーキテクチャが用いられる.しかしながら,前述のようなプラットホーム(車台)の共通化が進展している状況は,その中にモジュラアーキテクチャ的な要素も組込むことによって製造における総合効率を向上させようとしているということになる.これに対して,ドアやコンソールにおけるモジュール化は,形式的にはオープンなスロット型のモジュールであると見ることができるが,元来の製品全体における統合性を踏まえれば,相互の関係はそれほど単純なものではない.また,セダン系とトラック系を比較した場合,両者の構造上の違いから,後者のほうがモジュラアーキテクチャ的な要素が大きいということにもなっている.民間輸送機も,以上とほぼ同様であると考えられ,実体そのものの内容はクローズドなインテグラルアーキテクチャとなっているが,開発プロセスについては,開発コストを圧縮したりリスクを分散させるために,オープンなモジュラアーキテクチャ的な形態をとるようになっていると解釈することもできる.パーソナルコンピュータに関しては,随所にオープンなモジュラアーキテクチャが用いられていて,モジュールサプライヤから見れば,部品共有型モジュール性であり,カスタマイズを行うユーザから見れば,部品交換型モジュール性であるということになる.また,技術革新などによってインタフェースの形式が変化することもあって,周辺機器の多様化はそれらの接続方式をスロット型からバス型へ転換させたということになる.加えて,デスクトップパソコンとノートパソコンを比較した場合,後者においては小型化・軽量化が至上命題であり,それを達成するためにはある種のインテグラルアーキテクチャ的な発想が求められるということになっている.
