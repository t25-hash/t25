# 4・4・5 製品系列の統合化設計のための数理的手法

> **章**: 第4章 設計の管理
> **節**: 4・4 製品系列の統合化と設計

a. 設計プロセスから見た統合化の意味製品系列の統合化にはさまざまな内容が含まれることから、設計を進めるための手順すなわち設計プロセス(design process)を考えておくことも重要である。問題解決過程としての設計プロセスを考えると、統合化設計における処理内容は、単一の製品を対象としたシステム構造・形態・属性のそれぞれに関する一連の処理内容を基底としつつ、図できる。(1) 設計要求の分析処理:設計目標を確立し、多様な設計要求とそれらの間の相関性を分析する。(2) システム構造のシンセシス処理:システムの構造とその変形を確立する。具体的には、基盤となるシステム構造を確立し、適切なモジュールのサイズや相互関係についてのアーキテクチャを選定して、製品群を展開するうえでの基本概念を確立する。(3) 形態のシンセシス処理:製品形態とその変形を定める。具体的には、基盤となる形態を確立しつつ、そのもとでの幾何学的な属性などの制御によって製品群を展開するための可能性を模索する。(4) 製品群の具体化処理:具体的な製品群を確立する。その処理内容は、属性値を具体化することによって各モジュールの内容を確定するための属性値の具体化と、可能なモジュールの組合せの中から適切なものを定めることによって個々の製品を具体化するためのモジュール組合せの選定とから構成されている。これらのうち、第2段階のシステム構造と第3段階の形態に関する内容は製品群の多様性を拡大するための可能性を探求するものとなっている一方、第4段階の製品群の具体化に関わる内容は製品系列を確定するために具体的な製品を確認して定めるものとなっている。以上の分類は、そのまま実際の設計の進め方に対応するものではないが、それにはどのような内容が含まれているかを考え方として整理するうえでは有益である。また、何らかの支援手法やツールを導入する際には前提とすべきものである。b. 顧客ニーズの把握製品系列の統合化は顧客ニーズの多様化や流動化とも連動している。このため、図4・51における関係性をも含めつつ、顧客ニーズの幅や変化方向などを見定める必要がある。すなわち、与えられた顧客ニーズのもとで設計を行うのみではなく、設計が新しい顧客ニーズを呼び起こすという考え方にも立つことが求められる。一般に、製品のもつ価値は使用(性)価値(use value)と感性価値(kansei value)とに分類できるとされており、製品が社会へと進展していく導入期・成長期・成熟期を通じて、顧客の関心事は当初は使用(性)価値におかれるが、次第に使い勝手や意匠性などの感性価値にも注意が払われるようになって、最終的にはそれらが差別化の手段となるとされている。このような顧客ニーズの変化には何らかの様式が潜んでいたり、また要素技術の動向についても、別途、何らかの予測は可能であることから、統合化においては、設計に先立つ企画においてその種の様式を踏まえた戦略的な考察を行っておくことが、個別の設計を行う場合にも増して求められることになる。c. アーキテクチャの設計アーキテクチャの設計はいわゆる概念設計に関わる内容であり、機能構造そのものを処理することにもなっている。このこともあって、それについての明確な方法を手順として示すことは難しい。4・4・1・b項のスウォッチ社の事例は、腕時計のアーキテクチャをインタフェースの意味で再発見することによって、ある機能構造に新しい意味をもたらしている場合である。コンピュータ産業における状況は、業界標準としてモジュラアーキテクチャが定義されている典型例であるとともに、その技術革新が図4・46に示すような階段状に進展している事例でもある。いずれにしても、アーキテクチャの設計には、ある設計要求を実現するうえでの機能的な構造が確定しているとしたときに、それを構成する細かい部分をどのようなグループにまとめあげて個別モジュールとするかを決定する課題が含まれていて、それが重要な位置を占めている。その際、望ましいアーキテクチャとはどのようなものであるかに関しては、コストや利益を具体的に論じるよりも、それらの前提となるある種の構造に関してどのようなシナリオを描けるか、その中でいずれの戦略を選択するかが重要となる。例えば、ライフサイクルエンジニアリング(life cycle engineer-ing,3・6節参照)の視点を借りれば、製品が環境へ与える影響を低減するに当たって描けるシナリオや選択できる戦略には多種多様なものが存在している。具体的には、製品を製造した後に使用を進めて故障したらすべてを単純に廃棄する場合、廃棄製品を一定のレベルまで分解したうえでマテリアルリサイクルかサーマルリサイクルを行う場合、製品を保全(メンテナンス)しながらできるだけ長く使用した後でリサイクルを行う場合、保全に加えてモジュールごとに別製品での再利用(リユース)を繰返したうえでリサイクルを行う場合などが考えられる。図4・53は、ある冷蔵庫に関して、一定の想定条件のもとに三つのライフサイクルシナリオを想定したうえで、どのようなアーキテクチャ設計、つまり、モジュール化設計が最適であるかを比較したものである。図4・53(a)は冷蔵庫の部品構成をグラフとして示したものである。図4・53(b)は再利用とリサイクルを行う場合に最適であると考えられるモジュラアーキテクチャであり、機能面での関連の強い部品群が比較的大きなモジュールになる傾向が見られる。図4・53(c)は保全とリサイクルを行う場合のモジュラアーキテクチャであり、故障確率が相対的に高い部品が独立したモジュールや小さいモジュールになるなどの傾向が見られる。図4・53(d)は保全と再利用とリサイクルのすべてを行う場合のモジュラアーキテクチャであり、図4・53(b)と図4・53(c)の折衷型と見ることもできる。以上の例は、各モジュラアーキテクチャのもとでの環境負荷のレベルや期待される利益のレベルは相互に異なっていて、どの指標を最適性の基準として考えるかによって最適なモジュール設計が異なることを示している。一方、その背後ではさまざまな社会基盤の整備や消費者意識の変化などの要因も大きい。つまり、単純な最適性に基づいて設計を行うのみではなく、その結果を参考にしつつ諸要因をも念頭において戦略的な決定を行うことも求められているということになる。一方、上記の各設計案の違いには4・4・1・c項に示した統合化による副次的効果を左右する諸要因が潜んでいて、そのこ1―183とはアーキテクチャの設計が環境適合性のみならずさまざまな方面へ影響を及ぼすことを示している。d. 形態の設計製品の基本的な形状をどのように制御するかは、アーキテクチャに加えて、製品の多様化を支配する重要な因子である。例えば、直方体の容積を縦・横・高さのうちのどの変数によって制御するかについては、いくつかの自由度が存在する。図4・54は、このような操作に関する冷蔵庫のモジュール設計の事例を示している。図4・54の左は個々のモジュールの代表寸法がどのように関係しているかを示している(矢印は直接的な関係を示す)。図4・54の右は制御属性に関する二とおりの選択を示している。モジュールの幅を共通化した設計では、積み重ねることになるモジュールのいくつかは、製品群にわたってそれぞれに共通化することも可能である。一方、モジュールの高さを共通化した設計では、側面外壁のみが共通化可能となる。このように、制御属性の選定は製品系列の展開における柔軟性や効率性を左右することになっている。(b) 多目的最適化から見た共通化部分の特定多様な製品が求められるという事実を最適設計の視点から解釈すると、製品系列とは多目的最適化(multi-objective optimization)問題におけるパレート解集合(Pareto set)における異なる最適解の集合であると見ることができる。図4・55はその概略を示したものである。まず、製品系列は互いに競合する二つの設計目標と、例えば軽量化と性能容量の間で両者を異なる度合いで重視した製品群を提供する問題であるとする。共通化を考えずに、のみを考慮した最適化を行えばに相当する設計が可能であり、のみを考慮した最適化を行えばに相当する設計が可能であるとする。このとき、,という状況は実現はできないものの極限的な設計であり、製品における何らかの部分の共通化を前提とした統合化によるあらゆる設計はその端点の左下側の矩形領域内に位置することになる。数理的に見れば、共通化とは前述の式(4・2)から式(4・3)への転換の際に何らかの制約条件が追加されることを意味していて、それによって実行可能領域が矩形領域から後退して縮小し、その境界上にさまざまなパレート最適解(Pareto optima)が存在していることになる。この性質に着目すれば、製品のどの部分を共通化すべきかについての指針も得られる。以上の内容はあらゆる製品系列の統合化設計において適合するものではないが、モジュールの共通化が伴っている重要な性質を指摘している。f. モジュール組合せによる設計何らかのアーキテクチャと形態を想定することができ、加えて各モジュールとして利用可能なものが事前に確定している場合、製品系列の統合化設計問題は製品を構成するために実際に用いるモジュール群を候補群の中から製品系列の競争力を最大化するように取捨選択するという問題になる。個別の製品を機能面から見た場合、それぞれについての要求を充足する最小機能を有するモジュールがそれぞれに存在しているはずである。しかし、一連の製品を統合的に設計する場合、それらのすべてを導入したのでは必要なモジュールの種類が多すぎることになっていて、個別的には過剰な機能をもつモジュールを用いても全体としては必要なモジュールの種類をあるところまで削減したほうがよい場合が想定できる。このようなモジュール数の削減に向けては、モジュールの代替による共通化がどこまで可能かどうかが鍵であり、それについての条件は、図4・56にも示すように、以下の3種類の制約条件として整理できる。(1) 代替可能性制約:モジュールを代替した場合に、その製品が各要求機能を満足できるかどうかに関する制約。(2) 代替同時性制約:機能面の連結関係により、あるモジュールを代替した場合に、他の特定のモジュールも同時に代替しなければならないという制約。(3) 容量制約:エネルギーなどについての需給関係が全モジュール間で適切であるための制約。形式的には、その内容はある種の0-1整数計画問題(0-1 integer programmingproblem)となっていて、数理計画アルゴリズムを用いてその最適な設計を定めることができる。モジュールの共通化による効果を示すために、事例としてテレビ受像回路についての想定問題を考え、最適設計を行った結果を検討してみる。受像回路はテレビの主要な構成要素であることから、外部ケースの色や形状などを除いたテレビに求められる多様な仕様のほとんどと何らかの対応関係をもっている。具体的な状況として、表4・10に示す仕様をもつ,...,の製品をそれぞれ2万4000台ずつ生産する場合を考える。表4・11は最適な共通化を数理的に定めた結果を示したものである。表4・11中、●は共通化を考えない場合の個別製品におけるモジュールの選択を表しており、○は共通化によって製品系列全体で最適なモジュールの選択(代替)を行った場合を示している。は両者の間でそのスロットにおけるモジュール選択に変化がないことを示している。表4・12はその内訳を示したものである。数値そのものは仮想条件のもとでのものであり実状を反映したものではないが、表4・11に示した共通化によって、材料費などは増加するものの、モジュールの種類数に依存したコスト(設計開発費や設備費など)がそれ以上に減少しており、両者のバランスの結果として上記の総コストの低減が達成されていることがわかる。g. 製品系列の一括統合化設計各モジュールの内容も統合化設計における課題である場合には、図4・49における各モジュールの属性とそれらによって定まるモジュールの組合せの両方を同時に設計することが求められる。具体的な内容は、異なる製品間でモジュールの共通化を行うのか、ストレッチによる類似モジュールを導入するのか、あるいは、それぞれに独自の設計を行うのかなどを定めつつ、一連の方策のもとで各モジュールの内容を決定するという形式になっていて、双方の内容が相互に影響し合うことから、設計の内容は複雑で込み入ったものとなる。形式的には、その内容はある種の混合整数計画問題(mixed integer programming problem)となっていて、やや特殊な数理計画アルゴリズムを構成することによってその最適な設計を定めることができる。モジュール属性とモジュール組合せの両者を同時に設計することによる効果を示すために、異なる設計要求に対して5種類の航空機を設計する三つの事例を想定したうえで、最適設計を行った結果を比較してみることにする。各事例において、各航空機の要求仕様は、航続距離が1400,1600,1800,2100,2400,2600,2800kmのいずれかであり、座席数は90,120人のいずれかであるものとし、それらを台形状に配置する。三つの事例相互の違いは航続距離の相違の程度が順に大きくなっている点にある。航空機においてはさまざまなレベルのモジュール性を見出すことができるが、ここでは、図4・59のような胴体・主翼・尾翼・エンジンの四つの部分からなるモジュール構成を考える。また、各航空機の生産台数は200、生産期間は15年とする。図4・60中の各区画は、各ケースでの最適設計における胴体・主翼・尾翼・エンジンの各モジュールスロットごとの航空機間での類似性パターンを示したものである。矢印は根元から先に向かってストレッチを行う類似モジュール設計が行われることを表している。また、太い線で連結された点は、すべてのモジュールが同一のものとなって、航空機そのものが同一になることを示している。これに対して、事例2ではとが同一のものとなるものの、4種類の航空機が必要であることが導かれている。さらに、事例3では5種類の航空機が必要であることが導かれている。以上の傾向は後者のほうが要求仕様の相違が大きいことによるものであるが、具体的な相違の内容は各モジュールごとの類似性パターンが相互に影響し合いながら調和した結果であり、製品系列の統合化においては前提となる条件やさまざまな因子の相互作用をよく考える必要があることを改めて確認することができる。
