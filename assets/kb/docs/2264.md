# 5・1・1 設計学とは

> **章**: 第5章 設計者のために
> **節**: 5・1 設計学

設計工学は、工業製品の設計に関する研究領域である。設計に関して、ある研究は設計方法論(design methodology)と呼ばれ、また、ある研究は設計学(設計論、design theory)と呼ばれる。ここで、設計工学、設計方法論、設計論の関係については、おおよそ工業製品の設計に関する学問の総称が設計工学であり、そのなかで、設計方法論と設計論は、設計工学の一つの分野ととらえることができる。設計方法論とは、設計の手順を示したものである。設計方法論は、現実に行われている設計作業の分析のもとに、数多くのケーススタディを経ていることが多い。その結果、現実性の高いものとなっている場合が多い。例えば、5・1・3項で述べるポール(Pahl)とバイツ(Beitz)の設計方法論は、数多くの設計事例を詳細に分析して得られたものであり、設計方法論の代表例として評価が高い。しかし、設計方法論には、次のような問題もあると思われる。例えば設計経験の全くない者が設計方法論を学んだとして、はたしてそれで設計を理解できたといえるだろうか。この問題の原因はいろいろあると思われるが、その一つは、なぜその手順が必要なのかについての説明がないことにある。言い換えると、手順の意味が示されていないことである。例えば、ポールとバイツの設計方法論においても、このようにするとよいとは書いてあるが、この手順にはどのような意味があり、どうしてこの手順で進めるとよいのかについての説明はない。一方で設計論とは、設計を説明するための学問である。大づかみにいうと、なぜ人間は設計ができるのか、あるいは設計とは何か、という問いに答えるのが設計論である。具体的には、設計方法論の手順の根拠を説明することになる。設計を説明する理論は、設計教育や設計支援を科学的に行う場合に必要となる。設計の教育方法論や支援方法論の構築において、その根拠となる仮説を明確にするか否かによって、その構築方法が科学的といえるか否かが決まる。例えば、人間はなぜ設計ができるのか、ということに関する仮説なしに設計を教えたり、ある設計手順がなぜ必要なのか、ということに関する仮説なしにその設計手順を支援する方法を構築することは、まさに経験的に教育をしたり設計支援をしているといわざるを得ない。もちろん、人間はなぜ設計ができるのか、というもってまわった議論をせずとも設計はできるであろうし、方法論も構築できる。しかしそれでは、議論の積み重ねが難しくなってくるのである。たとえ優れた設計教育の方法や設計支援の方法が提案されても、その根拠を明確にすることができなければ、それらを体系的に発展させることはできないであろう。あるいは、同じような方法論が何度も提案されたり、同じような議論が何度も繰返されるということが生じるであろう。ところで設計論を追究することは、一般的には人間固有の概念生成過程を探ることになり、結果として人間の思考のより本質的な性質に接近する。したがって、設計論の成果は、設計者の能力のゆえんを明らかにする可能性がある。このことは、人類の本質的な能力を理解したいという好奇心を満足させるだけでなく、設計者の備えるべき工学的教養の姿を明らかにすることにもつながる。新しい概念を生成することの難しさと危険性を冷静に理解することは、革新と環境の時代の技術者に求められている倫理感の根底をなすと考えられる。このように考えると設計論は、その有用性を実際の設計に期待するよりは、設計工学を科学的な学問にすることにその意義を見いだすほうが適切であると思われる。なお設計の研究においては、設計学という用語も用いられる。設計学は、設計に関する学問を総称する場合と、なかでも特に設計論を意味する場合とがある。本節では、設計に関する科学的な学問の構築を目指すという意味を込めて用いることにする。以下、設計論の例として、吉川による一般設計学(generaldesign theory)とスー(Suh)の設計論を紹介する。スーの設計論は、上述の定義によるところの設計論と設計方法論の中間に位置する。また、完成度の高い設計方法論として評価の高いポールとバイツの設計方法論を示し、その意味を設計論の立場から説明する。
