# 5・1・2 一般設計学

> **章**: 第5章 設計者のために
> **節**: 5・1 設計学

一般設計学は,吉川によって提唱された下記の特徴を有する設計に関する理論である.(1) 設計プロセスは,要求概念(機能概念)から解概念(属性概念)への写像としてとらえられている.(2) 一般設計学では,両概念をそれぞれ集合としてとらえることから出発している.(3) 公理論的な構成となっている.一般設計学における定義のおもなものを示す.・定義1:実体集合.すべての実体を含む集合.・定義2:属性項目とその値.すべての実体は可算個の属性項目にそれぞれ値を与えることにより完全に記述される.・定義3:機能.実体を定める属性項目から適当に何項か(可算個)選んで発現させたとき,その全体は一つの機能を作る.この場合,発現とは外界からの作用で引き起こされるものであり,したがって機能とはその実体への外部からの働きかけの反作用であるといってもよい.これらの定義のもとに,下記の三つの公理が設定されている.・公理1:認識公理.どんな実体でもその属性を記述することで認識可能である.・公理2:対応公理.実体集合と実体概念集合は1対1対応である.すなわち同型である.・公理3:操作公理.抽象概念は実体概念集合の位相である.ここでいう位相とは,数学の位相空間論における位相であり,次のように定義される.・定義:集合に,次の性質をもつ部分集合の族옿が与えられたとき,を位相空間という.1∈옿(∈Γ)を옿の中からとった集合族とする.このとき욥∈옿2,∈옿ならば∩∈옿3∈옿4φ∈옿公理3が設計を可能にする概念構造に対する一つの鍵となっている.実体概念の分類としての抽象概念に高い操作性を与えるのがこの公理であり,これによって設計において要求を表現したり,それを詳細化したり,また解の発見,表現,修正というような作業が数学的な操作としてモデル化されることになっている.一般設計学では,これらの公理を用いて定理が導出されている.以下にそのいくつかを挙げる.・定理1:理想的知識(ideal knowledge,スーパーマンの知識)は,ハウスドルフ空間の性質を備えている.これは,理想的知識は現実の世界を完全に知っている(1対1対応)ということから導かれる.現実世界では異なるものの混同はあり得ない.したがってこれが概念として混同しないためには,どんな異なる二つの実体も,位相(近傍)として分離可能ということであり,これは数学的にはハウスドルフ空間の条件である.この条件を満たす位相空間は,分類能力の極めて高い性質をもつともいえる.・定理2:要求は適当に選んだ抽象概念(一般には機能概念)の積として表現される.これから要求というものが数学的に明快になる.これは,必ずしも積でなくても,和集合でも許される.a. 一般設計学における設計過程のモデル一般設計学は,理想的知識におけるプロセスとそうでない知識(現実的知識と呼ばれている)におけるプロセスに分けて考えることから始まる.まず,理想的知識においては,仕様の詳細化のプロセスに対応して実体概念の(元の)順序列が存在し,それは一点に収束することが知られている.したがって,その場合には設計プロセスはこの収束のプロセスととらえられる.ところが,現実の設計者の設計プロセスは,理想性のない条件下で行われる.このような現実的設計プロセスを考えるには,二つの方法があるとされている.その第一は,まず理想的プロセスにおける収束政策のモデルを確定し,次にその近似として現実的プロセスのモデルをたてるものである.第二は,理想的プロセスにおける収束政策とは独立に,現実的知識(real knowledge)のもつ特徴を理想的知識の静的構造との対比においてとらえ,その特徴から直接,現実的知識における設計プロセス,すなわち収束政策を導こうとするものである.第二の方法については,さらに具体的に考察が進められている.まず,現実的知識の欠陥については次のようにまとめられている.(1) 実体記憶に関する不完全性:有限個の実体記憶しかないので,実体概念を含まない抽象概念による部分集合が生じる.(2) 実体分類の不完全性:位相の欠陥である.異なる実体概念がうまく分離できないこと,あるいは全く近傍をもたない実体概念,すなわち全く抽象概念とは独立なものが存在する可能性がある.また,間違った抽象概念によって分類されている実体概念もある.(3) 操作上の不完全性:詳細化によって不可能な設計仕様を作ってしまったり,収束しなかったりする.また,仮に縮小列となっても,収束に無限時間がかかることもある.(4) 分析の不完全性:たとえ属性位相における部分集合を指定し得ても,近傍に関する記憶が不完全であるために設計解が正確に記述できない.このように現実的知識に理想的知識からのずれがあるとすると,具体的には,現実的知識の欠陥を補うという意味において設計プロセスが必要になるはずである.前述の各知識の欠陥を補うために下記のプロセスが必要となる.(1) 実体記憶に関する不完全性を補償するためには,既存の実体を収集することや,いままでは存在が想定されていなかった新たな実体について,その存在可能性を予測し評価する必要がある.それを行うプロセスにおいて,過去の設計物を参照するプロセスや,新たな人工物を創案しその製造可能性を評価するプロセスが発生する.(2) 実体分類の不完全性を補償するための方法としては,既知の機能概念や物理法則を適用すること,新たな機能概念を創出すること,さらには物理法則を発見することなどが考えられる.そこには,既存の人工物の仕様やメカニズムを理解するプロセスや,新たな仕様やメカニズムを想定するプロセスが発生する.(3) 操作上の不完全性を補償するためには,実現可能な操作手順を案出する必要がある.そのためには,過去に行われて成功した設計プロセスを参考にしながら実現可能な操作プロセスを考案することなどが考えられる.(4) 分析の不完全性を補償するためには,形状や製造に関する情報を追加する必要がある.したがって,既知の製造知識を収集することや,新たな製造方法を開発することなどが考えられる.以上に述べたプロセスは現実の設計において発生するプロセスであり,また主要なプロセスの多くが含まれている.このことより,設計プロセスのある部分は,仮に理想的知識というものがあるとす1―189図5・1
