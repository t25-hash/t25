# 5・1・3 ポールとバイツによる設計方法論

> **章**: 第5章 設計者のために
> **節**: 5・1 設計学

ポールとバイツによる設計方法論は,ドイツにおいて長年にわたり経験的に検討された成果をもとにまとめられたものである.本方法論では,機械は,入力と出力がシステムの境界を横切って外界と連結しており,入力を出力に変換するシステムとして定義される.入力および出力としては,次の三つの基本概念が取扱われる.(1) エネルギー:機械,熱,電気,化学,光,核など,また,力,電流,熱(2) 物質:気体,液体,固体,粉体など,また,素材,試験材料,工作物など,最終製品,構成部品など(3) 信号:大きさ,表示,制御パルス,データ,情報などいずれのタイプの入出力の変換においても,設計においては量的1―19 0なものと質的なものを考慮に入れる必要がある.本方法論では,設計は,役割の明確化,概念設計,実体設計,詳細設計の順に進むとされている.(1) 役割の明確化:設計解に求められる要件や制約条件を整理し,詳細な仕様書を作成する.(2) 概念設計(conceptual design):大まかな構造やメカニズムを創案する過程である.本方法論では,機能構造を構築し,適切な設計解原理を探索し,これらの原理を組合わせて代替概念を作成する,という手順が示されている.(3) 実体設計(embodiment design):レイアウトと形態を決定する過程である.遅くともこの段階までに,経済的実現可能性をチェックする.(4) 詳細設計(detail design):個々のすべての部品の寸法,材質などを決定し,技術的,経済的実現可能性を確認して図面や生産に必要なドキュメントを作成する.本方法論の意味については,その概念設計過程を,下記のように分析することができる.図5・2に示すように,概念設計は,問題の抽象化,機能分解(function decomposition),設計解原理の探索,その組合せの順に進めるとよい,とされている.本方法論では,図5・3に示すように,入力を出力に変換する機能がその流れに従っていくつかの下位機能(部分機能)に分解される.また本方法論における概念設計過程の手順は,図5・4のように書き直すことができる.図5・4において,上部(要求仕様と設計解候補)は,設計対象物の全体に関するものであり,下部(部分機能と部分構造)は,設計対象物の部分に関するものである.一方で,左側に書いてあるもの(要求仕様と部分機能)は,言葉で表現されるような抽象的なものであり,右側に書いてあるもの(設計解候補と部分構造)は,形の描かれている具体的なものである.このことより,概念設計過程は,全体↔部分と,抽象↔具体の操作の組合せである,と分析される.さらに概念設計プロセスは,図5・5のように書き改めることができる.図5・5において,機械全体の機能(要求仕様)と機械の部分機能の関係は,合成↔分解の関係にある二つの具体的な概念(構造)をそれぞれ抽象化したものとなる.この関係においては,部分を合成しても全体を特定できないし,逆にある全体を部分に分解すること図5・4 概念設計過程の構造図5・2 概念設計過程のモデル図5・3 全体機能の下位機能(部分機能)への分解図5・5 概念設計過程における概念の関係図5・6 機能階層の例もできない.すなわち,合成↔分解の関係は成立しなくなる.このことは,実際の工学設計では,機能分解と呼ばれる全体機能を部分機能に分解するプロセスが重要であり難しいと指摘されていることの説明となる.また,ポールとバイツの設計方法論において,機械の機能をエネルギー,物質,信号の流れととらえ,その流れに従って部分機能に分解するようになっているのは,抽象度の高い機能レベルの全体↔部分において,一般的には成立しない合成↔分解の演算を,あえて可能にするための方法と解釈できる.
