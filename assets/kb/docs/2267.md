# 5・1・4 スーによる公理的設計論(axiomatic design theory)

> **章**: 第5章 設計者のために
> **節**: 5・1 設計学

スーは、いわゆるよい設計と呼ばれる設計の原理として、次の二つの公理を提案している。・公理1:独立公理(independence axiom)必要機能()の独立性を保て。・公理2:情報公理(information axiom)情報量を最小化せよ。スーは、設計を機能空間(必要機能:)から実体空間(設計変数:)への写像として定義している。そして、これらの関係には次の二つの性質があると述べている。(1) とは階層構造をもち、分解できる(図5・6)。(2) 設計は、機能空間と実体空間の間の往復の連続である。本理論においては、独立公理の意味について数学的表現を用いて説明している。まず、とは次のように表現できるとされている。=ここで、は(必要機能)ベクトル、は(設計変数)ベクトルであり、は設計行列である。また、の性質により、設計を三つの種類に分類できるとしている。(1) が対角行列:独立設計(独立公理を満足する)(2) が三角行列:準独立設計(決定の順序を工夫することにより、は独立に決定される)1―19 1(3) その他:干渉設計次に、情報公理については、設計の分野における情報量として、以下の式のように考えている。=log(全範囲/許容範囲)ここで、全範囲とは生産システムの実現可能な範囲であり、許容範囲とは設計者の決める範囲である。情報量を小さくするためには、生産システムが実現可能なように設計変数の範囲を決め、かつ全体としての変数の数を減らせればいいことになる。逆にいうと、これが、情報公理の意味する設計の原理であるといえる。
