# 5・1・4 スーによる公理的設計論(axiomatic design theory)

> **章**: 第5章 設計者のために
> **節**: 5・1 設計学

スーは、いわゆるよい設計と呼ばれる設計の原理として、次の二つの公理を提案している。・公理1:独立公理(independence axiom)必要機能()の独立性を保て。・公理2:情報公理(information axiom)情報量を最小化せよ。そして、これらの関係には次の二つの性質があると述べている。(2) 設計は、機能空間と実体空間の間の往復の連続である。本理論においては、独立公理の意味について数学的表現を用いて説明している。=log(全範囲/許容範囲)ここで、全範囲とは生産システムの実現可能な範囲であり、許容範囲とは設計者の決める範囲である。情報量を小さくするためには、生産システムが実現可能なように設計変数の範囲を決め、かつ全体としての変数の数を減らせればいいことになる。逆にいうと、これが、情報公理の意味する設計の原理であるといえる。
