# 5・2・1 よい設計とは

よい設計(good design)の定義は極めて難しい。しかし重要な点として、これまでの少品種大量生産時代と異なり、現状は顧客要求が急激に多様化し多品種少量生産時代となっていることから、顧客の声を十分に反映した製品設計とする必要がある。少品種大量生産時代においては、設計目標、設計条件が大幅に変化することはなかった。したがって、設計も従来の案を踏襲し、それを改良することで対応できた。少品種大量生産時代には、設計案をいかに効率的に、精度よく、実現できるかが最大の課題であった。すなわち、この時代の設計は戦術的な展開が中心であった。この時代には設計→生産の一方向流れであり、各工程がそれぞれ独自に目標をたてて努力を払うことで、よい品質が確保できた(図リーであった。ツリー型(tree type)では、出力は根となるノード一つしかない(図5・8)。そのため、大きな設計目標の変更はできないが、目標があまり変化しなければ、各自がその責任を果たすことで必要な品質をもつ製品機能を実現できた。しかし、多品種少量生産時代に入ると、顧客要求が多様化してきたために、どのように顧客要求を取り入れるか、またそれを基礎にどのように設計目標を決定するかの戦略的な活動が重要となってきた。さらに、製品のライフサイクルも短縮化してきた。そのため、製品の開発サイクルを短縮化する必要があり、従来のような逐次処理の設計、生産方式では対応できなくなってきた。そこで、コンカレントエンジニアリングが提唱され、下流情報をできるだけ上流で処理し、工程活動を時間的に並行に処理することで製品の実現までの時間の短縮化を図ろうとした(図5・9)。しかし、下流情報を上流で処理し、工程を超えて情報を共有する図5・7
