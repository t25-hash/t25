# 5・3・3 設計知識の高度処理:データマイニング,知識発見

a. データマイニングと知識発見データマイニング(datamining)や知識発見(knowledge discovery)の研究開発が始まって10年以上が経過しているが,それらの技術の設計問題への適用は,現在あまり進展しているとはいえない.ここでは,その理由と問題点を洗い出し,設計知識の高度処理技術としての可能性を論じるために,データマイニングの概念について説明する.データマイニングは,研究が始まった1990年代前半にはKDDと呼ばれていた.ただしこの略語の意味には二つの解釈があり,一つはknowledge discovery and databaseの略だという解釈,いま一つはknowledge discovery and data mining の略だという解釈である.この分野の研究開始当初は,コンピュータハードディスクやネットワークに蓄積されていく膨大なデータに関しても,従来の統計的手法が適用可能と思われたことや,統計が比較的扱わない記号データや文書データなどが多かったため,統計研究者の参加は比較的少なく,機械学習研究者とデータベース研究者が中心であった.そのため,前者の意味合いでKDD が使われていた.ところが1990年代の半ばになると,コンピュータハードディスクの性能向上とコストの劇的な低下により,高速なハードディスクアクセスアルゴリズムや巨大なデータベースからのデータの切出しといったデータベース技術が,かつてほど重視されなくなってきた.もちろん,テラバイトを超えるデータ量の領域ではデータベース技術の重要性は言うまでもない.しかし,多くの実用分野で扱うデータは,通常のパーソナルコンピュータやデータサーバで処理可能な数千件から数千万件程度の量である.上記のような状況下で機械学習研究の適用が拡大し,また,ベイジアンネットなど確率統計分野の研究導入も盛んになり,知識発見に重点が移った.ただし現実の膨大なデータでは,多くの統計手法が仮定するデータ分布の均一性が成立しないことが多い.例えば,10年以上にわたって集められたデータや特性の違う地域で集められたデータは,時代背景や地域性によって均質なデータとはみなせず,データが膨大だとかえって一括して解析を行う妥当性の低下を招くという事態が起こる.したがってKDD では,膨大なデータをある程度均質な分布をもつセグメントに分解し,それぞれのセグメントについて適切なモデリングや解析を行うというアプローチがとられるようになった.すなわち,一様に鉄分を含む砂浜の上で永久磁石を引きずると自然に砂鉄が集まるのとは違い,少しでも掘る場所を違えると全然違ったものが見つかる(あるいは何も見つからない)という,考古学の発掘に似た状況である.こういうことからKDD の代表的研究者の一人であったファイヤッド(Fayyad)などがデータマイニングという言葉を使いだし,KDD はknowledgediscovery and data mining の略だとされるようになってきた.データマイニングと知識発見という言葉は,多くの点で重なった意味に使われるが,知識発見はデータが示す部分的傾向に関する説明性の高い関係を導出することに重点がおかれるのに対して,データマイニングはそれ以外にも,与えられたデータから目的量を的確に予測する最適予測技術を含んでいる.すなわち,説明属性と目的量の関係がたとえ人間には理解困難であっても,高い予測精度を得ることができればよしとする場合も含まれる.このような背景から,膨大なデータを性質の違うデータの固まりにセグメント化(segmentation)して解析を行うデータマイニングという言葉が多用されるようになり,そのもとで最適予測や知識発見の技術が発達してきた.したがって,現状のデータマイニングとは,非均質で膨大なデータを比較的均質なデータのセグメントに分割する機能とセグメントごとの最適予測か知識発見を行う手法の総称である,ということができる.特殊な実験環境下で取得されたデータなどを除くと,実問題として均質かつ膨大なデータが解析対象となることはほとんどなく,セグメント化機能とセグメントごとの解析機能はデータマイニングの本質といってよい.これは条件式によって膨大なデータから個別の該当事例を抽出する検索や,混合分布も含めて均質な分布の組合せをデータに対して仮定する統計的手法とは,一線を画する点である.b. データマイニングの多様性データマイニングの特徴は,その目的,対象データ,手法などについて多様性が大きいことである.データマイニングの解析目的は,これまで説明したように知識発見と最適予測の二つに大別される.知識発見の場合,予測能力を犠牲にしても,データから一定の規範のもとで指定された形式の規則を,知識の候補ないしは仮説として導出することが目的となる.あくまでも導出結果から,人間が知識として重要ないし有用であると思えるものを選び取ることが目的なので,データマイニングでは導出結果に統計的有意性検定を課さないか,粗いフィルタリングにとどめることが多い.これは統計的有意性と人間の解釈可能性が必ずしも一致するとは限らないためのみならず,人間がデータを追加収集して検討・解析したり,他の実験や専門的知見によって検証する余地を残すためである.データマイニング手法が提供するのは,人間が処理しきれない膨大・複雑なデータからの網羅的な知識候補や仮説の導出であり,最終的な知識発見は人間にゆだねられる.そのため,解析対象データや導出規則を人間にわかりやすく表示するヒューマンインタフェースも,知識発見には重要である.これに対して最適予測が目的の場合には,統計的手法による予測と同様に予測結果について,各種精度指標による性能評価や予測の妥当性,安定性検証などが行われる.データマイニングのもう一つの多様性は,対象データである.これは単にデータ数が数千個から数千万個までという広い範囲にまたがることに限らない.与えられるデータの形式が,数値,ノミナルな記号,天体写真のような画像,ホームページのような自然文やタグ・リンク付きハイパーテキスト,遺伝子配列のような系列,半順序を表す木構造,化学構造式のようなグラフなど,この世に蓄積されるありとあらゆる形式や構造をもつデータが対象となる.また,実世界で収集されるデータだけに,その品質もさまざまである.ノイズの多寡や,欠測値がなかったりほとんどが欠測値であったりする.実験や調査などデータ収集過程の制約上,不可避的に強いバイアスがかかっていることもある.また,上記解析目的に照らして遂行可能な品質や内容のデータかどうかは,実際にデータマイニングしてみるまでわからないことが多い.したがって,データの形式,構造,品質,目的に応じて,適切なデータマイニングの手法を選択しなければならないのみならず,型通りの手法を適用可能にするまでデータの前処理に多大な労力を要する.この前処理には,形式変換,データ項目の取捨選択,欠測値の補完(人間が介在する場合もある),他データからの新たなデータ項目生成などさまざまな処理が含まれる.1―203データマイニングのさらなる多様性は,データマイニング手法である.上記のような多様なデータ内容や解析目的に応じて,必要とされるデータマイニング手法そのものも多様となる.現状では,大別しても数十種類を超えるデータマイニング手法が提案され用いられている.最終的には,データマイニングした結果を検討し,目的達成の度合いに応じて対象データを取り換えたり,前処理方法やデータマイニング手法を変えるといった,試行錯誤が要求される.場合によってはデータ収集にまでさかのぼってやり直す場合もある.その際には,現場のデータ収集体制の変更にまで踏み込まなければならないこともしばしばであり,組織論までが関係してくる.このように,データマイニングは統計やデータベース,機械学習といった一学問分野だけではなく,それらが集まり,さらに人間工学や組織論,業務基幹システム設計にまでも関連する,総合工学の一種である.c. データマイニング解析手法の動向ここではデータマイニングの作業過程のうち,データ解析のための主要な手法について簡単に説明する.ただしデータマイニング作業のほとんどは,データの選択,前処理,変換,結果の解釈・評価,さらにはデータ収集系の再構築や組織改革にまで及び,解析手法自体はそのうちのほんの一部を構成するにすぎない.さらにデータマイニング解析手法自体も大別して数十種類を超えるため,ここでは代表的な二つの手法について簡単に述べ,次に設計知識の処理に関係が深いと思われる主要な研究動向を述べる.. 決定木学習手法(decision tree learning)データマイニングにおける実応用分野の知識発見手法として,現在最もよく用いられているものが,機械学習分野で発展した帰納的決定木学習手法である.対象データは表形式で,各行が項目値からなるベクトルである.この中で目的項目をクラス,その他の説明項目を属性という.この手法では,まず対象データのクラス値分布において,属性の値によって場合分けをした際に最も偏りやずれを生じる属性を選ぶ.そして,場合分けされたそれぞれのデータについて,さらに他の属性値によって場合分けをした場合に最も偏りやずれを生じる属性を選ぶことを再帰的に繰返す.場合分けしたデータにおけるクラス分布が十分に一つのクラス値に偏った時点で,それ以上の場合分けを打ち切る.クラス値分布の偏りやずれを評価する指標としては,情報量や情報量利得などが用いられる.このようにして得られる決定木によって,クラス値をほぼ決定する属性値の条件組み,すなわち決定規則が得られる.この方法の特徴は,クラス値に最も偏りをもたせる属性を優先的に用いて,データを自動的かつ再帰的にセグメント化していくことである.そして,そのセグメント化の結果がそのまま決定規則となる.この手法はどのような条件下でクラス値が決まるかが決定規則で与えられるので,結果の解釈が容易で,知識発見の糸口となる仮説を提供してくれる.その代わり知識発見に重点をおくため,対象問題によって予測精度においてはほかによい手法が存在する場合が多い.. バスケット分析(basket analysis)もう一つの代表的な手法はバスケット分析である.上述の手法が,固定長,固定項目のベクトルデータの集合を扱うのに対し,この手法はアイテム(item)と呼ばれる記号の集合であるトランザクション(transac-tion)の集まりを扱う.バスケット分析では,まずトランザクションの中で一定のしきい値(最小支持度)以上の頻度で共起するアイテムの集合である多頻度アイテム集合をすべて導く.例えば,コンビニエンスストアのレジにおける1回の売上記録は,{パン,牛乳,ジャム,チョコレート,雑誌}のような一人の顧客が購入したアイテム(品物)のトランザクション(集合)で表される.多量のトランザクションデータの中で,あるアイテムの組合せが最小支持度以上の頻度で含まれる場合,この組合せを多頻度アイテム集合という.さらに多頻度アイテム集合中で,特定のアイテムの組合せが現れる条件付きで,ほかに一定以上の頻度(最小確信度)で他の特定のアイテムの組合せが現れるとき,それら二つのアイテム集合の関1―204設計工学係を相関(association,アソシエーション)規則という.一つのコンビニエンスストアで売られるアイテム(品物)の種類は数千にもわたり,これらすべての組合せについて一つ一つデータ中の出現頻度を調べることは,組合せ爆発によって計算量的に困難である.そこで詳細は省略するが,より少ない数のアイテムからなる多頻度集合をもとに,逐次的により多くのアイテムからなる多頻度集合を導出する完全探索アルゴリズム(aprioriアルゴリズム)が用いられる.これによって通常のパーソナルコンピュータやワークステーションでも,多くの場合,実用的な計算時間内にすべての相関規則を導出することが可能である.この方法はあらわにはセグメント化の過程を含まないが,最小支持度を超える局所的なアイテム間の関係を網羅的に探索するので,相関の意味で部分的傾向が完全導出される.その代わり,導かれる相関規則が一般に膨大になることが多く,統計的に自明な関係や対象分野の知見に照らして明らかに無意味な関係をフィルタリングするなどの処理を行う.以上の代表的手法以外にも,K-means法やK-nearest neigh-bors法,べイジアンネット,EM アルゴリズムなどによる統計的なクラスタリングや分類,ニューラルネットワークによる分類,決定木と回帰分析を組合わせた回帰木分析,ILP(inductive logicprogramming)という帰納的述語論理学習など,多くの統計や機械学習,論理学習の手法が用いられる.また,従来的な統計検定や主因子分析,判別分析手法なども併用される.ただしこれらの手法の中には,本来データのセグメント化機能をもつものともたないものが存在する.もたないものを用いる場合には,あらかじめ他の方法でセグメント化するなどの前処理が必要になる場合がある.. 構造データマイニング(structure data mining)上記のデータマイニング解析手法それぞれが,さらなる手法拡張に向けた展開を見せているが,さらに近年急速に発展した手法について簡単に述べる.従来のデータマイニングのほとんどは,1行がクラス(目的項目)と複数の属性(説明項目)からなる表形式データか,アイテムの集合からなるトランザクションデータを扱ってきた.しかし,ゲノム配列や時間を追って起こる事象などの系列,化学式やウェブのリンク構造,機構の接続関係などを表すグラフのように,幾何学的構造を有するデータが増えつつある.このようなデータについて,前処理で構造を表形式データへと近似的にしてつぶしてしまうのではなく,直接にデータを特徴付けるパターンを導出しようとする研究が,この数年の間に盛んになった.手法の詳細は省略するが,文字列の系列については,間に複数の不定記号(ワイルドカード)が入ることを許す飛び飛びの文字列のパターン導出や,系列から読み取れる文字の半順序関係の導出などの手法が提案されている.グラフ構造のマイニングについても,たくさんの構造とその構造が有する性質の組がデータとして与えられているとき,性質を決めている部分グラフ構造はどこかを導出する方法や,1枚の大きなグラフに多頻度で現れる部分グラフ構造を導出する方法などがある.いずれも部分グラフ同型問題というNP-完全問題を含む計算上の難しさがあるが,アルゴリズムを工夫することで実用上十分に高速な完全探索を実現したり,最良優先探索で近似的に十分な解を得たりする方法が提案されている.また対象グラフも連結グラフとは限らず,飛び地があるような非連結グラフにまで一般化した研究や,逆に閉路を含まない木構造グラフの部分構造導出の研究も行われ,実用レベルの性能が得られている.構造データマイニングは,元データを前処理で近似的に表形式に押し込む必要がないため,結果の解釈容易性が高く,知識発見に向いていると考えられる.. 科学的法則式発見(scientific equation discovery)経験的に得られるデータに潜み,かつわれわれにとって何らかの意味を有する規則性の中で,いくつかの厳しい条件を満たすものを法則という.法則式発見で重要な点は,対象分野やその観測過程に関する先験的知識に基づいて,対象を支配する原理が観測数量間に課す制約と同型ないしは準同型である関係式を探索することである.これは,従来のシステム同定や統計的手法が観測数量間の漸近式を見出そうとすることと異なる点である.現状において最も一般性をもつ法則発見手法の一つが,各観測数量の尺度に関する知識を用いるものである.大半の観測数量は比例尺度と間隔尺度に分類される.比例尺度は質量や距離などのように,絶対的原点をもち,その比に意味がある数量である.一方,間隔尺度は温度の摂氏や華氏などのように,原点が人為的に決められ,値の差の比にしか意味のない数量である.これら2種類の尺度を含む法則式形式には,一定の数学的許容性が存在する.例えば,測定数量とが両方とも比例尺度であり,仮にその関係が対数関数=log であるとする.にある正数を掛け単位を変更すると,=log +log となり,log 分だけの原点が移動してしまう.これはが絶対的原点をもつ比例尺度であることと矛盾してしまう.したがって,との関数関係は対数であってはならない.詳細は省略するが,二つの比例尺度間に許容される式は=しかない.これは一辺の長さがの体積が=であることや,距離の二つの天体間に働く万有引力が=(ただしは両天体の質量に依存)であることに反映されている.このような制約を一般化すると,比例尺度や間隔尺度で構成される測定データ間に許容される関数系が得られる.それをもとに先見的に関数系を限定して,漸近式ではない法則式の候補をデータから導出することは可能である.観測数量に課されている制約に従う式は,その背後の物理的原理と矛盾しない可能性が高く,予測精度を多少犠牲にしても人間の物理的解釈が得やすい.科学的法則発見は知識発見の典型例である.また,紙数の都合上詳述は避けるが,最適予測問題の解法として,複数の分類学習器を並列に組合わせて単一の分類学習器よりも高い予測精度を得ようとするbagging やboosting などのアンサンブル学習(ensemble learning),実属性空間のデータを関数によって高次元ないしは無限次元のヒルベルト空間に写像し,そのうえで1枚の判別超平面で2値クラス予測を行うサポートベクタマシン(SVM : support vector machine)などの研究も盛んになっている.さらに関連して,高次元空間を張る属性どうしにラティスや木などの幾何学的構造関係を与えて判別距離を計算する,カーネル関数法の研究も進みつつある.次元間に先見的構造関係を導入することで,分類基準の説明能力を高めることができれば,知識発見にもつながる可能性があり,さまざまな制約を取り入れたカーネル関数の開発が期待される.d. 設計知識の高度処理への適用可能性設計分野へのデータマイニングや知識発見の適用は,インタネットのウェブサービス設計や医薬品設計において進められている.ウェブは多数のホームページ(URL:uniform resource locator)が互いをリンクによって指し示すネットワーク構造をしている.ウェブページの閲覧では,このネットワーク上を閲覧者が逐次移動する.ある閲覧者の閲覧履歴は,ウェブサーバに記録された一つのIP(internet protocol)アドレスからのウェブネットワークアクセス履歴で近似できるので,ウェブサーバログからそのサイトを訪れた多数の閲覧者の閲覧履歴グラフを取り出すことができる.このデータに前述した構造データマイニング手法を適用して,ある内容のウェブページをある順序で見る閲覧者は,ほかにどのような内容のページをどのような順序で頻繁に見るか,という知識を発掘することができる.このような知識を閲覧者の興味に応じた広告バナー配置やショートカットリンクの付加などの形で,ウェブサービスの設計に反映させることが可能である.一方,化学分子構造中のいかなる部分構造や物性が,人体にどのような影響を及ぼすかに関する知識が利用可能であれば,それらの構造や物性を適切に組合わせた分子を有する医薬品を設計することが可能になる.そのために,さまざまな生理活性を有する化学分子構造やその物性に関するデータを集め,それに構造データマイニングや科学的法則式発見の技術を適用して,必要な設計知識を得ることが考えられる.現在,このような考え方に沿って,さまざまな試みがなされている.現状では,機械工学分野の設計へのデータマイニングや知識発見の適用事例はあまり見られない.この一つの理由は,機械工学分野においては,設計者が膨大で複雑な生データをもとにして作業を進めなければならない場面が,比較的少ないためと考えられる.多くの場合には,設計に必要な機構に関する知識は設計者が有しており,また材料や部品に関してはデータベースとして整理されていることが多い.このような状況では,非均質で膨大なデータを比較的均質なデータのセグメントに分割する機能とセグメントごとの最適予測か知識発見を行う手法であるデータマイニングは,あまり適用される機会がないと思われる.機械工学分野の設計にデータマイニングが広く使われるようになるとすれば,それは膨大で詳細な設計事例データベース(design example data base)が整備されるような状況であると思われる.現状においても,特許データベース等での設計事例検索は利用されているが,機構や部品,材料などの細かい粒度の生情報が事例間で比較可能な詳細さで提供されているわけではない.このような詳細な設計事例データベースが提供されれば,設計目的や部分的機能が共通な設計事例群をそれらの内容によってセグメント化し,各セグメントがもつ共通な特徴を設計知識として抽出する目的でデータマイニングを用いることが可能になると期待される.機械工学分野での設計知識高度化へのデータマイニングや知識発見の適用方法に関する研究が待たれるところである.文献(1)
