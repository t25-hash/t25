# 2・2・1 二次元情報と三次元情報

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・2 三次元形状処理

2･1節で述べられているように，従来の設計では図面が情報伝達
の手段であり，すべての作業の基準であった．しかし，設計，生産
作業に計算機を利用したシステムが導入されるにつれて，図面基準
から数値基準へと作業が移行しており，情報も二次元から三次元へ
と変化してきている．三次元の形状情報が計算機で扱えることによ
り，単に三次元的な立体の絵（陰影図）が出力できるだけでなく，
製品の機能や特性の解析，試作品の製作，量産化のための工具や金
型の加工，製品の組立などにおいて，コンピュータシステムにより
援助を行うための入力データとして必須のものとなっている．現
在，設計から生産，さらには保守，回収と，製品のライフサイクル
を通して，統合したシステム化が進められている．
以上のような，計算機支援システムを実用化するうえで，その核
となるデータが三次元形状であり，また，これを実際の製品と同じ
ように計算機の中で扱うためには，単に数値的な三次元データがあ
ることでは不十分である．立体としての各種演算や，他システムの
要請に応じて必要なデータを出力するには，これらを構造化したモ
デルとして表現されていなければならない．このようなモデルを形
状モデルと呼び，1960年代初めのコンピュータグラフィックスの
研究と並行して開発が進められてきた．当初は，図面に書かれた稜
線を三次元的に表現するワイヤフレームモデルが中心であり，その
後，金型製作のための曲面モデルを経て，現在は三次元立体を表す
ソリッドモデルが実用化されている．さらに最近は，図面に書かれ
ていたすべての情報を含んで表現する製品モデルへと発展してい
る．
