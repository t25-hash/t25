# 2・3・2 (統合）三次元エンジニアリングシステムの構成

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・3 設計情報としての形状

三次元システムは，形状を中心とした設計支援機能（CAD:
computer aided design），作成した形状を解析・評価する機能
（CAE:computer aided engineering），型製作のための数値制御
（NC:numerical control ）機能（CAM:computer aided manu-
facturing）などの主要な応用機能のほか，グラフィック制御機能，
データベース機能，マンマシンインタフェース制御，通信制御，入
出力装置制御等，広範囲にわたるソフトウェアにより構成される．
また，そのソフトウェアが稼働するコンピュータシステム，表示装
置，各種入出力装置，設計データ管理のためのサーバや通信系イン
フラストラクチャなど，適切なハードウェアの選択も必要である．
ハードウェアの目覚しい性能向上と低価格化により，普及へ向けた
障害が取り除かれつつある．
a.ハードウェアの構成（図2･104) 三次元システムは1970
年代に第一期の実用期を迎えたが，高価なメインフレームを必要と
したことや，一部大企業による研究開発成果の実験的なシステムで
あったこともあり，一般的な普及にはほど遠い状況であった．プラ
ットホームがメインフレームからEWS（engineering work sta-
tion），PCへとダウンサイジングし，CPU性能が飛躍的に向上し
たことにより，いまでは標準的なPCでも三次元システムの稼働が
可能となってきた．いまや一人1台のPCの上で，他のオフィス系
のソフトウェアと違和感なく共存して利用できることが，三次元シ
ステムの適用範囲を広げ，それがソフトウェア価格を押し下げる力
となっており，導入のためのコスト的な条件は飛躍的に改善されて
きている．

.フレームワーク機能群
(１)幾何演算ライブラリ
三次元システムが扱う幾何形状としては，点，直線，円・円弧，
平面などの解析形状のほか，自由曲線・曲面があり，自由曲線・曲
面については数学的な表現手法が長く研究されてきた．クーンズ曲
面（Coons surface），ベジエ曲面（Bezier surface），B-スプライ
ン曲面（B- spline surface ），NURBS（non uniform rational B-
spline），グレゴリー曲面（Gregory surface）などが代表的であ
り，それぞれ面の接続性や形状の表現力などに特徴をもつ．幾何演
算ライブラリは，曲線・曲面等の幾何形状の生成と，分割，トリ
ム，延長，投影，交点・交線計算等の演算の機能を提供する．幾何
演算ライブラリごとに，表現形式や精度，仮定・保証する連続性な
どに差異があり，これがⅱ項(５)で述べるデータ交換の大きな障害
になっている．
(２)位相処理ライブラリ
立体表現法として，立体と基本立体（プリミティブ）間に，和・
差・積などのブーリアン演算（集合演算）を施して，複雑な立体を
生成していく過程をデータ表現したCSG（constructive solid
geometry）方式と，過程は表現せず，演算結果の立体の表面を面
境界線の連なりと面境界線を介した面の連なりで表現するB-Rep
（boundary representation）が考案されてきたが，現在のソリッド
モデラではB- Repが主流である．ソリッドモデラの内部では，モ
デリングの過程で位相構造の構築・変更・検索を頻繁に行うので処
理効率が重要である．ウイングエッジ（winged edge）構造，ハー
フエッジ（half edge）構造など，位相処理の処理効率を上げるた
めのデータモデルとアルゴリズムが数多く提案されてきた．
(３)パラメトリックエンジン
2･3･3項で述べる拘束処理を行うソフトウェアには，スケッチ
（sketch）として定義された二次元の幾何拘束問題を解くソルバ，
立体間の配置拘束を解くソルバ，パラメータ間の関係式を評価する
ソルバ，そして操作履歴の再実行機構などのソフトウェアがある．
再実行機構は，応用機能群に属する機能を順次起動していくスケジ
ューラの役割を果たす制御プログラムである．いずれもある程度の
実用レベルに到達しつつはあるものの，制約条件の可読性の低さの
問題と，それに起因する拘束ソルバの処理エラーに対する原因分析
の難しさは共通の課題であり，設計業務上の十分な効果を得るため
には，相応の経験の蓄積が不可欠である．

(２)製図
図面は，設計意図を記録し伝達するために不可欠な手段である．
三次元CADを導入しても，多くの場合，図面は健在である．三次
元CADの導入や，二次元CADから三次元CADへの移行がスム
面作成を行ったうえで三次元モデルを作成するのでは，手間が増え
るというものである．三次元CADでは，二次元CADのように一
から図面を作成するのではなく，三次元形状から自動的に図面を作
成する機能が提供されている．ここで得られた図面に対して，線や
寸法，注記などの追加や編集も行え，元の三次元形状に対する変形
操作やパラメトリック変形が，図面側にも反映される．三次元形状
の中に直接注記を記入することで，設計要件や設計意図を三次元側
に埋め込み，図面はあくまでも補助的な情報と考え，図面自体の簡
易化を試みるケースも見られる．三次元CADのさらなる普及へ向
けての障害は，もはや導入コストではなく，三次元化のメリットを
帳消しにする三次元モデルの作成コストを低減することにある．こ
れはCADベンダの課題であるが，利用部門においても三次元モデ
ルの流通を前提とした図面の位置付けや詳細度，規約等の見直しも
必要であろう．

対して，加工対象が自由曲面の場合には，三次元のCADデータに
対する三次元加工が必要となる．昨今の傾向の一つとして，三次元
形状さえ与えれば，それ以外の仔細な条件を与えなくても，工具の
選択などの加工方法を自動選択したうえで，切削経路を算出できる
システムも多くなってきており，未経験者でもある程度の加工がで
きるようになってきたが，やはり最適な加工のためには，ある程度
の経験・見識が必要とされる．三次元形状を利用する利点として，
干渉検査や切削量の認識，加工シミュレーション（machining
simulation）などが挙げられる．切削経路に沿って工具が移動した
ときに，工具自身やホルダ，アタッチメントなどが，切削箇所以外
の部分とぶつかることによる削り込みや，工具・加工機の破損を回
避するためには，経路計算ソルバ自身が三次元形状との干渉検査を
行うことが必要である．素材形状と目的とする製品形状をソリッド
データとして定義すれば，素材形状から工具形状ないしは工具軌跡
の包絡形状をブーリアン演算（差）することで，工具の通過後の素
材（削り残し）形状を認識することができる．素材の削り残し部や
切削量の認識は，工具選択，送り速度や工程の自動決定のために利
用できる．加工シミュレーションは，工具の移動と素材の切削状況
を画面上でアニメーション表示する．実際のところ，切削経路を表
示しただけでは，経路の良否をチェックすることは難しい．かとい
って，試切削を行うのではコストも時間もかかる．画面上で仮想的
な加工を行うことは，品質の事前検証を短期間かつ低コストで行う
ために極めて有効である．前工程で加工特徴と加工属性を定義して
おけば，加工条件指示の手間が低減される．加工技術データベース
を基に，形状特徴から加工特徴（machining feature）へ自動変換
するような形状認識の試みもある．このような設計と加工の連携も
重要なテーマである．
(４)CAE
三次元形状作成の最大の効果として，試作・実験を計算機上で仮
想的に行えることが挙げられる．通常の製品開発においては，設計
→評価→再設計のサイクルが繰返し実施されるが，実際に試作品を
作成してテストするのではなく，計算機内部で計算することによ
り，コスト削減，開発期間短縮が期待される．また試作による実験
では不可能な条件下でのテストなど，多数のテストケースの設定，
設計の初期段階からの解析の実施による品質の早期折込みなども可
能となる．
機能・性能を検討するための解析として構造解析（強度，剛性，
ひずみなど），振動・音解析，熱伝導解析，電場・磁場解析，流れ
解析，衝突・衝撃解析，機構解析などが，成形性，加工性検討のた
めの解析として樹脂流動解析（mold flow analysis ），鋳造湯流れ
解析（metal flow analysis ），板成形シミュレーション（panel for-
ming simulation），鍛造シミュレーション（forging simulation）
などがある．解析法の選択，解析条件の設定，解析結果に対する判
断を正しく行うためには，ある程度の解析理論の理解が不可欠であ
り，そのため，設計現場に解析の専門家・専門部門をおくことも多
い．一方で，詳細な解析である必要はないが，設計の初期段階から
設計者自身の手で簡易的な解析を行うことで，設計の手戻りを抑制
したいというニーズも強い．後者の解析を支援する試みとしては，
機構の動作シミュレーションや動的な干渉検査機能の提供が挙げら
れる．例えば，プレス機械が上死点から下死点まで移動する間の各
部品の挙動をディスプレイ上で確認し，部品間の干渉の発生やすき
まをリアルタイムに確認することは，試作に代わる仮想実験ともい
える．さらに，特定断面で切断した状態で動作シミュレーションを
行えば，試作をもってしても行えないプレス装置内部の検査を行う
こともできる．パラメトリック変形との連携による自動最適化の試
みも，将来の知識・ノウハウの活用と自動設計へつながる可能性を
秘めた取組みである．
(５)データ交換
CADシステムはそれぞれ固有のデータ形式をもち，あるCAD

ることはできない．異なるCADを利用している部門間，あるいは
企業間でデータ授受が必要となることは多く，そのため異なる
CADシステムのデータを相互に変換する仕組みが必要となる．こ
のように，CADデータを翻訳する処理をデータ交換，そのソフト

意する必要があるが，お互いのCADの特性に合わせた専用の変換

SET( Standard d’Exchange et de Transfert ），アメリカ標準で世
界的に普及しているIGES (Initial Graphics Exchange
Specification），これらを統一する意図で開発された国際規格であ

1970年代後半に制定され，ソリッドモデルの交換仕様も含むが，
