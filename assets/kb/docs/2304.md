設計活動にかかわらず，人間が協調作業を行うとき，何を，い
つ，どこで，どんな手段で行うかの意思のコミュニケーションが必
要となる．このようなコミュニケーションにおいては，意図を表現
する手段が重要となる．表現とは英語のrepresentationに対応し，
対象（object ）をほかの何か（記号等）で表すことを意味してい
る．設計活動に限らず，人間は対象を表現するために，文字，言
語，数式，図面，模型，計算機言語等を発明し，表現方法の体系化
とともに，知識・情報の生成と蓄積・利用を通して高度な文明を築
いてきた．生産活動においても，対象（商品，product ）の仕様，
機能，形状，性能，製作法，利用法，保守等を表現するため，図
面，文書（document ）などを用いてきた．紙を媒体とした表現
は，表現の自由度が高い，持ち運びが便利（ポータビリティ），ど
こでも利用できる，保管・管理が比較的容易である，長期保存に比
較的耐えられるなどの特徴のため，技術情報がディジタル化される
までは情報媒体の中心であった．この文書という形態による知識・
情報の伝達と蓄積のプロセスについて考察してみよう．まず文書
は，知識・情報を正確に表現しているものでなければならない．そ
のうえで知識・情報の伝達および蓄積に利用されてきたと考えるべ
きである．本節では，知識，情報，データの意味は以下の定義に従
って用いることとする．
a.知識（knowledge)・情報（information)・データ（data）
の定義 ISO/TC97（データベース委員会）において知識，情
報，データを以下のように形態学的に定義している（図2･145）．
(１)知識：五感，経験，直感等によって人間の頭脳に蓄えられた
すべて
(２)情報：人間がもつ知識の中で記号（データ）によって表現で
きる部分
(３)データ：情報を表現するもの（記号），または情報を運ぶも
の
この定義に従えば，機械は知識を取扱えないことになり，扱えるの
は情報とデータになる．
b.対象とモデル（model ) 対象を表現するとき，細部にわた
り完全に表現することは，多くの場合不可能に近い．人間がある対
象を表現（認識）するとき，ある観点（view）からその対象の特
徴（attribute）を抽出し，その特徴で対象を表現（認識）する場
合が多い．このように，ある観点からその対象の特徴を抽出し，そ
の特徴で対象を表現（認識）したものをモデルと呼ぶ．すなわちモ
デルとは，ある対象の，ある側面からの見方であり，かつ対象の近
似表現であると考えることができる．
c .モデルの表現 モデルを表現するとき，表現のために用い
る記号として，文字，言語，数式，模型，図式，計算機言語等があ
る．用いる記号の種類により，われわれは数式モデル，言語モデ
ル，物理モデル，図式モデル，計算機モデルと区別する場合があ
る．文書とは，対象の物理媒体（紙）上に人間の視覚認識を基盤と
する言語，数式，図式による表現であって，言語・数式・図式モデ
ルと呼ぶこともできる．
d.図面による情報の伝達と蓄積 製品設計に用いられる表現
の一つである製図は，設計者が頭に浮かんだ概念を図として具体化
し，その図を通して，概念の確認と問題点の発見，改良すべき点を
見つけ出し，自己の思考を高めていく作用をもつ．また設計者が製
図能力をもつことは，意思伝達の手段をもつことになる．このよう
に製図の目的は，意図・情報の表現・伝達・蓄積・再利用にある．
図面とは平面上の表現により，対象物の形状，機能，精度，製作法
などの意図をあいまいさなく表現し，設計者・製作者，発注者・受
注者間で必要な情報を交換するための情報モデルであり，製図通則
は，その情報モデル生成と共通理解を可能とする環境ともいえる．
図面を作成する目的は，図面製作者の意図を，図面使用者に確実か
つ容易に伝達することにある．さらに，その図面に示す情報の保
存・検索・利用が確実に行えることを期待している．
製図の利用法に関しては各国で，その理解が異なる．例えばドイ
ツでは，図面は設計者の意図を表現したものとしているので，工作
法を記入することはないといわれている．このように図面が，他者
に対して確実に情報を伝達するためには，図面の解釈が客観的に一
義でなければならない．また容易に伝達するためには，理解しやす
い表現となっていなければならない．さらに，図面に示す情報の保
存・検索・利用が確実に行えることが期待されている．
製図は企業における設計技術を凝縮した技術資料である．したが
って，その図面による製品の保守・修理，補給部品の製作などのた
めだけに保存するのではなく，将来の新設計の情報として検索・利
用できる必要がある．図2･146 は製図の目的（drawing aim）
と，それに関連する用件の関係を示したものである．また図2･
147 は，図面を作成するために必要な製図規格の性格（drawing
standards property）を示している．
