# 2・5・4 PDMによる設計データの一元管理

> **章**: 第2章 設計情報の表現と伝達
> **節**: 2・5 設計における情報管理

製品データの表現と交換を体系的に整理した規格（ISO10303
なされた．STEPはPDMにおける中立的な製品データ参照モデル
（product data reference model ）を規定しているので，その概要
について述べる．
a.STEP PDMスキーマ STEPは，複数の分野別規格
（AP:application protocol ）をもち，代表的な分野別規格として
AP203（三次元製品モデル交換規格），AP214（自動車工業会用規
格）等がある．これらのAPにおけるモデル管理データを整理統合
してSTEP PDMスキーマが提案されている．以下にその機能の
概要について述べる．
b.PDMスキーマの基本機能 PDMスキーマは基本的に以
下の機能をもっている．
(１)製品・部品に対する，識別と分類，特性，構成と関連
(２)ドキュメントに対する，識別と分類，特性，構成と関連
(３)製品とドキュメントとの関連付け，識別子の名称変更，承
認，有効情報，作業管理
(４)PDMスキーマの機能単位（unit of functionality）
STEP PDMスキーマの具体的な23の機能モジュールを，表2･
16に示す．STEP PDMスキーマの特徴を以下に示す．
(１)STEP手法を用いてデータを表現している．強固で国際的
なデータ表現である．
(２)国際規格（ISO10303「STEP」）をベースにしているため，
各種APとの親和性が高い．
(３)製品構成データの中立的表現であるため，PDM間のデータ
交換に利用できる．
設計データの一元管理とは，多くは企業内で設計開発に関わる設
計者が，いつでもどこでも製品と製品データの正しい組合せを検
索・利用・保管できる管理方法であって，基本的には設計データの
構成管理技術に依存するといえる．
