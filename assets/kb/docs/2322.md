a.調査分析手法 物流システムの計画に長年取組んでいる人
は従来は頭の中で経験というツールによって現状のデータから物流
システムを分析し，評価してきた．しかしシステムが複雑化する中
で，これらの人の行動を論理的にかつ多方面から支援するための分
析ツールが提唱されている．またこれらは経験の浅い技術者にとっ
ても有効で計画手法の伝承には不可欠となる．
工場内に何（形状，重量，質ほか）がどこからどこへどれだけ流
れるかを分析しないと工場内の物流計画は立てられない．このため
に物量を算定したり，分析するためのツールの代表例を表2･9に，
図2･31にその事例を示す．
b.シミュレーション技術 シミュレーションを行う場合，は
ん用プログラミング言語やシミュレーション言語を活用した専門的
知識が必要な時代から，現在では搬送車や生産機械，倉庫のシンボ
ルを張り付けてその間をつなぎ，必要な与条件を入力するだけで活
用が可能なものも出てきている．こうした中でシミュレーション技
術が求められる理由として，
･物流システムの構築が複雑になっている
･他のシステムとの連携が複雑になっている
･計画時間の短縮を求められる
･生産ラインの寿命が短くなり頻繁に計画が必要
･トップへのシステム投資物のビジュアルな説明
などのニーズが出てきており，このためシステムの構築前に設計の
検証を行い，手戻りを少なくするためにシミュレーションによる事
前評価が設計行為の中で重要な作業となっている．図2･32に加工
と組立の間の搬送をシミュレートした例を，また図2･33に半導体
製造ラインの例を，また図2･34には計画初期段階でトップへの提

c .モデル化ツール システムをモデル化し，現状把握と設計
システムの形を表現するためにもさまざまなモデル化手法が提唱さ
れている．その中で比較的物流計画に有効で標準手法として活用さ
れているIDEF（ICAM definition）について述べる．
IDEF0は複雑な対象を，要素図形とその接続関係で表現し，関
係者が共通に理解するための概念と手法，規約の集合体で複雑なシ
ステムのモデル化を行うための手法である．だが，その表現方法は
単純で，アローと呼ばれる矢印とボックスと呼ばれる四角の箱だけ
で表現される．アローはインプット，コントロール，アウトプッ
ト，メカニズムの四つからなり，それぞれボックスの接続位置が定
められており，特にコントロールはインプットが出力されるための
条件を与える重要な役割をもつ．ボックスは作業や活動などを表し
（図2･35），複数のボックスは4種のアローによって接続される．
また，これらボックスは階層化構造を取るところが大きな特徴であ
り，一般的なモデルでも4～5階層のモデルとなることが多い．そ
のため，これらの関係の把握と修正にはコンピュータの機能の活用
が不可欠となる．
IDEFの記述は単純で記述方法もシンプルだが，モデル化の概
念，モデルを記述するうえでの文法，用語からレビューの方法など
は全プロセスの広い範囲にわたって規定されている．
IDEF手法を用いることにより，さまざまな諸活動を普遍的な形
で表現でき，システムの設計，構築，運用を考える際の共通の理解
が得られ，システムの改善も効率的に行われる．IDEFは当初，構
造化分析手法としてアメリカ空軍がICAM（integrated computer
aided manufacturing）プロジェクトで開発したものだが，現在は
標準記述言語として使用されるようになっている．
IDEFは数種のファミリーをもっており，0はその中でも最も活
用されている手法となっている．IDEF0は，業務のプロセスや生
産のプロセス等を正確にまた視覚的にわかりやすく記述でき，複数
の間での共通の理解をもたらす．しかし，IDEFはシステムをモデ
ル化しビジュアルに表現するためのツールであって対象を限定する
ものではない．図2･36は物流システムを構築する課程で設計，シ
