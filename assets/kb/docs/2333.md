従来からのシミュレーションの応用に加えて，例えば，以下のよう

(１) サプライチェーンにおいて，要求された納期に間に合う製

(２) 新生産ラインを導入する場合，工場内の他の設備との能力
的バランスや操業面から見て運用上の問題はないか．
(３) あるエリアで作成したスケジュール，あるいは故障等によ
り生じたリスケジュールは，前後の加工エリアにどのような影響を
及ぼすか．
(４) 新製品に対する生産ライン群を複数の企業に発注した．開
発された個別システムは，全体システムとして整合が取れている
か．
図2･ 55 分散型工場シミュレーション
(１)～(３)のような状況に対応するためには，大規模な工場全体
のシミュレーションをある程度詳細に行うことが必要となる．ま
た，(４)の場合には個別に構築されたシミュレーションモデルを統
合し，全体システムのシミュレーションを行う必要が生じる．これ
らの要請に応えるものとして，分散シミュレーション（distribut-
ed simulation）技術の適用が考えられている．
分散シミュレーションにおいては，対象システムをいくつかのサ
ブシステムに分割し，各サブシステムのシミュレーションモデル
（以下シミュレータと呼ぶ）をネットワーク結合した計算機群に割
当て，全体として統合されたシステムとして作動するようにしたも
のである．生産システムに対しては，図2･55に示すように対象と
する工場あるいは工場群を加工エリア，組立エリア，素材・部品・
完成品倉庫群，エリア間搬送システムなどのサブシステムに分割
し，それぞれのシミュレータを個別計算機上に構築し，それらを統
合することにより全体システムのシミュレーションを実行する．必
要に応じて生産管理システムを組込むことも考えられる．
このような分散シミュレーションの実現のためには，サブシステ
ムのシミュレータ間の通信機能の確立，シミュレーションクロック
の同期方法の選択等の問題がある．また，このような環境に適合し
たシミュレーション言語が必要である．
分散シミュレーションをサポートするネットワークシステムとし
てTCP/ IP（transmission control protocol /internet protocol ）に
基づく通信システムを開発するのが一般的である．しかし，もとも
とは軍事用に開発された分散シミュレーション統合アーキテクチャ
HLA（high level architecture）が2000年にIEEEにより標準仕
様として採用され，今後これを基盤とする分散シミュレーションの
開発が期待される ．
個々のシミュレータはそれぞれの有する時計（ローカルクロッ
ク，LC）に基づいてシミュレーションを実行する．このため，現
実の同時刻における各シミュレータのLCは異なる時刻を指してい
るのが普通である．システム全体として時刻に関する整合性をとる
ために全体を統合する時計（グローバルクロック，GC）を定義し，
LCとGCの同期をとる仕組み，すなわち同期法を考えなければな
らない．一般の分散シミュレーションに用いられる同期法としては
基本的な方法としてタイムホイール法，ヌルメッセージ法，タイム
ワープ法等があり，多くの変形が報告されている ．しかし，こ
れらの同期法を適用するためにはそれぞれの同期法をサポートする
機能を組込んだシミュレーション言語（simulation language）を
開発する必要がある．生産システムシミュレーションの場合，多く
のシミュレーション言語が市販されている．それらを利用するに
は，市販のシミュレーション言語に組込み可能であり，生産システ
ムの特徴を考慮に入れた同期法が必要となるため，下記のような同
期法が提案されている．
一つは大規模システムのサブシステム間を搬送システムで結合す
るという考えに立つものであり，タイムバケット法と呼ばれる方法
である ．もう一つは，サブシステム間に存在する倉庫やバッ
ファを共有し，その在庫状態に基づいて同期をとるストレージ法と
いうものが提案されている ．
