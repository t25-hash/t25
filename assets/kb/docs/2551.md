# 1・2・5 工程設計用データベース

> **章**: 第1章 加工学一般
> **節**: 1・2 における第2段階においては,生産ラインを構成する加工設備

加工工程の設計を体系的に行うには,各種の必要な技術情報を標準化し,データベース(data base)に格納しておくことが重要になる.特に,三次元CAD/CAE/CAM に代表される設計および解析システムとの統合化を図るには,工程設計に必要な各種の情報を整理,標準化し,データベースを作成することが必須である.図す.a. 生産資源情報生産資源情報は,生産システムに存在する生産設備,ジグ,工具および人的資源の生産機能,生産能力,単位時間当たりのコストなど,生産資源に関する情報を表すものである.b. 加工特徴情報加工作業を行ううえで単位となる製品の部図1・4 加工に必要な時間図1・5 加工に必要なコスト図1・6 工程設計に必要なデータとその関係位(加工特徴)を整理,分類したカタログ情報である.すなわち,過去の生産対象製品の加工特徴,今後生産すると考えることができる加工特徴の形状,寸法,公差,表面性状,材質などを整理,標準化したものである.これらの情報は,工程設計で有用であるだけでなく,製品設計における参照情報として利用することで,加工工程を考慮した製品設計を可能にすることができる.c. 加工技術情報生産資源と加工特徴情報を結びつけるための技術情報を与える.すなわち,加工特徴情報に対して,その加工を実行することができる生産資源を対応付けるとともに,その生産資源で加工を行う際の作業指示書,あるいはNCデータなどを作成する際に必要な技術データを与えるものである.以上のデータベースを用いる場合に,製品設計データから加工工程データに変換するプロセスにおけるデータベースの役割を図1・6の下部に示す.すなわち,加工特徴のカタログは,CAD で設計された製品データから加工特徴を抽出し,工程設計用の製品モデルを作成するために用いることができる.さらに,加工特徴のデータから,加工技術データベースおよび生産資源データベースを参照することで,加工工程データ,加工指示書およびNCデータを作成することが可能になる.このようなデータの変換プロセスおよびデータベースの整備は,工程設計を体系的に行ううえで極めて重要である.文献(1) 岩田一明・ほか5名,生産システム学,(1982),129-130,コロナ社.(2) 岩田一明・中沢弘,生産工学,(1988),15-16,コロナ社.(3) 坂本千秋, STEP の進め方にかかる調査研究報告書, (2001), 1-18~1-29,型技術協会.(4) 岩田一明・ほか2名,技術者研修テキスト機械加工, (1980), 219-221,中小企業調査会.(5) 吉川武男,オペレーションズ・リサーチ,43-1(1998),7.→ Next Chapter
