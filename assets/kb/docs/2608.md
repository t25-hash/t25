# 3・8・2 設備保全管理のフレームワーク

> **章**: 第3章 管理システム
> **節**: 3・8 設備管理

設備の運用中には，種々の不具合が発生する．これは，設備の機
能が，ユーザの要求レベルを満たさなくなるためであるが，その原
因は，以下の4項目に大別される．
(１)使用環境，ニーズの変化 使用環境の変化や要求機能の
変化により，設備が実現できる機能が要求機能を下回ってしまう．
(２)設備に作り込まれた欠陥 設計不良，製作・据付不良，
あるいは，材料，部品，機器等の不良により設備に故障が発生す
る．
(３)強制劣化 運転条件，使用条件が不適正であるために，
設備の劣化が加速され，故障が発生する．
(４)自然劣化 使用に伴い不可避的に生じる劣化により故障
が発生する．
これらの問題に対処するのが設備保全管理の目的である．(１)へ
の対応が設備改良，(２)(３)への対応が設備改善，(４)への対応が
設備維持活動となる．欠陥や強制劣化の排除のための改善活動につ
いては，本来は，設備の立上げ時にこれらの問題が解決されている
ことが理想であるが，現実としては，保全活動の多くの部分を占め
ており，また，その効果は大きい．(１)の使用環境，ニーズの変化
を除くと，(２)～(４)は，いずれも設備に生じる故障が問題である
ので，その対応が設備保全管理の主要な課題となる．
設備保全管理においても，PDCA（plan-do- check-action）の
図3･30設備保全管理のフレームワーク
図3･ 31保全方式の分類（JIS Z8115：2000）
管理サイクルを回すこと，すなわち，保全方式を選び，それに基づ
いて保全作業を実施し，その結果を評価し，保全方式の見直しにつ
なげていくということが基本となる．この考え方に基づいた設備保
全管理のフレームワークを示したのが図3･30である ．図3･30
には，基本保全計画を軸とした三つの管理サイクルが示されてい
る．最も内側のサイクルが，運用段階での保全作業実施のサイクル
である．ここでは，基本保全計画で決定した保全方式に基づいて保
全作業計画を立案し実施する．保全作業の結果は，基本保全計画時
に想定された状況（劣化の進行や故障の発生形態など）に照らして
評価される．結果が想定した状況の範囲であれば，次の保全作業計
画に移り，作業の実施・評価が繰返される．一方，選択された保全
方式に不都合が認められたり，予想していなかった問題が現れたり
した場合は，基本保全計画に戻り，運用段階で得られたMP情報
を考慮して保全方式の見直し・改訂を行う．これが，保全管理の第
二のループである．基本保全計画において，改良保全が適当と判断
された場合には，さらに左側に戻って設備の改良が行われる．これ
が第三のループになる．このように，基本保全計画を軸とした管理
により，計画的な保全の実施と，環境の変化やMP情報のフィー
ドバックに基づく保全方式の更新と改良が可能となり，ライフサイ
クルを通じた設備状態の維持と改善の実施という設備保全管理の目
標が達成できる．
このような設備保全管理を実現するためには，種々の技術が必要
となるが，以下ではそれらの中でも特に重要と考えられる，保全方
式の決定などを行う保全計画技術と，実際の設備状態を把握するた
めの設備診断技術について述べる．
また，生産設備に関しては，設備管理活動の改善が生産効率の向
上に大きく貢献する．そこで，3･8･5項では，設備管理を基礎とし
た改善活動として知られているTPMの概要を説明する．
