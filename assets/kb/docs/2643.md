.目標仕様（what ) 対象分野が決定されたら，次に，そ
の対象分野において具体的にどんな製品を作るのか，その製品が達
成すべき主要な目標仕様（target specification）を決定する必要が
ある．これは，作るべき製品の定義付けにほかならず，目標仕様の
決定こそが，製品企画そのものともいえる最も重要な部分である．
この目標仕様の決定については，顧客と製品の二つの視点がある．
なお，対象分野で利用可能な技術の視点は，こと製品企画における
目標仕様の決定に際してはややあいまいなため，一般には適さない
ことに注意が必要である．
(１)顧客
この視点では，製品の目標仕様を，顧客に提供する便益から決定
する．例えば，東京～大阪間を片道2時間以内，到着時刻の誤差
10分以内，料金1万5000円以内で，人間が安全に移動可能な手段
というのが顧客の視点である．
(２)製品
製品の視点は，顧客の視点を，さらに具体的な製品レベルの目標
仕様へ具体化していくものである．通常，目標仕様は，この視点に
よって定められる．上記の例でいうと，この市場の要求を満足する
具体的な製品としては，飛行機と鉄道が考えられる．ここで，飛行
機では到着時刻の誤差10分以内という要求は実現不可能と判断し
たとする．すると，選択肢としては鉄道が残る．次に鉄道には，新
幹線とリニアモータカーがあるが，後者では1万5000円以内とい
う料金の実現は極めて困難と判断したとする．その場合，結果的に
新幹線という製品を選択することになり，例えば，安定的に走行可
能な時速300km以上という目標仕様が決定される．
この顧客と製品という二つの視点は，それぞれ利点と欠点があ
る．すなわち，顧客の視点は，この製品は何のためのものかという
目的が明確になるという長所があるが，実際の設計を行うにはやや
具体性に欠けるという短所がある．一方，製品の視点は，より具体
的であるという長所があると同時に，ともすれば製品を作りあげる
ことだけに視点が限定され，製品の目的そのものから注意をそらし
がちになるという短所がある．したがって，目標仕様の決定につい
ては，可能な限り，顧客と製品の両面から行うのが望ましい．
c .製品企画の要件（why，how，how much) 対象分野と目
標仕様を決定すれば，製品企画の内容が一応定まったことになる．
しかし，これで製品企画が完了したわけではない．すなわち，製品
企画の内容は，市場性，競争優位性，実現性，収益性の四つの要件
を満足する必要がある．
.市場性（why・その1) 市場性（marketability）に
は，ミクロおよびマクロの二つの視点がある．
(１)ミクロの視点
ミクロの視点は，上述した目標仕様（what ）で定義した製品が，
顧客のニーズに合致するのかという視点である．今日のようないわ
ゆる成熟期にあっては，単に顧客のニーズがある，言い換えれば，
あったほうがよいという程度のニーズでは不十分であることが多
い．ましてや，あってもよいくらいの弱いニーズでは，赤字製品に
なってしまう可能性が高い．したがってここでは，ぜひとも購入し
たい，なくては困る，あるいは，買わざるを得ないといった強力な
ニーズがあるか否かが検討の焦点となる．
(２) マクロの視点
もう一つの視点は，現在および将来の市場規模というマクロの視
点である．この際，さまざまな市場予測のデータを活用することに
なるが，客観的で正しい予測というものに過度に期待するのではな
く，必ず複数の予測データを比較したうえで，製品企画の責任者が
自らの予測を行うことが必要不可欠である．
さらに，市場規模は大きいほどよいということにはならないこと
にも注意が必要である．自社の売上げ規模と比較して大きすぎる市
場規模は，ともすれば現実から目をそらさせる．たとえシェア5 %
でも，大変な売上げ規模になるといった粗っぽい議論に陥る可能性
が大きい．シェア5%程度では，次に説明する競争優位性の要件を
満たすことが難しく，よって赤字製品に陥る可能性が大きい．この
場合，ミクロの視点から市場全体をもう一度冷静にとらえ直して，
自社が本当に対象とする市場を絞り込む必要がある．例えば，市場
規模を5分の1に絞り込んだうえで，その中で5%×5倍＝25%の
シェアをねらったほうが，より的確な製品企画になる場合が多い．
.競争優位性（why・その2) 競争優位性（competitive
advantage）とは，顧客の視点から判断して，競合製品と比較した
場合の自社製品の優劣をいう．ここでは，まず顧客の視点から判断
という点が重要である．かつて，VTRが成熟期にさしかかって売
上げの伸びが鈍化し始めたころ，メーカ各社が差別化と称して録画
再生機能の多機能化に注力し，リモートコントローラのボタンがど
んどん増えていくという時期があった．ところが，それらの多機能
化について，ユーザは関心を示さず，操作が面倒なこともあって，
結局，一度も使われない機能が続出してしまった．この例では，メ
ーカ各社の差別化を目指した努力は，結果としてユーザには訴求し
なかったのである．言い換えれば，顧客の視点を無視した差別化
は，競争優位性にはつながらない．
また，競争優位性とは，あくまでも競合との比較であることに注
意する必要がある．どんなに優れた製品でも，他社製品がそれより
も優れていれば競争優位性はないということになる．反対に，たと
え完璧な製品でなくても，他社よりも一歩でも優れていれば，競争
優位性があるということになる．
なお，競争優位性には，企業や製品のブランド力なども含まれる
が，設計の観点から特に重要となるのは，技術の独自性である．
.実現性（how) 製品企画のhowとは，製品を実際に作
れるのかどうかという，実現性（feasibility）のことである．これ
については，以下の二つの点に注意する必要がある．第一に，実現
性と競争優位性（技術の独自性）とを混同しないことである．過去
長年にわたって，わが社が取り組み，豊富な実績のある技術，とい
うものは，多くの場合実現性を指すのであって，競争優位性（技術
の独自性）のことではない．そして第二に，この実現性を考慮しな
い，例えばスケッチで表現しただけの製品企画は，製品企画とはい
えないということである．ただし，製品企画の時点では，必ずしも
すべての技術が実現性を満たす必要はない．この場合，その時点で
実現性を満たさない技術は，企画された製品の実現を目的とした研
究開発（技術開発）テーマとなる．
.収益性（how much) 製品企画のhow muchとは，売
上高とコスト（研究開発投資を含む）から製品の収益性
（profitability）を計算し，予測することである．ここで大切なの
は，収益性の計算そのものを精緻に行うことだけではあまり意味が
ないことである．むしろ，収益性の計算を通じて，その製品開発に
伴うリスクとリターンを定量的に明確にすることにこそ，本来の意
義があることを再認識すべきであろう．ここで，リスクとリターン
は均衡しているのが基本となる．例えば，改良型の製品では，通
常，リスクとリターンはともに低い．反対に革新型の製品では，一
般に，両者はともに高い．また，リスクが高いのにリターンが低い
製品は，企画そのものに問題があるので，設計に入る前に企画を徹
底的に見直す必要がある．さらに，リスクが低いのにリターンが高
い場合は，見通しが甘いのではないかと疑ってみる必要がある．
さて，収益性の計算には，前述した市場性，競争優位性（技術の
独自性），実現性という，それぞれ個別の視点からの分析が基礎と
なっている．すなわち，市場性があって，競争優位性があり，かつ
実現性が高い製品は，結果として収益性も高いはずである．これら
のそれぞれ個別の視点を統合して再度分析してみることに，収益性
の考え方の基本がある．これに関してしばしば見られるのが，始め
に製品開発ありきで，そのために必要なコストを正当化するために
収益性を計算し，そのつじつま合わせとして，個々の計算が必要以
上に緻密になっているというものであるが，これでは本末転倒であ
ることは言うまでもない．
d.ジレンマの存在と意義
.ジレンマの存在 製品企画における市場性，競争優位性
（技術の独自性），実現性の三つの要件間には，一般にそれぞれ相矛
盾する条件，すなわちジレンマが存在することに注意する必要があ
る（図3･8）．
(１)市場性と競争優位性（技術の独自性）のジレンマ
企業の独自技術が，すんなりと市場，すなわち顧客に受け入れら
れることはまれである．顧客は，自らの要求を満たしてくれればそ
れでよいのだから，よほどのメリットがない限り，平凡かつ安全な
技術を優先するのが普通である．
(２)市場性と実現性のジレンマ
市場性が明らかに高いと思われる製品は，技術的に実現すること
が困難な場合が多い．一方，技術的に実現が容易な範囲だけでは，
一般に平凡な製品となりがちであり，市場性が高い製品を企画する
ことは困難である．
(３)競争優位性（技術の独自性）と実現性のジレンマ
企業には，できるだけ早くかつ確実に製品を市場に投入したいと
いう要求があり，この場合，実現性が重視される．しかし，実現性
を追求すればするほど，独自性は失われる．一方，独自性を追求す
ればするほど，実現性は低くなってしまう．
.ジレンマの積極的な意義 これらのジレンマは，一見，
製品企画を妨げる阻害要因にも思えるが，実はそうではない．むし
ろ，このジレンマの存在こそが，製品企画における創造性の源泉な
のである．もしもジレンマがなければ，誰が製品を企画しようと
も，その結果には差異が出ない．この場合は，製品企画そのもの
は，製品開発・製品設計において，それほど重要な位置付けにはな
り得ない．したがって製品企画の成否は，このジレンマを具体的に
見抜き，その的確な解決方法を見出すことができるか否かにかかっ
ている．
