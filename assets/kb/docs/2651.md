# 3・10・6 システム的な視点に基づいた設計方法論の展開

> **章**: 第3章 設計のための個別方法論
> **節**: 3・10 システマティックデザイン

さて，3･10･5項での内容は4･4節の内容とも関係するものであ
るが，3章で取り上げられたさまざまな方法論もシステムという視
点のもとでとらえると，その意味も改めて明確になる．
品質機能展開（QFD:quality function deployment ) はバラ
ンスのよい設計を導き出すためのツールあるいは思考法であるとい
える．その背景には，設計において対象物を把握するうえでの視点
がさまざまに存在していて，その視点ごとにシステムを階層的に分
解していった際のサブシステムの構成が異なっている一方で，設計
結果そのものは一体のものであることがある．QFDはサブシステ
ム間での重要度の配分を視点間で相互に整合性のとれたものにする
ことによって，優れた設計を行おうとするものである．その考え方
は基本的には図3･154に示すような二元表が連鎖したものである．
すなわち，まず，ある視点から見たシステムの特性（重要度）をそ
の各部分に階層的に割り振る．それらを各表の縦軸と横軸に配置
し，縦軸の各指標についての重要度を基にしつつ，縦軸の各指標か
ら横軸の各指標への寄与度を定義することを通じて，横軸の各指標
の重要度を算出しようとするものである．さらに，ある表の横軸の
内容を別の表の縦軸の内容に据えて，さらに異なる視点を横軸に据
えれば，さらに連鎖させて同様の操作を展開していくことができ
る．つまり，QFDはシステムである製品のさまざまな視点のもと
でのさまざまな内容についての統合性を検討するうえでの媒体とな
っている．
信頼性の確保は設計において必須の課題であるが，それに関して
も，概念設計の段階で信頼性を評価するためのツールあるいは思考
法であるFMEA（failure mode and eﬀect analysis ）やFTA
（fault tree analysis ）も設計対象をシステムとして理解することが
基本にある ．図3･148に従えば，FMEAはあるレベルにお
けるサブシステム間でのつながりに関して，ある部分での故障が他
の部分での機能にどのような影響を及ぼすかを考えようとするもの
であり，FTAは，それに加えて上下方向への展開を加味すること
によって，ある目的が達成できなくなる事態に至るシステム構造上
の因果関係をトップダウンに明確にしようとするものであると見る
ことができる．つまり，これらの思考法に関しても，信頼性設計の
ための考え方それ自体からはやや離れて，システムと設計との関係
から眺めてみることでそれについての理解が深まり，さまざまな方
法論や考え方の連係性についても明らかになって，一連の考え方の
総合的な運用が可能になる．
このほか，設計対象を個別の技術領域から離れてとらえる場合に
はシステムとして理解することになることから，システム工学にお
ける手法，例えばISM（interpretive structural modeling）や多
変量解析（multivariate analysis ）などは設計における概念操作や
構造分析において有効な手段となることが知られている ．
