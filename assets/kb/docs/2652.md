# 3・2・1 品質における設計の重要性

> **章**: 第3章 設計のための個別方法論
> **節**: 3・2 機能と品質の設計

ボーダレス化がヒト・モノ・カネ，情報のすべての分野で急速に
進み，顧客満足に基づいた魅力ある製品作り，技術開発の効率向
上，製造の垂直立上げが，コスト低減，リードタイム短縮とともに
必要不可欠となってきている．これらの企業ニーズにこたえるため
には，従来の技術開発手法，品質管理手法ではもはや限界に来てい
る．
従来の品質管理手法に対して，その限界に言及した指摘を以下に
示す．
a.品質管理の反省 日本製品の品質向上のため，各企業の
品質管理（QC:quality control ）が効果のあったことは言うまで
もない．しかし今日の品質新時代における品質問題に対応するため
には，従来の品質管理活動では必ずしも十分ではなく，次のような
問題点があることは否めない．
(１)従来のQCは再発防止に重点がおかれている．しかし，製造
物責任（PL:product liability）問題では，不良が発生して
しまってからではすでに遅い．すなわち予防が必要である．
(２)従来のQCは製造段階での品質の向上，維持に重点がおか
れ，大量生産の場では大きな効果を収めている．しかし，設
計品質の確保，すなわち市場要求に適合し，自己のもつ技術
を高度に活用して優秀な品質の製品を企画・設計するという
点については，いまだ十分威力を発揮しているとはいいがた
い．
b.品質意識の誤解 製造現場とか市場で問題が発生するの
は，いわゆる品質問題である．したがって，技術の問題はすべて品
質問題という風潮が企業の中に染みついている．日本の品質管理の
優秀さが海外に鳴り響いたということからも，このような考え方に
拍車がかかった．QCサークルというものを考えた品質管理の推進
者は実に素晴らしいと思うが，よく考えてみれば，これは開発や設
計の技術者の敗北である．つまり，開発や設計で見落としたことを
現場で指摘されたということである．
品質問題を起こしている技術というのは，設計が悪いケースが多
いのである．ここでいう設計とは，製品設計と生産技術の設計を含
む．設計の悪さをさらにたどれば，設計の考え方を決めた，研究・
開発の不備に至ることも多いのである．
従来は，製造現場が設計の問題点について対策を講じた．これを
得意とした日本の製造技術が強いといわれた理由である．
商品化の流れを開発・設計，製造，市場とたどるとき，問題が顕
在化するのが下流になればなるほど，解決までの時間とコストが増
える．例えば3･2･3項で後述する品質工学では，商品化の流れをさ
かのぼる形で品質を下流・中流・上流・源流の4階層に分類し，で
きるだけ上流にさかのぼって改善を行うのが能率がよいとしてい
る．ここで下流の品質は消費者にわかる市場の品質で，中流の品質
とは製品の設計図とスペックであり，上流，および源流の品質が開
発・設計の領域である ．
質を保証する方策を立案する．
商品化までの時間とコストの低減は，生き残りをかけた企業の切
実なニーズであり，開発・設計にまでさかのぼった品質の確保がま
すます重要となってきたのである．それを進めるに当たって，従来
の品質管理手法ではもはや限界に来ているのは，上述したとおりで
ある．
開発・設計にまでさかのぼる品質の確保の手法として，より正確
ように展開したのが機能系統図である．
に言い換えれば品質の確保を重視した開発・設計手法として，本節
では二つの手法に注目した．まず3･2･2項で品質機能展開
（QFD）に触れ，ついで3･2･3項で品質工学（タグチメソッド）に
ついて述べる．
さらに3･2･3項で，最新の手法として品質工学（タグチメソッ
ド）によるロバストデザインにも言及する．
