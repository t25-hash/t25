# 3・6・3 製品ライフサイクル設計

> **章**: 第3章 設計のための個別方法論
> **節**: 3・6 ライフサイクル設計

図3･93のようなさまざまな循環生産の実現を考えた場合，リサ
イクルだけでなく，リデュースやリユースを含めた3R（reduce，
reuse ，recycling）を適切に組合わせて，インバースマニュファク
チャリングを実現する必要がある．このように，製品の一生を適切
に設計することをライフサイクル設計（life cycle design）と呼ぶ．
ライフサイクル設計が必要な理由は，製品の多くの特性は設計時
点で決められてしまうからであり，さらには，製品とその製品がた
どるライフサイクルとには不可分な関係があるからである．例え
ば，金属と多種類のプラスチックが分解困難な形で締結された構造
の製品は，リサイクルすることが極めて困難，高コストである．す
なわち，ライフサイクル設計は，製品と製品のライフサイクルを統
合的に設計しようとするものである．表3･35に従来の設計，3･6･2
項の(１)で述べた多くの製品で実施されているリサイクル設計，お
よびライフサイクル設計の違いをまとめて示す．
このようなライフサイクル設計と従来の製品設計を設計者から見
た相違は，以下のようにまとめられる．
(１)製品ライフサイクル全体への視野の拡大
従来の設計はおもに製品使用時の性能に着目してきたが，ライフ
サイクル設計においては製造から使用終了後の再利用，廃棄に至る
までの製品のライフサイクル全体を見通して設計を行わなければな
らない．この問題は技術的には設計と製造の両工程を考えるコンカ
レントエンジニアリングの延長上にあると考えられるが，設計に要
求される視野の広さの点で質的な変化が起きる．
(２)多視点性
上記に付随して，設計に要求される，もしくは設計時点で考えな
ければならない項目，属性が多様かつ複雑になる．極端にいえば，
従来の設計の目標はより安い価格でより高性能の製品を作るという
ことであり，ある意味でその目標は明確であったが，環境問題を考
えた場合，省資源，省エネルギー，高リサイクル性，有害物質の削
減など非常にさまざまな要求項目を満たす設計を行わなければなら
なくなる．なおかつこれらの要求項目間にはトレードオフ関係があ
ることが多い．このように，設計問題が複雑化，多様化することが
ライフサイクル設計の第二の特徴である．

図3･ 95 製品シリーズと世代
略の策定，ライフサイクルシナリオ（life cycle scenario）の作成
（ライフサイクルオプションのベストミックスを選択し，循環方法
のシナリオを作成すること），製品設計，プロセス設計を統合的に
行うことである．
ここで，ライフサイクルオプション（life cycle option）とは，
3･6･2項で述べたメンテナンス，リマニュファクチャリング，リユ
ース，マテリアルリサイクル，サーマルリサイクル，さらには，ア
ップグレードや適正廃棄など，製品の循環経路に関する選択肢のこ
とである．欧米では，end-of -life option といわれることもあ
るが，メンテナンスやアップグレードなど狭義のend- of - lifeでは
ないオプションも対象にする場合，ライフサイクルオプションと呼
ばれる．
図3･94では，おもな製品ライフサイクル特性として8項目挙げ
た．この中で製品の寿命［lifetime，物理寿命（physical life-
time），価値寿命（value lifetime）］が鍵となるのであるが，それ
以外にも製品の販売期間，製品カテゴリの寿命（8ミリビデオなど
製品カテゴリがどのくらいの期間，市場に存在するか），使用量・
廃棄量（単品生産物と大量生産するものではおのずからその処理方
法が異なる），使用方法・ユーザ，製品の構造，回収システムの有
無などを十分に把握する必要がある．
製品の寿命は，以下の2種類に分類して考えられる．すなわち，
従来の意味で製品が故障する，劣化するといった物理寿命と，パー
ソナルコンピュータや携帯電話に見られるように，製品自体は正常
に動作するが新製品に比べて機能が大幅に劣った，デザインが古く
なったために捨てられてしまうという意味での価値寿命であり，実
は現代の工業製品では価値寿命が支配的な製品が多い．製品の価値
寿命を維持，向上させながら，物理寿命まで製品や部品を使い尽く
すことがライフサイクル設計の一つの大きな課題である．
先に述べた設計の視野の拡大にも関連するが，ライフサイクル設
計の対象は単一の製品とは限らない．むしろ限定しないほうがよ
い．例えば，3･6･1･ b項で述べたインバースマニュファクチャリン
グを前提とすれば，提供すべきものはユーザが要求するサービスで
あるので，製造業においても従来のようにハードウェアを販売する
という形態ではなく，部分的にソフトウェア化しオンラインで販売
したり，製品自体はレンタルするといった多様な機能実現の方法を
考えるべきである．また，機械自体を対象とする場合にも，単一の
製品のみで考えるのではなく，製品系列（シリーズ）や世代を視野
に入れ（図3･95），シリーズ間や世代間での設計情報や部品の共通
化を検討することが有効である．
