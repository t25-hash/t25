シミュレーションの手順は概略以下のようになる．
(１)対象とする製品，現象の範囲を特定する．
(２)対象とする製品，現象をモデル化する．
(３)数学モデルに置換える．
(４)結果を予測する（厳密解あるいは実験結果による予測）．
(５)数学モデルを解く（理論的あるいは数値的に解く）．
(６)結果を評価する（厳密解あるいは実験結果との比較）．
上記の数学モデルを解く際に大規模な計算機を必要とするのがコ
ンピュータシミュレーション，手計算，電卓，あるいは小規模な計
算機で可能なものが設計シミュレーションと，ここでは定義する．

は，設計者が細かいことを知らなくてもある程度の答えを出してく

ことである．コンピュータシミュレーションは一種の数値実験であ
るが，実験の場合は計測という行為が入るために，ある程度結果を
予測する必要があり，この行為が結果として現象の理解につなが
る．一方，コンピュータシミュレーションはこのような手順を踏ま
ないでも使えるため，便利な反面，危険でもある．また，コンピュ

あるにもかかわらず，答えに差のあることがわかる．詳細に検討す
ればこれらの違いの原因・理由はわかるが，一般に設計者がそこま
で理解することは困難である．実験結果と同様に，コンピュータシ

次に，コンピュータシミュレーション手法について述べる．
物理現象の多くは偏微分方程式で表現できる．これを計算機で数
値的に解くためには，方程式を離散化する必要がある．離散化の方
法としては差分法（finite diﬀerence method），有限要素法（finite
element method），境界要素法（boundary element method）があ
り，離散化された方程式は多元の連立方程式となり，最終的にこれ
を解く．

範囲を超えて設計に適用してしまう危険をはらんでいる．
