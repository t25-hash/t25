> **章**: 第3章 設計のための個別方法論

a.ユーザニーズ収集の方法 ユーザのニーズは，3P（ポイン
ト）タスク分析，直接観察やグループインタビュー，アクティブリ
スニング法 ，評価グリッド法 などの方法を用いて収集する
ことができる．HDTでは，それらの中でも，ユーザの無意識の行
る3Pタスク分析や直接観察を重要視している．それらの方法は，
ユーザを用いないことも可能であり，コストの面からも魅力的な手
図3･143 HDTのプロセス
.直接観察の方法（direct observation method) 直
接観察による具体的な方法を下記に示す．
(１) ヒューマンマシンインタフェース（HMI ）の5側面から
問題点を抽出する
HMIの身体的側面，頭脳的側面，時間的側面，環境的側面，運
用的側面からインタフェース上の問題点を抽出する．身体的側面
は，①トルク，②位置，③フィット性の3項目から身体面での問題
点を抽出する．頭脳的側面では，情報のインタラクションでの問題
点，時間的側面では作業時間に関する問題点，環境的側面は照明，
空気調和など環境に関する問題点，そして運用的側面はHMIを効
率よく運営していくうえでの問題点を抽出する．
(２)痕跡から問題点を探す
システムとユーザとの不都合は痕跡となって現れるので，痕跡を
見つけることにより問題点を抽出することができる．
(３)操作，行動の手がかり（cue）から問題点を探す
われわれが操作したり行動したりする場合，手がかりが必要であ
る．この手がかりが何か，あるいはそれが効果的なのかを検討す
る．
(４)識別性から問題点を探す
情報は識別することにより特定できる．この点から問題点を探
す．
(５)システム側によるユーザに対する制約条件から問題点を探
す
券売機のコイン投入口での制約された操作など，ユーザの操作，
行動に対する制約条件を調べることにより，問題点を抽出できる．
以上(１)～(５)の観察により抽出された問題点は，グループごと
にまとめて要求事項化される．
.3 Pタスク分析（3P task analysis ）の方法 タスク
分析とは機器を使う際のシーンごとのタスクに関して，問題点を抽
出する方法である．大まかにいうと，タスクが連続して切り替わっ
ていくシーケンシャル型と，タスクが階層構造になっている階層型
の2種類がある．この場合，ユーザを使って問題点を抽出すること
が理想的であるが，ユーザを使わず実験者がユーザになったつもり
で問題点を抽出してもよい．
従来のタスク分析の大部分はおもに機器の入出力部分の問題点を
抽出する構造になっていて，ユーザの情報処理レベルでの視点が弱
い．このような従来のタスク分析に対し，評価検討抜けがないこと
とユーザの視点で評価できるようにするため，ユーザの情報処理レ
ベルでの問題点抽出をできるようにしたものが3Pタスク分析（図
3･144）である．この方法では，情報処理レベルにおいて，情報入
手→理解・判断→操作の3ステップから問題点を抽出する．従来の
方法では問題点の抽出段階で終わっているが，さらに製品コンセプ
ト構築につなげるため，それぞれのタスクにおいて抽出された問題
点を要求事項に変換する欄を右端に設けている．変換は7項目の手
がかりを参考にして行う．この欄は現実的に解決できる項目を書く
欄と，現実にはコスト，技術的に無理ではあるが，近い将来実現が
見込まれる項目を書く欄の二つに分かれている．
三つのステップのうち，情報入手ステップは，ユーザが情報を入
手する段階で，情報をいかに容易に手に入れるかがポイントであ
る．人間の情報処理の感覚，知覚レベルに該当する．
理解・判断ステップは，知覚情報を理解して判断を下す段階であ
る．わかりやすさの構築がポイントである．
操作ステップは，判断された情報を人間の効果器（手や足など）
を使って機械に与える段階である．身体面での快適，効率性がポイ
ントである．
なお，以上の各ステップにより30機種の電化製品のタスク分析
を行ったときの問題発生頻度の高い評価項目は，以下のようなもの
である．これらの項目を手がかりにすると，予備知識がなくとも問
題点の検出は容易になる．
(１)情報入手：①レイアウト（位置）が悪い，②見にくい，③強
調されていない，④情報がない，⑤マッピングがされていな
い
(２)理解・判断：①意味不明，②アフォーダンスがない，③紛ら
わしい，間違える，④フィードバックがない，⑤手順が問
題，⑥一貫性がない，⑦メンタルモデルが問題
(３)操作：①身体的特性と不一致［特に，姿勢，フィット性およ
びトルク（操作に必要な力）の3項目が重要］，②面倒であ
る
b.状況把握の方法 企画する製品のイメージを構築するため
には，現在市場に出ている商品がユーザにどのように知覚されてい
るかを確認する必要がある．
これについては，コレスポンデンス分析（correspondence anal-
ysis method) を活用することができる．
調査したい商品群に対して，評価項目（高級感がある，モダン
な，など）がどの程度該当するのか，ターゲットユーザに対するア
ンケートにより各項目の頻度を求める．アンケート結果は，コレス
ポンデンス分析により二次元座標上に，商品と評価項目が同一面上
に布置され，それらの関係が深い場合，近くに布置される．見方は
原点からの方向と原点からの長さから検討する．それらの商品が置
かれていない領域をねらい競争を避けるか，あえて同じ土俵上で勝
負するのかを決める．
c.製品コンセプト（product concept ）構築の方法
.コンセプト項目の体系化 ユーザニーズ収集により得ら
れたユーザの要求事項は，同じ機能どうしでまとめて階層化を行う
ことによりコンセプト項目として体系化し，製品コンセプトを構築
する．まず，上位の項目と最上位項目の構造の文脈から，新しい上
位の項目があれば追加する．また，この要求事項はもともと問題点
図3･144 3Pタスク分析
から作られた項目であるため，その製品の必要十分条件の項目がな
い場合や企画者として追加したい項目がある場合，必要に応じてこ
れらも追加する．さらに上位項目，下位項目のウエイト付けを行
う．このウエイト付けは非常に重要な作業であり，これにより製品
コンセプトの骨格が決まる．ウエイト付けは，ユーザの意向を反映
させるため，アンケート調査やAHP（analytic hierarchy proc-
ess ) 等により行う．これらのウエイト値は製造のコスト比にも
なるため，コストが合わない場合，ウエイト値が低い項目を排除す
ることも検討できる（図3･144）．最後に，体系図の最下位のコン
セプト項目に対して抽象化された意味（設計項目）を与えることに
より製品コンセプトの体系図を完成させる．図3･145 の上半分
はその一例である．
.設計項目への対応付けとそれらの種別 製品コンセプト
を明確に整理するうえで設計項目の種類や分類を知っておければ，
デザイン経験にかかわらず要求事項の具現化が可能となる．HDT
では，設計項目を八つの大設計項目とその下位の70の小設計項目
に分類して体系化している．
一連の項目のもと，製品コンセプトの最下層のコンセプト項目に
関係する設計項目を選択すれば，これらの項目から具体的なアイデ
アを創出することができるようになる．例えば，壁掛け時計のデザ
インで，文字の見やすさの獲得が最下層のコンセプト項目であるな
らば，これに関係するユニバーサルデザイン設計項目の情報入手を
参照して文字のサイズとか形状のアイデアを固める．この種の設計
項目はすべての製品に対応できるわけではないが，基本的で重要な
項目である．下記に，製品に必要な検討要素に対応した八つの設計
項目を示す．また，この8項目には，合わせて70の下位の設計項
目があるので，()内に示した．
(１)インタフェースや使い勝手の配慮→①ユーザインタフェース
設計項目（29項目）
(２)高齢者・障害者に対する配慮→②ユニバーサルデザイン設計
項目（9項目）
(３)製品の魅力や感性に関する配慮→③感性設計項目（9項目）
(４)安全性に対する配慮→④安全性（PL）項目（6項目）
(５)頑強性に関する配慮→⑤ロバストデザイン項目（5項目）
(６)メンテナンスに対する配慮→⑥メンテナンス（保守性）項目
（2項目）
(７)地球環境に対する配慮→⑦エコロジーデザイン項目（5項
目）
(８)その他→⑧その他（HMIの5側面など）（5項目）
.ユーザインタフェース設計項目（user interface design
item，29項目)
(Ａ)システムとして要求される項目：システムとして備えるべき
ユーザインタフェース関係の項目である．
(１)寛容性・柔軟性：ユーザの知識，経験，習熟度，好みなど
に応じて，機器が柔軟に対応できるようにする．
(２)習熟度対応：ユーザの習熟度に対応させる．
(３)ユーザの保護：ユーザの身体に危害を加えないよう保護す
る．
(４)ユニバーサルデザイン：健常者だけでなく身体障害者，高
齢者などでも健常者と同等に操作できるインタフェースを
提供する．
(５)異文化対応：対象となるユーザの言語，習慣，宗教など，
文化的背景を考慮する．
(Ｂ)ユーザのやる気の醸成：ユーザにやる気を起こさせる項目で
ある．
(６)楽しさ：ユーザが積極的に操作に取り組み，また使いたい
と感じられるような楽しさを提供する．
(７)達成感：操作がうまくできる喜びをユーザに感じさせ，ま
た使いたいと感じさせる．
(８)ユーザの主体性の確保：業務の開始から完了まで，ユーザ
の意思で思いどおりに操作できるようにする．
(９)信頼感：ユーザとの間で信頼関係をもてるようにする．
(Ｃ)効率のよいインタラクションの構築：情報入手→理解・判断
→操作の各ステップにおいて，効率のよいインタラクション
を実現させる項目である．

.ユニバーサルデザイン設計項目（universal design item，
9項目)
(１)調整：調整とは，体形，身長，作業姿勢，操作に対する熟達
度，障害の有無などさまざまに異なるユーザに対し，機械を
対応させることである．
(２)冗長度：冗長度とは，機能面での代替案があることである．
(３)仕様，機能が見える：取扱説明書を読まなくとも，いろいろ
な手がかりによりどのような操作を行うのか，あるいは機器
の機能を理解できることを意味する．
(４)フィードバック：ユーザの操作に対するシステム側の反応の
ことをいう．
(５)エラーに対し寛容：ユーザがエラーを行った場合，何らかの
対応が機械側からなされることである．
(６)情報の入手：ユーザが行動するためには，最初に情報を入手
しなくてはならないが，情報を効率よくユーザに伝えること
である．
(７)情報の理解・判断：入手した情報を誰にでもわかるようにす
ることである．
(８)操作：ユーザが容易に操作を行うことができることである．
(９)情報や操作の連続性：目的を達成するまでに，情報や操作
（作業）の流れが途切れないことである．
.感性設計項目（kansei design item，9項目)
(１)デザインイメージ：近代的，懐古的やしゃれたなどのデザイ
ンイメージに関する要素
(２)色彩：安心感のある色，斬新な色などの色の要素
(３)フィット性：形状がフィット，包み込む感覚などの機器と人
間との一体感のある感覚
(４)形態：シンプルな形，スマートな形などの形態の要素
(５)機能性・利便性：機能がよい，使いやすいなどの機能や利便
性に関する要素
(６)雰囲気：インテリアがよい，落ち着いた雰囲気など
(７)新しい組合せ：映像と音楽の組合せ，異種の調和などの，い
ままでにない組合せによる効果
(８)質感：素材のもつ豊かさ，素材の新しい使い方など質感のも
つ要素
(９)意外性：新しい組合せと関係が深いが，感性を感じさせる基
本的な項目

第1階層：雰囲気
第2階層：デザインイメージ
第3階層：色彩，フィット性，形態
第4階層：機能性・利便性，質感
第5階層（最下層)：新しい組合せ，意外性
さらに機能的側面からグループ化を図ると，下記の3層にまとめ
ることができる．
(１)感性そのものを感じさせる要素として，雰囲気，デザインイ
メージ（第1＋2階層）
(２)感性のインタフェース要素として，色彩，フィット性，形
態，機能性・利便性，質感の5項目（第3＋4階層)：人間と
機器との関係を円滑にするもの
(３)感性の基本構成要素として，新しい組合せ，意外性（第5階
層)：設計対象物に感性を引き起こすための要素

d.デザインを総合化する方法 デザインの具体案は，製品コ
ンセプトの最下層のコンセプト項目とこれに関係する設計項目から

ているが，コンジョイント分析（conjoint analysis ）によりスペッ
クの特定とウエイト付けを行うこともできる．まず，未定の製品属
により属性（attribute）の寄与率（rate of contribution）と水準
（level ）の効用値（utility value）が計算されるので，高得点の項
目を組合わせ，スペックを決めることができる．
e .デザイン評価の方法 前ステップで作成したデザイン案と
市場に出回っている商品との評価をAHPを用いて行う．この評価
によりある程度の評価を得ることができれば商品化へのめどがつく
ことになる．この場合，製品コンセプト項目を基準にして評価を行
うが，このことは案の妥当性（verification，仕様の確認という意
味）を検討する意味もある．さらに製品のモックアップ（製品を模
した模型）や3Pタスク分析などで有効性の確認（validation）を
行う．
あるいは，製品は有用性（useful ），利便性（usable）と魅力的
（desirable）の3属性 から構成されていると考えて，この3項
目からデザイン案の評価を行うこともできる．また，製品の特性に
対応して3項目のサブ項目で評価を行ってもよい．
