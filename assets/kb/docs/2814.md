# 13・3・2 加工システムの遠隔操作

> **章**: 第13章 遠隔加工システム
> **節**: 13・3 遠隔加工システム事例

研究レベルではあるが、工作機械を遠隔地から実感を伴って操作する例がある。ここで紹介する例では、工作機械のコントローラ(オープンアーキテクチャCNC,open architecture CNC)とネットワークに結合された外部コンピュータとを高速シリアル通信回線で情報伝送可能とし、外部コンピュータを介して遠隔地からでも操作可能とするものである。遠隔画面操作/遠隔運転操作と遠隔運転操作サポートとを実現するために、遠隔地で操作画面を提示するコンピュータから外部コンピュータに実時間制御データ、あるいはNCプログラムを送信し、高速シリアル通信回線経由でオープンアーキテクチャCNCに制御情報を送る。また、オープンアーキテクチャCNCからコントローラに関するパラメータ情報等を送り、遠隔の画面に必要な情報を提示する。遠隔地に提示する画面は、標準的なAPIを用いて実装されている。プログラム運転モードと手動運転モードの操作画面が設けられており、おのおのの画面は、工作機械やCNCの状態を表示する部分、運転モードの切換えと非常停止とを行う部分、運転モードに対応して切換える部分とから成る。また、遠隔教育/訓練(remote education/training)をサポートするために遠隔地から力帰還(force feedback)の可能なジョイスティックを用いて実感を有しながらマシニングセンタを操作可能としている。ジョイスティックを用いて実時間で制御するときには、外部コンピュータにデータを送信し、高速シリアル通信回線経由でマシニングセンタの工具位置、主軸回転数を制御する。東京-高知間をISDN 回線で結んだ実験では、視覚・聴覚情報の時間遅れ(time delay)は300ms程度、制御情報の時間遅れは40ms程度であった。ところで、エンドミル加工では加工条件が一定であっても、工具が1回転する間に加工反力も変化する。高い周波数で変化する力情報をそのまま力感に提示することは困難であるため、変動する切削力を平滑して提示する。具体的には、工具1回転の間における力の軌跡の重心を計算し、これを代表的な力とする。さらに、力ベクトルの軌跡の重心の方向は送り方向とは角度を有するため、この方向に力を提示したのでは操作感が低下する。このため、工具の送り方向に加工反力を提示する。加工においては、加工後の部品の表面粗さが重要である。しかしながら、例えば切削加工中は、加工表面は切りくずや切削油剤に覆われ、高性能のTV カメラを用いても加工表面を見ることは不可能である。そこで、加工中に発生する多軸力情報をセンシングし、それを視覚情報に変換することによって加工表面を強調提示することも可能である。文献(1) (財)製造科学技術センター、 機械産業における情報システム化促進総合調査成果報告書、(2001)。(2) 光石衛、精密工学会誌、64-1(1998)、18.(3) Pritschow,G.、ほか8名、Annals CIRP,50-2(2001)、463.(4) 高田祥三、精密工学会誌、64-1(1998)、23.(5) 高田祥三、機械技術、49-12(2001)、45.(6) Byrne,G.、ほか5名、Annals CIRP,44-2(1995)、541.(7) 光石衛、機械技術、49-12(2001)、49.(8) 長尾高明・ほか3名、知能化生産システム、(2000)、176-190、朝倉書店。(9) Mitsuishi,M.and Nagao,T.,Annals CIRP,48-1(1999)、409.
