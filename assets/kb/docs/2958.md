# 4・2・6 不確実性を前提とした手法

> **章**: 第4章 設計の管理
> **節**: 4・2 設計開発プロジェクト計画と管理

多くの場合，開発プロジェクトを構成する各アクティビティの遂
行には不確実性やリスクを伴う．場合によっては目的としたアクテ
ィビティを完了できずに，開発プロジェクトそのものが途中で中止
になることもある．以下に，所要時間に不確実性を持ち込んだ
PERT手法としての3点見積り法，そしてアクティビティそのも
のにリスクを伴うプロジェクト計画法としてPDPCについて紹介
する．
a.3点見積り法（three points estimation method) PERT
における3点見積りとは，プロジェクトを構成する個々のアクティ
ビティの所要期間について，最初から不確実性を盛り込む方法であ
る．ここでは，各アクティビティの所要時間の楽観値（下限値），
最も実現できそうな最頻値，そして悲観値（上限値）の3点があら
かじめ見積もられる．そして，この3点から各アクティビティの所
要期間の平均と分散が，以下の式に基づいて計算される．
平均値＝(楽観値＋4×最頻値＋悲観値)/6
分散＝(悲観値－楽観値)/6
これは所要時間の分布がβ分布に従うことを仮定したものであ
る．そして各アクティビティの所要時間の平均値を用いて，4･2･4
項で示したPERT計算等が行われる．そのとき，クリティカルパ
スの完了期間がプロジェクトの平均完了時刻となる．そしてその完
了時刻の分散は，クリティカルパス上のアクティビティの分散の総
計として計算される．この分散を用いてある指定された日程内にプ
ロジェクトが完了できる確率や，逆に完了できないリスクの計算が
可能となる．
b.PDPC（process decision program chart ) PDPCと
は，開発プロジェクトのように不確実性やリスクを前提としたプロ
ジェクトの計画立案に対して，それらをあらかじめ見込み，あるい
は読みきってプロジェクトの成功に導くための過程を示す計画チャ
ートである．新QC七つ道具（N7）の一つとしても知られる手法
である．実際の活用事例では表記の仕方，また成功確率を高めるた
めの計画の立案だけでなく，事前のボトルネック技術の予測・発
見，進捗過程でそのつど実績や新たな情報を取り込むやり方や，4･
2･7･ a項で述べるデザインラショナーレと同様に，プロジェクトの
進行過程をチャート化して残すことにより，次のプロジェクトのた
めの問題点を摘出する用法もある．
そのために表記の仕方等多くのバリエーションが存在する．図
4･21にはPDPCの簡単な例を示している．これにはプロジェクト
の目的が最上段に指定された記号のもとで示され，最下段にゴール
が示されている．そして図右上には成功と判断するための前提条件
が書かれている．そして，この目的とゴールを結ぶまでの想定され
る実施事項，予測される結果，それに基づく判断（デシジョン）等
が列挙される．図4･21の場合には，当初三つの代替案とその優先
順位が示され，それぞれの結果と判断によって次に打つべき手，す
なわち実施事項が結ばれている．
新商品開発や設備開発では，プロセスに必要になるフローがある
程度想定できるものについては，多くの場合，このような図の横
（例えば左側）に工程フローを明示し，それとの関係でPDPCのチ
ャートを描く．想定されるリスクやボトルネックを事前に関係者が
描いて，それを特定，共有することによって，プロジェクトの短
縮・効率化や，何より途中で中止せざるを得ないというような状況
を最小限にしようというものである．
なお，TOCの中の思考プロセス（thinking process ）と呼ばれ
る手法の中にも，PDPCと類似した手法がある．前提条件ツリー
（PT:prerequisite tree）と呼ばれているものである．これはまず
目標を掲げ，それを達成するための前提条件がその下に列挙され
る．次に，その前提条件を達成するために中間目標が順序関係を含
めてその下に列挙され，同時に中間目標から前提条件に到達するた
めの障害が同時に書かれる．さらに，中間目標を達成するための手
段と想定される障害がその下に展開される．このように目標に到達
するための目標のブレークダウンとその達成に伴う障害やリスクを
明確にし，事前にそれに対して手を打つことによって確実な成功に
結びつけようというものである．
