# 4・3・5 ボーイング777開発における展開例

> **章**: 第4章 設計の管理
> **節**: 4・3 設計プロセスのコンカレント化

コンカレントエンジニアリングを大規模かつ本格的に最初に実践
した例として，有名なボーイング777（B777）の開発における実
施例について述べる ．
B777プロジェクトは，1990年にスタートした大型民間旅客機開
発のプロジェクトである．
B777は静粛性を損なうことなく燃費の革新的な改善を行うこと
と，コックピットと材料技術において最も先進的な技術を取り入れ
ることで，ボーイング社が社運を賭けて取り組んだプロジェクトで
ある．なお，B777はB767とB747の間に位置付けられる360～
390人乗りの双発旅客機である．このプロジェクトは機体自体の先
進性はさることながら，従来と全く異なる革新的な開発プロセスを
採用することで，今までのやり方に比べ60%のコスト（開発費総
額5000億円）で開発を実現しようとしたものである．このプロジ
ェクトはボーイング社と日本企業との共同プロジェクトであった
が，遠く離れたアメリカと日本でコンカレントに開発を進めるた
め，徹底的に三次元CADを利用しようとしたものである．
コンカレントエンジニアリングによる開発プロセス革新の最大の
眼目は，従来作っていた実物大モックアップを全面的に廃止するこ
とにあった．そのため，コンピュータ内に三次元のモデルを作るこ
とで，実験に代わる解析シミュレーションや仮想世界で組立性の確
認を済ますなど，最大限に活用することで，実際にモックアップを
廃止することに成功した．
a.どのように取り組んだか 戦略として，(１)製品のディジ
タルデータによる定義，(２)共通データベース，(３)共通ツールの
三つを掲げ，all digital airplane，すなわち飛行機1機分すべてを
ディジタルデータで作り上げることを約束（コミットメント）して
取り組んだ．そのための道具として，三次元CAD，検査用
（verification）ソフト，ディジタルモックアップ内部における人間
工学上のシミュレーションソフト，統合データベース，自動干渉チ
ェックや解析シミュレーション用のソフトなどを準備した．これら
の新たな道具を徹底的に活用するため，以下のような活動を行っ
た ．
(１)CPD（concurrent product definition，コンカレントプロ
ダクトデフィニッション)
通常，新製品開発において製品の性能やコストは開発の初期段階
でほとんど決まってしまう．この段階における業務の質が製品とそ
の後の開発業務の成否を規定するといって過言でない．CPDは開
発初期段階の作業展開の方法であり，設計部門だけでなく，製造，
資材，品質保証，サービスなど開発に関係する部門や企業はもちろ
ん，さらには顧客であるエアラインまで参加して最適な製品や作り
方を検討し決定する方法をとった．
(２)DBT（design built team，デザインビルトチーム)
CPDを効率的に行うための組織としてDBTを作った．機体の
部位，機能ごとにチームを分けて，それぞれの検討に必要なメンバ
を集めた部門横断的なチームを編成した．この組織を作った初期は
チームの単位が大きすぎ，むだが多かったというが，チームの単位
を小さくしてからはねらいどおりの活動が実現できたという．この
活動の結果，設計変更を大幅に減らすことができ，さらに設計品質
の向上に大きく貢献した ．
(３)CAD/ CAMの活用
上述の活動を支えるツールとして三次元CADを全面的に活用し
た．100 %データ化を目指し，ソリッド化対象部品は13万点にのぼ
ったといわれる．そのために大型のコンピュータ数台と2000台の
CAD端末が設置された．しかしながらこれだけ膨大なコンピュー
タリソースを準備しても，干渉チェックをリアルタイムでは行え
ず，夜間バッチ処理を行い，翌朝日本を含めた関係者にチェックの
結果が報告され，不具合があればただちに検討を開始，設計変更に
必要なフィードバックや調整が行われた．
b.何がチャレンジであったか この取組みにおいてテクニカ
ルな課題は全体の20%程度で，残りの80%はカルチャーを変える
ことにあったという．ここでいうカルチャーチェンジとは，
(１)以前こんな方法で仕事をしたことがない，
(２)われわれには十分な準備ができていない，
(３)変革するにはいまは忙しすぎる，
(４)十分にトレーニングされたオペレータがいない，
(５)今でも十分に効率的だ，
(６)すべてがうまくいっているのになぜ変えなくてはならないの
か，
(７)どうやっていいのかわからない，
といった抵抗に対し，理解を得ながら変革していくことを意味して
いるが，この作業は大変なチャレンジであったとしている．一方の
20%のテクニカルな課題としては，
(１)ソリッドモデルによる三次元設計をどうやるか，
(２)製品定義をディジタルにどう定義したらよいか，
といったことであった．当時三次元CADがまだ十分な機能や性能
を持ち合わせていなかったことを考えれば，これとて大変なチャレ
ンジであったと思われるが，そのチャレンジもカルチャーを変える
ことに比べれば些細であったということは，いかにカルチャーを変
えることが困難であったかという証左であろう．カルチャーを変え
るために，プロジェクトリーダは，チームメンバには以下のような
要求を行った．
(１)タイムリーかつ正確なフィードバック．
(２)チームの期待値に対する共通理解．
(３)共通のゴールに対するコミットメント．
(４)責任の分担と共有．
(５)オープン，正直，タイムリーなコミュニケーション．
(６)チームの成功が最重要であると全員が感じること．
(７)仕事に対して有益な情報と資源の提供を．
(８)チームメンバに対する相互信頼と関心を持ち合うこと．
(９)権威を与える，それに見合った説明責任の義務をもつ前提
で．
これらは本来日本人の得意としていたカルチャーであるが，この
ような日本的な強みを積極的に取り入れているのが特徴的である．
いくつかのサブグループに分けて開発を行ったが，それぞれのグル
ープのリーダには以下のような責任をもつよう意識的な動機付けや
教育を徹底した．
(１)具体例でリードする．
(２)タイムリーな方法で情報を共有する．
(３)絶えず目標を確認し他人の言うことに耳を傾ける．
(４)能力開発に努める．
(５)会社の理念，ねらい，目標，目的にコミットする．
(６)顧客満足に照準を合わせるよう継続的な改善，改良をリード
する．
(７)メンバに対し，フェアに，信頼して，尊敬をもって接する．
(８)障壁を排除し，チームワークの向上に努める．
c.効果 B777プロジェクトは，“working together”を合言
葉に，コスト削減のために設計変更を減らすこと，そのために
design built team（設計品質作り込みチーム活動），concurrent
production definition（関連部門の同時並行作業の展開）など新し
-
い考え方を導入し，よりよい開発プロセスを追求した．このような
取組みの結果，品質，コスト，日程，安全性，モラルのすべてにお
いて改善され，完成したB777は顧客から高い満足を得ることがで
きた．具体的な効果としては，B777の10年前に開発したB767
と比べ設計変更を3分の1以下に激減したこと，40%の開発コスト
の削減を実現したこと，そして計画どおりの日程で初フライトを成
功させたことなどである．この成功の要因は日本のものづくりを徹
底的に研究し，TQC（total quality control ）を全面的に取り入
れ，リーンエンジニアリングを忠実に実践したことにある．またコ
ンカレント設計を実現するため，当時はまだ確立されていなかった
三次元CADの導入に果敢にチャレンジし，プロセスのデジタル化
による将来のコンカレントエンジニアリングのあり方を指し示し
た，時代を画するプロジェクトであったといえる．
