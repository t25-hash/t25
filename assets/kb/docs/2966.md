# 4・4・1 製品系列の統合化における背景

> **章**: 第4章 設計の管理
> **節**: 4・4 製品系列の統合化と設計

図4･43 PDMの研究
a.共通化・共有化・モジュール化 設計がより優れた機械や
製品の実現を目指していることには疑う余地はない．それがさまざ
まな制約のもとにあることも自明であるが，それにはさまざまな基
盤設備や知識生産なども含まれていることまでには考えは及びにく
い．しかしながら，設計の内容がさまざまな意味で高度化してくれ
ば，その種の制約を考えておくことが優れた製品を提供していくう
えで重要な視点ともなる．異なる製品の間で一定の部分を共通化
（commonalization）したり，異なる企業間でそのような部分を共
有化（sharing）したり，あるいは，製品の一定の部分を独立させ
てモジュール化（modularization）したりする動向は，上記の視
点に関わるものである．それらの潜在的で本質的な意義は，複数の
製品すなわち製品系列（product family）を統合的に考えることに
よって旧来からの意味を超えた優れた設計を実現しようとする点に
ある．
本節では，共通化・共有化・モジュール化として象徴される方面
における設計に関して，典型的な実践例，概念的な意味，アーキテ
クチャという視点から展開される理論，統合化（integration）に
おける最適性やそれに向けての数理的な設計の方法論などについて
示す．
b.製品系列の統合化についての実践例 さまざまな機械の間
でそれらの構成要素を共通化しようとする考え方は，なんら特殊な
ものではなく，各種ねじの標準化などにも見られる．標準化のねら
いはおもに設計の効率化とコストの低減にあり，それらは製品系列
の統合化がねらいとするところでもある．統合化が標準化を超えて
改めて重要である理由は，それが機械や製品の性質そのものを左右
する主要な部分を対象としている点にある．なお，本節の以下の部
分では，機械の物理的な機能だけではなく価値やコストなどについ
ての内容をも考える必要があることから，設計の対象物を製品と呼
ぶことにする．以下の各製品は統合化が効果的に行われている実践
例である．
腕時計メーカのスウォッチ（Swatch）社は，独特のデザインを
もつ極めて多くの種類の製品を次々と妥当な価格で提供してい
る ．その背後には，高性能な時計の心臓部が極めて安価に手に
入るようになって以後，時計が正確な時間を刻むことは当たり前の
機能になって，意匠が製品を差別化するようになったことが大きく
影響している．つまり，文字盤や針，腕輪などの形状を統一的なも
のとすることによって，それらのもとで多様な意匠を組合せ的に展
開することが可能になっている．
民間輸送機は製造コストもさることながら開発コストが大きく，
新規の開発は容易ではない．さりとて，航空需要は年ごとに変化し
て，新しい機体に対する需要は生まれてくる．これに対応する一つ
の方策は，既存の設計を流用しつつ新しい設計を生み出すことであ
る．例えば，DC-9は胴体の伸張（ストレッチ）などを何回かにわ
たって行いMD-80に至っている ．また，ボーイング777におい
ては，1990年の開発開始の当初から，初期型に続いて，将来，胴
体のストレッチによる座席数の多いタイプや，やがて導入が見込ま
れる高性能エンジンを積んだタイプなどを展開することが計画され
ていた ．
自動車の開発方法においても2000年前後から同様の傾向が見ら
れるようになってきている．それについてはさまざまな要因が交錯
しているが，環境問題への対応や衝突安全への対応は開発コストの
増大に直結するものである．また，社会の成熟に伴って消費者ニー
ズは多様化するとともに付加価値に焦点が移ってきていて，開発期
間の短縮や迅速な多品種展開を要請している．少なくとも従来の価
格を維持しながら前者に対応するには，より大量の台数を生産して
車1台当たりのコストを圧縮すればよいが，それでは後者に対応す
ることはできない．両者への対応を同時に実現する方法が，車の全
体ではなく部分を共通化することによって，部分としては超大量生
産を実現しつつ，車自体としては多品種展開を行うというものであ
る．具体的には，企業内や企業グループ内でプラットホーム
（platform，車台）を共通化し，そのもとでさまざまな車種を順次
展開していくことが行われたり，ドアやコンソールなどについて
は，サプライヤが主体となり開発と生産を行うことによって，個別
のモジュールについての開発に関する知識を集約的に投入したりす
ることが行われつつある．
開発コストが膨大となる製品の典型はコンピュータ産業のそれで
あろうが，その実態は上記の2例とは趣が異なる．その形成期は別
として，目下のところ，コンピュータの全体を単独で製造すること
ができる企業は成立し得ない．標準化や共通化を超えて，開発に先
だってある種の規格が策定され，そのもとでさまざまな企業が各部
分についての開発と製造を行った後に，最終的には組立を担う別の
企業が最終製品としての取りまとめを行うようになっている ．
それに至る理由は製品開発に関わるコストと関与する知識が膨大で
あることにあり，分業を通じてそれらを分担し合うことを前提とし
て産業が成り立っている．
c .統合化の及ぼす副次的効果 b項の内容は製品系列の統合
化が著しい効果をもたらしている事例ではある．例えば自動車にお
いて，さまざまなオプションが用意されていたり，特定車種に異な
る排気量のエンジンを搭載するモデルがあったり，右ハンドルの車
と左ハンドルの車が並立していたりすることは，同様のことを小規
模に行っている事例とみなすこともできる．つまり，製品系列を統
合的に考えることは，製品を全体と部分の関係のもとで考える，す
なわち，製品を適切な部分に分けたうえで，それぞれの部分につい
ての設計と相互の関係についての設計をまとめあげることによって
全体の設計を成立させるということになっている．
以下のことも付随していて，それによって製品の柔軟性を増大させ
ることができる ．
(１)オプション追加：製品を多様化するために，基本的な製品に
(２)アップグレード・順応：システム全体の性能を向上させた
(３)サービス性（修理・補充)：製品の寿命を確保するために，
(４)使用における柔軟性：異なる条件下で使用するために，一時
(５)再利用：いっそうの経済性を確保するために，寿命を終えた
これらについての設計はそれぞれに重要な事項ではある が，以
下ではb項で示したような規模での統合化を主題とする．
化などと称される製品系列の統合化は特異なことではなく，ある種
の因果関係のもとで今日的な課題としておのずと表出してきたもの
である．統合化の本質的な意味を理解するためには，まず，その因
果関係を把握しておくことが重要である．突き詰めればものづくり
とは何かということになる．物すなわち道具の利用は人類を特徴付
けているが，ものづくりとなれば，その生産が組織的な産業を通じ
て行われていることが本質である．産業における一つの根幹は分業
によって労働力や設備，資金や知識などの資源を特定の対象に集中
させて効率を向上させることにあり，そのような効率向上が新たな
資源の集中を可能にする循環が再帰的に作用することによって，物
性があるということになっている．
て，人間の欲求は生存欲求，安全欲求，社会的欲求，自我欲求，自
己実現欲求へと段階的に発展していくとマズローが指摘してい
る ように，物への要求は可塑的であって，充足とともに高度化
し多様化していく傾向がある．つまり，産業としての可能性と市場
における必要性が相互に作用しながら，ものづくりの対象である製
品の複雑化や多様化，さらにはその価値の短寿命化は増進し続けて
いるということになる．言い換えれば，製品の複雑さと対峙しつつ
圧縮された時間の壁を乗り越えていくことは設計の管理における不
可避の課題であるということになる．
b.視野の拡大による最適性の拡大 a項の内容は，とにかく
製品には膨大な内容が関与していることと，それらを短期間で処理
することが必要であることを意味している．一方，設計には当然の
ことながら最適性（optimality）が求められる．この最適性の意味
を大まかに概念的に考えてみれば，以下のようになる．
(最適性)＝(最適性指標)×(操作範囲)×(探索コスト)
(4･1)
つまり，優れた設計を達成するには適切な指標を用意して，より多
くの可能性を調べ尽くすことが必要である．より適切な指標とは物
事をより大局的にとらえることを指し，例えば，個別の製品の性能
やコストを考えるのでなく，特定のプロジェクトによる利益や企業
のある期における利益を考えることを指す．そのような指標が関与
する範囲は当然のこととして拡大することから，対応して操作範囲
は拡大しなければ，指標に対応した設計解を探し出すことはできな
い．さらに加えて，指標を置換えて範囲を拡大したことによって生
じる膨大な代替案を調べ尽くすには相応のコストが必要となる．つ
まり，三つの要因がそろって初めて，優れた設計が達成できるとい
うことになる．裏を返せば，いずれかが不足すれば，得られる解は
部分最適にすぎず，全体最適は達成されないということになる．
しかしながら場合によっては，設計に許されるコストがそのレベ
ルに追いつかないこともありうる．製品系列の統合化においては，
b項のような統合化とは離れても，部分の設計を考えることには
むしろ，そういう場合のほうが多い．それを回避するための方策は
設計問題を解く際の構造そのものを組換える ということにある．
すなわち，事前に望ましい最適解を導き出す手順を想定することが
できれば，そのもとで構成される各部分設計問題のそれぞれは相対
付加的な機能を任意に追加する．
的には容易に解決可能であり，個別の設計問題から得られる結果を
統合することによって製品全体についての全体最適に準じる設計を
り，使用環境の変化に適合させるために，製品のある部分を
達成することが期待できるようになる．なお，実際の設計におい
入れ換える．
て，この種の問題解決構造を組換えるには，組織そのものの構造を
変革するなり，適切な戦略のもとで製品開発を進めるなりの対応が
摩耗するなどした部分を入れ換えたり，消耗したものを充て
求められるものと考えられる．
んしたりする．
