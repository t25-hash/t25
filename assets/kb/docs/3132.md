# 5・6・2 単一生産機械のヒューマンインタフェース

> **章**: 第5章 生産設備
> **節**: 5・6 ヒューマンインタフェース

工作機械等の画面の標準化については，次の議論がされてい
る ．特にオープンコントローラ（open controller ）用のヒュー
マンインタフェースについては，メーカ各社が独自のヒューマンイ
ンタフェースを作成したいところであり，NC画面の統一や標準化
は難しく，むしろ使いやすい画面（ヒューマンインタフェース）を
使用される環境に合わせて容易に作成することが可能であることが
求められている．単一NC工作機械のヒューマンインタフェース
については，ISO7000：1989，JIS B6012：1998で規定されてお
り，それらについては，一部差異はあるものの工作機械メーカが製
作する機械操作盤，NC，いずれの場合も基本的にはだいたい類似
したシンボルが使用されている．機能群別に見ると，シンボルは
NC装置関連，NC設定関連，自動運転関連，機能表示関連，座標
関連，座標速度関連，手動運動関連に分けられる．画面全体は一般
的に現在位置など各種状態を表示する部分，運転モードや表示ラン
プを表示する部分，運転モードに応じて切換える部分からなる．こ
こで規定されていないものについては，各社が個々に考案したシン
ボルが使用されており，今後統一化の必要性がある．また，画面を
容易に構築するための道具であるビルダの要望は強く，これについ
ては，操作が簡単であること，拡張性が高いこと，FAコントロー
ラ機能のオブジェクト形式で抽象化可能であること，抽象化された
機能のNC部の明確な標準API （application program inter-
face）が定義可能であること，ビジュアルツールによる直感的な画
面作成が可能であること，ビジュアルツールに使用するシンボルや
用語が標準化されていることなどが要求されている．
