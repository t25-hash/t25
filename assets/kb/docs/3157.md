図5･14 機能固定方式と機能成長方式
多いが，ソフトウェアの設計も重要な分野である．ハードウェアの
設計では機能固定方式（fixed function system）の設計がまだまだ
多いのに対して，ソフトウェアの設計では機能成長方式（growing
function system）の設計が主流となってきている（図5･14)．
ソフトウェアの分野でも以前は，ハードウェアと同様に機能固定
方式であり，製品の機能は設計時に決定され，その機能をもつ製品
が出荷された．しかし，知識工学の導入以来，ソフトウェアは，顧
客のもとへ届けてから顧客とともにその機能を成長させる方式へと
変換した．
ハードウェアの世界では，まだ機能成長方式の製品実現は十分に
普及していないが，これまでの設計時の機能を出荷後も単に保持す
る保守から，ライフサイクルにわたりその状況に応じた適切な機能
を実現する方式へと急速に変化しつつあり，保守もこうした適応的
に機能を発現できるためのインテリジェントな活動へと変化してい
る．
また，ネットワーク型へ移行した当初は，各ノードがもつ情報が
重要であり，それらを結ぶリンクは情報交換の機能のみしか考慮さ
れなかった（図5･15)．しかし，次第にリンク自体が新しい価値を
生む，すなわち情報を交換することで，新しい価値を創造できるこ
とが明らかとなってきた．こうしたリンクのもつ価値に注目したネ
ットワークを，バリューチェーンネットワーク（value chain net-
work）と呼ぶ（図5･16)．
これにより，最終製品機能のみを価値として考えてきた従来の考
え方を超え，それに付随するプロセスをサービスとして価値化でき
る可能性が生じてきた．このような視点から，新たにサービス工学
が提案され，注目されている．
したがって，新しい時代のよい設計とは，ライフサイクルにわた
り，顧客を満足できる優れた適応性と，循環社会に適合する機能，
品質をもつ製品を実現し，また，製品実現に付随する各種のサービ
スを最大とする，プロダクトとプロセスの両面に適切な配慮を払っ
た設計である，ということができよう．
