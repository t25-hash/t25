# 6・1・4 データ収集装置

> **章**: 第6章 各種応用計測
> **節**: 6・1 電気計測器

a. 概説データ収集装置(data logger)は、電気信号や電気信号に変換された工業量・物理量などを測定し、アナログ波形もしくはディジタル値として記録する機器で、大別するとチャートレコーダ、PCフロントエンドデータロガー、ペーパレスレコーダに分けられる。最近のディジタル技術の進歩は著しく、特に1980年代のマイクロプロセッサ、1990年代のPCおよび1990年代後半のコンピュータネットワークの普及に伴い、データ収集装置の形態をアナログからディジタル、遠隔集中監視へと大きく変ぼうさせている。データ収集装置の種類と変遷を以下に示す(図6・11)。. チャートレコーダ(chart recorder)チャートレコーダは1950年代初頭に電子平衡式記録計が初めて国産化されたが、記録紙上にアナログ波形として信号を記録するものであった。1980年代にマイクロプロセッサが採用され、記録紙上へのディジタル印字記録や、表示器上への信号表示が可能になった。入力回路や記録機構およびチャートレコーダ内部でのデータ処理方法等の改良を重ね、現在でも多くの領域で工程監視、品質管理等の目的で使用されている。. PCフロントエンドデータロガー(PC front end data log-ger)PCフロントエンドデータロガーは、PCの普及に伴い発達したデータ収集装置で、コンピュータ通信を経由しPCにデータを転送、PCで記録を行うタイプである。一部、データロガーに記憶装置を内蔵し、IC(integrated circuit)メモリカードのような図6・11 データ収集装置の変遷図6・12ペーパレスレコーダ機能ブロック図図6・13 ペーパレスレコーダ外観記憶媒体を経由してPCにデータを移動する方法も採用されている。電子データでデータを管理することにより、PC上のソフトウェアを使いさまざまなデータ処理に柔軟に対応することが可能で、記録紙等の保管場所をとらない利点がある。特に1990年代後半のコンピュータネットワークの発達により、遠隔多点の測定、記録に威力を発揮する。. ペーパレスレコーダ(paperless recorder)チャートレコーダとPCフロントエンドデータロガー双方の利点を併せ持つタイプで、表示器に液晶ディスプレイを採用している場合が多い。チャートレコーダのデータ視認性と、電子データの利便性を活用したタイプが1990年代末に登場している。特に企業や研究団体等において既存のインフラストラクチャとして定着したLAN(localarea network、ローカルエリアネットワーク)や、インタネット等のWAN(wide area network、ワイドエリアネットワーク)を活用した遠隔・分散・多点の測定・記録が多く行われるようになった。図6・12にペーパレスレコーダの機能ブロック図、図6・13にペーパレスレコーダの外観を示す。b. データ収集装置と情報技術データ収集装置は近年のいわ5―123図6・14 ネットワークへの接続概念図ゆる情報技術により遠隔・分散・多点監視・記録が可能になった。インタネット・イントラネットのインフラストラクチャが普及し、技術者のデスク上にあるPCからさまざまな情報を入手可能になった現在、データ収集装置のデータをPCから集中監視する例が急増している。ネットワークのデファクトスタンダード(事実的標準)となったイーサネットインタフェース、TCP/IP(transmission controlprotocol/internet protocol)プロトコルを搭載し、データの自動転送やウェブ画面での監視を行えるデータ収集装置が登場するに至り、データ収集装置はネットワーク対応データターミナルという扱いを受けるようになっている。図6・14にデータ収集装置のネットワークへの接続概念図を示す。測定データをディジタルデータとして扱うことによる利点は大きいが、より簡単に高度な処理(解析)を行うためには、データ処理を行うPC上の処理ソフトウェアが重要である。測定データの検索性や他のソフトウェアとのデータ共有化などを支援する、ヒューマ5―124計測工学ンマシンインタフェース(HMI: human machine interface)の研究・開発が望まれている。図6・15にHMIの例を示す。c. データ収集装置の今後データ収集装置は今後、電子化がさらに進むことは必至であるが、大きく分けて二極化すると思われる。一つはローカルにおけるデータ収集で、ネットワーク等を介さず、記録紙または磁気記憶装置等による簡易データ収集である。これには簡便性が強く望まれ、バッテリ駆動や省サイズの特徴をもつ。紙は今後も重要な記憶メディアとして生き続けるであろう。もう一つは、ネットワーク経由・遠隔・分散・多点の大規模データ収集システムである。データの集中化が図られるため、信頼性・冗長性が望まれるが、上位システムの負担を減らすために、現地分散処理を行う方向へ進むであろう。このような大規模・分散システムを構築する場合は、特に負荷分散や時刻の同期性に注意する必要があることを付け加えておきたい。
