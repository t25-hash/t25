# 1・1・2 状態方程式

> **章**: 第1章 システム解析
> **節**: 1・1 システム表現

a. 状態方程式(state equation)伝達関数がシステムの動特性を入力と出力の間の直接的な関係で表現するのに対し、状態方程式は、入力と出力の間にシステム内部のふるまいを記述する変数を介在させ、間接的な関係で表現する。状態方程式は、その表現自体が因果律を満たす構造になっているので次元間の制約はないが、モデルとしての冗長性を排除するとき、通常>,>が成り立っている。式(1・14)(1・15)を合わせて状態方程式と呼ぶこともあるし、式(1・14)を状態方程式、式(1・15)を出力方程式(output equation)と区別して呼ぶこともある。このことから、状態変数はシステムの未来の挙動を決定するのに必要かつ十分な過去の情報をもつ変数であるということができ、その次元はこの特性から定まる。状態方程式表現は微分方程式表現の一種とみなせるが、その特徴は、形式的には高階の微分係数を用いないで1階の連立微分方程式で表現していること、構造的にはシステム内部のふるまいを記述する状態変数を介在させていることにある。また、伝達関数表現に比べ、多入力多出力のシステムや時変システム、非線形システムも容易に表現することができる。入力の微分項をもつ場合を含めて、より一般的な場合の取扱いについては、1・2節を参照されたい。なお、式(1・20)のの係数行列の形(あるいはその転置)を、同伴型(companion form)と呼ぶことがある。b. 非線形システムの線形化実際のシステムでは、その動特性が非線形であることが多い。しかし、非線形システム表現のままで解析や設計を行うには、ロボット制御などの一部を除いて、理論が整備されていない。このため、しばしば線形近似によって線形化システムが作成される。線形化(linearization)の過程では、単なる数学的な変換だけでなく、工学的な視点も重要である。そこでまず、図1・5のタンクシステムを例にして、線形化の過程を具体的に見てみよう。いま、水槽(タンク)の下流へ一定量の給水をするという状況を考えてみる。この場合、水槽を介さずにパイプから直接に給水すると、パイプ内の水圧変動が給水量に影響を及ぼすおそれがある。こ6―3図1・5 タンクシステムのタンクシステムは、水槽の水位を一定にすることでそのような影響の軽減を目的として用いられるものとする。この非線形システムを線形化するために、まず動作点を考える必要がある。線形化システムは動作点での線形近似によって得られ、動作点が変われば線形化システムも異なったものとなる。平衡状態は、平衡点(equilibrium point)ともいい、各変数が一定値で変化がない状態のことである。これが、式(1・24)(1・25)の非線形システムを線形化したシステムである。この線形化システムについての留意点を挙げておこう。さて、状態方程式(1・18)(1・19)で記述された一般の非線形システムに対して、線形化システムを導いておこう。このとき、システムが可制御かつ可観測(1・2節参照)ならば、特性多項式の次数は状態変数の次元に一致し、(初期値を除いて)互いに等価なシステム表現になっている。この逆に、伝達関数から状態方程式表現を求めることを実現(realization)という。これについては1・2節を参照されたい。ところで、多入力多出力の状態方程式に対しても伝達関数が定義でき、やはり式(1・37)で与えられる。一方、多入力多出力システムの零点にはいくつかの定義がある。このほかに伝達零点などがあるが、詳しくは文献(3)を参照されたい。この変換は、数学的には次元ベクトルで張られる空間(状態空間)の基底座標を変換することにより、同じベクトルの座標表現がからに変わることを意味し、状態変数変換あるいは座標変換とも呼ばれる。このとき、式(1・14)(1・15)と式(1・39)(1・40)のシステムは互いに相似(similar)であるという。相似なシステムは、同じ入力を加えると同じ出力を生じるという意味で等価であり、状態変数の選び方には依存しない。また、相似なシステムでは、伝達関数、極、不変零点が一致し、可制御性・可観測性も不変である。また、パラメータは時間刻みの順番で、サンプリング周期がであるサンプル値システムでは時刻を意味する。
