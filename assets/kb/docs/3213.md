# 1・1・3 中間標準形式

> **章**: 第1章 システム解析
> **節**: 1・1 システム表現

a. 中間標準形式(intermediate representation)中間標準形式はもともと、集中定数素子からなるシステムを状態方程式表現するための前段階として、素子の特性表現と結合関係を羅列したシステム表現で、これが名称の由来になっている。状態方程式表現と本質的に異なるのは、正方行列が正則ではないことである。もしが正則ならば、式(1・44)の両辺に左側からを掛ければ状態方程式になる。すなわち、中間標準形式は1階の微分方程式だけでなく、代数方程式も同時に含んだシステム表現である。これは、中間標準形式が因果律を満たさない微分方程式も含みうることを示している。したがって、中間標準形式から状態方程式につねに変換できるとは限らないし、式(1・44)を満たす解が存在することの保証もない。このような突発的なふるまいはインパルスモードと呼ばれ、状態方程式の解には現れないものである。b. インパルスモード(impulsive mode)インパルスモードを詳しく調べるために、中間標準形式を解析しやすい形式に書き換えよう。これに対し式(1・53)は、中間標準形式独特のふるまいであるインパルスモードを呈する部分である。このように、≠0のとき解(1・56)はデルタ関数やその微分を含み、このダイナミクスをインパルスモードという。これは式(1・51)で=0の場合に相当し、≠0だからインパルスモードをもつ。したがって、当然ながら、式(1・57)式を状態方程式に変形することはできない。インパルスモードをもたない中間標準形式からは状態方程式を導くことができるが、この手順については文献(6)を参照されたい。
