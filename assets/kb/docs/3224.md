# 1・4・1 システム同定とは

> **章**: 第1章 システム解析
> **節**: 1・4 システム同定法

システム同定(system identification)とは、対象とする動的システム(プラント)の入出力データの測定値から、ある目的のもとで、対象と同一である、なんらかの数学モデルを構築することをいう。このとき、目的、同一である、数学モデルの三つの単語がキーワードになる。まずたいせつなことは、何のためにシステム同定を行うかという目的である。おもだった目的を列挙すると、制御系設計、異常診断/故障検出、モデルに基づいた計測、適応信号処理などがある。このように、システム同定は最終目的ではないことに注意する。以下では、この中で制御系設計を目的としたシステム同定を取扱う。次に、同一であるということは、identificationの名の由来である。一般にプラントと同一のモデルを作成することは不可能なので、制御系を構成するうえで重要な特性がモデルに盛り込まれているとき、同一であるとみなす。このとき、モデルに含まれなかった動特性はモデルの不確かさと呼ばれる。最後に、制御系設計で用いられる数学モデルの代表例は、伝達関数、周波数伝達関数やステップ応答などであり、どのような数学モデルを利用するかは、システム同定法と制御系設計法の双方に依存する。線形制御系設計の発展とそれに必要とされるモデルの関係を表数学モデルと設計仕様が与えられれば、設計者の能力に依存しない標準的な制御系設計が行えるようになってきた。このとき、最も重要なものが高精度な数学モデルであり、システム同定の重要性が再認識された。しかしながら、システム同定には多くのノウハウが必要であり、必ずしも標準的な方法が確立されているわけではない。すなわち、システム同定を行う技術者のわざ(art)に頼る部分が残っているため、対象となるプラントやシステム同定の目的に応じて、さまざまなシステム同定法が存在する。システム同定の基本的な手順を表1・2にまとめた。これらの手順について以下で説明する。
