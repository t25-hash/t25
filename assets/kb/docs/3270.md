# 6・2・3 CAMシステムの変遷

> **章**: 第6章 情報システム
> **節**: 6・2 生産準備と情報システム

次に，NCプログラム作成作業を含めた工程設計業務へのコンピ
ュータ適用の変遷を見ることにする．
現在までのCAMシステムの変遷を見ると，それはNCプログ
ラム作成システムの変遷といっても過言ではないと思われる．それ
を三つの時代に分けて追って見ることにする．
a.言語型バッチ処理システムの時代 NCプログラム作成シ
ステムの起源は，1950年代にアメリカで開発されたAPT（auto-
matic programming tool ）である．これは，数値制御工作機械
（以下，NC工作機械と略す）の開発とほぼ同時期に開発された
NCプログラム作成用の言語で，APT言語と呼ばれている．その
おもなねらいは，手計算が不可能である自由曲面を含む複雑形状部
品のNCプログラム作成であった．このAPTは，幾何形状の定
義，形状に対する工具の動かし方の指示などの機能を有しており，
その後のNCプログラム作成システムの基本形ともなった．その
例として，現在でも使用されているAPTのCL（cutter loca-
tion）データ様式に関して以下で紹介する．
NCプログラムは，NC工作機械の種類ごとにすべて異なる様式
を要求されることが大半である．これに対応するために，CAMシ
ステムからはNCプログラムを直接出力せずに，まずCLデータを
出力して，ポストプロセッサ（post processor ）が各NC工作機械
向けのNCプログラムを作成する方法が採用されている場合が多
い（図6･7）．このCLデータの様式として，APTで定められた様
式が，現在でも使用されていることが多く見受けられる．
APTの後にドイツでEXAPT（extended APT）が開発された．
APTが工具軌跡作成のための数値計算処理が主体であったのに対
して，EXAPTは切削条件設定や工具選択など加工作業の自動設
定に主眼をおいたシステムであった．ただし，その対象は，2.5次
元までのフライス・穴あけ加工，旋削加工である．この2.5次元加
工とは，切削時には同時に二軸までしか使用しない加工をいう．
なお，この時代は多くのCAMシステムは大型コンピュータを使
用したバッチ処理で使用されていた．
b.対話型システムの時代 コンピュータが，ミニコン，EWS
（engineering workstation）などと進化するにつれて，リアルタイ
ムで出力を手にすることができる対話型システムへと移行した．当
初は先に述べた言語型が主流であったが，次第に言語を使用しない
メニュー選択方式が普及するようになった．また，NCプログラム
作成システム専用のコンピュータも出現し，さらには，NC制御装
置の前で対話型のNCプログラムの作成を可能とする対話型NC
が登場するまでに至った．
c.CADデータ活用の時代 CADが普及するにつれて，CAD
で作成された形状データをCAMで利用することが進んだ．この対
象は，おもに自動車のボデーや金型部品に代表される曲面加工であ
った．
が急速に普及したが，そこで作成されたCADデータのCAMでの
活用は，それほど進められなかったと見える．その大きな理由とし
ては，生産側で図面データから加工部位を探して加工部位の仕様な
どをCAMに入力するための作業量が多く，二次元CADで作成さ
れた一部の形状データを活用できても，大きな効果が出せなかった
ことが挙げられる．
図6･9 加工形状特徴を活用した設計・生産部門におけるデータの流れ
は，手計算では不可能な複雑な形状に対するNCプログラムの作
成作業であり，2.5次元加工に関しては，一部で加工技術データベ
ースを活用した加工作業の設定作業の自動化を実現しているシステ
ムも存在するものの，CAMの主流であったとはいいがたい．
