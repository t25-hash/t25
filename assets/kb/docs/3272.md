# 6・3・1 企業内情報システムの統合の流れと全体図

> **章**: 第6章 情報システム
> **節**: 6・3 企業内統合情報システム

a.企業内システムの抱えている課題 企業内情報システム
は，生産管理，生産現場支援，販売管理，設計支援，会計計算等々
の日常業務のルーチンワークを情報システムに置換えることにより
発展してきた．これらの情報システムは，開発投資に対して，何人
の省力化ができるかという観点で投資効率が測定され，個別に導入
および機能強化が行われてきた．その結果，企業内情報システムは
個々の機能ごとの効率化を目指した情報システムが乱立し，それぞ
れの情報システムの間がつながらない問題を抱えている．さらに，
個々の情報システムが，現場の要望に応じてシステムの機能強化を
行ってきたため，当初は整合性がとれていた各情報システムの内部
構造が複雑になり，機能強化が容易に行われなくなってきた．
b.情報システムの企業内での新たな位置付け 新しい時代の
企業内統合情報システムには，個々の省力効果以上に，経営の道具
としての機能が求められている．それには，変化の早い環境にも対
応するスピード経営を実現するために，受注，生産実績，部品納入
実績，在庫状況，販売実績，販売見込み，債権回収状況等々の企業
運営に必要な情報が即時に提供されることが必要となる．つまり，
企業内統合システムは，従来の月末等の締め日までに必要なデータ
を蓄積して企業活動の後処理をするだけではなく，システムの提供
する情報に基づいて企業の経営活動を支援できることまで求められ
るようになってきた．
さらに，企業内情報システムは，情報による企業間統合の支援を
求められるようになってきた．これは，競争の場が顧客主導の市場
に移り，これまでのように個々の企業内生産活動だけで十分な競争
