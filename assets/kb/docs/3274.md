b.生産・販売のための企業内統合情報システムの構成要素
生産・販売のための企業内統合情報システムは，大きく図6･26の
ような構成からできている．まず，市場の需要と経営計画を反映し
た長期生産計画を立て，それに受注状況を反映して基準生産日程を
作成する．それに基づいて資材所要量を計算し，購買情報と生産日
程計画を作成する．
生産計画を作成する中核となるMRPには，狭義のMRP（資材
所要量計画）と広義のMRP（クローズドループMRP）がある．
狭義のMRPは，図6･27のような機能をもち，基本的には，独立
需要，従属需要という考え方を導入し，すべてのワークオーダ，購
買オーダの実行計画（何を，いつ，どれくらい）の決定を可能にし
た．MRPが正しく機能するには，正確な在庫把握（製品，仕掛
品，部品，素材）と正確な部品表（生産部品表）が大前提となる．
これらの精度に不安があれば，MRPは架空の計画を作ることにな
る．
クローズドループMRPは，狭義のMRPを内部構成として含
み，プライオリティプランニングとキャパシティプランニングを結
合している．プライオリティプランニングとそれに基づくキャパシ
ティプランニングによる計画が実行され，その結果に計画との差異
が生じれば，ただちにその状況を計画に反映する機能をもつ．クロ
ーズドループMRPシステムの機能概要を図6･28に示す．なお，
プライオリティプランとは，何を，いつ，いくつといった製造・調
達すべきものの数量と時期を計画するものである．具体的には，生
産計画，基準生産計画，資材所要量計画，手配計画，発注計画，外
注計画などになる．
プロダクションプランニングは，プランニングの最上位に位置
し，ビジネスプランで示された利益，在庫水準，生産性などの経営

諸活動を規定し，製品モデルや製品グループの期間ごとの生産計画

f.ERP（enterprise resource planning) ERPパッケージは

れにより，社内の情報伝達のスピードおよびその精度が，従来と比
べると格段に向上する．
従来，情報システムを開発および導入する場合，企業の業務の仕
組みに合わせて開発をしていた．しかし，ERPはベストプラクテ
ィス，あるいはグローバルスタンダードを内包していると主張し，
業務に合わせて企業内情報システムを開発するのではなく，ERP
に合わせて業務自体を変えることを要求している．もちろん，業務
を変えられない場合は，追加開発で業務にERPを近づけることも
可能であるが，随時機能強化されていくERPのメリットをタイム
リーに生かすことができない．したがって，ERPを導入する企業
は，自社の業務上のニーズとERPが求める業務プロセスを合わせ
込むことが必要となり，これが導入上の最大の壁となる．さらに
ERPはリアルタイムシステムであるため，情報と物が一致するこ
とが要求され，業務遂行結果をリアルタイムに情報システムに反映
する必要があり，これも導入上の大きな壁となる．しかし，ERP
に必要な情報が蓄積されると，いつでも最新の情報を入手できるた
め，企業内の各部門で他部門との交渉用にデータを隠して開示のタ
イミングを制御するなどができなくなり，よりフラットでフレキシ
ブルな組織運営が可能になる．つまり，ERPを導入するというこ
とは，単に企業内の情報を統合化することのみならず，組織間の関
係も変えることになり，経営陣がERP導入・実施計画に直接かか
わることが必要となる．
g.CRM（customer relationship management ) CRMは，
情報システムを活用して企業の顧客志向を強化する機能である．市
場が成熟化してきている中で，市場制御の主導権は急速にカスタマ
側に移ってきている．そうした中で企業は，見込生産による市場へ
の押込みを主体とする販売方式から，カスタマからの受注により生
産および納入を行う市場要求をとらえた方式へ切替えていく必要が
ある．そのように対応していくことにより，売れない在庫をため込
まずに生産することが可能となり，利益確保を行うこともできる．
つまり，大量生産・大量販売の時代から，カスタマのニーズを個人
の単位で把握し，カスタマが必要とする商品を，必要なときに，必
要な量を生産し，販売することが求められており，それを支援する
のがCRMである．
CRMの機能としては，顧客情報管理，営業活動管理，営業機会
管理，顧客取引管理，顧客要望管理，営業報酬管理などがある．新
しい企業内統合情報システムの視点では，二つの大きなポイントが
ある．一つは，顧客接点の強化である．これは，顧客からの情報を
見込みであろうとクレームであろうと大切に扱い，企業内のさまざ
まな活動につなげるように情報を他の企業内システムに連携する．
もう一つのポイントは，すべての企業内情報システムおよび企業内
活動を顧客接点に対する影響度および貢献度の視点で再設計するこ
とである．すべての企業活動は，顧客接点で始まり，顧客接点で終
わる．CRM情報システムは，図6･32にあるように，顧客接点と
企業内システムをつなぐ役割をもち，今後，よりいっそう重要性を
増していく．
h.SCM（supply chain management ) SCMは原材料調達，
生産，物流，販売という一連の企業活動の中で，顧客へのサービス
を維持・向上させつつ，抜本的業務の効率化，在庫圧縮，および原
材料調達から製品の生産，販売までの時間短縮を実現する機能であ
る．SCMにより，企業の壁を超えて，計画，調達，出荷の一覧の
情報と活動を連動させる．つまり，売れる商品を素早く過不足なく
市場に供給し，かつ売れない商品は速やかに生産を止めることによ
り，機会損失を最小化させるとともに在庫の最小化を実現し，資金
回収の期間短縮を進めることで，キャッシュフローの改善を行う
（図6･33）．これらを効率良く実現するには，CRMからの鮮度の高
い情報が必要となり，SCMはその情報に感度良く対応できる生
産・調達・物流を実現する仕組みである．
SCMでは，納期回答期間，納期遵守率（要望納期遵守率，回答
納期遵守率）の測定および向上，さらに在庫削減（あるいは在庫回
転率の向上）を代表的な成果指標として，業務改革と対となって企
業内の情報を統合化して最適解を導いていく．SCMは，図6･34
のように，いくつかの製品系列の中で，需要予測と供給能力を考慮
