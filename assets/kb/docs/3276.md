b.業務プロセス（business process ) 企業間統合情報システ
ムによる情報共有により企業間連携が実現できる．企業間連携で
は，単に業務間をつなげたり，適用する業務の範囲を広げるにとど
まらず，相互に関連した形に業務プロセスを統合していく必要があ
る．したがって，企業間統合情報システムを利用した企業間連携の
場合，どの業務プロセスを対象にするかという対象業務プロセス
と，どの程度連携するかというプロセスの連携度合いを考えるべき
である．これにより企業間統合情報システムによる業務プロセスの
多くの形態が可能となる．
.対象業務プロセス 企業間統合情報システムの対象とす
る業務の範囲は，製品のライフサイクル全体では，その企画から廃
棄に至るまでに，おおむね以下のような業務がある．
･企画・開発業務
･生産・構築業務
･運用・支援業務
･撤去・廃棄業務
企業間統合情報システムの対象範囲を，どの業務プロセスにする
か，またはどのように特定の業務プロセスから徐々に適用範囲を拡
大していくかにより，企業間統合情報システムの実現形態が異なる
可能性がある．
.プロセスの連携度合い プロセスの連携度合いは，おお
むね，連係（つながりの意）から連携（ともに手を携えてものごと
を行うことの意）への段階が考えられる．企業間統合情報システム
として見た場合，連係の段階では，企業間の接点を明確にし，そこ
でのオンラインによる技術情報の受渡しを効率良く行う実現形態が
必要である．連携，統合と連携程度が深まるにつれて，企業間で協
調して業務を遂行するために，業務プロセスの標準化やBPR
（business process reengineering）が必要になる．企業間統合情報
システムとしては，利用者が企業間統合情報システムと企業内シス
テムを別システムとして意識せず，最終的には一体となったシステ
ムとして実現する形態が必要になる．

統合化のパターンは，ライフサイクル全体におけるさまざまな業

c .システム構築 企業間統合情報システムの構築においては，
複数の企業・業種との接続を考えるうえでの相互接続性の側面，シ
ステム構築の容易さ，アプリケーション依存性，機能レベル等に関
わる情報機器の利用の要件を明確にしておく必要がある．
.相互接続性（interoperability) 企業間統合情報システ
ムの構築において，複数企業を前提に考えた場合，その相互接続性
は重要な問題である．ここで考えるべき相互接続性は，ネットワー
クプロトコルレベルの接続にとどまらず，企業間統合情報システム
が相互に接続し，その機能を実現できることが必要である．接続対
象となる企業の増加に伴って，以下のような段階が考えられる．
･専用線＋独自インタフェース＋独自フォーマット
･インタネット＋独自インタフェース＋独自フォーマット＋データ
変換
･インタネット＋標準化された企業間統合情報システム間のインタ
フェース＋標準フォーマット
当然，これらの中間的な段階も存在する．
.情報機器の利用 企業間統合情報システムの構築におい
て，どの程度情報機器を利用するのか，どの程度情報機器の機能に
依存するのかは，構築のための期間・コストや，各種の制約・要
求，求められる機能のレベル等により異なる．高機能な情報機器を
利用して，企業間統合情報システムを構築する場合は，利用する機
能や使い方，標準化への対応動向等を十分考慮する必要がある．
