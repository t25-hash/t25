図6･50 OSI参照モデル
図6･51 ORBの機能
request broker ）を利用した分散オブジェクト技術が活用されてい
る．ORBは，ネットワーク環境に分散するオブジェクトへのアク
セスを実現するメカニズムの総称であり，具体的にはCORBA，
JavaRMI ，EJB，DCOMなどの仕様が用いられる．
分散オブジェクト技術の主要機能は，図6･51のような分散環境
下でのオブジェクト（アプリケーション機能単位）呼出しの実現で
ある ．図6･51のようにクライアント側は，オブジェクトのネッ
トワーク上での位置を特定しなくても，ローカルなオブジェクトと
同じようにそのメソッドを呼出すことができる．一方サーバ側は，
あたかもローカルなオブジェクトのように，外部アプリケーション
から呼出し可能なオブジェクトを実装できる．その実行には，呼出
されたオブジェクトを保有するサーバ位置の特定や，送受信メッセ
ージの組立と通信など複雑な処理が必要であるが，オブジェクト呼
出しの仲介役であるORBがその処理を担当する．また分散オブジ
ェクト技術では，単なる遠隔オブジェクト呼出しのみならず，セキ
ュリティ，ネーミング，トランザクション，イベント等の付加サー
ビスを提供する．ORBの利用により，組織やシステムの変更に対
するアプリケーションの柔軟性，オブジェクトベースでのアプリケ
ーションの機能拡張性・規模拡張性を確保できる．
b.分散オブジェクト処理のためのミドルウェア CORBA
（common object request broker architecture，コルバ）はOMG
（object management group）で標準化が進められている分散オブ
ジェクトのミドルウェア仕様であり，最も普及したORBである．
図6･52にCORBAのアーキテクチャを示す ．CORBAでは，オ
ブジェクトを呼出すときに使われるインタフェースなどの情報を
IDL（interface definition language）という言語を使って記述し，
これをアプリケーション開発言語（Java，C＋＋）へIDLコンパ
イラにより変換し，アプリケーションと組合わせて利用する．
CORBAの主要構成要素は，①ORB，②ネーミングやトランザク
ション等のシステムレベルサービス群からなるCORBAサービス，
③印刷やメール機能など抽象度の高いサービスを提供するCORBA
ファシリティ，④医療・通信・製造等の産業分野別のドメイン・イ
ンタフェース仕様の四つからなる．生産システム関連では，生産設
備のデータ収集や時系列データ管理，分散シミュレーション，
PDM/ PLMに関するインタフェース仕様の標準化が検討されてい
る ．
CORBA以外のORB仕様としては，Java言語アプリケーショ
ンにおいてメソッド呼出しをネットワーク上で実現するJavaRMI
（Java remote method invocation)，Java言語でプログラム部品
を作成し，それらを相互接続してアプリケーションソフトを構築す
るためのJavaBeans ，ネットワーク分散型ビジネスアプリケーシ
ョンのサーバ側の処理に必要な機能を追加したEJB（enterprise
JavaBeans )，Windows OS上のアプリケーション間でソフトウェ
ア部品（COMオブジェクト）どうしがネットワークを通じてデー
タの交換や処理依頼のやりとりなどを行えるメカニズムである
DCOM（distributed component object model ）なども利用されて
いる ．またCORBAとCOM間など，異なるORB環境間での
相互運用も可能である．
さらに，もともと軍事用の分散シミュレーション用IEEE標準で
あるHLA（high level architecture）とそのミドルウェアである
RTI （real -time infrastructure）を利用した ，製造ラインの分
散協調シミュレーションも試みられている ．
