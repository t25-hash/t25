a.データベースシステム（data base system) 生産情報シス
データベースシステムは，共有データの集合であるデータベース

をもった専用システムソフトウェアである ．さらに特有機能と

データベースシステムのおもな構成を図6･54に示す．ユーザイ
ンタフェース部はデータベースを利用する部分であり，アプリケー

guage）を利用し，論理データ部に対するデータ検索や更新を行
う．論理データ部はデータベースが管理するデータの論理的な構造
を，データモデル（data model ）に基づいて記述する．データモ
デルによって記述されたデータの論理的構造の定義は，スキーマ
（schema）とも呼ばれる．また物理データ部は，データベースが実
現されている記憶媒体上の物理的な操作を担当する．データベース
システムを複数計算機上に実装し，ユーザ側からは見掛け上一つの
システムとして扱うことが可能な分散データベースシステムも一般
的である．
b.データモデル データモデル（data model ）は，その表現
形式により階層モデル，ネットワークモデル，関係（リレーショナ
ル）モデル，オブジェクトモデルなどに分類されるが，柔軟性・操
作体系の統一性・理解のしやすさなどの理由から，現在最も普及し
ているのは関係モデル（relational model ）である．関係モデルは，
図6･55のようにデータを関係表（relation）という二次元配列で
統一的に表現し，関係表の集合としてデータベースを表現し，データ
操作を関係表間の関係演算と集合演算に帰着させたものである ．
関係モデルに基づいたデータベースとその管理システムは，それ
ぞれ関係データベース（relational data base），関係データベース
管理システム（RDBMS:relational data base management sys-
tem）と呼ばれる．代表的なRDBMSとしてはOracle，DB2，
Access等がある．また関係データベース向けのデータベース言語
の標準として，SQL（structured query language）が広く用いら
れている ．
また生産システムにおいて管理すべきデータの特性からデータベ
ースシステムを分類すると，表6･6のようにトランザクション
（transaction）型，バルク（bulk）型，アーカイブ（archive）型
の3種類となる ．バルク型データの取扱い，製品構成情報や設
計変更情報のような構造が柔軟に変化する情報の管理，長いトラン
ザクションへの対応などが生産システム用DBでは求められるが，
上述のような関係データベースのみでは不十分な場合も多く，ユー
ザアプリケーションにより機能を補って実装されることも多い．
