# 4・1・3 ロバスト適応制御(robust adaptive control)

> **章**: 第4章 適応・学習制御
> **節**: 4・1 モデル規範型適応制御系(MRACS)

適応制御システムは、フィードバック制御器内の可調整パラメータを現在までに得られたデータに基づいてオンラインで調整する手法であるため、基本データに外乱やモデル化誤差の影響があると、フィードバックループ内にある可調整パラメータのふるまいが、簡単に制御系を不安定化させてしまうことが知られている。したがって、外乱やモデル化誤差に対するロバスト性を考慮せずに適応制御系を構成することは現実的ではない。ここでは、ロバスト性を考慮したロバスト適応則(robust adaptive law)を紹介する。次の簡単な制御対象を考える。例えば、式(4・76)の設定が簡単である。パラメータ調整則を駆動する誤差には、パラメータ誤差の影響のみが含まれている場合が理想であり、調整はうまくいきやすい。しかし、式(4・77)のような場合には、ηの影響もパラメータ誤差の影響であると思って間違った調整を行う危険性があり、さらに、調整パラメータが制御器内のパラメータである場合には、それが閉ループ内におかれているために、容易に全系の安定性が崩れる。そこで、これらの不安定性を回避するようなロバスト適応則を用いる必要がある。ここでは、リーケージ手法、射影アルゴリズム、不感帯法の三つのロバスト調整法を紹介する。a. リーケージ手法(leakage method)次のようなロバスト調整法を考える。これは、リーケージ手法の代表的手法であり、固定σ-修正法と呼ばれている。式(4・78)を採用したときの性能を解析しよう。次のリアプノフ関数の候補を考える。この欠点を補うために、次のような二つのリーケージ手法(スイッチングσ-修正法、ε-修正法)が提案されている。もう一つのリーケージ手法として、ε-修正法を紹介する。b. 射影アルゴリズム(projection algorithm)実際問題では、真のパラメータベクトルθが存在する、次の凸集合が既知であることが多い。射影アルゴリズムは、いつでも調整パラメータが凸集合内に存在することを保証することができ、実用的である。c. 不感帯法(dead zone algorithm)不感帯法は、誤差にはパラメータ誤差の情報だけではなく、外乱やモデル化誤差の情報も含まれているので、それらの情報については不感とし、パラメータ調整を行わないように改良したアルゴリズムである。ここで、ηは式(4・94)を満足する定数である。次のリアプノフ関数の候補を考える。不感帯法を実装する場合には、式(4・94)にあるように、ηの上界値を見積もる必要がある。
