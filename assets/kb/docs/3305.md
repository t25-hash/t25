# 5・1・1 生物にならう制御

> **章**: 第5章 知的制御
> **節**: 5・1 知的制御概論

知的制御は生物(特に動物)のもつ能力をならおうとしていることはいうまでもないが、それを制御の観点から分類すると次のようになる。(1) 生物的人工物の実現(2) 補装具や人工臓器など生体機能の一部代替(3) 人間の計測・制御行動のコンピュータ化なわち人間のもつ能力をうまく利用することにより、従来の制御理論の欠点を補完する。ここで、情動などの機能を別にして、ヒト脳の機能を工学的に見れば、図5・1(a)のようになる。他方、図5・1(b)は人工システムの制御機能の階層構造を示したもので、図5・1(a)の脳の機能と各レベルにおいて対応する。実行レベルとは対象に制御動作を加えるレベルのことである。図量とする。としては伝達関数、微分方程式あるいは一般化して関数方程式などで記述される。制御動作の基本は制御対象の前に制御器を置き、この制御器がの逆システムとして動作するものがとれれば、は目標値に一致するので、これは理想的である。しかし、制御対象は因果的な特性をもつから、その逆システムは因果性からいって実現できない。ゆえに、例えば制御器としてPID動作を設定して、望ましい応答を得るようにそのパラメータを決める。これは実現可能な応答を導くような近似の逆システム制御器を作っていると見ることもできる。実行レベルでの制御器としては近似的な逆システムを構成するという問題が多く、線形理論の範ちゅうではこの問題は解決されているとみてもよいであろう。実際、最近の分散制御システムと呼ばれるものは、実行レベルとしてPIDのみならずアドバンスド制御と呼ばれるいくつものツールを用意している。協調レベルと管理・計画レベルは現実のシステムにおいてはそれほど明確に区別されないこともある。現時点では管理・計画レベルは人間のオペレータがその役割を果たしていることも多い。このような場合には、協調レベルはヒューマンインタフェースの役割ももつことになる。そして人間の場合は学習による経験知識の蓄積、それの検索による状況判断と意思決定、対象状態の将来予測などにより、いろいろの事態に対応できる。例えば、プロセス制御における(a)(b)図5・1 (a)脳の機能と(b)制御系のレベル図5・2 制御器と制御対象下位制御器の動作モード(制御モードとしてのPID、フィードフォーワードあるいはスミス法などや、プラントパラメータの同定モードなど)の選択、制御器パラメータ変更、プラントの運転立上げ、異常検出や故障診断などである。また離散事象的に駆動されるシステムのスケジューリングやプラントの運転状況に関する評価などもある。ここでのレベルは、実行レベルでの制御に用いられる伝達関数や微分方程式といった手法で制御対象のモデルを記述するのではなく、人間は、対象の動特性に関する知識を学習により脳内に得て、これにより望ましい状態を導くための近似的な逆システムを構成していると見られる。ここをコンピュータ化することが知的制御の主目的である。このような人間のもつ機能をコンピュータ化しようとするとき、そのモデルの記述能力としては次の事項が要求される。(1) 非線形特性を入出力写像として記述できる。(2) 数値のみでなく、記号表現の知識処理が可能であり、かつ状態の履歴特性を記憶し、かつ高速に検索できる。(3) 外部環境変化に対応する構造変化が可能。これは学習により達成可能であるが、そのための安定なアルゴリズムが必要である。
