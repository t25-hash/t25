# 5・1・2 知識表現としてのルールと推論

> **章**: 第5章 知的制御
> **節**: 5・2 においては制御対象の動作を記述し、は操作量、は制御

上に述べた(1)(2)の条件を満足できる手法として、ルール表現がある。これは人間のもつ知識をルールとして取出し、これを利用するものである。人工知能の分野において研究がなされ、多くのエキスパートシステムが開発された経緯がある。制御の面から見れば、複雑な知識の表現や高度の推論は不要であり、通常はプロダクションルールの形式が利用される。a. プロダクションルール(production rule)R 1: if A,then Bの簡単な形式で、A は条件部(前件部)、B は結論部(後件部)と呼ばれる。プロダクションシステム(production system)は、知識ベース、データベースおよびインタプリタよりなる。プロダクションルール群は知識ベースとして蓄積され、ワーキングメモリは外部対象の状態を記述するデータベースである。インタプリタはワーキングメモリ内のセンサ情報に適合する知識ベース内のルールを参照し、その推論結果をワーキングメモリに書き込み、その結果はさらに操作量として外部に出力される。このときの推論法はよく知られた三段論法である。そしてプロダクションシステムはこの繰返しを行う。プロダクションルールを制御動作に用いる場合、これは5・1・1項で述べた逆システムに対応したルールとなる。例えば温度制御系を考えるとき、制御対象の動作をもしプロダクションルールで記述するなら、if ヒータの電流を増す、then 部屋の温度が上がる。といった順方向の因果的記述になる。他方、制御器はif 温度が低ければ、then ヒータの電流を増す。という動作を記述するわけであるから、逆方向の意味が理解される。ただし、このプロダクションルールは単純すぎることもある。人間が知識を提供するときに、このルールはどの程度の確実さをもって成立するかといった情報をもっていることも多い。また、プロダクションシステムの運用時に、データベースのデータに適合するルールが多数存在することもある(ルールの競合)。このような場合にどのルールを優先するかの問題も生じる。そこでルールに確信度を付けた形式が考えられる。R 2: if A,then B with certainty factor確信度は例えば0,1や-1,1の範囲に定めればよい。これにより、人間の知識のあいまいさもある程度処理できる。しかし、この場合には確信度の更新のルールが必要となる。b. ファジィプロダクションルール(fuzzy production rule)ファジィ制御はわが国を中心に研究が進み、実用化も多い。ファジィの概念は人間の知識のあいまいさを数量化することが契機になったものである。ここでの知識もR 1の形のプロダクションルールに表現されるが、自然言語による知識表現にはあいまいさを含むので、ファジィ集合(fuzzy set)とそれに関する推論のほうが利用しやすい。次にR 1の形のプロダクションルールを制御則に使うとき、前件部は通常は測定データ(センサからの出力)であり、後件部は操作量(すなわち操作器への命令)である。ここで制御に関する言語知識をルールとしてR 3: if x is A,then y is Bと表現できるとしよう。簡単な例をとれば、前件部は例えば、「温度センサの値が高い」であり、後件部は「ヒータ電流を少し減らす」である。このときには集合は「高い」というメンバシップ関数を定義していなければならないし、は「少し」という関数を定義していなければならない。次に測定データが、別のファジィ集合に属するとしよう。すなわち、F : x is Aこのときの推論は上のR 3とF(事実)から、しかるべき結論C : y is Bを得ることである。Fが測定データを記述する場合にはは確定値をとり、またR 3が制御則を記述する場合には操作量もまた確定値をとるのが普通である。そこで、まずR 3とFから集合を推論する。この推論法としてはいくつかの方法が提案されている。ファジィ集合が決定されれば次に確定値のを得る必要がある。これをデファジィ化と呼んでいる。実際の問題ではルールはR 3のように単純ではなく、少なくともその前件部は二つ以上の条件文で構成されているのが普通であり、またそのようなルールは複数個存在する。これらについては5・2節において詳細が述べられる。
