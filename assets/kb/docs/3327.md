# 6・1・3 分散する機械

> **章**: 第6章 自律分散・創発システム
> **節**: 6・1 自律分散システム

効率を最優先とする場合には,すべての要素を考慮に入れた最適化をせざるを得ない.例えば,省スペースを目的とすると1箇所に集中したシステムが最適解となる.しかしながら,省スペースだけを評価として得られる最適解は,拡張性,耐故障性,保守性などの他の面からの評価を犠牲にしている場合が多い.実際,省スペース化と拡張性は相反する場合がほとんどである.情報技術の発達が市場に即応した生産,工場を連係させた生産への大きな変化を後押ししている時代には,拡張性,耐故障性,保守性などの評価を省スペースなどの効率に加えて重視せざるを得ない.拡張性を重視すると,要求に応じて機能が追加または削除できる柔軟性が必要である.そしてこのような柔軟性を確保するには,全体を部分に分ける必要がある.部分が機能に対応しており,要求に応じた機能の追加・削除を部分の追加・削除で実現するものである.もちろん,このような柔軟性が確保されれば,同じ機能をもつ部分を複数個用意することで耐故障性が向上するし,部分を交換するという形で保守性を向上させることが可能である.加えて,全体を部分に細分化できれば,人は部分から理解を始めることが可能となり,設計,保守,改善などにも好都合である.この部分を部品と読み替えると,古くから機械の作成・保守で用いられてきた部品化が拡張性,耐故障性,保守性を向上させることを目的とするものだったことがわかる.機械を部品の組合せとすることで理解しやすくするとともに,不良部品の入れ替えという形で保守性を向上している.もっとも,部品と部品が物理的に結合している必要があり,システム全体を眺めるだけでは集中と分散の違いが見つけにくかった.しかしながら,エレクトロニクスの発達,情報技術の発達は,部品と部品の結合を電気的なものだけに限定することを成功させた.これをもう一段抽象化させて,部品と部品との結合を情報交換だけに限定できれば,部品と部品は密接に結合している必要はない.つまり,近くにいる必要がない.すなわち,部品と部品は分散し,その間には有線や無線などの情報通信網が存在すればよい.これが分散システム(distributed system)である.このように部品,すなわち工程や工場も含む部分を物理的に分散させることが可能であれば,地域に依存しない地理的,経済的,政治的な障害に強い生産システムを構築することができる.これが自律分散システムが目指す機械や工場である.もちろん,既存の設備や工場を情報網で結合することにより,効率や生産性,耐故障性を向上させることを目的とするERP(engineering resource plan-ning)やSCM(supply chain management)も,ここで考える分散システムの一つに含まれる.この場合は全体を分割化して部品化するというよりは,すでに存在する部門や会社などを結合して新たな組織を作るという,結合化とも呼ぶべきものである.もっとも,全体が部分から構成されるという分散システムの本質に違いはない.
