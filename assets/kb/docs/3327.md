# 6・1・3 分散する機械

> **章**: 第6章 自律分散・創発システム
> **節**: 6・1 自律分散システム

器側は自律しているとはいえず，拡張性，耐故障性，保守性を限定
効率を最優先とする場合には，すべての要素を考慮に入れた最適
してしまう．
このような限定を解除するために求められているものが自律性で
ある．FAオープン推進協議会では，自律性を「一人でも生きられ
る」と定義した．つまり，情報網，すなわちネットワークから切
断されても機器が最低限の機能を発揮できる能力を自律性と呼んで
いる．もちろん，ネットワークに回復できれば，他の機器と高度な
協調を図れることはいうまでもない．
このように，情報網の切断を許容できる能力が，自律分散システ
ムと通常の分散システムを区別する指標である．また，このような
拡張性を重視すると，要求に応じて機能が追加または削除できる
切断に対する耐性はつながるかつながらないかという二段階だけで
なく，接続先の機器が稼働しているか否かということも含めて，多
段階で考える必要がある．このような多段階の場合に応じて機器側
の動作が変わるものが自律分散システムである．それゆえ，各機器
には高度な知能が要求される．
このような機器が知能化されたシステムを動かすとき，他の機器
への情報伝達は命令であってはいけない．命令は服従を想定し，知
能をもつ機器への働きかけとしては好ましくない．知能化は状況に
応じた多様な応答を機器側に期待するものなのに，命令はルーティ
この部分を部品と読み替えると，古くから機械の作成・保守で用
ンとしての応答しか許さないものである．このため，命令しかでき
ない機器が，機器側から命令を拒絶された場合に対処できないおそ
れがある．
この問題を回避するためには，他の機器への動作要請を質問中心
に変えることである．質問であれば，相手の答えとして他段階を想
定せざるを得ず，自然に自律分散機能が導入される．これは，命令
に比べて機器どうしの関係を疎にするということである．機器どう
しかしながら，エレクトロニクスの発達，情報技術の発達は，部
しを密結合すれば発揮できる機能を，疎にして実現することが自律
