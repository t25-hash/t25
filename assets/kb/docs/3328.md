# 6・1・4 自律する機械

> **章**: 第6章 自律分散・創発システム
> **節**: 6・1 自律分散システム

大規模,複雑化する機械や工場が分散システム化していく動きの中で不可欠なものが,部品の知能化である.つまり,部品と部品の結合を情報だけに限定する以上,各部品は情報以外は独立している必要がある.これが部品として見たときの自律性(autonomy)である.具体的には,他の部品から切り離して存在できること,そして,孤立しても動作できる自前のエネルギー源をもつことである.このように部品と部品の結合を情報だけに限定しているため,情報以外の自律性は多くの場合確保されている.問題は情報に関する自律性である.分散システムで昔から使われているポーリングやC/S(client/server)という情報伝達形式は,マスタまたはサーバと呼ばれる機器に情報が集中しており,スレイブやクライアントと呼ばれる分散機器はそれらに支配されている.これでは,分散機器側は自律しているとはいえず,拡張性,耐故障性,保守性を限定してしまう.このような限定を解除するために求められているものが自律性である.FA オープン推進協議会では,自律性を「一人でも生きられる」と定義した.つまり,情報網,すなわちネットワークから切断されても機器が最低限の機能を発揮できる能力を自律性と呼んでいる.もちろん,ネットワークに回復できれば,他の機器と高度な協調を図れることはいうまでもない.このように,情報網の切断を許容できる能力が,自律分散システムと通常の分散システムを区別する指標である.また,このような切断に対する耐性はつながるかつながらないかという二段階だけでなく,接続先の機器が稼働しているか否かということも含めて,多段階で考える必要がある.このような多段階の場合に応じて機器側の動作が変わるものが自律分散システムである.それゆえ,各機器には高度な知能が要求される.このような機器が知能化されたシステムを動かすとき,他の機器への情報伝達は命令であってはいけない.命令は服従を想定し,知能をもつ機器への働きかけとしては好ましくない.知能化は状況に応じた多様な応答を機器側に期待するものなのに,命令はルーティンとしての応答しか許さないものである.このため,命令しかできない機器が,機器側から命令を拒絶された場合に対処できないおそれがある.この問題を回避するためには,他の機器への動作要請を質問中心に変えることである.質問であれば,相手の答えとして他段階を想定せざるを得ず,自然に自律分散機能が導入される.これは,命令に比べて機器どうしの関係を疎にするということである.機器どうしを密結合すれば発揮できる機能を,疎にして実現することが自律分散であり,機器側の知能化が欠かせない理由でもある.図6・1 自律分散システム
