# 6・1・6 設計図をもつ機械

> **章**: 第6章 自律分散・創発システム
> **節**: 6・1 自律分散システム

システム内の機器が自律的に動くためには，各機器が自らの機器
情報を提供する必要がある．機器情報は，機器の名称や保持する機
能，バージョン情報などである．これは，設計情報の一部である．
そのことから，機器情報として機器の設計図を丸ごともたせるとい
うことが理想の一つである．もちろん，記録のコストが高ければ丸
ごともたせることは難しい．その場合には，設計図が保存されてい
るURLアドレスのみでももたせられれば，保守性が大幅に向上す
る．
もっとも，設計情報は部品や部分を作るときだけに必要となるも
のではない．システムができあがった後の維持管理や更新，そして
破棄まで設計情報は不可欠である．その意味で，少なくとも部品や
部分の寿命程度は設計情報を管理していく必要がある．
記憶に関するコストは下がる傾向にあるので，部品には設計情報
が存在するアドレスだけでなく，すべての設計情報をICカードで
張り付けられる時代がくるかもしれない．同様に，ソフトウェア自
体もそれを生成するのに関連するすべての情報を付加させる方向に
発展していく可能性もある．
