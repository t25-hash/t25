# 6・1・6 設計図をもつ機械

> **章**: 第6章 自律分散・創発システム
> **節**: 6・1 自律分散システム

システム内の機器が自律的に動くためには、各機器が自らの機器情報を提供する必要がある。機器情報は、機器の名称や保持する機能、バージョン情報などである。これは、設計情報の一部である。そのことから、機器情報として機器の設計図を丸ごともたせるということが理想の一つである。もちろん、記録のコストが高ければ丸ごともたせることは難しい。その場合には、設計図が保存されているURLアドレスのみでももたせられれば、保守性が大幅に向上する。もっとも、設計情報は部品や部分を作るときだけに必要となるものではない。システムができあがった後の維持管理や更新、そして破棄まで設計情報は不可欠である。その意味で、少なくとも部品や部分の寿命程度は設計情報を管理していく必要がある。記憶に関するコストは下がる傾向にあるので、部品には設計情報が存在するアドレスだけでなく、すべての設計情報をICカードで張り付けられる時代がくるかもしれない。同様に、ソフトウェア自体もそれを生成するのに関連するすべての情報を付加させる方向に発展していく可能性もある。もっとも、生体はこのレベルを大きく超えている。つまり、システム全体の設計図を遺伝子情報という形で各部品に相当する細胞が6―157保持している。そして、同一の設計図をもつ部品がシステムを構成する仲間であり、違う設計図をもつ部品は免疫系が異物として排除する。この生体で行われている情報管理が自律分散システムの目指すものと考えると、すべての構成部品にシステムの設計図を保持させ、その設計図をもたない部品を排除するというシステムとなる。現状では、設計情報の一部として部品が生産されるという認識である。それに対して、部品が設計情報を包含することになる。これは、部品と設計情報との関係が逆転することになる。このように考えると、将来の機械は設計情報だけでなく、更新や変更などの維持管理情報も機械自身にもたせる必要性が見えてくる。設計と履歴情報を付加することで、機械の動作をわかりやすくするとともに信頼性も向上する方向である。
