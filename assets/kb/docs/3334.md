# 6・3・1 強化学習

> **章**: 第6章 自律分散・創発システム

a. 枠組と歴史。 定義強化学習(reinforcement learning)とは、生物6―160制御システム図6・2 強化学習の枠組が報酬(reward)と呼ばれる特別な信号を手掛かりとし、試行錯誤を通じて不確実な環境に適応していく過程に啓発された機械学習の一種である。報酬獲得に役立つ行動をしかるべく強化(rein-force)することから、強化学習と呼ばれる。. 枠組強化学習は極めてシンプルな枠組をもつ(図6・2)。ある環境に置かれたロボットのようなエージェントを考える。ある時点でエージェントには環境の状態に応じた感覚入力욧が与えられる。エージェントは行動욧を出力して環境に働きかける。環境は次の時点の状態욧용に状態遷移する。ときどき報酬と呼ばれる特別な信号욧용が送られる。報酬とは、環境で生き残る、えさをとる、子孫を残すなどのプラスの評価、あるいは、敵に出会う、けがをするなどのマイナスの評価を抽象化した量である。ある時点である感覚入力に対してどのような行動をとるかを政策(pol-icy)と呼ぶ。エージェントの目的は報酬を最大化する政策を発見することである。強化学習の枠組には二つの際立った特徴がある。まず、感覚入力や行動出力は不確実である。さらに、報酬はそれぞれの行動の直後ではなく、遅れてやってくる可能性がある。必然的にエージェントは最適政策(optimal policy)を試行錯誤によって求めていかなければならない。これがニューラルネットワークなどの教師付き学習(supervised learning)との違いである。逆にいえば、ヒューマンエキスパートなどが不要であることになり、強化学習が未知要素を多くもつ領域での基礎技術として期待されるゆえんである。. 歴史強化学習は歴史的には二つの側面をもつ。機械学習としての強化学習は、人工知能の草創期から研究されている。1950年代のミンスキー(Minsky)による信頼度割当法(creditassignment)、サミュエル(Samuel)によるチェッカープレーヤーがよく知られている。当時の計算技術の限界もあって、理論的な検討が浅い状況が続いていた。一方、同じく1950年代にはベルマン(Bellman)による最適制御・動的計画法が提唱されている。機械学習と最適制御とは、後者が環境に関する完全な記述を要求するので、一見無関係に見える。ところが、1988年のサットン(Sutton)によるTDλ,1989年のワトキンス(Watkins)によるQ学習によって、両者の密接な関係が明らかとなり、理論面での飛躍的な進展を見て今日に至っている。標準的な教科書にはサットンによるものが便利である。b. 部分観測マルコフ決定過程・関数近似。 適用範囲の拡張マルコフ決定過程は現実応用には厳しい条件である。c項で述べたマルコフ決定過程を前提とする強化学習法の拡張もさまざまに追及されている。環境がマルコフ決定過程であっても、感覚器の制約から、エージェントにとってはマルコフ性が満たされない場合がある。異なる状態が同一に感覚される場合、隠れ状態があるといい、部分観測マルコフ決定過程(POMDP: partially observable Markov decisionprocess)という。逆に、同じ状態が異なって感覚される場合をperceptual aliasing と呼ぶ。いずれの場合でも、マルコフ決定過程を前提とした強化学習法を適用すると収束性が損なわれる。マルコフ決定過程は有限状態と有限行動上で定義される。感覚や行動が連続である場合、なんらかの関数近似との組合せが必要である。最も簡単には、状態空間の格子分割があるが、状態数の増加は試行錯誤回数の爆発をもたらし、好ましいことではない。適応的な状態分割について多くの方法が提案されている。. 非マルコフ環境への取組み。 非マルコフ性の所在最近の強化学習の理論的発展は、マルコフ決定過程を前提とした最適制御との関連付けに負うところが大きい。マルコフ性の仮定は数学的扱いには便利だが、機械学習としての強化学習に立ち戻って考えると、概念を不当に狭めている面がある。例えば、学習するエージェントが複数集まったマルチエ6―161ージェントシステムや、環境条件が刻々と変化するオープンシステムにおかれたエージェントでは、時不変のマルコフ決定過程の最適政策発見に特化された手続きはうまく働かない。マルコフ決定過程に依存しない合理性の基準が必要である。. 利益分配法の合理性定理マルコフ決定過程に依存しない合理的強化学習の例として、宮崎による利益分配法の合理性定理について説明する。状態と行動の対をルールと呼ぶ。各ルールには重みが付けられているものとする。Q学習における値と同じサイズのテーブルだが価値関数ではない。スタートから報酬が得られるまでのルールの選択系列をエピソードと呼ぶ。エピソードが何回も繰返される。このとき、グレフェンシュテッテ(Greffenstette)の利益分配法(profit sharing)とは、エピソード上の各ルールに、得られた報酬をある基準に従って分配することで、ルールの重みを強化する方法である。報酬からのステップ数に対して、ルールに割当てる報酬量を与える関数を分配関数という。分配関数には、等配分、等差減少配分などさまざまな方法が考えられるが、場合によっては報酬から遠ざかるルールを強化してしまうおそれがある。宮崎は、報酬から遠ざかるようなむだなルールを強化しない局所合理性条件と、強化されたルールが全体として報酬を全く含まないことはない大域合理性条件とが、式(6・12)と等価であることを示した。욢<욪(6・12)∑ここで욢:報酬から番めのルールへの報酬分配量、:エピソード長である。式(6・12)を満たす最も簡単な関数型は、指数減少である。利益分配法とQ学習の違いについては、捕獲問題などのマルチエージェント学習の標準問題で簡単に効果を確認することができる。
