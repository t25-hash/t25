# 7・1・1 離散事象システムの特徴

> **章**: 第7章 離散事象システム
> **節**: 7・1 システム表現

a. 概念離散事象システム(discrete event systems)とは、以下の特徴をもつシステムのことである。. 離散的状態システムのとりうる状態の集合が離散集合として与えられる。次のような場合がある。(1) 各状態変数のとりうる値が離散値であり、それらの値の組合せによりシステムの状態が定義される場合(例:スイッチが入っている/切られている、機械が稼働中/待機中、バッファ内の部品の個数)。(2) 対象をモデル化するときに、本来は連続値をとる状態を離散的に扱う場合(例:タンクの水量をバルブの開閉により制御するシステムでは、水面の高さは連続値をとるが、目的によっては、レベル低、中、高の3値のみをとる状態変数を考えれば十分な場合がある)。. 事象駆動事象(event)とは、ボタンが押される(特定の動作が起こる)、あるいは、タンクの水量がある水準を超える(特定の条件が満たされる)など、離散的状態の瞬間的な遷移のことである。状態の変化が微分方程式や差分方程式のように、時間の関数として与えられるようなシステムは、時間駆動(time-driven)であるという。例えば、システムのクロックに同期して状態が変化するようなシステムは、時間駆動である。これに対し、任意のタイミングで発生する事象によって状態が変化するシステムは、事象駆動(event-driven)であるという。事象生起のタイミングは事前には与えられず、また、クロックと同期する必要もない(例:計算機における多くの動作はクロックに同期して実行されるので時間駆動であり、一方、割込み処理は事象駆動である)。このような非同期的な事象生起は、複数のプロセスがシステム内に存在し、それらが相互作用を繰返しながら並行的に状態遷移を行っていく状況において出現する場合が多い。b. 離散事象システムの例離散事象システムとして扱うことのできる対象は、表7・1に示すように、計算機システム、通信システム、生産システムなどさまざまな領域に存在する。c. モデルの分類離散事象システムのモデル化には、オートマトン、形式言語、ペトリネット、代数的モデル、論理式、待ち行列ネットワークなど、さまざまなものが用いられる。これらは、使用目的から大きく次の二つに分類できる。. 論理モデル(logical model)システムの論理的な正しさを検証するために用いるモデル。時間を導入するか否かにより、さらに次の二つに分類できる。(1) 非時間モデル事象の生起順序のみを考慮するモデル。(2) 時間モデル事象の生起に時間的制約をもつような対象を表現するためのモデル。例えば、事象が生起を許可されてから実表7・1離散事象システムとして扱われる対象分野応用例計算機システム/通信システムオペレーティングシステム、ソフトウェア設計、データベースシステム、VLSI設計、非同期回路、通信プロトコル生産システムシーケンス制御、組立ライン、無人搬送システム、制御用ソフトウェア、組込みシステム、生産スケジューリングオフィスオートメーションシステムグループウェア、ワークフローシステム交通システム信号制御システム、航空管制システムその他プロジェクト管理、物流システム意思決定支援際に生起のするまでに遅延時間が存在する場合、ある時間内に事象が生起しなければ別の処理に切換えるタイムアウト処理などがその例である。なお、論理モデルで扱われる時間は一般には決定性の時間である。. 性能評価モデル(performance evaluation model)システムの性能(スループット、反応時間、稼働率など)を定量的に評価するために用いるモデル。事象が生起するまでの遅れ時間を確率変数として与えるなど、事象生起のタイミングは確率的にモデル化される。
