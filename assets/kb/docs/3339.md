# 7・1・4 時相論理

> **章**: 第7章 離散事象システム
> **節**: 7・1 システム表現

a. 概要時相論理(temporal logic)は通常の論理(古典論6―168制御システム理)に時間の概念を導入した論理であり、時間によって論理式の真偽値が変化する状況を記述できる。離散事象システムのように、状態が動的かつ離散的に変化するシステムの挙動や性質の記述に適している。例えば、ペトリネットの解析における到達可能性、活性、公平性など多くの性質は時相論理により記述することができる。その意味で、時相論理式により記述されたシステムの性質が成り立つことを判定する。あるいは、その性質が成り立つようにシステムを設計する問題は、離散事象システムの解析問題の一般形であるといえる。時相論理では、(否定)、∨(または)、∧(かつ)、(ならば)などに加え、時間に関するいくつかの演算子が使用される。どのような演算子を使用するかは論理ごとに異なるが、代表的なものとして以下のものがある。これらは未来に対する演算子であるが、過去に対する演算子を導入した論理もある。b. 線形時間論理と分岐時間論理時相論理は時間の導入方法により、大きく線形時間時相論理(linear time temporal logic)と分岐時間時相論理(branching time temporal logic)の2種類に分類される。. 線形時間時相論理次状態が一意に決定される場合に対する論理である。論理式は原子命題(それ以上分解できない命題)、および、それらが論理演算子により結合されてできた命題からなる。. 分岐時間時相論理次状態が一意ではなく、状態遷移の過程が初期状態からの木として表される場合に対する論理である。線形時間論理は一つのパスに対する論理であるといえる。をパスに対する論理式としたとき、どのようなパスでが成立するかを表現するために、すべてのパスで成立する(∀)、あるパスで成立する(∃)の2種類のパス限量記号が用いられる。オートマトンやペトリネットの状態遷移図をこれらの論理の解釈を与える構造と見れば、時相論理により離散事象システムの動的な性質が記述できる。c. 実時間時相論理数値で与えられる時間制約を扱えるように拡張された論理として、実時間時相論理(real-time temporallogic)がある。d. モデル検査モデル検査(model checking)とは有限状態のシステムを検証するための手法であり、三つのステップからなる。(1) モデル化設計したシステムをオートマトンやペトリネットなどの形式的モデルに変換する。(2) 仕様記述設計したシステムが満たすべき性質を時相論理などを用いて記述する。(3) 検証形式的モデル上で仕様が満たされているかどうかを検証アルゴリズムにより判定する。もし初期状態がこの集合に含まれていれば、仕様が満たされることになる。モデル検査で用いられる検証アルゴリズムとしては以下のものがある。(1) 構造を状態遷移図で表現し、各状態にその状態で真であるようなの部分論理式の集合をラベルとして付けていく方法。(2) タブロー法(tableau method):構造および論理式から、を満たすようなパスの集合を表現する有向グラフ(タブロー)を作成する方法。(3) 記号モデルチェッキング(symbolic model checking):二分決定グラフを用いたブール関数の計算により、論理式が真であるような状態集合を効率的に計算する方法。
