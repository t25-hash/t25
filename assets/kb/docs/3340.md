# 7・2・1 スーパーバイザ制御と可制御性

> **章**: 第7章 離散事象システム
> **節**: 7・2 スーパーバイザ制御

a. 離散事象システムにおける制御問題。 排他制御(mutual exclusion)複数の計算機が1台のプリンタを共有しているとき、プリンタを利用できる計算機はたかだか1台である。したがって、ある計算機が使用している間は他の計算機の使用を禁止しなければならない。このように、複数のプロセスが同時に存在するとき、同時に利用できるプロセスの数が限られている資源を有効に利用するための制御を排他制御という。. 並行処理制御(concurrency control)データベースでは複数のトランザクションを同時に実行するので、あるデータが複数のトランザクションから読み書きされることがある。このとき、データの内容が矛盾することなく、並行性をできる限り損なわないようにデータへのアクセスを制御しなければならない。このような制御を並行処理制御という。. デッドロックの回避(deadlock avoidance)デッドロックとは複数のプロセスが干渉し合ってお互いに処理ができなくなることをいう。デッドロックが発生するとあるプロセスをリセットさせることが一般的である。したがって、デッドロックを避けるように制御を行うことは重要である。デッドロックは排他制御や並行処理制御においてもしばしば問題となる。. ライブロックの回避(livelock avoidance)事象が生起するにもかかわらず、いつまでたっても目標状態に到達しないような現象をライブロックという。ライブロックとデッドロックは目標状態には達成しない点では共通であるが、デッドロックは完全に状態遷移が止まるのに対して、ライブロックは状態遷移が起こる点が異なる。. 安定化(stabilization)通信システムにおいては各サイト間で来るべきメッセージが来ないことがある。このようなときに正常な状態に戻るような制御が重要である。特に、自動的に正常な状態に戻るようなプロトコルは自己安定(self-stable)であると呼ばれる。. 公平性生起する事象があるプロセスに関係するものばかりではなく、どのプロセスに関係する事象もいつかは生起できることを公平(fair)であるという。例えば、プリンタの使用では、使用要求を出したプロセスから順番に使用を許可するFIFO(first-in first-out)制御が公平な制御の典型的な例である。. 統合問題(coordination problem)複数のプロセスが同時進行しているとき、各プロセスが行う処理自体には全く制約を与えず、その処理の相対的な順番のみを制御することによりCPUの有効利用やプロセス間の干渉を抑える。このことは、オペレーティングシステムの設計上重要な問題であり、統合問題と呼ばれる。排他制御、並行処理制御、デッドロックの回避、公平な制御等を適用することによって統合問題は解決される。. スーパーバイザ制御(supervisory control)と状態フィードバック制御(state feedback control)システムが大規模化すると、統合問題のように制御仕様もさまざまな部分制御問題が複雑に絡み合うことになる。そこで、システマティックに制御系を構成するために提案された手法がスーパーバイザ制御と状態フィードバック制御である。いずれの方法でも、システムが制御仕様を満たすように次に生起できる事象を制限することで制御を行っている。スーパーバイザ制御では、制御仕様を形式言語で与えて、制御対象が生成する事象列に基づいて制御を行う。状態フィードバック制御では、制御仕様を状態上の述語で与え、システムの現在の状態に基づいて制御を行う。一般にδは部分関数であり、定義されているとき、δσ,!と書くことにする(7・1・2・b項参照)。各状態で、生起が許される事象の集合を制御パターンという。このとき、その生起が外部から禁止できる事象と禁止できない事象がある。前者を可制御事象、後者を不可制御事象という。例えば、タスクの開始、プリンタのような資源へのアクセスなどは可制御事象である。また、システムダウン、タスクの終了等は不可制御事象の例である。可制御事象と不可制御事象の集合をそれぞれ∑,∑とおく。Γを制御パターンの集合とおく。この制御パターンを割当てる制御器はスーパーバイザと呼ばれる。スーパーバイザによって制御された離散事象システム、すなわち閉ループシステムは次のオートマトンで表される。において次に生起可能な事象は、遷移後のの状態に対してξによって割当てられた制御パターンの要素でなければならない。また、スーパーバイザによって割当てられた制御パターンの要素であっても、において生起ができない事象は、制御を行っても決して生起できないことに注意してほしい。このような意味で「制御パターンによって事象の生起は許容される」という言い方をすることがある。制御仕様を満たす言語を制御仕様と呼ぶことにする。[定理1]はブロッキングがない離散事象システムとする。制御仕様が空でない接頭語閉な正規言語で、も有限オートマトンであるとする。が可制御であることと下記の条件は等価である。補題2より、の可制御性は、以下のアルゴリズムによって判定できる。成立していれば可制御であり、成立していなければ可制御ではない。e.
