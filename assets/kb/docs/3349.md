# 7・4・2 待ち行列理論

> **章**: 第7章 離散事象システム
> **節**: 7・4 パフォーマンス解析

待ち行列理論(queueing theory)とは、有限の資源が多くの利用者によって共有されている状況を数学的に記述した確率モデル(待ち行列モデル)に関する理論である。多くの利用者が有限の資源を共有している状況では、利用したい資源が他の利用者によって占有されている場合、他の利用者は資源が解放されるまで待つか、あるいは、その資源の利用をあきらめなければならない。このような待つという状況、あるいは競合により資源利用を拒否されるという状況を抽象的に表すため、待ち行列理論では、通常、資源利用要求を客、共有資源をサーバ、客が共有資源を占有する時間をサービス時間と呼ぶ。待ち行列理論において興味の対象となるおもな指標には、システム内客数分布や客の待ち時間分布などがある。a. 待ち行列モデルの記法待ち行列モデルは、共有の資源への利用要求発生に関する統計的情報(到着過程)、個々の要求が資源を占有する時間に関する統計的情報(サービス時間過程)、共有資源の数(サーバ数)、システム内にとどまることが許される客の最大数(システム容量)などによって記述される。このような待ち行列モデルを記述する方法としてケンドールの記法(Kendall’snotation)がある。これは、一般に///の形をしており、それぞれ、到着時間間隔分布/サービス時間分布/サーバ数/システム容量を示している。サービスの順序に関しては通常、先着順サービスが仮定されている。λを単位時間当たりの平均到着客数(これを平均到着率という)とする。
