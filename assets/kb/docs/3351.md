# 7・4・4 順序最適化

> **章**: 第7章 離散事象システム
> **節**: 7・4 パフォーマンス解析

順序最適化(OO: ordinal optimization)とは、解析的な扱いが困難な大規模かつ複雑なシステムに対する現実的な問題解決策と6―181して、1990年代初頭にハーバード大学のホー(Ho)によって提案された一つの考え方である。a. 背景例として、次のような最適化問題で表されるシステム設計問題を考えてみる。ここでいう設計方針としては、連続的なパラメータでもスケジューリング政策のように離散的なものでも構わない。ただし、Θは有界な連続型の集合または有限集合である。こうした問題では、システムが不確実または不確定な要素を含んでいる場合など、与えられたθに対してθの値が解析的・数値的に求められないことも少なくない。そのため、シミュレーション、あるいは実際に稼働しているシステムを観測することにより、θの値を推定するというのが一般的な手法となる。その際の問題点として、システムが大規模かつ複雑になると、実行可能な設計方針の数も膨大になることや、性能評価尺度を正確に推定しようとすると、システムを長時間にわたって観測し続けなければならないことが挙げられる。すなわち、大規模かつ複雑なシステムに対しては、実行可能なすべての設計方針に対して長時間のシミュレーションを行い、そのうえで最良の方針を決定するという策は、莫大な時間とコストをかけることになってしまう。このような問題に対して、OOの基本的な考え方は、必ずしも最適でなくてもよいから、とりあえず満足できる解をなるべく早く提供しようというものである。実際、OO自身は厳密な最適解を与えるものではないが、他の最適化アルゴリズムに対して補足的に用いることにより、確率的なシステムの最適化問題に有効であることも報告されている。b. 実装OOでは、目標の緩和(goal softening)と順序比較(ordinal comparison)により、その考えを実現する。目標の緩和とは、Θの中から厳密に最適な設計方針を見つける代わりに、良い設計方針をいくつか含むようなΘの部分集合を提示しようという考え方であり、最適解を与えることはできなくても、とりあえず満足できる解を提供することによって、工学的な要求にある程度応えようという姿勢を表している。具体的には、まず第一段階の目標の緩和として、実行可能な設計方針すべてに対して性能評価尺度を推定するのは到底無理なので、Θの中からでたらめに(一様に)個だけ選ぶことにする。そこでOOでは、順序比較すなわちθ,...,θの間の順序関係を決定することのみに重点をおく。短時間で順序関係が決まるといっても、推定のノイズによって誤った答えを導いてしまうことも起こりうる。以上がOOの基本的な考え方とその実現方法である。OOは、決して既存の最適化手法に代わろうというものではなく、むしろ補足的に用いられるものである。例えば、初めは個の設計方針をΘから一様に選んでいたとしても、順序比較の結果に基づいて設計方針の選び方を更新するという手続きを繰返すなどにより、確率的なシステムに対する最適化問題を効率良く解く一つの可能性を与えているのである。
