# 7・4・4 順序最適化

> **章**: 第7章 離散事象システム
> **節**: 7・4 パフォーマンス解析

順序最適化(OO: ordinal optimization)とは、解析的な扱いが困難な大規模かつ複雑なシステムに対する現実的な問題解決策と6―181して、1990年代初頭にハーバード大学のホー(Ho)によって提案された一つの考え方である。a. 背景例として、次のような最適化問題で表されるシステム設計問題を考えてみる。Minimizeθsubject toθ∈Θここに、Θ:実行可能な設計方針全体からなる集合であり、θ:設計方針θを選んだときのシステム性能評価尺度である。ここでいう設計方針としては、連続的なパラメータでもスケジューリング政策のように離散的なものでも構わない。ただし、Θは有界な連続型の集合または有限集合である。こうした問題では、システムが不確実または不確定な要素を含んでいる場合など、与えられたθに対してθの値が解析的・数値的に求められないことも少なくない。そのため、シミュレーション、あるいは実際に稼働しているシステムを観測することにより、θの値を推定するというのが一般的な手法となる。その際の問題点として、システムが大規模かつ複雑になると、実行可能な設計方針の数も膨大になることや、性能評価尺度を正確に推定しようとすると、システムを長時間にわたって観測し続けなければならないことが挙げられる。すなわち、大規模かつ複雑なシステムに対しては、実行可能なすべての設計方針に対して長時間のシミュレーションを行い、そのうえで最良の方針を決定するという策は、莫大な時間とコストをかけることになってしまう。このような問題に対して、OOの基本的な考え方は、必ずしも最適でなくてもよいから、とりあえず満足できる解をなるべく早く提供しようというものである。実際、OO自身は厳密な最適解を与えるものではないが、他の最適化アルゴリズムに対して補足的に用いることにより、確率的なシステムの最適化問題に有効であることも報告されている。b.
