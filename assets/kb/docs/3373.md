# 2・1・1 生産システム開発プロセスの概要

> **章**: 第2章 設計・評価技術
> **節**: 2・1 生産システム設計・評価技術の体系

一般に大規模なシステムの開発プロセスにおける各段階において、問題の解決手順は六段階から成り立っている。これを生産システムの開発プロセスに適用すると、以下のようになると考えることができる。a. 問題の設定(problem definition)解決すべき問題、すなわち生産システムの開発問題を認識し、その開発項目を明確にする。b. 目的の選択(selecting objectives)システム合成、すなわち開発すべき生産システムの設計の代替案を評価するための基準を設定し、選択する。これは、目的関数として定量的に定義できる場合もあるが、達成すべき項目の羅列として定性的にしか表現できない場合もある。c. システム合成(systems synthesis)開発すべき生産システムの設計案を複数創出する。ここでは、システム解析および選択を行うために、複数の代替案を創出することが重要である。d. システム解析(systems analysis)生産システムの設計案の特性を定性的、定量的に把握し、開発の目的に基づいて解析する。これにより、代替案が目的をどの程度満足するかを評価する。e. システム選択(selecting systems)システム解析の結果を評価し、評価結果と設定された目的との比較検討を行って、適切な設計案を選択する。f. 実行計画(planning for action)選択された設計案について、その実施計画を立案し、組織体制などを決定する。この段階において、開発過程が1ステップ進展するとともに1レベル詳細化され、以下詳細化されたそれぞれの開発問題に対して、a~f項までのプロセスが繰返される。上記のプロセスの中で、問題設定および目的の選択が、生産システムの開発を進めるうえで極めて重要である。問題設定においては、以下のような項目を明確にする必要がある。・生産システム全体または一部の開発・新規の開発または既存システムの改善・投資費用、人的資源、開発期間などの制約次に、問題設定と密接に関係するが、生産システムを開発するための目的を明確にすることも必要である。生産システムの開発目的の例を以下に示す。・投資額および/または開発期間の最小化・生産量および/または生産品種の増大・生産コストおよび/または運用コストの低減・信頼性および/または品質の向上・フレキシビリティおよび/または拡張性の向上・労働環境および/またはヒューマンインタフェースの向上上記のような問題および目的を設定したのち、生産システムの設計および評価のプロセスに入るが、そのプロセスは、対象となる製品の生産工程および生産システムの形態により異なる。そのため、以下では、自動化、省力化が最も進んでおり、自動化設備を念頭において設計および評価を行う必要がある機械加工システムについてその設計・評価技術をまとめる。
