# 2・3・4 物流計画のための手法

> **章**: 第2章 設計・評価技術
> **節**: 2・29 に新たな生産システムを構築しようとする企業が加工、組立、

a. 調査分析手法物流システムの計画に長年取組んでいる人は従来は頭の中で経験というツールによって現状のデータから物流システムを分析し、評価してきた。しかしシステムが複雑化する中で、これらの人の行動を論理的にかつ多方面から支援するための分析ツールが提唱されている。またこれらは経験の浅い技術者にとっても有効で計画手法の伝承には不可欠となる。工場内に何(形状、重量、質ほか)がどこからどこへどれだけ流れるかを分析しないと工場内の物流計画は立てられない。このため図2・32 加工組立工場の搬送車図2・33 半導体製造ライン図2・34 ビジュアルシミュレーション例に物量を算定したり、分析するためのツールの代表例を表2・9に、図2・31にその事例を示す。b. シミュレーション技術シミュレーションを行う場合、はん用プログラミング言語やシミュレーション言語を活用した専門的知識が必要な時代から、現在では搬送車や生産機械、倉庫のシンボルを張り付けてその間をつなぎ、必要な与条件を入力するだけで活用が可能なものも出てきている。こうした中でシミュレーション技術が求められる理由として、・物流システムの構築が複雑になっている・他のシステムとの連携が複雑になっている・計画時間の短縮を求められる・生産ラインの寿命が短くなり頻繁に計画が必要・トップへのシステム投資物のビジュアルな説明などのニーズが出てきており、このためシステムの構築前に設計の検証を行い、手戻りを少なくするためにシミュレーションによる事前評価が設計行為の中で重要な作業となっている。図2・32に加工と組立の間の搬送をシミュレートした例を、また図2・33に半導体製造ラインの例を、また図2・34には計画初期段階でトップへの提案のためのビジュアル表現を行った例を示す。c. モデル化ツールシステムをモデル化し、現状把握と設計システムの形を表現するためにもさまざまなモデル化手法が提唱されている。その中で比較的物流計画に有効で標準手法として活用されているIDEF(ICAM definition)について述べる。IDEF0は複雑な対象を、要素図形とその接続関係で表現し、関係者が共通に理解するための概念と手法、規約の集合体で複雑なシステムのモデル化を行うための手法である。だが、その表現方法は単純で、アローと呼ばれる矢印とボックスと呼ばれる四角の箱だけで表現される。アローはインプット、コントロール、アウトプット、メカニズムの四つからなり、それぞれボックスの接続位置が定められており、特にコントロールはインプットが出力されるための条件を与える重要な役割をもつ。ボックスは作業や活動などを表し(図2・35)、複数のボックスは4種のアローによって接続される。また、これらボックスは階層化構造を取るところが大きな特徴であり、一般的なモデルでも4~5階層のモデルとなることが多い。そのため、これらの関係の把握と修正にはコンピュータの機能の活用が不可欠となる。IDEFの記述は単純で記述方法もシンプルだが、モデル化の概念、モデルを記述するうえでの文法、用語からレビューの方法などは全プロセスの広い範囲にわたって規定されている。IDEF手法を用いることにより、さまざまな諸活動を普遍的な形で表現でき、システムの設計、構築、運用を考える際の共通の理解が得られ、システムの改善も効率的に行われる。IDEFは当初、構造化分析手法としてアメリカ空軍がICAM(integrated computeraided manufacturing)プロジェクトで開発したものだが、現在は標準記述言語として使用されるようになっている。IDEFは数種のファミリーをもっており、0はその中でも最も活用されている手法となっている。IDEF0は、業務のプロセスや生産のプロセス等を正確にまた視覚的にわかりやすく記述でき、複数の間での共通の理解をもたらす。しかし、IDEFはシステムをモデル化しビジュアルに表現するためのツールであって対象を限定するものではない。図2・36は物流システムを構築する課程で設計、システムのモデル化の手段としてIDEF0を活用したものである。こ図2・35 IDEFの表現方法図2・36 IDEFの記述例7―23うした物流システムの構築は複雑多岐なエンジニアリングプロセスを踏んで、クライアントと仕様の詰めを行うための多大な作業が存在する。このため、これらクライアントとのシステム設計上のやりとりに共通言語としてIDEF0を活用し、システム仕様書の一部として活用したものである。
