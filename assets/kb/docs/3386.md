# 2・3・4 物流計画のための手法

> **章**: 第2章 設計・評価技術
> **節**: 2・29 に新たな生産システムを構築しようとする企業が加工、組立、

a.調査分析手法 物流システムの計画に長年取組んでいる人
は従来は頭の中で経験というツールによって現状のデータから物流
システムを分析し，評価してきた．しかしシステムが複雑化する中
で，これらの人の行動を論理的にかつ多方面から支援するための分
析ツールが提唱されている．またこれらは経験の浅い技術者にとっ
ても有効で計画手法の伝承には不可欠となる．
工場内に何（形状，重量，質ほか）がどこからどこへどれだけ流
れるかを分析しないと工場内の物流計画は立てられない．このため
に物量を算定したり，分析するためのツールの代表例を表2･9に，
図2･31にその事例を示す．
b.シミュレーション技術 シミュレーションを行う場合，は
ん用プログラミング言語やシミュレーション言語を活用した専門的
知識が必要な時代から，現在では搬送車や生産機械，倉庫のシンボ
ルを張り付けてその間をつなぎ，必要な与条件を入力するだけで活
用が可能なものも出てきている．こうした中でシミュレーション技
術が求められる理由として，
･物流システムの構築が複雑になっている
･他のシステムとの連携が複雑になっている
･計画時間の短縮を求められる
･生産ラインの寿命が短くなり頻繁に計画が必要
･トップへのシステム投資物のビジュアルな説明
などのニーズが出てきており，このためシステムの構築前に設計の
検証を行い，手戻りを少なくするためにシミュレーションによる事
前評価が設計行為の中で重要な作業となっている．図2･32に加工
と組立の間の搬送をシミュレートした例を，また図2･33に半導体
製造ラインの例を，また図2･34には計画初期段階でトップへの提
案のためのビジュアル表現を行った例を示す．
