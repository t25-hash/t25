# 2・5・4 コンピュータシミュレーションの仕組み

> **章**: 第2章 設計・評価技術
> **節**: 2・5 節においては、上記のようなさまざまなシミュレーション技

コンピュータシミュレーションでは対象システムのモデルをコンピュータプログラムとして記述する。プログラムははん用プログラミング言語により記述することもできるが、シミュレーションの実行に必要な機能を内蔵したシミュレーション言語や特定のシステムのシミュレーションに特化・専用化した専用シミュレーションシステムなどを用いて、プログラムを構築することができる。このようにユーザが構築したプログラムは、外部モデルであり、構築の容易さやモデルの理解のしやすさに重点がおかれている。外部モデルは実際のシミュレーションの実行に関わる内部モデルに変換されるものであり、実際のシミュレーションの機構や演算手順に即しているとは限らない。シミュレーションの実行の仕組みは以下のとおりである。コンピュータシミュレーションを行うに当たって、対象システムのどのような状態変化に着目し、その変化の仕組みをどのように記述するかによって、構築するソフトウェアモデルとしてのプログラム、特に内部モデルに関わる部分は異なったものとなる。ここでは状態変化の取扱いについて概観した後、離散型シミュレーションについてその実行手順を中心に述べる。a. 状態変化の基本的な取扱いシミュレーションはシステムの状態変化を時間の経過とともに観察できる仕組みに基づいて行わ7―30生産システム工学れる。したがって、時間の経過とともに連続的に生起するシステムの現実の状態変化をどのように表現するかが問題となる。シミュレーションの目的によって、連続的な状態変化をそのまま忠実に取扱わねばならない場合と、連続的変化を捨象して離散的な時点において生起する状態変化に着目すれば十分な場合とがある。これに対応して、ソフトウェアモデルにおけるシステムの状態変化の仕組みを表現する方法は連続型、離散型の二つの基本型に大別できる。連続型シミュレーションでは、制御システムにおける状態変化や速度、加速度の変化等のように、状態変化の機構が微分方程式系で記述される場合を取扱うことが多い。従来では、アナログコンピュータの適用分野と考えられてきたが、時間増分を非常に細かくとることにより、ディジタルコンピュータで実行する方法が一般的となっている。生産システムにおいて、機械加工中のワークや動作中のロボットの形状の変化、搬送中のワークや移動中の搬送台車の空間的な位置を対象とする場合には連続型となる。しかし、形状変化や位置変化などの連続的な状態変化を捨象し、ワークなどに対する作業に着目すると、加工作業や運搬作業といった活動の離散的な時点における一連の変化をシステムの状態変化に関連づけることができる。これらの活動は必要に応じてさらに細分化することも可能である。また、活動の開始と終了時点に起こる状態変化に着目することもできる。このような開始、終了といった特定の離散的な時点に状態変化を生起させるものを事象(離散事象、イベント:event)と呼ぶ。このように、離散的な時点におけるシステムの状態変化に注目してモデルを構築し、実行するものが離散型シミュレーションである。また、事象に着目することから事象型、離散事象型、事象駆動型シミュレーションなどとも呼ばれる。b. 離散型シミュレーションの実行手順生産システムのシミュレーションにおいては、連続型、離散型のシミュレーションが用途により使い分けられている。以下では、離散型シミュレーションを実行する手順を中心に考える。離散型シミュレーションでは、対象システムの状態変化は何か事象が生起したときにのみ起こると考える。したがって、さまざまな事象が実世界で生起するのと同じ順序で時間の進行に伴って生起するような仕組みをプログラムとして実現できれば、実世界における状態変化をモデル上で再現することができる。このような考えに基づいて基本的なシミュレーション実行手順を考えると、図2・53のように表すことができる。シミュレーションプログラム内における時間の進行を管理するシミュレーション時計の進め方により次の二つに分類できる。(1) 離散型シミュレーションの進行手順(時間管理)(1)の方式は離散型シミュレーションに対して一般的なものである。(2)は連続型シミュレーションに対して一般的であり、生産システムにおいて搬送台車の空間的位置の連続的な変化を考慮に入れる場合などにも用いられる。図2・53に用いた基本的な用語は以下のような内容のものであり、その相互関係は概念的に図2・54のように示される。事象表:現在時点で将来生起することが確定している事象をまとめた表である。その生起予定時刻と事象の種類、生起場所などが属性として記述される。新しい事象が生起したり、予定事象が確定すると更新される。状態表:対象システムの構成要素の現時点における状態を記録した表である。事象の生起に伴って参照されるとともに、状態の変化が起こると表は更新される。条件集合:事象の生起に伴って起きる状態変化や、その結果として次に生起する事象等を決定するために必要な条件や決定規則を整理したものである。対象とするシステムによりその内容は異なる。生産システムの場合には、加工順序を決定する差立規則、搬送先の決定規則、その他さまざまな運用規則等が含まれる。シミュレーション時計:シミュレーション実行中における現在時刻を記憶する。時間の進行はシミュレーション時計に基づいて管理される。初期化:シミュレーションの開始時点における状態にプログラムを設定することであり、事象表、状態表、シミュレーション時計の時刻などを開始時刻の状態に設定する。乱数発生装置やシミュレーションの結果を解析するための統計諸量の初期化、シミュレーション終了条件の設定等も必要である。図2・53(a)に示した可変時間増分方式によるシミュレーションの実行手順は、シミュレーションがある程度進んだ時点で考えると以下のようになる。ステップ1:で事象表に登録されている生起予定事象の中から最も早く生起する事象(最早事象)を探索する(ステップ4であらかじめ最早事象を探索しておくこともできる)。ステップ2:シミュレーション時計を最早事象の生起時刻に進める。ステップ3:生起する事象の種類によって次に行うべき処理内容は多様である。条件集合の中から適切な条件や決定規則を選び、それを適用することにより、システムの状態の変化を決定し、状態表を更新する。さらに、次の生起予定事象と時刻を決定する。シミュレーション結果に関する統計諸量の計算も行う。ステップ4:決定した次の生起予定事象と時刻を、事象表に登録する。シミュレーション終了条件を満たしていなければステップ1に戻る。なお、ステップ3で次の生起事象や時刻が設定できない場合も多いが、その場合には終了条件を調べた後にステップ1に戻る(生起予定事象の事象表への登録時に、生起順序も併せて決定しておくことにより、ステップ1の最早事象探索は簡略化できる)。上記の処理過程からも明らかなように、可変時間増分方式の場合のシミュレーション時計は、時刻からのように相続く事象の生起時刻に応じて進められ、その増分は一定していない。したがって、シミュレーションの実行時間は生起する事象の総数に依存する。図2・54 離散型シミュレーションの仕組みこれに対して、固定時間増分方式の場合には、シミュレーションの目的に合った最小時間増分単位を設定し、図2・53(b)に示すように時計が進むごとに生起事象の有無を検討する。活動を直接的に取扱う場合には、活動の開始、終了が生起し、状態変化が起こるかどうかをごとに探索し、このような活動が発見された場合にのみ可変時間増分方式と同様なステップ3,4の処理が行われる。ただし、次に生起するのは事象ではなく活動となる。連続型のシミュレーションの場合には毎回なんらかの事象が生起し、状態変化が起こることとなる。この方式では、すべての事象がの倍数の時刻に生起せねばならないという制約を受けるため、の設定値によりシミュレーションの結果は影響される。したがって、の設定に際しては対象システムの特性とシミュレーションの目的を十分に考慮することが重要である。また、シミュレーションの実行時間はシミュレーションの期間とに依存し、可変時間増分法の場合よりも長くなるのが普通である。
