# 2・5・5 製造業の情報化と分散型シミュレーション

> **章**: 第2章 設計・評価技術
> **節**: 2・5 節においては,上記のようなさまざまなシミュレーション技

図2・49に示したようなバーチャルファクトリーの実現は製品開発環境の整備を推進するものであり,離散型シミュレーションは製造ラインの開発に活用されている.また,製造部門においても,各生産ショップなどにおいてはシステムの改善や効率的スケジューリングのためにシミュレーションが用いられていることはすでに述べた.企業内外における情報ネットワークの整備が進み,高速・大量の情報流通を活用した意思決定の重要性が高まっていることから,従来からのシミュレーションの応用に加えて,例えば,以下のような意思決定支援にシミュレーションを効率的に適用できる環境を整備することが望まれている.(1) サプライチェーンにおいて,要求された納期に間に合う製品を要求された価格で提供できるか.(2) 新生産ラインを導入する場合,工場内の他の設備との能力的バランスや操業面から見て運用上の問題はないか.(3) あるエリアで作成したスケジュール,あるいは故障等により生じたリスケジュールは,前後の加工エリアにどのような影響を及ぼすか.(4) 新製品に対する生産ライン群を複数の企業に発注した.開発された個別システムは,全体システムとして整合が取れているか.(1)~(3)のような状況に対応するためには,大規模な工場全体のシミュレーションをある程度詳細に行うことが必要となる.また,(4)の場合には個別に構築されたシミュレーションモデルを統合し,全体システムのシミュレーションを行う必要が生じる.これらの要請に応えるものとして,分散シミュレーション(distribut-ed simulation)技術の適用が考えられている.分散シミュレーションにおいては,対象システムをいくつかのサブシステムに分割し,各サブシステムのシミュレーションモデル(以下シミュレータと呼ぶ)をネットワーク結合した計算機群に割当て,全体として統合されたシステムとして作動するようにしたものである.生産システムに対しては,図2・55に示すように対象とする工場あるいは工場群を加工エリア,組立エリア,素材・部品・完成品倉庫群,エリア間搬送システムなどのサブシステムに分割し,それぞれのシミュレータを個別計算機上に構築し,それらを統合することにより全体システムのシミュレーションを実行する.必要に応じて生産管理システムを組込むことも考えられる.このような分散シミュレーションの実現のためには,サブシステムのシミュレータ間の通信機能の確立,シミュレーションクロックの同期方法の選択等の問題がある.また,このような環境に適合したシミュレーション言語が必要である.分散シミュレーションをサポートするネットワークシステムとしてTCP/IP(transmission control protocol/internet protocol)に基づく通信システムを開発するのが一般的である.しかし,もともとは軍事用に開発された分散シミュレーション統合アーキテクチャ7―31図2・55 分散型工場シミュレーションHLA(high level architecture)が2000年にIEEEにより標準仕様として採用され,今後これを基盤とする分散シミュレーションの開発が期待される.個々のシミュレータはそれぞれの有する時計(ローカルクロック,LC)に基づいてシミュレーションを実行する.このため,現実の同時刻における各シミュレータのLCは異なる時刻を指しているのが普通である.システム全体として時刻に関する整合性をとるために全体を統合する時計(グローバルクロック,GC)を定義し,LCとGCの同期をとる仕組み,すなわち同期法を考えなければならない.一般の分散シミュレーションに用いられる同期法としては基本的な方法としてタイムホイール法,ヌルメッセージ法,タイムワープ法等があり,多くの変形が報告されている.しかし,これらの同期法を適用するためにはそれぞれの同期法をサポートする機能を組込んだシミュレーション言語(simulation language)を開発する必要がある.生産システムシミュレーションの場合,多くのシミュレーション言語が市販されている.それらを利用するには,市販のシミュレーション言語に組込み可能であり,生産システムの特徴を考慮に入れた同期法が必要となるため,下記のような同期法が提案されている.一つは大規模システムのサブシステム間を搬送システムで結合するという考えに立つものであり,タイムバケット法と呼ばれる方法である.もう一つは,サブシステム間に存在する倉庫やバッファを共有し,その在庫状態に基づいて同期をとるストレージ法というものが提案されている.
