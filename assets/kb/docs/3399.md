# 2・6・1 生産システム開発プロセスのあり方

> **章**: 第2章 設計・評価技術
> **節**: 2・6 生産システム開発プロセス

製品ライフサイクルの短縮と高品質,低コストが要求される中で,製品を作り出す生産システム開発プロセスは,開発期間の短縮とそのライフサイクルコスト(LCC: life cycle cost)を極小化するような開発体制が求められている.このことを実現するために,図2・56に示すような生産システムの企画から,仕様決定,設計,製作,据付・試運転,初期流動に至る開発プロセスに求められる要件は,新製品開発と連動した源流管理をいかに実行できるかにかかっていると言っても過言ではない.LCCとは,生産システムや設備の開発・購入コストだけでなく,その操業,メンテナンス等の一生涯にわたってかかるコストをいう.新製品開発プロセスとしてコンカレントエンジニアリングの実践の中で,生産システム開発プロセスも製品の設計・開発と連動,オーバラップさせた活動による開発期間短縮の必要性はいうまでもない.そのような短納期の中でシステムを立上げてから問題点が顕在化し,その対応に追われ,安定稼働になるまでに時間を要することは,余分なコストを強いられLCCを増加させる大きな要因となる.同時に技術革新の激しい現代では,売上の機会損失にもつながり,経営上にも大きな影響を与える.源流管理(management at the source)とは,開発プロセスのなるべく前段階でシステムや設備の品質やコストを作り込むことである.製品開発において,品質とコストは設計・開発で80%決まるといわれるように,設備開発についても同様なことがいえる.図トと,LCCを削減するための機会の大きさを,生産システム(設備)開発プロセスとの対応で示した概念図である.生産システム開発プロセスの下流で起きる設計変更ほど手戻りとなり,開発時間を長引かせるとともに大幅にコストが高くなる.逆にプロセスの上流,すなわち源流ほどLCC削減機会は増大し,製作後になるとほとんどコスト削減機会は失われる.以上のことから,生産システム開発プロセスのあり方として,計図2・56 生産システム開発のプロセス(ステップ)画・企画の段階で将来の市場や技術変化の見通しのもとでのLCCの作り込みをしたうえで,プロセスの上流において,システムや設備の潜在的な問題点や不具合を設計・製作以前の段階で摘出し,据付と同時にねらいの品質や稼働率を100%確保できるような垂直立上げの実現がその目標であるといえる.図2・58のような状況から図2・59のような状況の実現である.図2・58は,納期に迫られたシステムの製品品質,信頼性,操作性,保全性,経済性,安全性,省資源性,そして製品の品質保証といった側面からの問題点や不具合が十分摘出・対策されずに据付され,初期流動の期間中にそれらの問題多数が顕在化していることを示す.そのため,なかなか当初のねらいの設備総合効率(overallequipment efficiency,時間効率×性能効率×良品率)が達成できずに,初期流動解除までに多大な時間を要している.当然のことながら,多大な設計変更や対策のコストを費やすだけでなく,その間の生産量減に伴う機会損失も著しく大きい.一方,図2・59は,全体的に問題点摘出の曲線が図2・58に比べて開発プロセスの上流にシフトしているように,問題点の前出しがうまくでき,据付運転開始と同時にねらいの設備総合効率に短期間で達している.このような初期流動期間を短期間で解除させ,安定生産に移行できる垂直立上げができている.新製品開発と連動したねらいの納期で垂直立上げができるような生産システム開発を行うためには,まず生産技術や設備開発部門だけでなく製品開発,製造等の関連部門や設備メーカとの連携に基づく情報共有が必要となる.さらに過去の設備の故障や品質トラブル等から蓄積されたMP情報(maintenance prevention informa-tion)の活用と,設備開発の要所要所でデザインレビューの適切な実施がなされなければならない.また必要に応じて信頼性や品質保証面からの工程FMEA,最適な生産条件を設定するためのパラメータ設計等の品質工学の適用,そしてコスト面からはVE(価値工学)等のツールの活用も必要である.図2・58 後工程での問題点・不具合の発見による初期流動期間の長期化図2・57 生産システム開発における設計変更コストとLCC 削減機会図2・59問題点・不具合の前出しによる開発期間の短縮と垂直立上げこのような問題点の前出しによる垂直立上げを目指した生産システム開発のマネジメントを,TPM(total productive mainte-nance)の分野では,初期管理と呼ぶことがある.
