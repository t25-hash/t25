formanufacturability、製造容易性設計)のノウハウを製品設計に盛り込むことが必須である。加えて新製品開発の各ステップで行われるデザインレビュー(DR : design review)に生産システム開発の観点から参画が必要である。また、b項で述べるQFD も製品開発との双方向のコミュニケーションを図るうえで有用なツールである。b. 仕様決定生産システム開発の構想が決まり、具体的なシステムや設備仕様に落し込む段階では、システムが作り出す製品の品質保証の観点からの検討が必要となる。図2・62は、そのプロセスをQFD(quality function deployment、品質機能展開)の形式でまとめたものである。まず製品の要求品質をリストアップしたものから、それを製品という形で実現するため代用特性との関係をマトリックスで表現したものが品質表である。これを用いて要求品質の重要度を考慮して、代用特性の特性値が決められる。そしてその代用特性値と構成部品の部品特性との因果関係を勘案して部品特性値に展開する(構成部品展開表)。その結果から、特に重要な部品品質特性と品質保証上のポイントを設計から生産技術に伝達するQA(quality assur-ance)表を作成する。このQA 表および構成部品展開表の部品特性から、それを作り出す生産システムについて、構成する工程・設備のリストとの関係を記述したものが工程QA マトリックスである。この工程QA マトリックスを作成することにより、各工程や設備の実現すべき管理特性およびその特性値が決められ、それに基づく工程・設備仕様書が作られていく。その過程で、製品・部品の品質保証や信頼性上特に重要な特性については、この段階でc項に示す工程FMEA を用いて重要故障モードを取上げ、その致命度の大きいものは事前に処置をとるような対策が必要である。また、現在の生産技術では達成できない特性値が要求されるような工程・設備については、ボトルネック技術(BNE)として生産技術そのものを先行して開発することも重要なことである。生産システムの仕様構想が固まり、次に社外設計を含めて基本設計、詳細設計に入る前にデザインレビューを実施する。デザインレビュー(デバッギングとも呼ばれることもある)とは、新製品開発の場合と同様に、仕様や設計の漏れやマイナス要因を前出しするために開発の節目で関連部門が参画して行う審査である。企画、設計、製作というそれぞれの段階で、システム、設備の本来果たすべき機能に加えて、基本的に具備すべき信頼性、保全性、保守性、操作性、安全性、省資源性について漏れや不具合がないか検討がなされる。その際、c項で述べるMP情報やQFD、信頼性解析等のドキュメントを準備したうえで、参加者の固有技術や経験を踏まえた議論が行われ、最終的に次のステップへの移行可否が決められる。c. 初期流動管理初期流動管理とは、設備の据付・試運転の終了後、実際に製品を生産しながら不具合や問題点を摘出し、早期に安定稼働を図る活動である。初期流動管理が解除され、安定生産に入ることで一連の生産システム開発プロセスは終了する。その際、初期に定めた稼働率、良品率、そして設備総合効率をクリアすることが前提となる。同時に、操作・段取標準や給油・点検、定期整備などの保全基準を作成し、教育・訓練の実施とともに製造・保全担当者へ引渡す活動も行われる。無論、源流管理がうまくいき、この段階で不具合が出ずに垂直立上げ(vertical ramp up)ができることが理想である。初期流動管理期間中に発生する機能上、製品品質上、あるいは保全上の問題点については、早期に対策処置をとることが機会損失を最小限にするためにも必要である。同時に、この段階で出てくる不具合が、なぜ源流で摘出できなかったのか原因を明確化し、次の開発への反省点として生かすことが不可欠である。
