# 3・1・3 二つの管理システムの比較

> **章**: 第3章 では生産システムを所与とし、それを運用する管理システ
> **節**: 3・1 生産システム管理の体系

ここで二つの管理システムの違いを整理しておく(表3・1)。a. 納期見積り(due date estimation)両者の大きな違いは納期見積りの精度の違いである。資材手配中心の管理システム(以下、前者と呼ぶ)では、通常、負荷計画に基づいて行われるため、精度が低くなる。一方、生産スケジューリング中心の管理システム(以下、後者と呼ぶ)は詳細な生産スケジュールに基づいて行われるため、必然的に精度が高くなる。b. 生産リードタイム前者の場合、負荷計画はタイムバケットを単位として作成され、タイムバケットと加工時間の差が累積されて生産リードタイムの見積り値は長くなる。また、実際の生産リードタイムは見積り値に合わせたものになる。後者の場合、詳細な生産スケジュールが作成され、オーダ間でのリソースの干渉がなくなり、待ち時間は最小限に抑えられる。また、資材は生産スケジュールに合わせて供給されるため、資材到着待ちから生じるリソースの遊休が避けられ、生産リードタイムは短縮される。必然的に、納期の見積り時点から製品納入までの期間を意味するリードタイムも短くなる。c. 仕掛在庫前者の場合、生産リードタイムが長くなるため、仕掛在庫も増大する。また、共通部品はロットまとめの結果、実際の使用に先行して作られることが多くなり、その量が増えると仕掛品の在庫水準を引上げることになる。後者の場合、生産リードタイ表3・1 二つの生産管理システムの比較資材手配中心の生産管理システム生産スケジューリング中心の生産管理システム基本的な特徴資材手配が生産スケジューリングに先行して行われる。生産スケジューリングが資材手配に先行して、あるいは両者が同時に行われる。納期見積り多くの場合、負荷計画に基づいて行われるため、精度が低い。詳細な生産スケジュールに基づいて行われるため、精度が高い。生産リードタイムタイムバケットと加工時間の差が累積され、見積もられた生産リードタイムが長くなり、実際の時間もそれに合わせたものになる。スケジューリングの段階でリソースの干渉のない計画が作成され、資材がスケジュールに合わせて供給されるため、必然的に短くなる。仕掛在庫生産リードタイムが長くなることは仕掛在庫の増大を意味する。共通部品は一般に先行して作られるため、それが多量になると仕掛在庫は高水準になる。生産リードタイムが短くなることは仕掛在庫の減少を意味する。共通部品の紐付き管理を原則とするので、仕掛在庫の増加が抑制される。原価管理仕掛在庫の増大や変動により、単位期間当たりに発生する材料費、諸経費の算定が難しくなる。仕掛在庫が低水準に抑えられる場合は、単位期間当たりに発生する材料費、諸経費の算定が容易になる。出荷高が売上高に一致する経営を目指しやすくなり、単位期間当たりの付加価値の算定ができる。パッケージソフトMRPあるいはMRPIIスケジューリングソフトとMRP、あるいはAPSデータBOMM-BOM、加工時間ムが短くなるため、それに対応して仕掛在庫は減少する。共通部品の紐付き管理が原則として実施されるので、共通部品の先行生産は特別な場合を除いて実施されず、仕掛品の在庫水準は抑えられる。d. 原価管理前者の場合、仕掛在庫の増大や変動により、単位期間当たりに発生する材料費、諸経費の算定が難しくなる。後者の場合、仕掛在庫が低水準に抑えられ、単位期間当たりに発生する材料費、諸経費の算定が容易になる。さらに、すべての製品が受注に基づいて製造されるならば、出荷額は売上にほぼ一致し、売上から人件費を除く諸経費を差引いたものは、いわゆる付加価値に近いものになる。それを所定の期間で割ることによって生産システムが創出する時間当たりの付加価値(近似値)が求められ、日々の生産活動の経営に対する貢献の程度が測定できる。e. パッケージソフト前者の場合は、通常、MRPあるいはMRPIIが使用される。後者の場合、スケジューリングソフトとMRPが併用されることが比較的多い。その場合、MRPは第2レベル以下の部品の発注に利用される。近年は、スケジューリングと資材手配を同時に行う機能を備えたAPS が用いられ始めている。f. データMRPあるいはMRPIIでは、BOM(bill ofmaterials)と呼ばれる製品構成に関するデータが利用される。一方、APS ではスケジューリングと資材手配を同時に実施する必要から、BOM に加工手順のデータを取込んだM-BOM(manufac-turing-BOM)と称する新しいBOM が利用されることが多い。当然、MRPと違って加工時間のデータは不可欠であり、その作成と保全には大きな労力を要する。
