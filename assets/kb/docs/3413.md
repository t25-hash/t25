# 3・2・8 生産管理システム

> **章**: 第3章 管理システム
> **節**: 3・2 生産計画

MPS が作成されると,このスケジュールから各製品の出荷時期と所要量が判明するので,生産管理者は製品を生産するために必要な原材料の調達,部品や組立品の生産,その他もろもろの活動を計画し,管理しなければならない.この計画と管理の方式は,企業がどのような生産管理システムを採用するかによって異なる.以下では,代表的な四つの生産管理システム:ポンドドレンニングシステム,プッシュシステム,プルシステムおよびボトルネック中心システムについて述べる(図3・6).a. ポンドドレンニングシステムこのシステムの特徴は,生産を支える工程に設定されているストックポイント(原材料,部品や部分組立品,製品などの在庫点)の管理水準をコントロールするところにある.図3・6(a)に示すように,これは最もシンプルな生産管理システムの一つで,顧客-生産者-供給者をつなぐ生産システムを最も少ない情報で結び付けている.このシステムでは,生産者には顧客需要の時期と量があらかじめ知らされていないために,各種の製品を前もって生産し,ストックしておかなければならない.そして顧客から製品の出荷要求があったとき,最終製品在庫ポンドから製品がドレン(排出)される.最終製品在庫が減少すると,前もって作られ,ストックされている部品や部分組立品を使って最終組立が行われる.同様に,部品や部分組立品の中間在庫が減少したとき,原材料在庫から原材料がドレンされ,部品や部分組立品が生産される.そして原材料在庫が減少すると,原材料供給者に対して原材料購入のためのオーダが発せられる.このように,各在庫点をトリガとして生産命令が発せられ,下流から上流へと伝達される.以上は,ストックベース生産工場の場合についての説明であるが,オーダベース生産工場の場合は,最終製品在庫ポイントを顧客オーダバックログに置換えればよい.(a) 四つの生産管理システム7―43情報やコミュニケーション手段が発達した今日では,このシステムの考え方は過去のものになりつつあるが,概念は非常に単純でわかりやすいために,まだ応用的価値は十分あるように思われる.ポンドドレンニングシステムの利点は製品中心生産もしくは工程中心生産のいずれの場合でも応用できることである.そして,顧客,供給者,生産者に対して,有益で少ない情報だけでシステムが機能することである.しかし,欠点も多い.システムそれ自体は過剰在庫を生む危険性をはらんでおり,顧客ニーズに柔軟に対応する能力に欠ける嫌いがある.また,ブルウィップ現象と呼ばれる現象のために,上流に行くほど在庫の変動が増幅されて,極端な在庫過剰/不足をきたしてしまうことがある.b. プッシュシステム(push system)このシステムの特徴は,顧客,原材料/部品供給者,生産者に関する情報から詳細なスケジュールを作成し,それに基づいてマテリアルフローを前進させる方式を採るところにある.原材料や部品がいつごろどれだけ必要かを予測し,その時点前に到着するように計画する.部品や部分組立品は最終組立がいつ行われるかを予測して,必要な時期に供給できるように計画する.同様に最終製品についても,顧客が必要とするときまでに組立てられて出荷される.このようにプッシュシステムは原材料が工場の後門から中へ入り,上流から下流に向かって次々と押出されて,最後に製品となって工場の前門から出荷されていくシステムである[図3・6(b)].プッシュシステムがうまく機能するか否かは,生産スケジュールの正確性に依存する.さらに,生産スケジュールの正確性は顧客需要情報の精度やリードタイムの正確性に関係している.このようにプッシュシステムでは生産スケジュールが重要な鍵となる.プッシュシステムの代表例としてMRP(material requirements planning,資材所要量計画)が挙げられる.これはエンドアイテムに対するMPS を基に,エンドアイテムを構成するすべての部品や原材料の所要量をBOM(bill ofmaterials)に基づいて計算し,在庫情報を考慮しながら各部品や原材料の正味所要量と所要時期を決定する方法である(図3・7).この計算には,データが膨大になるためにコンピュータ支援が必須で,多くのコンピュータソフトウェアがパッケージされて用意されている.現在,MRPはさらに拡張されて企業の経営管理全般を取扱うMRPIIやERPに発展している.c. プルシステム(pull system)このシステムは生産のすべての段階においてWIP(work-in-process,仕掛在庫)を低減させることに主眼をおくところに特徴がある.具体的には,プルシステムは次工程のみに着目し,そこで必要なもののみを決定し,それのみを生産する.すなわち,必要としないものは生産しないし,また必要としないものを次工程にも送らない.もし生産の必要性が生じた場合,次工程の作業者がWIPを引取りにくる.このようにすれば,製品は上流工程から下流工程へ最小のWIPの下に流れることになる.このように,工場の後門に到着した部品や原材料は次工程の作業者によって順次引取られていき,最終的に製品となって,図3・7 MRPコンピュータプログラム構造7―44生産システム工学工場の前門から顧客によって引取られていく.現在,プルシステムの代表例としてJIT(just in time)がある.これはトヨタ自動車が開発した生産管理システムで,トヨタ生産方式と呼ばれている.この方式では,まず生産管理者にロットサイズの低減が要求される.そして,WIPを低水準に保ちながら,すべてのマテリアルが品質水準を満足し,約束の時間に,かつ所定の場所に正確に到着することが必要となる.そのためには,すべての機械が故障せずにその機能を十分果たすことができなければならない.これはかなり厳しい課題である.また,生産管理者は顧客の要求納期を満足させながら,できるだけ生産が一定のレベルで行われるよう要求される.これには生産管理者が現場を熟知していること,先取的戦略をもつこと,ある程度顧客需要をコントロールできる権限をもつこと,などの条件が必要になる.図3・8にJIT 生産によって生まれる波及効果を示す.プルシステムにおいて,マテリアルフローをコントロールする方法としてよく知られたものに,かんばん方式とCONWIP(con-stant WIP)がある[図3・6(c)].かんばん方式はかんばんを情報伝達の手段として用いるもので,かんばん枚数をコントロールすることによって,マテリアルフローをコントロールすることができるという考え方に基づいている.また,CONWIPは生産工程内のWIPをできるだけ一定に保つことを主眼におく方法で,生産工程へのマテリアルのインプットとアウトプットを同期化させる考え方である.この同期化の最も単純な方法は,WIPにある限界を設けておいて,WIPがその限界を超えたときはいつでも生産工程へマテリアルを投入しないようにすることである.CONWIPはこのプロトコルを採用している.d. ボトルネック中心システムこのシステムの代表例として図3・8 JIT 生産の波及効果図3・9 OPT の基本モジュール図3・10 生産システムOPT(optimized production technology)がある[図3・6(d)].OPT では,初めに,生産する製品群が決まると,生産工程の中でボトルネックとなる工程を見出す.もし製品が流れ生産ラインで作られる場合,ボトルネックでない工程で,いかに速く生産を行っても,ボトルネック工程がそのラインの生産能力を支配するために,在庫を作るばかりである.OPT はこの点に注目し,ボトルネック工程に対しては生産に支障をきたさないようにするために,作業者,機械,ツールを抜かりなくスケジュールするための特別なアルゴリズムを採用している(図3・9).OPT の根底に流れる思想は制約条件の理論(TOC: theory ofconstraints)である.別名を同期化生産(synchronous produc-tion)と呼ぶ.これは組織を構成するすべてのメンバは同じ組織目標に向かって仕事をしなければならない,という考えからきている.またOPT 思想の中心課題は生産パフォーマンスの継続改善(continuous improvement)である.これまでの単位コストや機械・作業者稼働率などの伝統的な評価尺度で生産パフォーマンスが測られるのではなく,スループット(製品の販売によって得られる金額),在庫(在庫に投資される金額),オペレーションコスト(在庫をスループットに変換するのにかかる費用)が生産パフォーマンスを測る新しい尺度として用いられる.その基本はスループットを増やして,逆に在庫とオペレーション費用を減ずることにある.OPT の管理方法はドラム,バッファ,ロープの概念に基づいている.すなわち,生産はコントロール点,すなわちボトルネックでコントロールされ,ボトルネックは集団を統制するドラムの役割を担い,その他の工程はドラムが打ち鳴らすビートに従って行動する.同時にドラムはボトルネック工程の生産スケジュールと符合した,MPS のような役割をもつ.在庫の形態をとるバッファがボトルネック工程の前に設定されて,つねにマテリアルの供給が途絶えないように図る.このようなバッファの設定によって顧客納期が高い信頼性を遵守することが保証される.ロープは生産スケジュールのようなコミュニケーションの手段としての役割をもつと同時に,在庫が増えるのを阻止するためと,MPS を遵守するために下流工程から上流工程にさかのぼって情報を伝達するための二つの役割をもつ.ロープはすべての工程がMPS と同期化されていることを保証している.作業管理(work management)とは,「作業方法の分析・改善によって,標準作業と標準時間を設定して,この標準を維持する一連の活動体系(備考:作業管理に必要な基礎技術を作業研究または方法研究という)」(JIS Z 8141:2001,-5101,-5103)とされている.管理対象が図3・10に示す生産システムであり,その中に管理システムと製造システムが包含され,そのいずれにも作業という活動が存在することを考えると,次のように読み替えることができるだろう.必要な機能,目的を達成するために有形・無形のものに物理的・化学的変化あるいは付加価値を与える人・機械の諸活動である作業を,設計・計画・調達・実施・統制・評価する活動の総称が作業管理である.
