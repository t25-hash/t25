# 3・4・2 生産統制の内容

> **章**: 第3章 管理システム
> **節**: 3・4 生産統制

生産計画が確定すると、設備・人員・材料についての手配を行い、日程計画で作業の開始・完了時刻が明らかになると、生産計画活動の状況を見ながら、各工程あるいは作業者に作業の準備や開始・終了時刻を指示することになる。この作業指示のことを差立(dispatching)ともいい、作業開始前に、資材、治工具や機械などが生産実施に支障のないように準備・整備されているかどうかをチェックし、さらに現状の負荷および余力を確認してから指示される。注文生産のように追加注文、緊急注文の到着、計画変更などの外部変動や、機械故障や加工時間見積誤差などの内部変動が無視できず、時々刻々に環境が変更するときには特に差立が重要な意味をも7―47図3・15 生産計画と統制図3・16 ガントチャートによる進度管理図3・17 生産進度図つことになる。経験豊富な担当者が逐次状況を判断して差立を行ったり、いくつかの差立ルール(dispatching rules)から状況に応じて次に差立てる優先規則を決める適切なルールを選んで差立が行われる。代表的な差立ルールとしてFCFS(先入れ先出し)、SPT(最小加工時間)、EDD(最早納期)、SLACK(最小納期ゆとり)などがある。具体的な指示方法は作業票、出庫票、検査票、移動票などと呼ばれる伝票を発行したり、関連する部署にコンピュータの端末機を配置して端末機を通して指示をしたりする。また、後述するかんばん方式(Kanban managing system)あるいはトヨタ生産方式(Toyota production system)で有名になったかんばん(移動式表示板)も伝票の変形であるといえる。いずれの手段を用いるにしろ、指令の最末端が差立であり、あとは実行あるのみということになる。これらの伝票の発行あるいは指示は生産計画の最終ステップであると同時に、生産統制が始まることになる(図3・15)。図3・16は注文生産の場合、ガントチャート(Gantt chart)による進度管理を表している。実線は予定であり、点線は実績を表す。設計で遅れが発生し、部品加工は設計で遅れた分だけ予定が遅れ、組立で少し遅れを取り戻し、検査で計画どおりに完了したことがわかる。連続生産の場合には、一定期間ごとに生産数の累計を調べることになるので、予定表を兼ねた進度表を作ることによって進度管理ができる。図3・17は横軸に1箇月の日付(休日は除く)を、縦軸は7―48生産システム工学累計生産数を表している。縦軸の予定月産数の線と横軸の月末の線との交点をとれば、これが月末の到達点であり、これと原点とを結べば生産予定線となる。次に毎日の生産実績の累計をプロットし、実績が予定線を下回っていれば遅れを意味し、予定線を上回っていれば進みを意味する。例えば、16日時点ではの長さ、つまり3日間の遅れがでており、の長さ、つまり、120個だけ遅れがでていることがわかる。前述したとおり、実際の生産活動はいろいろな原因で計画どおりには進まず、計画と実際との間に差異が生じる。生産統制はこの差異を発見して、計画どおりに近づくように調整することであるから、生産統制の内容は生産活動の状況を把握して差異を発見することから始まり、差異を小さくするように処置することで終わる。生産統制は生産計画に対応している。生産計画における工数計画(または負荷計画)に対して余力管理、日程計画に対しては進度管理(または進捗管理)、資材・仕掛品および製品の流れに対して現品管理が対応する。生産活動には物の流れと人(機械)の物への働きかけという二面があるから、生産統制の内容も物の流れの統制と人の動きの統制という二つがある。前者の物の統制に関しては、さらに動的(進行的)な面の統制(進度管理)と静的な面の統制(現品管理)とがあり、後者は作業者や機械設備の稼働に関する統制であり、仕事量と能力との調整を行うものである(余力管理)。最後に生産結果は生産活動情報として収集・処理し、関連部署に報告したり、その後の計画や管理に役立てるための情報管理が必要となる。a. 進度管理(progress control、進捗管理)進度とは、仕事の捗りのことをいう。生産活動における進度管理とは、何がどの工程まで、どのくらいの数量、加工が完了したかをチェックして、日程計画で決められた線に従って時間的進行面の統制を行うものである。進度管理は納期管理ともいわれ、生産統制機能の中で最も重要なものであり、その目的は納期の確保と仕掛り量の減少にある。進度を調査する場合、次の二とおりの考え方がある。1どこ(何工程)まで進んだか:過渡的進度2どれだけ(何個)できたか:数量的進度1は注文生産の場合の考え方で、一つ一つの物の加工手順が異なる場合、それぞれの物がどの工程まで完了したか、あるいはどの工程で仕掛かっているかについての進度を調査することになる。2は連続生産の場合の考え方で、常時、工程を同じ物が流れている場合には、各工程での進度を考える必要はなく、生産量の累計のみを把握すればよい。ロット生産の場合には、1か2のいずれかの方法をとることになる。b. 現品管理(WIP: work in process control、仕掛り管理)実際の生産で計画に比べて遅れが生じたり、加工中に不良や破損が生じたりすると、あるべき数量と実際の数量とに差異が生じる。このような食違いを放置しておくと、予定どおりの生産を達成できなくなったり、品切れが生じたりする。そこで、各工程に流動している材料、部品、製品などの所在位置と数量を確認し、過不足があった場合には、その原因を調べると同時に対策を打たなければならない。現品管理とは、どこに、何が、どのくらいあるかを適確に把握することである。c. 余力管理(load control、負荷管理)生産計画の段階で能力と負荷とのバランスがとれていても、生産を実施する段階では、機械の故障、作業者の欠勤、前工程からの材料遅れ、追加注文あるいは飛込みによって相対的に能力が不足すると、生産計画で決めた負荷を処理できなくなるし、逆に計画変更によって生産予定量が少なくなったり、注文取消しがあったりすると作業者や機械に遊びが生じることになりコストが増大する。余力管理とは、各工程あるいは個々の作業者について、現在の負荷状態と現有能力とを把握し、現在どれだけの余力あるいは不足があるかを検討し、作業の再配分を行って能力と負荷を均衡させることである。生産統制を上述どおり、進度管理、現品管理、余力管理に分類したが、これはあくまでも、説明の便宜上の分類であって、実際の生産統制はこれらが一体となって行われる。つまり、計画と実際との差異は一般に進度の遅れあるいは生産量不足という現象として表面化するから、進行状況を把握することによって差異を発見することが多い。この進行状況を把握するためには、各工程に流動している材料、部品、製品などの所在位置と数量を確認するために現品の管理が必要となる。また、進度状況の調査の結果、差異が生じた場合、あるいは、機械故障や作業者の欠勤によって差異が生じると予想される場合には、当然ながら能力と負荷とにアンバランスが生じる。そこで、どのくらいの能力の過不足が生じるかの把握は、差異を小さくするための対策を打つ具体的手段を見出すために行うのだと考えることもできる。つまり、納期を維持し、作業者や機械を有効に活用して計画どおりに生産するという目的を達成するために、進度の把握と現品の確認によって計画と実際との差異を明らかにし、処置をとるために能力の過不足を明らかにするのである。d. 生産実施情報管理生産が完了すると現場からは原始データとして作業票、移動票、出庫票、検査票などの諸伝票に実績値が記入されて戻されてきたり、現場に配置している端末機を通して実績値が入力される。また、このほかに日報、週報あるいは月報として生産実績や不良数量などの情報が定期的に報告される。これら生産に関する内部情報は、次のような目的に利用される。1作業の遅れや生産数量不足を把握してその処置をとる。2次期生産計画を作成するための基礎情報を提供する。3原価計算を行う基本情報とする。4品質、資材などの諸活動の業績評価をしたり計画の基礎情報を提供する。おもな生産実績情報には次のようなものがある。(1) 総括的な生産実績(生産活動を総括的に把握した情報)・製品別の完成数量・製品別・工程別の仕掛量・新規受注件数(数量)および手持ち件数(数量)(2) 人員関係に関するもの・総人員(延べ欠勤人員、延べ出勤人員、出勤率)・職場別、工程別の人員(欠勤人員、出勤人員、出勤率)(3) 時間に関するもの・製品別作業時間実績(定時と残業)・工程別作業時間実績(定時と残業)(4) 日程に関するもの・職場別遅延件数・製品別、部分別遅延件数(5) 不良に関するもの・製品別不良件数(不良率)・工程別不良件数(不良率)(6) 設備に関する物・設備別稼働時間(稼働率)・設備別故障時間(故障率)(7) 材料の消費に関するもの・材料別の消費量(直接材料、間接材料)・動力消費量
