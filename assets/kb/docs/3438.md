# 8・1・3 生産機能と生産情報のオープン化

> **章**: 第8章 社会と生産システム
> **節**: 8・1 生産システムのオープン化

a.生産システムモデルによるオープン化 通信機能の相互接
はできない．製造業には多くの機能が存在しているからである．そ
これらの機能を明確にして，ソフトウェアのマルチベンダ環境に
よる相互接続を提案しようとする試みがなされている．この試み
図8･1ネットワークシステムの構成例
ためにシステムを記述したI -CAM（integrated computer aided
manufacturing）プログラムによって始められた．IDEFと呼ばれ
る構造化図式モデル法が用いられた．その後，CIM（computer
integrated manufacturing）のアーキテクチャとして，CIM/ OSA
(open system architecture) がEUで提案された．
これらの提案は，製造業の基本的な構造や機能は共通であり，特
定の方法や枠組を決めることで相互運用が可能であるという考え方
で進められてきた．しかしながら，製造業は，製品や工程や市場の
状況でかなり異なり，一般的なモデルを作成することは困難であ
る．
図8･2 PSLXによるスケジューリングデータの交換例
そこで，最近では，オブジェクトモデリング手法を用いて，業種
ごとや機能ごとに柔軟にモデル化する方法が採られてきている．そ
の例として，半導体製造業に対象を特定したモデルがある．それ
は，SEMATECH モデルと呼ばれるもので，製造機器のインタ
フェース情報などの詳細な部分までをモデル化した例として評価さ
れている．
同様な方法で，組立製造業のシステムを定義しようとする試み
が，ISA（Instrument Society of America）においてなされ，
ISA-95 という規格として，国際標準化機構（ISO）に提案され
ている．
今後，製造業の操業にソフトウェアの比重が高くなるにつれて，
生産システムの機能を明確に記述して，第三者に開示していくこと
で，オープン化を促進していくことになる．
b.基幹システムのオープン化 生産管理の基幹システムは，
MRP（material resources planning）からERP（enterprise
resource planning）へと進歩してきている．ERPシステムとし
て，SAP社やBAAN社のシステムを導入する企業が増えてきて
いる．特に，薬品や化学工業製品を生産している企業では，その傾
向が顕著である．ERPは，基本的に，生産計画や受注などのデー
タをデータベースによって結合し，生産管理と経理あるいは生産管
理と営業，製造と品質管理など企業内のデータ連携を図るものであ
る．基幹システムを統合化することで，一企業内の複数のシステム
間の情報の共有を通して，生産情報のオープン化が行われてきてい
る．標準化によるオープン化ではないが，SAP社とBAAN社の
市場占有率の高さから，それぞれの構造に合わせたソフトウェア群
がアライアンスを組んだ企業から提供され，デファクトスタンダー
ド化されてきている．
c .スケジューリングデータのオープン化 スケジューリング
（scheduling）は，それぞれの企業の独自データと独自のアプリケ
ーションで作成されるのが基本である．これは，それぞれの設備や
能力などが含まれているので，公開がちゅうちょされるものであ
る．しかしながら，昨今の製造業では調達のオープン化やアウトソ
ーシングが進み，工場自身，あるいは自身の工場が管理できるとこ
ろのみで生産するのが困難になってきている．また，スケジューリ
ングの不確定さを補う在庫を十分にもつことも困難になってきてい
る．このような状況で，工場の壁を超えて，あるいは企業の壁を超
えてスケジューリングデータを共有する必要が生じてきた．
例えば，XML（extensible markup language）を用いて，スケ
ジューリングデータをウェブを介して交換する仕様がある．この仕
様 は，PSLXコンソーシアムが公開している．これは，グラン
ドデザイン，XML記述規約，用語などからなっている．図8･2に
データ交換の例 を示す．
XMLによるデータ交換は，交換すべきデータの記述に用いら
れ，処理に関する記述を制限しない特徴があるので，すでに存在し
ているソフトウェアなどの資産を有効に活用して，データ交換が可
能になる．そのため，スケジューリングデータのみならず今後のオ
ープン化のツールとして注目されている．
d.MESによる製造データのオープン化 生産管理や計画分
野は基幹システムやデータのオープン化が進み始めている．また，
工業用ネットワークを通して機器のオープン化も進んでいるが，製
造そのものの実態を表現するデータを機器間あるいは生産管理や生
産計画と結びつけることは難しい．そこで，ERPと現実の設備群
の間にMES（manufacturing execution system）を介して，実績
情報，制御情報を結びつけることによって，計画系，管理系と設備
系システム間に存在していた壁を取り除いてオープン化しようとす
る試みもなされている．
MESでは，UML（unified modeling language）を用いて，設
備やコントローラを仮想設備として記述し，CORBAなどの共通プ
ラットフォームを用いてデータ交換を行おうとしている．この仕様
は，(財)製造科学技術センターにおいて，Open MES仕様書 と
して公開されている．
