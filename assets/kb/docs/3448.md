# 3・8・2 設備保全管理のフレームワーク

> **章**: 第3章 管理システム
> **節**: 3・8 設備管理

設備の運用中には,種々の不具合が発生する.これは,設備の機能が,ユーザの要求レベルを満たさなくなるためであるが,その原因は,以下の4項目に大別される.(1) 使用環境,ニーズの変化使用環境の変化や要求機能の変化により,設備が実現できる機能が要求機能を下回ってしまう.(2) 設備に作り込まれた欠陥設計不良,製作・据付不良,あるいは,材料,部品,機器等の不良により設備に故障が発生する.(3) 強制劣化運転条件,使用条件が不適正であるために,設備の劣化が加速され,故障が発生する.(4) 自然劣化使用に伴い不可避的に生じる劣化により故障が発生する.これらの問題に対処するのが設備保全管理の目的である.(1)への対応が設備改良,(2)(3)への対応が設備改善,(4)への対応が設備維持活動となる.欠陥や強制劣化の排除のための改善活動については,本来は,設備の立上げ時にこれらの問題が解決されていることが理想であるが,現実としては,保全活動の多くの部分を占めており,また,その効果は大きい.(1)の使用環境,ニーズの変化を除くと,(2)~(4)は,いずれも設備に生じる故障が問題であるので,その対応が設備保全管理の主要な課題となる.設備保全管理においても,PDCA(plan-do-check-action)の図3・29設備ライフサイクル図3・31 保全方式の分類(JIS Z 8115:2000)管理サイクルを回すこと,すなわち,保全方式を選び,それに基づいて保全作業を実施し,その結果を評価し,保全方式の見直しにつなげていくということが基本となる.この考え方に基づいた設備保全管理のフレームワークを示したのが図3・30である.図3・30には,基本保全計画を軸とした三つの管理サイクルが示されている.最も内側のサイクルが,運用段階での保全作業実施のサイクルである.ここでは,基本保全計画で決定した保全方式に基づいて保全作業計画を立案し実施する.保全作業の結果は,基本保全計画時に想定された状況(劣化の進行や故障の発生形態など)に照らして評価される.結果が想定した状況の範囲であれば,次の保全作業計画に移り,作業の実施・評価が繰返される.一方,選択された保全方式に不都合が認められたり,予想していなかった問題が現れたりした場合は,基本保全計画に戻り,運用段階で得られたMP情報を考慮して保全方式の見直し・改訂を行う.これが,保全管理の第二のループである.基本保全計画において,改良保全が適当と判断された場合には,さらに左側に戻って設備の改良が行われる.これが第三のループになる.このように,基本保全計画を軸とした管理により,計画的な保全の実施と,環境の変化やMP情報のフィードバックに基づく保全方式の更新と改良が可能となり,ライフサイクルを通じた設備状態の維持と改善の実施という設備保全管理の目標が達成できる.このような設備保全管理を実現するためには,種々の技術が必要となるが,以下ではそれらの中でも特に重要と考えられる,保全方式の決定などを行う保全計画技術と,実際の設備状態を把握するための設備診断技術について述べる.また,生産設備に関しては,設備管理活動の改善が生産効率の向上に大きく貢献する.そこで,3・8・5項では,設備管理を基礎とした改善活動として知られているTPM の概要を説明する.
