# 4・1・1 生産システムの構成

> **章**: 第4章 自動化システム
> **節**: 4・1 生産システム自動化の体系

生産システムは図4・1のように、設計-加工-組立-検査-出荷の構成をもつ。この中で加工・組立を特に製造と呼び、自動化の対象としてきた。生産システムの自動化とは本来この五つの工程すべての自動化を意味するのであるが、4・1節は特に製造工程の自動化を対象としてその考え方を探る。生産の自動化は図4・2で模式的に示すようにまず機械化が先行し、追って自動化となる。例えば切削加工は手動でも可能だが、機械加工でなければ生産性・精度ともに比較にならないくらい悪い。よって自動化する前の段階として、切削用機械である旋盤などがまず発明される必要がある。しかし、旋盤の場合でも、まずは工作物の回転が実現され、工作物をどのような形状に削るか、すなわち工具経路の制御は作業者による手作業であった。そこで、工具を工具台に載せ、次に工具台をねじで駆動した。ここで機械化の完成である。次に、工具台の動きを意図するように自動的に動かす必要が生じる。この機械の動かし方の自動化には二つある。一つは倣い旋盤のようにテンプレートを使用する方法であり、もう一つは数値制御(numerical control)装置である。このようにして、単独の機械が自動化されてきた。単独の機械で単一の製品を自動的に製造できるようになると、複数種類の製品も可能としたい。つまり、フレキシビリティの増大を追求する。また、異なる工具でも同一製品に加工可能としたい。このような要求に対応してマシニングセンタ(MC: machining cen-ter)が登場した。単独の機械の中での自動化が進むと、今度は生産機械間での工作物や治工具の管理と移動とが大きな手間となる。その結果、自動搬送車(AGV : automated guided vehicle)を使い、立体自動倉庫(automated storage/retrieval system)を導入して、加工システム全体で物流が管理できるようになり、FMS やFA と呼ばれる加工システムが構築された。これらについては後述する。以上は、加工についての自動化の流れである。一方、組立の分野では異なる発想があった。組立はもともと機械化されないで残った作業の寄せ集めと定義してもよいくらい多様な作業からなる。それゆえ、組立工程ではまず手作業を機械作業へ変換することが必要であった。作業者による生産と比較して機械によ図4・1 生産システムの構造図4・2 自動化の進展る生産がコスト的に優位でなければ組立機械を導入できないことから、一見容易に見えるような作業もなかなか機械化できない場合が多い。高速作業や微細作業は機械でなければ不可能であるから、やむを得ず、専用機として個別に実現する。しかし、ふたを取付ける、穴に棒を通す、配線をまとめるなど一見単純に見える作業は、人手作業のほうがコストが安いことから自動化が遅れた。また、大量生産ラインでは機械化できた作業も、生産量が少ないとコスト高になる。よって、多様な組立作業を自動化するには、はん用のハンドリング機構であるロボットの登場を待たざるを得なかった。図4・2で注意すべきは、機械化・自動化の前に半機械化、半自動化が存在する点である。つまり、完全な機械化・自動化である無人化には、すべての要素の機械化・自動化がそろわなければならず、その工程でさして重要ではない部分でも機械化・自動化の対応をとる必要があることを意味する。そこで今日では、経済的に意味のある自動化のレベルに止めるような工夫がなされている。組立作業は図4・3のように、組込まれる部品[通常、ワーク(work)と呼ばれる]が搬送され、手あるいは組立工具を用いて、組込む部品[パーツ(parts)]がワークに取付けられる。パーツの搬送は、部品供給用の荷姿(ばら積み状態や重ねた状態)から姿勢をそろえて整列し、一つ一つ分離して、工具(グリッパ、あるいはハンドと呼ばれる)が把持して組付ける。この部品供給を信頼性高く実現することが組立機械化の急所である。部品を把持した工具は組立箇所へと移動して、挿入などの組付動作を行う。よって、組立作業の自動化とは、1搬送、2部品供給、3組付、4後工程への接続の自動化を総合的に進めることを意味する。一方、加工工程も組立と類似工程からなるが、加工では部品供給の代わりに治工具(刃具、ジグ)の供給が求められ、ハンドの動きが刃具の動きとなって、ワーク上に形状を生成する。よって加工作業の自動化とは1ワークの搬送、2工具の交換、3加工、4後工程への接続からなる。このように自動化生産システムとは、作業固有のプロセスである切削や組立があり、それらの準備のための部品・製品の搬送(供給・移送)作業と単独機械間の接続からなる。
