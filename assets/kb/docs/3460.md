# 4・1・4 自動化システムの歴史

> **章**: 第4章 自動化システム
> **節**: 4・1 生産システム自動化の体系

組立作業を例に自動化のために発達した機器の歴史を見てみよう。図4・3の組立システムの場合、ワークの搬送はベルトコンベヤにより機械化されていた。そこで、二重線枠に囲まれた部分である部品供給が自動化対象として研究された。特定の部品の特定の形状・物理特性を利用して、部品を整列・分離・供給する。すなわち、専用機を作ることから始め、次にその専用機をはん用化した。自動化には大別して物理的作業の自動化と情報処理の自動化があり、それを表4・1で示すように装置内、複数設備間、設備間と異なるシステムで実現することが求められる。自動化が大きく進展するには設備のはん用化が重要であり、その典型はNC工作機械とロボット(はん用NC位置決め装置ともいえる)であろう。産業用ロボットは1960年代の後半に、ユニメート(Unimate)とバーサトラン(Versatran)という2種類のロボットがアメリカで開発された。これらはともに油圧駆動で教示再生方式を採用して表4・1 自動化の単位と進展作業機械化情報処理自動化レベル専用はん用専用はん用設備内専用機ロボット専用ソフトロボット言語設備間ベルトコンベヤAGVスケジューリング設備状況管理システム――CAD/CAM/CAE生産管理ソフトCIM7―74いた。しかし、繰返し位置決め精度も悪く、かつ、MTBFが当初200時間もないなど、まだ工場で働く機械としては不十分であった。しかし、日本の産業、特に自動車産業ではこれをスポット溶接用に採用した。その後、コンピュータと電動モータ技術の進歩によって精度・信頼性が格段に進歩した。加えて、組立における柔らかさ(compliance)利用の制御技術が進歩し、組立作業への適用も拡大した。同様に、大型ロボットにおいてもその使い勝手が良くなり、荷役のような作業も低価格で実現可能となった。一方、加工機械におけるNCはロボットの発達に先だって発達した。1950年代にMIT で開発された数値制御工作機械は当初複雑曲面の加工を目標としていたが、日本では旋盤やフライス盤の自動化を考慮して発達した。当初は2~4軸制御と軸数の少ない工作機械が中心であったが、マシニングセンタの開発とともに、6軸も一般化した。NCを使うためには、加工形状の計算と加工条件設定が必要となる。そこで、CAD(computer aided design)とCAM(computer aided manufacturing)とが開発された。1980年代以降は制御盤にコンピュータが組込まれたNC(CNC: computernumerical control)にCAD やCAM の機能が搭載され、容易に加工方法を決定できるようになっている。このように、生産システムの自動化は、当初プロセス機械の自動化が開発対象であったが、その後は周辺の情報処理、特に、機械をどのように動かすかの計算に開発対象が移る。その際、プロセス機械をはん用化・知能化することで情報処理の自動化が加速され、その成果がまたプロセス機械の高度化を推し進めるとのスパイラル現象が見られる。生産システムの自動化は、表4・1の左から右、上から下へと進むことが多いが、経済状況に応じて、時代時代でそれぞれ最適なシステム構築が選択される。これは1980年代には自動化を是としてきた家電業界が、いまでは人手によるセル生産方式に価値を見いだしていることからも明らかである。また、生産システムは経済的評価によって選択されるので、既存システム(現在生産しているシステム)の改良のほうが、全く新しいシステムより性能・投資効果・効率などが良くなるように思えるので、どうしても新規生産システムの導入では保守的になりがちである。
