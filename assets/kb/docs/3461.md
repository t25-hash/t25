# 4・1・5 自動化生産システムの概念

> **章**: 第4章 自動化システム
> **節**: 4・1 生産システム自動化の体系

生産システムの構築方法については多数の概念が提案されてきた.多くはコンピュータをシステムの中心におく情報流通を志向したもので,生産工程が機械化・自動化できることを前提としている.一方,日本で提案されたトヨタ生産方式などの生産システム概念はむしろ物流のあり方を考えている.その中で,本書でも説明がある三つの概念について自動化の視点から説明する.(1) FMS(flexible manufacturing system,4・2・4項参照)多様な製品を生産できるシステム.通常,切削加工システムをいう.マシニングセンタ,AGV,自動倉庫,自動洗浄機などで構成され,ATC(auto tool changer),AWC(auto work changer)などを装備して,長時間自動運転が可能なシステム.(2) FA(factory automation,6・3・3・d項参照)工場自動化.OA(office automation,事務室の自動化)と対比される.工程管理,資材管理,工場内物流を含む工場全体の自動化を意味する.(3) CIM(computer integrated manufacturing,6・3・3・d項参照)生産に必要なすべての情報を現場で収集し,コンピュータにより計画・管理・実施するシステム.具体的には,CAD/CAM/CAE/CAPPなどを統合して,設計から販売まで統合するシステム.これらの概念は加工システムから始まったものであるが,組立システムの場合にも同様の概念が存在する.この三つのシステム概念が示すように,そしてまた,前述の自動化の歴史も示すように,自動化は物理作業から情報処理へ,そして,単独の機械から設備間へと広がっていく.さらに,設計から加工・組立,そして出荷までを連結して情報と製品とを流そうとしてCIM が生まれた.その構築には,次のことが求められる.(1) モデル化工場内での情報処理,工程内での製品変化などをすべてコンピュータ内にモデルで表現し,シミュレーションできるようにする.(2) 情物一致現場での物流・情報流とコンピュータ内モデルとを一致させる.しかし,(1)(2)ともに決して簡単ではない.特に生産システム内のパーツやワークなど物の流れを測定することはセンサや人手による入力作業を必要とするため,コストがかかる,入力データの信頼性確保が困難,などの問題点がある.CIM のような統合化は多くの利点をもたらすが,同時に過度のシステム化(マニュアル化,機械化・自動化)を招き,ソフトウェアシステムとして固定化につながる危険性がある.以上のコンピュータによるシステム化の方向付けは,生産現場での情報の流れにむだを省き,生産を円滑に進めることが目標なのであって,モデル構築ばかりしても意味はない.むしろ現場における生産改善活動を中心におくべきとする考えがある.TPM(totalproductive maintenance),TQC/TQM(total quality control/total quality management)などがその典型である.またトヨタ生産方式のかんばんシステムなども現場における物流管理のほうがコンピュータ内のデータより重要であると考える.このような現場重視の方法は,生産環境の変化には迅速に対応できるが,生産性を向上させにくい,場当たり的になりやすいとの欠点もある.
