# 5・5・1 プログラマブルコントローラ

> **章**: 第5章 生産設備
> **節**: 5・5 制御装置

a. 定義JIS B 3500:1990「プログラマブルコントローラ用語」で、「論理演算、順序操作、限時、計数及び算術演算などの制7―106生産システム工学御動作を行わせるための、制御手順を一連の命令語の形で記憶するメモリをもち、このメモリの内容に従って諸種の機械やプロセスの制御をディジタル又はアナログの入力を介して、ディジタル方式で制御する工業用電子装置」と定義されている。b. 誕生と歴史シーケンス制御は古くから、メカニカルなカムシャフト方式やドラム式などがあったが、いずれも単純な制御しかできなかった。その後電磁リレーの出現により電気的制御が可能となり、シーケンス制御の分野でリレー盤が使用されるようになった。1945年にノイマン博士が開発したストアードプログラムのコンピュータ技術がIC技術の発展とともに大きく発展してきた。一方、シーケンス制御においても単体制御からライン制御、集中制御などへ大規模・複雑化し、リレー制御盤の設計・製造・調整・変更や拡張に多大な労力がかかるようになってきた。このような背景の中から、1969年アメリカのゼネラルモーター社が自社設備のトランスファマシンの制御装置に関する要求仕様を発表した。この仕様に基づき、1970年にミニコンベースのプログラマブルコントローラ(programmable logic controller、以下PLCと呼ぶ)が出現した。要求仕様は(1) プログラミングおよびプログラムの修正が容易であり、シーケンスの修正が現場でできること。(2) 保守点検が容易なこと。(3) リレー制御盤より信頼性が高いこと。(4) リレー制御盤より小型であること。(5) 上位のコンピュータと結合できる機能をもっていること。(6) リレー制御盤と経済的に競合できること。(7) AC115V までの入力を受付けること。(8) AC115V,2A の出力が供給できること。(9) システムの変更を最小にし、基本システムを容易に拡張できること。(10) 最低4K 語まで拡張できるプログラマブルなメモリを有すること。などである。その後、顧客からのニーズや半導体技術革新により、演算機能の高度化、ネットワーク対応、高速処理、メモリの大容量化など目覚ましく発展し、今日では生産システムの中核コントローラとなってきている。c. 構成要素図5・40にPLCの構成要素を示す。はん用コンピュータと基本的には同じである。. CPU 部CPU はアプリケーションプログラムに従い、外部接続された入力スイッチなどのON/OFF状態を入力部を経由して取込み、演算し、その結果を出力部を経由して電磁開閉器など外部に出力する。CPU には8ビット処理から32ビット処理のマイクロプロセッサが使用されている。最近では、専用ASIC(appli-cation specific IC:特定用途向けIC)を組込んだ高機能高性能PLCが多く出てきている。. メモリ部シーケンスプログラムと入出力/数値などデータを格納するためのメモリである。メモリの容量は機種によりさまざまな容量のPLCが提供されている。メモリ素子も各種用意さ図5・40 PLCの構成要素図5・41 PLC 動作概要れている。. 入力部リミットスイッチや押ボタンスイッチ、各種センサなどが入力コネクタ(または端子)を経由して接続され、PLCで扱う内部信号レベルに変換する機能をもっている。. 出力部CPU で演算された結果は、外部機器と接続できるようにレベル変換されて出力部を経由して外部機器に接続される。. 電源外部電源から変換してPLC内部を動作させる内部電源である。. 周辺装置ヒューマンマシンインタフェース(HMI)は、ユーザプログラムの読出し/書込みおよび動作モニタをするサポートツールを接続するインタフェースである。基本構成は以上であるが、このほかに通信とのインタフェースやアプリケーション専用モジュールである位置決め、プロセス制御などのインタフェースを用意しているPLCもある。d. 動作概要PLCは図5・41に示す一連の処理をサイクリックに繰返すことでリレー盤と同じ処理を実行している。. 共通処理PLCハードウェアに異常がないか各種診断している。診断内容としては、一連の処理が一定の時間内で完了しているか、メモリのチェック、I/Oのチェック、電池のチェックなどがある。. I/OリフレッシュCPU は演算処理に先立って外部入力信号をI/Oメモリに取込み、I/Oメモリにある演算結果を外部出力する。したがって、演算処理はCPU とI/Oメモリ間で実行される。他の方式として逐次実行方式と呼ばれ、I/Oリフレッシュによる一括処理ではなく、命令処理時に外部入出力するものもある。. 演算処理ユーザプログラムメモリに格納された命令語を順次呼出し、実行する。. 各種サービス(1) 周辺サービスプログラミング/モニタ機器などサポートツールに対するサービスを行う。サービスとしてはプログラムの読出し/書込み、I/O状態のモニタ、I/Oの強制ON/OFFなどがある。(2) 通信サービス上位-PLC間、PLC-PLC間、PLC-下位間など通信からの要求処理をする。e. 国際規格PLC の国際規格化がIEC(InternationalElectrotechnical Commission)で、IEC61131として進められている。併せて日本ではJIS 化も進んでいる。その対応表を表5・4に示す。f. プログラミング言語IEC61131-3:1996では、図5・42に示すように、図式言語としてラダー図(LD)言語と機能ブロック図(FBD)言語、テキスト形式言語として命令リスト(IL)言語と構造化テキスト(ST)言語および制御システムの遷移を記述するシーケンシャルファンクションチャート(SFC)が規定されて表5・4 PC規格IEC規格規格NO.規格名称General InformationEquipment requirements and testsProgramming LanguagesUser guidelinesMessaging service communicationsJIS 規格JIS NO.規格名称B 3500プログラマブルコントローラ用語B 3501プログラマブルコントローラ一般情報B 3502プログラマブルコントローラ装置への要求事項と試験B 3503プログラマブルコントローラプログラム言語TR B 0004プログラマブルコントローラ使用指針図5・42 プログラミング言語の種類いる。このほかにもフローチャート式やタイムチャート式などPLCメーカが独自で開発した言語もあるが、一般的にはラダー図言語が使用されていることが多い。ただし、PLCメーカごとに同じラダーであっても表記方法が異なるので、ユーザはPLCメーカごとに習熟する必要があり、教育、保守サービス上課題となっていたが、言語の標準化が進めば課題が解決し、ソフト生産性向上に大きく寄与すると期待されている。g. FA ネットワーク動向FA ネットワーク体系を図5・43に示す。. 情報系ネットワークPLCより上位のネットワークはイーサネット(Ethernet)がデファクトスタンダードといってよい。最近、情報系から下位のネットワークまでプロトコルを統一する動きがあり、代表例としてEtherNet/IP,ProfiNetおよびModbus/TCPがある。またFA 環境でも使えるように産業用イーサネット(Industrial Ethernet)が検討されている。. コントローラ系ネットワークコントローラ間ネットワークとして、IECでフィールドバスが標準化されている(表5・5)。日本では(社)日本自動車工業会が「FA ネットワーク要件書」を発表した。FA ネットワーク要件作成のねらいは・マルチベンダ指向のネットワーク(オープン仕様)・安くて使いやすいネットワーク・ユーザが必要な通信の規格と性能保証それに基づいてMSTC[(財)製造科学技術センター]でFL-netが仕様策定され、FA 機器メーカから各種機器が発表されている。このほかに有力PLCメーカが推進するネットワークがある。. 下位ネットワークPLC以下のコンポーネントレベルネットワークは世界的に多数あり、それぞれ特徴をもっている。ヨーロッパ、北アメリカおよび日本では有力なFA 機器メーカや工業会・団体がデファクトスタンダードを目指して推進している。このレベルのネットワークはセンサやリレーなどON/OFF制御7―107図5・43 FA 用ネットワーク体系表5・5 フィールドバス標準タイプ名称Fieldbus(ANS/ISA S 50.01)ControlNetPROFIBUSP-NETFieldbusFoundation HSESwiftNetWorldFIPINTERBUS図5・44 制御システムの設計手順中心(以下ビット系と呼ぶ)のネットワークとON/OFF制御に加えてインテリジェント機器をつなぐために数十から数百バイトのデータ伝送(以下データ系と呼ぶ)のネットワークの2種類がある。ビット系ではヨーロッパのセンサメーカが推進しているASIが著名である。データ系ではヨーロッパはPROFIBUS-DP,INTER-BUS、北アメリカではDeviceNet,SDS,LON、日本ではOPCN-1,DeviceNet,CC-LINK が著名である。h. PLCシステムの設計。 PLCシステム設計プロセスPLCを選定する場合、制御システム全体のシステム概念からPLCシステム仕様を決定する。図5・44に示す。設計手順の第1ステップは制御システム全体の概念(考え方、思想)を明確にする。対象システムの機能目的、性能目標、信頼性目標、予算などシステム分析を行い、システムの前提条件、ターゲットを明確にし、システム概念図をまとめる。第2ステップはシステム概念図に基づいてシステム設計を行い、システム構成図、機能構成図、機能別構成図を作成する。全体システム構成に基づきPLCで制御する対象機械、システムの範囲(制御対象)とPLCが受持つ制御の範囲を決める。制御対象の制御内容と規模によってPLCを選定するのが一般的であるが、システム概念に基づいてPLCのダウン時の影響範囲などを判断基準として集中制御とするか、分散制御とするか、また自動、手動運転をPLCが受持つか、PLCがダ7―108生産システム工学ウン時のバックアップはリレーで製作するなどPLCの制御対象、制御範囲、制御仕様を決定する。このPLC仕様に基づいてPLCを選定し、第3、第4ステップの設計、製作を行う。図5・45はPLCシステムの設計から稼働までの具体的な進め方である。. PLCによる制御システムPLCで制御する対象機械、システムの範囲によって、単独システム、集中システム、分散システム、階層システムがある。(1) 単独システム1台のPLCで対象の機械を制御するシステムで、対象機械の自動運転、手動運転をPLCですべて制御するか、PLCダウン時のバックアップの手動運転はリレーで製作するかで、PLCが受持つ制御範囲を決める。PLC制御範囲の入出力点数によりシステムを構成する。(2) 集中システム集中システムとは、1台のPLCが複数の機械を制御するシステムである。このシステムは複数の機械が互いに関連して動作するような一連のライン制御に適用する。1台のPLCで制御するため、ある対象機械にトラブルが発生し、PLCを停止すると他の機械も停止する欠点があるが、逆に機械1台が停止すれば、他の機械は運転しないシステムに有効である。(3) 分散システム分散システムは、対象機械ごとにPLCを設け、制御する。各機械間のインタロック、フィードバック、制御指令を各PLC間で制御信号のやり取りをするシステムである。PLC間の制御信号のやり取り方法は、PLC間ネットワークを利用する方法が一般的である(g 項参照)。このシステムは1台のPLCが停止しても、他の機械は運転できる。現地据付調整、試運転、保守の面でやりやすいばかりでなく、改造、増設などにも対応しやすいなどの特徴をもっている。(4) 階層システム階層システムは図5・46に示すようにコンピュータとPLCをイーサネットまたはRS 232Cなどで接続し、生産情報管理処理と機械制御を機能分散し、階層システムを構成することでトータルシステム制御を行う。このシステムは、機械制御はすべてPLCが受持ち、コンピュータとPLC間で制御情報のやり取りを行い、次のような処理をシステムとして合わせて行う。・生産指示、生産工程制御指示・生産個数などの生産データの収集と表示・日報、月報の作成図5・45 PLCシステムの設計から稼働までのフロー図5・46 階層システム・制御対象の運転状況監視表示・制御対象の故障、異常の検出と表示また1台のコンピュータから複数台の管理PLCを接続し、複数の系統をもつラインブロックをコンピュータが管理処理するトータルシステム制御ができる。. PLCシステム選定上の留意点PLC選定に当たっては、制御対象、制御規模、価格、安全性、メンテナンス性、信頼性、規格・法規制など、総合的に検討する必要がある。. PLC機種選定制御システムに最適な機種を選定するだけでなく、将来の拡張性、サポートツールの充実や操作性なども選定に加味する必要がある。(1) 入出力点数制御対象の入出力点数を算出する。(2) プログラムメモリ容量ユーザプログラムを格納する記憶容量であり、制御が複雑になればメモリ容量は大きくなる。プログラム容量は入出力点数とほぼ比例している。制御内容の複雑さにもよるが、入出力点数×(8~12)といわれてきた。しかし最近では、ヒューマンマシンインタフェース機能や故障診断機能の充実により、入出力点数×20以上のシステムも多くなってきている。(3) データメモリ容量プログラムメモリと同様に、入出力点数と制御の複雑さによってデータメモリも増加する。プログラムメモリ容量に比例して不足しないようにデータメモリが用意されていることが多い。(4) データバックアップPLCに格納されたプログラムやデータメモリの一部は、PLC電源がOFFでも保持しなくてはならない。不揮発性のメモリや電池によるバックアップをしていることが多い。(5) 処理速度PLCに求められる処理速度は制御対象からの入力信号を受けて、演算処理し、出力するまでの時間であり、ディジタル総合応答時間(TRT : total system response time of digital)と呼ばれる。このTRT は数msから数十msまでである。図5・41に示すように、PLCはサイクリックに実行しており、この1サイクル時間をスキャンタイムという。通常TRT には1スキャンタイム分のばらつきが出る。複雑な演算機能を使用している場合、演算している場合としていない場合でスキャンタイムのばらつきが大きくなり、制御のばらつきとなることがあるので注意が必要である。(6) 演算機能シーケンス論理処理(ビット単位の演算)機能のほかに、タイマ/カウンタ機能を保有しているが、シフトレジスタ、比較、四則演算、浮動小数点演算など高度な命令機能を保有している機種もある。どんな機能が必要なのかによって機種選定が変わる。(7) 入出力ユニットの種類交流/直流、電圧/電流、シンク/ソース、接点、トランジスタ、トライアックなど各種あり、その中から選定する。(8) 高機能入出力ユニットアナログ、位置決め、カウンタなど各種用意されているが、メーカによって仕様が大きく異なるため、事前に仕様チェックが必要である。(9) 外型PLCのタイプには次の三つがある。(i) ボード型ワンボード型とも呼ばれるPLCで、プリント基板上にCPU 部、I/O部、メモリ部などを一体化したもので、一般的に入出力点数が少ない。小型化、軽量化、薄型をねらったもので、入出力点数の変更や拡張性の制限があるが、安価である。(ii) ブロック型小規模PLCに多く、増設単位がブロック単位で拡張する。ブロック型は盤外設置の取付けの容易性を考えて、DIN レール取付けが多い。基本ユニット、増設ユニットともに入出力点数や種類を豊富に取りそろえており、選定しやすいようになっている。(iii) 入出力モジュール取付型ビルディングブロック型とも呼ばれるPLCで、CPU、電源、入力、出力などがモジュール化され、ベースに取付けて増設していくものである。入出力点数の増設単位は、8点、16点、32点、64点などがある。1枚のベースに実装できるモジュールの数も、2,4,8枚など各種用意されている。高さは250mmと130mm前後のものが多いが、最近では100mm以下の小型機種が増加している。このタイプの機種はアナログ、位置決めなど高機能入出力やネットワークなどを組合わせて使用することが多く、中大規模制御に適している。(10) 耐環境性PLCが使われる使用環境で注意すべき点は、耐温度、耐湿度、振動、衝撃、じんあい、耐油性、腐食性ガス、ノイズ環境などである。これら設置環境に適合したPLCを選定することが必要である。(11) 価格制御システム最適なPLCを選定することは必要であるが、拡張性や保守性も考慮することが望ましい。(12) 周辺機器周辺機器は、プログラミング、デバッグ、モニタリング、ドキュメント作成、プログラム保存などが効率的に操作できることが必要である。最近はパーソナルコンピュータを周辺機器として活用することが多い。携帯用のハンディプログラマや表示器にモニタ機能を内蔵させたものも出てきている。(13) 規格海外輸出する場合、適用規格に適合している機種を選定する。
