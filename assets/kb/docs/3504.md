# 5・6・2 単一生産機械のヒューマンインタフェース

> **章**: 第5章 生産設備
> **節**: 5・6 ヒューマンインタフェース

工作機械等の画面の標準化については、次の議論がされている。特にオープンコントローラ(open controller)用のヒューマンインタフェースについては、メーカ各社が独自のヒューマンインタフェースを作成したいところであり、NC画面の統一や標準化は難しく、むしろ使いやすい画面(ヒューマンインタフェース)を使用される環境に合わせて容易に作成することが可能であることが求められている。単一NC工作機械のヒューマンインタフェースについては、ISO7000:1989,JIS B 6012:1998で規定されており、それらについては、一部差異はあるものの工作機械メーカが製作する機械操作盤、NC、いずれの場合も基本的にはだいたい類似したシンボルが使用されている。機能群別に見ると、シンボルはNC装置関連、NC設定関連、自動運転関連、機能表示関連、座標関連、座標速度関連、手動運動関連に分けられる。画面全体は一般的に現在位置など各種状態を表示する部分、運転モードや表示ランプを表示する部分、運転モードに応じて切換える部分からなる。ここで規定されていないものについては、各社が個々に考案したシンボルが使用されており、今後統一化の必要性がある。また、画面を容易に構築するための道具であるビルダの要望は強く、これについては、操作が簡単であること、拡張性が高いこと、FA コントローラ機能のオブジェクト形式で抽象化可能であること、抽象化された機能のNC部の明確な標準API(application program inter-face)が定義可能であること、ビジュアルツールによる直感的な画面作成が可能であること、ビジュアルツールに使用するシンボルや用語が標準化されていることなどが要求されている。
