# 6・2・3 CAM システムの変遷

> **章**: 第6章 情報システム
> **節**: 6・2 生産準備と情報システム

次に、NCプログラム作成作業を含めた工程設計業務へのコンピュータ適用の変遷を見ることにする。現在までのCAM システムの変遷を見ると、それはNCプログラム作成システムの変遷といっても過言ではないと思われる。それを三つの時代に分けて追って見ることにする。a. 言語型バッチ処理システムの時代NCプログラム作成システムの起源は、1950年代にアメリカで開発されたAPT(auto-matic programming tool)である。これは、数値制御工作機械(以下、NC工作機械と略す)の開発とほぼ同時期に開発されたNCプログラム作成用の言語で、APT 言語と呼ばれている。そのおもなねらいは、手計算が不可能である自由曲面を含む複雑形状部7―116生産システム工学品のNCプログラム作成であった。このAPT は、幾何形状の定義、形状に対する工具の動かし方の指示などの機能を有しており、その後のNCプログラム作成システムの基本形ともなった。その例として、現在でも使用されているAPT のCL(cutter loca-tion)データ様式に関して以下で紹介する。NCプログラムは、NC工作機械の種類ごとにすべて異なる様式を要求されることが大半である。これに対応するために、CAM システムからはNCプログラムを直接出力せずに、まずCLデータを出力して、ポストプロセッサ(post processor)が各NC工作機械向けのNCプログラムを作成する方法が採用されている場合が多い(図6・7)。このCLデータの様式として、APT で定められた様式が、現在でも使用されていることが多く見受けられる。APT の後にドイツでEXAPT(extended APT)が開発された。APT が工具軌跡作成のための数値計算処理が主体であったのに対して、EXAPT は切削条件設定や工具選択など加工作業の自動設定に主眼をおいたシステムであった。ただし、その対象は、2.5次元までのフライス・穴あけ加工、旋削加工である。この2.5次元加工とは、切削時には同時に二軸までしか使用しない加工をいう。なお、この時代は多くのCAM システムは大型コンピュータを使用したバッチ処理で使用されていた。b. 対話型システムの時代コンピュータが、ミニコン、EWS(engineering workstation)などと進化するにつれて、リアルタイムで出力を手にすることができる対話型システムへと移行した。当初は先に述べた言語型が主流であったが、次第に言語を使用しないメニュー選択方式が普及するようになった。また、NCプログラム作成システム専用のコンピュータも出現し、さらには、NC制御装置の前で対話型のNCプログラムの作成を可能とする対話型NCが登場するまでに至った。c. CAD データ活用の時代CAD が普及するにつれて、CADで作成された形状データをCAM で利用することが進んだ。この対象は、おもに自動車のボデーや金型部品に代表される曲面加工であった。一方、CAD としては、製図作業の支援を目的とした二次元CADが急速に普及したが、そこで作成されたCAD データのCAM での活用は、それほど進められなかったと見える。その大きな理由としては、生産側で図面データから加工部位を探して加工部位の仕様などをCAM に入力するための作業量が多く、二次元CAD で作成された一部の形状データを活用できても、大きな効果が出せなかったことが挙げられる。以上で述べたCAM の歴史をまとめると、CAM の適用の中心は、手計算では不可能な複雑な形状に対するNCプログラムの作成作業であり、2.5次元加工に関しては、一部で加工技術データベースを活用した加工作業の設定作業の自動化を実現しているシステムも存在するものの、CAM の主流であったとはいいがたい。
