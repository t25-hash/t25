# 6・2・4 次世代CAM の動向

> **章**: 第6章 情報システム
> **節**: 6・2 生産準備と情報システム

1990年代の後半から普及しつつある三次元CAD(ソリッドモデル)、および国際規格として開発されたデータモデルの活用により、CAM の世界も新たな展開を見せようとしている。a. 加工形状特徴による部品仕様の伝達加工形状特徴の概念そのものは新しいものではなく、従来からCAM の中で穴、ポケットなどと加工部位の分類用に使用されていたが、その記述仕様は統一されていなかった。この加工形状特徴の分類と各加工形状特徴のもつ属性情報の記述方法が、STEP規格の中のAP224で規定された。その分類の例を図6・8に示す。この規格のねらいは、設計部門から生産部門への加工部位とその仕様の受渡しに加工形状特徴を使用することであり、これが実現されると、加工部位およびその仕様の理解を人が行う必要がなくなることになる。なお、加工形状特徴と三次元CAD との関係は次のとおりである。(1) STEP AP224においては、加工形状特徴の形状情報は、ソリッドモデル(solid model)で表現されなければならない。(2) 加工形状特徴の多くは、ソリッドモデルで表現された形状情報から自動認識可能である(すでに実用システムも存在する)。b. 新しいデータの流れ加工形状特徴を活用した設計・生産部門におけるデータの流れの例を図6・9に示す。この流れの中核を図6・8 加工形状特徴の例図6・9加工形状特徴を活用した設計・生産部門におけるデータの流れ以下で解説する。. 加工形状特徴データの作成まで(ステージ1~3)具体的な例を先に示すと、図6・10のモデル部品の三次元形状およびその属性情報から、図6・11の加工形状特徴を作成することである。図6・9のステージ1およびステージ2から、三次元形状+属性データが作成できると、加工形状特徴の的確な自動認識の実現だけでなく、ここから図面データを自動的に作成することも可能である。しかし、現実を見ると、三次元CAD を形状データ作成だけに使用していることがまだ多い。したがって、三次元形状データだけから加工形状特徴を自動認識するシステムの利用が多く見受けられるが、この場合には、作成された加工形状特徴に対して、仕上面粗さや公差情報を付加する必要が生じる。. 加工工程の設定(ステージ4)ステージ3で作成された加工形状特徴データを入力として、加工工程の設定を行う。モデル部品に関する加工工程設定の結果の中で、工程への加工形状特徴の割付および部品の保持位置(clamping position)を表示した例を図6・12に示す。図6・12では、テーブル回転機能付の横型マシニングセンタを使用して、図内の前面および背面をこの工程で加工しようとしている。ここで設定された部品の保持位置も重要な情報であり、これを用いて設計されたジグの例を図6・13に示す。このステージ4の加工工程の設定をサポートしているCAM システムは非常に少なく、多くのものは、ここを飛ばしてステージ5の加工作業の設定に移っている。. 加工作業の設定(ステージ5)このステージは、事前に準備された加工技術データベースを利用して、加工作業の設定を図6・10 モデル部品の形状7―117自動化することと考えられる。そのためには、図6・9の準備ステージは、以下に示す加工技術情報をモデル化しておく必要がある。・加工形状特徴の加工方法・工程全体の作業順序を設定するためのルール上記の二つの情報により、この工程に割付けられた加工形状特徴に関する加工作業の自動設定が可能になる。このうちの加工形状特徴の加工方法を記述した例を図6・14に示す。図6・12 工程への加工形状特徴の割付の例図6・13 ジグ設計の例図6・11モデル部品に含まれる加工形状特徴7―118生産システム工学図6・14加工形状特徴の加工方法記述の例図6・15 工具動作タイプ(2.5次元フライス加工用)図6・14は、加工形状特徴の加工方法に関して、ISO14649:2003で開発されたデータモデルを使用して記述したものである。まず、対象とする加工形状特徴の加工に必要となる作業とその順序を記述した表を作成する。さらに、設定された各加工作業の詳細内容を記述した表を作成する。上記の二つの表に記述すべき項目は、すべてISO14649:2003で規定されている。また、作業方法の詳細の表に記述する項目の中で、作業タイプ、使用工具、工具動作タイプ(tool path strategy)などは、同規格で規定している種類の中から指定することになる。その工具動作タイプ(2.5次元フライス加工用)を図6・15に示す。. NCプログラム(NC program)の作成(ステージ6)ステージ5で作成された作業設計情報からNCプログラムを作成する。NCプログラムの指令方式としては、過去長年にわたり、ISO6983:1982(通称G コード)が使用されてきたが、ISO14649:2003(通称CNCデータモデル)では、それに代わる新しい指令方式を開発している。この方式では、・加工形状特徴の種類とその幾何形状・加工形状特徴の加工方法(図6・14の内容)をNC装置に与えることを基本としている。この方式のおもなねらいは、すべてのNC装置やNC工作機械に共通に使用可能なNCプログラムとすることであり、図6・7に示したポストプロセッサ処理を不要とすることである。. まとめステージ4(加工工程の設定)およびステージ5(加工作業の設定)を支援するソフトウェアを、欧米ではCAMと区分して、CAPP(computer aided process planning、自動工程設計)と呼ぶことが多い。また、図6・9中の中間出力に、国際規格として開発された・形状データモデル(STEP AP203:1994)・加工形状特徴データモデル(STEP AP224:2001)・工程情報用加工プロセスモデル(STEP AP240:2005)・作業情報用加工プロセスモデル(ISO14649:2003)などのデータモデルを使用することにより、図6・9の流れをすべて同一システムで実現するのではなく、機能ごとに、最適なシステムの選択が可能になると考えられる。c. 技能の技術化熟練技能者の個人に特化された技能を、抽出・整理・体系化して客観化を図るとともに、IT により再現性のあるディジタル技術に可能な限り置換えるための活動が、「ディジタルマイスター関連プロジェクト」として、2001年より国の支援のもとに実施されている。ここで抽出された技術は、図6・9中の加工技術データベース(manufacturing technology data base)として可能な限り活用されることが望ましい。これに関連する技術としては、切削加工データベース構築システム(S-CAM)が1999年に開発されている。S-CAM(standard-ized CAM)は、製造科学技術センター(MSTC)が情報処理振興事業協会(IPA)からの委託で開発したソフトウェアであり、図6・14に示した記述方法で、加工技術データベースが構築できる。
