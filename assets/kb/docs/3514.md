# 6・3・1 企業内情報システムの統合の流れと全体図

> **章**: 第6章 情報システム
> **節**: 6・3 企業内統合情報システム

a. 企業内システムの抱えている課題企業内情報システムは、生産管理、生産現場支援、販売管理、設計支援、会計計算等々の日常業務のルーチンワークを情報システムに置換えることにより発展してきた。これらの情報システムは、開発投資に対して、何人の省力化ができるかという観点で投資効率が測定され、個別に導入および機能強化が行われてきた。その結果、企業内情報システムは個々の機能ごとの効率化を目指した情報システムが乱立し、それぞれの情報システムの間がつながらない問題を抱えている。さらに、個々の情報システムが、現場の要望に応じてシステムの機能強化を行ってきたため、当初は整合性がとれていた各情報システムの内部構造が複雑になり、機能強化が容易に行われなくなってきた。b. 情報システムの企業内での新たな位置付け新しい時代の企業内統合情報システムには、個々の省力効果以上に、経営の道具としての機能が求められている。それには、変化の早い環境にも対応するスピード経営を実現するために、受注、生産実績、部品納入実績、在庫状況、販売実績、販売見込み、債権回収状況等々の企業運営に必要な情報が即時に提供されることが必要となる。つまり、企業内統合システムは、従来の月末等の締め日までに必要なデータを蓄積して企業活動の後処理をするだけではなく、システムの提供する情報に基づいて企業の経営活動を支援できることまで求められるようになってきた。さらに、企業内情報システムは、情報による企業間統合の支援を求められるようになってきた。これは、競争の場が顧客主導の市場に移り、これまでのように個々の企業内生産活動だけで十分な競争力を維持できなくなり、サプライヤ等を含めた企業間をまたがった競争力の実現が必要になってきたためである。これらを実現するために、まず、企業内情報システムを統合化し、さらに企業活動の上流である市場情報および下流のサプライヤ、協力企業への統合化を、グローバルな視点を取り入れて進めていく必要がある。これらのような情報システムの位置付けの変化をe-businessと呼ぶ場合がある。e-businessは、単にインタネットで行われる個人および企業カスタマへのセールスプロモーションや受発注業務だけではなく、ネットワークで企業内および企業間が統合化された時代のリアルタイム経営戦略なのである。c. 企業内統合情報システムの構成要素および統合の視点企業統合情報システムの構成要素は製造業の主要な活動に対応して、大きく三つから構成される。まず、需要予測・受注により材料・部品の調達を行い、製品の生産および出荷を行い、顧客に製品を納入し、代金を回収する生産活動である。次に新製品を開発し、それを量産ラインに載せる開発活動である。そして会計、人事等の管理活動である。これらを全体の流れとして説明したものが図6・16である。縦軸が技術動向あるいは市場動向を反映して、商品開発をするプロセスで、横軸が顧客要求から生産・調達・出荷するプロセスである。さらにこれらのプロセスを支援する管理系のプロセスが基盤である。これらの構成要素を含む企業内情報システムの統合化の進め方は、企業の特性、戦略によって異なるが、企業内の三つの主要プロセスを企業戦略に沿って全体最適の観点で整合性をとって進めていく必要があり、四つの視点で整合性をとって統合化する方法がある(図6・17)。一つめは顧客視点、二つめはサプライヤ視点、三つめは従業員視点、そして四つめの企業戦略の視点に立って全体を統合化し戦略を立案することである。そのうえで各機能間の役割および情報インタフェースを、論理的に整合性をとって定義していくことが重要となる。図6・16 企業レベルのプロセス全体図7―119
