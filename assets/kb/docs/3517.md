# 6・4・1 企業間統合情報システム概説

> **章**: 第6章 情報システム
> **節**: 6・4 企業間統合情報システム

a. 企業間統合情報システムの定義企業間統合情報システムとは、異なる組織間で、合意した電子的な情報を、各組織が構築した情報システムや情報機器を介して、標準的な規約を用いて共有することを実現している情報システムを意味する。異なる組織とは、企業、政府組織、公益法人などを表すが、6・4節では、異なる組織を企業と称している。b. 企業間のビジネスプロセスを実現するための課題企業のビジネスプロセス(business process)における処理の高度化・効率化において、情報システムはさまざまな役割を果たしてきた。企業の情報システムへの投資において、対象とされた範囲は、自社部門内・企業内または関連会社・系列企業内が中心であり、企業または企業グループ内に閉じたビジネスプロセスにおける処理の高度化・効率化が実現されてきた。しかし、今日のように競争が激しくなっている環境下では、さらなる高度化・効率化が求められている。このために、生産・流通・販売支援システムのような、いかに安定した広い市場を確保できるかを目的として開発されるシステムでは、企業または企業グループの枠を越えて、戦略性も含めて製品のライフサイクルを支援するシステムの必要性が叫ばれている。すなわち、企業内のビジネスプロセスを、顧客・取引先および供給者までを含めたビジネスプロセスとして統合化することにより、品質・価格面でより優れた製品を提供していくことが求められている。このビジネスプロセスの統合化により、企業およびサプライチェーンの機敏性の改善、生産およびサポートにおけるリードタイムの短縮などが実現され、この中で各企業が有機的に機能することになる。このような、企業間のビジネスプロセスを実現するためには、以下のような課題がある。(1) 他企業との正式な情報の授受手段は紙によるものが多い。そのため、送り手側では情報の配布の手間が増大し、受け手側では、情報の再利用のための再ディジタル化などの新たな作業や最新版の管理などの手間が発生する。7―126生産システム工学(2) 企業間の業務を実現するための、情報の授受方法に関する標準が少ないため、企業間業務連携を行うに当たっては、個別にその連携方式(企業内システムと他企業内システム間の連携を行うためのプロトコル)および連携システムを構築しなければならず、多大なるコストがかかる。(3) ある企業のビジネスプロセスにおける慣習と他企業のビジネスプロセスにおける慣習の違いから、それらを統合化したビジネスプロセスの構築が困難である(実現に当たっては、立場の強い企業側の企業文化の押付けが発生し、立場の弱い企業で、多端末現象が発生したり、各企業の独立性・戦略性が損なわれてしまうという問題が発生する)。(4) 企業間連携時のトラブル、企業間ビジネスプロセス内でのデータ矛盾等、企業間の責任所在に関連するトラブル発生に備えた対応を実現する仕掛けを構築する必要がある。(5) 企業間での情報の授受に伴う第三者からの脅威、当事者による内容の改ざん等、セキュリティを確保するための情報技術上の問題が発生する。これを、取引上の課題と情報技術上の課題にまとめると、以下のようになる。. 取引上の課題・企業文化の押付け・慣習による分担の不明確化(責任所在、分担範囲の不明確化)・紙による情報の授受による手間の増大(情報の再利用性、情報の授受における管理方式等の問題)。 情報技術上の課題・統合化をトータルに対応する情報システム基盤がない(ネットワーク、セキュリティ、責任分界点の監査等)・独立した情報システム間の連携性の確保[情報システムの孤島化、情報交換(データフォーマット、コード体系、管理体系等)等の問題]取引上の問題については、企業間の水面下でのやりとりにより解決を図っていることも少なくない。しかし、このような慣習は、公平な取引・責任範囲の明確化の観点から好ましくない。また、情報技術上の問題に関しては、電子化されたデータによるオープンな環境下での企業間連携において顕在化する要素が多く、今後の情報技術の進展とともに解決されていかなければならない。c. 企業間連携を実現するための情報技術と標準化企業間統合情報システムには、利用面において、必要なときに、必要な情報を利用できることが求められる。これを実現するためには、バッチ伝送のデータ交換だけでなく、データ共有による業務実現のための管理機構(情報共有として受渡される情報の管理、業務の状況管理等)が必要となる。しかし、このような情報共有における管理機構がないのが現状である。また、企業間連携を実現するための取組みとして、情報インフラレベルでは、UNIX 等OS レベルでのネットワーク連携から、ウェブ(WWW : world wide web)の普及、コルバ(CORBA :common object request broker architecture)によるインタオペラビリティの確保等の技術進展がある。アプリケーションレベルでは、ビジネスオブジェクト標準化に向けての検討が行われている。今後、これらの成果を踏まえたビジネスプロセスの統合化の実現が図られていくと考えられる。また、企業間連携に当たっての重要な要素として、標準化が挙げられる。企業間で統合化を行うに当たっては、業務上のやりとり内容や業務連携方法などについて、当事者間で事前に合意されなければならない。したがって、複数企業間で自由に企業間連携を実現するには、この当事者間の業務上の合意が、少なくとも業界などの範囲で標準となることが望ましい。しかし、このような標準化された企業間連携を実現するに当たっては、企業間連携に関連する各企業内の業務の標準化が必要であることを念頭におかなければならない。さらに、企業間業務の連携の標準と企業内での業務標準が結びついて企業間連携が実現する。この標準には、用語の意味の標準化、業務ルールの標準化なども含まれる。したがって、企業間連携の実現にとっては、企業間および各企業内の業務が、それぞれの標準に立脚したうえで連携が可能なような、基盤となる標準化技術が望まれるのである。このような形態で情報共有を利用した企業間業務連携の環境が実現されることが情報共有環境のあるべき姿といえる。d. 企業間統合情報システムに必要な基本的考え方業務連携の環境における課題を解決するためには、次の考え方が必要となる。(1) 企業間の連携を、電子化されたデータの情報共有を中核として行う。(2) 企業間の業務を連携させるための標準的な基盤を提供する。(3) 企業間の利用内容や運用などについて事前の合意に従った業務連携を実現する。ただし、この考え方を実行するには、企業文化の押付け、および慣習による分担不明確化等を防ぐため、管理者の設定を行い、利用する内容・運用についての合意を行い、合意内容に従った標準基盤による連携が前提となる。効果は、(1) 情報の電子化により、紙による情報の授受における弊害が解消され、情報配布等による情報流通・授受の効率化、情報の再利用、更新内容の即時反映、ドキュメント保管コストの低減が図られる。また、情報共有の実現により、授受された情報の一元管理による管理の効率化、情報検索の容易性が図られる。(2) 企業間の業務連携のためには、企業間での情報の授受における標準的な方法、企業間での業務連携を支援するための仕掛け、トラブル発生に備えた対応を実現するための仕掛け(責任分界点の確保)、セキュリティの確保等の標準的な基盤の提供が前提となる。効果は、参画の容易化(連携のための準備作業の低減)、連携先範囲の拡大、業務の標準化の推進を図ることができる。(3) 企業間で事前に合意をとることにより、効果は対等なビジネスプロセスの構築、責任所在・分担範囲の明確化、データの信頼性確保が実現できる。
