# 6・5・3 製造制御ネットワークの要件

> **章**: 第6章 情報システム
> **節**: 6・5 製造制御システム

現状の製造制御ネットワークでは特定のベンダによるネットワークを用いたり,標準化あるいはオープン化されたネットワークと混在で用いる場合が多い.しかし,製造システムを構築する観点からは,特定ベンダに依存した自動化機器の構成はシステムの柔軟性や拡張性を阻害する要因ともなってくる.従来から,マルチベンダ環境を実現するためにはRS 232Cなどの標準通信手段を用いてきたが,通信方式の違いによりパソコンやインタフェースなどの中継器を介して自動化機器と接続しなければならないことやユーザ自身が複雑な通信設定値の設定を行う必要があったり,さらに通信ケーブルが煩雑となって機器の移動に伴う再配線など多くのデメリットを有していた.そこで製造システムの高度化に対応するためには,次のような視点からネットワークを選択することが望ましい.すなわち,・マルチベンダ環境実現のためのオープン化・使いやすいネットワーク実現のためのプラグ& プレイ化・ユーザ指向の仕様と性能実現のための標準化とオープン化などが求められている.特に制御ネットワークの基本的仕様としては,オフィス環境と異図6・42 工場内ネットワークの階層7―131なって特別な仕様が求められる.製造環境ではとりわけデータのリアルタイム性が要求される.すなわち,応答時間の要求値として監視制御の際には10~数百ms,I/O制御では1~数十msといった速い応答速度が求められる.さらに比較的劣悪な環境で使用されるために耐環境性や高信頼性が不可欠な要件となる.また工場内で多数のネットワーク敷設と機器の接続がなされるので,ネットワークの施工性や保全性も求められる.今後の制御ネットワークには,上述した条件や仕様を満足するようなオープン性や標準化を指向したネットワークが多く求められるであろう.
