# 6・6・3 ネットワーク技術

> **章**: 第6章 情報システム
> **節**: 6・6 生産システム情報化のためのソフトウェア基盤技

a. 生産システムの機能階層と通信ネットワークISO/TR10314-1:1990では、生産システムの基本機能として図6・48に示す6階層の工場自動化参照モデルを提案している。生産システムでは、装置単体からより大規模なシステムへと、複数階層に分散する計算機や機器群が、さまざまな規格の通信ネットワークにより相互接続されている。上位階層の企業レベル・工場レベルでは、遠隔地の事業所や工場の基幹情報システム間が、広域通信網(WAN)で相互接続される。広域通信網として、従来は専用回線や電話回線が利用されてきたが、近年はインタネットの活用が増えている。セル・エリア・工場の階層間を接続するネットワークとしては、はん用のLAN が利用されることが多い。この工場や企業内LANの構築にも、インタネット標準が活用されることが多い。これらは、イントラネット(intranet)と呼ばれる。b. フィールドバス(fieldbus)下位階層のエリア、ステーション、デバイスレベルでは、センサ・アクチュエータ・表示器、制御機器間のディジタル通信機能に特化した産業オートメーション用ネットワークが利用されている。これらは、広義のフィールドバスと呼ばれる。フィールドバスの特徴として、センサからのディジタル計測データやアクチュエータ制御データのような、比較的データ長の短い計測制御情報が伝送対象である点、また一定期間内に通信が必ず完了するリアルタイム性をもつ点が挙げられる。広義のフィールドバスをさらに機能分類すると、図6・49のとおり、1コントロールバス(control bus):PLC・セル制御用PC・マンマシンインタフェース相互間の通信向け、2狭義のフィールドバス:PLCや制御機器と、工作機械やロボットなどの機器間のディジタル通信向け、3センサバス:センサ信号、アクチュエータ信号(接点開閉信号や、アナログ入出力信号)の高速通信向け、の3種類に大別される。フィールドバスには、異メーカ・異機種の機器が自由に接続できなければならない。このため、通信仕様が完全に公開されたオープンなネットワーク規格が活用されている。IEC/TR 61158:2003,IEC62026:2000においてフィールドバスの国際標準化が試られたが、現在も表6・4のようなさまざまな規格が共存しているのが実情である。おもな規格としてデバイスネット、PROFIBUS(プロフィバス)、Foundation Fieldbus(ファウンデーション・フィールドバス)、CANopen(キャンオープン)、LonWorks(ロンワークス)等がある。また、日本自動車工業会、日本電機工業会が提案するフィールドバス規格であるFL-net,OPCN-1もある。c. 通信プロトコル(communication protocol)ネットワーク上での通信を行うには、送受信双方において通信手順規約であるプロトコル(protocol)を一致させる必要がある。図6・50に示すISOが提唱する異機種間データ通信における通信プロトコルの機能階層モデルを、OSI参照モデル(open systems interconnectionreference model)と呼ぶ。この機能階層の導入によって、上位プロトコルの通信制御ソフトウェアは、その直下の階層のプロトコルさえ知っていれば通信可能となる。インタネット用の通信プロトコルでは、OSI参照モデルのトランスポート層にTCP(transmission control protocol)が、ネットワーク層にIPが使われている。また、ほとんどのLAN では、データリンク層と物理層として、通信速度100Mbpsの100BASE-TX,1Gbpsの1000BASE-T をもつイーサネット規格や、2.4GHz 帯無線で約11Mbpsの通信を行うIEEE802.11規格が利用されている。一方フィールドバスでは、表6・4のように各規格で異なる通信プロトコルが用いられているのが現状である。
