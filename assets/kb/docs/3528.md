などの仕様が用いられる。分散オブジェクト技術の主要機能は、図6・51のような分散環境下でのオブジェクト(アプリケーション機能単位)呼出しの実現である。図6・51のようにクライアント側は、オブジェクトのネットワーク上での位置を特定しなくても、ローカルなオブジェクトと同じようにそのメソッドを呼出すことができる。一方サーバ側は、あたかもローカルなオブジェクトのように、外部アプリケーションから呼出し可能なオブジェクトを実装できる。その実行には、呼出されたオブジェクトを保有するサーバ位置の特定や、送受信メッセージの組立と通信など複雑な処理が必要であるが、オブジェクト呼出しの仲介役であるORB がその処理を担当する。また分散オブジェクト技術では、単なる遠隔オブジェクト呼出しのみならず、セキュリティ、ネーミング、トランザクション、イベント等の付加サー図6・52 CORBA アーキテクチャビスを提供する。ORB の利用により、組織やシステムの変更に対するアプリケーションの柔軟性、オブジェクトベースでのアプリケーションの機能拡張性・規模拡張性を確保できる。b. 分散オブジェクト処理のためのミドルウェアCORBA(common object request broker architecture、コルバ)はOMG(object management group)で標準化が進められている分散オブジェクトのミドルウェア仕様であり、最も普及したORB である。図6・52にCORBA のアーキテクチャを示す。CORBA では、オブジェクトを呼出すときに使われるインタフェースなどの情報をIDL(interface definition language)という言語を使って記述し、これをアプリケーション開発言語(Java,C++)へIDLコンパイラにより変換し、アプリケーションと組合わせて利用する。CORBA の主要構成要素は、1ORB,2ネーミングやトランザクション等のシステムレベルサービス群からなるCORBA サービス、3印刷やメール機能など抽象度の高いサービスを提供するCORBAファシリティ、4医療・通信・製造等の産業分野別のドメイン・インタフェース仕様の四つからなる。生産システム関連では、生産設備のデータ収集や時系列データ管理、分散シミュレーション、PDM/PLM に関するインタフェース仕様の標準化が検討されている。CORBA 以外のORB 仕様としては、Java言語アプリケーションにおいてメソッド呼出しをネットワーク上で実現するJavaRMI(Java remote method invocation)、Java言語でプログラム部品を作成し、それらを相互接続してアプリケーションソフトを構築するためのJavaBeans、ネットワーク分散型ビジネスアプリケーションのサーバ側の処理に必要な機能を追加したEJB(enterpriseJavaBeans)、Windows OS 上のアプリケーション間でソフトウェア部品(COM オブジェクト)どうしがネットワークを通じてデータの交換や処理依頼のやりとりなどを行えるメカニズムであるDCOM(distributed component object model)なども利用されている。またCORBA とCOM 間など、異なるORB 環境間での相互運用も可能である。さらに、もともと軍事用の分散シミュレーション用IEEE標準であるHLA(high level architecture)とそのミドルウェアであるRTI(real-time infrastructure)を利用した、製造ラインの分散協調シミュレーションも試みられている。
