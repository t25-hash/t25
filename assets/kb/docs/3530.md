# 6・6・6 データベース技術

> **章**: 第6章 情報システム
> **節**: 6・6 生産システム情報化のためのソフトウェア基盤技

a. データベースシステム(data base system)生産情報システムは数多くのアプリケーションソフトウェアからなるが、これらはお互いに製品構成、製造プロセス、生産計画、生産実績、在庫、保守サービスなどの情報をデータベース(data base)として管理・保有している。データベースシステムは、共有データの集合であるデータベースと、データベース管理システム(DBMS : data base managementsystem)で構成される。データベース管理システムは、共有されるデータの定義・登録・更新・検索・削除などの要求に応える機能をもった専用システムソフトウェアである。さらに特有機能として、一貫性制御機能、排他制御機能、障害回復機能、機密保護機能などを有している。データベース管理システムの利用により、データの定義形式や操作手順が標準化され、これらを特定のアプリケーションソフトウェアから分離させることができる。データベースシステムのおもな構成を図6・54に示す。ユーザインタフェース部はデータベースを利用する部分であり、アプリケーションプログラムやユーザがデータベース言語(data base lan-guage)を利用し、論理データ部に対するデータ検索や更新を行う。論理データ部はデータベースが管理するデータの論理的な構造を、データモデル(data model)に基づいて記述する。データモデルによって記述されたデータの論理的構造の定義は、スキーマ(schema)とも呼ばれる。また物理データ部は、データベースが実現されている記憶媒体上の物理的な操作を担当する。データベースシステムを複数計算機上に実装し、ユーザ側からは見掛け上一つのシステムとして扱うことが可能な分散データベースシステムも一般的である。b. データモデルデータモデル(data model)は、その表現形式により階層モデル、ネットワークモデル、関係(リレーショナル)モデル、オブジェクトモデルなどに分類されるが、柔軟性・操作体系の統一性・理解のしやすさなどの理由から、現在最も普及しているのは関係モデル(relational model)である。関係モデルは、図6・55のようにデータを関係表(relation)という二次元配列で統一的に表現し、関係表の集合としてデータベースを表現し、データ操作を関係表間の関係演算と集合演算に帰着させたものである。関係モデルに基づいたデータベースとその管理システムは、それぞれ関係データベース(relational data base)、関係データベース図6・54 データベースシステムの機能構成図6・55 関係表管理システム(RDBMS : relational data base management sys-tem)と呼ばれる。代表的なRDBMS としてはOracle,DB 2,Access等がある。また関係データベース向けのデータベース言語の標準として、SQL(structured query language)が広く用いられている。また生産システムにおいて管理すべきデータの特性からデータベースシステムを分類すると、表6・6のようにトランザクション(transaction)型、バルク(bulk)型、アーカイブ(archive)型の3種類となる。バルク型データの取扱い、製品構成情報や設計変更情報のような構造が柔軟に変化する情報の管理、長いトランザクションへの対応などが生産システム用DB では求められるが、上述のような関係データベースのみでは不十分な場合も多く、ユーザアプリケーションにより機能を補って実装されることも多い。
