# 6・7・3 自動車部品メーカにおけるDE

> **章**: 第6章 情報システム
> **節**: 6・72 に示すように,基本的には計画フェイズで工場の能力と負荷

a. 自動車部品メーカのディジタルエンジニアリング(DE: di-gital engineering)の動向バブルの崩壊後,消費者はメーカの製造した製品を購入するのではなく,自分自身のニーズにあった商品を厳選して選択し,購入するようになってきた.このため,市場ニーズをいかに早くとらえ,いかに早く作り,いかに早く商品を市場に出せるかが,企業として生き残っていくための重大な課題となってきている.そこで,自動車メーカは,車両の開発期間を従来の24箇月から,18箇月,12箇月,さらには1けたにすることを検討している.部品を製造し,自動車メーカに納入する自動車部品メーカとしては,これまで以上に密接に自動車メーカと連携をとり,商品を速く納入することが求められてきている.このような状況の中,従来のように試作品を作り,性能評価,搭載検討して納入する仕事のやり方を,根本的に変える必要性が出てきた.フォードは1990年代の半ば,FPDS(Ford product development system)構想を打ち出し,従来の試作中心のものづくりから,ディジタルデータを使ったものづくりに変革していくことを提言した.この動きは,現在も欧米を中心に強力に進められてきている.自動車部品メーカにとって,ディジタルデータを用いて商品開発を実施すれば,物がなくても自動車メーカと会話ができることになる.すなわち,開発期間の大幅な短縮が可能となり,自動車部品メーカにとって重要かつ有効な仕事のやり方である.ディジタルデータを使用して,商品開発期間短縮を検討している事例について,以下に述べる.b. 自動車部品メーカのDE のコンセプト図6・76に,ディジタルデータを用いた,自動車部品開発のあるべき姿を示す.自動車部品メーカは,自動車メーカから提示された計画図を基に,三次元CAD を用い,三次元ディジタルデータ(ソリッドモデルデータ)を作成する.この作成された三次元データは,図面として使われるだけでなく,後工程であるCAE,生産準備,デザインレビュー等に積極的に活用される.最初に作成された三次元データは,後工程で型製作用の付加的なデータが加えられ,データとして成長していくことになる.また,この三次元データは,納入先である自動車メーカや,部品の仕入先にも送られ活用される.自動車メーカでは,部品メーカから提出された三次元データを使って,車両図6・76 自動車CALS のあるべき姿図6・77 CAD データの欠陥そのものの性能評価,例えば,衝突性能評価や車両への組付け性の検討を実施する.部品の仕入先では,三次元データによって製品情報が先出しされることにより,製品開発の早い段階から,型の製作準備等に取り掛かれることになる.前述のように,この三次元ディジタルデータは,製品開発の各段階において成長させられながら後工程に伝授されていくことになる.このため,同一製品のデータであっても,複数の三次元データが存在することになる.すなわち,CAE用や,型製作用の三次元データが存在する.これらの三次元データの履歴も含めたデータの全体を管理するシステムがPDM(product data management)である.PDM は,品番情報も含む製品のすべての情報を管理するDEの要である.以下,自動車部品メーカのDEを構成する代表的な技術,CAD,CAE,PDM について,以下に記述する.c. 三次元CAD データ作成技術ディジタルデータを活用し,b項で示したあるべき姿を実現するためには,三次元CAD で作成されたデータが,(1)後工程で活用しやすくなっていること(PDQ),(2)社外とのデータ互換が取れることが重要である.この2点について中心に記述する.. PDQ(product data quality)三次元データを作成しても,後工程で活用できなければ,今まで二次元図面を基に進めてきた仕事のやり方と全く変わらない.しかし,後工程で三次元データを活用する場合に,さまざまな問題がある.三次元データとしては完全な形状に見えても,データの作成途中で細かな欠陥(図6・77)ができ,これを後工程(例えは,CAE,金型製作)で活用しようとしたとき不具合となる.このため,この欠陥を手作業で修正しなくてはならない.欠陥が発生しにくいデータの作り方をマニュアル化したり,あるいは,頻繁に使用される形状はライブラリ化することによって不具合を防止している.. 異機種CAD データ間でのデータ変換現在,世界中の自動車メーカで使用されている三次元CAD は,大まかに3種類ある(CATIA,Unigraphics,Pro/Engineer).自動車部品メーカは,納入先の要請に基づき,おのおののCAD にあった形式でデータを提出しなくてはならない.取引先が多くなればなるほど,複数のCAD を導入し,納入先と同じCAD を使ってデータを出さなくてはならない.部品メーカにとって,複数CAD を使用することとは,1コストがかさむ,2複数のCAD を操作できる人材を育てなくてはならない,3生産システムとの連携も複数考慮しなくてはならない,という大きな問題がある.このため,データをソフト的に変換し,異機種CAD のデータであっても同じデータを作り出せる技術が強く要望されている.しかし,現在開発が進められているが,十分な機能を有するソフトはまだできあがっていない.d. CAE 技術今まで実験で製品性能を求めていたが,CAEを用いると計算機の中で性能を求めることができ,さらには,製品加工上の作りやすさも評価できる.自動車部品メーカのDEを実行していくうえで重要となる,下記二つのCAE技術を中心に記述する.. 接合部のモデル化技術CAEを実行するうえで,単品の強度解析等は,現在頻繁に実行されている.しかし,自動車メーカとの連携を考えたとき,部品どうしや,部品と車両がボルトや溶接等によって接合される部分を考慮し,解析を実施しなくてはならない.接合部の考慮は,近年非常に頻繁に実施されている衝突解析の精度を向上させるうえでも重要である.接合部は,非線形的挙動を示すことが多いため,実験での合せ込みが重要である.. CAE 簡易解析ツールCAEは,計算機の中で実験をし,性能を求めるものである.CAEをCAEの専門家ではなく,設計者が活用できれば,良い製品を早く作り出すことができる.一般的にCAEは,計算用のメッシュを作成することが難しく,さらには,計算に使用する境界条件の設定が難しいという問題がある.そのため,CAEは,CAEの専門家が使用するのが常であった.しかし,自動車部品は自動車に比較して小物であり,その結果自動車部品メーカで実施されるCAEは,自動車メーカに比較して小規模である場合が多い.このため,設計者自身がCAEを活用するメリットが大きい場合が多々ある.近年,設計者が容易にCAEを活用できるツール(SOL!BOX,MSC.Acumen等)が販売されてきた.図6・78にそのツールの一例を示す.これは,あらかじめ計算のやり方(メッシュ作成,境界条件等)を設定しておけば,パラメータ変更をした計算が,自動に実行できるシステムである.設計者がCAEを活用するうえでの負荷を大幅に低減している.このツールを用いて,製品加工時に発生する塑性変形をCAE解析することによって,設計者が製品設計を行っている段階で,製品の作りやすさを簡単に評価できる.e. PDM(product data management)図6・79に現在考えられているPDM システムの構成図を示す.イメージ的に書くことは簡単であるが,これを体系的に整理し,システム化することは非常に難しい.このため,自動車部品メーカに限らず,自動車メーカ図6・78 CAE簡易解析ツール(板ばね強度シミュレータ)7―147においても,大きなプロジェクトを企画し構築している.PDM システムとは,拡大解釈をしていくと,製品のライフサイクル,サプライチェーンマネジメント等を含んだシステムとなる.それゆえ,システムが巨大化し,各機能の連携を考慮したシステムが作りにくくなってしまっている.まずは,製品設計の技術データ(三次元ディジタルデータ,品番等)を中心にシステム構築することが肝要である.
