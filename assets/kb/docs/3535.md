# 6・7・4 航空機CALS(aero CALS)

> **章**: 第6章 情報システム
> **節**: 6・72 に示すように,基本的には計画フェイズで工場の能力と負荷

a. ボーイング777旅客機開発での事例1991年に日本側での作業が始まった777の開発では,日本の航空機機体5社が共同開発に参加して,おもに胴体の構造物を分担した.この開発を日米で同時並行的に進めるため,日米間にコンピュータネットワークを作り,各社ともボーイング社が提供した同一のシステムを利用して,設計データを共有するコンカレントエンジニアリング(concur-rent engineering)を実施した.この結果,ライフサイクルコスト(life cycle cost)の削減につながる設計変更が従来機種に比べて70%減少し,設計期間は30%短縮した.また次の開発方法が定着した.. CAD データ基準従来は紙の図面が基準であり,CADデータは参考としてCAM で使われていた.しかし図面を100%CAD 化し,これを基準とすることにより全面的にCAM の展開が可能となった.. 三次元CAD による設計従来の3面図を描く設計から,三次元CAD によりモデリングを行う(立体図を描く)設計に転換した.これにより,コンピュータ上での部品や装備品の仮想組立による干渉チェックができ,設計変更を大幅に減らすことが可能となった.b. 航空機CALS 実証実験(社)日本航空宇宙工業会では,1996年から2年間にわたり,情報処理振興事業協会の企業間高度電子商取引推進事業の一つとして30社の参画を得て,「航空機CALS プロジェクト」を実施した.図6・80に示すシステム機能を開発し,実証実験によりその効果を確認した.. 組立シミュレーション(electronic mock up simulation)参加各社が利用している各種のCAD システムを用いて,主翼取付エンジン部と降着装置の三次元ソリッドモデルを作成した.この結果,従来の作業時間に比べて45%削減でき,効果は次によることが実証できた.(1) 各社が各種のCAD システムで作成したモデルをSTEP規格のデータを介して交換することにより,機体会社における装備品等のCAD モデルの再作成が不要となった.(2) 従来はCAD システムを用いて数点の部品間での干渉チェックを実施していたが,多数の部品間ではチェックの対象となる部品の選択が困難であった.この選択を自動化することで,干渉チェックがもれなく実施可能となった.. 分散データの統合管理主契約会社と従契約会社および関連会社に役割を分担して,図面の出図,改訂,配付の業務を実施図6・80 航空機CALS システムの構成図6・79 PDM による製品情報管理7―148生産システム工学した.この結果,従来の作業時間に比べて56%削減でき,効果は次によることが実証できた.(1) 各社で図面データを共有することにより,図面番号の採番通知や図面等の紙を送付する業務および出図状況等の台帳管理業務を電子化する効果が大きい.(2) 社内承認のワークフローを電子化して時間短縮するとともに出図遅れが減少する効果も大きい.. 技術文書の作成・管理取扱説明書等をSGML(stan-dard generalized markup language)規格を用いて作成,変更,管理を行う業務を実施した.この結果,技術文書を20年間維持管理する業務では,従来の作業時間に比べて47%削減でき,効果は次によることが実証できた.(1) 文書の初版作成時間は若干増加した.これは市販SGML作成ツールの機能不足のため,ワープロ文書データからSGMLに自動変換したが,変換不具合発生時の処置等に問題を残したことによる.(2) 頻繁な維持改定作業時間が,紙による作業に比べて半減した.c. 小型民間機への適用事例近年,日本の航空機機体各社は,海外各社との共同開発による小型旅客機の開発に当たり,設計の初期段階から三次元CAD を使用したコンカレントエンジニアリングを適用している.図6・81は小型民間機の後胴部分の事例であるが,約4000点の部品を三次元モデル化したものである.この結果,約2000箇所の部品の干渉を設計段階で未然に防止することができた.また,三次元CAD モデルを重量計算やパーツカタログ,治工具の設計・製作,部品のNC(数値制御)加工等に活用し,大きな効果を得ることができた.d. 防衛調達CALSCALS の目的はライフサイクルコストの削減であり,航空機業界では図6・82に示すように,設計段階や製造,後方支援の各段階でのCALS の開発と実用化を進めている.防衛庁では新規主要装備品の調達業務の100%電子化を目指している.企業側では,この「防衛庁CALS」に対応するための必要な情報を電子化して,防衛庁と企業間で情報交換を実施する必要がある.このため,通産省の支援を得て1999年より2年間で「防衛調達CALS」の開発と実証実験を実施した.この概要は次のとお図6・81 小型民間機への三次元CAD 適用事例図6・82 航空機ライフサイクルでのCALS の適用りである.. 防衛庁技術研究本部との技術審査資料の交換主契約会社は,関連会社と協同して図面や資料を作成し,技術研究本部との間で修正指示受けなどの情報を電子交換する.. 防衛庁との調達情報の交換防衛庁における調達業務の電子化に対応するため,企業側にて各種の調達関連文書を電子的に作成して交換する.. 自衛隊との技術情報の交換主契約会社は関連会社と分担して取扱説明書などを作成・改訂する.この膨大な量の技術文書を,分担する各社に電子的に配分し改定のうえ,再度編集する.. 企業間EDI(electronic data
