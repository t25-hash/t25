# 8・1・2 通信ネットワークのオープン化

> **章**: 第8章 社会と生産システム
> **節**: 8・1 生産システムのオープン化

a. MAP生産システムのオープン化は、通信ネットワークのオープン化から始まった。1982年にGM をはじめとするビッグスリーが工場内ネットワークのプロトコルの標準化の提案を行ったのが最初である。工場内のネットワークは、悪環境における信頼性、高速データ通信における信頼性、リアルタイム性などの理由により、オフィス環境の通信とは異なるネットワーク仕様が必要であるとされ、MAP(manufacturing automation protocol)が提案された。これは、OSI(open systems interconnection)の7層構造に準拠して作られた工業用通信ネットワークのプロトコルで、CATV の同軸ケーブルで接続するLAN(local area network)が基本である。MAPは、イーサネットに比べてインタフェースボードの価格が高いこと、LAN が基本的な概念であったことから、一般的には普及しなかった。しかしながら、ネットワークを介して仕様の異なるFA 機器を結合することでマルチベンダのシステムを作ることができること、基本的なネットワークをインフラストラクチ7―169ャとして整備することで、部分的な設備更新によってもシステムの相互互換が保証されるという概念が定着した。また、MAPでは、各種のコントローラなどのFA 関係の機器の機能を通信の視点からモデル化することで抽象化し、仮想設備(virtual manufacturing device)を定義することによってはん用性を高めるアプローチを示した。この考え方は、その後の設備のオブジェクト化やプロファイリングなどの基本となっている。現在は、この部分のネットワークも、管理系や設計系の上位のネットワークとの親和性や経済性から、イーサネットベースによる接続が普及してきている。その一つが、日本の自動車産業からの要求に基づいて、(財)製造科学技術センターで開発したFL-net(エフエルネット)である。b. フィールドネットワークFA 機器のネットワーク接続は、制御機器レベルだけでなく、センサやアクチュエータなどの機器レベルの接続も問題となっている。生産システムの監視、制御においては、さまざまなセンサやアクチュエータを自由に組合わせ、システムの特徴に合わせて構成する必要があり、マルチベンダ環境で、かつ、信頼性のあるデータ互換、相互運用を保証する必要がある。このレベルでは、情報量は多くないが、高速で信頼性のあるデータをリアルタイムで交換する必要がある。このレベルのネットワークはフィールドネットワークと総称され、プログラマブルコントローラ(PLC)やFA コンピュータを中心にネットワークを構成している。ヨーロッパ、アメリカ、日本などのPLCベンダを中心に国際コンソーシアムを形成しており、FieldBus,ODVA,ProfiBusやOPCNなどのネットワークの提案が行われている。c. 工場内のネットワークの構成オープンな生産システムにおけるネットワークの構成例として、図8・1に示すような例がFL-netの仕様書に記載されている。生産計画や設計などの生産情報を上位のLAN ネットワークで結合し、コントローラをFL-netのようなイーサネットベースのネットワークで結合し、PLC以下でフィールドネットワークを用いている。工場のネットワークは、取扱うデータの量やタイミングの程度によって図8・1のように階層化されるのが一般的である。これらの各層に共通なプロトコルを採用することによって、さまざまな機器がベンダを意識しないで接続されることになる。これがオープン化の第一歩である。
