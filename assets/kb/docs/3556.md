# 8・1・3 生産機能と生産情報のオープン化

> **章**: 第8章 社会と生産システム
> **節**: 8・1 生産システムのオープン化

a. 生産システムモデルによるオープン化通信機能の相互接続が実現しても、生産システムのオープン化は十分に達成することはできない。製造業には多くの機能が存在しているからである。それらが有機的に結合して製品の生産を実現しており、多くのソフトウェアがそれを支援している。これらの機能を明確にして、ソフトウェアのマルチベンダ環境による相互接続を提案しようとする試みがなされている。この試みは、1970年代にアメリカの空軍が生産資源の調達や材料の調達の図8・1 ネットワークシステムの構成例7―170ためにシステムを記述したI-CAM(integrated computer aidedmanufacturing)プログラムによって始められた。IDEFと呼ばれる構造化図式モデル法が用いられた。その後、CIM(computerintegrated manufacturing)のアーキテクチャとして、CIM/OSA(open system architecture)がEU で提案された。これらの提案は、製造業の基本的な構造や機能は共通であり、特定の方法や枠組を決めることで相互運用が可能であるという考え方で進められてきた。しかしながら、製造業は、製品や工程や市場の状況でかなり異なり、一般的なモデルを作成することは困難である。そこで、最近では、オブジェクトモデリング手法を用いて、業種ごとや機能ごとに柔軟にモデル化する方法が採られてきている。その例として、半導体製造業に対象を特定したモデルがある。それは、SEMATECHモデルと呼ばれるもので、製造機器のインタフェース情報などの詳細な部分までをモデル化した例として評価されている。同様な方法で、組立製造業のシステムを定義しようとする試みが、ISA(Instrument Society of America)においてなされ、ISA-95という規格として、国際標準化機構(ISO)に提案されている。今後、製造業の操業にソフトウェアの比重が高くなるにつれて、生産システムの機能を明確に記述して、第三者に開示していくことで、オープン化を促進していくことになる。b. 基幹システムのオープン化生産管理の基幹システムは、MRP(material resources planning)からERP(enterpriseresource planning)へと進歩してきている。ERPシステムとして、SAP社やBAAN 社のシステムを導入する企業が増えてきている。特に、薬品や化学工業製品を生産している企業では、その傾向が顕著である。ERPは、基本的に、生産計画や受注などのデータをデータベースによって結合し、生産管理と経理あるいは生産管理と営業、製造と品質管理など企業内のデータ連携を図るものである。基幹システムを統合化することで、一企業内の複数のシステム間の情報の共有を通して、生産情報のオープン化が行われてきている。標準化によるオープン化ではないが、SAP社とBAAN 社の市場占有率の高さから、それぞれの構造に合わせたソフトウェア群がアライアンスを組んだ企業から提供され、デファクトスタンダード化されてきている。c. スケジューリングデータのオープン化スケジューリング(scheduling)は、それぞれの企業の独自データと独自のアプリケーションで作成されるのが基本である。これは、それぞれの設備や能力などが含まれているので、公開がちゅうちょされるものである。しかしながら、昨今の製造業では調達のオープン化やアウトソーシングが進み、工場自身、あるいは自身の工場が管理できるところのみで生産するのが困難になってきている。また、スケジューリングの不確定さを補う在庫を十分にもつことも困難になってきている。このような状況で、工場の壁を超えて、あるいは企業の壁を超えてスケジューリングデータを共有する必要が生じてきた。例えば、XML(extensible markup language)を用いて、スケジューリングデータをウェブを介して交換する仕様がある。この仕様は、PSLX コンソーシアムが公開している。これは、グランドデザイン、XML記述規約、用語などからなっている。図8・2にデータ交換の例を示す。XMLによるデータ交換は、交換すべきデータの記述に用いられ、処理に関する記述を制限しない特徴があるので、すでに存在しているソフトウェアなどの資産を有効に活用して、データ交換が可能になる。そのため、スケジューリングデータのみならず今後のオープン化のツールとして注目されている。d. MES による製造データのオープン化生産管理や計画分野は基幹システムやデータのオープン化が進み始めている。また、工業用ネットワークを通して機器のオープン化も進んでいるが、製図8・2 PSLX によるスケジューリングデータの交換例造そのものの実態を表現するデータを機器間あるいは生産管理や生産計画と結びつけることは難しい。そこで、ERPと現実の設備群の間にMES(manufacturing execution system)を介して、実績情報、制御情報を結びつけることによって、計画系、管理系と設備系システム間に存在していた壁を取り除いてオープン化しようとする試みもなされている。MES では、UML(unified modeling language)を用いて、設備やコントローラを仮想設備として記述し、CORBA などの共通プラットフォームを用いてデータ交換を行おうとしている。この仕様は、(財)製造科学技術センターにおいて、Open MES 仕様書として公開されている。
