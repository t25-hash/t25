# 8・1・4 オープン化の傾向

> **章**: 第8章 社会と生産システム
> **節**: 8・1 生産システムのオープン化

生産システムのオープン化に対しては、企業において依然、強い抵抗感がある。それは、独自のものとして培ってきた生産技術をはん用的な技術やソフトウェアに置換えられることに対する抵抗感、生産上のノウハウなどの流失、企業がもっている能力に関する情報の漏えいなどの心配がその根底にあるからである。一方、自社内の設備だけで生産するのではなく、他の企業と協力して生産を効率化するサプライチェーン管理やグローバル生産など企業との協調が重要になってきており、オープン化の重要性が顕在化してきている。そこで、最近では、CIM システムに代表される一企業のみを対象とした総括的な統合ではなく、他企業あるいは協力企業群を対象とした情報の統合が議論されている。企業における強い抵抗感と、事業上のオープン化の要求を調和させる必要がある。情報処理技術では、オブジェクト指向言語やJAVA あるいはXMLやUMLなど部分のモデル化を可能とするコンピュータ技術が成熟しつつあり、これらの利用による柔らかい統合が可能となってきているので、オープン化に対する抵抗感も減少しつつある。生産システムにおけるソフトウェアの比重は、今後大きくなることはあっても、少なくなることはない。このような状況において、独自にアプリケーションソフトウェアを開発していくには、費用がかかりすぎる。また、そのソフトウェアの保守なども問題となってくる。このようなことからオープン化の必要性はますます強くなってきているが、この分野では国際標準化機構からの提案ではなく、多くのコンソーシアムからさまざまな提案がなされるのもオープン化の特徴であり、長期の利用可能性を検討して、導入を決定しなくてはならない。オープン化の流れは、国際標準化機構であるISOやIECなども、安全や健康、環境、品質などに関わる問題以外の相互運用に関する問題に対してこの傾向を推進するように、従来の厳格な規格から、より開かれた柔軟な規格体系へと変化し始めており、まさにオープンな環境を提供するように変わってきている。今後も注目しなければならない問題である。
