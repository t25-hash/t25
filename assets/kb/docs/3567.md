# 8・3・5 PLP(product liability prevention、製造物責任予防)

> **章**: 第8章 社会と生産システム
> **節**: 8・3 安全の国際的整合化

としての国際規格機械文明がこれまで正常な発展を遂げてきたとは必ずしもいえないが、それは別として、機械による恩恵はそれを求める者にひとしく享受されるべきである。絶対安全があり得ないという条件のもとで機械の恩恵を求めようとすれば、設計者は使用者に対して残留リスクに対する説明責任を負う。その説明責任で果たすべきことは、使用者に対し、正当なリスク低減の協力の承認を得ることである。ISO12100に基づくリスク低減のための設計手順を、リスクアセスメントと本質安全設計を中心に説明した。リスクの受容とは、機械の使用者による安全の判断であるが、リスク低減の責任はあくまでも機械の設計者が負う。設計者が可能な限りのリスク低減を行った結果、使用者は残るリスクを甘んじて受容し、事故防止を引き継ぎ、場合によっては災害そのものが受容されることが期待できるのである。したがって、設計段階でリスクが許容できるか否かを判断する場合、リスク低減が正当な手順に沿って実行したか否かが問われる。国際規格はその正当な手順を与えるガイドラインであり、これに従って事故防止を実行したことによって、事故が起こってもやむを得なかったかどうかの判断が期待できるのである。安全確認の原理は、安全を確認して機械を運転するシステムを求めている。技術の時代的背景(state of the art)を考慮し、その時代の最高レベルの安全確認システムを設計して設計者が責任を果たすのは、これを怠って課せられる事故の補償責任に対する事前の免責を得るためである。国際規格は、設計者からの立場から、PLP7―179(製造物責任予防)を本来の目的としているのである。
