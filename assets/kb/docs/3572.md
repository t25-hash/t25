# 8・5・3 分散協調エンジニアリング

> **章**: 第8章 社会と生産システム
> **節**: 8・5 国際分業と協業

基本設計を日本で行い、その後、設計拠点を海外へ移す、トランスナショナルといわれるプロジェクトは数多くの実績があるが、以下に紹介するユニットごとに世界に分散して同時に作業を進めていくプロジェクトは、日本のエンジニアリング会社の先進的な例である。1999年末に南米の製油所近代化プロジェクトの基本設計業務を受注し、2000年5月に完了した。その後、基本設計以降の詳細設計と機器資材調達業務を2001年3月に受注した。このプロジェクトの特色は、図8・19に示すようにマルチタスクでユニットごとに韓国、インド、ブラジル、日本と、世界の4箇所に設計拠点が分散して業務のやり取りが行われており、顧客の要望でインタネットベースの集中型ドキュメント管理が必要であったことである。また、設計のプロダクトである図面やあらゆるドキュメントは、従来ならばアウトプットしたものを国際宅配便で送って承認を得るという手続きが行われていたが、今回は顧客より逐次電子データでの納入が要求された。そこで、社内に勤務している顧客の駐在員のために別のインタネット回線を用意して、ドキュメント管理サーバで直接アクセスする使い方をしており、各設計拠点と顧客のエンジニアリング情報の共有を実現した。なお、韓国では、高速なインタネット環境が整っているものの、すべての拠点が同様のインフラ環境でないため、出力したドキュメントを要求するベンダもまだ多数存在した。ただ、メインコントラクタのエンジニアリング会社では自社で作成したCAD 図面は2割あるかどうかで、他は200を超すベンダが作成しており必ずしもCAD でない図面もある。このようにエンジニアリング会社のプロジェクトにはさまざまなドキュメントが混在している状態でもある。
