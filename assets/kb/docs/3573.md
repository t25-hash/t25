# 8・5・4 コラボレーションによるアフタセールスサービス

> **章**: 第8章 社会と生産システム
> **節**: 8・5 国際分業と協業

エンジニアリング企業が対象とする製品の納入先は世界中に分散しており、かつ製品ごとに構成するコンポーネントとそのベンダが異なるので、アフタセールスサービスに対して設計・建設プロジェ図8・19ネットワーク構成図8・20 VERAM7―183クト時のような体制をとることができない。通常は、製品納入時にプロジェクトを解散し、顧客の要望に対してそのつど必要な体制を整えている。IMS/GLOBEMEN プロジェクトでは、企業ネットワークによるグローバル生産のあり方を研究開発しており、その概念と方法論を用いたコラボレーションによるアフタセールスサービスを紹介する。a. IMS/GLOBEMEN の概要GLOBEMEN プロジェクトの共通成果としては、VE構築の方法論であるVERAM(virtualenterprise reference architecture and methodology)、その中でもVE(virtual enterprise、仮想企業)構築のアーキテクチャであるVERA(virtual enterprise reference architecture)などがあり、以下に簡単に説明する。VERAM は、図8・20に示すようにVE構築のための手法、モデル、ツールやガイドラインの集合であり、これを用いることにより、設計・建設プロジェクトのVEやアフタセールスのためのVEなどがより迅速に構築されて運用に入ることができることをねらったものである。VERAM の中の主要コンポーネントであるVERA は、図8・21に示すようにネットワーク、VEおよび製品のライフサイクルとそれらの間の関係を表現したもので、機能、情報、組織、資源、の各視点でモデル化していくためのアーキテクチャである。企業群が集合してネットワークを構成し、その運用フェーズでVEを構築する。また、VEの運用フェーズで製品が開発・製造されるそのライフサイクルを表現するものである。各ブロックはライフサイクルの各フェーズで実行するアクティビティを表現している。VEの運用フェーズでカバーする製品のライフサイクルの範囲は場合によって異なるので、矢印で示している。b. VSE とはVSE(virtual service enterprise)とは、アフタセールスサービスのためにサービスネットワークから顧客の要求を満たす企業群が、サービスを提供するために構築するVEをいう。顧客ごとに要求は異なるので、図8・22に示すように、顧客の図8・21 VERA図8・22 VSEの概念7―184生産システム工学サービス要求1に対しては2社が共同でVSE1を、顧客のサービス要求2に対しては3社が共同でVSE2を構築するといったように、顧客ごとに最適なVSEを構築する必要がある。例えば、顧客が製品を提供したエンジニアリング会社から離れている場合、サービス会社(オンサイトサポート)、部品供給会社(スペアパーツ供給)と協力して図8・23のようなサービス体制を構築することが考えられる。すなわち、図8・23では、顧客の設備はエンジニアリング会社の遠隔診断システムにより常時モニタリングされている。もし、異常が生じて、部品の交換やプラントの最適化などの作業が必要になった場合、エンジニアリング会社はサービス会社にオンサイト支援を依頼する。一方、エンジニアリング会社はサービス会社に対して、マルチメディア保守マニュアルを用いた遠隔コンサルティングを提供する。また、部品の供給が必要になった場合、エンジニアリング会社はサプライヤに対して、部品の製造・輸送・納入・組付けを依頼する。この際も、複雑な部品に対してはエンジニアリング会社がサプライヤに遠隔トレーニングを行い、スムーズな部品の組付けを支援する。c. ホスティングサービスによるリモート支援顧客に納入した製品をサービスするためには、製品の稼働状況をできる限りリアルタイムでモニタリングして把握しておく必要がある。また、プラントのような大規模なシステムの場合、そのコンポーネントはマルチベンダ対応になり、複数のベンダの協力が必要となる。そこで、図8・24に示すようなシステムを構築し、顧客、エンジニアリング会社、メンテナンス会社、ベンダが協力して運転や保守の支援を行える環境を構築した。図8・23 VSEの事例ただし、一般に顧客は運転データの開示に保守的で、ましてや他社の運転データと同じデータセンタに保存することには消極的である。したがって、この課題を解決するためにはより安全なプラットフォームの開発が必要となる。文献(1) GM, Manufacturing Automation Protocol、(1982)。(2) FL-Net,http://www.mstc.or.jp/jop/fl-net/tech.html、(2001)。(3) FieldBus,http://www.fieldbus.org/、(2002)。(4) ODVA,http://www.controlnet.org、(2002)。(5) ProfiBus,http://www.profibus.com、(2002)。(6) OPCN,http://www.jema-net.or.jp/Japanese/news/opcn/top-opcn.htm、(2002)。(7) MSTC,FL-net共通マニュアルガイドライン、(2000)、7.(8) 福田好朗・大見孝𠮷, CIM のためのオープンアーキテクチャ、(1991)、日刊工業新聞社。(9) SEMATECH, Computer Integrated Manufacturing(CIM )Application Framework Specification 1.2、(1995)。(10) ISA-95,Enterprise-Control System Integration、(2000)。(11) PSLX コンソーシアム、PSLX 仕様書、(2002)。(12) Nishioka,Y.,Proc.
