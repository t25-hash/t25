# 10・1・3 技術開発における重要性 技術開発・利用に関し，

> **章**: 第10章 知的財産に関する法制度
> **節**: 10・1 知的財産制度概要

β9－210 β9編 法 工 学
知的財産問題の発生の形態の例について以下に説明する．
a．研究開発における発明の権利化 一般に企業では従業員の
発明の譲渡を会社が受ける職務発明規則を特許法に基づいて定めて
いる．研究開発に携わる開発者が発明と認識し，会社にその内容を
届け出なければ，ノウハウとして社内に蓄積されることはあっても，
権利化をすることはできない．開発者は研究開発を優先し勝ちであ
るが，先願主義を踏まえ，研究開発の成果は商品（図面）と知的財
産であることを認識し，発明を速やかに届け出ることが要求される．
第三者が先に出願すると発明が早くても権利を取得ができなくなる．
b．入手技術（ノウハウ）の無断使用 研究開発の過程で，開
発者は多くの技術情報に接するが，社外からの情報について守秘義
務を負うものがあるときに，情報を完全に分離して管理することが
必要である．複数社に同一仕様の設計を依頼し，一社の設計を採用
するような場合，意図的ではなくとも一方の会社のノウン・ウを別の
会社に適用させるようなケースであり，十分な注意が要求される．
c．商品の知的財産権侵害 開発成果は商品の形で実施されるこ
とが多い．この商品が第三者の権利を侵害すると，実施料の支払い
や販売を中止せざるを得なくなることがある．この問題を回避する
ために，研究開発の初期から発売まで，商品の仕様を決めるつど第
三者の権利の有無を調査し，権利がないことを確認するのがよい．
d．技術提携での権利・財産としての主張と尊重 共同開発や
技術支援を行う場合，開発以前から所有する発明・ノウハウは所有
者に帰属することは当然として，開発や支援の開始後になされた新
規及び利用発明等の帰属の取り扱いは，事後の企業活動に影響する．
成果への寄与度は互いに尊重し合うとしても，帰属決定には十分な
検討が必要である．場合によって，当初から所有する権利を含めた
ライセンスを受けることも考えなければならない．
e．技術力の主張 特許出願や特許権の取得は，会社もしくは
商品の技術力の高さを示すものとして利用される．また第三者の参
入を排除して長期間独占的に実施することで，更に技術力を示すこ
ともできる．海外進出やメーカとの共同開発を行う場合でも技術水
準の高さを示すものとして用いることができるので，ポートフォリ
オに基づく特許網を構築することは重要である．
