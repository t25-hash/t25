# 6・4・2 リスクマネジメントシステム(1)

> **章**: 第6章 マネジメントと法制度
> **節**: 6・4 リスクマネジメント

a.規格化の経緯 近年企業を取り巻くさまざまなリスクが発現し、企業の危機管理のありかたを問われる事件が多数発生している。これらの結果生じるさまざまな被害は当該企業の運営や存続に大きな困難を与え、またその被害が関係者はもとより一般市民にまでおよび大きな社会的損失をもたらすこともまれではない。これらのリスクに企業が適切な組織的対応を行い、自らの経営を安定させ、また、万一・の事態発生の際にはその影響を極小化し、社会的損失をできる限り発生させないことが求められてきている。そのため、1995年に発生した阪神・淡路大震災を契機に、自然災害や火災爆発事故などのさまざまな原因から生じる危機への対応を標準化するための検討が開始され、2001年3月にJIS Q2001「リスクマネジメントシステム構築のための指針」が制定された。この規格は、企業や自治体などがリスクに対し組織的な取り組みをするため、日常の予防活動と緊急事態時の対応などを総合した経営の仕組みを定めたものである。同様な規格は海外でも制定されており、例えばオーストラリア・ニュージーランド共同規格(AS/NZ 4360:1995)はリスクの洗い出しとリスク評価手法に秀でており、カナダ規格(CAN/CSA-Q850-97)は経営者の意思決定のために関係者とのリスクコミュニケーションを重視したものとなっている。またイギリスでもPD 6668が制定されている。なお、ISOでは現在リスクマネジメントの用語の統一が検討されており、その後規格の検討が開始される予定である。b.規格の狙いと特徴 過去の災害や事故などの教訓を踏まえ、危機対応には経営者の関与が不可欠であること、また成功した危機管理はそもそも危機に陥らないようリスクの発現を未然に防ぐ日常活動が適切に行われていることが重要であるとの認識から、リスクへの対応は、日常の経営活動の一環として取り組むことが肝要であり、したがって経営の仕組み、すなわちマネジメントシステムとして規格化されることになったものである。マネジメントシステムといえば品質マネジメントシステムのISO9001や環境マネジメントシステムのISO 14001が知られているが、これらと同様に、JIS Q 2001に規定されているリスクマネジメントシステムは、次の特徴をもっている。(1) 経営者が関与 組織の最高経営者が自らの責任を明確にし、方針を立て、実施状況を定期的にレビューし、次のサイクルをスタートさせる。(2)継続的改善の実施 今年よりも来年、来年よりも再来年と日常活動のなかで継続的に改善活動を実施する息の長い体質改善運動である。(3) 自己責任による身の丈にあった活動の選択 規格では経営者が実施する必要のあるさまざまな項目を要求事項として規定している。この各要求事項に対してどこまでどのように行うかは企業の自己責任で決定される。したがって、企業の規模、業種、環境や立地基盤などに応じた企業の身の丈にあった目標の設定や実施内容の選定が可能である。なお、ここでいうシステムとは仕組みや制度のことで、人事異動などで要員が入れ替わっても活動自体が継続し機能する必要がある。c.規格の概要と適用事例 ここではJISQ2001「リスクマネジメントシステム構築のための指針」の概要を説明する。また、リスクマネジメントシステムは現在普及の緒についたところであるが、先進的企業を中心にすでに導入しているところもあるので、その事例をいくつか合わせて紹介する。なお、この規格は、各企業がリスクマネジメントシステムを構築するにあたって参照することを目的として制定されたもので、ISO9001やISO 14001などと異なり、第三者認証制度ではなくガイドラインである。したがって、各企業がこの規格の適合を自己宣言することはなんら問題ない。また、すでにISO 9001やISO 14001あるいは個人情報保護JIS Q 15001などの個々の業務を対象としたマネジメントシステム規格を導入している場合には、それらと重複した要求事項を新たに設定する必要はない。さて、この規格は図6・11に示すように七つの原則から成り立っている。ここでいう原則とは要求事項をわかりやすくいくつかの項目にまとめたものを指す。以下順に七つの原則を説明する。原則1 リスクマネジメント方針Plan原則2 リスクマネジメントに関キる計画策定会t原則組織の最高経営者によ診ゴ原則7潟Xクマネジメンgシステム維持のスめの体制・仕組ンDo原則3 リスクマネジメントの実{Check原則4 リスクマネジメントパフHーマンス評価及びリスクマネジ<塔gシステムの有効性評価原則5 リスクマネジメントシスeムに関する是正・改善の実施図6・11JIS Q 2001 リスクマネジメントシステム概念図i.原則1 リスクマネジメント方針 組織の最高経営者(企業全体に適用する場合は通常は社長)がリスクマネジメントに関する方針を明確に定める。何を守るのかを明確にすることが大切である。方針の事例として、「当社は経営方針の実現を阻害するすべての要因を可能な限り排除する。万一の事態発生に際しては、株主をはじめ関係者への影響を極力小さくするよう最大限の努力を行うとともに、経営として事態を反省し再発防止策を適切に構築し関係者からの信頼回復に務める」をあげておく。ii.原則2 リスクマネジメントに関する計画策定(Plan)リスクマネジメントに関する具体的なさまざまな計画を立てるプロセスである。まずリスクの発見、リスクの特定、リスクの見積りなどのリスク分析を行った後、リスク評価を行う(これらはリスクアセスメントと呼ばれ、詳細は6・4・3項参照)。ここで重要なのはリスクの優先順位の決定である。優先順位の高いリスクに対しては、リスクマネジメントの目標を設定し、事前準備にあたるリスク対策を選定する。そして、具体的な事前準備の内容や有事の際の業務内容を定めるなどのリスクマネジメントプログラムの策定を行う。ここで、目標を設定するにあたっては、守るべき対象を明確化するとともに、法的要求事項、関係者へ悪影響を与えるリスクの低減などを考慮する。また、プログラムの策定にあたっては、具体的な実施内容を定めるとともに、責任を明確にし、組β9-157織、日程、投入する経営資源(予算、要員等)などをそれぞれ決める必要がある。リスクとして地震を取り上げて具体例を以下に示す。リスクマネジメントの目標は、1震度6の地震に対して従業員のなかから死者を出さない、2震度6の地震に対して3日以内に業務を再開する、などである。リスク対策は、15年以内にすべての工場、社屋の耐震補強を実施する、2本年度は第一工場の機械設備の固定を実施する、3主力製品のラインの一部を第二工場に設置し第一工場の被災に際し3日以内に代替生産を可能とする、などである。プログラムの策定は、1耐震補強の見積りおよび実施、2第二工場へのライン移設および受発注システムの修正、第二工場の部品業者選定、第二工場への物流ルート開拓、3安否確認および地震対策マニュアルの改訂、4全従業員への地震に関する基礎知識教育の実施、などである。なお、日本の先進企業では、すでにリスク評価を実施し、優先順位の高いリスクを特定し、対処している。それらの企業が特定したリスクを見ると野製品の蝦疵(P:L、リコール)、自然災害や火災爆発等による製品供給の停止、コンプライアンス(遵法)違反、会社および役員や従業員を対象にした犯罪などを定めている例が多い。iii.原則3 リスクマネジメントの実施(Do)原則2で定めたおのおのの計画について具体的な実施手順を作り、日常活動として実施するプロセスである。日常的な具体的な実施業務については、JIS Q 2001では原則7のなかでまとめて規定しているので、ここでもそれにしたがい後述する。なお、JIS Q 2001の特徴として、ここで緊急時と復旧時の対応手順の策定について述べられている。特に緊急辱め実行組織の構築の仕方として、実行責任者のもとに、1情報機能、2分析評価機能、3対応機能、4広報機能、の四つの機能を持つことが重要であると定めている。このように機能で整理すると、日本の企業はほとん、どが上記3の対応機能しかなく、1の情報収集整理、2のその情報の分析評価、4の広報による対応策の関係者との共有、という三つの機能が欠けていることに気付かされる。iv、原則4 リスクマネジメントパフォーマンス評価及びリスクマネジメントシステムの有効性評価(Cllec:k)計画に則ってリスクマネジメント活動を実施していくが、実際にどの程度成果を上げているのか、すなわちリスクマネジメントのパフォーマンスを評価するプロセスである。具体的には、実施状況の監視、進捗状況の管理、実行度合の測定などによりパフォーマンスを評価する。例えば、実施状況の評価では、顧客からの苦情件数、重大クレームの内容など、また進捗状況の評価では、教育訓練の受講者数、備蓄物資の配備状況などが対象となる。ここで、規格では、目標として定量化できるものは目標設定時から定量化し。この評価時点でその指標と実施結果を比較することが望ましいとされている。発生頻度が比較的多いリスク、例えば従業員が多数いる事業所等における交通事故や労災事故などでは数値目標の設定とそれに基づく評価も可能であるが、特定はされているもののまだ経験のないリスク、例えば役員の誘拐や巨大地震などについては評価が難しい。無理に定量化できる部分のみだけで評価をしょうとすると、全体を見誤る可能性があるので実際の適応には注意が必要である。このようにして、リスクマネジメントのパフォーマンスを評価するが、同時にリスクマネジメントシステムのシステム自体の有効性も評価する必要がある。すなわち、例えば何年かリスクマネジメントシステムのサイクルを実行後、到達目標に対して着実に改善が見られているかどうかを判断し、システムの有効性を評価する。具体例をあげれば、交通事故の半減を5年間で達成しようとしたとき、毎年の交通事故件数の評価はパフォーマンス評価でみるが、3ヤたっても一向に減少しない場合は現在のリスクマネジメントシステムが有効でないと評価される。v.原則5 リスクマネジメントシステムに関する是正・改善β9-158の実施(check)必要に応じてシステムの是正・改善をするプロセスである。是正・改善をするきっかけには、1日常点検で不備・不具合を発見した場合、2リスクマネジメントシステム監査の結果から改善が必要と判断した場合、3緊急事態や危機管理を経験して反省した場合、4リスク情報を監視した結果に基づき必要と判断した場合、などがある。上記2のリスクマネジメントシステム監査は、各部門が実行しているリスク対策の進捗状況を監査するものである。一般には内部監査として、リスクマネジメントシステムの観点から実施される。なお、リスクマネジメントシステムではその遂行に対する取締役の役割が明確にされているので、監査役監査の一環として実施される。また、4は常設組織(通の(2)参照)等が、日々のマスコミ情報やあるいは研究会、論文等で知見を新たにしたり、企業の内外から寄せられるさまざまな情報から新たなリスクを発見したり、あるいは注目していたリスクが実際に具現化すると判断した場合、などを意味する。例えば食品安全では、0157の出す毒素は耐熱性が高く、従来の方式では細菌は死滅しても毒素が残留し食中毒の原因になることが判明したとき、これらの事実に基づきその対策を製品製造工程に組み入れることが是正改善にあたる。vi.原則6 組織の量高経営者によるレビュー(Act) リスクに強い組織になるために継続的に改善を行っていくプロセスである。そのために自ら定めた間隔(1年が多いが半年や四半期でもよい)で、組織の最高経営者がリスクマネジメントシステム全体をレビューする。この結果を踏まえて、組織の最高経営者の責任で次のサイクル(Plan→Do→Check→Act)をスタートさせる。レビューでは、定期的な取り組みの内容を把握し、例えば、交通事故対策を5年間実施した結果最終目標である「50%削減」を達成したことを評価し、今後は「この率を維持」するよう目標を改定する、また、次年度以降は新たなリスク対策として「苦情対応マネジメントの導入」を図る、などを行う。あるいは、地震対策を実施しているが、満足にマニュアル改訂や訓練がされていないので、これらの状況を改善するために、専任者を1名人事異動で手当するなどである。vii.原則7 リスクマネジメントシステム維持のための体制・仕組み このようなリスクマネジメントシステムを構築して維持するためには、次のような各種の要求事項を検討して必要に応じて実行する。いくつか重要な点を次に述べる。(1)体制 組織の最高経営者とリスクマネジメントシステム担当責任者の役割がそれぞれ定められている。組織の最高経営者(通常は社長)は、方針を明確にし、取り上げるべきリスクの優先順位の最終決定を行い、有事には最高責任者としての責務を果たし、レビューを実施し、リスクマネジメントに関するすべての責任を果たす。リスクマネジメントシステム担当責任者は、リスクマネジメントを遂行する実務の最高責任者である。特に重要なのは、経営者(常務会に出席するなど意思決定ができる地位の役員、中小企業では社長が兼務してもよい)のなかから責任者を選任しなければならない点である。従来日本の大企業では、業務ごとに担当役員が一般業務およびリスク処理を執行するのが一般的であったが、新たな危機や事件が発生した場合担当部門が不明確であると初動が遅れ危機の拡大につながる。これを防ぐために、担当が不明な事例や複数の部門にまたがり調整を要するリスクの発現にあたっては、リスクマネジメントシステムを担当する役員が責任者として一元的に対応し、初動の混乱を回避する。体制の一例を図6・12に示しておく。(2)常設組織の業務 リスクマネジメントシステムの構築や円滑な運営にあたり、常設組織を持つとノウハウの蓄積にも有効である。図6・12の例では、リスクマネジメント委員会がこれに該当する。このような常設組織は企業全体を包括して以下の業務を行う。1リスク情報の収集と分析。2 自社を取り巻くリスク動向について把握し、経営者に報告する。3 重要な取り組みリスクに対するマニュアルを作成し、メンテナンスする。4法務リスク、財務リスクなどリスクごとに特定の部門にそのリスク対策を委ねた場合、それらのリスクにつきリスク管理状況を調査し経営者に報告する。5 経営者および従業員に対して教育訓練を行う。社長組織の最高経営者リスクマネジメントシステム担当責任者副社長リスクマネジメント委員会監査事務局;経営企画製造情報システム総務営業本部法務人事広報品質管理図6・12 リスクマネジメントシステムの体制例(3)仕組み 原則7には、リスクマネジメントシステム維持のために総務、製造、品質管理などの各部門が日常的に実施する業務が仕組みとしてまとめられている。仕組みとして要求されている事項として、1リスクに関する教育訓練、2リスク対応のシミュレーション、3関係者とのリスクに関する情報の共有化および情報開示などのリスクコミュニケーション、4リスクマネジメントシステム文書の作成、5リスクに関連した文書の管理、6発見したリスクの監視、7リスクに関連した記録の維持管理、8リスクマネジメントシステム監査、などが定められている。ここで、上記4のリスクマネジメントシステム文書とはマニュアルが代表的なものであるが、それ以外のりスクマネジメン。トに関するさまざまな提案書、調査報告書、稟議書などのすべての文書が該当する。また7のリスクに関連した記録の維持管理とは、事故記録等の他、教育訓練等のシナリオや訓練の議事録などの記録をとり維持管理することで、それを振り返ることがマニュアルの改訂や対策の改善につながる。引用文献・参考文献(1)
