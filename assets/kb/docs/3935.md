> **節**: 2・2 コンピュータの基礎

コンピュータの基本的な構成を非常に簡略化して示すと図2･1の
ようになる．CPU（central processing unit ）は計算を行う装置
で，機械命令（machine instruction）によって動作が規定される．
機械命令（以下，単に命令と記す）の種類や形式はCPUによって
固有であり，それらを総称して命令セットアーキテクチャ（ISA:
instruction set architecture）と呼ぶ．代表的なISAに，IBM社
の体系を有している．
の370アーキテクチャ，インテル社の8086アーキテクチャ，サ
ン・マイクロシステムズ社のSPARC（スパーク）アーキテクチャ
などがある．主記憶装置（main memory unit ）は命令や演算対象
となるデータを格納する装置である．コンピュータの動作は，主記
憶装置から読み込んだ命令に従って，主記憶装置からデータを読み
込み，演算を施し，結果を再び主記憶装置に書き戻すことの繰返し
ととらえることができる．これらの動作を正しく行うためには，主
記憶装置に格納された命令やデータの位置を一意に指定する仕組み
が必要になる．これは，主記憶装置内のすべての位置に1対1に対
応する番地（アドレス：address ）を付与することで実現される．
すなわち，CPUと主記憶装置との間のデータの授受は，必ずアド
レスを指定して，そのアドレスによって定まる主記憶装置内の位置
を対象に行われる．
‘1’の値をもつビット（bit ）の並びとして与えられる．標準的な
長さ（ビット数）をもつデータはワード（word）と呼ばれ，デー
タ間の演算は基本的にワード単位に行われる．また，主記憶装置上
の位置を指定するアドレスにもワード長のビット列が用いられるこ
とが多い．一般にワード長が大きいコンピュータほど，より広い空
間のデータにアクセスでき，より多くのデータを一度に処理できる
ので，能力が高い．コンピュータの発達とともにワード長は増加し
てきており，現在は32ビットないし64ビットが一般的である．コ
ンピュータに意味のある処理をさせるためには，単なるビット列で
あるデータに，文字や数値，さらには画像データや音声データなど
