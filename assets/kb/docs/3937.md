キャッシュメモリと主記憶とのマッピングは，メモリ領域を
32～64バイト程度のブロックに区切り，ブロック単位に行うのが
一般的である．主記憶のどのブロックがキャッシュメモリのどこに
存在するか（あるいは存在しないか）を判定するための機構とし
て，ディレクトリ（またはタグともいう）と呼ばれる対応表が用意
されている．キャッシュアクセス時には，まずディレクトリが検索
されてキャッシュヒットあるいはミスヒットが判定され，ヒットな
らばキャッシュメモリの該当するブロックがアクセスされ，ミスな
らば主記憶がアクセスされる．
主記憶とキャッシュメモリの領域のマッピングにはいくつかの方
法がある．主記憶の任意のブロックをキャッシュメモリの任意の場
所にマッピング可能にする方式をフルアソシアティブ（full as-
sociative）方式と呼ぶ．この方式はキャッシュメモリ上の領域を
フルに活用できる点で優れているが，ディレクトリの構成が複雑に
なり，ハードウェア的なオーバヘッドが高い欠点がある．逆に，主
記憶のブロックのアドレスによってキャッシュメモリ上の位置がユ
ニークに定まる方式として，ダイレクトマップ（direct map）方式
がある．この方式はフルアソシアティブ方式と反対に，ハードウェ
アがシンプルになる長所と，キャッシュミス率が高くなる短所をも
つ．また，これらの中間的な方式としてセットアソシアティブ方式
がある．これは，ハードウェアの複雑化を抑えつつキャッシュ領域
の活用度もある程度高めることをねらったものである．以下，セッ
トアソシアティブ方式について詳しく説明する．
セットアソシアティブ方式は，主記憶およびキャッシュメモリの
領域を，セットと呼ぶ複数の部分領域に分割し，主記憶とキャッシ
ュのマッピングを対応するセット内に限定するものである．セット
内には複数のブロックが格納可能で，この数をウェイ数あるいはレ
ベル数と呼ぶ．ダイレクトマップ方式はウェイ数が1のセットアソ
シアティブ方式と考えることもできる．実際のキャッシュメモリで
は，ウェイ数が2～4程度のセットアソシアティブ方式が採られる
ことが多い．図2･10はセットアソシアティブ方式を図示したもの
である．
セットアソシアティブキャッシュにおいて，すべてのウェイにブ
ロックがマップされている状態で，キャッシュ上にない同一セット
内のブロックがアクセスされると，そのブロックのための領域を新
たに確保するため，既存のいずれかのブロックをキャッシュから追

スされてからの経過時間が最も長いブロックを優先的に選ぶLRU
（least recently used）という方式が広く用いられている．
b.仮想記憶 プログラムがアクセス可能な主記憶の大きさは
命令セットアーキテクチャによって規定されており，例えば32ビ
ットアーキテクチャでワード長をフルにアドレスとして使えば，メ
モリ空間の大きさは4GB（ギガバイト）に達する．また，現在の
コンピュータは複数のプログラムが同時に動作できるのが普通で，
1個のプログラム当たり4GBのメモリ空間とすれば，トータルの
空間サイズは膨大なものになる．ところが，物理的に存在するメモ
リは有限であり，すべてをまかなうには明らかに不足する．このよ
うに，実際のメモリサイズをはるかに超える空間サイズを提供する
仕組みが仮想記憶（virtual memory）である．
仮想記憶システムは，プログラムがアクセスする全空間のうち，
ある時点でアクセスする部分のみを実メモリ上に置き，それ以外は
ディスク等の外部記憶に追い出すことで，上記を実現している．ア
クセスが外部記憶に追い出された領域に及ぶと，実メモリ上の適当
な領域を選んで，外部記憶との交換が行われる．外部記憶は一般に
実メモリに比べてはるかに大きな容量を有しているので，実メモリ
サイズを超えた空間が仮想的に実現されることになる．図2･11に
概念図を示す．
仮想記憶は，プログラム上のメモリ空間と実際のメモリ空間を対
応付ける仕組みととらえることができる．前者のアドレスを論理ア
ドレスまたは仮想アドレスと呼び，後者を物理アドレスまたは実ア
ドレスなどと呼ぶ．仮想記憶を実現するためには，双方のアドレス
空間をページという固定長のブロックに分割し，ページ単位で両者
をマッピングする手法が多く用いられている．これをページングと
呼ぶ．同時期に物理メモリ上に存在できるページ数は実メモリの大
きさで制限され，それ以外のページは外部記憶上に置かれることに
なる．ここで論理空間サイズを4GB，物理メモリサイズを1GB，
ページサイズを4kBとすれば，論理空間は1Mページ，物理空間
