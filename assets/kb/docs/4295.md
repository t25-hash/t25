# 3・3・1 実装設計

> **章**: 第3章 ロボット・メカトロニクス要素
> **節**: 3・3 実装技術

実装（implementation）技術とは，要求された機能を満足する
ようにものをまとめる技術である．各技術分野の機能をまとめあげ
る際に，相反する要求があり，種々のトレードオフを技術分野間に
わたり考慮する必要がある．このため，実装設計には広い範囲の知
識が必要となる．特にメカトロシステムでは，機構，電子回路，ソ
フトウェア，制御と広い分野を横断する実装技術が必要であり，そ
の実装技術レベルにより性能，コストに大きな差が生じる．
以下にメカトロシステムの実装設計を行う場合に注意することを
記述する．
a.メカ部と制御部の分担 機構で実現する部分と制御部で実
現する部分の調整が必要である．例えば，コスト低減のためメカ部
の剛性，精度を低下させ，高度な制御アルゴリズム，センサにより
要求される精度を満足させるという手法がとられる．
b.ソフトウェアとハードウェアの分担 半導体技術の進歩に
より，ソフトウェア，ハードウェアの境界を柔軟に設定できるよう
になったが，今後はその傾向がより強まると考えられる．例えば，
CPU，DSPの高速化により従来はハードウェアで実現せざるを得
なかった機能がソフトウェアによって実現可能となる一方で，
FPGA，設計技術等の進歩によりディジタル論理回路がソフトウェ
アのように柔軟に開発できるようになっている．
c .新規開発部の限定 一つの組織でメカトロシステムに関す
るすべての技術を保有することは不可能である．このため，差別化
するべき部分を決定し，その部分に注力するべきである．また，外
部から調達する部分を決め，その部分は特殊なものを避け，規格
化，標準化されているものを使用する．
モジュール構成がとられるが，近年ではモジュールの複雑化に伴
い，各モジュールにCPUを搭載して制御を分散化させることが多
い．また，配線数削減のため，同一ハウジング内においてもモジュ
ール間を通信ラインで接続するという実装が増えてきた．
電磁ノイズ規制，安全規格等の規制を守る必要があり，実装設計時
の重要事項となる．
