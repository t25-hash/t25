# 4・6・1 人とかかわるロボットに関する概念

> **章**: 第4章 ロボティクス
> **節**: 4・6 人間・ロボットインタフェース

a.コミュニケーションロボット（communication robot )
ロボット研究には二つの方向性がある．一つは工場内等の限られた
環境（ロボットが働きやすいように工夫された環境）で特定の作業
だけを行うタスク指向ロボット（特定の仕事のみを行うロボット）
の開発であり，もう一つは，人間の日常生活の場のようなオープン
な環境（特にロボットのために作られた環境ではない）で人間と協
調して動作するコミュニケーション指向のロボット，コミュニケー
ションロボットの開発である．産業用ロボットは前者であり，工場
や家庭内の限られた環境で組立や動物のようなふるまいといった特
定のタスクを行う．一方，コミュニケーション指向のロボットは特
定のタスクを実行するのではなく，パートナーとしてわれわれの日
常生活の中で活動するために人間と対話し，互いの存在を認識し合
う機能が求められる．すなわち，コミュニケーションロボットと
は，コミュニケーション機能を目的として開発される知能ロボット
である．
b.擬人化（anthoropomorphism) 人間に親しみやすい形状
として人とかかわるロボットは，たいてい人間に似た姿形，すなわ
ちヒューマノイド（人間型ロボット）であることが多い．工場で単
一の目的を正確に達成する従来のロボットとは異なり，近年研究が
盛んな人間の日常生活の場で働くロボットの最も重要な目的は，人
間と直接かかわることであり，人間にとって最もかかわりやすい相
手であることである．その理由は，人間が対話する対象を擬人化す
る傾向があることにある．人間は，たとえ動かぬ物体であっても，
対象を擬人化することによって対話することができる．また，人間
の視覚や聴覚などさまざまな感覚はたいてい，他の人間を認識する
ために都合よくできている．すなわち，人間が対話する理想の相手
は人間なのである．ゆえに，人間とのかかわり合いを目的としたロ
ボットを作る場合，そのロボットは必然的にヒューマノイドになる
ことが多い．
c .ロボットメディア（robotic media) 人間とかかわり，人
間と対話するという目的で開発されるロボットは，コンピュータイ
ンタフェースやマルチメディアの研究とのかかわりが深くなる．日
常生活の場で働くロボットは，人間と対話しながら，人間からの要
求を理解し，与えられたタスクを実行するというものである．しか
し，対話そのものもロボットの重要なタスクである．技術として重
要なのは，対話というタスク，すなわち人間から対話によってタス
クを得るという機能の実現である．この機能が実現できれば，たと
えロボットが重い荷物を運ぶことができなくても，たとえロボット
が食器を洗うことができなくても，情報提供などさまざまなサービ
スを行うことができる．
このように考えれば，ロボットを新しいメディアとみなすことが
できる．メディアとは情報の媒体であるが，現在の社会においてそ
の根幹をなすのが，ディスプレイやマウスを用いたコンピュータで
ある．しかしながらこれらは，人間にとって完全なシステムではな
い．より人間が利用しやすいシステムにするためには，多くの開発
課題が残されている．ディスプレイやキーボードなど，それを利用
するために特殊なトレーニングを必要とすることなく，人が人に話
しかけるように情報交換できるメディア，それがメディアとしての
ロボット研究の目的である．
d.環境知能（ambient intelligence) ロボットをメディアと
してとらえると，必然的にロボットの知能に対する見方も変わって
くる．従来の知能ロボットは，与えられた環境の中で，より効率よ
く特定のタスクを達成するものである．例えば溶接ロボットであれ
ば，巧妙にマニピュレータの先端を動かしながら，複雑な形状をも
つ対象に溶接する．腕をぶつけることなくなるべく短い時間で，短
い経路をたどりながら，なるべく正確に溶接する．そのようなよう
すを見て，われわれはロボットの知能を感じていた．
しかしながら，このような知能ロボットの研究においてロボット
を知的と感じているのは，開発者本人のみである場合が多い．例え
ば，障害物回避をするロボットを見て，一般の人はそのロボットの
行動を知的と感じるだろうか．そこに複雑なメカニズムを埋め込ん
だ開発者自身は，そのメカニズムがどのように障害物回避に反映さ
れているかを知っており，それゆえ複雑なメカニズムの効果を単純
なタスクを通して観察することもできる．すなわちロボットの知能
とは，ロボットの内部メカニズムにあるというよりは，ロボットと
ユーザとの間の相互作用に宿る主観的現象である．これら両者を区
別するために，前者を個体知能，後者を環境知能と呼ぶ．個体知能
はロボットの内部メカニズムこそが知能のメカニズムであるという
見方であるのに対し，環境知能は，ロボットと人間の相互作用（そ
の手段は問わない）に主観的現象として現れるのが知能であるとい
う見方である．もっとも，個体知能と環境知能を無関係に考えるこ
とはできない．先にも述べたように，相互作用から知能が発現する
という見方においては，個体知能と環境知能は密接にかかわるもの
であり，両者の関係そのものが重要であることは言うまでもない．
e .分散認知（distributed cognition) 人工知能や知能ロボッ
トの研究では，1台のコンピュータや1台のロボットが限られた環
境で動作する際の知能，言い換えれば行為主体（ロボットや人間）
に閉じた知能を実現する機能を研究してきた．一方で，心理学や社
会学など人間を対象にしてきた分野では，古くから知的な活動にお
ける環境の重要性が議論されており，そこでは知能は個体に宿ると
いうより，個体間や個体と環境との相互作用として発現するものと
考えられている．
このような個体間や個体と環境との相互作用という視点は，近年
の人工知能や知能ロボットにおいても取り上げられるようになって
きた．人工知能研究での発端は，マルチエージェントシステムの研
究（複数の自律的に行動するプログラムが協調して作業を行う研
究）であり，知能ロボット研究では群ロボットの研究（多数のロボ
ットが協調して作業を行う研究）である．これらは個体間の関係の
重要性を再認識させるものであった．一方，環境との相互作用につ
いて，環境からの刺激に対して反射的に対応する要素行動に基づく
アーキテクチャ（behavior -based architecture）や，同様の問題
を概念的に説明するアフォーダンスの考えは，まさに知能の所在を
環境との相互作用に見い出したものである．
ここで個体が相互作用する対象について改めて考えると，その対
象，すなわち環境にも記憶や学習のメカニズムがあるとしても不自
然ではない．このような考えに立つのが分散認知である．例えば，
人は次の日にすべきことを忘れないように冷蔵庫の扉にメモを張
り，いったんそのことを忘れても翌朝，冷蔵庫から飲み物を取出す
ときにそのメモに気づき，その日すべきことを思い出す．これは相
互作用を通して環境に記憶の機能をもたせていることにほかならな
い．このような分散認知の考え方は，日常的認知（everyday cog-
nition）と呼ぶ枠組においても議論されるとともに環境知能の考え
方の源にもなっている．
対象が冷蔵庫であるとあまり知的な感じを受けないが，コンピュ
ータ技術の発達を思い返してみるとよいだろう．今日の高度情報化
社会があるのは，ある程度人間の知能が発達したからであるともい
えるが，それ以上の勢いで進んでいるのは，コンピュータやコンピ
ュータネットワークなどの人間を取り巻く環境である．コンピュー
タの情報記憶・処理能力はまさに知能といえるレベルに到達してい
る．このように環境が特にコンピュータ技術によって高度に知能化
されていく例は，ほかにも数多くある．例えば，ITS（intelligent
transport systems ）はその一つの例である．道路は最初，人間が
運転する自動車がただ通るためだけのものであった．道路網が複雑
になると，より効率的に複数の自動車を通行させるためにさまざま
な道路標識や信号が設置された．さらに近年では，道路状況などを
道路に沿って設置されたさまざまなセンサが検知し，運転者に能動
的に情報を伝えるITSの研究開発が進んでいる．このようなITS
の運転者は道路標識や信号に従うだけでよく，言い換えれば運転者
は環境によって知的に制御されている．
