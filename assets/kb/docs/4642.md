# 4・7・2 モデルベースド制御（model -based control )

> **章**: 第4章 ロボティクス
> **節**: 4・7 知能・システム技術

ロボットは，人間の意図を受けて忠実にその命令を実行し目標を
達成する機械である．このような観点に立つと，ロボットが直接の
作業対象を含めその環境に関する知識（モデル）をもっていると都
合がよい．人間は直感的にわかりやすい言語表現でその意図を伝え
ることができ，究極的には作業対象とその目標状態を指定すればよ
くなる．ロボットは，与えられた目標を達成するための行動を計画
し，必要であればそれをアニメーションなど人間に理解しやすい形
で提示できる．計画された行動に誤りがあれば，人間は修正の指示
ができる．修正を加えるなどして承認が得られた行動計画を規範と
して，最終的には4･7･1項で述べた感覚フィードバック制御により
作業を実行する．実行段階では，モデルに基づいてセンサ情報を解
釈することにより，作業エラーの検出やリカバリを試みることもで
きる．モデルベースドなアプローチはこのようにロボットの運用に
至る各段階で有益と考えられ，研究が進められてきた．
このアプローチでは，ロボット自体と作業対象の形状や表面光学
特性，材質や質量など属性情報を記述したモデルと，対象に対する
操作手順や動作決定アルゴリズムとを分離して，それぞれ一般性の

a.物体認識 二次元濃淡画像や，奥行きを含めた三次元距離
画像を用いて視野内に存在する物体を認識するには，入力データか
ら顕著な特徴を抽出し，その特徴をもつ形状モデルを探し出して物
体候補とする手続き（ボトムアッププロセス）と，候補物体の形状
モデルに基づいて，観測されるであろう顕著な特徴を抽出し，それ
が実際に画像中に存在するかどうかを検証する手続き（トップダウ
ンプロセス）とを相補的に用いる．
三次元物体は光源配置や視点により見え方が異なる（図4･149）．
同一シーン内に複数の物体があると相互に隠ぺいが起こり，さらに
見え方が変化する（図4･150）．存在可能性のある物体の数が多く
なると，それらの特徴や見え方の組合せが爆発的に増加する．さら
に，ノイズによる不確定性も加わるので一般的な認識は困難にな
る．
しかし，環境内に存在する物体についての事前情報が得られ，そ

b.環境モデリング 三次元作業環境において，存在する個々
の物体の認識は一般的には困難であるが，空間との境界である物体
表面の構造を知るだけであれば，レーザレンジファインダやステレ
オビジョンにより表面形状を計測できるようになった．移動ロボッ
トのための地図の作製や，マニピュレータが動きうる空間の記述が
可能になるので，以下に述べる動作計画が現実的になり，環境モデ
ル（environment model ）の構築から自律動作までの一貫した作業
が射程に入ってきている．
c .動作計画 二次元平面内を移動するロボットであれば，障
害物や壁の配置を記述した二次元マップを用いて，また三次元空間
で作業する多自由度マニピュレータであれば，環境内の三次元物体
の形状および配置を記述した環境モデルに基づいて，目標作業を遂
行する障害物回避経路を計画できる ．二次元平面内移動ロ
ボットの動作は，位置と姿勢を規定する3変数を軸とする三次元空
間で規定できる．また三次元空間で動作する多関節マニピュレータ
の動作は，その関節軸数の次元をもつ多次元空間で一意に規定でき
る．このように対象の状態を一意に規定できる空間をコンフィギュ
レーション空間（configuration space，C空間）と呼ぶ ．動作
計画はこの空間内での経路探索問題となる．特に，空間の次元が高
くなる後者は計算量爆発のため全探索はできず，実用的な時間で経
路を得る工夫が必要となる．
.セル分割 多次元C空間では，障害物と自由空間との
境界は複雑な曲面となり，これを境界とする空間を正確に記述する

もの（自由空間セル）に分けることにより空間形状を近似的に表現
する．任意の2点が与えられたとき，それぞれが属するセルから隣

.ロードマップ法 多次元C空間において体積をもった
連続空間としてロボットの自由空間を表現する代わりに，一次元の
曲線を連結したグラフにより，自由空間のトポロジカルな情報を保
持することができる．これをロードマップと呼ぶ ．任意の2点
が与えられたとき，それぞれの点から近傍のロードマップまでの障
害物回避経路を生成できれば，ロードマップ内でそれら接続点を結
ぶ経路を探索することにより，2点を結ぶ障害物回避経路を生成で
きる．
ボロノイ図（Voronoi diagram）はロードマップとして用いる

くなってくる．しかし，近傍の障害物からの距離を厳密に評価した

d.マニピュレーション計画 個々のロボットがとるべき位
置・姿勢の系列として作業を規定する代わりに，作業対象そのもの
の状態変化の系列として作業を規定するほうが，人間にとっては直
感的で記述量も削減でき，負担が少ない．マニピュレーション計画
は，作業対象の初期状態と目標状態とからロボットの実際の動作を
計画するものである．
作業対象の初期状態からその把握位置を計画し，必要ならば持替
えの計画や，作業スペースを作り出すための他物体の移動の計画を

それらの計画結果を環境内の障害物を回避しつつ一貫した作業シー
ケンスとしてロボットが実行できるようにC空間での動作計画を
行う必要がある．しかし，すべてを考慮して同時に計画することは
できないので，以降の計画が成功しそうな解を各段階で求めてさら
に計画を進め，失敗したら前段階の計画に戻って別の解を試みると
