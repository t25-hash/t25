# 4・8・1 安全に対する考え方

> **章**: 第4章 ロボティクス
> **節**: 4・8 ロボットの安全

安全は，許容できないリスクが存在しないことと概念的に定める
考え方が国際的に広く受け入れられ，国際規格［例えば文献(303)］
としてもうたわれている．したがって，対象としている状況や系が
安全であることを確かめるためには，リスクを一つ一つ洗い出し，
これが許容できるかどうかを確かめる必要がある．この確かめるた
めのプロセスはリスクアセスメントとして，これもまたロボットの
みならず機械安全分野全般にわたってすでに広く受容され，技術的
により詳細なレベルまで国際規格により規定されている ．以下，
これに沿ってリスクアセスメントの概要を説明する．
a.リスクアセスメント（risk assessment ) リスクアセスメ
ントの基本的なプロセスは，図4･173中右側に示すように以下の要
素で構成される．
(１)使用環境（条件）の整備
(２)危険源の同定
(３)リスクの見積り
(４)リスクの評価
(１)の使用条件の整備は，ロボットの使用される状況を特定する
ためのプロセスで，ロボットシステムの構成，機能，仕様，さらに
使用環境を考慮して定める．次に，(１)の使用状況に沿って危害の
潜在的な源と定義される(２)の危険源の同定，例えば文献(304)の
付録Aの表A. 1に掲げられているリスト中から該当する危険源を
抽出する作業が行われる．危険源を同定するプロセスは，その具体
的な作業である事故シナリオ想起の精緻さが，その後のアセスメン
トプロセスからリスク低減手法の決定まですべてのプロセスの枠組
を決定付けるため，極めて重要なプロセスである．危険源は
source of hazardsの訳語であるが，hazardsを，起こりうる事故
と定義して，その発生条件を明示する立場もある ．(３)のリス
クの見積りは，リスクの定義に依存してフォーマットが変わりうる
ものであるが，基本的には人間が危険源から被る可能性のある危害
の程度とその危害の発生確率の組合せによって表現され，さらに危
害の発生確率は，人間が危険源にばく露される頻度（あるいは時
間），危険事象の発生確率，人間が危険事象を回避する可能性の三
つの要素の関数となる．最後に，(４)のリスクの評価は，その後に
ひかえるプロセスとしてリスク低減措置を必要とするか，あるいは
すでに安全性が達成されたかを決定するプロセスである．このため
に，リスクの許容値を定め，他の類似機器の場合との比較も必要に
応じて行いながら，個々のリスクに見積もられたリスク値が許容値
か否かを網羅的に判断し，対象機器システムに対して適正なリスク
のレベルを総合的に定める．
図4･173 リスクアセスメント
b.リスク低減 リスク評価の結果，リスク低減措置が必要と
判断されたら，図4･173中左側に位置付けられる保護方策プロセス
によってリスク低減措置を講じる．このプロセスでは，リスク低減
が必要と評価されたそれぞれのリスクに対して，3ステップ法と呼
ばれる3段階からなる措置を講じる．それらはすなわち，(１)本質
安全設計，(２)安全防護策・付加保護方策の安全方策をこの順番で
講じ，さらに(３)残留リスク情報のユーザへの提供を行う．
本質安全設計は，3ステップ法の中では最優先で講じられるべき
手段であり，より危険の少ない材料への代替等，設計を工夫するこ
とによって，安全防護物など付加的な設備の設置を行うことなく安
全性の向上を図ることである．安全防護策とは，本質安全設計によ
ってだけでは許容できないリスクが残ってしまう場合に，さらに機
構や制御によって安全性向上を図ろうとするもので，リスクのレベ
ルに応じて適切な技術的方策を選定することになる．
近年，機能安全（functional safety）と呼ばれる安全防護策段階
の設計概念がヨーロッパから提言され国際規格化されて ，これ
を反映した安全技術が急速に普及されようとしている．これは，ラ
ンダム故障と系統的故障のいずれの故障によってもハザードをきた
す機器として，電気/電子/プログラマブル電子機器［electrical /
electronics / programmable electronics （E/E/PE）機器］および
ソフトウェアを広くシステマティックに対象として，それらのライ
フサイクルにわたる管理，技術要件に加えて，さらにこれらシステ
ムにかかわる人から組織までの責任要件をまとめたものである．同
規格Part3あるいはさらにPart6には，ハードウェア・ソフトウ
ェア統合システムのライフサイクルにソフトウェアの開発を組込む
目的で定めたソフトウェア要求事項およびその適用例が示されてお
り，知的なふるまいのためにソフトウェア開発の比重がますます高
くなるロボットの安全にとって，今後必要欠くべからざる技術要件
となろう．
産業機械や製造システムの安全技術としては，安全状態＝エネル
ギーが零の状態の考え方に従って，機械システムへの電力供給源に
直接かかわる安全関連系（safety-related system）の安全性能の
妥当性が，近年SIL（Safety Integrity Level ，安全インテグリテ
ィレベル）等によって評価されるようになっている．SILは，E/
E/ PE機器の安全関連系において，ハードウェアの危険側故障の検
出ができない確率をどれだけ抑えられるか，また，ソフトウェアで
安全な状態間の遷移がどの程度保証できるかを統一的の評価するこ
とができる指標となっている．
