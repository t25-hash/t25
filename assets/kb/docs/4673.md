# 6・3・1 コンピュータへの応用

> **章**: 第6章 熱輸送デバイス
> **節**: 6・3 ヒートパイプの応用

a. ノートブックPCの冷却近年、パーソナルコンピュータ(PC)用マイクロプロセッサ(MPU)の高性能化は著しく、それに伴いMPU の発熱量も指数関数的増加を示しつつあり、PCの熱設計は、重要な設計要素となってきた。特にノートブック型PCでは、そのスペースの制限により、大変細やかな設計が必要である。図6・18は、近年のMPU 発熱量の増加状態を示す。1993年のインテル社のペンティアムプロセッサ윣윃の登場を契機に、ヒート3―9 8熱機器パイプ(heat pipe)の実機搭載は進み、現在では、90%のノートブックPCにヒートパイプが使用されていると言っても過言ではない。1993~1995年の間は、ほとんどのMPU の発熱量が10W 以下と少なかったため、直径3mmまたは4mmのヒートパイプ1本でも十分であったが、最近は発熱量の増加により、4mmのヒートパイプを2本使用したり、6mmのヒートパイプが採用され始めている。図6・19に、MPU の熱をキーボード表面から周囲に放熱するシステムを示す。MPU の熱は、ヒートパイプによりアルミニウム等の放熱板に熱拡散され、キーボードを介して周囲に放熱される。吸熱ブロックと周囲温度間の熱抵抗は、5~8°C/W 程度であり、キーボードの大きさと許容温度の制限により、約10W の放熱が限界である。一方シャシから周囲放熱することも可能であるが、対流熱伝達は上から下には伝えにくいため、7W 程度が限界である。近年では、プラスチックの筐体に代わり、マグネシウムやチタンの軽量金属を採用し、放熱対策を施しているものもある。図6・20に、ヒートパイプを組込んだ放熱部品の実用例を示す。図6・21に、ヒートパイプ式RHE(remote heat exchanger)を示す。MPU の熱は、銅ブロックを介してヒートパイプに伝わり、筐体の端部の放熱しやすい箇所にまで伝熱され、ヒートシンクを介して周囲に放熱される。放熱効率を高めるために、ファンが取付けられ、周囲より吸引された低温度の空気により、冷却しそのまま排気する。筐体内部の空気をファンが吸引すると、他の熱源で加熱された空気でヒートシンクを冷却することになり、冷却効率が低減することがあるので、筐体内部の空気の流れを併せて解析することが必要である。図6・19キーボードより放熱する構造図6・20 デスクトップ、サーバへの応用ノートPCのCPU とデスクトップやサーバのCPU は、構造や実装方法が異なるとともに、PCの本体の構造も異なっているために、ヒートシンクに要求される放熱量や形態も異なっている。デスクトップやサーバのCPU からの発熱量は、数十W から100W を超えるものまである。特にサーバの場合は、1台に複数のCPU を搭載していることも多いため、全体の発熱量はかなり多いものとなっている。そのため、デスクトップやサーバのCPU レベルの発熱量になると、強制空冷による冷却が基本となる。強制空冷はCPU が置かれている筐体内の温度が所定温度を超えないように、外気導入するためのファンと、CPU 冷却用のものを、兼用あるいはそれぞれ独立して使用することで行われる。. カートリッジ(cartridge)素子対応型CPU は多くの場合一つのチップにより形成されるが、いくつかの素子を基板上に搭載して、それを一つの素子(ユニット)とするものがある。その場合、ユニット自体が大型化するとともに冷却方法も異なってくる。すなわち、熱源となるチップ自体の大きさは、同一性能のCPU に対しては小さくなる傾向にあるため、チップの熱をうまくユニット全体に広げ、かつ効率良く放熱することが重要となってくる。図6・24にヒートシンクの構造例を示す。図中左部のブロック部図6・22 ヒンジヒートパイプによる冷却構造図6・23 各種形式の熱抵抗比較分を素子に取付け、ヒートパイプにより吸熱し、フィン側へ熱を移動させてファンで冷却することにより放熱を行っている。集熱部分であるブロック部の厚さは薄く、CPU に取付けても厚さを所定値に納めることが可能であるため、複数の素子を実装する際に実装ピッチを狭めることが可能となる。また、風を流す部分を分離しているためにファンの風を効果的に流すことが可能であり、かつ伝熱面積の変更が容易であるため、設計の自由度も大きいものとなっている。ここでいう設計の自由度とは、フィン側の構造に関するもので、本ヒートシンクで用いられるフィンの形状がCPU の設置レイアウトによって幅や高さを変えたり、またそれに伴ってフィンの枚数を変えるなどの変更が容易なことを意味している。.
