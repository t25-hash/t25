# 4・4・2 セントラルモニタ

> **章**: 第4章 検査・診断機器
> **節**: 4・4 生体情報モニタ

日本においてICU（intensive care unit ，集中治療室）が開設さ
れるようになってから約30年が経過し，多数の総合病院でICUを
もつまでに定着した．その間，ICUの位置付けとしての考え方も
少しずつ変化してきた．当初は，大きな外科的侵襲を受けた患者の
術後管理を行う回復室であったが，救急医療システムの中で内科
系・外科系を含めた総合的なICU（general ICU）へと発展した．
次にこれとは別に，特定分野の患者を対象にしたICU（special
ICU）へと分化してきた．これには心筋梗塞をはじめとする心疾患
を扱うCCU（coronary care unit ）が代表的であるが，このほかに
呼吸疾患を扱うRCU（respiratory care unit ），脳外科における
NCU（neurosurgical care unit ），新生児を対象にしたNICU
（neonatal intensive care unit ）などがある．また，最近では複合
臓器不全の総合的な治療・看護の立場から，general ICU的な考え
方が見直され，患者の重症度に応じて必要なときに必要な看護を提
供できるPPC（progressive patient care）の考え方が定着してき
ている．
一般にICUでは，ベッドサイドに一人用生体情報モニタを置き，
ナースステーションにセントラルモニタ（central monitor ）を置
くことにより，複数患者の同時監視を行っている．
一方，CCUで扱う心疾患の中でも急性心筋梗塞は発生直後に最
大の危機が到来し，発症後2時間以内に全患者の約50 %が死亡す
るといわれるが，初期死亡のおもな原因は心室細動や心臓ブロック
などによる不整脈である．したがって，虚血性心疾患においてはそ
の早期発見，早期治療が極めて重要である．虚血性心疾患を発見
し，その進行状況を判断するには，長時間にわたる心機能の監視が
必要であり，CCUでは心電図の連続監視体制が敷かれている．
CCUにおける患者監視は，急性期の患者を主体にしたものと急性
期を過ぎた患者を主体にしたものの二つが考えられる．急性期にお
ける患者監視は多くの場合，ベッドサイドモニタによって行われ，
装置の構成はICUと同様になる．CCUの患者は一般に意識が明り
ょうであるから，個室が多く用いられる．この場合，ある部屋のベ
ッドサイドモニタから他の部屋の患者（ベッドサイドモニタ）の状
態（情報）を呼び出して見ることのできるインタベッド通信機能は
有用である．急性期を過ぎた患者の監視はおもに患者の容態の急変
に備えることに主眼がおかれる．患者は自由に行動できるため，監
視の主体はベッドサイドモニタからセントラルモニタに移る．セン
トラルモニタはおもにナースステーションに設置され，多人数の心
電図情報を素早く得る必要があることから，装置の操作が簡単であ
りコンパクトなものが要求される．
CCU用セントラルモニタとしては，4人用の無線テレメータ方
式（図4･19）を採用したものがある．通常は4人用の心電図，呼
吸曲線などと心拍数を表示し，スイッチを操作して画面を切換える
と，過去数時間分の心拍数変化，すなわち心拍トレンドグラフを表
示するものである．心電図以外のパラメータとしてSpO ，非観血
血圧，呼吸数，体温の数値情報を表示することもできる．また，い
ずれかの患者の状態が設定された限界値を超え危険となったとき
に，警報を発してナースなど医療スタッフに対して緊急事態発生を
伝えたり，心室性期外収縮の発生を知らせる不整脈警報機能もあ
る．最近ではスタッフの労力を軽減するために，さまざまな機能が
追加されている．例えば，患者に投薬などの治療を行った場合な
ど，不整脈の発生頻度の変化，心拍リズムの変化を連続して観察す
る必要があるときには，圧縮心電図表示が用いられる．また，圧縮
心電図から，離散的に現れる不整脈やリズムの変化を一目で読み取
ることもできる．そのほか，ST解析機能，不整脈リコール，トレ
ンドグラフ表示，バイタルサインデータリスト表示が可能である．
一方，患者は携帯型の送信機（図4･20）をもち，心電図・呼吸
数の情報は無線信号でモニタに伝送される．電源は単3乾電池1本
で8日間連続して伝送することができ，重量は85g（電池，電極
リード線を含まず）と軽量であるので，患者の動作にも負担をかけ
