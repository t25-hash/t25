# 5・3・5 携帯端末とJavaの標準化動向

> **章**: 第5章 情報通信機器
> **節**: 5・3 モバイル情報機器（PDA）

現在，PDA分野の主要なOSとしては，PalmOS，Windows-
CE，EPOC，ザウルスOSなど，特定企業あるいは企業連合によ
って定められたOSが利用されている．
従来の携帯端末ではビルトインのアプリケーションを中心に構成
され，限られたカスタマイズ手段しかなかったが，さまざまなユー
ザニーズの拡大，利用シーンの分化により，PDAの世界にもオー
プン化の波が押し寄せている．機器メーカ自身ですべてのユーザニ
ーズに応えることはもはや不可能になってきており，サードパーテ
ィやインハウスの開発者によるアプリケーション開発の重要性が高
まっている．そのためには開発ツール，開発環境の提供と技術情報
の公開が必須となりつつある．
C言語等から利用する固有のライブラリが機器やOSごとに提供
されている場合は，各個の技術の習熟に時間を要すること，機器ご
とにアプリケーションを移植する必要があることから，クロスプラ
ットホーム性の高い技術，特にJava（ジャバ) に対する関心が
高まっている．
デスクトップやサーバの世界では実用化の進んでいるJavaだ
が，携帯電話や携帯端末をはじめとする組込み分野においても，
J2ME（Java2platform micro edition）として仕様の策定が進め
られている．J2MEは，VM（virtual machine，バーチャルマシ
ン）などプラットホームの基本部分を規定するコンフィギュレーシ
ョンと呼ばれる部分と，その上でバーティカル市場や製品カテゴリ
ごとに定義されるプロファイルと呼ばれるAPI群からなる．コン
フィギュレーションとしては，CLDC（connected limited device

configuration）と，それよりもリッチな機能を提供するCDC
（connected device configuration）の二つが定義されている．
CLDCは，主として携帯電話など，極めて限られたハードウェ
アリソースしかもたない機器を対象とするもので，Javaの言語仕

information device profile）やローエンドPDAを対象とした
PDA profile（仕様策定作業中）がある．また，NTTドコモは，
Javaコミュニティプロセス（JCP）の外でMIDPとは別の独自の
プロファイルを定義している．
Java相当の機能をサポートする機器を対象としているが，機器の
カテゴリによってはGUI （graphical user interface）を必要とし
ないものや分散オブジェクト機能を必要としないものもあり，基本
部分のAPI （application programming interface）群となる

一方，CDCは，Javaアプレットを実行できる従来のPersonal-
profile，RMI （remote method invocation）機能を提
供するRMI profile，PersonalJava相当のGUIを提供するPer-
sonal profile（仕様策定中）がある．こちらは，CLDC上のプロフ

これらJava関連の仕様は，Javaコミュニティプロセスという

家グループによる開発作業，仕様の公開レビューなどの手続きをル
ール化し，外部の監査機関によってこのルールにのっとって仕様化
作業が進められているかどうかがチェックされている．
