# 6・5・1 インテリジェント（ロボティック）ルーム

> **章**: 第6章 メカトロニクスの応用展開
> **節**: 6・5 ネットワークメカトロニクス

a.定義 ロボットをわれわれの生活空間に導入する目的は，
端的にいえば日常活動の支援である．その場合，ロボットに期待さ
れる支援の内容はさまざまである．日常使っている物品や書類など
の配達といった典型的な運搬作業はもちろん，家電（家庭用電気製
品）や住宅設備の操作，さらにインタネットが当たり前に利用され
るようになっている現在では，情報伝達のようなこともロボットの
役割の一つと考えられるようになっている．
これらの支援機能は，機能だけを取出してみれば必ずしもロボッ
トの形に依存しない．いわゆるヒューマノイドのような人間型であ
る必要はなく，介助犬のような動物型でもかまわないし，だいたい
手，足，眼，頭脳などのすべてを一つにまとめ上げた一体型，スタ
ンドアローン型である必要もない．眼の機能を取出してみれば壁や
天井に設置されたカメラ，その視覚情報処理を行うのは別の場所に
設置された計算機，情報処理の結果に応じて物を運ぶのは天井のレ
ールを走行するマニピュレータといった具合でもよい．このような
考え方から生まれたのが，部屋や環境の中にロボットの機能要素を
分散配置して，それらの協調的作業によって室内にいる人間が求め
る仕事を行い，支援を行ってくれるという空間型のロボットであ
る．これはインテリジェントルーム（intelligent room），あるいは
ロボティックルーム（robotic room）と呼ばれることが多いが，
その呼称についてはいまだ一定するものはない．国内ではロボティ
ックルーム，インテリジェントルームなどの呼称が多く，外国では
スマートルーム（smart room）と呼ばれることもある．
図6･129に一体型のロボットと空間型のロボットを模式的に比較
したものを示す．図6･129(a)は一体型ロボットのイメージで，ロ
ボットに想定されるすべての機能要素を一つの 体に積み込んでい
る．図6･129(b)は空間型ロボットのイメージで，ロボットの機能
要素を部屋の中に分散的に配置し，ネットワークで統合している．
空間型に分類されるインテリジェントルームやロボティックルーム
を構成する要素は，モータやセンサなど基本的にロボットを構成す
るさまざまな機能要素と同じものであるが，それらを空間的に分散
配置したものが基本形である．もちろん，機能要素を単にばらばら
(ｂ)
図6･ 129 一体型ロボットと空間型ロボットのイメージ比較
においただけでは意味がなく，分散配置された機能要素がお互いの
役割を適切に果たし，統合的な効果を発揮することで初めてこの空
間がロボットとみなせるようになる．すなわちロボティックルーム
の実現にはこれらロボット的機能要素を統合するための手段も重要
となる．一体型ロボットの場合，図6･129(a)のように，要素の統
合はロボットの 体という単なる入れ物への実装にすぎず，当該機
能要素が 体のほとんどを占める．また，ひとたび実装された機能
要素を交換することは，故障による以外はほとんどない．一方，ロ
ボティックルームの場合，図6･129(b)のように埋め込む機能要素
の総量に対し，空間の広さは圧倒的に広い．また，埋め込む機能要
素もさまざまであり，追加・変更もたびたび行われる．すなわち，
ロボティックルームの場合，従来の一体型ロボットと実装手法はま
ったく異なったものになる．具体的には，空間の自由な位置に機能
要素を配置できるようになったが，その一方で，それらの間の密な
連携が達成できるような仕組みも必要であり，これには最近進歩の
目覚しいコンピュータネットワークが利用されるようになってき
た．また，部屋の中にいる人にどのようなサービスを提供するかと
いうソフトウェア，コンテンツも重要な構成要素である．

c.RTミドルウェアを用いたシステム 佐藤・森らのロボテ
ィックルームの提案以降，ロボット的な要素機器を実際の環境の各
部に分散して埋め込むことで空間自体に機能をもたせようとする研
究はさまざまな形で展開されてきた．一方で，それらの構成手法に
ついては特別な方法論がなく，ロボット的な要素機器をアドホック
に結合して構築することが多かった．
ロボット機能要素のモジュール化手法に関しては，森・佐藤らは
分散オブジェクト（distributed object ）を導入している ．さら
に産業技術総合研究所では，2002年度から2004年度に行われた
NEDO（New Energy and Industrial Technology Development
Organization，新エネルギー・産業技術総合開発機構）のロボッ
トの開発基盤となるソフトウェア上の基盤整備プロジェクトにおい
て，ロボット的な要素機能を標準的な切り口を備えたモジュールと
して整え，システム統合を柔軟かつ迅速に行えるようにするための
RTミドルウェア（robot technology middleware）の研究開発を
行った ．
RTミドルウェア自体はソフトウェア工学における分散オブジェ
クト指向システムのミドルウェアであるCORBA（common
object request broker architecture）をベースとして，ロボットを
構成する機能要素をソフトウェア的にモジュール化するためのツー
ルとして開発されたものである．要素モジュールを分散オブジェク
ト化することのメリットはいくつかあるが，その一つとして，ネッ
トワークに接続するソフトウェアモジュールを，ネットワークの物
理的な構造を考慮することなく機能的な接続を簡単に記述できるこ
とがある．そのため，ロボティックルームのように，空間のさまざ
まな場所に機能要素モジュールを配置し，しかもその変更を随時行
うような状況に適している．

RTコンポーネントと呼ばれ，その基本構造は図6･132のようにな
っている．コンポーネントの作動に直接かかわるデータの入出力チ

びコンポーネントそのものの制御などを行うためのコマンドを受け
るインタフェース部とで構成される．例えばサーボモータであれ
