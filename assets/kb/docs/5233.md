# 1・1・7 ロボットとサービス機能

> **章**: 第1章 総論
> **節**: 1・1 ロボティクス

ロボットは1・1・5項で述べた各種技術を搭載して構成されるが、ここで構成要素として研究され実現されるものは、一般にロボットが必要とする基本機能である。例えばロボットの腕を構築する場合、腕の関節をいかに正確に位置決め制御するか、また複数の関節を備えたリンク状の構造体である腕の先端をいかに目標軌道に沿っ(b) 自由型ロボット(例:クローラ+アーム、車輪+視覚センサ、脚+アームなど)(a) 形にこだわるロボット(例:人型、昆虫型、蛇型ロボットなど)(c) 部屋(空間)型ロボット[例:電動化されたドアや窓、家電などがホームLAN(local area network)で接続・統合制御されてサービスを提供するシステム]に応じてロボットを開発することである。図1・3 ロボットの機能と形て正確に位置決め制御するか、さらに移動機能においては、歩行ロボットの脚をどのように協調制御すれば倒れず安定に移動させることができるか、障害物を回避して目標軌道に沿って移動体を動かすためにはどのように移動体を制御すればよいか等々、ロボットの基本機能動作を実現するために必要な理論が検討される。これが、いわば従来から行われてきたロボティクスの学問の具体的内容であり、本書の4章でこうした学問の成果を体系的に解説している。しかし、ここで、社会のニーズに応えるロボットを開発するにはそれで十分かということについて考えてみる必要がある。ロボットへの期待は、病弱高齢者の介護、日常生活における高齢者支援、障害者の機能補助、災害時のレスキュー、工場の自動化、危険作業代替などさまざまであるが、これらを実現するには、単に正確に動ける、障害物を回避できるといった機能だけでは不十分で、それを必要とするユーザにとって役立つ機能をさらに備えていなければならない。近年、人型ロボットの技術進歩は著しい。安定して歩く技術はかなりの完成度で実現化しており、最近では駆け足する機能をはじめ、倒れても起き上がる機能、全身でものを抱え操作する機能など、その達成機能のレベルは日進月歩の状況にある。しかし、人型ロボットもこうした基本機能だけでは社会の役に立つ存在とはならない。ユーザは歩くロボットや走るロボットを見て楽しむことが目的ではない。ユーザが欲していることは、例えば家庭では介護、留守番、家事の支援、清掃等々、ビジネスの場では案内、警備等々、工場内ではワーク(作業対象)の搬送、溶接、塗装、検査等々といった具体的な作業である。もし、ロボットが社会に役立つとすればこうした人に提供する作業を通してであり、またロボットが経済効果を生む商品として開発された場合にユーザがお金を払うのは、ユーザに提供されるこうした作業に対してである。ここでは、ユーザがしてほしい機能をロボットが提供するとき、そのしてほしい作業の内容をサービスと呼ぶことにする。ロボットが社会で役立つためには、ユーザの要求に合うサービスをいかに的確に見つけ、それをロボットの基本機能の技術を背景にいかに実装するかが重要である。ユーザの要求に合うサービスをロボットに実装する場合の技術的課題は、技術的に達成でき、かつユーザが満足するサービスを見い出すことである。ユーザはさまざまな問題を抱えており、大衆はSFの影響もありロボットに対して多様かつ過大な要求を抱くことが少なくない。こうした要求の中から商品価値のある、すなわちユーザがそれにならば資金を投資してもよいというサービスを見つけることが重要である。こうしたサービスは、自動車や半導体など大規模市場が確立した産業では比較的明確である。ユーザは自動車とそこに搭載された技術が自分にとってどんなサービスを提供してくれるものであるかをすでに熟知しているからである。したがって、商品を開発し提供する側も、ユーザの認識を前提に、さらにどのような技術を開発しサービスを追加すればよりいっそうユーザの関心を引き、市場を拡大できるかの見通しが立てやすい。しかし、ロボットにおける現状は、工場の自動化に使われている産業用ロボット以外に一定規模の市場に展開した、あるいは展開している商品は、ソニー(株)のエンターテイメントロボットのアイボ(2006年に生産中止)や、アメリカのアイロボット(iRobot)社によるロボット掃除機ルンバ(Roomba)程度と極めて少なく、ユーザがお金を出してまで欲しがるサービスは何であるかを特定することが困難な状況にある。この課題に答える学術的手法は現時点では未開発であり、一般に技術者の経験に基づいて問題解決が図られることが多い。こうした中で、ユーザの要求に応じたサービス機能をもつロボットを開発するためにロボット技術で行われてきた一つの方法は、ユーザの注文一般にわれわれは商品としてのロボットを白物家電のように大量生産品として販売・購入することをイメージしがちであるが、現状において特に産業の現場で使われている多くのロボットはこうした白物家電のように販売店に陳列されたものを購入する形式ではなく、注文生産で作られている。例えば、日本航空(株)やルフトハンザドイツ航空などの航空会社は、ジャンボジェット機のボデーを清掃するロボットを開発して使用しているが、これらのロボットはまずユーザである航空会社がロボットに期待するサービスの仕様を定め、それをロボットの製造メーカに持ち込み、ロボットの専門家と航空会社の技術者が一致協力して開発したものである。そのほか、スペースシャトルに搭載されている宇宙用マニピュレータ、火星探査ロボット、本書の6・1・3項(2)で解説する活線作業ロボットなども注文生産品である。日本やアメリカ、ドイツなど世界のロボット先進国といわれる国には、こうした注文生産品のロボットが個々の機種当たりの数は少ないが多数存在する。注文生産ベースによる製品開発はユーザからの要求に応じて開発がスタートするため、ユーザの要求に確実にマッチしたロボットの開発が保証される。この意味で、自動車産業のような大市場に展開する先行商品のない分野での確実な商品開発のモデルとなっている。しかし、一般に注文生産品は数量効果が期待できないためコスト高となり、現状では一般大衆を対象とした商品の開発には向いていない。そこで、多様なユーザの要求にマッチしたロボットを安価にかつ迅速に提供する仕組みとして、ユーザの要求に応じて、ハードウェア、ソフトウェアのコンポーネントやロボットどうしを独自に組み合わせることで新しいロボットが構築できるように、アクチュエータ(actuator)やセンサ(sensor)などロボットに使われる要素機器や、またマクロに見ればマニピュレータ(腕)、移動装置(脚)、ハンド、センサ、ヒューマンインタフェース、基本動作ソフトウェア、アプリケーションソフトウェアというようなハードウェアおよびソフトウェアのコンポーネントをオープンモジュールとして市場に提供する技術が検討されている。これらのコンポーネントがロボット開発者やシステム設計者が活用できる形で市場に提供されていれば、市場で調達できるものは調達し、手に入らない要素や機能だけを自己開発することで新ロボットを構成することができ、ユーザの要望に応じた製品開発が迅速化される可能性がある。また、注文生産品のレベルでは製品としてのロボットは異なっても、コンポーネントレベルでは共通に使われるため、コンポーネントの量産効果によるコスト低減も期待できる。言うまでもなく現状でもあるシステムを開発する場合、開発者がすべてを作るわけではなく、例えばテレビカメラやモータ、ロボットアームなどは市販品を購入することが一般的に行われている。上記でいうコンポーネントは、ソフトウェアを内蔵した機能モジュールを意味している。例えば、ハンド・アイシステム(アームと視覚センサからなる自由型ロボット)を構成するためには、テレビカメラとロボットアームが必要となるが、この二つがあればシステムができるわけではない。図1・4(a)に示すように、要求される作業に応じた両者を結びつけるソフトウェアが必要である。現状の注文生産ロボットの開発では、テレビカメラ、アームは市販品を購入するとしても、ソフトウェア部分はすべて一社で開発を行っている。最近のコンピュータ技術は分散化が進み、こうしたソフトウェアもテレビカメラ、アーム固有の制御ソフトウェアと、ハンド・アイシステムに要求されるサービスを提供する機能を実現するアプリケーションソフトウェアに分けて設計が行われる[図1・4(b)].アプリケーションソフトウェアはその商品が提供するサービスの内容に依存するが、制御ソフトウェア部分はサービスの内容によら7―5ず共通性が高い。したがって、テレビカメラや、組込み型のコンピュータをアームに装備したうえでこうした制御ソフトウェアも組込んで市場に提供すれば、システム開発者は提供するサービスに注目してアプリケーションの部分のみの開発を行えばよく、開発が迅速化される。しかしその際、開発者が異なるそれぞれの制御ソフトウェアとアプリケーションソフトウェアを問題なく接続できる仕組みが必要である。これは分散オブジェクト技術で使われる、ソフトウェアどうしを結合するソフトウェアであるミドルウェア(middleware)を開発することで解決することができる。そこで、コンポーネントやロボットシステムの要素として使うロボットそのものを分散オブジェクトとして扱えるようにするためのソフトウェアとしてのRT(robot
