# 1・2・6 ロボティクスとメカトロニクス

> **章**: 第1章 総論
> **節**: 1・2 メカトロニクス

ロボットとは1・1・2項で述べたように、何らかの人間あるいは生物的な特徴を備えた、実世界に作用を及ぼす能力をもつ機械であり、こうした機械を目指して人・生物の行動、仕組みを記述する理論を構築したり、それらの理論を背景に具体的に人にサービスを提供する人工物を設計したりすることが、学問としてのロボティクスである。一方、メカトロニクスは、ある工学的問題の解決が要求されたときに、機械工学、電気工学、電子工学、情報工学の知識を活用して、既存の手法を超える解を追求する学問と定義される。開発された機器レベルでロボティクスとメカトロニクスを比較するとき、両者ともに機械、センサ、コンピュータといった共通の要素をもつゆえに、両者を同一視したり、あるいはメカトロニクスが生物模倣以上の広範囲な機器を扱うことから、ロボティクスをメカトロニクスの一分野と解釈する傾向も見られる。しかしそれは、ロボティクスを単にロボットというハードウェアを構築する学問としか見ない誤った見方である。図1・10 メカトロニクスに関連するロボティクスの分野すでに1・1・3項でも指摘したように、ロボティクスにはロボットというハードウェアを構成することだけでなく、人・生物の仕組みを構成論的立場から解明するという分野がある。また、ロボットの構成要素である運動機構や運動制御機構、知能などを実現するための理論構築を行う分野も含む。これらの成果物は理論であり、機械的な構築物ではない。機械、電気、電子、情報技術の知識を総合して実システムを開発することを目指すメカトロニクスは、ロボティクスのこの分野とは重複しない部分である。一方、ロボティクスにおいて成果物がハードウェアになるのは、サービスが実装される場合である。介護、警備、清掃等、具体的なサービスを提供するロボットを開発する場合には、そこに機械、電気、電子、情報のほか、活用できるさまざまな分野の技術の統合が必要になるが、このプロセスはまさにメカトロニクス的研究開発そのものになる。ロボティクスとメカトロニクスの接点は、このサービスの実装という点において初めて生じることを理解しておく必要がある。以上に述べたロボティクスとメカトロニクスの関係を図1・10に示す。文献(1) チャペック、K.(千葉栄一訳)、ロボット(R.U.R.)、 (1994)、岩波書店。(2) 橋本秀紀・渡邊朗子、 空間知能化デザイン―建築・ロボティクス・IT の融合、(2004)、NTT 出版。(3) 谷江和雄、東芝レビュー、59-9(2004)、9.(4) 日本ロボット工業会、 21世紀におけるロボット社会創造のための技術戦略調査研究報告書、(2001)、21-59.(5) 尾崎文夫・ほか4名、日本ロボット学会誌、21-6(2003)、22.(6) 神徳徹雄、計測と制御、44-11(2005)、761.(7) 日本ロボット工業会ホームページ、http://www.jara.jp/(8) Goertz,R.C.,Nucleonics,10-11(1952)、36.(9) Manicini,C.and Roncaglia,F., Acta Frequenza, 32-6(1963)、379.(10) Mosher,R.S.,Proc.
