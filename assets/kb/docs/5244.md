# 2・1・3 数値表現とコード化法

> **章**: 第2章 基礎技術
> **節**: 2・3 項参照のこと。

数、文字、命令などを2進コードの形にすることをコード化(coding)、それを行う装置を符号器(coder)、逆に2進コードを数、文字、命令などに直すことを復号化、装置を復号器(de-coder)という。10進数と2進数との間の対応付けの方法がいろいろ考えられる(1や0の並べ方が異なる)が、そのうち何かの特徴をもつコード体系のみが広く使われている。a. 各種のコード体系。このコード体系の特徴は簡明である点である。. 2進化10進コード[BCD (binary-coded decimal)code]ビット数を増やせば、どんなに大きい10進数も8421コードで表せるはずであるが、一見しただけでは2進数から10進数が、あるいはその逆がわかりにくい。2進化10進コード(BCD コード)はこれを改善するためのもので、10進数の各けたを4ビット2進数で表して並べるものである。例えば、235は001000110101となる。. 3増しコード(excess-3 code)8421コードの10進数3から12までが新しく0から9までを表すと定めたコードを3増しコードという[表2・4(a)].このコード体系の特徴は、10進数0,1,2,...の2進コードで0と1とを入れ替えれば10進数9,8,7,...が得られる(9の補数が容易に得られる)自己相補性があることである。このことは数の引き算の操作を簡単にする。また8421コードでは10進数0の転送に際し、受信側では信号0000が転送されてきたのか故障により瞬断したのかの区別がつかないが、3増しコードではそのようなことがない。. 2-5進コード(biquinary code)表2・4(b)は2-5進コードと呼ばれる。最初の2ビットを見れば、その数が4以下か5以上かがわかり、残りの5ビットの重みは4,3,2,1,0である。このコード体系の特徴は、1語のうち最初の2ビットにも、また残りの5ビットにも1が必ず1個含まれていることである。. 5の中の2コード(2out of5code)表2・4(c)は5の中の2コードであり、10進数0の場合を除き、重みは7,4,2,1,0となっている。このコード体系の特徴は、1語に1の個数が2個含まれていることである。. グレイコード(gray code)対象物の変位、角度等をディジタル的に得るために、図2・4に示すようなラベルを対象物に付け、これを検出器で読み取ることを考える。図2・4(a)は1,2,...ごとの各位置に対応して8421コードの白黒を付けたものである。検出器の直前に例えばAA の位置がくれば出力信号は011であり、BB の位置がくれば100である。しかし実際上、第1ビットが1から0に変わると同時に第2ビットも1から0に変わり、かつ第3ビットが0から1に変わる、ということはあり得ない。最悪の場合に、第1ビット、第2ビットがまだ1であるのに第3ビットのほうが一足早く1に変わってしまい、境界では4を7と読み誤ることもありうる。しかしもし図2・4(b)のように隣接値が1だけ違うようなコードを使うならば、誤差はどの境界でも1にすぎない。これがグレイコード(交番2進コード)といわれるもので、機械式のA-D 変換に欠かせないコード体系である。表2・4(d)は4ビットのコード体系を示す。b. コード誤りの検出と訂正。例えば表2・4(a)の3増しコードにパリティビットを追加すると表2・5となる。2-5進コード、5の中の2コードはそれ自身がすでにパリティチェック可能なコードとなっている。. 誤り訂正コード(error correcting
