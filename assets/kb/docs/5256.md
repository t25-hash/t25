# 2・3・1 ハードウェアの基礎

> **章**: 第2章 基礎技術
> **節**: 2・3 制御用コンピュータの基礎

昨今,コンピュータの小型化が進み,かつてはスーパーコンピュータのみが行えたような計算も,現在では一般の家庭にあるコンピュータで行うことができるようになってきている.そして,パーソナルコンピュータ(PC)で利用されていたCPU はさらに小型化,省電力化されるとともに,用途ごとに特化した仕様で設計され,身の回りにあるほぼすべての家電(家庭用電気製品),自動車などに組込まれてきている.こうしたシステムへの組込みを意識して設計されている組込みコンピュータは,処理速度も有している機能もさまざまであり,用途に応じた選定・利用が必要となってきている.本節では,制御コントローラとして,ロボティクス・メカトロニクス分野でも広く利用されている一般的なPCではなく,ロボットなどのシステムへの組込みを前提とした制御用コンピュータに注目して紹介する.システムへの組込みを意識して設計されている制御用コンピュータは,マイクロコントローラと呼ばれており,計算処理を行うためのCPU に加えて,接続した機器を制御するための機能を有している.接続機器を制御するために,デジタルI/O(input/output)ポート,メモリ,A-D 変換器,カウンタユニットなどが存在する.すべてを有する万能型のマイクロコントローラもあるが,必要でない限りは要求機能に応じたマイクロコントローラを選定する必要がある.マイクロコントローラは,ロボティクス・メカトロニクス分野において,スイッチのような簡単なオンオフの制御から,アクチュエ7―31ータの回転制御やセンサ情報の獲得まで多種の用途で利用されてきている.マイクロコントローラを構成するCPU の基本設計は大きくCISC(complex instruction set computer)方式とRISC(reduced instruction set computer)方式に分類されている.CISC方式はマイクロコントローラが登場した当初から利用されており,高級言語に近づけることによって複雑な処理を実現し,処理能力の向上を図っている.一方でRISC方式は,単純な命令を効率よく実行することで処理性能の向上を図っている.現在では双方ともに互いの利点を取込みながら成長を続けていることから,厳密な区分けは難しくなってきている.現在のマイクロコンピュータ(マイコン)ではRISC方式を採用しているものが多く見られる.このように広く家電などのシステムに組込まれて制御用コンピュータとして用いられているマイクロコントローラであるが,現在では多くの会社より用途に応じたさまざまな種類の製品が販売されている.ここでいくつかのマイクロコントローラについて紹介する.a. PIC(peripheral interface controller)PICはマイクロチップテクノロジー社によって製造されているマイクロコントローラ製品である.PICにCPU,メモリ,I/Oポートが1チップ上に集積されており,プログラムを書き込むことによって,単体でコンピュータとして動作することが可能である.PICを用いることで回路構成が簡単になり,安価なために,それほど性能を必要としない組込みデバイスとして広く用いられている.最近ではDSP(digitalsignal processor)を内蔵したdsPICも登場しており,b項で後述するH 8/SH シリーズと同様に用途の幅が広がってきている(図b. H 8/SH シリーズルネサステクノロジ社製のマイクロコントローラ製品であり,H 8Tiny,H 8,SH-1,SH-2,SH-3,SH-4など用途に応じて広く選択できるラインナップである.SH-2はロボティクス・メカトロニクス用途において広く利用されているマイクロコントローラの一つであり,浮動小数点演算ユニット(FPU : floating point unit)を搭載している点も広く利用される理由の一つである.また,ロボティクス・メカトロニクス分野において広く利用されるものとして,H 8シリーズが挙げられる.これは開発キットなどが手に入れやすいのと同時に,豊富なインタフェースを有することが利用されている大きな理由として挙げられる.組込みOS(operating system,オペレーティングシステム)の開発も盛んに行われており,H 8に搭載できるようにカスタマイズ化されたLinux(リナックス)も存在する(図2・31).c. AVRAVR はアトメル社によって製造されているマイクロコントローラ製品である.PICと同様にロボティクス・メカトロニクスで用いるデバイスを制御するためのインタフェースを有している.また,PICよりも後発であるために,C言語での開発を意識したプロセッサやメモリ構造をもつなど,PICの不足点を補う構成をもっている.図2・30 マイクロチップテクノロジー社のマイクロコントローラの概要7―32メカトロニクス・ロボティクスd. PSoC(programmable system-on-chip)PSoCはサイプレス・セミコンダクタ社によって製造されているマイクロコントローラ製品である.PSoCにはマイクロコントローラと通常の組込みシステムの場合にコントローラの周辺回路として配置されるアナログ-ディジタルコンポーネントが集積化されており,設計者は内蔵ディジタル-アナログブロックの組合せを選択することで,さまざまなアナログ-ディジタル周辺回路を1チップに集積することができる.近年注目を浴びているマイクロコントローラの一つであり,ロボティクス・メカトロニクスで求められる機能[A-D 変換やPWM(pulse width modulation)機能など]を適宜実装できる点が利点といえる.e. PLD(programmable logic device)PLD は内部論理回路を定義・変更可能な集積回路である.内部構成,集積度に応じてさまざまな種類があるが,代表的なものはCPLD(complex pro-grammable logic device)およびFPGA(field programmablegate array)である.性能的には専用設計された集積回路であるASIC(application specific integrated circuit)には及ばないが,論理回路の書換えによって機能向上やバグ修正が可能である.ASICに比べ開発コストが低減できることから,近年用いられることが多くなっている.また,近年ではPLD の内部論理回路を定義・変更可能な特徴を備えたアナログICとして,アナダイム社のFPAA(field pro-grammable analog array)等が製品化されている.画像処理など高速な処理を必要とする用途において,多くの利用例が見られる.マイクロコントローラがロボティクス・メカトロニクス分野で利用されていることに対する共通点として,これらのマイクロコントローラを採用した開発キットが市場に出回っており,一般に手に入りやすい点が挙げられる.そのため,ロボティクス・メカトロニクスの教育用キットや,分散制御モジュールのコントローラとして広く利用されるようになっている.また,これらのマイクロコンピュータを使用した作品事例などがウェブや書籍によって広く紹介されており,マイコン初心者が利用しやすくなっている点も挙げられる.これらのマイクロコントローラはプログラムによってその動作を変更でき,その開発においては各メーカから開発環境が無償で提供されているものもあれば有償のものもあるが,いずれにせよ第三者によって作られている開発環境を利用している.また,このとき利用する開発言語についてもアセンブラ言語,C言語,BASIC(ベーシック)言語などが用意されているものが多い.また,開発したソフトウェアをマイクロコントローラで利用するために,ライタ図2・31 H 8/SH シリーズマイクロコントローラの製品一覧(回路)が必要とされる.多くのマイクロコントローラは動作プログラムを書き込む部分に不揮発性のメモリを採用しているため,回数制限(数千から数万回)はあるものの,電気的に内部のソフトウェアを更新することが可能である.マイクロコントローラは安価で入手しやすく,開発環境も比較的安価にそろえることが可能であることから,これからもロボティクス・メカトロニクスの研究用途において広く利用されることが考えられる.マイクロコントローラは多数存在し,機能も多種多様である.よって,利用するときは用途を明確にし,その用途に合ったものを選定することが重要といえる.2・3・2項において,多くのマイコンのもつインタフェースについて紹介する.
