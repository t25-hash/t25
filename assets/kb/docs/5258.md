# 2・3・3 制御用ソフトウェア

> **章**: 第2章 基礎技術
> **節**: 2・30 )。

a. 制御用ソフトウェアの概要ロボティクス・メカトロニクスで用いる機器の制御において、一つ一つの処理にかかる時間は一定かつ高速であることが求められる。通常の力制御などでは1ms程度、カメラを用いた視覚システムでは30msがリアルタイムとされ、この時間をつねに満たすようにソフトウェアを構築する必要がある。視覚システムの30msに比べて、力制御等で求められる1msは高速であり、これを周期的に実現するためにはb項で示す専用のOS が必要となる。一方で、ハードウェアから少し離れ、操作インタフェースを考える場合、それはコンソール上でコマンドを打ち込むことよりも、GUI(graphical user interface)を用いたユーザフレンドリーなインタフェースが望まれる。こうしたインタフェースを実現するための多くの開発言語が登場してきている。さらに、開発したソフトウェア資産を一度きりの利用でなく、同じ用途においては再利用が望まれるとの知見から、ミドルウェア技術が近年注目を集めている。以上のように、現在ロボティクス・メカトロニクス用途のソフトウェアは単純にハードウェアに近い部分のソフトウェアだけでなく、アプリケーションに近い部分まで階層的な開発が求められてきている。b項以降、すべての基盤となるOS から、その上で動作するソフトウェアを作成するための開発プログラミング言語、それらの開発を促進することを目的として作られている統合開発環境について紹介し、最後にそれらのソフトウェア資産の再利用を目指したミドルウェア技術について示す。b. OS(operating system)メカトロニクス用途で利用されるOS としては、マイクロソフト社のウィンドウズやLinux の各種ディストリビューション、Mac OS が利用されている。ウィンドウズは2007年現在、市販の多くのコンピュータにおいて標準的に搭載されているOS である。ロボティクス・メカトロニクス開発においては、マイクロコンピュータ用の開発環境やアプリケーション等の作成に用いる統合開発環境、シミュレーションソフトウェアなどが利用可能である。Linux は多くのディストリビューションを抱え、それぞれが特徴をもっている。2007年現在、Vine LinuxやFedora,Debian,Ubuntuなどのディストリビューションを利用している例が多く見られる。Mac OS もベース部分では、Linux と同様にUNIX(ユニックス)ベースで作られている。オープンソースで開発されていることから、内部の構造を詳細に把握することが可能であり、ドライバなどで独自の仕様で作成することが求められることの多いロボティクス・メカトロニクス要素の制御などでは、ウィンドウズ以外のこうしたOS を利用するケースが多く見られる。ウィンドウズやLinux は基本的にOS 自体がタスクスケジューリングを行っているため、メカトロニクス要素の制御を一定時間で実行しようとソフトウェアを書いていても必ずしも定期的に実行されるとは限らない。GUIでの操作インタフェースなどではこうしたことは問題とはならないが、1msでのリアルタイムの制御が要求される場合には致命的である。そこで、リアルタイム性を保証する必要がある用途においては、リアルタイムOS(real time OS)と呼ばれるものが利用される。代表的なものとして、VxWorks,RT Linux とART(advanced real time)Linuxなどが挙げられる。VxWorksは古くから利用されるリアルタイムOS で信頼性も高い。一方でRT Linux やART Linux は無償で手に入れることができるリアルタイムOS である。RT Linux がカーネル空間でリアルタイムタスクを実行しているのに対して、ART Linux はユーザ空間でリアルタイムタスクを実行する。2007年現在において、RT Linux の開発は止まっているように見えるが、ART Linux は新カーネルへの対応が進んでいる。リアルタイムOS を用いること7―34メカトロニクス・ロボティクスで安定した制御を実現することが可能となるため、サーボコントロールが必要となる用途において広く利用されている。PCを用いた用途では、前述のようなOS が利用されてきているが、マイクロコントローラにおいてもOS が存在する。広く利用されているOS としてITRON(マイクロアイトロン)仕様に基づいたOS があり、多くの家電で採用されている。さらに、従来のPCが安価かつ小型に実現可能となってきているため、組込み機器向けのウィンドウズであるWindowsCEや、組込み機器用にチューニングされているLinux も登場してきている。このようにメカトロニクス用途において数多くのOS が利用されており、それぞれにおいて実績がある。実際に利用する場合には、用途に合ったOS を選択することが望ましいといえる。c. 制御用ソフトウェアの開発環境メカトロニクス用途でのソフトウェア開発では、対象とするものによってさまざまなプログラミング言語が利用されてきている。ここでは、その中で特に利用されているプログラミング言語やその中で利用するライブラリについて示す。. コンパイル言語(1) C言語C言語は手続き型の言語であり、コンパイルすることで実行が可能となるコンパイル言語である。低レベルの記述が可能であり、デバイスドライバなどの多くはC言語で書かれていることが多い。組込み機器でもC言語に合わせたアーキテクチャを採用しているものもあり、メカトロニクス研究開発において幅広い利用が可能な言語である。(2) C++もともとC言語の拡張として開発された言語である。従来のC言語はオブジェクト指向が導入しにくい仕様となっていたため、オブジェクト指向プログラミングを可能とするための要素が取り入れられている。オブジェクト指向を実現するために、名前空間や多重継承、オーバロード、例外処理の導入や、テンプレートなどC言語にはない機能が追加されている。商用用途において広く利用されている言語であり、開発環境としてもマイクロソフト社製のVisual C++やボーランド社製のC++コンパイラ、さらにはインテル社のC++コンパイラ、GCC(GNU Compiler Collection、グニューコンパイラコレクション)など多くのコンパイラが存在する。(3) C#マイクロソフト社によって開発されているオブジェクト指向プログラミング言語である。同社の。NetFramework(ドットネットフレームワーク)を利用することで開発効率の向上を図っている。共通言語基盤(CLI:common language infrastructure)を採用することで、従来は独立した言語であった、Visual BasicとC言語系のC#など、.NetFramework を基盤とした言語を同じレベルで利用することが可能となっている。構造的にはJAVA(ジャバ)によく似た構造をとる。(4) JAVAサン・マイクロシステムズ社によって開発されたオブジェクト指向プログラミング言語である。C++に比べて、JAVA はガベージコレクションの機能がついているなど、ソフトウェアの安全性と保守性を保証する設計になっている。重要な点として、JAVA 言語はプラットホーム非依存で作られているため、動作するOS を選ばない。CORBA(common object request broker architecture)など分散オブジェクト言語との相性もよいため、ネットワークを介するようなソフトウェア作成においても力を発揮する。C++やC#の開発を支援するツールとしてVisual Studioがあるように、JAVA においても、サン・マイクロシステムズ社で開発されているNetBeans(ネットビーンズ)や、もともとIBM 社を中心に開発されていたオープンソースソフトウェアであるEclipse(エクリプス)などが統合開発環境として存在する。(5) Visual Basicマイクロソフト社が提供するBASIC言語の開発環境であり、開発環境がそのまま言語名としても利用されている。マウスによるドラッグアンドドロップによって簡単にGUIを作成することができ、操作インタフェースの作成などに広く利用される。.NetFrameworkをベースにするようになってから他の言語との親和性が増しており、オブジェクト指向の概念が導入されるようになってきている。. スクリプト言語(1) Python(パイソン)オープンソースのオブジェクト指向スクリプト言語である。Perl(パール)などとともに世界的に広く利用される言語であり、簡単な処理を行わせることから、ウェブプログラミングやGUIの開発など、幅広い分野での利用が可能である。ライブラリも多く存在し、多数のパッケージ提供サイトが存在する。いまや最大の検索エンジンであるグーグル(Google)でも採用されており、近年では日本国内でも書籍が登場し始めており、今後広まる可能性のある言語である。多くのOS で動作することも特徴である。(2) Ruby(ルビー)日本で開発されたオブジェクト指向スクリプト言語であり、Perlなどが用いられてきた領域でのオブジェクト指向プログラミングを目指している。Cプログラムなどを拡張モジュールとして組込むことが可能である。Pythonと同じような仕様であるが、日本で開発されたこともあり、関連書籍が多く存在する。. グラフィックス・画像処理ライブラリ(1) OpenGL(Open Graphics Library)シリコングラフィックス社が中心となって開発した三次元グラフィックス用のライブラリである。OpenGLそのものは、ハードウェアに近いレベルのライブラリである。OpenGLのみでは、実際の表示の部分などをOS に合わせて適宜手順を踏む必要があり、初心者には開発が困難であったことから、より抽象度の高いレベルでのOpenGLの利用を目指したGLUT(The Open GL UtilityToolkit)が作られている。基本的にOS 非依存のライブラリであり、メカトロニクス機器のシミュレーションをはじめ、広い用途で利用されるライブラリである。(2) OpenCV(Open Computer Vision Library)オープンソースで開発されているコンピュータビジョンライブラリである。従来までは、コンピュータの処理速度がそれほど速くなく、画像処理に必要な速度が十分得られないことから画像処理ボードなどのハードウェアに頼っていたが、近年コンピュータの高速化に伴い、以前はボードで行われていた処理をコンピュータ上で実行することが可能となってきている。そうした中で登場したOpenCV は、基本的な画像処理から応用的な画像処理まで多岐にわたりカバーしており、現在メカトロニクス研究において広く利用されるようになってきているライブラリである。. シミュレーション(1) ODE(open dynamics engine)これまで動力学計算には高価なソフトウェアが利用されてきたが、この動力学計算エンジンは、オープンソースで開発されているフリーのエンジンである。マルチプラットホームでの動作が可能であり、開発言語もC,C++を利用できる。最近メカトロニクス研究者の間でも利用されるようになってきている。OpenGLの機能が組込まれていることや、サンプルプログラムも豊富であることから、シミュレーション用途において重宝することが期待できる。(2) OpenHRP(Open Architecture Humanoid RoboticsPlatform)産業技術総合研究所、東京大学、ゼネラルロボティクス社が中心となって開発している、ロボットを対象とした統合シミュレーションプラットホームである。動力学計算エンジンを搭載しており、独自のロボットモデルと制御プログラムを用いた動力学シミュレーションを可能としている。v項で述べるロボットのための分散ミドルウェアであるOpenRTM(Open Robot Technology Middleware)に対応することで、その基本単位であるRT(robot technology)コンポーネントを用いたシミュレーションも今後可能となる予定である。(3) マイクロソフト・ロボティクス・スタジオ(Microsoftrobotics studio)マイクロソフト社によって開発された、ロボットのシミュレーションを行うためのプラットホームである。物理計算エンジンとしてPhysX(フィジックス)エンジンを利用しており、三次元描画には同社の技術であるDirect3D が利用されている。インストール時にいくつかのロボットモデルがインストールされていること、VPL(visual programming language)によりグラフィカルなプログラミングモデルを採用している点などにより、初心者にとって利用しやすくなっている。マイクロソフト・ロボティクス・スタジオに対応したロボットも増えてきており、今後が期待できるロボット用シミュレータの一つである。(4) MATLAB(matrix laboratory)マスワークス社によって開発されている数値解析ソフトウェアであり、メカトロニクス分野においても広く利用されているソフトウェアの一つである。行列計算、ベクトル演算やグラフ表示、画像処理機能など、さまざまなライブラリをもつ。シミュレーション用途にも利用可能であり、そのためのツール群も用意されている。. ミドルウェア(1) OpenRTM-aist産業技術総合研究所を中心にメカトロニクス要素を制御するために開発された資産を有効に再利用するための共通ソフトウェアプラットホームとして開発された、ロボット用ミドルウェアである。ソフトウェアの国際標準化団体であるOMG(Object ModelingGroup)において、その基本単位であるRT コンポーネントの仕様の標準化活動が進められている。基本部分に分散オブジェクト技術であるCORBA を採用しているため、ネットワークを介したロボット要素の制御を可能としている。(2) ORiN(Open Robot Interface for the Network/OpenResource Interface for the Network)産業用ロボットや、各種FA(factory automation)機器向けのミドルウェアである。おもにアプリケーションソフトウェアから呼び出すAPI(application program interface)と、そこで用いるデータの仕様を定めている。幅広い産業用装置を適用対象とすることで装置ごとの仕様の違いをカバーし、アプリケーションソフトウェアにおいて統一的に取扱うことを可能としており、NC(numer-ical control)工作機やプログラマブルコントローラ、ロボットにおいて実績をもつ。(3) RSi(Robot Service initiative)パーソナルサービスロボットなど、人と接するロボットにおけるサービスを簡単かつ便利に利用できる枠組を提供することを目指している。相互運用性のあるネットワークプロトコルを目指しており、複数の企業が参画してその規格を定めている。他のミドルウェアとの相互運用も検討されており、RT ミドルウェアなどとの相互運用が行われている。(4) CORBA(common object request broker architecture)OMG によって定められた分散オブジェクト技術の仕様である。ネットワーク上に分散するコンピュータ間でオブジェクト間の通信を行うための仕様を定めている。多くの開発言語に対応しており、プラットホーム非依存のミドルウェアである。RT ミドルウェアもCORBA をベースに開発が進められている。その他のミドルウェアにおいてもCORBA をベースに開発されているものは多い。情報通信ネットワークは、正確な情報を、どこに対しても、より速く、相手に伝える技術である。古くは電話系ネットワークや放送7―35系ネットワークなど、目的別にそれぞれ進化してきたが、近年ではインタネットの登場やディジタル化の進展に伴いさまざまな機器がネットワークに接続可能となり、かつ一つのネットワークとして相互に連携可能になりつつある。近年、メカトロニクスの分野でもネットワーク利用が盛んとなり、コントローラからの遠隔地のアクチュエータの制御や、複数のロボットの連携システムなどが実現されつつある。本節では、それらを支える技術である情報通信ネットワークの基礎およびそのアプリケーション例を説明する。
