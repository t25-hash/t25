# 2・4・2 インタネット技術

> **章**: 第2章 基礎技術
> **節**: 2・4 情報通信ネットワーク制御の基礎

a. インタネットの概要インタネット(internet)とは,OSI参照モデルの第3,4階層に対し,通信プロトコルTCP(trans-mission control protocol)/IPを用いて全世界のネットワークを相互に接続したパケット交換ネットワークである.図2・34に全体構成を示す.ユーザ計算機はホストとしてデータリンクに接続される.各ホストはIPアドレスと呼ばれるアドレスによって一意に識別される.IPプロトコルとは,ホストから発信されるパケットを最終目的地まで転送するレイヤ3プロトコルである.データを送信するホストは,宛先としてIPアドレスを指定すると,パケット交換ノードであるルータを介して目的地に応じて経路制御(ルーティ図2・33 OSI参照モデルとデータ単位図2・34 インタネットの構成ング)されてパケットが最終目的地まで届けられる.TCPプロトコルとは,IPパケットを用いてホスト間で信頼性のあるデータ転送を行うためのレイヤ4プロトコルである.インタネットでは,データリンク以下の伝送手順(レイヤ1,2相当)の規定はなく,通信距離や速度に応じてさまざまな方式が利用されている.また,メールシステムなどアプリケーションとして位置付けられるOSIの5層以上のプロトコルも,インタネットプロトコルとして広く利用されている.b. データリンク技術インタネットでは,ネットワーク構成や規模に応じて幅広いデータリンク技術(伝送メディア)が利用されている.一般に,広域ネットワーク(バックボーンネットワーク)側はISP側で管理されており,一般のユーザはその構成を把握する必要がない.一方,アクセス系ネットワークやローカルネットワークはユーザにおける利用判断,設計が必要であり,それぞれの伝送メディアの特性を理解しておく必要がある.表2・7に代表的なデータリンク技術について整理する.c.
