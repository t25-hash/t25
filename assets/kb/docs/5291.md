# 4・2・3 運動分解制御と動的制御

> **章**: 第4章 ロボティクス
> **節**: 4・2 マニピュレーションの力学と制御

ロボットアーム(マニピュレータ、manipulator)は先端部に装着されたエンドエフェクタ(end-effector)やハンド(hand)に運動(力を含む)を与えることで作業を遂行する。マニピュレータは通常、多リンク・多関節機構から成り立ち、各関節(自由度)の運動を制御することで全体の運動を実現する。作業のやり方はエンドエフェクタの運動で記述されるから、その運動を実現するためにはそれを各関節の運動に変換(分解)し制御することが必要である。ここではその方法について述べる。マニピュレータの制御の精度を考えたとき、フィードバック制御(feedback control)だけでなく、フィードフォワード制御(feed-forward control)を加味することが非常に有効である。そのためのマニピュレータの動力学を考慮した動的補償制御について述べる。a. 速度分解制御(resolved motion rate control)エンドエフェクタの速度(位置・姿勢の時間変化率)を各関節の回転速度(またはスライド速度)に変換し、制御する方法である。これにより、エンドエフェクタ速度から関節速度を求める次の式が得られる。ここでは、実現すべきエンドエフェクタの速度を分解して得た速度を、関節速度サーボ系の目標値として与えている。関節速度サーボ系の代わりに関節位置サーボ系を用いても構成できる。この場合は、分解された速度を積分して、関節位置サーボ系の目標値として与えればよい。この制御法はもともとテレオペレータの制御法として考案されたものであり、オペレータが意図する方向にエンドエフェクタを導くために使われた。産業用ロボットの応用に必要なエンドエフェクタの位置・姿勢や力を制御する場合は、本制御系の外側にもう一つの制御系を設定する必要がある。速度分解制御と同じことは、エンドエフェクタの位置・姿勢を関節角度(またはスライド量)に変換する方法でも実現できる。しかしそれは多次元の代数方程式を解く問題に帰着され、式(4・170)のような一般解は存在しない。本方式は複雑な自由度配置をもつマニピュレータにも適用できる点が特徴である。b. 加速度分解制御(resolved acceleration control)エンドエフェクタの加速度を各関節の加速度に変換し、必要な関節トルク(または力)を割り出すことで、エンドエフェクタの運動自由度ごとに任意のサーボ系を構築する方法である。速度分解制御からの類推では、関節加速度に分解した後、関節加速度サーボでそれを実現することになるが、そのような運動学(kinematics)的な変換だけでは有用な機能は発現せず、動力学と組合わせるところがポイントである。結局、式(4・171)のエンドエフェクタの加速度を生じさせるためのトルクは次式で計算できる。右端のブロックは関節角度をエンドエフェクタの位置・姿勢に変換するブロックである。この計算を行い、現在の位置・姿勢をフィードバックすることにより、エンドエフェクタの位置・姿勢の各成分を次のような多様なモードで制御できる。. 自由状態=0(4・177)と設定すれば、運動に関する制御はかからず、外力により運動が定まる状態となる。力の開ループ制御を行う場合はこのモードを用いる。ここでは力センサにより計測した、力のフィードバック量である。さらに、インピーダンスを設定するもう一つの方法がある。次の微分方程式で求めたをi項の位置制御系の目標値として入力する方法である。また、エンドエフェクタが環境と硬い接触を行う場合も、パラメータの適当な設定範囲で有効に機能する。第二の方法は高剛性の位置制御に基づく方法であるので、正確な特性が実現できるが、エンドエフェクタと環境との間に硬い接触があると制御が不安定化する可能性が高い。そのため、手首部や力センサに柔軟部を入れるような工夫が要求されるが、動力学モデルに多少の誤差があっても有効に機能するという特徴がある。c. 動的補償制御加速度分解制御においては、式(4・174)の計算を実時間で行う。このような制御を動的補償制御と呼ぶ。b項に述べた例で一つ省略した事項があるが、それはマニピュレータの動力学モデルに関する重力項や速度二乗項に関する動的補償である。このとき、式(4・174)は次のようになる。ここでは非線形力学の動的補償をも含んだ制御系になっている。このような動的補償はフィードフォワード補償として位置付けられ、フィードバック制御の誤差要因を取り除き、制御性向上のために有効である。
