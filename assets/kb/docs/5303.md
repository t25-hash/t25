# 4・4・3 二足歩行技術

> **章**: 第4章 ロボティクス
> **節**: 4・4 ヒューマノイド技術

図4・102に、HRP-2が前提としている基本的な歩行制御の枠組を示す。ロボットを歩かせるために必要な関節角度データの時系列を歩行パターン、これを計算するソフトウェアを歩行パターンジェネレータと呼ぶ。床面が完全に平らで指令どおり正確にロボットが動作する理想的な状況では、単に計算したパターンどおりにロボットを動作させるだけで歩行を実現することができる。しかしこのようなオープンループによる歩行では、床面にわずか数mm程度の凹凸が存在するだけで数歩のうちに不安定化してしまう。そこで、床面の凹凸や外乱の影響を適切なフィードバックで補償し、安定した長距離の歩行を可能とするソフトウェアが必要となる。これを安定化制御系と呼ぶ。a. 倒立振子モデルによるダイナミクスの簡略化二足歩行ロ7―127図4・101 HRP-2プロトタイプ機による転倒時の対応と復帰動作図4・102 歩行制御の基本的な枠組図4・103 二足歩行ロボットを表現する三次元の倒立振子ボットが片脚支持相にあるとき、その支配的なダイナミクス(動力学)は床面上の支持中心点とロボットの重心を結ぶ仮想的な倒立振子(とうりつしんし、inverted pendulum)によって表現できる(図4・103)。ここで倒立振子の先端は質量の質点で表現され、脚は無質量で伸縮でき、けり力を発生できるとする。質点にはけり力と重力が作用し、けり力はの各方向成分に分解できるので、運動方程式は次式のように書ける。次式で表される平面を考え、倒立振子の重心がつねにこの平面内を運動するように拘束制御することを考える。これを2回微分して次式を得る。図4・104にそのイメージを示す。このとき、重心の水平方向のダイナミクスは式(4・295)にけり力の式(4・298)を代入すれば得ることができる。このようなダイナミクスを三次元線形倒立振子と呼ぶ。三次元線形倒立振子を用いると二足歩行のダイナミクスを見通しよく考察できるほか、これに基づいて歩行パターンジェネレータを構成することもできる。b. ZMP を利用した歩行パターン生成の考え方倒立振子に基づく歩行パターン生成が順動力学をベースに考えているのに対して、逆動力学をベースに考えるのが、ZMP(zero momentpoint)を利用した歩行パターン生成である。図4・105のようなロボットを考える。この歩行ロボットは人間型の脚と補償用おもりからなり、各関節はすべて位置制御されるとする。脚は人間の歩行データをもとにあらかじめ設定されたパターンで動作させるが、そのままではロボットが転倒する可能性がある。そこで、上半身の補償用おもりを振ってバランスをとるのだが、そ図4・104 三次元線形倒立振子図4・105 ブコブラトビッチらが想定していたロボットモデルの軌道をどうやって決めればよいだろうか。ブコブラトビッチ(Vukobratovic)らは、つねに支持多角形の内部に位置するようなZMPの目標軌道をまず設定し、これが実現されるように補償用おもりの運動を決定するという方法を示した。具体的にはZMPの設定値と運動方程式からおもりの加速度が得られ、これを積分すればおもりの運動パターンが決定できる。ただし、適切な初期条件を与えないと可動範囲を超えて発散してしまうので、周期的な運動を与える初期条件を繰返し計算によって求める。このように補償用おもりを搭載した二足歩行ロボットとして、東京工業大学の船橋らによるMEG-2と、早稲田大学の加藤・高西らによるWL-12がある。一方、今日のヒューマノイドでは補償用おもりを用いる代わりに胴体の加減速によってZMPを目標値に調整している。その具体的な方法を次に示す。c. ZMP に基づく実時間歩行パターンジェネレータZMPによる歩行パターン生成をより一般的なヒューマノイドに適用し、さらに実時間でこれを行う方法について述べる。まず、図4・106のようなモデルを考える。ここでは、全質量が重心1点に集中し、水平方向にのみ運動を行うという粗視化を行い、ロボットを無質量のテーブル上を質量の台車が走行するというモデルに置換えている。これをテーブル・台車モデル(cart-table model)と呼ぶ。テーブル・台車モデルで台車重心の位置を、高さをとすると、床反力の圧力中心(ZMP)は次式で与えられる。次式のような離散時間の予見制御系を用いることによって、この計算を実時間で行うことができる。上式で計算されたを使って台車の躍度(加速度の時間微分)を決定する。人間サイズのヒューマノイドの場合、予見時間を2s程度に設定すれば、適切な軌道を作ることができる。図4・107は予見制御系によって3歩前進して静止する歩行パターンを計算し、得られた重心軌道を水平面に投影したものとZMPを示している。ZMP(細線)は1歩ごとに着地足(破線の長方形)を順番に直線的にたどっていく。これに対して、重心軌道(太線)は滑らかな曲線を描いている。図4・106 テーブル・台車モデル図4・108では同じ歩行パターンを時間軸を横にとって前進方向と左右方向に分けて示した。歩行中の支持多角形の境界が破線で示されている。ZMP(細線)がつねに支持多角形の内部を移動するように設定されているのに対し、重心軌道(太線)はしばしば支持多角形の外へ出ており、動的な歩行パターンになっていることがわかる。こうして得られた重心軌道と、別途決定された遊脚軌道から逆運動学を用いてロボットの関節角度を計算して歩行パターンが完成する。以上の方法ではロボット全体のダイナミクスを大幅に簡略化しており、例えば遊脚を振り出す際の反動や手の振りなどの影響は考慮されない。しかしヒューマノイドでは、胴体部にバッテリやコンピュータが搭載され、かなりの質量が集中しているため、たいていは上記の方法で動歩行を実現できる。ただし、歩行中に胴体姿勢を大きく変化させるような場合には、以下に述べる動力学フィルタ等を用いてパターンをより安定なものに改善する必要がある。
