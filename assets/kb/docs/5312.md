# 4・5・5 多自由度マスタスレーブシステム:同構造型と異構造

> **章**: 第4章 ロボティクス
> **節**: 4・5 テレロボティクス

型機械式マニピュレータでは、マスタアームとスレーブアームの構造は同一である必要があった。これはマスタアームとスレーブアームで対応する個々の関節が機械的に結合されていたため、同じ構造でないと操縦者が指令するマスタの手先の動きがスレーブ側で再現できなくなってしまうからである。サーボマニピュレータにおいても、初期のころは対応する個々の関節ごとにサーボ系を組んでいたので、基本的にマスタとスレーブは同構造である必要があり、大きさが異なる場合でも相似形が許されるだけであった。このようにマスタアームとスレーブアームの構造が同一(もしくは相似形)であるものを同構造型マスタスレーブシステム(master-slave systemwith same configurations)と呼ぶ。同構造型は、マスタとスレーブの構造が同一でなければならないという制約があるが、基本的には対応する関節ごとに1自由度の制御系を組めばよく、システムとしては簡単になる。また同構造のため操縦者は自然にスレーブアームの特異状態を避けることができ、万が一特異状態になっても関節レベルの制御のため制御上問題になることはない。ロボット工学における順運動学と逆運動学によりロボットアームの関節変位と手先の位置との関係が記述できるが、バイラテラル制御の作業変数を同構造型では必ず用いられていた関節変位ではなくある作業座標系で記述された手先の位置・姿勢とすることにより、異なった構造のアームを組合わせてもマスタスレーブシステムが構成できるようになる。これを異構造型マスタスレーブシステム(master-slave system with different configurations)と呼ぶ。異構造型の考え方を最初に提案したのは、アメリカのJPLのベッジ(Bejczy)らである。図4・123に、異構造型マスタスレーブマニピュレータの例として当時のシステムの流れをくむJPLのシステムを示す。異構造型バイラテラル制御の定式化は一とおりではないが、ここでは先に示した式(4・306)(4・307)を拡張したものを示す。異なった構造をもつマスタアームとスレーブアームであっても、ある共通の作業座標系で手先位置および手先力を表現することは可能である。ここではこのようにして表現されたマスタとスレーブの手先位置ベクトルを욇,욦とし、手先力ベクトルを욇,욦とする。先の場合と同様に욇がオペレータからマスタアームに加える力であるのに対し、욦はスレーブアームが環境に加える力であることに注意されたい。욇,욦は、それぞれのアームの関節変数욇,욦から順運動学問題を解くことで得られる。욇=욇욇(4・319)욦=욦욦(4・320)ここで、욇,욦はそれぞれマスタアーム、スレーブアームそれぞれの順運動学計算を表す関数である。両アームの手先力ベクトル욇,욦は手先に取付けられた力センサの計測値から適当な座標変換によって求められるものとする。このように、作業座標で表現したマスタスレーブシステムの制御系の一般的表現は次式のように与えられる。욇=욇욣욇+욇욋욇dd욇욞욇욇욇-욇욣욦+욇욋욦dd욇욞욦욦욦(4・321)욦=욦욣욇+욦욋욇dd욦욞욇욇욇-욦욣욦+욦욋욦dd욦욞욦욦욦(4・322)ここで、微分演算子はベクトルの各要素に作用するものとする。上式の一般表現の各ゲインは通常、対角行列とする。作業座標で表現された制御入力욇,욦を関節駆動力に変換するには、ヤコビ行列の転置をかければよい。욇=욇욇욇(4・323)욦=욦욦욦(4・324)異構造型の手法を用いれば、一つのマスタアームを用意していればそれだけで種々の異なった構造のスレーブアームを操縦できるようになる。このように操縦装置を一般化できるのが異構造型の利点ではあるが、一方で構造が異なるがゆえにスレーブアームの作業範囲の限界や障害物を回避したいときのスレーブアームの姿勢を操縦者がイメージしづらい点や、アームの特異点の場所が異なることで操縦に支障がでる場合があることが欠点である。
