# 4・5・7 通信時間遅れを含む遠隔操作システム

a. 時間遅れ(time delay)の問題とその対処法マスタとスレーブの間の距離が大きくなると、その間の通信に要する時間が無視できなくなる。通信時間の遅れがあると、通常のバイラテラル制御では系が不安定になることがよく知られている。時間遅れのある通信環境下でも安定性が保証されるバイラテラル制御についてはb項で詳説するが、時間遅れが大きすぎると元来バイラテラル制御がもつ直接作業に近い作業形態という特徴が失われてしまうことになる。例えば極端な例として、火星と地球との間の通信時間遅れ(約30分)のもとでも安定なバイラテラル制御を構成することは理論上は可能であるが、実際のところバイラテラル制御では作業が不可能なのは明らかである。環境からの情報のフィードバックが遅れる状況下では、どうしても動かしては待つ(move-and-wait)という操作をせざるを得ない。このような操作性の悪化を補う目的で、予測情報を提示する手法がいくつか提案された。まず図4・128に示すように、マスタの指令に遅れなしで応答するバーチャルなスレーブアームの映像を実映像に重ね合わせて表示する予測ディスプレイ(predictive dis-play)の手法が提案された。次に、環境との接触力を予測して力フィードバックとして提示する試みも行われた。また、時間遅れが大きくなると予測の精度も悪くなるため、スレーブ側にある程度の自律性を付与する手法が有効である。これについてもc項で詳しく説明する。b.
