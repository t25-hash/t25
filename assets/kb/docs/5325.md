# 4・7・3 行動ベースド制御(behavior based control)

モデルベースド制御は一般に計算時間がかかるので、構造が動的に変化する環境では適用困難である。また、センサによる環境情報入力、環境モデルの構築、モデルに基づく動作計画、実際の行動制御が機能別に直列になっているので、そのうちのどの段階であってもエラーを生じれば最終結果もエラーとなる。このため、頑健性に関しても問題があることが指摘されている。行動ベースド制御はこれらの問題を解決するものとして提案されたもので、いくつかの異なったアプローチがある(図4・157)。a. 感覚-行動対をルールとしたルールベース制御センサを用いたローカルな環境状態認識に基づいて、短期的な行動戦略を決定し行動に移すことを繰り返す(図4・158)。モデルをもたず、複雑な探索計算もしないので、実時間の反応的な行動が実現できる。最適性や合目的性は保証されないが、センサ誤差のために状態認識に失敗し誤った行動をとったとしても、観測と行動が短い周期で繰返されるので、以降の観測により正しい環境認識がなされれば修正行動が起動され、いずれ目標が達成されることが期待できる。過去の行動履歴に基づいて書き換える内部変数や、達成すべき目標に関する内部変数を用意し、これらも合わせた形で環境状態として認識し、行動を決定するようにルールを拡張することにより、単純な条件反射行動にとどまらず、より複雑な行動シーケンスを生成できる。動的に変化はするが、生じうる状況が限定されている環境における比較的簡単な作業であれば、ロボット制御に適用できると考えられる。b. 学習によるルールの獲得とルールベース制御即応型のルールベース制御では、設計者が内省あるいは試行錯誤を繰返してルールの作成と蓄積を行うため、動的な環境で複雑な作業を実現するまでには膨大な労力が必要となる。そこで、学習によりルールを自動生成するシステムが考えられてきている。代表的なものとして強化学習がある。強化学習は教師なしの学習であり、目標を達成したら環境から報酬を得る枠組を用意しておけば機能する。クラシファイアシステム(classifier system)とQ学習(Q-learning)が代表的なものである。クラシファイアシステムの基本構造は、図4・159に示すようにルールベースシステムに、信頼度割当てシステム、GA(genetic al-gorithm)によるルール発見システムを加えたものである。センサ入力は作業記憶に書き込まれ、その内容と起動条件が一致するルールが実行される。これを一定の短周期で繰返して目標を達成しよう7―152メカトロニクス・ロボティクスとするものである。作業記憶の記入内容によって複数のルールが起動されうるときは、各ルールに付与された信頼度と呼ばれる評価値の高いものが実行される。信頼度は、学習前にはすべて同一値が割り当てられる。種々の条件下でシステムを駆動し、試行錯誤的な動作を繰返す過程で学習がなされる。すなわち、目標を達成し報酬が得られたときには、そこに至るまでに寄与した一連のルールの信頼度を一定のアルゴリズムに従って高め、失敗に終わった場合には信頼度を下げる。このようにすると、学習が進み、環境と作業目標に適合したルールは高い信頼度をもつようになる一方、適合していないルールは低い信頼度となる。ある程度学習が進んだ時点で、ルール発見システムが起動され、ルール群に対し遺伝的操作を行ってルールを進化させる。信頼度の高いルールは高い確率で次世代のルールを残すことができる反面、信頼度の低いルールは次世代のルールを残す確率が低く、淘汰されていく。このような学習と進化の繰返しにより、適応システムが実現される。信頼度割当てのアルゴリズムとして、BBA(bucket brigade algorithm、バケツリレーアルゴリズム)や利益共有法(profit sharing)が知られている。Q学習は、センサで観測される環境状態sとその状態でとるべき行動aの組合せに対して、その行動をとったときに将来得られるであろう期待累積報酬(Q値と呼ぶ)を学習によって獲得するものである。学習の結果はQテーブルに残されるので、センサで状態を観測したらQテーブルを参照して、期待報酬の大きな行動をとればよいことになる。環境やロボットに関する事前知識を必要とせず、実環境で稼働中でも学習を継続していれば、予期できない変化が生じるような環境にも適応できるとはいうものの、実際には学習に時間がかかる。また、複数のサブゴールを満たさなければならないような複雑な作業はできていないという問題も残されている。c. 行動ベースド制御の意義サブサンプションアーキテクチャやより広義の行動ベースド制御は、実際に人間社会に役立つ作業行動をする機械の制御に用いる技術と、行動による問題解決という知能の本質や認知科学の追求という二つの観点がある。モデルを用いた探索計算をしないことから実時間性が確保できるが、実現される行動は反応的であり、目的に合った行動が発現する保証はない。サブサンプションアーキテクチャでは、障害物を避ける、うろつくといった基本的な行動から、人間にとって有益な作業のレベルの上位行動までの階層行動分解と再構成ができるのかは不明である。また、各構成要素が並列でつねに動いているため、一部が故障しても何らかの行動ができるという意味では頑健であるが、それぞれの階層で行動指令を出すので、どの行動を採択し実現するかを決定する調停機構がキーになる。行動ベースド制御は、移動ロボットへの適用例は多いが、精密な手作業ロボットへの適用は極めて少ない。世界で最も多く売れた掃7―153図4・163 第1層と第2層で構成されたうろつきおよび衝突回避モジュール除ロボットRoomba(ルンバ、2007年1月末時点で200万台以上と発表されている)は、サブサンプションアーキテクチャに基づいているとも伝えられているが、詳細は公表されていないようである。実用過程で試行錯誤や失敗が許されるロボット応用分野(清掃、エンターテイメント)と、作業の失敗が製品品質や社会基盤の安全にかかわるようなロボット応用分野(生産ラインでの精密組立、極限作業など)とでは、適用できるロボット制御技術が違っているのが現状であろう。人間は行動を起こす前に結果を予測する。複雑で達成することが難しい目標であればあるほど熟考することになる。一方、サブサンプションアーキテクチャは、行動の予測をしない。モデルをもたないのでシミュレーションできない。予測をしない認知主体はあり得ないという意見もある。サブサンプションアーキテクチャは、認知科学や人工知能の分野に大きな影響を与えており、より一般化した行動規範型アーキテクチャも提案されている。最初の提案から20年が経過し、広範な議論から種々の批判や評価が生まれている。
