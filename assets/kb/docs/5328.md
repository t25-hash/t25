# 4・8・1 安全に対する考え方

> **章**: 第4章 ロボティクス
> **節**: 4・8 ロボットの安全

安全は,許容できないリスクが存在しないことと概念的に定める考え方が国際的に広く受け入れられ,国際規格[例えば文献(303)]としてもうたわれている.したがって,対象としている状況や系が安全であることを確かめるためには,リスクを一つ一つ洗い出し,これが許容できるかどうかを確かめる必要がある.この確かめるためのプロセスはリスクアセスメントとして,これもまたロボットのみならず機械安全分野全般にわたってすでに広く受容され,技術的により詳細なレベルまで国際規格により規定されている.以下,これに沿ってリスクアセスメントの概要を説明する.a. リスクアセスメント(risk assessment)リスクアセスメントの基本的なプロセスは,図4・173中右側に示すように以下の要素で構成される.(1) 使用環境(条件)の整備(2) 危険源の同定(3) リスクの見積り(4) リスクの評価(1)の使用条件の整備は,ロボットの使用される状況を特定するためのプロセスで,ロボットシステムの構成,機能,仕様,さらに使用環境を考慮して定める.次に,(1)の使用状況に沿って危害の潜在的な源と定義される(2)の危険源の同定,例えば文献(304)の付録A の表A.1に掲げられているリスト中から該当する危険源を抽出する作業が行われる.危険源を同定するプロセスは,その具体的な作業である事故シナリオ想起の精緻さが,その後のアセスメントプロセスからリスク低減手法の決定まですべてのプロセスの枠組を決定付けるため,極めて重要なプロセスである.危険源はsource of hazardsの訳語であるが,hazardsを,起こりうる事故と定義して,その発生条件を明示する立場もある.(3)のリスクの見積りは,リスクの定義に依存してフォーマットが変わりうるものであるが,基本的には人間が危険源から被る可能性のある危害の程度とその危害の発生確率の組合せによって表現され,さらに危害の発生確率は,人間が危険源にばく露される頻度(あるいは時間),危険事象の発生確率,人間が危険事象を回避する可能性の三つの要素の関数となる.最後に,(4)のリスクの評価は,その後にひかえるプロセスとしてリスク低減措置を必要とするか,あるいはすでに安全性が達成されたかを決定するプロセスである.このために,リスクの許容値を定め,他の類似機器の場合との比較も必要に応じて行いながら,個々のリスクに見積もられたリスク値が許容値か否かを網羅的に判断し,対象機器システムに対して適正なリスク図4・173 リスクアセスメントb. リスク低減リスク評価の結果,リスク低減措置が必要と判断されたら,図4・173中左側に位置付けられる保護方策プロセスによってリスク低減措置を講じる.このプロセスでは,リスク低減が必要と評価されたそれぞれのリスクに対して,3ステップ法と呼ばれる3段階からなる措置を講じる.それらはすなわち,(1)本質安全設計,(2)安全防護策・付加保護方策の安全方策をこの順番で講じ,さらに(3)残留リスク情報のユーザへの提供を行う.本質安全設計は,3ステップ法の中では最優先で講じられるべき手段であり,より危険の少ない材料への代替等,設計を工夫することによって,安全防護物など付加的な設備の設置を行うことなく安全性の向上を図ることである.安全防護策とは,本質安全設計によってだけでは許容できないリスクが残ってしまう場合に,さらに機構や制御によって安全性向上を図ろうとするもので,リスクのレベルに応じて適切な技術的方策を選定することになる.近年,機能安全(functional safety)と呼ばれる安全防護策段階の設計概念がヨーロッパから提言され国際規格化されて,これを反映した安全技術が急速に普及されようとしている.これは,ランダム故障と系統的故障のいずれの故障によってもハザードをきたす機器として,電気/電子/プログラマブル電子機器[electrical/electronics/programmable electronics(E/E/PE)機器]およびソフトウェアを広くシステマティックに対象として,それらのライフサイクルにわたる管理,技術要件に加えて,さらにこれらシステムにかかわる人から組織までの責任要件をまとめたものである.同規格Part 3あるいはさらにPart 6には,ハードウェア・ソフトウェア統合システムのライフサイクルにソフトウェアの開発を組込む目的で定めたソフトウェア要求事項およびその適用例が示されており,知的なふるまいのためにソフトウェア開発の比重がますます高くなるロボットの安全にとって,今後必要欠くべからざる技術要件となろう.産業機械や製造システムの安全技術としては,安全状態=エネルギーが零の状態の考え方に従って,機械システムへの電力供給源に直接かかわる安全関連系(safety-related system)の安全性能の妥当性が,近年SIL(Safety Integrity Level,安全インテグリティレベル)等によって評価されるようになっている.SILは,E/E/PE機器の安全関連系において,ハードウェアの危険側故障の検出ができない確率をどれだけ抑えられるか,また,ソフトウェアで安全な状態間の遷移がどの程度保証できるかを統一的の評価するこ7―159とができる指標となっている.
