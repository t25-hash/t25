動作計画産業用ロボットの動作は、希望の位置と姿勢を教示して、それを正確に実現するのが基本である。指令はロボットのエンドエフェクタの位置と姿勢、速度の点列として与えられる。それを単位時間ごとのロボット各軸の指令値に生成するのが、本制御の目的である。教示の点列には、点間の軌跡を制御する必要のないもの[PTP(point to point)制御]と、教示点間の軌跡を規定するもの[CP(continuous path)制御]とがある。CP制御をする場合は、サーボ制御の時系列指令値を生成するために教示点列間の補間を行う必要がある。点列間の移動は、直線であるか、円であるかなどの教示データがあるので、それに従う。また、補間は教示の座標系に対応して行われる。座標系には、ロボット基盤部の一点を原点としたもの(ベース座標系)、エンドエフェクタの一点を原点にしたもの(ツール座標系)、ワークの一点を原点にしたもの(ワーク座標系)、各関節角度で表現する関節座標系等がある。補間の後、サーボ系の制御周期に対応したおのおのの点を各関節角度に座標変換する。基本的にはこれがサーボ指令になるが、そのままでは教示どおりの動作は不可能な場合があり、各種の対策をとることになる。まず、ロボットの軸構成によっては特異点を通過すROM :read only memoryRAM :random access memory LCD :liquid crystal displayIC:integrated circuitることがあり、その場合は各軸指令値を修正する。その具体策はロボットメーカごとに種々の工夫が行われている。また、ロボットの動特性を考慮して、加減速部や曲率の高い部分に対しての指令値の調整が行われる。d. サーボ制御サーボ制御の構成例を図5・4に示す。基本はPID(proportional-integral-derivative)補償の位置決め制御であり、電流、速度、位置の三重のサーボループで構成される。産業用ロボットでは高速性と高精度が要求され、各種の追加補償を行うため、現在はソフトウェアサーボシステムが一般的である。図ダイナミクスを解き、振動を抑制した信号を追加しながら高速応答性を得ようとしている。状態推定オブザーバ部では、ロボットアームや減速機部のねじれを推定し、適切なゲインでフィードバックすることで制御系の安定性向上を図っている。また、関節型ロボットでは、その動作や姿勢によって重力トルクや軸間干渉トルクが変動する。これが外乱となって精度に影響するので、外乱トルクの補償値を計算して制御系に追加している。サーボ動作の高精度化の要求は高いので、各種の現代制御理論が適用されつつある。e. 知的制御産業用ロボットでは、あらかじめ入力された動作を正確に再現する教示再生動作が基本であるが、前提となる精度や条件が不確定の場合にも対応できる制御システムが求められており、外界センサによる補正や人工知能技術の適用が試みられている。外界センサとして多用されるのが視覚である。作業のトリガの検出や安全情報の検出のほかに、サーボ制御において教示データの補正に使われる例が多い。例えば、自動車の生産ラインで基準位置を示す穴の位置を計測して作業位置を補正するもの、溶接作業でワークのひずみを検出してトーチ位置を修正するものなどである。視覚センサとしては、二次元画像処理を行うもの、レーザポイントやスリット光を照射するものなどが多く、三次元処理を行うものもある。また、組立作業などでは、位置制御だけでは作業ができない場合が多く、力覚センサで作用力をフィードバックしてロボット動作の指令に利用する力制御が行われる例が増えている。力覚センサはエンドエフェクタ部に設置され、3軸方向の力とトルクをひずみゲージによって検出する方式が一般的であるが、過負荷への耐性、耐ノイズ性等に課題がある。また、力制御における安定性や応答性等は今後改良の余地がある。その他、各種の最適制御、学習制御等が採用された例があるが、一般的にはなっていない。f. ロボット言語ロボット言語は、人間がロボットに所望の動作や作業を効率よく行わせるために作られた言語である。JIS B8439:1992で「産業用ロボット―プログラム言語SLIM」が定められているが、実際はメーカごとに独自のロボット言語が使用されている。言語レベルは、低いほうからコマンドレベル、原始的動作レベル、構造的動作レベル、対象物状態レベル、作業目標レベル等に分類される。コマンドレベルは、アセンブラやNC(numericalcontrol)コードに相当するもので、原始的動作レベルはMOVETOといったエンドエフェクタの動作を指示するものに相当する。構造的動作レベルは、ロボットの個々の動作を効果的に記述しうるように言語仕様が拡張されたものであるが、ソフトウェアに熟練していないと活用できず、一般的ではない。対象物状態レベルは、作業を対象物間の状態の変化としてとらえて記述するものである。状態変化が大幅なものになると動作手順の自動生成が困難になることから、部品をある状態に置いたり、他の部品と組合わせるといった単純な状態変化を単位にしている。さらに、作業目標レベルは作業の最終目標だけを与えて、その達成のための動作手順やデータを自動生成させるものであるが、実現には高度の問題解決機能が必要となる。現在実用化されているロボット言語は、そのほとんどが原始的動作レベルまでであり、構造的動作レベルは研究開発段階である。将来はより高レベルなものが実用レベルに達すると思われるが、実際に広く使用されるためには、各メーカの言語とのインタフェースが統一されることが課題となる。7―169サーボオフ教示ダイレクトティーチサーボオン教示オンライン方式ペンダント教示リモートティーチジョイスティック教示マスタスレーブ教示位置・姿勢教示法ダミーロボット教示間接ティーチ専用具教示(キューブなど)オフライン方式数値教示オフラインティーチグラフィック教示言語教示。..(a)
