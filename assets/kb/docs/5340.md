# 7・1・2 映像圧縮技術

> **章**: 第7章 情報圧縮・セキュリティ技術
> **節**: 7・1 圧縮技術

映像圧縮技術について，代表的な圧縮方式（国際標準化方式）と
その技術について解説する．映像圧縮方式は，大きく分けて，静止
画像の圧縮方式と，動画像の圧縮方式に分類される．また最後に，標
準化方式には採用されていないが，代表的な圧縮方式を紹介する．
a.静止画像圧縮方式 静止画像圧縮方式は，二値画像の圧縮
方式と，多値画像の圧縮方式に分かれる．二値画像とは，0,1の二
値で表される画像であり，代表的なものとしてはファクシミリ
（facsimile，fax）の画像が挙げられる．一方，多値画像とは，例
えば0～255の256値で表される画像であり，代表的な画像として
は，ディジタルスチルカメラの画像が挙げられる．
.二値画像の圧縮方式
(１) MH（modified Huﬀman)，MR（modified read)，
MMR（modified modified read）
G3ファクシミリやG4ファクシミリに用いられている圧縮方式
である．画像の0,1の連続長を符号化するランレングス方式であ
る．MHは，一次元符号化方式であり，MR，MMRは，二次元逐
次符号化方式である．
(２) JBIG（Joint Bi - level Image experts Group）
JBIGは，算術符号化を用いて圧縮を行う．圧縮する画素に対し
て，周囲の画素のパターンからコンテクストが決定される．このコ
ンテクストを用いて，圧縮する画素が，0であるか1であるかの予
測を行うことで，圧縮を行う．JBIGは，可逆圧縮アルゴリズムで
あり，従来のMMRなどに比べて，圧縮率が高い．
.多値画像の圧縮方式
(１) JPEG（Joint Photographic Experts Group，ジェーペ
グ）
JPEGは，ISO/IEC JTC1/SC29/WG1とITU-T SG8CCIC
の共同グループとして1986年に設立され，多値静止画像（連続階
調画像）を扱う圧縮方式を制定した．
JPEGは，DCT（discrete cosine transform，離散コサイン変
換）を用いる非可逆圧縮方式と，DPCM（diﬀerential pulse code
modulation，差分パルス符号変調）を用いる可逆圧縮方式をもつ．
(ａ)非可逆圧縮方式は，DCTにより，10：1から20：1の高
圧縮率が可能である．非可逆圧縮方式は，さらに，ベースラインプ
ロセスと拡張DCTプロセスの二つに分類される．ベースラインプ
ロセスは，DCT方式をサポートする最小限の機能であり，拡張
DCT方式は，大まかな画像を表示し，その後，その画像を逐次的
に向上させるプログレッシブビルドアップ方式や，算術符号化の採
用による付加機能を提供する．
(ｂ)可逆圧縮方式はDCTに代わりDPCMを採用したことに
より，完全に原画像を再現することができるが，圧縮率は2：1程
度である．
(２) JPEG-2000
圧縮方式を，JPEGが用いるDCTからDWT（discrete wave-
let transform，離散ウェーブレット変換）に変更し，ブロック単
位ではなく画面全体に対して処理を行うため，低ビットレート時の
ブロックノイズが軽減されるほか，選択領域の画質を向上させる
ROI （region of interest ）機能や伝送誤りへの耐性，複数のカラー
モデルへの対応，セキュリティ情報の付与など，従来のJPEGに
ない拡張機能が盛込まれている．
(３) DCT（離散コサイン変換）
DCTは同一画面上での画素の相関関係を基に数学的な処理を施
して，空間方向に対する冗長度を削減する．画像のように相関性が
高い信号は，これを周波数成分に変換したとき，低周波成分に電力
が集中し，また低周波成分が視覚的にも重要な役割を果たすので，
この性質を利用して圧縮を行う．
実際の画像では，水平方向と垂直方向の両方向に相関が存在する
ために，通常，二次元DCTが用いられる．すなわち，複数の画素
（例えば8×8画素）をまとめたブロック単位にDCT演算を行い，
画素の細かさや粗さを表す64個のDCT係数を得る．これらの
DCT係数は平均的な明るさを表す直流成分と，画像の変化を表す
低周波数成分と高周波成分を示す．低周波数成分は画像の大まかな
形などの情報を含み，高周波成分は細かい変化に関わる情報を含ん
でいる．画像の統計的性質から，DCT係数のパワーは直流成分と
低周波成分に集中する．一方，人間の目は細かい情報に鈍感な性質
をもっている．そこで量子化の段階で重みを付けて，細かい変化を
表す高周波成分を粗く量子化することによりデータ量を削減するこ
とができる．
b.動画像圧縮方式 動画像圧縮方式には多くの種類がある
が，ここでは用途別に，テレビ会議・テレビ電話用圧縮方式，蓄積
メディア用圧縮方式を紹介する．
.テレビ会議・テレビ電話用圧縮方式
(１)H.261
1990年末にITU-T（旧CCITT）によって国際標準方式として
勧告された．テレビ信号を，64kbps ～1.92Mbpsの比較的低い画
像ビットレートに圧縮する．圧縮アルゴリズムとして動き補償フレ
ーム間予測とDCTを用いるハイブリッド圧縮方式である．まず
16×16画素のブロック単位（これをマクロブロックという）で動
き補償予測を行い，次にその予測誤差を8×8画素のブロックサイ
ズでDCT変換を行う．その後に量子化および可変長符号化を行
う．入力画像フォーマットとして共通中間フォーマット（CIF:
common intermediate format ）を用いる．CIFの解像度は水平
352×垂直288画素である．64kbpsなどの低速での利用（おもに
テレビ電話）を考慮し，水平，垂直ともに半分の画素のQCIF
（quarter common intermediate format ：176×144画素）も定め
られている．
(２)H.263
H.263は，H. 261の改良版であり，H.261は挟帯域ISDN（64
kbps ）への対応を目的としているのに対し，H. 263はアナログ電
話モデムのような低ビットレートの通信回線（28.8～33.6kbps ）
への対応を目的としている．圧縮アルゴリズムはH.261を基にし
ており，半画素精度動き補償の採用，可変長符号化の改善，オーバ
ヘッド情報の削減が行われている．H.263は，H.261と同じく
CIFを採用し，さらに，16CIF（1152×1408画素），4CIF
（576×704画素），CIF，QCIF，sub-QCIF（96×128画素）をサポ
ートしている．
.蓄積メディア用圧縮方式
(１)MPEG（エムペグ）
MPEGとは，正式にはISO/ IEC JTC1/SC29/WG11のこと
で，Moving Picture Experts Groupの略称である．MPEGの規格
には，MPEG-1，MPEG-2，MPEG-4がある．
MPEG-1は，転送速度が1.5Mbps程度で，おもにCD-ROM
などの蓄積メディアを対象としている．MPEG-2は転送速度が数
Mbpsから数十Mbpsで，テレビ信号に用いられるインタレース信
号を含めて，テレビ映像などHDTVを含む高品質の符号化映像を
対象としている．また，MPEG-4は，おもに移動体通信，アナロ
グ電話網での動画通信などを対象としている．
(ａ)MPEG-1 MPEG-1は，H. 261とJPEG方式をベース
にしながら，蓄積メディアの特質を考慮して改良を加えている．例
えば，蓄積メディアの再生には早送り，途中からの再生，逆再生な
どの機能が必要であり，こういったランダムアクセスはH.261の
ように順方向予測を行う方式では実現できない．そこでMPEG-1
では，複数の画面データを一つのまとまりとして扱うGOP
（group of pictures ）構造を採用している．
GOP構造の先頭には，GOPヘッダがあり，これを検出すること
でランダムアクセスが可能となる．GOPは，少なくとも1枚の独
立して復号できる画像データをもち，その画像データから復号を開
始することにより，GOP内の他の画像データを復号する．また，
過去の画像からの順方向予測だけでなく，未来の画像からの逆方向
予測，および，順方向予測と逆方向予測の加算である両方向予測を
採用し，圧縮効率を高めている．
(ｂ)MPEG-2 MPEG-2は，通信や蓄積メディアに限られ
ていた動画像圧縮方式H.261やMPEG-1を，放送，通信，蓄積な
どのあらゆる分野へ適用できるように発展させた圧縮方式である．
ノンインタレース（順次走査）だけでなくインタレース（飛び越し
走査）のテレビ信号をサポートし，スケーラビリティをもつなど，
多くの特徴を備えている．MPEG-2では，テレビ信号を高品質に
保ちながら圧縮効率を上げるためにフレーム間/フィールド間符号
化が採用されている．
(ｃ) MPEG-4 MPEG-4は，インタネットや携帯電話など
のモバイルを介したAV通信，遠隔監視，AVデータベース検索，
放送さらにはビデオゲーム等への応用が考えられており，5kbps
～4Mbpsの幅広いビットレートが適用領域とされている．
MPEG-4の目的は，1)AVオブジェクト単位のアクセスおよびイ
ンタラクティビティとスケーラビリティの実現，2)圧縮性能の向
上，3)誤り耐性および回復力の強化，4)CGと自然画像の統合記
述環境の実現が挙げられる．
(２) 動き補償（motion compensation）
動画像の連続する二つのフレーム間での被写体の動きの方向と大
きさ（これを動きベクトルという）を検出し，動きベクトルで示さ
れる1フレーム前のブロック（16画素×16ライン）を予測値とし，
現フレームのブロックとの差（予測残差と呼ぶ）を伝送する方式で
ある．
復号側には，動きベクトル情報と予測誤差のみを送るだけで，1
フレーム前のブロックから現フレームのブロックを復号することが
できる．動きベクトル情報と予測誤差は，フレームを構成するブロ
ックの情報よりもはるかに小さいので，圧縮ができる．なお，動き
補償予測には，過去のフレームのみから予測する順方向予測と，時
間的に未来のフレームを用いて予測を行う逆方向予測がある．
(３) ハイブリッド符号化
動画像圧縮の国際標準であるH.261，H.263，MPEG-1，
MPEG-2，MPEG-4などに共通した画像圧縮方式である．前後す
る画面の違いのみを伝送する動き補償と，同一画面上での画素間の
相関を利用して，数学的な処理によって情報を減らすDCTの二つ
の技術を組合わせる．実際には，ハイブリッド符号化は，動き補
償，DCT，量子化およびエントロピー符号化を組合わせている．
DCTと量子化で1/10～1/20，動き補償で1/2，エントロピー符号
化で2/3～1/2の圧縮が可能である．したがって，これらを合計し
た圧縮率は1/30～1/80程度となる．
c.その他の映像圧縮方式
.ベクトル量子化（vector quantization) 画像を構成す
る画素値をいくつかまとめて，ベクトルとみなす，もしくは，ある
画素の輝度，色差成分等のいくつかのデータをまとめてベクトルと
みなし，そのベクトルを量子化することにより，単なるスカラ量子
化に比べて量子化誤差の少ない量子化が可能になる．これをベクト
ル量子化と呼ぶ．音声やオーディオの圧縮では広く使われており，
画像圧縮への適用も試みられている．
.フラクタル符号化（fractal coding) 画像の自己相似
性を利用する圧縮方式である．自己相似性とは，画像の任意の部分
を拡大すると拡大前の画像と同じになることをいう．圧縮では，輝
度変換，幾何学変換などの縮小写像により，画像のある局所領域を
これとよく似た周辺領域から探し出し，量子化を行う．フラクタル
符号化の特徴は，高い圧縮率と，さまざまな解像度をもつ復号画像
を生成できる点にある．
.ブロック・トランケーション符号化（block truncation
coding) 画像を，N×M画素の小ブロックに分割し，その後，
ブロック内の各画素の画素値をある一つの画素値に近似する圧縮で
ある．この圧縮方式は，画像が周りの画素との相関が高いことを利
用した圧縮方式である．
