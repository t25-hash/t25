b.データ圧縮の原理 データ圧縮とは，記号列を受取り，そ
れを符号に変換することである．その際，頻繁に現れる記号列を短
い符号で表し，めったに現れない記号列を長い符号で表す．これが
データ圧縮の原理である．
効果的なデータ圧縮が行われたときには，得られた符号列が元の
記号列よりも小さくなる．
c .データ圧縮の方法 データ圧縮は，モデルと符号化とから
なる．記号または記号列は，モデルに基づいて符号化する．モデル
とは，入力記号または記号列を処理してどの符号を出力するかを決
める規則集合のことで，統計的モデルと辞書モデルとがある．符号
化とは，モデルに基づいて符号を出力することである．統計的モデ
ルでは，ハフマン符号（Huﬀman coding）とシャノン・ファノ符
号（Shannon- Fano coding），算術符号（arithmetic coding），LZ
符号（Lembel - Ziv coding）などが使われる．辞書モデルでは，後
ほどd項で具体的に触れるとおり単純な符号化が行われている．
データを圧縮するには，入力記号を各記号がもっている情報量と
同じビット数で符号化する必要がある．データ圧縮という観点から
いって，英文テキストをASCII （アスキー）コードで符号化する
ということは，最適な手法ではない．
最も単純な静的統計モデルでは，出現確率を固定的な表にして使
用する．入力される記号列に対してこの表による統計情報が合わな
いとき，圧縮率は悪くなる．入力記号列を処理しながら出現確率の
表を適応するように修正する適応的統計モデルもある．
辞書モデルでは，入力記号列を読み込み，辞書に現れる記号列を
探し，合致があればその記号列に対する符号を出力する代わりに辞
書に対するポインタを出力する単純な符号化が行われている．入力
記号列の性質が途中で変わらないならば，固定的な辞書を使うのが
単純であるが，圧縮した記号列と一緒に辞書も送らねばならない．
ただしLZ77やLZ78による辞書モデルでは，移動窓が使われて
いるので辞書を送る必要がない．例えば学術論文など，略称による
語句の置換えは，読者が頭の中で辞書を使って理解しているのであ
る．
