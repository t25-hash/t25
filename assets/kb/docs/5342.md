# 5・5・1 アミューズメントロボット(amusement robot)

> **章**: 第5章 ロボティクスの応用展開
> **節**: 5・5 アミューズメント・セラピーロボット

ロボットは従来からアミューズメント目的にも用いられてきたが、近年の小型化、二足歩行などの制御技術、センサ処理やそれに基づく知能化技術により、人間の行動に反応してさまざまな動きや発話、時にはダンスなどを行うことで人と相互作用し、人々を楽しませるアミューズメントロボットという新しいロボットが開発されるようになってきた。このようなロボットは、物を運ぶといった物理的作業を目的としないという点で他のロボットと大きく異なる。アミューズメントロボットの定義を考えると、広義にはアミューズメントに利用可能なロボットをすべて含むと考えられるため、非常に広い範囲のロボットが該当する。例えば、歩く、踊るといったふるまい(ビヘービア、behavior)をするロボットは、ロボットの目新しさを利用して人々を楽しませるという意味でアミューズメントロボットといえる。ごく少数のアクチュエータを内蔵したぬいぐるみロボットや遊園地などで自動的に動く人型ロボット、組立てて遊ぶようなロボットの玩具ももちろんアミューズメント性をもつ。部品組立用のロボットアームといった物理的作業を目的とするロボットさえも、博覧会やパーソナルロボットの展示会で繰返し部品を素早く移動させるようすは人々を楽しませている。しかし、単に機械的な目新しさからアミューズメント性をもつロボットに関してではなく、ここではむしろ狭義のアミューズメントロボットとして、おもに動物や人間の形に似せて作られ、自律的に知的に行動し、多様なふるまいのコンテンツをもつことで、アミューズメント目的等の人との相互作用を主目的とするロボットについて、その要素技術を示す。相互作用を主目的とするロボットは近年盛んに研究されるようになってきたが、いまだ新しい研究・技術分野であり、さまざまな可能性が模索されている段階にある。将来的には、相互作用を主体とするロボットは人間のパートナーとして日常生活の場で人間と共存し、人間を支援する活動をするようになっていくと考えられている。例えば、音声コミュニケーションも交えて、高齢者(elder-ly person)の話し相手や街中での情報提供など、人間との相互作用機能を主体として多様なサービスを提供する可能性がある。アミューズメントロボットは、このような相互作用を通じて人間の活動を支援するようなロボットの、現在における実用化・商用化の一形態であるといえる。a. アミューズメントロボットの種類このようなアミューズメントロボットは、自律性の有無、外見の擬人性によって大きく4タイプに分類することができる(図5・46)。(1) トイロボット(toy robot)トイロボットはアミューズメントロボットの中でも最も安価で普及しているものである。動物の形をしているものが多いが、二足で立つ人間型のものもある。簡単な外界刺激に反応することはできるが、自律性は低い。例えば、なでると鳴き声をあげるぬいぐるみや、発話に合わせてうなずくロボットがこれに分類される。後述の3タイプのロボットの一部の機能をもつ廉価版であるといえる。(2) パフォーマンスロボット(performance robot)擬人性が高いことを利用して、歩行やダンスの機能を見せることで人々を楽しませるロボットで、あまり自律性の高くないものである。例えば、初期の本田技研工業(株)のアシモやソニー(株)の図5・46 さまざまなアミューズメントロボットSDR(後のQRIO)といった二足歩行ロボットがこれに分類される。(3) ペットロボット(pet robot)外見は動物に似せたものが多い。しっぽを振る、音を出すなどの表現により、人間からの働きかけに自律的に反応し、また自動的にさまざまな表現をするといった方法によって人間との相互作用をするロボットである。例えば、ソニー(株)のアイボなどがこれに該当する。ペット動物を飼っている場合と同様に、生き物のように自律的にふるまうロボットとかかわることで人々を楽しませる。また、精神的な癒し効果などをもたらす場合もある。(4) ソーシャルロボット(social robot)相互作用により人と長期的・社会的な関係を築き、人間の日常生活を幅広く支援するようなロボットのことである。擬人的な外見をもつものが多く、また高い自律的な相互作用能力が特徴である。コミュニケーションロボット(communication robot)、パートナーロボット(partner robot)とも呼ばれることがある。アミューズメントにとどまらず、コミュニケーションを通じたさまざまなタスクが期待されている。商品として実用化されているものはまだないが、研究開発が進められており、プロトタイプとして例えば本田技研工業(株)のアシモ、ソニー(株)のQRIO(キュリオ)、ATR のRobovie(ロボビー)などのロボットが発表されている。家庭や個人での所有が期待されているパーソナルロボット(personalrobot)もこれと同様のコンセプトによるものである。b. ヒューマンインタフェースアミューズメントロボットのインタフェースは他の機械装置などと大きく異なる。通常の機械装置では、順番にボタンを押すといったように、人は一定の取決めに従った操作を行う。また、操作に対して機械装置が完全に正しい反応を行うことが求められる。これに対して、多くのアミューズメントロボットはユーザの入力をほとんど、あるいは全く取り決めない。また入力手段も、ロボットが適切に反応できるかどうかはともかくとして、音声、身振り手振り、ふれあいといったさまざまな方法が可能である。これは、むしろ人間どうしのコミュニケーションや人間が動物に働きかける場合に近い。また、人間側がロボットに期待することも、指示に対する完全に正しい反応というよりはむしろ、時に間違えるのも愛嬌のうちとして、人間の反応になにかしらリアクティブに反応し、あるいはロボットがアクティブに何か行動をしてみせることでロボットの知的なふるまいを見せ、言葉や非言語のやりとりによる相互作用を行うことにある。c. アクチュエータ、表現系自由度は、犬型のペットロボットや上半身のみが人間型のロボットで20程度、二足歩行人間型のもので40程度である。特にペットロボットやソーシャルロボットでは高速な動作よりも、柔らかい制御、安全性、小型であることが重視される。手先などの可動範囲よりも、むしろ表現能力の点で動物や人間に近いかどうかという視点で設計されることが多い。これに加えて、LED(light emitting diode)などにより喜怒哀楽の顔表情を作り出すものもある。ロボットの内部には金属製の部品が使われても、表面にはプラスチック、シリコーンゴム、布、毛皮などを利用することで柔らかさや暖かさを実現するものも多い。d. センサアミューズメントロボットの内界センサにはポテンショメータ、ジャイロなどが用いられるが、むしろ特徴的なのは外界センサである。アミューズメントロボットには、人間からの働きかけに対して反応するための多様な外界センサが搭載される。利用されることが多い外界センサの種類と用途について以下に示す。(1) 視覚人の位置、動き、ジェスチャ、顔の表情といった情報の取得のために、CCD カメラが多くのアミューズメントロボットに搭載されている。たいていの場合、ロボットの眼に該当する位置に1~2台取付けられる。これに加えて、より視野の広い魚眼カメラや全方位カメラも用いられる場合がある。7―185(2) 聴覚マイクロホンも多くのアミューズメントロボットの耳の位置に搭載され、音声認識に用いられている。やや高価であるが、多数の同期入力可能なマイクロホンから構成されるマイクロホンアレイが胴体部などに搭載される場合もある。マイクロホンアレイは、音源方向の検出や音源分離による雑音の低減に利用される。(3) 触覚人とふれあう状況を想定したロボットでは、ロボットをなでる、たたくといった入力が期待される。そのため、多くのペットロボットと一部のソーシャルロボットでは触覚センサが利用されている。単純なプッシュスイッチ式のもののほか、圧式のセンサを利用するものもある。e. 知能化アルゴリズム特にペットロボットやソーシャルロボットに該当するアミューズメントロボットにとって、人間との相互作用機能は最も本質的な機能である。そのためには、人間の働きかけに応じた知的に見えるふるまい(ビヘービア)が重要である。アーキテクチャは人間の動作に反応するといった短時間の知能化に関連し、いかにビヘービアを遷移させるかといった行動制御が重要となる。(1) アーキテクチャ(architecture)アーキテクチャはロボットの知的なふるまいの基本設計である。アミューズメントロボットに関しては、人間の行動といった非常に動的な要素に反応するために、高い反射行動能力が必要とされ、多くの場合、サブサンプション(subsumption)アーキテクチャに代表されるような行動ベースド(behavior based)制御(4・7・3項)を基本としたアーキテクチャが採用される。(2) ビヘービアの遷移と行動制御多くのアミューズメントロボットは、インタラクティブなふるまいを実現する要素行動としてビヘービアを用いている。ビヘービアは、しっぽを振る、ボールをける、お辞儀をする、握手をするといったロボットの一つの具体的なふるまいに対応する。そして、このビヘービアを逐次的に切り換えるような制御によって知的なふるまいや相互作用が実現されている。ビヘービアの遷移に当たっては状況依存性の利用が重要になる。ロボットの多くのビヘービアは特定の状況下では適切に実行できるが、すべての状況において実行できるものではない。また逆に、特定の状況を仮定することで効率的で信頼性の高いセンシングが可能になる。さらに、ロボットがビヘービアを行うことによって状況が限定される。例えば、人間型ロボットが人間と握手をするというビヘービアを例にとると、ロボットが人に握手を求めるという状況は、人がロボットの前方の一定距離にいる状況であると考えられる。また、この状況で人がロボットの手先に触れると、それは握手が成立したと考えることができる。しかし、一般的な状況でロボットの手先が触れられたからといって握手が行われたと認識することはできない。このように、ロボットが現在の状況を利用し、また自ら状況を作り出すことで、状況に依存した期待待受け型のセンシングを行い、一般的な状況では困難な人間行動認識を実現することができる。ビヘービアの遷移に関しては状態遷移モデルやその拡張形式が利用されることが多い。遷移は、本来は行動のプランニングに関する問題であるが、人との相互作用を行うアミューズメントロボットにとって、相手となる人間の将来的な行動を予測することは非常に困難であり、人工知能分野での二人ゲーム問題といったプランニングが適用可能な状態にはなかなかならない。むしろ、ロボット自らの過去の行動や状態と照らし合わせて一見矛盾しないような行動を行うための状態遷移モデルや、ルールベースの制約メカニズムによって遷移行動が設計される。f. 相互作用の設計モデル数分から数時間程度のロボットのビヘービアの遷移は、上記の状態遷移モデルやルールによって書き表すことが可能だが、より長期的にロボットの行動を変化させ、ビ7―186メカトロニクス・ロボティクスヘービア遷移のバラエティを作り出すことも、ロボットと相互作用する人間を飽きさせないために重要である。このようなロボットのふるまいの決定は、ややアートに近い部分もあるが、どのような相互作用のモデルを設計ポリシーに採用するかは工学の領域であろう。以下に、アミューズメントロボットに用いられることの多い相互作用の設計モデルについて解説するが、複数のモデルが同時に用いられることも多い。(1) 行動モデルロボットの行動が人間に直感的に理解できるように、しばしば実際の動物や人間の行動をモデルとしてロボットの行動が構成される。ペット型ロボットの実現に際しては、動物の典型的な行動パターンを見いだす動物行動学(ethology)の手法が利用されることがある。例えば、アイボは動物行動学により解明されたイヌの典型的な行動(周囲の探索、摂食、休息、遊びなど)をビヘービアとすることにより、イヌと類似したふるまいをするように設計されている。(2) 感情モデル感情(emotion)とは、喜怒哀楽といった人間の気持ちや心的な内部状態のことを指す。人間どうしのコミュニケーションの際には、顔の表情や発話の韻律、ニュアンスなどに感情が表出する。アミューズメントロボットでも、たたかれると怒る、なでられると喜ぶなど感情モデルに沿ったビヘービアをもたせ、人間や動物と似た反応を返すことで、実際の生き物のように見せるために感情モデルが利用されている。また、その場での直接的な反応のみでなく、例えば、なでられるとしばらくは機嫌よく歩き回るというように、感情モデルに関する内部パラメータを変化させて状態遷移モデルの遷移を制御する手法が用いられる。ソニー(株)のアイボ、オムロン(株)のNeCoRo(ネコロ)、日本電気(株)のPaPeRo(パペロ)など、多くのロボットに感情モデルが実装されている。類似のモデルとして、食欲や睡眠欲といった欲求をモデルに取り入れることで、たくさん歩き回ると眠くなって休む行動が頻出するようになるといった欲求モデルによる状態遷移モデルの遷移制御が行われることもある。(3) 学習モデル長期の相互作用で重要となるのが学習や適応のメカニズムである。自分にだけなついている、だんだん言葉を話すようになった、お手を教えたらお手を覚えたなどのように、ロボットが何か新しいことを覚え、成長することは、人間の興味を引き続けるために有効な手法である。実際には、このために見掛け上の学習が行われる場合が多い。すなわち、ロボットは実は多くのビヘービアや機能があらかじめ実装されているが、初めのうちはこれらのビヘービアや機能を表出させず、学習や成長のトリガが生じた場合にこれらのビヘービアを新たに表出させることで、あたかもロボットが成長し、新しいことを学んだかのようにふるまわせるのである。
