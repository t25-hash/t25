b.公開鍵暗号 公開鍵暗号（public key cryptosystem）は，
非対称鍵暗号（asymmetric key cryptosystem）とも呼ばれ，暗号
化する際に使用する鍵と，復号化する際に使用する鍵が異なる特徴
を有し，一般に処理負荷が大きい．公開鍵暗号で使用される鍵のう
ち，広く公開する鍵を公開鍵（public key），利用者が秘密裏に保
持する鍵を秘密鍵（private key，secret key）と呼ぶ（共通鍵暗
号における鍵を秘密鍵と呼ぶ場合もあるが，公開鍵暗号における秘
密鍵と混同する場合があるため，前者は共通鍵や対称鍵と呼ぶほう
がよい）．
公開鍵暗号の代表的な暗号方式としては，RSA（Rivest Shamir
Adleman）暗号とECC（elliptic curve cryptosystem）がある．
RSA暗号は1980年代から今日まで使われ続けてきたが，高速素因
数分解法（おもに数体ふるい法）の進歩によりその安全性が危ぶま
れ，徐々に鍵長が長くなってきている．実際，当初は512ビットの
鍵長が使われていたが，2000年に入って1024ビット，2048ビッ
トの鍵長が使われ始めている．鍵長の伸長に伴い，演算速度の低下
やハードウェア化の困難さが指摘され，他の暗号方式への移行が模
索されている．
一方ECCは，歴史的には1985年にコブリッツ（Koblitz）とミ
ラー（Miller ）により独立して提案されていたものの，実用化され
始めたのは1990年代ごろからであり，その意味で実績は少ない．
しかしRSA暗号の1024ビットに対し，ECCの160ビットが同程
度の強度を有するといわれているように，ハードウェア化しやす
く，高速に演算できることから，今後大いに使われていくものと考
えられる．
公開鍵暗号の特徴は，暗号鍵と復号鍵が異なることである．この
ことを利用し，暗号鍵として公開鍵を，復号鍵として秘密鍵を用
い，メッセージを特定の人にだけ安全に送ることができる．例え
ば，Aさんが公開鍵，秘密鍵のペアをもっていて，BさんがAさ
んにメッセージを秘密裏に送る場合，Aさんの公開鍵でメッセー
ジを暗号化してAさんに送信する．すると，このメッセージを復
号化できるのはAさんだけなので，親展としてメッセージを送る
ことができたことになる（秘密鍵はAさんしか知り得ないため）．
なお，公開鍵暗号を用いてメッセージ全体を暗号化するのは非効率
であるため，共通鍵でメッセージを暗号化し，この共通鍵を公開鍵
で暗号化して送信するほうがよい．
