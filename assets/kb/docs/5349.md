a.攻撃技術
.コンピュータウイルス（computer virus ) コンピュー
タウイルスは，正常なシステムの動作を意図的に妨げる目的で作成
された不正動作を行うプログラムであり，他のコンピュータに入り
込むことを感染と呼ぶ．コンピュータウイルスにはさまざまな種類
が存在するが，他のファイルにコンピュータウイルス自身を寄生さ
せる機能や，一定の条件が成立すると悪質な行為を始める機能，プ
ログラム，データ等のファイルの破壊を行ったり，設計者の意図し
ない動作をする機能などを備えることが多い．
初期のウイルスは画面にメッセージを表示させたり，勝手に画面
を書き換えたりするといった簡単なものであったが，やがてデータ
を破壊する悪質なものが登場した．また感染は，フロッピーディス
クなどのメディア経由から次第にネットワーク経由に移っている．
ネットワークには多くのコンピュータが接続されているため，感染
が爆発的に起こりやすい．ネットワーク経由の場合には，電子メー
ルやWWW（world wide web）の仕組みを用いて感染することが
多い．
なお，コンピュータウイルスは，自分のプログラミング能力を誇
示したり，世間の騒ぎを見て楽しんだりするために作成されること
が多いと考えられている．
以下ではコンピュータウイルスの代表的な種類について述べる．
(１)ファイル感染型
アプリケーションなどの実行型ファイルに感染するウイルス．感
染したファイルを実行するとウイルスも動作を開始する．
(２)システム領域感染型
パソコンのOSなどシステム領域に感染するウイルス．ユーザが
明示的にファイルを実行しなくても動作を始めることがある．
(３)マクロウイルス（macro virus ）
文書作成や表計算などのアプリケーションで作成するデータファ
イルに埋め込まれるマクロの仕組みを悪用したウイルス．マクロは
文書の作成などにおいて，作業を自動化するための簡易プログラム
である．同一のアプリケーションソフトであれば，異なるOS上で
も動作できるため，複数のOSにまたがって感染できてしまう．実
行型ファイルに比べて，企業等の日常業務の中では文書を交換する
機会は多く，その文書ファイルの中にマクロウイルスが潜んでいる
と爆発的に広がりやすい．また仕組みが簡単であることから，それ
ほどウイルスの知識やノウハウがなくても改造できてしまうため，
興味本位で亜種が再配布されることも多い．

b.防御技術
.データの暗号化・署名 ネットワーク上での通信を安全
かつ正当に行うためには，以下を保証することが必要である．
(１)秘匿：第三者に内容を知られないこと
なお，セキュリティホールとはハードウェアやソフトウェアに存
(２)認証：相手が正当な受け手だと確認すること
(３)完全性：内容の改ざんが行われないこと
(４)否認防止：内容を後に否定できないこと
これらを実現するために，データに対して暗号化や電子署名を施
す方法がある．文書ファイルなどの一般的なファイルの暗号化や署
名付けを行うためにPGP（pretty good privacy）など数多くのソ
ネットワークに接続されているシステムにセキュリティホールが
フトウェアが開発されている．PGPはジマーマン（Zimmerman,
P. ）がフリーソフトウェアとして開発した暗号化ツールであり，現
在では商用版も発売されている．
.電子メールの暗号化・認証 インタネットにおいて頻繁
に利用されるアプリケーションである電子メールについてもプライ
バシーの保護は非常に重要であるため，暗号化や認証の枠組みが検
討されている．代表的な方式は以下の2方式である．
(１)S/ MIME（secure /multipurpose internet mail exten-
sions ) S/ MIMEはRSAラボラトリーズが策定したPKCSに
よって提供される暗号化技術を，電子メールでさまざまな文字コー
ドやデータ型を取扱うためのMIMEに適用した方式である．各利
用者の公開鍵の正当性を保証するためにCAが発行する公開鍵証明
書を用いる．
一般的なコンピュータシステムにおいては，アクセス権限の認証
(２)PGP 先に述べたPGPは電子メールの暗号化や署名の
機能を備えている．公開鍵の認証は，利用者が相互に信頼を与え合
MIMEも利用されている．
.通信路の暗号化・認証 アプリケーションソフトウェア
が個別にプライバシの保護への対策を行うのではなく，通信路を暗
号化する技術としては，SSL（secure socket layer ）あるいは
TLS（transport layer security）が挙げられる．これらは通信路
の暗号化のほか，サーバの正当性認証，クライアントの正当性認証
の機能を備える．SSLはネットスケープコミュニケーションズ社
が開発し，自社のWWWブラウザに組込んだことから広く普及し
た．主要なWWWブラウザ，WWWサーバをはじめ多くのソフ
トウェアが対応している．SSLの最新バージョンは3.0であるが，
IETFではこれを基に若干改良を加えてTLSとして標準化（RFC
2246：1999）を行った．
SSL/ TLSの特徴は次のとおりである．
(１) OSI参照モデルにおけるセッション層のプロトコルであ
るため，HTTPに限らずFTP，Telnet等の上位層を用いるアプ
リケーションは，特に意識することなく透過的に利用できる．
(２) さまざまな共通鍵暗号方式や公開鍵暗号方式，ハッシュ関
数を組合わせることが可能であり，公開鍵証明書を用いた認証にも
対応している．
暗号化の処理は一般的に負荷が高いため，通常のサーバで処理を
行うと性能面でのボトルネックとなってしまう．そこで暗号化処理
部分を担うSSLアクセラレータと呼ばれる専用ハードウェアが開
発されている．
また，通信路の暗号化および相手認証のプロトコルとして，
IPSec(IP security)が挙げられる．IPSecはRFC2401：1998
～2408：1998，2411：1998，2412：1998で規定されている．IPSec
の特徴は，SSL/ TLSでは暗号化できないUDPパケットも含めた
あらゆる通信をIPレベルで暗号化できるというところにある．
IPSecにおいては利用する暗号方式や暗号鍵等を通信相手との間で
交渉して決定する．
なお，IPSecはIPバージョン4ではオプションであるが，IPバ
ージョン6では必須となっている．
.ファイアウォール（firewall ) 組織でのセキュリティ
を守るためには，各コンピュータおよびネットワーク機器に対して
対策を施すのが理想である．しかし，管理対象のコンピュータ数が
多くなってしまうことや，利便性が損なわれる場合が多いことか
ら，実現は困難であることが多い．そのため，組織内ネットワーク
と外部ネットワークとの境界にセキュリティ機能を高めたファイア
ウォールと呼ばれるハードウェアおよびソフトウェアを設置して内
部のコンピュータ等を保護するという考え方が普及した．
ファイアウォールの基本的な機能はパケットフィルタリング
（packet filtering）である．ネットワークを流れるパケットのヘッ
ダにはプロトコルや送信元アドレス，送信先アドレスやポート番号
などの情報が含まれており，ファイアウォールはパケットを監視し
てこれらの情報を検査し，あらかじめ設定しておいた方針に基づい
てパケットを通過させるかどうか決定する．外部からの不正なアク
セスを排除するために必要なパケットだけを外部から内部に通過さ
せるのである．通過できなかったパケットは送信元に通知された
り，破棄されたりする．管理者はパケットを通過させる方針を決定
することができるが，つねに最新の情勢を踏まえた方針にすること
が肝要である．
不正アクセスに対してより効率的に対処するために，通信回線を
監視し，組織内ネットワークへの侵入者を検知して管理者に通報す
る侵入者検知システム（intruder detection system）を導入する場
合もある．このシステムはネットワーク上を流れるパケットを分析
し，パターン照合により不正アクセスと思われるパケットを検出し
た場合には管理者に通知したり，疑わしい通信を切断したりするな
どの措置を行う．
パケットフィルタリングは最も一般的かつ簡単なセキュリティ技
術であるが，パケットのヘッダだけを検査しているため，ヘッダが
正当なパケットであるように偽造された場合には無力である．その
ため，ファイアウォールにおいては，さらにアプリケーションゲー
トウェイ（application gateway）やプロキシ（proxy）によるア
プリケーションレベルでのアクセス制御などの技術を併用すること
が重要である．
あるが，組織の内部と外部との通信をすべて処理することになって
負荷が非常に高くなるため，専用のハードウェアを組合わせる場合
もある．
