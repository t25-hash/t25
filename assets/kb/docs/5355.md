# 6・5・1 インテリジェント(ロボティック)ルーム

> **章**: 第6章 メカトロニクスの応用展開
> **節**: 6・5 ネットワークメカトロニクス

a. 定義ロボットをわれわれの生活空間に導入する目的は,端的にいえば日常活動の支援である.その場合,ロボットに期待される支援の内容はさまざまである.日常使っている物品や書類などの配達といった典型的な運搬作業はもちろん,家電(家庭用電気製品)や住宅設備の操作,さらにインタネットが当たり前に利用されるようになっている現在では,情報伝達のようなこともロボットの役割の一つと考えられるようになっている.これらの支援機能は,機能だけを取出してみれば必ずしもロボットの形に依存しない.いわゆるヒューマノイドのような人間型である必要はなく,介助犬のような動物型でもかまわないし,だいたい手,足,眼,頭脳などのすべてを一つにまとめ上げた一体型,スタンドアローン型である必要もない.眼の機能を取出してみれば壁や天井に設置されたカメラ,その視覚情報処理を行うのは別の場所に設置された計算機,情報処理の結果に応じて物を運ぶのは天井のレールを走行するマニピュレータといった具合でもよい.このような考え方から生まれたのが,部屋や環境の中にロボットの機能要素を分散配置して,それらの協調的作業によって室内にいる人間が求める仕事を行い,支援を行ってくれるという空間型のロボットである.これはインテリジェントルーム(intelligent room),あるいはロボティックルーム(robotic room)と呼ばれることが多いが,その呼称についてはいまだ一定するものはない.国内ではロボティックルーム,インテリジェントルームなどの呼称が多く,外国ではスマートルーム(smart room)と呼ばれることもある.図6・129に一体型のロボットと空間型のロボットを模式的に比較したものを示す.図6・129(a)は一体型ロボットのイメージで,ロボットに想定されるすべての機能要素を一つの体に積み込んでいる.図6・129(b)は空間型ロボットのイメージで,ロボットの機能要素を部屋の中に分散的に配置し,ネットワークで統合している.空間型に分類されるインテリジェントルームやロボティックルームを構成する要素は,モータやセンサなど基本的にロボットを構成するさまざまな機能要素と同じものであるが,それらを空間的に分散配置したものが基本形である.もちろん,機能要素を単にばらばら7―227(a)(b)図6・129一体型ロボットと空間型ロボットのイメージ比較においただけでは意味がなく,分散配置された機能要素がお互いの役割を適切に果たし,統合的な効果を発揮することで初めてこの空間がロボットとみなせるようになる.すなわちロボティックルームの実現にはこれらロボット的機能要素を統合するための手段も重要となる.一体型ロボットの場合,図6・129(a)のように,要素の統合はロボットの体という単なる入れ物への実装にすぎず,当該機能要素が体のほとんどを占める.また,ひとたび実装された機能要素を交換することは,故障による以外はほとんどない.一方,ロボティックルームの場合,図6・129(b)のように埋め込む機能要素の総量に対し,空間の広さは圧倒的に広い.また,埋め込む機能要素もさまざまであり,追加・変更もたびたび行われる.すなわち,ロボティックルームの場合,従来の一体型ロボットと実装手法はまったく異なったものになる.具体的には,空間の自由な位置に機能要素を配置できるようになったが,その一方で,それらの間の密な連携が達成できるような仕組みも必要であり,これには最近進歩の目覚しいコンピュータネットワークが利用されるようになってきた.また,部屋の中にいる人にどのようなサービスを提供するかというソフトウェア,コンテンツも重要な構成要素である.b. ロボティック病室ロボティックルームの先駆的な開発事例として東京大学の佐藤・森研究室のロボティック病室がある.1996年ごろから始められたプロジェクトで,いくつかの実現バージョンがある.図6・130はバージョン1と呼ばれているもので,患者の状況をモニタリングするためのセンサとしてここに埋め込まれている機能要素は,天井に設置されたカメラ群,ベッドの台に並べられた圧力センサ群,床に埋め込まれたセンサ群,壁のレールに沿って動くマニピュレータ,患者に情報を伝えるペットロボットなどである.ロボティックルームにはもともと決まりきった形はないし,構成要素の組合せ方にも決まりはない.利用者の目的に合わせて必要な機能を適当に組合わせて環境として整えられるところにポイントがある.この事例では,患者の行動をモニタすることで簡単な物取りなどの介助を行ったり,ベッドセンサなどによる体調チェックから必要に応じて看護師や医師に連絡することなどを想定している.簡単な物取り介助というのは,ティッシュペーパーやテレビのリモー7―228メカトロニクス・ロボティクストコントローラなど,簡単な仕事であればわざわざ看護師を呼ばなくてもよいように利用されることを想定している.ペットロボットはロボティックルームの作動状態のインジケータとしての役割をはじめ,ロボット(人工物システム)と人との直感的コミュニケーションの手段として用いられている.機能構成は図る.(1) ベッド上の患者のモニタリング:ベッドの布団の下に221枚の圧力センサがマトリックス状に並べられ,患者が寝ているときの圧力分布を解析することで,あおむけ,左向き,右向き,うつ伏せなどの体位,さらに圧力分布の変動を解析することで呼吸の状態もリアルタイムで計測できる.(2) 室内の人の検出と追跡:天井に据え付けられたカメラ群と床に埋め込まれた圧力センサ群で構成されている.これは,室内における人の居場所を検出・追跡するためのものである.天井カメラはまた,人の顔を検出し,三次元的な高さも計測できるようになっている.(3) 指差しと被指示物体の推定:天井カメラにより患者の両手をモニタしてジェスチャ(gesture)の解析を行う.特に指差し行為を検出した場合にはその指し示している方向を計算して,その先にある物体を,患者が取ってほしいものとして推定する.バージョン2のシステムでは,これらのハードウェアの上で計測された人の行動を蓄積し,そこからより高度な知的サービスの機能図6・130 ロボティック病室例図6・131 ロボティック病室の機能ダイアグラムが研究されている.これはインテリジェントルーム内で活動を行う人々の行動履歴から個々の人の活動パターンに応じたサービスをさまざまに展開しようというものであり,コンテンツサービスの新しい姿として着目されている.c. RTミドルウェアを用いたシステム佐藤・森らのロボティックルームの提案以降,ロボット的な要素機器を実際の環境の各部に分散して埋め込むことで空間自体に機能をもたせようとする研究はさまざまな形で展開されてきた.一方で,それらの構成手法については特別な方法論がなく,ロボット的な要素機器をアドホックに結合して構築することが多かった.ロボット機能要素のモジュール化手法に関しては,森・佐藤らは分散オブジェクト(distributed object)を導入している.さらに産業技術総合研究所では,2002年度から2004年度に行われたNEDO(New Energy and Industrial Technology DevelopmentOrganization,新エネルギー・産業技術総合開発機構)のロボットの開発基盤となるソフトウェア上の基盤整備プロジェクトにおいて,ロボット的な要素機能を標準的な切り口を備えたモジュールとして整え,システム統合を柔軟かつ迅速に行えるようにするためのRT ミドルウェア(robot technology middleware)の研究開発を行った.RT ミドルウェア自体はソフトウェア工学における分散オブジェクト指向システムのミドルウェアであるCORBA(commonobject request broker architecture)をベースとして,ロボットを構成する機能要素をソフトウェア的にモジュール化するためのツールとして開発されたものである.要素モジュールを分散オブジェクト化することのメリットはいくつかあるが,その一つとして,ネットワークに接続するソフトウェアモジュールを,ネットワークの物理的な構造を考慮することなく機能的な接続を簡単に記述できることがある.そのため,ロボティックルームのように,空間のさまざまな場所に機能要素モジュールを配置し,しかもその変更を随時行うような状況に適している.RT ミドルウェアによりモジュール化されるロボット機能要素はRT コンポーネントと呼ばれ,その基本構造は図6・132のようになっている.コンポーネントの作動に直接かかわるデータの入出力チャネルである入力ポート(inport)と出力ポート(outport),およびコンポーネントそのものの制御などを行うためのコマンドを受けるインタフェース部とで構成される.例えばサーボモータであれ図6・132 RT コンポーネントの仕組み図6・133 RT コンポーネントを用いたロボットシステムの構築ば,その制御プログラムの外側にこの枠組がかぶさって,サーボモータを一般化したモデルとなる.入・出力ポートはサーボのデータの授受(目標値,現在値)を行う一方,モータのオンオフはコマンドインタフェースから行うようになっている.外側から見れば,まさに標準的なパラメータを通じて目標値と現在値をやりとりするだけのものとみなせるようになる.このように一般化された接続口をもったモジュールを,図6・133のようにネットワークに接続する.各RT コンポーネントはRT ミドルウェアにより付加される標準的なネットワークインタフェースをもっており,ネットワークに接続するだけでシステムの一部となる.システムの構築としては各RT コンポーネントの入出力チャネルを通じたデータのやりとりとして記述される.具体的には,あるモジュールの中で他のモジュールの入出力チャネルのパラメータを参照したり,パラメータに値を書き込むことを記述することが,モジュール間の接続を指定することに相当する.RT ミドルウェアという標準的なコネクタがあって,パラメータは個々の足のようなものである.標準化されたコネクタなので同等部品との交換が可能であり,より性能の高い部品,より安価な部品,ユーザのニーズ,好みに合ったそれぞれのロボティックルームの構築が容易になる.RT コンポーネント自体には階層的な関係はなく,相互の情報の参照の仕方,指令を送る方向等によってさまざまな階層関係が組まれることになる.例えば図6・133の中で,ロボットアプリケーションモジュールがこのシステム全体のサービスモデルを記述したものであれば,このモジュールがユーザの指令をいったん受け取り,その指令実行に必要な部分仕事を他のコンポーネントに割り振るなどのプランニングやスケジューリングを行うことになる.一方,中央集権的なモジュールをおかずに自律分散的なモジュール群のシステム構成も可能と考えられる.7―229図6・134 関連研究開発動向ロボティックルームに関連する要素機能,インフラストラクチャ技術についてもさまざまな研究が進められている.橋本らのインテリジェントスペースでは,空間に,観測する,理解する,働きかけるという機能をもたせた空間知能化を提案し,観測機能を分布型センサで行うDIND(distributed in-gelligent network Technology,アメリカ国立標準技術研究所)のSmartSpace(スマートスペース),スタンフォード大学のInteractive Work-space(インタラクティブワークスペース),マイクロソフト社のEasyLiving(イージーリビング),ジョージア工科大学のAware Home(アウェアホーム)など,情報サービス中心であるが,知能化された空間の研究開発が盛んに進められている.
