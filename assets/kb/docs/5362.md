# 1・1・3 音声帯域上のデータ通信

> **章**: 第1章 情報通信ネットワーク
> **節**: 1・3 に示す。

本項では、電話網を用いたデータ通信技術について述べる。電話網は本来、音声信号の通信を行うための通信網であるが、音声信号帯域内で変復調を行う装置(モデム、modem)を介することで、データ通信を行うことが可能になる。このような音声帯域上のデータ通信端末としては、モデムによるデータ通信とファクシミリ通信が一般的に用いられている。a. モデム通信。 おもなモデムの規格アナログ通信網である電話網を介してデータの送受信を行うため、“0”,“1”のディジタル信号を、送信側で音声通信のための3.1kHz 帯域のアナログ信号に変調(modulation)して電話網に送信し、受信側でアナログ信号を復調(demodulation)してディジタル信号を再生する。モデムは片方向ずつの通信のみが可能な半二重式と、同時に双方向の通信が可能な全二重(full duplex)式の2種類があり、現在は通信効率の良い全二重式がおもに用いられている(図1・7)。全二重式モデムでは、データ通信機器との間で上り/下りの信号を同時に送受信し、2線式の電話加入者回線で送受信処理を行うため、信号の混入を回避する必要がある。V.22bisなどでは上り/下りで周波数を分離する方式を用いている。これに対し、V.34以上のモデムでは、内部のハイブリッド回路で2線/4線変換を行い、またハイブリッド回路で発生するエコーによるエラーを最小とするため、エコーキャンセラ処理を行う。おもなモデムの規格を表1・4に示す。. 拡張機能公衆網を介したモデムの利用の普及と利用形態の多様化、通信速度の向上に伴い、モデム通信の品質、データ転送速度を向上するため、エラー訂正、データ圧縮などの機能が拡張されている。市販モデムで最も利用されている、MNP(Microcom networking protocol、アメリカのマイクロコム社開発)では、クラス1~クラス4がエラー訂正、クラス5~クラス9がエラー訂正とデータ圧縮を規定している。また、クラス10は回線品質の悪い移動体通信などを想定して、回線状態を監視し、パケットサイズや通信速度を回線品質に応じて動的に変動する機能をもつ。また、ITU-T 勧告としては、パケット型のデータ通信で利用されるHDLC(high-level data link control)プロトコルを基本としたLAPM(link access procedure for modems)により、誤り検出とフレーム再送によるエラー訂正を行う。同じく、ITU-T勧告のV.42bisでは、V.42通信においてLZ 法によるデータ圧縮を規定する。圧縮率はデータ内容に依存するが、最大400%のスループットが得られるといわれている。. モデム接続手順図1・8に、モデムを介してデータ端末間で接続を行うシーケンスの例を示す。一般に、データ端末とモデムの間は、シリアルインタフェース(RS-232C,USB 等)で接続され、モデム制御用のコマンドで発着信制御などを行う。表1・5に、市販モデムで一般に利用されている標準AT コマンド(ATcommand、アメリカ、ヘイズ・マイクロコンピュータ・プロダク(a) 半二重式モデムによる通信(b) 全二重式モデムによる通信図1・7 半二重式および二重式モデム表1・4 おもなモデム勧告おもな勧告番号概要V.221200bps全二重モデム。V.22bis2400bps全二重モデム。V.324800bpsおよび9600bpsの2線式全二重モデム。これ以上の速度は、エコーキャンセラ方式を採用。V.32bis14400bpsの2線式全二重モデム。12000,9600,7200および4800bpsへのフォールバック機能をもつ。V.3428800bpsの2線式全二重モデム。24000,19200bpsへのフォールバック機能と、V.32/V.32bisとの後方互換性を有する。V.34bis33600bpsの2線式全二重モデム。31200bpsまたはV.34通信速度へのフォールバック機能をもつ。V.90下り方向56000bps、上り方向33600bpsの2線式全二重モデム。ホスト側をディジタル通信回線とすることで、下り方向の品質を向上。図1・8 モデム接続シーケンスの例ツ社開発)の例を示す。. ネットワークエコー制御
