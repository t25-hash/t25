# 1・3・3 サーバサービス提供装置

> **章**: 第1章 情報通信ネットワーク
> **節**: 1・3 インタネット

インタネットは、研究者用ネットワークとして利用されていた時期から、その分散システムとしての性格を生かして、FTPや電子メール、ニュース配信等のサービスが実現されていた。これらのサービスでは、サービスを提供するサーバに対して利用者がアクセスする処理形態(クライアントサーバモデル)をとる。この場合、サーバの処理能力が問題となるのであるが、当時はユーザ規模が小さかったことと、扱うデータが文字中心であったため、それほど深刻な問題になるケースは少なかった。しかし、近年では、ユーザ数の急増やマンマシンインタフェースに優れたアプリケーションの登場、さらにはインタネット上のマルチメディア情報充実等により、サーバに対する負荷が増大している。例えば、WWW(world wide web)は、文字データだけでなく、画像イメージ、サウンドなどのマルチメディアデータの扱いに優れていることから急速に利用が広がったハイパーテキスト形式サービスであり、その性質上、大容量データの伝送が発生する。また、WWW を利用するクライアントソフト(ブラウザ)は、GUI(graphical user interface)を駆使して非常に使いやすく設計され8―11ているため、利用者数の急増の一因ともなっている。従来から利用されてきた電子メール(e-mail)についても、ビジネス利用の急増や大規模メーリングリスト、ファイル添付等により、扱うデータ量が非常に大きくなってきている。このような背景から、性能、負荷耐性、信頼性などが非常に高い水準でサーバに要求されており、インタネット利用者数の増加状況から鑑みても、この傾向は今後も継続するものと思われる。
