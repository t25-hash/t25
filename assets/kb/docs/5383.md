# 2・2・3 メモリシステム

> **章**: 第2章 コンピュータ
> **節**: 2・2 コンピュータの基礎

a. キャッシュメモリ半導体テクノロジの進歩に伴い、CPUの動作速度は高速化の一途をたどっており、その動作周波数(通常、2・2・2・c・i項のパイプラインステージ1段の実行時間に相当する)が1ns(ナノセカンド、10億分の1秒)以下のプロセッサも現れている。一方、主記憶アクセスに要する時間も徐々に短縮されているものの、CPU の速度とは大きな差があり、その差はむしろ拡大する傾向にある。つまり、このままではメモリアクセス命令に要するパイプラインステージ数が膨大になり、パイプラインが効率良く流れないのは明らかである。この問題の解決策として、主記憶に比べて高速で小容量の記憶装置をCPU の近くに置き、メモリアクセス命令の実行時には主記憶ではなく、この記憶装置にアクセスする方法が実現されている。この記憶装置をキャッシュメモリ(cache memory)と呼ぶ。キャッシュメモリには主記憶の部分的なコピーを保持するものである。図2・9にCPU、キャッシュ、主記憶の位置関係を示す。図2・8 複数命令を並列実行するパイプライン図2・9キャッシュメモリキャッシュメモリへのアクセスは数ナノ秒程度と高速なため、アクセスするデータがキャッシュメモリ上に存在する(これをキャッシュヒットと呼ぶ)限りはパイプラインがストールすることはない。ただし、キャッシュメモリ上にデータがない場合(これをキャッシュミスヒットと呼ぶ)は多大なサイクル数を要して主記憶までデータを取りに行く必要があるので、キャッシュメモリを用いた場合、そのヒット率が非常に重要になる。例として、5命令に1回の割合でメモリアクセスを行い、キャッシュミスしなければ1サイクル当たり平均1命令実行されるプログラムを考える。また、キャッシュミスによる主記憶アクセスには50サイクル要するとする。このプログラムを100命令実行するとメモリアクセスは20回発生するが、これがすべてキャッシュヒットすればトータルの実行時間は100サイクルである。しかし、20回のうち1回キャッシュに外れただけで実行時間は1.5倍の150サイクルに、2回外れると2倍の200サイクルになってしまう。このことから、コンピュータの高い性能を得るためには、かなり高い確率でキャッシュメモリにヒットしなければならないことがわかる。実際のキャッシュメモリのサイズは16~256kB(キロバイト)程度のものが多い。ヒット率はプログラムの性質等によってさまざまであるがおおむね良好な値を示している。この程度のサイズのキャッシュメモリでも高いヒット率が得られるのは、多くのプログラムに、一度アクセスしたメモリ領域は短い時間の後に再びアクセスされる可能性が高いという性質があるからである。この性質のことを参照の局所性と呼ぶ。また、最近のコンピュータではキャッシュメモリと主記憶の性能差が大きいため、それらの間に中速で中容量[2~8MB(メガバイト)程度]のメモリを置いた階層構造を取るものが多い。このようにした場合、CPU に近いほうのキャッシュメモリを一次キャッシュ、主記憶に近いほうを二次キャッシュと呼ぶ。キャッシュメモリと主記憶とのマッピングは、メモリ領域を32~64バイト程度のブロックに区切り、ブロック単位に行うのが一般的である。主記憶のどのブロックがキャッシュメモリのどこに存在するか(あるいは存在しないか)を判定するための機構として、ディレクトリ(またはタグともいう)と呼ばれる対応表が用意されている。キャッシュアクセス時には、まずディレクトリが検索されてキャッシュヒットあるいはミスヒットが判定され、ヒットならばキャッシュメモリの該当するブロックがアクセスされ、ミスならば主記憶がアクセスされる。主記憶とキャッシュメモリの領域のマッピングにはいくつかの方法がある。主記憶の任意のブロックをキャッシュメモリの任意の場所にマッピング可能にする方式をフルアソシアティブ(full as-sociative)方式と呼ぶ。この方式はキャッシュメモリ上の領域をフルに活用できる点で優れているが、ディレクトリの構成が複雑になり、ハードウェア的なオーバヘッドが高い欠点がある。逆に、主記憶のブロックのアドレスによってキャッシュメモリ上の位置がユニークに定まる方式として、ダイレクトマップ(direct map)方式がある。この方式はフルアソシアティブ方式と反対に、ハードウェアがシンプルになる長所と、キャッシュミス率が高くなる短所をもつ。また、これらの中間的な方式としてセットアソシアティブ方式がある。これは、ハードウェアの複雑化を抑えつつキャッシュ領域の活用度もある程度高めることをねらったものである。以下、セットアソシアティブ方式について詳しく説明する。セットアソシアティブ方式は、主記憶およびキャッシュメモリの領域を、セットと呼ぶ複数の部分領域に分割し、主記憶とキャッシュのマッピングを対応するセット内に限定するものである。セット内には複数のブロックが格納可能で、この数をウェイ数あるいはレベル数と呼ぶ。ダイレクトマップ方式はウェイ数が1のセットアソシアティブ方式と考えることもできる。実際のキャッシュメモリでは、ウェイ数が2~4程度のセットアソシアティブ方式が採られることが多い。図2・10はセットアソシアティブ方式を図示したもの8―37図2・10 仮想記憶システムの概要である。セットアソシアティブキャッシュにおいて、すべてのウェイにブロックがマップされている状態で、キャッシュ上にない同一セット内のブロックがアクセスされると、そのブロックのための領域を新たに確保するため、既存のいずれかのブロックをキャッシュから追い出す必要が生じる。このとき、追い出すブロックを適切に選ぶことで、キャッシュヒット率を高く保つことができる。最後にアクセスされてからの経過時間が最も長いブロックを優先的に選ぶLRU(least recently used)という方式が広く用いられている。b. 仮想記憶プログラムがアクセス可能な主記憶の大きさは命令セットアーキテクチャによって規定されており、例えば32ビットアーキテクチャでワード長をフルにアドレスとして使えば、メモリ空間の大きさは4GB(ギガバイト)に達する。また、現在のコンピュータは複数のプログラムが同時に動作できるのが普通で、1個のプログラム当たり4GB のメモリ空間とすれば、トータルの空間サイズは膨大なものになる。ところが、物理的に存在するメモリは有限であり、すべてをまかなうには明らかに不足する。このように、実際のメモリサイズをはるかに超える空間サイズを提供する仕組みが仮想記憶(virtual memory)である。仮想記憶システムは、プログラムがアクセスする全空間のうち、ある時点でアクセスする部分のみを実メモリ上に置き、それ以外はディスク等の外部記憶に追い出すことで、上記を実現している。アクセスが外部記憶に追い出された領域に及ぶと、実メモリ上の適当な領域を選んで、外部記憶との交換が行われる。外部記憶は一般に実メモリに比べてはるかに大きな容量を有しているので、実メモリサイズを超えた空間が仮想的に実現されることになる。図2・11に概念図を示す。仮想記憶は、プログラム上のメモリ空間と実際のメモリ空間を対応付ける仕組みととらえることができる。前者のアドレスを論理アドレスまたは仮想アドレスと呼び、後者を物理アドレスまたは実アドレスなどと呼ぶ。仮想記憶を実現するためには、双方のアドレス空間をページという固定長のブロックに分割し、ページ単位で両者をマッピングする手法が多く用いられている。これをページングと呼ぶ。同時期に物理メモリ上に存在できるページ数は実メモリの大きさで制限され、それ以外のページは外部記憶上に置かれることに8―38情報・メディア機器なる。ここで論理空間サイズを4GB、物理メモリサイズを1GB、ページサイズを4kB とすれば、論理空間は1M ページ、物理空間は256k ページで構成されることになる。論理アドレスは32ビットで表現され、上位20ビットが論理ページアドレス、下位12ビットがページ内アドレスになる。また、物理アドレスは30ビットで表現され、上位18ビットが物理的なページ(ページフレームと呼ぶ)を特定する番号として用いられ、下位12ビットがページフレーム内アドレスになる。論理ページと物理ページの対応関係はページテーブルと呼ばれる表に格納される。ページテーブルは個々の論理ページに対して1個のエントリが存在する。エントリの内容は有効フラグと呼ばれる、該当するページが物理メモリ上に存在するかどうかを表すフラグと、(存在する場合に)ページフレーム番号を格納する領域で構成される。ページテーブル自身もメモリ空間の一部に存在する。図す。論理アドレスから物理アドレスへの変換はメモリアクセスのたびに行う必要がある。これはデータへのアクセスだけでなく、命令語を読み出す場合も同様であり、このような頻繁なアドレス変換のたびにページテーブルをたどっていたのでは、オーバヘッドが非常に大きくなってしまう。そこで、ページテーブルのうちアクセス頻度の高い部分のコピーを格納した高速小容量のメモリを別に備え、アドレス変換を高速に行う方式が広く採用されている。この小容量メモリをTLB(translation lookaside buffer)と呼ぶ。メモリアクセスの際にはまずTLB が参照され、対応するエントリがTLB に存在すれば、アドレス変換は速やかに行われる。エントリがTLBに存在しない場合は、ある程度の手間をかけてページテーブルから情報を得る必要がある。TLB のエントリ数は64~1024程度で、ページテーブルのエントリ数よりはるかに小さいが、それでも十分に高いヒット率を示し、有効に活用されている。これはメモリ参照の局所性によりキャッシュが有効に働くのと同じ理屈である。TLB の構成はキャッシュと同様にダイレクトマップ方式またはセットアソシアティブ方式のものが多い。また、一次キャッシュ、二次キャッシュのように階層構造をもつTLB を採用しているプロセッサもある。これまでは1個の論理空間を実空間とマッピングする仮想記憶システムを説明してきたが、実際のコンピュータシステムでは複数のプログラムを同時動作させるため、上記を拡張した多重仮想記憶が実装されている。ここではそれぞれの空間に識別子を付加することで空間が多重化されているが、基本的な原理は単一空間のものと同じである。
