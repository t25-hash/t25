# 2・3・1 マイクロプロセッサ

> **章**: 第2章 コンピュータ
> **節**: 2・3 実際のコンピュータ

マイクロプロセッサ(microprocessor)とはコンピュータの中央処理装置(CPU)の機能を一つの半導体チップに集積したものである。半導体技術の進歩により、最近のCPU はほとんどの機能を一つのチップに集積しているため、CPU の同義語として用いられることが多い。a. マイクロプロセッサの速度向上1971年に世界初のマイクロプロセッサ4004が誕生して以来、現在に至るまで10年で約100倍の速度向上というペースで急速の進歩を遂げてきた。この進歩は命令セットの工夫、動作周波数の向上、キャッシュやパイプラインといったハードウェアアーキテクチャの工夫による。特に後者二つは、半導体チップの製造技術の進歩が寄与するところが大きい。つまり、より高速なトランジスタをより多く搭載できるようになったことで、動作周波数が向上すると同時にスーパースカラを代表とする複雑な並列処理の実現、キャッシュのオンチップ化によるメモリアクセスの高速化が可能になった。図2・13にインテル社のマイクロプロセッサのトランジスタ数、およびプロセッサの動作周波数の変遷を示す。動作周波数とトランジスタ数が約25箇月で2倍に増図2・12 multi processor、チップマルチプロセッサ)とマルチスレッドプロセッサ(multi thread processor)がある。CMPは1チップ内に複数のCPU を構成したマルチプロセッサシステムであり、高速なCPU 間通信と高いチップ全体性能を特徴としている。CMPの代表的なものとして2001年にIBM 社が発表したPower4がある。一方マルチスレッドプロセッサは、1個のマイクロプロセッサがあたかも複数個のマイクロプロセッサであるかのようにデータを扱うことで、命令の処理効率を上げるマイクロプロセッサである。2002年インテル社はこれを実現するため技術(ハイパースレディング技術)を用いたインテル좲XeonTMプロセッサを発表した。Xeonは二つの実行スレッドを同時に実行する機構をもつ。これによりチップ全体性能が30%向上するという。CMPとマルチスレッドプロセッサは互いに共存できるアーキテクチャであり、今後これらを融合したアーキテクチャが開発されていくものと考えられる。b. Pentium좲4(インテル)Pentium4(ペンティアム4)は2000年にインテル社が開発した32ビットインテルアーキテクチャ(IA-32)のマイクロプロセッサである。図2・15にブロック図を示す。Pentium4はハイパーパイプラインと呼ばれる20段の長いパイプラインを採用し、Pentium3に比べて動作周波数を1.5倍(同一テクノロジでの比較)に向上させている。2002年に2.8GHzで動作するものが発表されている。IA-32アーキテクチャは16ビットプロセッサ8086以降のx86命令もサポートしており、多くの複雑な処理を行う命令を含んでいる。一般に複雑な処理を行う命令はパイプライン化が難しい。これに対してPentium4では、デコード時にマイクロオペレーションと呼ばれる単純な命令に分解することで実行効率を高め、問題を解決している(この機構は1995年に発表されたPentium Proから実装された)。1サイクルでマイクロオペレーションは6個まで同時発行が可能で、さらに命令完了を待たずにマイクロプロセッサに投入できる命令数が128と大規模なスーパースカラプロセッサとなっている。また分解後のマイクロオペレーションをトレースキャッシュと呼ばれる命令キャッシュに記録することで、再実行する際のデコード処理を省いている。二次キャッシュは512kB 8ウェイのセットアソシアティブキャッシュでチップ内に搭載されており、高速なデータ転送を可能にしている。さらにストリーミング拡張命令2(SSE2)と呼ばれる命令をもち、3D グラフィックスやアニメーションといったマルチメディアアプリケーションを高速に動作させる。8―40情報・メディア機器。 Itanium2(インテル、ヒューレットパッカード)Itanium2(アイテニアム2)はヒューレットパッカード社とインテル社によって、ハイパフォーマンス向けに開発された64ビットプロセッサである。2002年第3四半期に発表された。x86アーキテクチャとは異なり、EPIC(explicitly parallel instruction comput-ing)アーキテクチャと呼ばれるアーキテクチャを採用している。EPICアーキテクチャは、同時に実行できる操作をパックした命令(バンドルと呼ばれる)を実行する(図2・16)。命令間の並列性の抽出はコンパイラが行うという点でVLIW アーキテクチャの一種であると考えられる。しかし、バンドルは通常のVLIW 命令とは異なり並列実行の単位とはなっていない。テンプレートフィールド内に記録された情報(ストップビット)が並列実行できる命令群の区切りになっている。これにより実行バイナリの非互換性を解決している。Itanium2では1サイクルに2バンドルフェッチで、1サイクル当たり11命令発行が可能となっている。VLIW(very long instruction word)という性質から、マイクロプロセッサの性能はコンパイラの最適化にゆだねられている。コンパイラによる最適化を支援するため、命令語中にプリフェッチ、分岐、キャッシュアクセスに関するヒント情報をもたせてコンパイラが制御できるようになっている。また、レジスタ数も整数レジスタ128個、浮動小数点レジスタが128個と、他のマイクロプロセッサと比較して大規模なレジスタセットを有することでコンパイラの変数割当ての柔軟性が増している。さらに投機的ロード(データ読込みを先行的に実行し、データ使用時にロードしたデータの正しさをチェックする機構)や条件付き実行命令をサポートしている。図2・17にItanium2のブロック図を示す。パイプライン段数は8で、最近のハイエンドのマイクロプロセッサとしては非常に少ない。これにより分岐予測ミス時の実行遅延を軽減している。x86命令セットのデコードコントロールユニットをもつことで、従来のx86用アプリケーションの動作も可能にしている。2001年に発表されたItaniumより動作周波数の向上(800MHz~1GHz)、命令同時実行数の増加(9命令から11命令)、キャッシュアクセスの高速化(ほぼ半減)およびメモリスループット向上(11.7B/s~32GB/s)によって2倍の性能を達成している。. Power4(IBM)Power4(パワー4)はIBM 社によってサーバマシン向けに開発されたチップマルチプロセッサ(CMP)である。2001年第4四半期に1.1GHz と1.3GHz のPower4が発表された。1チップ内に64ビットPOWER アーキテ図2・16 PICアーキテクチャの命令形式(バンドル)クチャのCPU を二つ搭載し、さらに4チップをパッケージ化して8CPU のメモリ共有型マルチプロセッサシステムの構築が可能である。図2・18にPower4のブロック図を示す。1CPU は8命令同時発行が可能で、先行命令の完了を待つことなく200命令を投入することが可能なスーパースカラプロセッサである。一次命令キャッシュ(64kB)および一次データキャッシュ(32kB)はCPU ごとに存在するが二次キャッシュ(1440kB)および三次キャッシュ(32MB)はチップごとに存在している。また12.8G/sという大きなメモリバンド幅をもち、高負荷なサーバ処理(トランザクション処理など)にも耐えうる設計となっている。浮動小数点数演算性能の評価値であるSPECfp2000のピーク値も1169と世界最高レベルであるため科学技術計算サーバ用マイクロプロセッサとしても注目されている。. ARM1020E(ARM)2002年ARM 社より開発された組込み用32ビットマイクロプロセッサである。絶対性能ではなく周波数当たりの性能を重視した設計になっている。図2・19にブロック図を示す。パイプライン段数は6段で1サイクル当たり1命令発行、動作周波数は325MHz である。また消費電力は通常動作時では約0.2W であるが、さらにNAPモード、SLEEPモードと呼ばれる動作モードによって、非動作時の消費電力を抑える工夫が施されている。命令セットにも工夫が施されている。Thumb命令セットと呼ばれる16ビット命令もサポートしており、これを用いることで使用メモリ量が最大35%削減できる。また、組込み用マルチプロセッサとしての利用のためDSP用の拡張命令をもっており、16ビットあるいは32ビットのSIMD 整数演算の高速な処理を可能にしている。組込みプロセッサとしてさほど頻度が多くない浮動小数点演算は、マイクロプロセッサ内部では処理をせず別の処理チップ(コプロセッサ)で処理を行う設計になっている。これによりマイクロプ図2・18 Power4のブロック図図2・17 Itanium2のブロック図図2・19 ARM1020Eのブロック図(http://www.arm.com/products/CPUs/ARM1020E.htmlより抜粋)ロセッサ自体の回路規模を小さくし、消費電力を抑えている。
