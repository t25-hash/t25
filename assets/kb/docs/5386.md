# 2・3・2 項では、メインフレーム、スーパーコンピュータなどのハ

> **章**: 第2章 コンピュータ
> **節**: 2・3 実際のコンピュータ

イエンドのコンピュータシステムについて概観する。a. 概説気象予測、バイオインフォマティクス、構造解析、衝突解析などの科学技術計算分野では大規模な数値計算処理が必要とされている。そのため、数百、数千のプロセッサにより並列計算処理を行う超並列型スーパーコンピュータ(super computer)が使用される。後述するように並列型スーパーコンピュータにはベクトルプロセッサを使用するベクトル並列型と、RISC方式(reduced instruction set computer)プロセッサなどのスカラプロセッサを使用するスカラ並列型が存在する。また、1990年代からのスカラプロセッサおよびパーソナルコンピュータの低価格化、高性能化により、SMP サーバ(symmetric multiprocessingserver)、PCクラスタ(personal computer cluster)が利用されるようになった。SMPサーバは数台から数十台のスカラプロセッサが主記憶、IOを共有する。OLTP(online transaction processing)やデータベース処理、スカラ並列型スーパーコンピュータの計算ノードなど、中規模以上の処理用途に広く使用されている。PCクラスタは多数のPCを結合網で結合する。安価な高性能処理装置として、データマイニング、検索エンジンなどの負荷分散が容易な並列計算用途で使用される。また、多重化による高可用性のPCサーバとして小規模の処理用途で広く使用されている。販売事務、経理事務など、高度な信頼性や耐障害性を必要とする基幹系システムにはメインフレームが使用される。メインフレームは耐故障性、保守・管理性が高められており、平均年間稼働率は99.99%に達する。b. ベクトル並列型スーパーコンピュータ。 概説ベクトル並列型スーパーコンピュータは多数のベクトルプロセッサをクロスバー結合網(CrossBar network)で並列に接続した計算機である。ベクトルプロセッサは多数のデータを一度に処理でき、これを複数並列に動作させることにより、同時に多量のデータを処理する。これにより、多量のデータに繰返し処理を行う科学技術計算を高速に実行することができる。最新のベクトル並列型スーパーコンピュータには地球シミュレータ(Earth Sim-ulator,NEC)、SX-6(NEC)、VPP5000(富士通)がある。特に地球シミュレータは2002年6月の時点で35.86TFlopsで世界一高速な計算機であり、2位ASCI White(IBM)の7.226TFlopsを大きく上回っている。ベクトル並列型スーパーコンピュータの実例として、SX-6,VPP5000の構成例を図2・20に示す。SX-6は主記憶を複数のベクトルプロセッサで共有しており、さらにクロスバー結合網で全体が接続される。VPP5000では各ベクトルプロセ(a) VPP5000の構成(b) ベクトルプロセッサベクトルプロセッサ(vectorprocessor)は複数のデータを一つのベクトルレジスタに格納し、ベクトルレジスタどうしの演算によって1命令で多数の演算を行う。また、ベクトル演算器は深いパイプラインで構成され、連続した演算のスループットが高い。ベクトルプロセッサはベクトル演算を行うベクトルユニットと制御を行うスカラユニットで構成される。ベクトルプロセッサの構成例を図2・21に示す。ベクトルユニットは主記憶上の大量のデータに対する演算を目的とし、同じデータに繰返しアクセスすることが少ないためキャッシュをもたない。また、条件分岐を含むループを効率的に処理するため、演算結果をマスクするマスクレジスタをもつ。c. スカラ並列型スーパーコンピュータ1990年代にRISC(reduced instruction set computer)プロセッサを代表とするスカラプロセッサの高性能化、低価格化が進み、ローエンドからミドルレンジ市場では数台から数十台のスカラプロセッサを搭載したSMPサーバ(d項参照)が使用されるようになった。SMPサーバはスカラプロセッサとともに性能を上げたため、これをノードとするスーパーコンピュータが作られた。スカラ並列型スーパーコンピュータは複数のSMPノードを結合網で接続し、大規模並列処理を行う。SR8000(日立製作所/1999年)、HPC2500(富士通/2002年)はベクトル演算を高速に処理する仕組みをもっており、従来のベクトル並列機ユーザをサポートする。また、ASCI White(IBM/2001年)は2002年6月の時点で地球シミュレータに次いで世界で2番目に速いコンピュータである。d. PCクラスタPCの高性能化、低価格化に伴い、多数のPCによる並列、並行処理が行われるようになった。ラックサーバ、ブレードサーバはPC1ユニットをラック型もしくはブレード型の基板に集積し、筐体に多数のユニットを収めることができる。これにより、多数のPCの効率的な設置、保守が可能になった。一方、科学技術計算処理では多数のPCを高速な結合網で接続し、PCクラスタとして並列計算を行う。2002年6月の時点で、PCクラスタはドイツ、ハイデルベルク大学のHELICS が世界で35番目、東京工業大学のPrestoIIIが世界で47番目に高速なコンピュータとなっており、安価なスーパーコンピュータとして注目されている。f. メインフレーム1960年代のSystem/360(IBM)より始まるメインフレームは長年基幹業務に使用される中で信頼性、安定性が高められてきた。現在でも高度な信頼性や耐障害性を必要とする基幹系システムにはメインフレームが使用される。メインフレームは耐故障性、保守・管理性が高められており、平均年間稼働率はPCクラスタの約99%,PCサーバの約99.9%に対して99.99%を超える。現在販売されているメインフレームはzSeries(IBM)、GS8000(富士通)、AP8000/MP(日立製作所)、ACOS(NEC)がある。近年ではOS/390のほかにオープン系オペレーションシステムでメインフレームを稼働する例も増えており、高信頼性、保守性が求められる場面での使用が広がっている。g. 超並列型スーパーコンピュータ1990年代にスカラプロセッサの高性能化、低価格化に伴い、数千のスカラプロセッサを結合網で接続する超並列型スーパーコンピュータ、SR2201(日立製作所/1995年)、Cenju-4(NEC/1997年)、AP3000(富士通/1997年)などが作られた。1ノード1プロセッサを基本構成としたためSMPノードに比べてノード性能で劣り、ノード数が多いため結合網も複雑化したため、各社とも1990年代終わりにはスカラ型スーパーコンピュータに移行した。
