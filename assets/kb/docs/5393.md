# 3・2・2 参照局所性と記憶の階層構成

> **章**: 第3章 記憶装置・記憶メディア
> **節**: 3・2 記憶システム

計算機の主たる役割はプログラムの実行にあるが,記憶装置上のプログラム参照には局所性(空間局所性,時間局所性)が存在することが知られている.この特徴を生かし,プログラム参照対象を高速なキャッシュメモリに保持することにより,また,近傍のデータを近傍の記憶領域に格納することによりプログラムの実行を大きく高速化することができる.a. キャッシュメモリと主記憶装置主記憶に比べて数倍高速なアクセスが可能であり,極めて小容量な高速メモリと位置付けられる.主記憶との置換えの単位はブロックと呼ばれる数語の大きさが採用される.キャッシュメモリと主記憶とのブロックの割当て方式には,主記憶ブロックを任意のキャッシュブロックに割当てるフルアソシアティブキャッシュ法,主記憶ブロックを一義的なキャッシュブロックに割当てるダイレクトマッピング法,主記憶ブロックを複数セットにまとめ任意のキャッシュブロックセットに割当てるセットアソシアティブ法などが知られている(図3・5).現在は,キャッシュメモリ管理の簡便さとはん用性を特徴としたセットアソシアティブ法が主流である.キャッシュ容量は一般に極めてわずかであるが,高速処理が要求されるため,新しいブロックとの置換手法として,ハードウェアによるLRU(least recently used)置換が用いられることがある.CPU がデータの値を変更する場合,キャッシュ上の値と同時に主記憶の値まで書き換えるライトスルー方式と,キャッシュ上の値だけを変更し,主記憶上の値をブロックの置換が発生した時点で更新するライトバック方式とがある.最近では,アドレスの衝突による性能劣化を防ぐためにビクティムキャッシュが導入され,また,エンコード化された命令列を格納するトレースキャッシュなど新しい手法が採用されるに至っている.主記憶のアクセスタイムがほとんど改善されないことから,キャッシュレベルの多段化に加え,さらに,キャッシュ上に所望のブロックが存在する確率(キャッシュヒット率)を向上させるため,コンパイラが必要となるであろうデータを事前にキャッシュに取込むプリフェッチ命令の活用が模索され,キャッシュの動作に適合したキャッシュヒット率向上手法,すなわち,キャッシュアウェアなアルゴリズムの開発が進んでいる.また,複数個のCPU から構成されるマルチプロセッシングシステムでは,各CPU ごとに設けられたキャッシュ間のデータの一貫性が問題になる.キャッシュデータの一貫性が保たれなくなることにより,CPU 間で多大なオーバヘッドの発生による処理性能低下図3・5 主記憶とキャッシュメモリ図3・6 RAID 5の構成が発生することが知られており,種々のキャッシュ一貫性維持プロトコルが提案されている.とりわけ,バス結合型マルチプロセッサにおいては定着した技術となりつつある.ハードウェアコストの低減とともに,スライスと呼ばれる余剰プロセッサを用いた投機実行によるミスの回避も最近の技術として注目される.b. 主記憶と二次記憶装置(secondary storage system)主記憶は半導体素子の集積技術の飛躍的進歩および低価格化によってその容量は増大しているが,一方で,アプリケーションが扱うデータ量も恒常的に増大している.アプリケーションが必要とするデータは,比較的高速で中容量の主記憶と,低速であるが十分な容量のある磁気ディスク装置などの二次記憶装置とを組合わせた仮想メモリ方式を用いて処理される.仮想メモリ方式は,オペレーティングシステムにより制御されており,数kB のページを単位として,主記憶容量(実空間)よりもはるかに大きな記憶容量(仮想空間)を提供するものである.ページ単位での実空間と仮想空間の管理,つまり,実空間と仮想空間のアドレス変換,ページの置換え,メモリプロテクションなどがおもな処理である.アドレス変換は,ページ単位での単純な変換を行うページ方式,ページを複数まとめたセグメントで管理しプログラムレベルで制御を行うセグメンテーション方式,両方式を複合し現在広く用いられているセグメンテーションページング方式がある.ページ置換えアルゴリズムは,ページ単位での主記憶利用効率で管理するのが一般的であり,最も長期間存在するページを最新のページと置き換えるFIFO(first-in first-out)アルゴリズム,ページ置換え時点で最も参照されていないページと置換えるLRU アルゴリズムなどがあり,一般にはLRU アルゴリズムが用いられている.
