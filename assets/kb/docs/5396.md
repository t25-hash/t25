# 3・2・5 ストレージネットワーキング(storage networking)

> **章**: 第3章 記憶装置・記憶メディア
> **節**: 3・7 )。

2000年に入り、ネットワーク技術の大きな進展に伴い、ストレージ技術とネットワーク技術の融合が図られた。ストレージシステムの格納するデータが指数関数的に増大していく中で、システムの運用/管理コストの増大は切実な問題となってきた。これを回避するため、ストレージコンソリデーション(ストレージの統合化)と呼ばれるファイバチャネル技術を用いたストレージエリアネットワーク(SAN)によるシステム統合化が進められた。ストレージエリアネットワークとは、ストレージシステム内のネ図3・7メインフレーム/オープンサーバの二次記憶システムの共有図3・8 ストレージエリアネットワーク8―49ットワークであり、従来のサーバ間を接続するローカルエリアネットワーク(LAN)に相当する(図3・8)。ストレージシステムは、従来、サーバに対し従属的に接続されてきたが、1990年代末から2000年にかけては、ストレージシステム全体を専用のネットワーク(SAN)で接続し、これらにサーバが複数接続される構成が登場した。SAN の登場により、すべてのストレージは、SAN を介して一元的に管理することが可能となり、データの管理コストが大きく改善された。また、従来のバックアップデータは、LAN を経由していたためサーバやネットワークに大きな負荷を発生したが、SAN に磁気テープ装置を接続することにより、ストレージシステムからバックアップ装置を共有することが可能となり、一元的なバックアップを効率よく実現できることにもなった。さらに、サーバが介在しないサーバフリーバックアップも開発が進められている。ファイバチャネルと同時にIP-SAN と呼ばれるイーサネットを利用したSAN も登場しつつある。IP-SAN は、イーサネット上のTCP/IPプロトコルにファイバチャネルで使用されているSCSIプロトコルを内包させることにより、現状のイーサネット接続環境に必要となるネットワーク機器ならびに各種管理ソフトウェアを流用し、低価格なネットワーク接続環境を利用することが可能となる。また、イーサネット上のプロトコル処理を高速に処理するハードウェアエンジンの開発も進められており、ストレージ接続インフラとして大きな期待が寄せられている。ネットワークとストレージの融合により、今後、種々のストレージ管理技術の開発が進むと予想される。
