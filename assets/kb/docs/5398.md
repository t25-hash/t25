# 3・2・7 ストレージ、ファイル、データベース

> **章**: 第3章 記憶装置・記憶メディア
> **節**: 3・7 )。

ストレージ(storage)にはデータが格納され、それを管理する最も基本的なソフトウェアがファイルシステムであることから、ストレージにファイルシステムというソフトウェアを組込むことが容易に考えられる。ファイルサーバとも呼ばれてきたNAS(net-work attached storage)と名付けられている製品がこれに相当する。2004年時点ではファイバチャネルの性能のほうがイーサネットに比べて高いため、入出力負荷の高い本格的なアプリケーションはSAN 上に実装される場合が多い。ファイルよりもさらに厳密な8―50情報・メディア機器データ管理を司るソフトウェアがデータベース管理システムであり、今後ストレージのアプライアンス化が進むものと予想される。NAS とSAN は排他的な技術ではなく、NAS を実現するためにストレージとしてSAN を利用することが考えられる。また、メタデータをNAS で管理し、実データの転送をSAN 経由とする手法も提案されている。記憶装置とそれに格納されるデータを管理するソフトウェアの一体化は今後の大きな方向と考えられる。
