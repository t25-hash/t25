# 4・1・2 コンピュータにおけるマンマシンインタフェース

> **章**: 第4章 入出力装置
> **節**: 4・1 マンマシンインタフェース

コンピュータの分野におけるマンマシンインタフェースは、初期のパンチカードによる入力から、TSS 端末やパーソナルコンピュータ(パソコン)へのキーボードを用いた対話型コマンド入力、そしてマウス処理を中心としたGUI(graphical user interface)へと変遷してきた。近年では音声入力装置やテレビカメラを内蔵した機器も安価で市販されている。これらはコンピュータに指示を与え、結果を受取るうえで、より人間にとって自然で快適なシステムの実現を目指した結果である。コンピュータにおけるマンマシンインタフェースは、個々のデバイスであるハードウェアと、それらのハードウェアを個々に、あるいは統合的に処理するソフトウェアから構成される。a. マンマシンインタフェースのためのハードウェア技術現在ではおもに以下のデバイス(ハードウェア)が人間とコンピュータとの情報のやり取りのために使用されている。(1) 入力機器(a) キーボード(b) マウス、トラックボールなどのポインティングデバイス(c) タブレット、タッチパネル(d) スキャナ、OCR(optical character reader)(e) ジョイスティック(f) マイク(g) TV カメラ(2) 出力機器(a) CRT(cathode-ray tude)、液晶パネルなどのディスプレイ表示装置(b) スピーカ(c) プリンタ上述したデバイス以外にも、コンピュータにおける特殊なマンマシンインタフェースの一例として、身体障害者や知的障害者への入出力デバイスなどがある。例えば(1) 視覚や聴覚など感覚の障害者に対する音声やドットインパクト型の出力装置(2) 運動障害者に対する筋電位やオートスキャニング(自動走査)型入力装置などがその代表例である。b. マンマシンインタフェースのためのソフトウェア技術近年のプログラミング開発環境では個々のアプリケーションとマンマシンインタフェース(ユーザインタフェース)は別々に設計されるのが一般的である。ユーザインタフェースで使用される多種多様なデバイスを制御するためには、多くのハードウェア依存のソフトウェアを準備する必要がある。また、例えばGUIで実現されるウインドマネージャなどの機能を、アプリケーションごとに開発するのは非効率的である。そこでユーザインタフェース部分を分離、ライブラリ化(標準化)し、利用しやすい形で提供することで、プログラミング開発者はデバイスへの具体的なアクセス方法などを気にすることなく、ハードウェア構成に依存しないソフトウェアを構築することができる。またユーザにとっては、異なるアプリケーションでもほぼ同じ操作体系を利用できるというメリットもある。この分離されたユーザインタフェースをユーザインタフェース管理システム(UIMS : user interface management system)と呼ぶ。代表的な例としてはUNIX のMotifやWindowsのMFC(microsoftfoundation class)がある。
