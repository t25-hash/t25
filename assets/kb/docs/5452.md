# 5・3・4 インタフェース

> **章**: 第5章 情報通信機器
> **節**: 5・3 モバイル情報機器(PDA)

a. IrDA(アイアールディーエー). IrDA 概説現在,赤外線データ通信規格に関する標準化を進めているおもな団体としては,IEEE(Institute of Electri-cal and Electronics Engineers,アメリカ電気・電子技術者協会),IEC(International Electrotechnical Commission,国際電気標準化会議),JEITA(Japan Electronics and Information Technol-ogy Industries Association,電子情報技術産業協会),ICSC(Infrared Communication System Study Committee,光無線通信システム調査研究委員会),TTC(Telecommunication Tech-nology Committee,情報通信技術委員会),およびIrDA(Infra-red Data Association,赤外線データ通信協会)がある.8―103IrDA 委員会は低コストで便利な赤外線通信の相互接続性の確立と普及を目指して1993年6月に設立された業界団体である.構成企業は,パソコン/ワークステーション/PDA,周辺装置,ソフトウェア/OS,LSI,光デバイス,通信事業関連の企業などであり,160社以上が参加している.IrDA が定めた規格は,業界標準であるが,赤外線データ通信における事実上の標準規格となっている.通信可能な距離は0~1mで,角度は15°の円すいである.携帯電話やPDA の普及により,消費電力低減への要求が高まってきたため,通信可能距離が0~20cmの短距離通信モード(省電力オプション)も追加設定されている.通信速度としては,2400bps~115.2kbpsのIrDA 1.0に加えて,高速版として1995年10月に1.152Mbpsと4.0MbpsのIrDA 1.1が,1999年3月にはさらなる高速版の16MbpsのVFIR(very fast infrared)が新たに追加された.以下には,対象を限定し最近に策定されたIrMCとIrDA-Controlについて述べる.. IrMC(アイアールエムシー)IrMC(Infrared MobileCommunication)はIrDA 赤外線通信を携帯電話に利用する際の応用規格である.携帯電話経由のパケット通信網の利用や,携帯電話そのものによる電子メールやインタネットの利用などにより,携帯電話とその他機器間でのデータのやり取りそのものの需要が高まってきたことから策定された.IrMC規格を策定するに当たって,IrDA がまとめた要求仕様は以下のとおりである.・携帯電話小形化・薄型化を妨げないよう,デバイス(赤外線トランシーバ)自体が十分小形で実現できること.・携帯電話の待受け時間・通話時間を圧迫することのないよう,赤外線通信機能を実現するハードウェア(おもに赤外線トランシーバ)の十分な低消費電力化が図られること.・これまでにIrDA-Dataが実装されているPC/PDA などとの相互通信が確保できること.・ページャなど処理能力の低いハードウェアをもつ端末との通信も可能であること.これらの要求を満たすものとして,エリクソン社,ノキア社,モトローラ社,NTT ドコモなどが中心となってIrMC規格を作成した.IrMCでは,携帯電話における赤外線通信の応用として,電話帳やスケジュール,メッセージ,ノートなどのオブジェクト交換,データ通信(モデムアクセス),音声通信の三つを想定している.IrMCを搭載した携帯電話機では,携帯電話にいちいち電話帳やスケジュールを入力しなくても,PC上のPIM ソフトウェアから,電話帳・スケジュールをそのまま転送することができるようになる.また,特殊なソフトウェアをPCにインストールしなくても,PCの赤外線通信ユニットの横に携帯電話を置くだけで,ネットワークにダイヤルアップ接続することも可能となる.通話するためのスピーカとマイクを一体化した,いわゆるヘッドセットもIrMCを搭載することにより,ワイヤレス化することができる.. IrDA-Controlシャープ,ヒューレットパッカード,インテル,マイクロソフトの4社は,新しい双方向の赤外線通信規格であるIrBus(アイアールバス)の開発を続け,1998年2月にIrDA-Controlという名称で規格として採用された.IrDA-Controlは,マウスやキーボード,ジョイスティックといったパソコン入力機器の無線化や,セットトップボックスなどのディジタル家電用リモコンの高機能化など,機器の制御や少量の情報交換を目的としたものである.特徴としては,リモコン並みの長距離・広範囲の通信が可能なことと,1台のパソコンあるいはディジタル家電に複数台(最大8台)の入力装置を同時に接続可能なことが挙げられる.通信速度は75kbpsで,到達距離は8mである.このため,従来のIrDA-Data規格とIrDA-Control規格の通信方式は全く互換性がなく,互いに通信することはできない.IrDA-Controlの有力な応用として,双方向のデータ伝送性と75kbpsの通信速度を生かし,従来のリモコンの機能を拡張する新しいリモコンへの展開が検討されている.例えば,ディジタル放送で8―104情報・メディア機器配信される電子番組表(EPG : electronic program guide)を,セットトップボックスから手元のリモコンにダウンロードし,リモコン上の液晶画面上に表示させることで,番組内容を手元で確認しながら番組の選択をしたり,ビデオの録画予約をしたりするといったことが可能となる.また,家庭内の各機器からリモコンに操作メニューをダウンロードし,ユーザが操作メニューを組合わせることで,部屋の照明を消すと同時にテレビやエアコンの電源も切るといった家電の連携動作なども可能となる.JEITA では,IrDA-Con-trolを用いてあらゆる家電機器を操作することが可能なリモコンを実現するための規格として,2001年に赤外線高速リモコン規格を策定した.b. Bluetooth(ブルートゥース). Bluetooth のねらい数メートル程度までの距離にある携帯電話,ノートPCあるいはモバイル情報機器(PDA)などあらゆる種類のモバイル機器,さらにはその周辺機器を相互に無線で接続するための技術がBluetoothである.伝送媒体に電磁波を利用しているので,多少の障害物や赤外線のように対向する機器間の位置合わせを気にせず,厄介なケーブルなしで機器間を簡単に接続できる.Bluetoothでは利用目的さえ合致していれば接続ケーブルやコネクタの形状や規格を意識することなく,異機種間を含めた接続性を確保することを目的としている.また,携帯電話やPCといった情報通信機器だけでなく,電子スチルカメラや家電機器など多種多様な機器への搭載と今後の広範な応用を考慮して,・短距離の無線接続を利用して小形,低消費電力と廉価性を実現する・2.4GHz帯(ISM : Industrial, Scientific and Medical)バンドの使用など全世界的な利用を考慮する・誤接続,盗聴防止のためのセキュリティ機能を盛込む・上位に対して透過的なインタフェースを提供し既存技術への適用性を確保する・データだけでなく音声の併用も考慮する・1対1だけでなく1対複数の接続形態も提供する・相互接続性を確保するなどの観点から仕様が定められている.. 2.0+EDR(enhanced data rate)が発表され,2005年から対応製品が市場に出始めている.Bluetoothの規格は特定の企業に独占されたものでなく,この仕様に賛同する企業であれば仕様に基づく商品開発,製造販売を行うことができるというオープンな業界標準を目指している.2001年1月時点で2100を超える企業・団体(2004年時点で約2000)がこの仕様に賛同してSIG(SpecialInterest 1.1では13のプロファイルが定められている(表5・6).さらに,標準化として,SIG のワーキンググループの中で表5・7のような事項について検討が進められ2001年中ごろより順次仕様に追加された.8―105図5・17 Bluetooth認証. 相互接続性と認証Bluetooth SIG のメンバが開発したBluetooth仕様に基づく製品がこの技術仕様に適合し,仕様の範囲内での相互接続性(interconnectivity)を保証するためにBluetoothロゴ認証制度が設けられており(図5・17),ロゴ認定判定者(BQB : Bluetooth qualification body)と公認テスト機関(BQTF: Bluetooth qualification test facility)の設置が進められている.BQB で認証を受けた場合にのみBluetoothのロゴを表示することが許される仕組みになっており,認証を受けた機器はSIG のサイトに公開することが可能となる.Bluetoothでは相互接続性を確保するため用途ごとの接続規定としてプロファイルを定めている.同一のプロファイルをもった機器どうしは相互接続できるがBluetoothを搭載はしていても異なるプロファイル間での接続性は保証されない.Bluetoothを搭載さえしていれば何でもつながるという誤解を招かないよう,認証はプロファイルベースで行い,Bluetoothのロゴと併せて適合プロファイルも機器に明記することになっている.なお,Bluetoothのロゴ認証はBluetooth仕様への適合性と接続性を保証するためのものであり,製品を製造販売するためには製品を利用する国ごとの法規制に適合する必要がある.例えば日本国内であればTELEC(Telecom Engineering Center)やJATE(Japan Approvals Institute for Telecommunications Equip-ment)などの適合認定も併せて取得する必要がある.
