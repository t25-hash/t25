# 5・3・5 携帯端末とJavaの標準化動向

> **章**: 第5章 情報通信機器
> **節**: 5・16 に示す。パケットの伝送、接続とリンク制御を行うベースバ

現在、PDA 分野の主要なOS としては、PalmOS,Windows-CE,EPOC、ザウルスOS など、特定企業あるいは企業連合によって定められたOS が利用されている。従来の携帯端末ではビルトインのアプリケーションを中心に構成され、限られたカスタマイズ手段しかなかったが、さまざまなユーザニーズの拡大、利用シーンの分化により、PDA の世界にもオープン化の波が押し寄せている。機器メーカ自身ですべてのユーザニーズに応えることはもはや不可能になってきており、サードパーティやインハウスの開発者によるアプリケーション開発の重要性が高まっている。そのためには開発ツール、開発環境の提供と技術情報の公開が必須となりつつある。C言語等から利用する固有のライブラリが機器やOS ごとに提供されている場合は、各個の技術の習熟に時間を要すること、機器ごとにアプリケーションを移植する必要があることから、クロスプラットホーム性の高い技術、特にJava(ジャバ)に対する関心が高まっている。デスクトップやサーバの世界では実用化の進んでいるJavaだが、携帯電話や携帯端末をはじめとする組込み分野においても、J 2ME(Java2platform micro edition)として仕様の策定が進められている。J 2MEは、VM(virtual machine、バーチャルマシン)などプラットホームの基本部分を規定するコンフィギュレーションと呼ばれる部分と、その上でバーティカル市場や製品カテゴリごとに定義されるプロファイルと呼ばれるAPI群からなる。コンフィギュレーションとしては、CLDC(connected limited device8―106情報・メディア機器図5・18 コンフィギュレーションとプロファイルconfiguration)と、それよりもリッチな機能を提供するCDC(connected device configuration)の二つが定義されている。CLDCは、主として携帯電話など、極めて限られたハードウェアリソースしかもたない機器を対象とするもので、Javaの言語仕様やVM の仕様自体を大幅にサブセット化している。その上のプロファイルとしては、携帯電話を対象とするMIDP(mobileinformation device profile)やローエンドPDA を対象としたPDA profile(仕様策定作業中)がある。また、NTT ドコモは、Javaコミュニティプロセス(JCP)の外でMIDPとは別の独自のプロファイルを定義している。一方、CDCは、Javaアプレットを実行できる従来のPersonal-Java相当の機能をサポートする機器を対象としているが、機器のカテゴリによってはGUI(graphical user interface)を必要としないものや分散オブジェクト機能を必要としないものもあり、基本部分のAPI(application programming interface)群となるFoundation profile,RMI(remote method invocation)機能を提供するRMI profile,PersonalJava相当のGUIを提供するPer-sonal profile(仕様策定中)がある。こちらは、CLDC上のプロファイルと異なり、CDCに加えてFoundation profileのみの形や、Foundation profileとRMI profileといった形で、必要なプロファイルをネストさせてサポートすることになる(図5・18)。これらJava関連の仕様は、Javaコミュニティプロセスという中立的な手続きに従って策定される。当初サン・マイクロシステムズ社がJavaの仕様を独占しているという批判があったのに対して制定されたもので、新規仕様の提案、専門家グループの招集、専門家グループによる開発作業、仕様の公開レビューなどの手続きをルール化し、外部の監査機関によってこのルールにのっとって仕様化作業が進められているかどうかがチェックされている。ウェアラブル(wearable)という言葉が広く世界で使われるようになったのは1990年代終わりからである。これを最初に意識的に用いたのは1996年8月のボーイング社によるウェアラブルコンピュータ(wearable computer)に関する小規模な会議といわれ、パーソナルコンピュータ(PC)を小形化し身に付けたいという素直な欲求に端を発して用いられた用語である。その観点で改めてルーツをたどると、最初のウェアラブルPCは1966年にルーレット予測を目標にMIT(マサチューセッツ工科大学)が開発したものであろう。もっとも、時刻という情報を身に付けるようにしたことから、腕時計こそが最初のウェアラブル情報機器ともいえる。腕時計も機械式から電子式に変化する中で、時刻以外のさまざまな情報を扱うことのできる情報機器へと進化している。ウェアラブルとモバイルの境界は必ずしも画然としていない。携帯電話が5・1節で扱われていることから、5・4節ではウェアラブルを身体に固定する機器と狭義にとらえ、腕時計型を中心に解説する。情報機器の小形化、高機能化は確実に進展しており、ウェアラブル情報機器もポケットなどに入れるものまで含めた広義の解釈ではもちろんのこと、狭義においてもその重要度が飛躍的に増すことは間違いない。図5・19通信機能付き水深計腕時計図5・20
