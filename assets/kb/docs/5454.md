# 5・4・1 腕時計型

> **章**: 第5章 情報通信機器
> **節**: 5・4 ウェアラブル情報機器

時計の技術史の基軸は高精度化の追求にあった。この過程において1675年のホイヘンスによるテンプゼンマイの発明がなされたが、この発明が携帯化への道をも開いたことは注目に値する。以後、腕時計は人間が身に付ける唯一の機器として発展してきた。1969年には水晶腕時計が製品化され、半導体や液晶技術の発展に伴い腕時計は情報機器としての進化を始めた。1980年代に入ると、センサを内蔵し時刻以外の情報を扱うようになり、1990年代にはパソコンなど他の情報機器と通信するに至っている。a. センサタイプ電子化された水晶腕時計にさまざまな小形センサを搭載し、簡易な小形情報機器とする試みは数多く行われてきた。これらの腕時計を、各センサが計測する情報で分類すると以下のようになる。環境情報:温度、圧力、方位身体情報:心拍、血圧、加速度、指紋AV 情報:音声、画像このうちの環境情報:圧力の例として、通信機能付き水深計腕時計について少し詳しく解説する。図5・19は外観である。図5・20はシステムブロック図で、標準的な時計システム、水圧センサ、A-D コンバータで構成される。水圧センサは、ダイアフラムチップをセラミック容器に実装してシリコーンゲルで封止したものである。このチップはマイクロマシニング技術でダイアフラム状に加工したシリコーンで、ダイアフラム面上にブリッジ抵抗体を形成している。このセンサからの信号を増幅回路で増幅し、A-Dコンバータに入力する。A-D コンバータは、水深計測範囲、センサ特性のばらつき、気圧補正範囲などを考慮し、1V/12ビットとなっている。1ビット当たりの電圧が非常に小さいため、多重積分方式を採用している。また、センサは個々の感度とオフセットのばらつきが大きいため、基準圧力で測定した結果を不揮発性メモリに書き込んでディジタル的に補正している。メモリは潜水を行った際の潜水中の時間経過に対する水深データ等を記憶しておくものである。このデータは対角1インチ、56×32画素の液晶画面にそのままの数値、あるいはグラフィック化されて表示される。また、通信回路を介してシリアル信号に変換してPCに送ることもできる。ただし、時計回路は3V 系で動作しており、端子数を削減する目的から半二重通信を行っているので、専用のユニットを経由してPCと通信することとなる。PC側のインタフェースはシリアルまたはUSB を用い、接点が不要な赤外通信も備えている。マイコン(マイクロコンピュータ)は8ビット、クロック数は100kHz から数MHz,RAM は16kB(キロバイト)で、時計用としては高レベルの性能を実現している。腕時計型センサタイプとして今後の発展が期待されるものの一つは、身体情報センサを備えた常時装着型のウェアラブル生体情報機器である。その背景には高齢化や健康志向といった社会環境の変化と、機器の小形高性能化や通信ネットワークの発展といった技術の進化とがある。日常生活を行う中で健康状態などを継続的にモニタしてネットワークに発信することで、従来の医療システムではできなかった個に対応したきめ細やかな医療や遠隔診断が期待できる。この意味でウェアラブル生体情報機器には大きな社会的インパクトが期待される。このようなウェアラブル生体情報機器の実現に向けて、例えば、日常の心拍を計測し、そのスペクトル解析から心拍揺らぎを算出し、ストレスや疲労度を評価する研究が行われている。この研究では120×65×22mmとやや大きめの測定器を用いているが、時計型や指輪型の心拍計測器の開発も進んでおり、ウェアラブル生体情報機器としての展開が期待できる。心拍計測方式としては、裏ぶたと表側を電極として両上肢間に誘導される心電図を取得する方法、血液量変動を光学吸収で見て脈波を検出する方法、圧電素子をアレイ状に並べたセンサで血管部の圧力を見て脈波を検出する方法などがある。心電図、筋電図、脳波では体表面に誘導される電位は数V から1mV 程度の微弱信号であるため、50 (60)Hzの交流誘導雑音を高精度の差動増幅器で消去している。ただし、体を動かしている状態ではノイズの問題が残っており、いつでもどこでもという理想的なシステムの実現は、計測技術および生体理解の両面でいまだ過渡期といえる。腕時計型センサタイプとして、もう一つの期待される応用は個人認証である。電子決済あるいはキーレス社会などの便利な生活が今後実現されていくが、そのためには個人を正確に特定することが極めて重要となり、ふだん身につけている機器にその機能があると利便性が大きい。現在広く利用されているパスワードでは、それを知った他人との識別ができないという安全性にかかわる問題がある。このため、個人の身体的特徴、すなわち顔、声、指紋、虹彩、血管パターンなどで識別する手法が数多く検討されている。ウェアラブル情報機器で用いるには装置の小形化が必須であり、これらの中でも指紋認証装置は有力候補といえる。b. 通信タイプ現代社会ではテレビや携帯電話などに代表される無線情報が生活環境の一部となっている。これら通信のもつ利便性を腕時計型で実現しようとする試みも数多くなされてきた。古くはAM やFM ラジオ付き腕時計に始まり、1982年にはゲストホ8―107スト型液晶パネルを搭載した腕テレビが開発された。さらに、ページャ(無線呼出し)やメッセージ伝送通信機能の付いた腕時計型の情報受信端末が、数百MHz帯の専用電波や商用FM ラジオ放送のサブキャリヤを用いて実用化された。図5・21はFMサブキャリヤに多重化して送信された情報(ニュース、天気予報などブロードキャスト情報やメッセージ、予定などの個人情報)の表示例である。腕時計型のPHS や携帯電話についても精力的な開発が行われている。PHS では音声認識で入力キーを使わないタイプや、図するタイプなどが開発された。また携帯電話でもGSM 方式やCDMA 方式に対応した開発が行われている。図5・23はCDMA 2000対応の腕時計型携帯電話である。ディスプレイの右側には十字キー、時計のベルト部分にはテンキーが組込まれており、IrDA(アイアールディーエー)によるワイヤレスイヤホン、プラグイン形式のディジタルカメラ、カラー液晶、音声認識による応答なども備え、待受け時間は150時間、通話時間は100分である。さらに、図5・24に示したようにバンド部にマイク、時計相当部に振動体を備え、振動体からの振動を骨伝導で伝達する、いわば手を受話器にした腕電話の試みもなされている。ただし、現時点ではこれらのほとんどが製品として広く普及するには至っておらず、大きさ、動作時間、入力などで依然として多くの課題を抱えている。これらの課題が克服されれば腕時計型の通信機器も大きな発展が期待される。なお、GPS(global positioning system)信号を受信して経度緯度の位置を割出すGPS 時計も、このカテゴリの製品である。国内では正確な時刻情報が福島県(40kHz)と佐賀県(60kHz)にある二つの基地局から発信されているが、これを受信し時刻を修正する電波修正時計も同様である。c. PCタイプデータプロセシングやデータキャリヤ機能をもちながら、腕に付けることができるのがPCタイプである。例えば、ホストマシンとの通信により、カレンダー、アドレス帳、メモ帳、ToDoリストなどPIM(personal information manager)データを腕に移して運ぶ機能であり、このようなことを実現したさま図5・23 CDMA 2000対応の腕時計型携帯電話図5・22 腕時計型PHS図5・24 骨伝導を利用した腕時計型携帯電話図5・21 FM サブキャリヤ情報受信端末の表示例8―108情報・メディア機器ざまな超小形PCが製品化されている。システム基本構成は、携帯型のPCやPDA と同様であるが、腕時計型に小形化するために機能を特化し、CPU やメモリサイズを必要最低限のものにして低消費電力化することが必要である。具体的には、時計表示のみを行っているときに未使用のシステムを分離したり、複数のクロックを使い分け、どのような状態でも必要最小限の電力で動作させる工夫がなされている。つねに身に付けているという特徴を生かして、電子決済(elec-tronic settlement)への応用も有力である。電子決済は、決済した金額が後に口座から引き落とされるクレジット方式と、あらかじめある金額をチャージしておきその範囲で決済するプリペイド方式に分けられる。クレジット方式としては、内部に非接触ICカードのICチップならびに送受信用アンテナを搭載したものが製品化されている。この腕時計型PCを非接触ICカードリーダにかざすことにより、内蔵されたICチップとリーダとが通信して決済が実行され、後に指定口座から引き落とされることになる。プリペイド方式には磁気カード型と非接触ICカード型があり、腕時計型としては後者が製品化されている。あらかじめチャージ機で一定金額までのチャージができ、利用者はその金額の範囲で決済が可能となる。腕時計型での電子決済ではクレジット、プリペイド両方式ともに非接触ICカード機能を搭載することになる。時計ムーブメントには金属が多く使用されているので、腕時計に電子決済機能を付加する製品では、この時計ムーブメントの金属が通信機能に影響を与えないようケース内での空間配置に工夫がなされている。以下では、腕時計型での用途開発のために試作された超小形PCの例について少し詳しく述べる。これは音声と画像情報を使うマルチモーダルなユーザインタフェースを設計思想においた腕時計型PCであり、外観と仕様を図5・25と表5・8に示す。この腕時計型PCは単独のPDA 機能だけでなく、Bluetooth(ブルートゥース)通信と指紋センサを利用し、PCを操作してメールやスケジュールを見たり、携帯電話をポケットやかばんに入れ図5・25 腕時計型PC表5・8 腕時計型PCの仕様ハードウェア本体部分65×46×16mm/43gCPU32ビットMPU(18~74MHz)入力機能タッチパネル/リューズスイッチ/ボタン表示装置320×240画素モノクロ液晶パネルメモリDRAM 8MB/FLASH 16MB通信機能Bluetooth,IrDARS 232C(クレードル経由)その他スピーカ/マイク/バイブレータ指紋センサ/加速度センサ電源リチウムイオン二次電池ソフトウェアOSLinuxGUIMicrowindowsBluetoothスタックIBM좲BlueDrekar윣윃たまま利用したりすることをねらっている。すなわち、身近なIT機器を直接コントロールするセキュリティ機能をもったユーザインタフェース(本人認証、ポインティング、トリガ、ビューア)デバイスとしての機能である。OS としては、ユーザが機能を追加したりアプリケーションを開発したりしやすくするため、ソースコードやアプリケーションプログラミング環境が利用可能なLinux(リナックス)を採用している。このほか、Bluetoothを利用して近くのコンピュータに音声信号を送り、負荷の大きい音声処理を任せることで、あたかもこの腕時計型PCが音声処理機能をもつよう動作させることも可能である。またBluetoothのホットスポット(hotspot)を利用して、腕時計型PC画面に、現在いる場所に即したサービス(空港のフライトインフォメーション、レストランの本日のお勧めメニュー等)を表示させたり、これを個人のID としてホテルのチェックイン、クレジットカード代わりの支払いを行うなど、さまざまな利用方法が想定される。
