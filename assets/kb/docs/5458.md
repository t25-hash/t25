# 6・1・1 ルータ

> **章**: 第6章 ネットワーク接続機器
> **節**: 6・1 ルータ、サーバ

a. コアルータ(core router)WWW(world wide web)や電子メールに代表されるインタネットアプリケーションが急速に普及するにつれて、インタネットトラフィックは急激に増加し、同時にインタネットバックボーンに必要な回線容量も急激に増加している。このようなインタネットバックボーンに必要な高速のIPインタフェースをもち、インタネットバックボーンに接続されるルータをコアルータ(バックボーンルータ)と呼ぶ。インタネットバックボーンに必要な回線容量は、およそ4年で10倍のペースで伸びており、1997年には1G(ギガ)bpsを超え、それまでルータに装備されていたインタフェース速度を上回ってしまった。このため、各ルータベンダはOC-48(2.4Gbps)POS(point-to-point protocol over SONET/SDH)と呼ばれる高速IPインタフェースを備えた大容量ルータ(ギガビットルータ)の製品化を開始した。. 単体大容量ルータ従来型の単体ルータのアーキテクチャ(図6・1)を基に高速化を図ったルータである。従来はソフトウェア処理であったため高速化のボトルネックとなっていたパケットフォワーディングやフィルタリング処理といった経路制御部の処理がASICによりハードウェア化された。スイッチングファブリックはクロスバ型スイッチの採用によりスイッチング処理が並列化された。. 並列型大容量ルータ単体型ルータでは交換容量が足りないため、複数のルータを相互接続して交換容量を増加させたルータである。一般にはルータ間のネットワーク速度がルータ内のバックプレーン速度よりも低速のため処理全体のボトルネックになる。並列型大容量ルータではルーチングモジュール間の接続構造を多次元化することにより、上記ボトルネックの解消を図り、複数のルータがあたかも1台のルータであるかのように動作させている。現在、コアルータと呼ばれるものはOC-48(2.4Gbps)やOC-192(10Gbps)のインタフェースを装備し、単体型では数十から図6・1 ルータの内部構造図6・2 コアルータとエッジルータ数百Gbpsのオーダ、並列型ではT(テラ)bpsのオーダの交換容量をもっている。また、コアルータでは交換容量だけでなく、信頼性でも高いレベルが要求されており、ルーチングモジュールや電源等のルータ内の各機能の冗長化が図られている。b. エッジルータ(edge router)バックボーン間を接続するコアルータの周辺に設置され、エンドユーザを収容するアクセスルータを集線するルータをエッジルータと呼ぶ(図6・2)。エッジルータでは交換能力としては数Gbpsのオーダで十分であるため、通常は単体型ルータが使用される。コアルータとエッジルータの名称の違いは、ルータの配置位置による相対的なものであり、ルータ内部のアーキテクチャに関しては交換容量を除いてほとんど相違はない。IPパケットの大容量交換能力と高速IPインタフェースが求められるコアルータと異なり、エッジルータではファストイーサネット(100Mbps)からギガビットイーサネット(1Gbps)程度のイーサネット回線の収容数、POS やATM(asynchronous transfermode)などのイーサネットとは異なるデータグラム層をもつプロトコルの収容可能性、アクセスコントロールに代表される多彩なフィルタリング機能、IP-VPN のような仮想閉域ネットワーク構成機能等の多ポート多機能が求められる。c. RAS(remote access server)RAS とは、遠隔地にある端末(RA Client: remote access client)にRAS が設置されているネットワークへWAN(wide area network)回線を介した物理的または論理的な回線を確立させるサービス(リモートアクセスサービス、remote access service)を提供する機能をもったルータ装置のことである。RAS の中でも、大規模なサービスを提供するものを、BRAS(broadband RAS)と呼ぶこともある。RA Clientは、RAS が提供するリモートアクセスサービスを利用することで、RAS の設置されているネットワークにあたかも直接接続しているかのように利用することができる。リモートアクセスサービスの基本的な構成要素を図6・3に示す。. リモートアクセスプロトコルRAS は、PPP(point-to-point protocol)、 PPPoE(PPP over ethernet)、 PPPoA(PPP over ATM)などのプロトコルを用いて、RA Clientとの接続確立・データ伝送を行う。これらは、リモートアクセスに用いるWAN 回線の種類によって使い分けられる。. 認証機能RAS は、PPP接続ネゴシエーションの際に、認証プロトコルであるCHAP(challenge handshake authen-tication protocol)、PAP(password authentication proto-col)により、許可を与えていないRA Clientからの接続を阻止することができる。RADIUS(remote access dial-in user ser-vice、ラディウス)プロトコルを利用して、RAS と認証サーバ(RADIUS サーバ)とで連携して認証を行う場合もある。. IP ルーチング機能RAS はRA Clientとの間で確立したPPP接続と、RAS が接続されているネットワークとを結ぶルータとしての役割を果たし、RA Clientとネットワーク間のIP通信図6・3 リモートアクセスサービスの基本的構成要素を実現する。d. ダイヤルアップルータ、ブロードバンドルータ近年の家庭内LAN(local area network)構築の一般化や低価格・定額の通信サービスの普及に伴い、LAN をISPのネットワークと接続したいという需要が高まっている。これに伴って、ダイヤルアップルータやブロードバンドルータと呼ばれるルータ装置が普及しつつある。ダイヤルアップルータやブロードバンドルータは、WAN 回線を介してRAS とリモート接続を確立し、LAN と遠隔地のネットワークとのルーチングを行う。LAN 側の端末は遠隔地のネットワークのリソースを透過的に利用することができる。ダイヤルアップルータは、接続先ネットワークのRAS とのWAN 回線にISDN 回線やアナログ電話回線を利用する場合に用いる。このとき、ダイヤルアップルータは、c項で述べたリモートアクセスサービスのRA Clientの役割を果たし、LAN とネットワークとのPPPセッションとのルーチングを行う。ブロードバンドルータは、ダイヤルアップルータとほぼ同様の役割を果たすが、ADSLやCATV インタネットなどの常時接続型のブロードバンド回線を用いてネットワークに接続する場合に用いる。ADSLなどを利用するためのPPPoE・PPPoA 機能や、CATV 回線インタフェース機能を備えている製品が一般的である。市販のダイヤルアップルータ、ブロードバンドルータ製品は、さまざまな付加機能を備えている。代表的なものを以下に示す。. NAT機能、NAPT(IPマスカレード)機能LANでは一般的にプライベートIP(internet protocol)アドレスが用いられる。このため、接続先ネットワークでグローバルIPアドレスや別のプライベートIPアドレスが用いられているような場合には、一般に正しいIP通信ができない。ダイヤルアップルータ、ブロードバンドルータは、NAT(network address translation)機能、NAPT(network address port translation)機能を用いてIPアドレスやTCP/UDPポート番号の付替えを行い、この問題を解決する。. IP フィルタリング機能外部からLAN への不正アクセスを防止するために、管理者が指定したアプリケーション以外のIPパケットを、ダイヤルアップルータ、ブロードバンドルータで破棄する機能である。. 無線LAN アクセスポイント機能近年、IEEE802.11b無線LAN 製品が急速に普及している。ダイヤルアップルータ、ブロードバンドルータ製品にも、LAN インタフェースとしてIEEE802.11b無線LAN アクセスポイント機能を内蔵したものが登場している。e. MPLS(multi protocol label switching)。 MPLS 登場の背景1990年代半ば、拡大を続けるIPネットワークはIPトラフィックの増加と経路情報の肥大化に拍車をかけ、従来のレイヤ3技術を実装したルータの処理能力では、IPパケットを高速に転送することが困難になりつつあった。ルータは、経路情報からIPパケットの宛先アドレスに最長一致する情報を検索し、パケットを出力するインタフェースを決定してIPパケットを転送する。この処理がソフトウェアによって行われていたことと、経路数の増加に伴い最長一致アルゴリズムによる計算量が膨大になってきていたことが、ルータでのパケット処理負荷を大きくする原因であると考えられていた。この問題を解決するため、当時、高速広帯域な転送を実現するレイヤ2ラベルスイッチング技術であったATM(asynchronous transfer mode)を利用して、IPトラフィックをATM ネットワーク上で転送する方式が検討された。しかしながら、IPとATM は全く異なる思想に基づき設計された技術であるため、経路制御やアドレス割当等の方式が全く異なり、これら二つの技術を利用したネットワークの運用は複雑であった。この複雑さを軽減するために登場したのが、マルチレイヤスイッチング技術である。この技術は、経路制御にはIPの技術を利用し、転送にはATM のラベルスイッチング技術を利用することに8―113より、IPとATM の複雑な対応付けを必要とせず、速度とコスト効率が高いIPルータを実現するものであった。マルチレイヤスイッチング技術は、ハードウェアによるパケットの高速転送処理の実現や、宛先アドレスを含むさまざまなパケット情報に基づく、明示的な経路制御によるトラフィックエンジニアリング、およびQoS(quality of service)の実現といった利点を有していたが、いくつかの問題も抱えていた。すなわち、各ベンダが独自開発した技術を採用したため相互運用性に乏しかったこと、イーサネットを代表とするATM 以外のレイヤ2技術に対応できなかったことなどである。このような状況のもとで、さまざまなレイヤ2技術に対応できるマルチベンダ標準策定に対する機運が高まり、1997年の初め、IETFはMPLS の標準化作業を開始した。. MPLS の概要MPLS で利用するフレームワーク、およびラベルについては、それぞれRFC3031, RFC3032に規定されている。まずはラベルについて概略を示す。MPLS のパケット転送は、ラベル付替えアルゴリズムに基づいている。MPLSネットワークに到着するレイヤ2プロトコルのパケットが、ラベルのためのフィールドをもっている場合(例えばATM の場合)には、そのラベルがIPパケットに対して付与される。レイヤ2プロトコルがラベルフィールドをもたない場合には、MPLS ヘッダがレイヤ2ヘッダとIPヘッダの間に挿入される(図6・4)。MPLSヘッダは全部で32ビットからなる。具体的には、同様の特徴を有する(例えば宛先IPアドレスが同じ)パケット群に対して割当てるラベルを示すラベルフィールド(20ビット)、実験用に予約されたExpフィールド(3ビット)、ヘッダを多段に挿入したときの最終ラベルであることを示すスタック(S)フィールド(1ビット、MPLS ヘッダは二つ以上挿入することができる)、MPLS パケットの生存期間を示し、ルータを経由するごとに値が1ずつ減算されるTTL(time to live)フィールド(8ビット)からなる。次に、MPLS ネットワークとラベルスイッチングの動作の概略を示す(図6・5)。LER(label edge router)は、IPネットワークとMPLS ネットワークの境界に設置されたルータであり、IPパケットが入ってくる境界ルータを入口LER と呼び、IPパケットが出て行く境界ルータを出口LER と呼ぶ。また、MPLS ネットワーク内に設置されたルータをLSR(label switch router)と呼ぶ。MPLS ルータは、LDP(label distribution protocol)などのラベル配布プロトコルを利用して、同じ特徴(宛先/送信元IPアドレス、宛先/送信元ポート番号、アドレスプレフィクス、優先度、など)をもつパケット群に対して、どのラベルを割当てるかということを、MPLS ルータどうしで情報交換し、ラベル転送テーブルを作成する。図6・5では、アドレスプレフィクスが同じパケットに対して、ラベルを割当てた例を示している。入口LER では、到着したパケットの宛先IPアドレスを調べ、ラベル転送テーブルに基づいて、そのアドレスプレフィクスに見合うラベルを付けて、出力インタフェースに送出して、隣接するMPLS ルータに転送する。LSR は、IPの経路情報とは無関係な、ラベル転送テーブルだけに従って、到着パケットのラベルを付替え、出力インタフェースに送出する。出口LER では到着パケットのラベルを取り外した後、そのパケットのIPヘッダを参照して、IP経路情報に基づいてパケットの転送を行う。. MPLS の適用例1990年代の後半、インタネットの商用性が高まり、その標準プロトコルとしてIPの普及が進んだ。このような環境下でIPハードウェアの市場が確立され、新技術の開発が相次ぎ、IPルータの処理能力も飛躍的に向上した。このこと図6・4 MPLS ヘッダ8―114により、IPルータの処理能力はもはや問題ではなくなったが、MPLS は、パケットの高速転送以外にも、以下に示すような、従来のIPネットワークでは実現が困難であった機能を提供できる。(1) トラフィックエンジニアリング(traffic engineering)トラフィックエンジニアリングとは、ネットワーク全体の使用率を平均化し、ネットワークの性能を最適化するようにトラフィックをさばくことであり、ネットワークの設計や調整に有効である。従来の宛先アドレスに基づく最短経路計算による経路制御では、宛先が同じトラフィックは、すべて同一の経路を経由するため、ネットワークの性能低下の原因となりうる。従来の経路制御は、たとえ混雑していない最短経路以外の冗長経路が存在したとしても、そちらの経路を経由するようにトラフィックを制御する手段を持ち合わせていない。MPLS を利用すれば、ある特定の経路を通過させたいトラフィックに対して、特定のラベルを割当てることにより、ネットワーク管理者が明示的に経路を制御することが可能になる。このことにより、ネットワーク事業者は、増加の一途をたどるIPトラフィックをさばき、限られたネットワーク資源を効率的に運用できるようになる。(2) CoS(class of service)基本的に品質保証機構をもたないIPネットワークにおいて、パケットに優先度を付けることでトラフィックを差異化し、その優先度に見合った転送サービスを提供するフレームワークとして、Diffserv(differentiated services)が標準化されている。CoS とは、比較的大まかな基準に基づいて分類されたパケット群(クラス)を単位として、その単位ごとに適切なサービスを提供する考え方であり、Diffservの概念の基本となっている。MPLS ヘッダには実験用としてExpフィールドが用意されているが、このフィールドとDiffservで利用する優先度ビットを対応づけてCoS を提供することが検討されている。このことにより、従来のIPネットワークと比較して、ある程度の品質保証を含む柔軟性のあるサービスを提供できる可能性がある。(3) VPN(virtual private network)あらゆる企業において、ネットワークの利用を前提とした業務が図6・6 クライアントサーバ方式一般化しつつあり、各拠点間をWAN によって接続し、情報伝達の迅速化や業務効率の向上を図ることがますます重視されている。従来のWAN には、専用線、FR(フレームリレー)、ATM 等が利用されてきた。近年、公衆ネットワークを利用して資源を共有することにより、比較的安価なプライベートWAN を仮想的に構築する技術として、VPN が脚光を浴びている。ネットワーク事業者はMPLS を利用して、ユーザごとにラベルを割当て、トラフィックを完全に分離することにより、VPN サービスを提供することが可能になる。その利点としては、異なるユーザネットワークにおいて同じプライベートIPアドレス空間を使用することができる、ATM やFR と同等のセキュリティを提供できる、VPN を構築するためにユーザ側で特別な装置を必要としない、といったことが挙げられる。このようなことから、今後、MPLS を用いたVPN 構築機会の増加が見込まれる。
