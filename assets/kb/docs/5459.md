# 6・1・2 サーバ(server)

> **章**: 第6章 ネットワーク接続機器
> **節**: 6・1 ルータ,サーバ

a. サーバ専用機とはん用機コンピュータ群をネットワークによって結合し,特定の機能をサービスという形で提供するサーバ(群)と,サーバへサービスを要求しながら目的の処理を行うクライアント(群)に分割する処理モデルのことをクライアントサーバ方式(client-server method)と呼ぶ.これは従来の集中型システムに代わる新しい形態として使用され始めており,近年の計算機の高性能化,低価格化,ネットワークの高速化,広帯域化がその普及に拍車をかけている.特定のサービスを提供するサーバには,ディスクシステムなどの記憶装置をほかの計算機が共有できるようにするファイルサーバ,1台のプリンタを複数の計算機から共有できるようにするプリンタサーバ,データベースに対する関係演算を高速に処理するデータベースサーバ,さまざまなネットワークに接続する機能をもつ通信サーバなどがある.一方,サービスを要求するクライアントは,ネットワークを介してサーバにサービス要求(リクエストメッセージ)を送信し,サーバは要求された処理を行った後,クライアントにリプライメッセージを返信する.一般に多数のクライアントが少数のサーバを利用する形態となる.図6・6はクライアントサーバ方式を図6・5 MPLS ネットワークとラベルスイッチング図示したものである.サーバはデータ(ファイル,データベース)の集中管理や,高性能ハードウェア(プリンタ,通信)の共有といった目的に合わせ,ある機能に特化したハードウェア構成を有する計算機が使われる.また,複数のクライアントに対してサービスを行う性格上,大量のリクエストを処理できる十分な性能,連続運転に耐えうる高い信頼性が求められる.このうち高性能化に関しては,高性能CPU,高速大容量なメインメモリ,ディスクシステムの採用に加え,1台のCPU の能力を超えた処理性能を実現するマルチプロセッサ構成,データを複数台のディスクに分散配置しアクセス速度を向上させるディスクストライピング技術などが採用される.また多数の入出力装置を同時に効率よく動作させるためI/O関連の強化が行われる.例えばCPU,メモリ,各種制御装置を接続するシステムバスをクロスバとし,データ転送時にバスの占有を可能にする構成や,ディスクシステムやネットワークインタフェースを高速,広帯域化し,高いスループットを実現するといった構成が採用される.またクライアントサーバ方式では,クライアントの増加に合わせてサーバを追加していくことにより,性能の向上を図ることも容易である.一方信頼性に関しては,ハードウェアの障害を自動的に検出(パリティチェック,温度センサ),記録(ログ)し,管理者に通報する機能,ハードウェアを冗長構成とし,障害発生時には故障した機能を切り離す自動縮退機能,故障したディスクシステムやネットワークインタフェースを運用中に交換する機能(活線挿抜,ホットスワップ)といった技術が採用される.さらに,サーバの周辺環境についても,無停電電源装置による電源の供給,サーバ自体を一般のユーザからアクセスできない場所に隔離し,セキュリティを確保するといった特別な配慮を行うことが多い.b. 大容量サーバb項では,インタネットの主流なサービスを提供するWWW(world wide web)サーバを中心とした大規模化システム構成技術を記述する.. 大規模化サーバ技術の分類大容量サーバシステムを実現する基本的な考え方として,サーバマシンのスケールアップとスケールアウトがある.前者はCPU やメモリを追加してマシン単体の処理能力を向上させる方式であり,後者はサーバマシンを複数台増設する方式である(図6・7).スケールアップ方式はDB サーバなど,集中管理に適したシステムに向くが,能力向上度はサーバアプリケーションに依存する.また,スケールアウト方式は,WWW サーバなどの処理系に適しており,台数比例で効力が向上することが多い.処理が単純で,サーバを分散させてもデータの整合性がそれほど厳しくないためである.スケールアウト方式の中には,複数サーバマシンを管理する方法として,さらに負荷分散方式,DNS ラウンドロビン(DNS round robin)方式がある.さら図6・7 サーバマシンの大規模化技術図6・8 スケールアウト方式負荷分散方式は,複数のサーバマシンを一つに見せること,エンドユーザ(Webブラウザのクライアント)からの要求をあるルールに従って各サーバへ振り分けること,いったん接続されたエンドユーザとサーバの対向関係を維持することの三つを実現する仕組みであり,負荷分散装置を利用してシステムを構成する.DNS ラウンドロビン方式は,エンドユーザの閲覧要求に記載されたWebページの場所を示すURL(uniformresource locator)から実際のサーバマシンのアドレスを回答するDNS(domain name system)の仕組みを利用し,あらかじめ登録してある複数のサーバアドレスを順番に回答することにより,エンドユーザの要求を複数サーバマシンに振り分ける.負荷分散方式では,通常,負荷分散装置はサーバマシンと同じ場所で接続されることが多く,サーバマシンの稼働状態に合わせた振分け処理,サービス種別ごと(Webサーバ,DB サーバなど)への振分けなどが可能である[図6・9(a)].しかしながら,サーバからエンドユーザへのデータの流れに対して直列に負荷分散装置が構成されるため,負荷分散装置のデータ処理性能が全体の疎通能力に大きく依存する.現状では一般的に100Mbpsレベルの性能までの大規模化が可能である.一方,DNS ラウンドロビン方式は,複数サーバマシンはネットワークのどこに配置していてもよく,地理的な制約が少ないが,各サーバマシンの状態に合わせた振分けができないという欠点がある[図6・9(b)].データ疎通量は,負荷分散方式よりも性能改善効果が大きい.これは,WWW サーバとエンドユーザは直接接続され,DNSの性能に依存しないためである.. 高信頼サーバc項では,インタネット上で高信頼サービスを提供するためのサーバシステム構成技術に関して,最近主流のPCサーバの信頼化向上技術を記述する.. 高信頼サーバ技術の分類フォールトトレラントサーバ(FTS : fault tolerant server)とは,サーバマシンを無停止で運用することを目的としたアーキテクチャで,FTS 協議会などで具体的なハードウェア構成を検討している.そのFTS は,プロセッサ,メモリ,ハードディスクなどをすべて物理的に二重構成とし,一つの筐体に収容したものである.同一の処理を同時並行処理するため,一方のデバイスで故障が発生しても,システム全体を停止させることなく,他方のデバイスが処理を継続することができる(図従来の高信頼化技術(図6・12)としては,共有ディスクによるクラスタ構成により,障害発生時までにディスクに書き込まれたデータを,代替サーバへ引き継ぐこと(ロールバック,ロールフォワード)により,障害発生後のデータ整合性の矛盾を排除することが考えられていた.しかし,代替サーバへの切替え時間がかかるため,完全に無停止状態での運用はできなかった.また,b項で述べた負荷分散方式は,WWW サーバなど単純なトランザクション処理系を複数持ち合わせ,あるサーバが故障した場合は,それ以外のサーバによりサービスを継続できる構成である.しかし,故障サーバとエンドユーザとの間の接続はいったん切れてしまうため,エンドユーザが別サーバへ再度接続するまでの間,サービスは中断してしまう.FTS は,これらの二つが代替的システム技術であるのに対し,完全二重化システム技術により,はん用PCベースの技術を用いることにより,安価に無中断サービスを実現することを目的としている.. FTS の動作原理FTS のハードウェアの特徴は,ディ図6・11 FTS システム構成スクやNIC(network interface card)などのI/O部分を担うバススロットと,プログラムステップを実行する演算装置(CPU,メモリ,チップセット)を分離し,それぞれを故障検出/分離回路で接続する.さらにバススロットおよび演算装置を同期並行動作させることで二重化を図り単一障害点をなくしている.論理的には,FTS はクライアントから見て1台のサーバとして動作する.以下FTS の動作について記述する.1クライアントからFTS に処理要求を出す.2ネットワークを経由して処理要求が2台のバススロットに到着する.3それぞれのバススロットは故障検出回路を通じて2台の演算装置に処理依頼を行う.42台の演算装置で同期実行(ロックステップ)され演算結果を故障検出回路に出力する.5故障検出回路で,2台の演算結果をつねに比較し,一致していればバススロットに演算結果を通知する.6演算結果をFTS のディスクに蓄積する場合は2台のディスクにミラーリングされ,またネットワークに対して演算結果を流すのであれば,どちらか一方のバススロットを選択して当該スロットに接続されたNICからクライアントへ演算結果を通知する.もし,演算装置でハードウェア故障やソフトウェア停止などで正しく演算結果を返さなかった場合,故障検出回路で演算結果の不一致を検出し,正しく演算結果を返さなかった演算装置を切離す.バススロットも同様に常時故障監視され,故障検出した時点で故障スロットを切離す.いずれの場合もFTS はアラートを出し,当該装置の交換を促す.FTS は,システム停止することなく当該装置の交換を行うことができる(ホットスワップ).以上の動作原理により,FTS はシングルサーバとして動作し,各コンポーネントの故障検出・切離しを自動的に行うことで,完全無停止を実現している.協調型ディジタル機器とは,ネットワークに接続されることを想定した機器認識機能や,他の機器との連携を前提としたアプリケーションを備えた,パーソナルコンピュータ(パソコン)やPDA(personal digital assistance)などの情報機器,およびネットワーク機能をもつAV(audio visual)家電品の総称である.その対象はおもに個人,家庭およびオフィスでの利用を想定したものになってくる.また,協調型ディジタル機器とネットワークを解説するうえで,具体的な機器の例を挙げるが,その機械としての動作原理そのものについては,ここではあえて触れない.あくまでその機器がネットワークの中でどのような位置付けにあるか,他の機器との相関関係を明確にすることを目的としている.
