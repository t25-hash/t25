# 7・1・3 データ圧縮技術

> **章**: 第7章 情報圧縮・セキュリティ技術
> **節**: 7・1 圧縮技術

シャノン(Shannon, C.E.)は、情報理論を創始し、メッセージの生起確率により定まるエントロピーという概念を導入し、情報の性質を定量的に示した。情報に含まれる冗長性を取り除いた本質的な情報の量がエントロピーである。あるメッセージが記号で表され8―125ているとき、それぞれの記号のエントロピーは、その出現確率の対数の負数として定義される。エントロピーは、メッセージ中の情報量をビット数で表すために2を基数とする対数で表す。メッセージ全体のエントロピーは、それぞれの記号のエントロピーの総和となる。ビット数=-log(出現確率)エントロピーを超えて圧縮した場合には、元の情報をひずみなしで復元することができないので不可逆圧縮と呼ばれる。7・1・3項では、完全に元に戻せる可逆圧縮について述べる。a. データ圧縮の目的データ伝送時間短縮およびデータ記憶場所の利用効率の向上である。b. データ圧縮の原理データ圧縮とは、記号列を受取り、それを符号に変換することである。その際、頻繁に現れる記号列を短い符号で表し、めったに現れない記号列を長い符号で表す。これがデータ圧縮の原理である。効果的なデータ圧縮が行われたときには、得られた符号列が元の記号列よりも小さくなる。c. データ圧縮の方法データ圧縮は、モデルと符号化とからなる。記号または記号列は、モデルに基づいて符号化する。モデルとは、入力記号または記号列を処理してどの符号を出力するかを決める規則集合のことで、統計的モデルと辞書モデルとがある。符号化とは、モデルに基づいて符号を出力することである。統計的モデルでは、ハフマン符号(Huffman coding)とシャノン・ファノ符号(Shannon-Fano coding)、算術符号(arithmetic coding)、LZ符号(Lembel-Ziv coding)などが使われる。辞書モデルでは、後ほどd項で具体的に触れるとおり単純な符号化が行われている。データを圧縮するには、入力記号を各記号がもっている情報量と同じビット数で符号化する必要がある。データ圧縮という観点からいって、英文テキストをASCII(アスキー)コードで符号化するということは、最適な手法ではない。最も単純な静的統計モデルでは、出現確率を固定的な表にして使用する。入力される記号列に対してこの表による統計情報が合わないとき、圧縮率は悪くなる。入力記号列を処理しながら出現確率の表を適応するように修正する適応的統計モデルもある。辞書モデルでは、入力記号列を読み込み、辞書に現れる記号列を探し、合致があればその記号列に対する符号を出力する代わりに辞書に対するポインタを出力する単純な符号化が行われている。入力記号列の性質が途中で変わらないならば、固定的な辞書を使うのが単純であるが、圧縮した記号列と一緒に辞書も送らねばならない。ただしLZ 77やLZ 78による辞書モデルでは、移動窓が使われているので辞書を送る必要がない。例えば学術論文など、略称による語句の置換えは、読者が頭の中で辞書を使って理解しているのである。d. データ圧縮の実際モデルとして静的統計モデルを使い、シャノン・ファノ符号で符号化することにより、記号列aa bbcccc dddd eeeeeeeeをデータ圧縮してみることにする。手順1)
