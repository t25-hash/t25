# 7・2・3 鍵配信(key distribution)、鍵共有(key sharing)

> **章**: 第7章 情報圧縮・セキュリティ技術
> **節**: 7・2 情報セキュリティ技術

二者間で通信する際に通信路上でデータを秘匿化するためには、共通鍵暗号アルゴリズム等を用いてデータを暗号化する必要がある。しかし、送信者と受信者間で鍵の共有がなされていないと正しく相手が復号化できないため、安全な鍵共有の方式が必要とされる。その鍵共有の代表的な方式としては、第三者機関である鍵配送8―129センタを用いる方式(Kerberos等が有名)と、二者間で個別に鍵を共有する方式(Diffie-Hellman 鍵配送方式)、認証局(certification authority)を用いて鍵を共有する方式などがある。図7・10は、Kerberosにおけるデータの流れを示したものである。送信者A、受信者B は、鍵配送センタCと鍵を共有している(それぞれ、,)。送信者A は、B にデータを送信するに先立ち、鍵配送センタに送信者A と受信者B の識別子を送る(1)。これを受信した鍵配送センタCは、それぞれの識別子から検索した鍵,を用い、2に示すデータを生成する(はタイムスタンプ、はA とB の共有鍵、は鍵を用いて暗号化することを意味する)。これを受信したA は、第一メッセージからおよびを復号化し、鍵を用いて自己の識別子とタイムスタンプを暗号化して第二メッセージとともにB に送信する(3)。これを受信したB は、第二メッセージからおよびを復号化し、鍵を用いてタイムスタンプ+1を暗号化して送信者A に再送する(4)。送信者A はこのタイムスタンプの正当性を評価し、共有鍵を用いて以降の秘密通信を行う。一方、Diffie-Hellman鍵配送方式においては、素数を、原始元をとすると、送信者A および受信者B はそれぞれ=mod,=modを計算する(およびは、A,B がそれぞれ生成した乱数である)。次に、お互いが生成したデータを交換し、A,B はそれぞれ=mod=mod,=mod=modを計算し、この値を共有鍵とする。最後に認証局を用いた鍵共有方式であるが、送信者が生成した乱数は、相手の公開鍵で暗号化されて受信者に送られる。これを受信した受信者は自己の秘密鍵で乱数を復号化し、この乱数を共有鍵として利用する。なお、認証局に関しては次項に詳述する。
