# 7・2・4 項で述べたPKIの普及により、インタネット等のネット

> **章**: 第7章 情報圧縮・セキュリティ技術
> **節**: 7・2 情報セキュリティ技術

ワーク上での認証、承認が可能となり、それに従って電子署名に対する法律の整備が求められてきた。そのような流れの中で電子署名および認証業務に関する法律施行規則(電子署名法)が2000年5月に成立・公布、2001年4月1日に施行(一部規定については3月1日)された。電子署名法は、電磁的記録の情報に対して本人による一定の電子署名が行われているときは真正に成立したものと推定する規定で、対象を私文書に限定している。このことにより、電子署名が手書きの署名や押印と同等に通用する法的基盤が整備された。また、海外ではアメリカで2000年10月に電子署名法が、EU では1999年12月にEU 電子署名指令が施行されている。a. 電子署名とその効力電子署名法第2条1項では、電子署名とは「電磁的記録に記録することができる情報について行われる措置であって、次の要件のいずれにも該当するものをいう」と規定している。要件とは、「当該情報が当該措置を行った者の作成に係るものであることを示すためのものであること」(本人性の確認)、「当該情報について改変が行われていないかどうかを確認することができるものであること」(非改ざん性の確認)とされている。また、電子署名法では特定の技術を挙げてはいない。これは技術を中立にとらえ、特定のメーカの規格による電子署名技術のみを、法的に位置付けるものでないためである。b. 認証機関に関する規定電子署名に関する規定とともに、認証局を運営する認証業務のうち一定の水準を満たす民間の認証機関は国の認定を受けることができる制度もあわせて導入された。民間認証機関の安全・信頼性に関する必要最小限の要件を満たすものを特定認証業務と定義して、特定認証業務を行おうとする者は主務大臣の認定を受けることができる。認定を受けた認証機関は認証業務の安全・信頼性の維持、第三者を含めた利用者保護などの観点から、一定の義務を負う。内容は、業務に関する帳簿書類の作成・保存義務、利用者の真偽の確認に関する情報の目的外使用の禁止、主務大臣による報告徴収および立入検査などを受ける義務などである。認証機関に対しては、このほかに外国の認証事業者などに関する取扱いが規定されている。これは多くの諸外国で認定制度が導入されていく中で、日本の認証機関が海外でも通用するために、海外と相互性のある制度とすることも目的としたものである。海外の動向としては、IETFよりqualified certificate profileがRFC 3039:2001(proposed standard)として発行された。このRFCはRFC2459:1999を基として、X.509証明書を人間が使用する場合のプロファイルを定めたものであり、高度な確信性に適したもの、法的認知に適したもの(例:EU directive)であるという要件を満たすことを目的としている。
