# 8・2・4 装置構成

> **章**: 第8章 社会情報システム
> **節**: 8・2 金融ATM システム

図8・8に装置構成の例を示す。制御部に、必要な機能をもったI/O(input/output)ユニットが接続される。カード明細票ユニット(CIP: card reader/imprinter/printer)は、磁気ストライプ付きカードの磁気ストライプの内容を読み取り、取引明細票にカードのエンボスをインプリントし、さらに取引情報を明細票に印字する機能をもつ。CIPの構成は、カード読取り部、インプリント部、プリント部から構成される。取引明細票は連表となっており、必要な処理が行われた後にカットされ、利用者に放出される部分と取引記録として内部に巻き取られる部分との2枚構成となっている。また、使用されたカードの証拠を残すために、図8・8 ATM の装置構成例8―139カード上のエンボスが取引明細票に転写(imprint)される。通帳ユニット(PBPR : passbook printer)は、通帳に対して取引情報の印字処理を行う。PBPR の構成は、通帳の磁気ストライプの内容を読み取る磁気ストライプ読取り部と、印字部から構成されている。通帳への印字を行う際に、印字行が各ページの最終行に達すると、通帳のページをめくる機能を備えた装置もある。通帳がATM に挿入されると、PBPR は、磁気ストライプに記録されている印字開始ページ/開始行を読み取り、ページの確認および印字開始行までの通帳移動処理を行った後、印字処理を行う。また、ページめくり機能を備えたATM においては、取引結果の印字や、未記帳取引の印字により印字行が印字ページの最終行に達した場合は、ページめくり処理を行った後、再度ページ確認を行い、印字処理を継続する。紙幣ユニット(BRU : bill recycle unit)は、入金された紙幣の真偽鑑別/金種判別/計数/分類し、各金種別収納箱に収納するとともに、出金に際しては、各金種別収納箱より必要枚数の紙幣を繰出し、計数処理を行い放出する。BRU の代表的な仕様を表8・4に示す。仕様の中で、紙幣ニアエンド検知とは、収納箱の中の紙幣の残りが少なくなったことを検知する機能であり、紙幣フル検知とは、紙幣収納箱の収納限界が近くなったことを検知する機能である。BRU の構成は、繰出し部、真偽鑑別部、表裏整列部、紙幣収納部より構成されている。BRU の真偽鑑別部は、入金された紙幣の真偽を見分けるために、パターン認識技術等の非常に高度な技術が使用されている。また、紙幣の真偽鑑別に加え、金種、表裏、正損の判定を高速に処理するとともに、処理の信頼性を向上させるために鑑別部の二重化なども実施されている。近年、他の金融機関との差別化を図るため、消毒機能をもたせたものも登場している。硬貨ユニット(CRU : coin recycle unit)は、入金された硬貨の真偽鑑別/金種判別/計数/分類し、各金種別収納箱に収納する。また、出金に際しては、各金種別収納箱より必要枚数を繰出し、計数処理を行い放出する。CRU の代表的な仕様を表8・5に示す。CRUは、硬貨投入部、繰出し部、鑑別部、選別部、硬貨収納部より構成されている。CRU は硬貨を扱うため、処理速度の向上および鑑別精度・速度の向上を実現するために、硬貨搬送系、硬貨鑑別機構に高度な技術が不可欠となっている。ユーザ操作ユニット(UOP: user operation panel)は、利用者表8・4 BRU output executive)ソフトウェアおよびモニタが格納される。モニタは一種のミニOS であり、基本機能としてタスク管理および割込み処理、タイマ管理機能をもつ。従来、ATM 製造メーカ固有のオペレーティングシステム(OS)が主流であったが、近年、CD/ATM の高機能化とともに、インタネットとの融合が進むにつれ、はん用OS を採用する流れが加速しつつある。IOX は、各種I/Oユニットを制御する入出力制御ソフトウェアである。ベーシックソフトウェアは、IOX ソフトウェアとアプリケーションソフトウェアの間をつなぐはん用ソフトウェアであり、業務処理を実行するアプリケーションソフトウェアの設計仕様の変更の影響を受けない共通部であり、アプリケーションソフトウェアの開発生産性を向上させる効果がある。アプリケーションソフトウェアは、自動化機器による取引処理を規定するソフトウェアであり、金融機関が作成する利用者とのマンマシンインタフェースや、ホストコンピュータとのインタフェース仕様によりその内容が規定される。ソフトウェアの構成例を図8・9に示す。最新型のATM の多くは、上記各ユニットが独立した処理を行いながら、全体的に制御されるようになっている。ATM を構成する各部分を、相対的に独立したユニットとしたことによって、1ユニットごとに高い性能を追求しやすい、2それぞれが同時に動作することにより処理全体のスピードアップが図られる、3保守管理、障害対応がしやすい、4通帳ユニット、硬貨ユニット等をオプション化することにより、個々の金融機関のニーズに合わせた機能をもったATM を容易に構成することができる等のメリットがある。例えば、ICカードへの対応についても、カードユニットの交換により、ハード的にはすでに対応可能となっている機種が多くなっている。また、最新のATM の大きな動向として、従来、各メーカのATM は、それぞれの機種に特有のOS をATM の制御部に採用していたが、これがWindowsNT やOS/2等のはん用OS をベースとしたものに移行しつつある。これにより、金融機関が情報システムとして使っているパソコンやインタネットとも相互に接続が可能となり、商品情報や金利シミュレーション等の情報をATM で容易に提供しやすくなった。また、ATM における新しい機能の開発が、パソコンでさまざまなアプリケーションを作成することと同じ手軽さでできるようになっている。さらに、近年のパソコン用のはん用OS は、一般に画像処理や音声処理に優れており、これらの機能を使うことにより、ATM のガイダンス画面を非常に美しく、自然なものにすることができるようになった。
