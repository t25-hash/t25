# 8・3・1 端末制御装置

> **章**: 第8章 社会情報システム
> **節**: 8・3 金融機関店舗機器システム

営業店内の各種端末装置を制御する装置で、端末装置の多い都市銀行等の大規模店では営業店に設置され、一方、端末装置がそれほど多くない地方銀行や中・小規模店では端末制御装置を事務センタ等に設置し、通信回線を介して営業店内の各種端末装置を制御するケースが多い。端末装置とホストコンピュータとのデータ送受信はこの端末制御装置を経由して行われる。これまで、多数の各種端末をリアルタイムで制御する必要性から、多くの場合、高性能な専用コンピュータが用いられてきたが、最近では、はん用コンピュータの性能向上に伴い、高性能な演算処理装置と大容量のメモリをもつはん用コンピュータを用いるケースが増えつつある。端末制御装置は、各種端末装置の制御や通信処理を行うことに加え、ホストコンピュータや端末装置で業務処理の一部を処理する例も増えてきている。端末制御装置のおもな機能を以下に述べる。(1) 通信制御機能ホストコンピュータと各種端末装置の中間に位置し、通信処理を制御する機能であり、以下のようなメリットが得られる。・多数の端末装置とホストコンピュータをつなぐ回線を端末制御装置で集約することができるため、通信回線を効率良く使用することができる。・多種多様な端末装置の制御を端末制御装置で統一(仮想化)することにより、ホストコンピュータがそれぞれの端末装置の仕様の違いを意識せずに済むため、ホストコンピュータの負荷を軽減することができる。・端末装置相互の連係処理や、営業店内のローカル処理をホストコンピュータに負担をかけることなく処理することができる。(2) オフライン処理機能ホストコンピュータや回線が障害となった際に、障害中に端末装置から入力されたデータを端末制御装置内に蓄積しておき、障害回復後に自動的にホストコンピュータに送信する機能である。(3) 営業店内の運用管理機能資金の残高管理や、端末制御装置に接続される端末装置の稼働状況を管理する機能である。(4) ホストコンピュータと連係した運用管理機能端末制御装置で管理している端末装置の稼働状況や障害情報をホストコンピュータに送信することにより、ホストコンピュータでは、金融機関のすべての端末装置の状況を管理することができる。また、端末装置のソフトウェアに修正が必要になった場合、当該端末装置のソフトウェアをホストコンピュータから端末制御装置に転送し、さらに、端末制御装置から当該端末にソフトウェアをダウン8―141ロード・インストールする機能を実現しているシステムもある。多数の端末装置のソフト修正を、人手を介さずに実現できる本機能は、システムのメンテナンスをするうえで効果の大きい機能といえる。端末制御装置を含む端末システムの構成例を、図8・10に示す。
