# 8・3・4 イメージOCR システム(image optical character

> **章**: 第8章 社会情報システム
> **節**: 8・3 金融機関店舗機器システム

reader system)金融機関の営業店における事務処理の効率化は、オンラインシステム化、窓口端末装置や現金処理装置の高機能化により大幅に向上した。しかし、顧客が記入した伝票の入力については、長い間、テラーが窓口端末装置のキーボードから手で入力してきた。キーボードからのデータ入力にはある程度の熟練が必要であり、そのために金融機関が負担しているテラー教育のための時間と費用が無視できなくなってきている。事務処理の効率化とローコスト化のために、未熟練テラーやパートタイマーにも容易に操作できるデータ入力装置への要求が高くなってきた。このような背景から、窓口端末装置にイメージOCR 装置を接続し、顧客の記入した伝票をイメージOCR 装置で直接読み取って処理するシステムが出現した。8―143(1) システム構成イメージOCR システムは、営業店の後方に設置され、大量の帳票を一括して処理する連続読取り型と、窓口の一線、二線に設置され、取引一件ずつ処理を完結する一枚読取り型に大別することができる。連続読取り型の装置は、はん用のOCR システムそのものであるため、ここでは窓口の一線、二線に設置される一枚読取り型について説明する。(2) 運用イメージOCR システムの運用を次に示す。1顧客は目的の伝票に必要事項を記入し、現金等を添えてテラーに処理を依頼する。2テラーが受取った伝票をイメージOCR に挿入すると、読取りを開始する。まず、伝票の定められた位置に印字されている帳票ID を読み取り、そのID に対応した帳票定義体に従って読取りを実行し、読取り結果を窓口端末装置に通知する。帳票定義体とは、伝票の文字が記入されている位置や文字の種類(手書き、OCR-B フォントなど)を定義した情報の集合であり、通常、窓口端末装置のハードディスク上に格納されている。OCRシステムは、この定義情報に従って文字認識処理を実行する。3イメージOCR システムから通知された読取り結果は、窓口端末装置に表示される。表示のスタイルは、窓口端末装置のアプリケーションソフトにより規定されるが、一般的には、後で行う修正処理がしやすいように、読取り結果と、記入された文字のイメージが並んで表示される。4テラーは、画面に表示されている読取り結果と記入された文字のイメージを見ながら、読取り結果が正しく認識されているか、誤認識やリジェクト(認識不能)文字がないかを確認し、修正入力を行う。修正入力の効率化のために、リジェクト文字は、画面上で点滅したり、ほかとは違う色で表示したり、テラーが気がつきやすい表示方法をとっている。5画面上での修正入力終了後、完了キーを押下すると、取引データがホストコンピュータに送信される。6ホストコンピュータでの処理が完了すると、窓口端末装置に取引結果が送信され、イメージOCR システム内に取込まれた伝票に取引完了の証拠となる印字が実行され、伝票は排出される。イメージOCR システムでは、顧客が直接伝票に記入することから、記入字体の制限や教育は困難であり、誤認識やリジェクト文字の発生は避けることができない。窓口における事務の効率化を図るためにはイメージOCR システムの認識率の向上が鍵であり、知識辞書の応用や、システム的なチェックによる実質的な認識率の向上策も盛んに研究・応用されている。
