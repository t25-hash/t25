# 8・3・5 印鑑照会、照合システム

> **章**: 第8章 社会情報システム
> **節**: 8・3 金融機関店舗機器システム

金融機関では、顧客数の増大に伴い、取扱う印鑑票の数が増加している。また、取扱い業務の多様化による営業店面積の拡大、営業店の階層化(母店、出張所、インストアブランチなど)による営業店構成の複雑化により、印鑑票管理の効率化が課題となっている。顧客の口座番号等をキーとして登録されている印影を検索し出力する印鑑照会、および、顧客の押印した印影と登録されている印影の一致度を判定する印鑑照合システムは、印鑑処理業務の効率化、営業店内スペースの有効利用、顧客サービスの向上を目的としている。以下に、その概要を述べる。(1) 効果従来、印鑑票は紙ベースで管理されており、印鑑照会、照合は、一線もしくは後方のテラーによる手作業で実施していた。これに対して、印鑑票の印影や顧客の署名(住所、氏名)を電子ファイル化してシステムを構築することにより、次の効果が期待できる。・印鑑照会、照合時間の大幅なスピードアップによる顧客待ち時間の短縮・テラーが席を離れる必要がなくなることによる事務効率の向上8―144情報・メディア機器・印鑑票を管理する母店以外の店舗や事務集中センタからネットワークを介して印鑑照会、照合が可能になり、事務効率の向上と顧客へのサービス性向上・印鑑照会、照合機能を勘定系システムと連動することによる一線完結率の向上、事務効率の向上、顧客サービス性の向上(2) 概要1システム構成メーカから提供されている印鑑照会、照合システムの概観の例を以下に示す(図8・13)。図8・14に、印鑑照会、照合システムの基本的な構成を示す。端末制御装置を介して印影ファイルコントローラとホストコンピュータを接続することにより、印鑑照会、照合機能を営業店内での利用のほか、他の店舗、事務集中センタからの利用も可能となる。また、営業店内においては、端末制御装置に接続されている窓口端末装置と連動し、窓口端末装置から印鑑照会、照合指示を出したり、照会、照合結果を印影表示装置もしくは窓口端末装置のディスプレイに表示することができる。2印影ファイルコントローラ印鑑照会、照合システムの専用端末である印鑑登録機、印影プリンタ等の接続制御を行うとともに、印影イメージデータを印影ファイルに格納する機能をもつ。システムの信頼性を保つために、ファイルシステムは、ハードディスクを冗長構成することにより信頼性を高めるRAID(redundantarrays of inexpensive disks)を採用することが多い。3印鑑登録機印鑑票をイメージリーダで読み取り、ディスプレイ表示、印鑑部、署名部のイメージデータの切出し、イメージデータの圧縮を行い、キー入力された口座番号ととともに印影ファイルコントローラに登録する装置である。イメージリーダは、固定されたスキャナに対して印鑑票を搬送するスキャナタイプと、机上に置かれた印鑑票を、電気スタンドのような形をしたセンサが光学的に印鑑票をスキャンし、イメージを取込むスタンドスキャナタイプがある。スタンドスキャナタイプは、周囲の照明環境の影響を受けやすいという技術的に難しい面があるが、厚みのある資料やしわのある伝票からイメージを取込む高い能力をもつため、広く使われるようになってきた。各社から提供されているスタンドスキャナタイプのイメージリー図8・13 印鑑照会・照合システムダを備えた窓口端末装置の例を以下に示す(図8・11)。4印鑑照会、照合機キー入力された口座番号に対応する印影データを検索し、印影表示装置に表示する照会機能、伝票に押印された印影が印鑑票に登録されている印影と一致するかどうかを見る照合機能がある。印鑑照会、照合機は従来、専用の端末装置で実現されていたが、近年、窓口端末装置の能力が高くなり、窓口端末装置のアプリケーションソフトとして、印鑑照会、照合機能を実現する傾向にある。5印影プリンタ印鑑登録機、印鑑照会、照合機からの指示により印影イメージデータを実物大で印刷する装置である。
