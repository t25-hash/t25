# 8・5・2 航空交通システム

> **章**: 第8章 社会情報システム
> **節**: 8・5 交通情報システム

a. 航空交通システムの特徴航空交通システムは航空機,離発着をする空港や飛行場,飛行する空域や航空路,航行援助システム,さらに安全で効率的な運航を図る航空交通管制機関,運航者である航空会社,こうした施設,組織や関連システムを運用する要員などからなる.したがって,そのシステムは複雑で大規模である.また,航空機は重力に反して,人や貨物を輸送するため,高い信頼性,安全性,定時性や快適性が要求される.b. 航空交通システムの構成図8・30に航空交通システムの概要を示す.これは機能別には通信,航法,監視,そして航空交通管理(ATM : air traffic management)のサブシステムに分類できる.ATM は航空交通業務(ATS : air traffic services),航空交通流管理と空域管理からなる.ATS は航空機運航の安全性と効率性を確保するための業務で,これは航空交通管制業務,気象情報,航法援助施設や飛行場などの運用状況などを提供する飛行情報業務,そして航行中の緊急事態や遭難時の対応をする警急業務からなる.c. 航空交通管制航空機の飛行方式には,一定の気象条件下で,パイロットが他機との安全な間隔を設定して飛行する有視界飛行方式(VFR : visual flight rules)とつねに航空交通管制の指示に従って飛行する計器飛行方式(IFR : instrument flight rules)とがある.今日では,旅客機などほとんどの航空機はIFR で航空交通管制(以下,航空管制と呼ぶ)のもとで飛行している.航空管制では,航空管制官が航空機相互間に安全な最小間隔(管8―154情報・メディア機器図8・30 航空交通システムの略図制間隔という)を確保して衝突を防止する.監視システムにより航空機の動態を把握し,パイロットに通信システムを介して指示を与える.パイロットはこの指示に従って航空機を飛行させる.管制間隔は前後,左右,そして上下方向について個別に定められている.この値は管制方式,利用できる航法,監視や通信システムなどにより異なるが,航空機の位置情報が正確であるほど管制間隔を小さくできる.航空機は空港を出て管制圏と呼ばれる空港近傍の空域から進入管制区を経て管制区に入る.管制区ではおおむね水平飛行をし,目的空港の近くの進入管制区,管制圏に入り着陸する.管制は航空機の飛行段階でも分割されており,管制区内にあっても,セクタと呼ばれるいくつかの管轄空域に分割され,個別の管制官により管制される.管制官がセクタにより異なるため,セクタ間での管制移管の際は,当該セクタでは航空機に関するさまざまな情報がやり取りされる.運航者は飛行に先立ち,空港に飛行計画を提出し,これに基づいて出発空港から到着空港までの飛行計画が作成される.この際,航空機が飛行する予定のセクタの管制席に特定の地点の通過予定時刻,高度,便名,機種などの情報が表示される.こうした作業を計算機により自動的に行うのが管制情報処理システムである.航空管制では航空交通の円滑な流れを促進することも重要である.これには,各セクタでの局所的な航空の管制に加えて,出発から到着までの全経路を含めた大局的な交通流や空域の管理が必要となる.d. 管制情報処理システム管制情報処理システムの代表的なものには(1) 飛行計画情報処理システム(FDP: flight data processingsystem)(2) 航空路レーダ情報処理システム(RDP: radar data proces-sing system)(3) ターミナルレーダ情報処理システム(ARTS :automatedradar terminal system)(4) 洋上管制データ表示システム(ODP: oceanic air trafficcontrol data processing system)(5) 航空交通流管理システム(ATFM : air traffic flow man-agement system)(6) スポット総合調整サブシステム(SMAP: spot manage-ment and planning system)などがある.FDPは電子計算機により飛行計画情報を受信し集中的に処理し,運航票などの管制に必要な資料を管制官の手元に提供する.また,他の管制情報処理システムに表示に必要な情報を送る.RDPやARTS は航空路監視レーダや空港監視レーダの情報を処理し,各管制卓に設置された表示装置に航空機の位置や速度ベクトルに加えてFDPなどにより得た航空機の便名等の情報を表示するシステムである.ODPは洋上空域を飛行する航空機の位置を,飛行計画情報と航空機の位置通報から予測計算し,その空域の交通状況を表示するシステムである.なお,位置通報には,パイロットからの無線電話による通報と機上の測位情報を自動的に地上に送信する自動従属監視(ADS)システムによるものがある.ATFM は特定の航空路や空港に航空機が過度に集中することを事前に防止するため,RDPからのレーダ情報,FDPによる飛行計画情報,入力探査システム(IDP: input data processing system)からの航空情報などを用いて,適正な航空交通量の予測を行うものである.SMAPは計算機を利用して航空交通流管理に必要な空港のスポット運用状況等を全国のCADIN(e項参照)端末およびエプロン管理システムから集め,整理するとともにATFM から発出する出発制御時刻情報などの交通流制御情報を航空会社や空港等の関係機関に配信するシステムである.e. 航空交通情報システム上記の管制情報処理システムに情報を伝送する情報通信ネットワークが航空交通情報システム(CADIN : common aeronautical data interchange network)である.これは,国際航空交通情報処理中継システム(AFTAX :aeronautical fixed telecommunication automatic exchange andaeronautical data processing system),その国内版であるDTAX(domestic telecommunication automatic exchange and aeronau-tical data processing system),IDPや日本全国の空港等に設置された端末とそれらを網羅する情報通信ネットワークの総称である.国際航空固定通信網(AFTN)や管制情報処理システム,気象庁や防衛庁,航空会社等のシステムとも連接している.これは航空機の運航に必要な飛行計画,航空情報,気象情報,捜索救難に必要な情報などの情報の伝送,交換,通信処理,情報処理を行うシステムである.なお,AFTAX は,世界中の空港,管制機関,航空会社等を結ぶ国際航空固定通信網(AFTN)の日本における通信センタで,航空機の国際運航に必要な情報の処理・中継を行っている.DTAX は国内の空港端末の通信センタである.IDPはAFTAX やDTAX と連携して,国内および国外からの飛行計画などを受信し,データ形式の監査を行ってFDPにデータを提供するシステムである.f. 通信システム航空保安を目的とする通信,すなわち,航空通信業務には航空固定(地対地),航空移動(空対地,または空対空),航空放送(地対空),それに航空無線航法の業務がある.航空固定業務は特定の二地点間,航空移動業務は航空機局と地上局間または航空機局相互間で行われる業務である.航空放送業務は航空機に対して航行に関係する情報の伝達を目的とする放送業務である.固定地点間の通信はアナログの有線電話,無線電話とAFTN と呼ばれるテレタイプ通信が依然として使われている.航空移動通信は,通信内容では,パイロットと管制機関間の航空管制通信,航空機と運航者間の航空運航管理通信,航空乗客が利用する航空公衆通信とに大別される.航空管制通信はVHF帯(軍用機はUHF)を用いた音声通信が主である.洋上においてはHF帯の音声通信(間接通信)が主であるが,近年,VHFやINMARSAT を介した衛星データリンク(CPDLC)が使用され始めた.通信は音声からデータへ移行しつつあるが,運航管理通信ではACARS(ARINC communication addressing and reporting sys-tem)と呼ばれるVHFによるデータリンクシステムが使用されている.これはMSK 変調により2.4kbpsで文字を伝送する.現行のデータリンクは文字伝送方式(キャラクタ指向)であるが,将来は本格的なディジタル技術による(ビット指向)データリンクに移行するものと思われる.現在,国際民間航空機関(ICAO)では表8・7のようないくつかのモードのVHFデータリンク(VDL)システムを検討中である.なお,VDLモード4は監視のためのものである.公衆通信では地上系電話システムであるUHF帯を用いた機内公衆電話のサービスが提供されている.2001年からNTT ドコモがN-STAR 衛星を用いた衛星による機内公衆電話サービスを提供していたが,携帯電話の普及に伴い2004年に終了した.また,ボーイング社は2001年末から衛星を利用した機内インタネット接続サービス「CONNEXION」をアメリカで開始し,2004年には一部の航空会社の日本-ヨーロッパ間の路線でこのサービスが可能となった.航空放送業務には航空路/ターミナル情報提供業務(AEIS/ATIS)などがある.これは,航空路を飛行中の航空機に対して,気象状態や飛行場,航空保安施設の運用状態の変化などに関する情報をVHF無線電話で提供するものである.また,この情報はACARS を介してデータリンクにより機上で取得できるようになっている.このように,現行では目的ごとに異なる通信手段を用いているが,ICAOでは,最終的にはこれらの相互運用性を確保し,共通に利用する総合通信網を検討している.これは航空通信網(ATN :aeronautical telecommunication network)と称するもので,7層に階層化されたOSIプロトコルを採用することになっている.g. 航法システム航法とは移動体を目的地まで導く技術で,その中核は測位と誘導・制御である.航法システムは,一般には機上システムと地上の航法援助システムからなる.電波を利用する航法が電波航法,地上装置を必要としないものを自律航法と呼ぶ.最近の航空機では,航法センサ,対気,慣性センサ等の情報や自動操縦条件などに基づき操縦系,エンジン系を最適に制御する飛行管理システムが装備されている.航法援助システムとしては無指向性無線標識(NDB 単独では進入・着陸段階で使用するには測位精度や信頼性[特にインテグリティ(システムが航法への使用に不適な場合,利用者に対し適時に警報を出す能力)など]が十分ではなく,これを補強する必要がある.このため静止衛星や地上に補強システムを配置し,誤差を補正する情報やインテグリティ情報を利用者に送信することにより,性能要件を満たすための研究・開発がなされている.衛星を用いた補強システムをSBAS(satellite-based augmentation system),地上の補強システムをGBAS(ground-based augmentation system)と称している.このSBAS の例としてはアメリカの広域補強システム(WAAS : wide area augmentation system)が2004年から稼働している.その水平測位精度は2~3m(95%)である.h. はレーダに比べて地上局のコストが低く,レーダの代替監視手段として注目されている.レーダには航空機からの反射信号を用いる一次レーダと自動応答装置(transponder)からの応答信号を利用する二次レーダがある.二次監視レーダ(SSR : secondary surveillance radar)は質問モードと応答コードとの対応で通信機能を有している.このため,SSR では航空機の識別符号,気圧高度情報が入手できる.ちなみにASR やARSR は一次と二次レーダを併設しているが,ORSR はSSR 単独でモノパルス測角方式を採用して測角精度を向上させている.SSR は監視に加えて通信機能を兼ね備えているが,これを拡張したSSR モードS では,世界中の航空機に固有のアドレスが割当てられ,個別呼出しが可能である.しかも,データブロックに56ビットまたは112ビットのコードを用いており,多くの情報を伝達できる.これは,空対地のデータリンクや航空機衝突防止装置(ACAS : airborne collision avoidance system)に利用されている.ACAS はSSR の質問装置を搭載し周りの航空機の距離と高度差の情報から,衝突の可能性のある航空機を検知し位置情報の提供や衝突回避の指示を与えるシステムである.i. 航空機システム航空機はB 767以降の第四世代ジェット機では計器類の表示にCRT や液晶ディスプレイが使用され,A 320やB 777などでは操縦装置も機械系から電気式に置換えたFly-by-wireなどが採用されている.
