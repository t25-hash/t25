# 5・9・1 リスクマネジメント(risk management)

> **章**: 第5章 手術・治療機器
> **節**: 5・9 ガイドライン

極論すると、安全な外科手術はないといえる。だからこそ、手術の安全性をどのように確保するかがいま問われているのである。例えば、脳そのものは絹ごし豆腐のような柔らかさであり、非常にデリケートな臓器である。また、場所によっては1mmでも違うと異なる機能が存在するという微細な構造をもっている。脳の手術とは、非常にもろくデリケートな臓器を手術することなのである。このように手術対象臓器は、生命を維持するために多かれ少なかれ種々の有用な機能や役割をもっている。外科手術は、臓器の機能をできるだけ温存しながら、患者のQOLを高レベルで維持することが求められているのである。手術の安全を確保し、つねに検証し改善していくためには、新しい手術システムや治療器具の開発、手術方法の改良、精密な画像診断に基づく手術計画、合理的かつ系統的なトレーニングが必須である。a. 医療過誤(medical malpractice)の削減策医師らが事故事例を開示し原因を見極めて医療システムを改善することと、その積み重ねが重要であるとD. バーウィック(アメリカ、ハーバード大学)は述べている。すでにアメリカでは、医療の質を担保するために総合的質経営(TQM : total quality management)の考え方を医療へ応用し、審査から改善を重視する方向へと進んでいる(医療の質改善全米実証プロジェクト、1987~1991年)。医療過誤による死亡者は約9万人に上り、いまやアメリカ人の死因の8番めで、乳がん・自動車事故・エイズよりも多いという現状である。悪い医師(医療スタッフ)のために、医療過誤(事故)が起きるのではなく、システムの欠陥、例えば医療器具の設計ミスなどによる事故がおそらく98%を占めるといわれている。例として投薬システムについて考えてみると、処方してから患者が服用するまでに、50から60のステップが仮にあるとする。一つだけなら信頼性は95~99%であっても、信頼性95%のステップが25あれば、全体の信頼性は28%に下がることになる。さらにそのステップが50あれば、その信頼度はわずか8%なのである。解決策は、単純化することに尽きる。すなわち、コンピュータ制御によるステップ数の削減、薬品の種類の制限、処方数の削減等を実施し、また見た目が似た薬の廃止を行う。とにかく、仕事、機器の標準化・単純化に尽きる。またシステムの改善以外にも、人のモチベーションを向上させる仕組みも必要である。アメリカでも医師や看護婦が自分のミスを明らかにすることにちゅうちょしている。当然、わが国でもそうである。航空事故を防止するために、アメリカ航空宇宙局(NASA :National Aeronautics and Space Administration)では、事故につながる過ち、間違いをしたかもしれないという懸念を自主的に申告させることで事故防止に効果を上げている(最近わが国でもヒヤリハット事例の収集が積極的になされている)。制度として、インシデント・アクシデント報告として、全国の病院で確立されつつある。重要なのは報告を統計数字として扱うのではなく、誤りの内容を細かく記述し解析することである(背後要因分析)。また、個人が罰則を受けないように病院として謝罪、補償をするようにし、正直でありたいという個人のモチベーションを大切にすることが必須である。このように背後要因を分析公表することで、同様な事故は防げるのである。b. 医療トレーサビリティシステムすでに農産物の分野では、消費者の食の安全志向の高まりとともに、生産者が食の品質や安全性を保証するトレーサビリティ(traceability、原料・生産履歴の追跡管理)システムの導入が始まっている。しかし、それにとどまらず医療従事者が受益者である患者・家族に治療・看護過程の履歴を開示し、安全性と品質を保証する医療のトレーサビリティ(治療・看護過程履歴の追跡管理)システムの整備が求められるようになるのが必然である。安全性を追求するうえで医療情報の可視化とともに、イベント(行為)の履歴(ログ)を追跡管理するシステムが必須である。手術中に患部の状態を撮像できる術中オープンMRI手術室での手術中の画像の取得や、その画像に基づくリアルタイムアップデートナビゲーションは、その医療トレーサビリティの一つである。本システムは、術中の状況や経過をディジタルの履歴として残すだけではなく、脳の偏位(手術操作により脳が本来あるべき位置よりずれること)の問題の解決や、腫瘍の取残しのない手術などさまざまな優位性をもっている。東京女子医科大学に設置されたオープンMRI手術室では、2000年3月から2007年10月までに、550例の手術を行った。特に、言語野(言葉の中枢)・運動野(運動の中枢)などの機能領域に隣接する悪性脳腫瘍に対しては、術中MRIだけではなく、覚醒下手術(言語野などの機能領域を手術する際に、麻酔を覚醒させて言語機能の局在を確認する)、5-ALA(5-アミノレブリン酸)によるケミカルナビゲーション(脳腫瘍に取込まれて紫外光を当てると赤く光る状態)、軟X 線照射装置(フォトン放射線治療装置、PRS)による術中照射の併用など種々の方法を駆使して、摘出率の向上(平均摘出率93%)と重要な脳機能の温存に努めている。現在は、機能を温存し患者のQOLを高レベルで維持することが当然の手術結果と考えられる時代となったのである。c. リスクアセスメント(risk assessment)手術におけるリスクアセスメントは、医者と患者間でのインフォームドコンセントに、今後重要な役割を果たす概念である。ある手術例で考えると、手術は術式などにより危険(100%)から安全(0%)までのリスクが当然存在することは言うまでもない。医者と患者との話し合いで、私(患者)はこの程度の残存リスク(例えば40%)であれば、この術式で手術をお願いしたい(医者へ)ということになり、契約が成立するのである。これを受けて、医療サイドはこの残存リスクをさらに低下させる方策を考えることになるのである。残存リスクを可能な限り下げて手術を行うことが、医療サイドの努めである。すなわち、医療従事者は使用する医療器具・システムにおいても、残存リスクを低下させる方法をつねに追求する必要があるのである。d. 治療目標過程最適管理と戦略システムいつでもどこでも均質な医療を提供するのが、これからの医療システムである。これを実現するためには、一定のガイドラインに従ってチームで医療を行い、さらに予防・診断・治療を一貫させる仕組みが必須である。すなわち、健康を維持するための啓蒙活動を行うと同時に、病気になる前の生活習慣を含めた患者の医療情報を継続的に収集することが必要なのである。治療の実施に当たっては、患者情報をチームで共有し、チームで治療方針を策定する。治療実施中にも経過情報をチームで共有し、必要に応じ相互支援するわけである。このようなチーム医療を行うためには、医療機関の仕組みとして、治療過程を可視化して一元管理する情報センター(戦略デスク)が必須となる。目標最適管理(TCM : target controlled management)とは、目標(治療結果)を設定し、術前に精緻な手術シミュレーションを行い、治療プランニングを策定し、治療プロセスのロードマップを構築して、治療中には、治療に対する患者の反応の予測と実際の反応との違いを測定してフィードバックすることで治療手段を調節し最適化して設定したゴールに誘導する手法である。このためには、治療プロセスを可視化することが本質的に重要である。逆に、治療チームの一員として患者自身が参画するケースが増えるに従って、治療プロセスの可視化は否応なく進むはずである。効果部位コンパートメントは、実際の治療動態データベースに基づいている。日々蓄積される治療動態データベースに因子分析、実験計画法などを適用し、データ発掘(data mining)を行って、対象範囲の拡大、定量性の精密化を行うのみならず、因果関係を治療理論あるいは深い知識として構造化していき、最終的には治療効果全体を包括的にシミュレーションできる病態ディジタル生体モデルに発展させることが目標である。手術のプロセスを管理するために、ミクロなレベルでTCM を適用することができる。どこまで完全な手術を目指すか、どこで手術をやめるかという判断を手術中に冷静かつ客観的に下すことは、執刀医にとって非常に難しいことである。TCM を使って、手術の一つ一つの操作ステップごとに目標を明確化して達成度を評価し、次に行うステップや手術全体の目標をダイナミックに再設定することによって、判断の根拠と基準を可視化できるようにすることが必須である。これら治療プロセスを最適に管理するのが手術戦略デスクなのである。リスクマネジメントとは、リスクが発生するときに力を発揮することではなく、リスクをいかに予防し、インシデントのうちに悪循環の輪を断ち切りアクシデントを未然に防ぐかということなのである。危機が発生したときに、当事者に現在の状況をわかりやすく提示し、解決のオプショ9 ―105ンをわかりやすく示すことでパニックを予防するリアルタイムな決断をサポートするシステムを構築することが、リスクマネジメントのすべてなのである。文献(1) Slutsky,A.S.,Itensive Care Med.,20 (1994)、64.(2) Pierson,D.J.,Respir.
