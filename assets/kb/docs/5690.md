# 9・3・1 視覚障害者のパーソナルコンピュータ利用と操作

> **章**: 第9章 支援機器
> **節**: 9・3 視覚障害者用機器

a. 音声出力・音声入力パーソナルコンピュータ(PC)を使用できる重度の視覚障害者は少ない。それは、初心者であっても、俗にブラインドタッチと呼ばれる熟練者が行うようなタッチタイピングを強いられ、そのことだけでも相当な負担となるためである。しかも、一目でわかるディスプレイはなし、見て操作がわかりやすいマウスもなしで、操作順序を記憶して音声だけを頼りにPCを使うのは、まるで音声電話だけでPCを遠隔操作するようなものである。ようやく利用が広がってきた音声認識による入力は、視覚障害者にとってハンディキャップを補う意味で非常に大きい利点がある。しかしながら、主たる出力も音声を頼りにしての操作であるため、すべてのディスプレイが目で見ることを前提にして作られている以上、音声だけでは確認できない部分が多数あるのも事実である。b. ディスプレイ情報の音声出力1980年代に実現したMS-DOS(Microsoft disk operating system)PC操作の音声化によって、PCは徐々に視覚障害者の間に浸透していったが、1990年代に入ると視覚的な操作中心のGUI(graphical user interface)環境一辺倒となり、しばらくの間、視覚障害者は全く取り残された状態となった。ウィンドウズへの視覚障害者用音声対応については、日本障害者雇用促進協会が研究者を組織して続けていた地道な開発が実を結び、1996年に国内初の視覚障害者用ウィンドウズ音声化ソフトウェアが市販された。これらは音声スクリーンリーダ(screen-reader by voice)と呼ばれている。次いで、視覚障害者が音声で利用しやすいように特化したソフトウェアも開発され、視覚障害者の世界もようやくウィンドウズが主流となって、アプリケーションソフトウェアが利用できる基盤が整えられていった。このほか、視覚障害者の操作性に配慮し、キー操作のみで音声読上げによって比較的簡単にアクセスできるホームページ閲覧ソフトウェアも発売されている。c. 点字式キー入力初心者の視覚障害者にとっては、フルキーボードだと最初から熟練者と同じくブラインドタッチを行わなければならない。そのため、点字タイプライタなどの点字器具をよく使用している視覚障害者を対象に、点字の文字等が六つの点の組合せであることに対応させ、フルキーボード中のF,D,S とJ,K,L等の6個に10個程度プラスしたキーだけで文字入力ができるソフトウェアが開発されており、これだと指の位置が変わらないなお、六つのキーはその組合せを同時に押して、押している指がすべて離れたときにキーの組合せの信号を出す方式である。したがって、キーの同時押しが認識される必要があるが、キーボード部分の制約で認識できない機種も少なくないので、ウェブ上で点字キー入力が可能なリストを公開しているホームページもある。d. 自動点訳ソフト墨字(inkprint)文字データを点字データに自動変換する技術は、英語においては1980年ごろには実用化されたが、日本語においては漢字の読取り精度と漢字読みの選択、そして言葉の切れ目の判断が必要なため、技術的に大きな困難があった(墨字とは、点字に対して目で見る文字をいう)。急速な技術の進展により、近年ようやく日本語においても実用的なレベルに達してきており、非公式点字資料としては変換しただけのものが使用されることもある。ただし、英語の自動翻訳等と同じで、正確な点字書類とするには変換後の校正が欠かせない。そのため、きちんとした資料、とりわけ教材等や点字図書館等の蔵書においては、変換後に入念な校正を行い、点字エディタによって修正する必要がある。なお、墨字の図形情報の触覚情報への変換については9・3・2・d項で詳しく述べているが、文字データ変換とは比較にならないほど大きな触覚特性上の問題点があり、自動変換は将来の課題となっている。e. 点字エディタ点字では、文字の大きさが一定であること、レイアウトが大きく制約されることなどから、多くの場合、レイアウトについては墨字文書とはかなり異なる。そのため、点字形態での編集・校正を欠かすことができない。そのような点字でのデータ編集を行えるソフトウェアが市販品、シェアウェア、フリーウェアを含めていくつか存在しており、特に点訳を行うときによく利用されている。なお、触図を点字プリンタ出力するための点図(点字図形)編集ソフトウェアもフリーウェアとして開発されており、広く使用されている。
