# 15・1・1 セキュリティ

> **章**: 第15章 医療福祉の情報インフラストラクチャ
> **節**: 15・1 医療・福祉情報システムとしてのインフラスト

医療情報ネットワークでは，通常のネットワークシステム以上に
セキュリティに留意しなければならない．患者情報は個人のプライ
バシーに関するものであるため，この情報がインタネット上を流通
する場合には幾重にもセキュリティロックをかける必要があるだろ
う．医療情報ネットワークでのセキュリティには，医療情報そのも
のにセキュリティをかける方法，ネットワークサービスとしてセキ
ュリティをかける方法，アクセスする人を制限する方法の3種類が
ある．
現在，医療画像の標準として，画像フォーマットと通信プロトコ
ルを規格化したDICOM（digital imaging and communications in
medicine，ダイコム）があり，その中にセキュリティの規定があ
るが，その構成は基本的にIP（internet protocol ）レベルでのセ
キュリティとDICOMヘッダ部の暗号化であり，画像部自身への
暗号化は規定していない ．IPレベルのセキュリティは後述する
が，基本的にアプリケーションが関与しないセキュリティとなる．
DICOMヘッダへの暗号化はDICOM規定の補遺にて示され，ヘッ
ダ情報を暗号化し，個人を特定できないようにしている．暗号化さ
れたヘッダ情報は鍵情報を得ることによって解凍される．DICOM
画像としては規定に準じているため，暗号化により個人を特定する
例えば，かかりつけの医師は患者のすべての情報にアクセス可能と
ことはできないが，DICOMプロトコルによる転送，画像表示は可
能である．また，DICOMプロトコルとは別に，画像自身を暗号化
する方法もある．この場合，画像情報は容量が大きいので，個人情
報を含むヘッダ部と画像部に分けて，前者には暗号化強度は強いが
処理時間がかかる暗号化手法を，後者には暗号化強度は前者よりも
劣るが処理速度が速い暗号化手法をそれぞれ用い，安全性を保ちな
がら処理速度を向上させる手法が提案されている．
ネットワークサービスとしてセキュリティを保つ方法の一つに
VPN（virtual private network）と呼ばれる技術 がある．イン
タネットは基本的にオープンなネットワークであり，誰でも情報の
発信，取得が可能である．IETF（internet engineering task
force ）では，IPレベルのセキュリティのため，RFC（request for
comment ）でIPSec（IP security）を勧告している ．VPNは
IPSecを利用し，インタネット上でも構内ネットワーク（LAN:
local area network）と同様なセキュリティを保ちながら，特定の
グループ内のみでの通信を可能にする技術である．これにより，ユ
ーザは暗号化を意識せずに通常の使用によってある程度セキュリテ
ィの保たれた通信を行うことができる．
また，外部にある医療情報にネットワークを介してアクセスする
場合，アクセス者がその情報を取得できる者であるかどうかを確認
して制限するため，アクセス者を個人認証することも必要である．
端末側での個人認証の一手法として指紋認証がある．端末用PC
（パーソナルコンピュータ）に簡易に接続できる指紋認証システム
もすでに販売されている．また，ネットワーク上に個人認証を行う
認証局を設置する方法も検討されている．本手法は，医療情報ネッ
トワークのみではなく広くネットワークビジネスの世界で使用され
ていくものと思われるが，その最初のスタートはやはり，セキュリ
ティが重要な医療情報であろう．個人認証局では情報にアクセスす
る個人のレベルも規定でき，同じ医師であってもハンドリングでき
る情報レベルに差をつけ，プライバシーを保護することができる．
し，それ以外の専門医には患者情報を制限するなどの方法が可能で
ある．
介護支援への利用が可能となる．
