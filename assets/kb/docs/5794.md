# 15・3・3 電子カルテの目的と概念

> **章**: 第15章 医療福祉の情報インフラストラクチャ
> **節**: 15・3 電子カルテ

基本的に電子カルテは診療上の事実をすべて記録するシステムである。しかも単なる記録の羅列ではなく、記録と記録の関連付けを行うのが望ましいと考えられる。通常、診療は以下のプロセスで行われる。(1) 患者の訴えを聞く(主観的事実)。(2) 診察し、訴えを医学的に解釈する(客観的事実)。(3) 問題点を整理する。(4) 事実を評価(診断)する(アセスメント)。(5) 問題点に基づいて検査を行う(検査計画と実行)。(6) 治療を行う(治療計画と実行)。上記のプロセスがループ状に繰返されるのだが、このような問題解決のアプローチをPOS(problem oriented system、問題点に基づいた解決法の意)と呼び、問題点ごとに上記の(4)~(6)が行われる。紙のカルテではこれらの診療経過(特に問題点どうしの関連など)を効率的に表現することが非常に困難だが、コンピュータを使うと容易になる。そのほか、静止画、音声、動画など、いわゆるマルチメディア情報を収録することなど、紙のカルテにできないことが可能になる。また、通信を使って、医師どうしや看護スタッフとの意見の交換、施設を超えた意見交換、データベースを利用した文献データの検索などが電子カルテの機能の一部として利用できる。電子カルテとは医療情報データベースそのものであり、適切な情報提示のためのフィルタ機能をもった作業環境と考えることができる。電子化して初めて可能となるのが、情報の共有とフィルタ機能である(図15・3)。電子カルテシステムは、患者から見るとわかりやすい説明、インフォームドコンセント(informed consent)となり、医師等には医学的記録を中心とした専門的な内容として見えるとともにパワフルな検索・通信機能をもつ診療支援環境になる。
