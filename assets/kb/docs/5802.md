# 15・4・4 医用画像利用のためのインフラストラクチャ

> **章**: 第15章 医療福祉の情報インフラストラクチャ
> **節**: 15・4 医用画像

a. 概要診療用医用画像は医師によって診療目的のみに用いられるものであるが、同時に個人情報の一部に相当し、セキュリティや個人情報保護の対象にもなるものである。一方、今後は各人が自分自身の健康データとして画像をもちうるし、一部の理工学系の研究者やイメージングシステム開発に関係する技術者の利用への道も望まれる。これらの多様な用途に対応するために、ハードウェア、ソフトウェア両面にわたるさまざまのインフラストラクチャともいうべきものが考えられており、一部はすでに存在している。例えば、画像伝送ネットワーク、画像によるスクリーニングシステムはシステム面で、ディジタル画像の規格や標準化、テストパターン等の画質や互換性を保証する基盤的仕組みはソフトウェア面でのインフラストラクチャに相当するものとみなされよう。一方、ディジタル画像の流通(収集、配布、保管など)は技術的にはめどが立ちやすいが、実際はフィルムの形での画像の処理も完全には除外できない点に技術的課題が残る。b. 病院内ネットワーク(inside hospital network)大規模病院では画像関係、カルテなどの診断情報関係、予約や課金など運営・管理関係の情報を一括管理するネットワークが稼働している。ソフトウェアとしての病院情報システム(HIS : hospital informa-tion system)であるが、その中で画像関連では、各患者ごとの撮影から医師への配布までの管理、各医師の診療時の要望に対応する画像の保管からデータベース化に至るさまざまなレベルのシステムがある。また、各分野ごとに構築されるシステム[例えば、放射線画像システム(RIS : radiological image information system)など]の利用もある。画像に関しては、それらに共通するシステムとして画像情報の管理(収集、保管、伝送、処理)を一元化しようとする画像収集・伝送システム(PACS : picture archiving andcommunication system)の構想が1980年代から提唱されているが、最初は大規模病院内において種々のレベルでの実現が図られた。しかし、診断レベルでの利用では各医師個人の要求に対応できなくてはならないうえ、治療においてはそれを実施する現場(例えば手術室内)での対応が必要であった。今日では初期のこれらの問題点は着実に解決され、少なくとも病院の新設、改築に際してはPACS を組込んだ情報システムを含めてシステム設計がなされることが普通になっている(図15・6)。c. DICOMDICOM(ダイコム)は医用ディジタル画像と通信に関する国際的な標準規格であり、1992年ごろからDICOM 標準規格が検討され始め、順次高レベルのものができている(最新版は2000年)。国内では日本画像医療システム工業会(JIRA :Japan Industries Association of Radiological Systems)のDICOM 委員会が国内への紹介や国内版の開発などを進めている。そこではおもに画像記憶の形式と画像を記憶した媒体の交換手続きを定めている。d. 病院外ネットワーク(outside hospital network)より広域には、地域における病院間ネットワーク、救急医療システム、病理医等の特定分野の専門医あるいは病理診断画像伝送ネットワー9 ―231図15・6 病院内ネットワークの一例ク、離島などの地理的条件への対応等、多様なものがある。全国規模の広域ネットワークにおける画像情報流通には、医師が送信側に簡単には行けない状況下での画像伝送が要求され、またそれができて初めて意義が認められる。そこで、遠隔地からのディスプレイ上の画像で診断する遠隔診断、さらにこれにバーチャルリアリティ技術で生成した三次元映像を組合わせたテレイグジステンスの応用、および遠隔治療の検討とさまざまの実験が行われている。一方、災害時の支援では不完全な環境下での最小限の情報伝送、あるいは可能な範囲での高速通信を保証する仕組みも必要となる。e. スクリーニング日本ではがんのスクリーニングや職場での健康診断にも画像が活用されている。そこでは大量の画像の診断を一定の水準とコストで実行するシステムが必要とされる。現在ではほとんどが医師の診断のみに依存しているが、近い将来には前期CAD の活用によって医師の負担を増すことなく診断精度を高めることが期待されている。の連携
