# 15・8・5 今後の課題

> **章**: 第15章 医療福祉の情報インフラストラクチャ
> **節**: 15・8 健康モニタリングシステム

健康モニタリングシステムは、毎日変化するバイタルデータや食事、運動状況などを自動的に、もしくはユーザが定期的に入力することによりその状態を管理することができるシステムである。ここでは、これらのシステムにおける課題について述べる。a. バイタルセンサと情報転送ネットワークバイタルセンサを使用してデータを入力するシステムの場合、使用するバイタルセンサの大きさや電池寿命、体に直接取付けた場合の不快感など、バイタルセンサ自身に起因する課題が存在する。健康モニタリングに使用されるバイタルセンサ類は、バイタルセンシングするセンサ機能部と、センシングしたデータをネットワークを介してデータセンター等に転送するネットワーク機能部に分類できる。センサ部の詳細は4・2節にて述べられており、ここでは高齢者などが違和感なく使用できるための課題を列挙する。(1) 小型化(2) 電池寿命(3) ユーザが意識しない装着の仕方(4) プライバシーへの配慮次に、ネットワーク部については、これらのセンサにより取得されたデータをどのように転送するかによってその構成が変わる。ネットワークインフラストラクチャとしては以下のものが考えられる。(1) 携帯無線ネットワーク(2) ホームネットワーク(家庭用無線LAN に直接送信、もしくはバイタルセンサから微弱無線により中継器を介して家庭用有線LAN に送信)(1)は家の中だけではなく、外出中にもモニタリング可能である。しかしながら、データ発生時に携帯電話網に発呼することになり、データ転送のたびに課金される。(2)は家庭内の使用に限定されるが、装置の小型化が可能となる。また、インタネットなどに常時接続していることを前提とした場合、モニタリングデータの転送のたびに課金されることはない。逆にいえば、常時接続しているため、いつでも高齢者などの状態を把握することが可能であり、上述したプライバシーの面を十分に考慮したうえで、効果的なモニタリングが可能となる。b. バイタルデータ以外のモニタリング系バイタルデータは各種センサにより自動的に取得可能であるが、高齢者の状態情報や健常者の食事内容の情報などを取得するには、画像による認識や各種状態情報取得センサとの融合が必須である。例えば、高齢者の状態を把握するために、電気ポットの使用状況をモニタリングしてその回数をセンターに送出し、遠隔地に住む家族などが携帯電話などでセンターに確認することで、その使用状況から高齢者の状態を判断するシステム(みまもりほっとライン)があるが、毎日の電気ポットの使用を前提としているため、暑い夏場で使用しなかった場合はどうなるのかなどの問題があり、一つの情報のみを状況判断の材料とすると誤認識の要因となる。別の状態把握手段として、カメラからの画像情報により高齢者の行動パターンや食事の内容、量などを判定し、状態情報や食事内容データとする方法も考えられるが、これも画像データのみで判定すると誤差を生じる。このように、環境から所望の状況を取得するためのデータ取得系に関しては、複数のセンサを使用し、それらのデータから総合的に判断するアルゴリズムが必要である。
