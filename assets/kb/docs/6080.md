# 12・2・12 環境試験データ管理システム

> **章**: 第12章 シミュレーション技術と試験法
> **節**: 12・3 に概要を示す.

筑波宇宙センターでは,衛星開発におけるコスト削減,開発期間短縮,信頼性向上を目的として1998年から環境試験データ管理システムとして順次試験データのデータベース化を行っている.a. 機能・システム構成環境試験データ管理システムのおもな機能は以下のとおりである.(1) 各環境試験設備で取得された試験データの一括管理・分析・検索・閲覧(2) 衛星メーカとの情報共有図12・29にシステム構成を示す.環境試験データ管理システムはメインサーバ(DB サーバ),セキュリティサーバ,通信サーバ等より構成される.メインサーバは試験データ等(設備データ,供試体データ,試験コンフィギュレーション等)の管理を,セキュリティサーバは筑波宇宙センター内外との通信のセキュリティ管理,通信サーバは筑波宇宙センター内外との通信の全般管理を行う.これらのサーバは,試験設備のデータ処理装置とLAN を介して接続されており,また衛星メーカ端末とはIPバーチャル・プライベー図12・29環境試験データ管理システム構成図表12・11 環境試験データ管理システム概要IBM RS/6000 Model H70 2CPUMemory 2GBHDD約1TB(RAID 5構成)オートチェンジャ付きテープライブラリ無停電電源装置メインサーバ諸元基盤ソフトウェアOracle8i Enterprise EditionIBM WebSphere Application ServerIBM VisualAge for Java Enterprise端末要求Windows98/NT/2000Netscape4.6ト・ネットワークを利用してインタネットを介して接続する.b. 利用形態環境試験データシステムの利用形態を以下に示す.(1) 試験データの再利用:同一供試体のモデル間,または類似供試体間の試験データの比較等の統計分析を行う.(2) 試験関連情報の閲覧:衛星の予備設計,基本設計,詳細設計の各段階において,設計者が過去の環境試験条件,試験結果,問題点等を体系的に確認する.(3) 解析ソフトウェアとの併用:環境試験データ管理システムの試験データを用いて,解析ソフトウェアを用いた試験シミュレーション結果の検証を行う.(4) 試験関連情報の伝送:衛星メーカの担当者が本社・事業所から試験関連情報を閲覧・取得する.c. 概略仕様環境試験データ管理システムの概要を表12・11に示す.文献(1) Brancewell,R.N.and Garriott,O.K.,Nature,182(1958),760.(2) Kane,T.R.and Barba,P.M.,AIAA J.,4-8(1966),1391.(3) 狼嘉彰・ほか4名,航空宇宙技術研究所特別資料,SP-4(1985).(4) 狼嘉彰・ほか2名,計測と制御,26-10(1987),845.(5) 高橋孝・ほか2名,人工衛星の力学と制御,(2004),1-112,宇宙航空研究開発機構.(6) 木田隆,スペースクラフトの制御,(1999),123-144,コロナ社.(7) Wijker,J.,Mechanical Vibrations in Spacecraft Design,(2004),11―153第II部基礎編343-398,Springer.(8) 狼嘉彰・藤井裕矩,日本航空宇宙学会誌,32-364(1984),15.(9) 小松敬治,計測と制御,28-4(1989),358.(10) Kane, T.R., ほか2名, Spacecraft Dynamics, (1983), 318-332,McGraw-Hill.(11) Wie,B.,Space Vehicle Dynamics and Control,(1998),468-483,AIAA.(12) Likins, P.W., NASA-CR-108209, Dynamics and Control ofFlexible Space Vehicles,(1970).(13) Singh,R.P.,ほか2名,J.
