# 17・2・1 リスク概念

> **章**: 第17章 信頼性とリスクマネジメント
> **節**: 17・2 リスクマネジメント

a. リスクの定義と分類リスクは、企業の経営活動あるいは技術開発において、ある意思決定あるいは選択を行った結果、ある因子が生じ、それが発展して重大な結果(事象)に至ることが考えられる場合、その事象の発生確率に事象のもたらす価値(損失)を乗じたもの、すなわち期待値として定義される。結果には、よい結果と悪い結果とがあり、企業経営の場合などは、正負両方の期待値が存在しうるが、技術活動の場合には、悪い結果のみが対象となる。すなわち、リスクには、大別して二つの種類がある。小林はこれを広義のリスク、狭義のリスクと呼んでいる。広義のリスクは、企業経営の場合のように、成功追求型のリスクであり、プラス、マイナス両方のリスクが存在する。一方、狭義のリスクは、顕在化すれば、企業活動の継続性を脅かすであろう災害、事故、不祥事、設計・製造ミスに伴う製品の欠陥、納期遅延、コストオーバラン、環境汚染、機密漏洩、違法行為などに関するリスクであり、これらはさらに、企業外部要因(自然災害、金利、原材料価格、景気、為替レート変動など)に起因するリスクと企業内部要因(設計・製造ミス、不祥事など)に起因するリスクとに分けられる。狭義のリスクの中で、特に企業内部要因に基づくリスクは、基本的に期待値はマイナスであり、上記の定義は次のように言い換えることができる。リスク=意思決定あるいは選択の失敗の発生確率×失敗した結果のもたらす損失メレディス(Meredith)とロビンソン(Robinson)は、リスクを次のように分類している。(1) 政治的リスク(2) 市場リスク(3) 技術リスク(4) 製造物責任リスク小林の分類と対照させると、(1)(2)は広義のリスク、(3)(4)が狭義のリスクとなろう。セイジ(Sage)は、狭義のリスクに相当するものをさらに次のように分けている。(a) 性能上のリスク(b) 日程上のリスク(c) コスト上のリスク(d) 運用支援リスク(e) 計画支援リスクb. リスクの判断者への依存性リスクは決定に伴って発生する。技術者、特に設計者が行う決定は、セイジによれば、次のような種類がある。(1) 起こりうる事態が一つしかないことが確実にわかっている状態下での決定(2) 起こりうる事態はいくつかあり、それぞれの内容と発生確率11―206宇宙機器・システムが把握されている状態下での決定(3) 起こりうる事態はいくつかあり、それぞれの内容と発生確率が正確に把握されていない状態下での決定(4) 起こりうる事態はいくつかあり、その内容と発生確率とが部分的に把握されている状態下での決定(5) 複数の決定者がおり、それぞれの間の意思疎通を欠いている状態下での決定(1)の場合を除いては、決定は、個人あるいは集団の判断に依存する割合が大きい。すなわち、リスクは個人の資質と判断力あるいはその集団の文化に依存するのである。ここでいう文化(cul-ture)とは、「ある集団の構成員が、共通に直面する問題に対して示す解の集合であって、これが規則となり、風土を形成して、その集団の価値観となる」とのヴォーン(Vaughn)の定義による。技術者、特に上級技術者が決定を下す場合、起こりうる事態の内容と発生確率とが明確になっている場合は極めて少なく、個人の資質と判断力に依存する要素が大きいが、資質と判断力に基づいてなされる判断は、その個人の所属する集団の文化に大きく影響されているものとみなしてよいであろう。すなわち、技術者の判断は、もっぱら所属する集団の文化すなわち価値基準に準拠してなされると考えてよいであろう。しかし、結果の評価は、多くの場合、第三者(各種の委員会や多くは世間)によってなされる。問題はここに存在するのであって、判断する側の評価基準と、評価する側の価値基準とが一致していればよいが、一致していない場合には、それは極めて大きなリスク要因となるのである。このことに気づかず、一方的に集団の論理で判断を下している場合が往々にしてあるが、注意すべきである。c. リスクの時間依存性リスクは放置すると増大し、かつ時間が経過するにつれて、段階的にその(リスクが顕在化したときの)処理が困難となる。ある担当者が潜在的リスクを認識したけれども、それを上司に言うことをためらっていたりすると、その間にリスクは大きくなり、後になって言い出したときの処理の労力は、初期の段階で言い出したときの数十倍にもなる。上司が部下の報告を受けたときに、リスクのあまりの大きさに処理をためらっても同じことになる。リスクは、認識したら即座に明るみに出して処理をする必要がある。リスクマネジメントはそのためにあるのであって、バーンスタインは「リスクマネジメントは、コントロールできる領域を最大化し、結果の見えない、つまり、コントロールできない領域を最小化することである」などと述べている。d. リスクを許容する社会リスクマネジメントで重要なことは、リスクを許容することである。よく、絶対に打上げ失敗はできない、絶対に成功させる、絶対にミスはあってはならない、あるいは、事故は絶対にあってはならないという言い方を聞くことがあるが、これは、リスクを許容しない社会の言い方であると考える。もちろん、失敗、ミス、事故はあってはならないが、絶対にミスのないことはあり得ない。しかし、絶対にミスがあってはならないという、その集団の文化が担当者の判断を狂わせて、ミスを隠す、ミスを言い出さない、という行動を引き起こす可能性は十分にある。その結果、ミスが発見されるタイミングが遅れて処理に大きな労力を割かなければならなくなったり、ミスが表に出ないまま大きな失敗に至ってしまうといった事態が発生する。リスクを最小にする努力はする(リスクマネジメント)が、残ったリスクは許容する度量が必要であろう。
