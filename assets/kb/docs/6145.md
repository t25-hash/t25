# 17・2・3 確率論的リスク評価

> **章**: 第17章 信頼性とリスクマネジメント
> **節**: 17・2 リスクマネジメント

a. 確率論的リスク評価とは何か確率論的リスク評価(PRA : probabilistic risk assessment)は、リスクマジメントの重要なプロセスである、リスクを識別しその大きさを評価し、リスク軽減の戦略を立案することを確率論的に実施する手法であり、複雑なシステムやミッションのリスク(不確定性)に対してプロジェクトマネージャが実施する意思決定を支援するものである。システムを運用すると、思わぬ故障や運用上のミスに遭遇し、それらが引き金となって重大な事故や失敗に至ることがある。特に複雑なシステムが事故や失敗に至る場合には、事故の引き金となる事象(起因事象)と事故結果の間には分岐となる多くの事象(中間事象)があり、それらの状況(発生・不発生)の組合せにより事故経過の最終結果はシステムやミッションの成功から重大事故・失敗までの幅広い範囲に分岐する。確率論的リスク評価では、一つの起因事象からある最終結果に至る事象の系列をシナリオと呼び、不確定性をもったシナリオの発生確率(頻度)が信頼性データや確率論により定量的に評価され、意思決定の参考資料とされる。ロケット・人工衛星分離姿勢制御系太陽電池パドル展開電力系熱制御系通信系軌道変更主アンテナ展開あるシステム・ミッションに対して確率論的リスク評価を実施すると、システム・ミッションの外乱・故障に対するリスクモデル、すなわち、事故シナリオ、事故の最終結果、事故の発生確率に関する情報を得ることができる。その結果、プロジェクトマネージャは個々の事故シナリオに対してコスト、性能や信頼性・安全の観点から不確定性を考慮した優先順位をつけることができ、より的確な意思決定を行うことができる。言い換えると確率論的リスク評価は、「システムに発生するリスクを評価するために、外乱または故障によるシステムの応答をモデル化またはシミュレートする過程」であると定義でき、リスク評価を実施するということは、対象とするシステムの理解を深める過程であるといえる。b. 確率論的リスク評価の手順確率論的リスク評価の手順例を以下に示す。. 対象と範囲の決定確率論的リスク評価を実施する対象と範囲を決定する。. システムの理解対象システムをよく理解する。システムの構成要素、システム機器・部品の動作、サブシステム構成間の依存性、システム機器・部品の機能、システム機器・部品の故障モードを確認する。. 引き金事象の明確化引き金事象とは、システム外乱または性能・機能の故障である。内部要因としては、機器故障、ヒューマンエラー、ソフトウェアエラー、危険状態があり、外部要因としては、システム外の環境条件(放射線、雷等)がある。. トップレベルシナリオシナリオとはある結論に至る事象の連続である。その開始点が引き金事象であり、その終点が終局状態(エンドステート)である。このシナリオを、イベントツリー、フォールトツリーの組合せ、またはイベントシーケンスダイアグラムによって、どういう経過で起因現象から終局状態になるのか、トップレベルのシナリオを記述する。(1) 終局状態は、ミッションの成功に関与する結果であり、例えば、宇宙機損傷、搭乗員死傷、ミッション失敗、ミッション成功等がある。(2) イベントツリーは、システムにおける事象の時間的なシーケンスを図示するものであり、フォールトツリーと組合わせることによって、複雑で動的な要素を効果的に記述できる。イベントツリーの例を図17・3に示す。. 確率論的リスク評価実施上の留意点確率論的リスク評価は、原子力発電所、化学プラント等で実施しており、その分野での手順が整備されているが、宇宙とは異なる分野であるため、宇宙独自の実施プロセスを検討する必要がある。NASA では、宇宙開発のための確率論的リスク評価実施の手順を定めており、従来検討が不足していた、ソフトウェア、ヒューマンについての信頼性の考察を追加している。確率論的リスク評価を実施するには、支援ソフトウェアの利用が望ましい。現在数種類のアメリカ、ヨーロッパ製の確率論的リスク評価支援ソフトウェアが市販されており、これらを評価し使いやすいものを利用するのがよい。確率論的リスク評価を実施するのに重要なことは、信頼できるデータを準備することである。宇宙開発の場合、機器・部品の生産量が少ないため、少ないサンプル数による信頼性データしか入手できない。そのためデータとしてはハンドブックデータ集に頼っているのが現実である。よって確率論的リスク評価によって、効果の高い意思決定を行うには、精度の高いデータを収集・分析してデータベース化する必要がある。文献(1)
