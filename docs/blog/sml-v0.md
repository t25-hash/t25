# LLMならぬSMLを作ってみた。

> 大規模言語モデル（LLM）ではなく、**SML（Small Language Model＝小規模言語モデル）** を、ブラウザの中だけで・APIなしで作ってみた記録です。題材は Web に転がっている **JIS／機械工学** の知識。コードは GitHub 上にオープン実装しました。

---

## SMLとは

**LLM（Large Language Model）** が「巨大なパラメータ・巨大な計算資源・クラウドAPI」で成り立つのに対し、
ここでいう **SML（Small Language Model）** は、

- **極小**（パラメータは“赤ちゃん級”：埋め込み→tanh隠れ層→softmax の数千〜数万重み）
- **端末内（ブラウザ）で完結**（サーバ・API・外部送信なし）
- **特定ドメインに特化**（今回は機械工学）

という思想のモデルです。
単体の極小ニューラルネットでは流暢な文章は作れません。そこで本物のコーディングエージェント（Claude Code 等）と同じ **「検索 ＋ 重み」のハイブリッド**＝RAG（検索拡張生成）で“賢さ”を補います。
「巨大さ」ではなく「**仕組みの正しさ**」で動かす、いわば *LLM の構造を最小単位で再現した教材* です。

## 方針：GitHub 上にオープン実装

- **バニラ HTML / CSS / JS**（ビルド不要・フレームワーク不要・依存ゼロ）
- **完全オフライン**（API キー不要、外部送信なし、`fetch` するのは自分の静的アセットだけ）
- **GitHub Pages で公開**（スマホ／iPhone でもそのまま動く）
- 題材は **Web 上の JIS・機械工学関連**（機械要素・材料力学・歯車・軸受・熱処理…）を知識ベース化。
  → 事前に作った検索索引：**5,809 文書 / 約 16 万語**のプルーニング済み TF-IDF(BM25) 逆引きインデックス。

## 構築対象（できたもの）

「Ask the baby（赤ちゃんに聞く）」という1枚の SPA に、AIエージェント工学の構成要素を“動く実機”として実装しました。

### ● 赤ちゃん言語モデル（Ask）
本体。質問すると、機械工学KBから根拠を検索し、その文脈を極小ニューラルネットが学習して回答します（後述）。

### ● 周辺ラボ（すべて端末内・決定論的）
- **Neural Lab** — Ask の土台になる極小ニューラルネットを、ブラウザ内で**勾配降下学習**させて中身（loss・重み・softmax分布・埋め込み）を観察
- **Embedding Lab** — トークン分解 → ベクトル化 → 類似度 → クラスタ可視化（PCA）
- **RAG Lab** — チャンク分割・検索・MMR 再ランク
- **Memory Lab** — 短期／長期／意味／エピソード記憶、圧縮（要約）と想起（コサイン）
- **Agent Lab / Multi-Agent Lab** — 観察→思考→行動ループ／役割分担と合意形成
- **Tool Calling Lab / MCP Lab** — ツール選択・実行ログ／MCP の tools/call 表現
- **Claude Code Explorer** — while ループ・権限ゲート・ツール・コンテキストという中核を図解
- **Playground** — プロンプト設計の評価、トークン数（実カウント）とコスト試算
- **Research Lab / PDF抽出** — 端末内 PDF 解析（テキスト抽出・要約・抽出型QA）
- **Doc 生成** — プロンプトから Excel / Word / CSV / Markdown を生成（実ファイル）
- **Academy / Evaluation / Build / AI Coding** — 理論学習・指標・周辺

## Ask 構築ロジック

ユーザーの1問が、回答になるまでの流れ：

1. **ユーザーからの質問**（例：「軸受の種類と特徴は？」）
2. **検索（Retrieval）** — BM25 索引で機械工学KB 5,809 文書から関連文書 top-k を取得
3. **チャンク分割＋再ランク** — 取得文書を断片化し、TF-IDF コサインで質問に近い箇所を選別
4. **抽出最適化（前処理）** — 原典は**二段組 PDF 抽出**で文が崩れている。行交互の段組みを**デインターリーブ**し、数式行をスキップ、**言語モデル尤度**で「元順 vs 分離後」のどちらが流暢かを判定して、崩れた文を正しい全文に復元
5. **重みの学習（Weights）** — 取得文脈だけを材料に、極小ニューラルネット（埋め込み→tanh→softmax）を**端末内で数千ステップ学習**
6. **回答生成** — 根拠の“実文”を、**固有語ゲート**＋**関連性フロア**で選び、学習済みネットの確信度（系列対数尤度）で**再ランク**。約100字の自然な回答にまとめる
   - 実文ベースなので**文法が崩れない／事実を捏造しない**
   - 十分一致する記述が無ければ、誤答せず**「該当なし」**と答える
7. **メモリ要約** — 取得文脈を“作業メモリ”とみなし、関連文だけを抽出要約して回答の下に併記

> ポイントは **「生成」ではなく「検索した実文の選別＋再ランク」** にしたこと。極小モデルの自由生成では文法破綻を避けられないため、**赤ちゃんでも“自然で正しい”回答**を出すための現実解です。

### 工夫したところ（ハマりどころ）
- **語義の曖昧性**：表層の語一致だけだと「殻理論の**基礎**」→「ポンプの**基礎**(土台)」のように別語義へ誘導される。→ 汎用語を除いた**固有の主題語**の一致を必須化して抑制。
- **二段組 PDF の文崩れ**：左右カラムが1行ずつ交互に出力され、そのまま読むと文が割れる。→ 偶奇分離＋**言語モデルで流暢な並びを採用**。原典側の崩れを再構成で解消。

## 動作状況

- ✅ **完全オフラインで動作**（API 不要・外部送信なし）。PC／スマホ（iPhone）でブラウザを開くだけ。
- ✅ **GitHub Pages で公開**。リロード後も最新が反映。
- 🧪 **実証フェーズ**：KB 由来のランダム 30 問 × 複数バッチで回答品質を評価。
  - 形式面：ほぼ整文・約100字・文末まで自然に終止。
  - 内容面：固有語ゲート導入で**語義ハイジャック型の誤答を抑制**、抽出最適化で**二段組由来の文字崩れを大幅減**。
  - 弱い一致は誤答せず「該当なし」を返す方針（正直さ優先）。
- ⚠ **残課題**：カラム行数が極端に不均衡な稀な版面・数式主体ページ・検索再現率の限界。→ 次バージョンの宿題。

これを **v0（一旦完成）** とします。
